## Summary

When editors embed images in CKEditor content on a Drupal CMS site, they previously saw a long dropdown of view mode options that included a "large" size variant for every aspect ratio (16:9 large, 1:1 square large, 3:4 large, 4:3 large, 9:16 large, and a plain "large"). After a prior release widened the content column to 720px, "large" and "medium" rendered at nearly the same visual size, making the "large" options redundant and confusing. This change removes all six "large" view modes from the CKEditor media embed picker, leaving only medium and small variants per aspect ratio. At the same time, new 720px-wide image styles are introduced to ensure the "medium" responsive images actually serve a full-width candidate at the wider content column.

## Impact

Affects site owners.

- **Config change:** Six 'large' media view modes removed from `filter.format.content_format` allowed_view_modes; five new 720px-wide image styles added and wired into responsive image 'medium' breakpoint mappings; oEmbed video max dimensions updated to 720×405.

## Technical details

The `filter.format.content_format` text format's `media_embed` filter had its `allowed_view_modes` trimmed. Six keys were removed: `16_9_large`, `1_1_square_large`, `3_4_large`, `4_3_large`, `9_16_large`, and `large`. The format's config dependencies were updated to match.

Five new image styles were introduced at 720px width: `16_9_720x405_focal_point_webp`, `1_1_720x720_focal_point_webp`, `3_2_720x480_focal_point_webp`, `4_3_720x540_focal_point_webp`, and `uncropped_720w_webp`. Each uses a `focal_point_scale_and_crop` (or `image_scale` for uncropped) effect followed by `image_convert` to WebP. These styles were then added as the top candidate in the `sizes_image_styles` mapping of the corresponding responsive image style "medium" variants (`16_9_medium`, `1_1_medium`, `3_2_medium`, `4_3_medium`, `uncropped_medium`), so browsers serving the now-wider content column receive a correctly sized source.

The SVG image display width was updated from 600 to 720, and the remote video oEmbed formatter `max_width`/`max_height` was bumped from 608×342 to 720×405, both aligning with the widened content column established in a prior release.
