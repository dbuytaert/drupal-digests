# #3476802: User Experience (UX) Review of the SEO Dashboard

## Summary

The SEO Dashboard, added to Drupal CMS for DrupalCon Barcelona, has been removed from the Advanced SEO Recipe (`drupal_cms_seo_tools`). A UX review concluded that the dashboard delivered a list of links rather than meaningful actions, and that most of its content is already accessible under the "Search and metadata" section of the admin navigation. Site owners who had this recipe applied will no longer see a dedicated SEO tab in the Dashboard, and the shortcut to `/admin/dashboard/seo` is gone.

## Impact

Affects front-end developers.

- **Theme change:** The SEO Dashboard tab at `/admin/dashboard/seo` has been removed; any navigation or shortcuts pointing to that path will break.

## Technical details

The removal was implemented in commit `3322115` (issue [#3488285](https://www.drupal.org/node/3488285)). Three things were deleted from the `drupal_cms_seo_tools` recipe: the dashboard configuration file (`dashboard.dashboard.seo.yml`), the shortcut config entity (`shortcut.shortcut.seo_dashboard.yml`), and the `dashboard` module from the recipe's `install` list. The `drupal/dashboard: ^2` Composer dependency was also dropped from the recipe's `composer.json`. The prerequisite recipe `core/recipes/full_html_format_editor`, which had been included solely to support the dashboard's text blocks, was likewise removed. The recipe description was updated to drop the reference to the dashboard.

### Collaboration

pameeela provided a detailed annotated screenshot and written analysis of the dashboard's weaknesses: the primary CTA for adding URL aliases was rarely needed, "Fix broken links" was misleading because the target was a 404 report rather than a link checker, and the external resources section did not justify a dedicated dashboard. The conclusion in her comment was "what if we move the common 'jobs to be done' links into the main dashboard?" This analysis directly motivated the decision to remove the dashboard rather than revise it.
