## Summary

A new "Feature" Single Directory Component (SDC) has been added to the `drupal_cms_olivero` theme for use with Experience Builder (XB). The component renders a visually prominent featured section with an optional image, a required title, an optional summary, and an optional call-to-action link. The image can be positioned left or right, producing a flipped two-column layout on wider viewports. This component is marked `experimental` and is intended primarily for the XB preview experience in Drupal CMS 1.x.

## Impact

Affects module maintainers.

## Technical details

The component lives under `drupal_cms_olivero` and consists of three files: `component.yml` (metadata and prop schema), `feature.css` (responsive styles), and `feature.html.twig` (template). Defined props are: `title` (string, required), `summary` (string, uses `$ref: json-schema-definitions://experience_builder.module/textarea`), `link_text` (string, defaults to "Learn more"), `link_url` (string, URI format), `image` (object, uses `$ref: json-schema-definitions://experience_builder.module/image`), and `image_position` (enum `left`/`right`, defaults to `left`).

The Twig template applies modifier classes (`feature--has-no-image`, `feature--flipped`) via `attributes.addClass(classes)`, using SDC's standard `attributes` object rather than a custom prop. The image renders conditionally: when absent, the modifier class triggers alternate padding rules and the decorative colour bar moves to the wrapper instead. The `link_url` is also rendered conditionally to suppress an empty anchor element.

CSS uses container queries (breakpoint at `600px` inline-size) to switch from a stacked single-column layout to a flex row. The image is constrained to a `4:3` aspect ratio with `object-fit: cover`. A short coloured accent bar is placed at the top of the image wrapper (or, when no image is present, at the top of the outer container) using a `::before` pseudo-element; its inline start/end offsets are reversed in the flipped variant.

The prop naming went through several iterations during review. The final prop names (`summary`, `link_text`, `link_url`) were chosen over earlier candidates (`tagline`, `cta`, `ctahref`) to be more user-friendly and consistent with naming being adopted across other Olivero XB components. Tags were dropped from scope because XB cannot yet handle them programmatically. The `image` field was kept as a prop rather than a slot after discussion: Experience Builder does not yet support restricting which components can be placed in a slot or limiting slot count, making a prop cleaner for the demo context.

### Collaboration

The design debate over slots versus props was the central tension in this thread. `pdureau` argued strongly that the `image` field should be a slot following SDC best practices, and that using the XB-specific `$ref` for the image prop unnecessarily couples the component to Experience Builder. `balintbrews` pushed back, noting that XB cannot yet constrain slot contents or counts, and that a simple image prop would present a cleaner demo experience. `boulaffasae`, who implemented the component, initially switched to a slot to explore it, then reverted to a prop at `balintbrews`'s recommendation after finding that the slot-based approach still required a separate "Include image" boolean to handle the no-image variant. `kristen pol` drove the final prop naming iteration: replacing `tagline` with `summary` and `cta`/`ctahref` with `link_text`/`link_url` to be consistent with naming conventions being established across other XB components in Drupal CMS. `mherchel` pushed a style update to align the implementation with the designer's specifications (4:3 image ratio, centred text, flipped layout variant).
