## Summary

Drupal CMS's home page, which uses a basic page content type, received custom styling to set it apart visually from other basic pages. The key change is that the page title is kept in the DOM as an accessible `h1` but hidden from sighted users, ensuring screen reader compatibility while matching the design intent of a title-free hero layout. The Drupal CMS Olivero sub-theme (`drupal_cms_olivero`) handles this with a PHP preprocess hook and a small CSS utility file.

## Impact

Affects front-end developers.

- **Theme change:** The drupal_cms_olivero sub-theme now visually hides the h1 page title on the front page via a preprocess hook and a new utility.css class rather than removing the block from the DOM.

## Technical details

The original approach (issue [#3491494](https://www.drupal.org/node/3491494), December 2024) used a `request_path` block visibility condition on `block.block.drupal_cms_olivero_page_title` to suppress the page title block entirely on the front page. This left the home page without any `h1` element, which was an accessibility problem reported as issue [#3500408](https://www.drupal.org/node/3500408) and fixed in the 1.0.1 release (MR !462).

The fix removes the `request_path` visibility config so the page title block renders everywhere. In `drupal_cms_olivero_preprocess_block()` in `drupal_cms_olivero.theme`, a call to `\Drupal::service('path.matcher')->isFrontPage()` checks whether the current page is the front page; if so, and the block plugin is `page_title_block`, the CSS class `visually-hidden` is appended to the block's attributes. A new `css/base/utility.css` file defines that class as `width: 1px !important`, with the `!important` required to override Olivero's own specificity on visually-hidden elements. That stylesheet is added to the `global-styling` library in `drupal_cms_olivero.libraries.yml`.
