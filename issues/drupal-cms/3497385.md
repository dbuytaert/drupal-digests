## Summary

Drupal CMS's Olivero theme gains two new experimental Single Directory Components (SDCs) for Experience Builder: a `Card` component for individual content cards and a `Card grid` container that arranges them responsively. At the same time, the old `.teaser--card` CSS and its dedicated `card.css` stylesheet were removed, and `node--teaser.html.twig` was refactored to use the new card SDC's markup and CSS classes. Site owners using Experience Builder can now compose card-based content areas in the visual editor; developers and themers who previously relied on `.teaser--card` or `.teaser__*` selectors need to update to the new `.card` / `.card__*` classes.

## Impact

Affects module maintainers and front-end developers.

- **Theme change:** `.teaser--card` and `.teaser__*` CSS classes replaced by `.card` and `.card__*` from the new Card SDC; `node--teaser.html.twig` markup structure changed significantly.
- **New API:** Two new experimental SDC components `drupal_cms_olivero--card` and `drupal_cms_olivero--card-grid` added for Experience Builder use.

## Technical details

The `Card` SDC (`drupal_cms_olivero/components/card/`) defines props for `image`, `pretitle`, `title`, `subtitle`, `summary`, and `link_url`. When `link_url` is set, an `::before` pseudo-element on `.card__title a` makes the entire card clickable; `pointer-events: none` is applied to both `.card__image` and `.card__cta-icon` so the overlay does not block click events on the decorative arrow. The card uses CSS container queries at 600 px to switch from a stacked to a side-by-side image-plus-content layout, and a `card--no-image` modifier adjusts padding when no image is provided.

The `Card grid` SDC (`drupal_cms_olivero/components/card-grid/`) wraps a `cards` slot in a `.card-grid` div. Its CSS uses CSS `:has()` selectors to automatically set `--column-count` to 1, 2, or 3 depending on the number of child cards, including special handling for XB's `.xb--sortable-list` wrapper via `grid-template-columns: subgrid`.

`drupal_cms_olivero.libraries.yml` drops `css/components/card.css: {}` from the `views` library and adds `core/components.drupal_cms_olivero--card` as a library dependency instead. The old `css/components/card.css` file (162 lines of `.teaser--card` and `.teaser__*` rules) is deleted entirely. `node--teaser.html.twig` is refactored to use `card` and `card__*` classes and calls `attach_library('core/components.drupal_cms_olivero--card')` directly; a `@todo` comment notes that the markup is partially duplicated from the SDC template because XB cannot yet pass image data through slots.

## Upgrade

Themers who overrode `.teaser--card` or `.teaser__*` CSS selectors in a custom sub-theme of `drupal_cms_olivero` should migrate those rules to the new class names:

```css
/* Old selectors (removed) */
.teaser--card { ... }
.teaser--card .teaser__image { ... }
.teaser--card .teaser__content { ... }
.teaser--card .teaser__title { ... }

/* New selectors */
.card { ... }
.card__image { ... }
.card__content { ... }
.card__title { ... }
```

Themers who overrode `node--teaser.html.twig` should review the updated template: the outer element now carries `card` and (conditionally) `card--no-image` classes, and the inner structure uses `card__inner`, `card__image`, `card__content`, `card__header`, `card__pretitle`, `card__title`, `card__subtitle`, `card__lower`, and `card__cta-icon` elements instead of the previous `teaser__*` hierarchy.
