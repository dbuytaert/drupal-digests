## Summary

The site template selection step in the Drupal CMS installer now renders each template option as a styled card rather than a plain radio button. Each card displays a screenshot thumbnail, the template name, a description, and a "More Info" link. The `drupal_cms_starter` template is selected by default and listed first. This matches the Figma design that was referenced in the issue.

## Impact

Affects front-end developers.

- **Theme change:** New `form-element--site-template.html.twig` Twig template added to `drupal_cms_installer_theme` for rendering site template radio button cards with screenshot, name, description, and homepage link.

## Technical details

A new `drupal_cms_installer_theme_preprocess_form_element()` function in `drupal_cms_installer_theme.theme` handles the `form_element__site_template` theme hook. It reads a `screenshot.webp` file from the recipe directory using `Hooks::getRecipePath()` and embeds it as a `data:image/webp;base64,...` URI, bypassing the limitation that recipes are stored outside the web root and cannot be served as static files by the web server. It also reads the `homepage` key from the recipe's `composer.json` to populate the "More Info" link. A new Twig template `form-element--site-template.html.twig` replaces the default form element markup for these radio buttons, rendering the screenshot in an `<img>` tag, an `<h3>` for the name, the description block, and a conditional anchor for the homepage link.

CSS in `add-ons.css` was updated to match: labels switch from a fixed colored border to a transparent 4px border that fills with `var(--gin-color-primary)` when selected, images inside labels receive rounded corners and a light `#D8DADE` border, and `.cms-installer__form-group` wraps the radio group in a responsive CSS grid (up to three columns at viewport widths of 600px and above). The selected-state checkmark indicator now uses `var(--gin-color-primary)` as its background instead of black.

The old `drupal_cms_installer_theme_form_installer_recipes_form_alter()` hook implementation, which handled a different installer step, was removed from the theme file. The site template form alter now also sets `#default_value` to `drupal_cms_starter` and applies a `#weight` of `-100` to ensure it appears first.

The implementation involved real back-and-forth between designer pameeela and developer phenaproxima. The core difficulty was that `form-element-label.html.twig` had no theme hook suggestion specific to site templates, so the custom label styling bled into unrelated form fields on other installer steps. bernardm28 identified the problem after their first commit brought the layout close to the Figma design. phenaproxima proposed and implemented the `form_element__site_template` theme hook suggestion from the backend, which gave pameeela and bernardm28 the scoped template they needed to style only the template selection cards.
