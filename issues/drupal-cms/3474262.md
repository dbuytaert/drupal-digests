## Summary

Drupal CMS sites now display a Dashboard link with an icon in the navigation bar. Previously the Dashboard module's block was not allowed inside the Navigation module's hardcoded list of permitted blocks, so no dashboard entry appeared in the nav. A temporary composer patch wires the two modules together while the upstream fix for this limitation makes its way through the Drupal core queue.

## Technical details

The `drupal/navigation` module maintains a hardcoded allowlist of block plugin implementations it will render; the Dashboard block was not on that list. Drupal core issue [#3443833](https://www.drupal.org/node/3443833) ("Provide a way for other modules to flag block plugin implementations as 'navigation safe'") is the intended long-term solution, but it was not yet resolved at the time of this commit. As a stopgap, a patch from the Dashboard contrib issue [#3395527](https://www.drupal.org/node/3395527) was added to the `extra.patches` section of the project's `composer.json` under the `drupal/navigation` key, pointing to `navigation-dashboard-integration-d10.3_0.patch`. This is applied at install time via `cweagans/composer-patches` and will be removed once the upstream core API lands.
