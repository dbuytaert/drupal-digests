## Summary

Event card views in Drupal CMS were serving the original, unprocessed featured image instead of the optimized responsive image style used by every other content type. On a page with multiple event cards this could mean several megabytes of unnecessary image data per page load, invisible to visitors on fast connections but a real burden on slower ones. The fix changes the event card display to use the same `16:9 Medium` responsive image style already applied to blog, news, page, person, project, and case study content types.

## Impact

No upgrade or configuration changes required.

## Technical details

The `drupal_cms_events` recipe's card view-mode display config (`node.event.card.yml`) had `field_featured_image` configured with the `media_thumbnail` formatter and no image style set, causing Drupal to fall back to the raw source file. The fix replaces the formatter with `entity_reference_entity_view` using `view_mode: 16_9_medium`, exactly matching the configuration already present in all other Drupal CMS content type recipes. Along with the config change, `ComponentValidationTest` functional tests were added (or extended) for all seven content types: blog, case_study, events, news, page, person, and project. Each now asserts that `field_featured_image` in the card display uses the `entity_reference_entity_view` formatter, preventing this regression from recurring. The `dms/phpunit-arraysubset-asserts` library was added as a dev dependency to provide the `assertArraySubset` assertion used by these tests.

## Upgrade

Sites that applied the `drupal_cms_events` recipe before this fix will not receive the config update automatically. To correct the display:

1. Go to `/admin/structure/types/manage/event/display/card`
   (Structure → Content Types → Event → Manage display → Card)
2. Change **Featured Image** from **Thumbnail** to **Rendered entity**.
3. Open the settings gear and select **16:9 Medium** as the view mode.
4. Save the display.
