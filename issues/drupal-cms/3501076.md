## Summary

The Drupal CMS search recipe previously required `search_api_autocomplete`, which loaded approximately 280kb of JavaScript for anonymous users. This alone caused the Lighthouse performance score to drop from 95 to 86 after applying the search recipe. The proposed change removes `search_api_autocomplete` from the recipe entirely, eliminating that JavaScript payload and the associated Lighthouse regression. Autocomplete may be reintroduced later if a low-cost approach is found (see [#3500964](https://www.drupal.org/node/3500964) for the upstream performance work).

## Impact

No upgrade or configuration changes required.

## Technical details

The diff removes every trace of `search_api_autocomplete` from the `drupal_cms_search` recipe. Concretely: the `search_api_autocomplete.search.content_autocomplete` config entity file is deleted; `drupal/search_api_autocomplete` is dropped from `composer.json`; `search_api_autocomplete` is removed from the recipe's module install list; the `simple_search_form` block config for both the desktop and mobile form instances has its `input_type: search_api_autocomplete` setting and the entire nested `search_api_autocomplete` config block removed; and the `grantPermission: 'use search_api_autocomplete for content_autocomplete'` actions for both `user.role.anonymous` and `user.role.authenticated` are removed.

The maintainers noted that removing the module also requires cleaning up any autocomplete-specific theming in `drupal_cms_olivero` (tracked in [#3501069](https://www.drupal.org/node/3501069)), because the search form falls back to a broken state without additional theme adjustments. Those theme changes are considered out of scope for this MR and would be addressed separately.
