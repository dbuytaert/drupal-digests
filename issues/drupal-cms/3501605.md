## Summary

The `launch-drupal-cms.sh` convenience script has been removed from Drupal CMS 1.1.0 and later. The `README.md` now lists the equivalent manual DDEV commands to spin up a local site, and adds a note that any Drupal-compatible hosting stack will work, linking to the Drupal User Guide for those who do not want to use DDEV. On the 1.0.x branch the script itself is retained but the README is updated to use the manual commands.

## Impact

No upgrade or configuration changes required.

## Technical details

The `launch-drupal-cms.sh` bash script (in the Drupal CMS project template root) wrapped `ddev config`, `ddev start`, `ddev composer install`, and `ddev launch` with guards for duplicate project names and DDEV version constraints. Its removal means those four commands are now listed verbatim in `README.md`. The follow-up commit (`8e424cca`) deleted `LaunchScriptTest` in `drupal_cms_installer`, which had been the script's only unit test coverage.

A key technical argument raised in the thread by catch: because the script lived in the Composer project template, it could never be updated on sites that had already been created from the template. Dead, un-updateable code sitting in every project directory was an additional reason to remove it rather than fix it.

## Upgrade

If you were using the launcher to start your local Drupal CMS environment, replace it with:

```shell
ddev config --project-type=drupal11 --docroot=web
ddev start
ddev composer install
ddev launch
```

DDEV is not required; any stack that satisfies Drupal core's system requirements will work. See the [Drupal User Guide installation chapter](https://www.drupal.org/docs/user_guide/en/installation-chapter.html) for alternatives.
