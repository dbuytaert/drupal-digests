## Summary

All image styles shipped with Drupal CMS recipes now convert images to AVIF with WebP as a fallback for browsers that do not support AVIF. AVIF typically produces smaller file sizes than WebP, improving page load performance for visitors. This change applies automatically to new sites installed from the Drupal CMS recipes; sites that were already installed would need to update their image style configurations manually.

## Impact

No upgrade or configuration changes required.

## Technical details

The change is a bulk update across every `image.style.*.yml` file in the Drupal CMS `recipes/` directory. Each image style previously applied the `image_convert` effect (converting to WebP); the effect id is replaced with `image_convert_avif`, which is the AVIF-with-fallback effect introduced in Drupal core [#3510582](https://www.drupal.org/node/3510582) (requiring core 11.2 or later). The `data.extension: webp` value retained in each YAML file now acts as the fallback format, not the primary target. The `ComponentValidationTest` was updated so that all CMS-controlled image styles are asserted to produce `avif` derivatives, excluding third-party styles (`canvas_*`, `linkit_result_thumbnail`). The `ContentMetaTagsTest` was also updated to expect `og:image:type` to be `image/avif` instead of `image/webp`. The commit covers approximately 57 image style YAML files in total.

## Upgrade

Existing Drupal CMS sites that want to adopt AVIF conversion must update each image style configuration manually. The effect `id` field in each `image.style.*.yml` needs to change:

```yaml
# Before
id: image_convert

# After
id: image_convert_avif
```

The `data.extension` value (`webp`) should remain unchanged; it becomes the WebP fallback. Alternatively, use this shell command to batch-update image style exports in a `recipes/` directory:

```bash
find recipes/ -name "image.style.*.yml" -exec sed -i 's/^ *id: image_convert$/    id: image_convert_avif/' {} +
```

This requires Drupal core 11.2 or later.
