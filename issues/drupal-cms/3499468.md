## Summary

The "Recent content" panel on the Drupal CMS admin dashboard now shows a "Type" column that displays each content item's content type. Previously, editors had to open individual nodes to determine their type. As part of the same change, the "Author" column that had been added to this view was removed in favour of the more useful "Type" column, and column header labels for "Title" and "Updated" that were previously blank are now explicitly set.

## Impact

No upgrade or configuration changes required.

## Technical details

The change modifies two Views config files in the `drupal_cms` project. In the `recent_content` node-based view, the `uid` field (labelled "Author") is replaced wholesale with the `type` field (labelled "Type"). At the same time, the previously label-less "Title" and "Updated" (changed date) columns receive explicit `label` values of `Title` and `Updated` respectively, and the entity-reference link on the author field is disabled (`link: false`). In a separate canvas-page view, the `owner` (Author) field block is removed entirely. No PHP, no hook, no service change: the fix is purely configuration YAML.
