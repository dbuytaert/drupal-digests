## Summary

A new `drupal_cms_locations` recipe is added to the Drupal CMS repository, providing a Location content type with structured address, telephone, and automatic geocoding fields, plus paginated list and interactive Leaflet map views. The existing `drupal_cms_events` recipe is updated to depend on this new recipe and gains a `field_where` entity reference field that links events to location nodes, so event editors can choose an existing location from an autocomplete widget.

## Technical details

The `drupal_cms_locations` recipe installs `address` (structured address input), `geofield` and `geocoder`/`geocoder_address`/`geocoder_geofield` (auto-geocoding via Nominatim/OpenStreetMap), and `leaflet_views` for map rendering. The `location` node bundle carries four fields: `field_location_address` (required, hides personal name and organization sub-fields), `field_location_telephone`, `field_geofield` (auto-populated from the address on save via a `geocoder_field` third-party setting on the field instance), and a standard body field. Pathauto is configured with the pattern `locations/[node:title]`, and a metatag record sets `og_type: location`.

Two Views are configured: `locations` (a standard paginated node-teaser list at `/locations` with an "Add a Location" header button via `add_content_by_bundle`), and `locations_map` (a Leaflet map style view at the same path, using OSM Mapnik tiles, with an attachment display rendering location teasers below the map). The `drupal_cms_events` recipe is updated to list `drupal_cms_locations` in its `recipes:` key and in `composer.json`'s `require` block. It adds `field_where` as an `entity_reference` (cardinality 1) targeting the `location` bundle, with an autocomplete edit widget and `entity_reference_entity_view` (teaser mode) plus `entity_reference_label` (inline link) display formatters on the full and teaser view modes respectively.

The root `composer.json` adds a `conflict` entry for `drupal/leaflet < 10.2.24` and applies a patch for issue [#3469226](https://www.drupal.org/node/3469226) (removing an unnecessary `weight` key from Leaflet settings).
