## Summary

Two Drupal CMS recipes that were previously invisible in the Project Browser UI, `drupal_cms_ai` and `drupal_cms_google_analytics`, are now listed there and can be applied directly without using the command line. At the same time, Project Browser is updated to alpha7 and the `max_selections` setting is capped at 1, meaning only one recipe can be queued for installation at a time. Several dependency version constraints were also tightened: AI-related packages move from specific beta pins to stable `^1` ranges, `drupal/scheduler` moves to `^2.2`, and `drupal/webform` to `^6.3-beta1`. The project template's minimum stability is raised from `alpha` to `beta`.

## Impact

Affects module maintainers and site owners.

- **Config change:** `project_browser.admin_settings` gains `max_selections: 1`, limiting Project Browser to one recipe selection per install action.
- **New API:** Two new recipes, `drupal_cms_ai` and `drupal_cms_google_analytics`, are now listed in the Project Browser UI and can be applied without CLI access.

## Technical details

In `drupal_cms_starter`'s recipe YAML, the `project_browser.admin_settings` config is updated via `simpleConfigUpdate` to add `drupal_cms_ai` and `drupal_cms_google_analytics` to `allowed_projects.recipes` (the list is also re-sorted alphabetically, placing `drupal_cms_accessibility_tools` first). A new `max_selections: 1` key is added to the same config block, restricting the Project Browser UI to one recipe selection per install action.

The `drupal/project_browser` constraint in `drupal_cms_starter/composer.json` advances from `^2-alpha6` to `^2-alpha7`, and `drupal/project_browser` is added as a direct dependency of the project template itself with `@alpha` stability. The project template's `minimum-stability` setting in the release-tagging script moves from `alpha` to `beta`.

In `drupal_cms_ai`'s `composer.json`, five AI package constraints are relaxed from their former specific beta pins (`^1-beta7`, `^1-beta1`, etc.) to broad stable `^1` ranges, reflecting that those upstream packages have now reached a stable 1.x release. Two trailing commas in the AI setup recipe's YAML input configuration for `#maxlength` are also removed to fix YAML validity.
