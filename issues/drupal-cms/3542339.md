# #3542339: Make the blank site option usable

## Summary

When choosing the blank site option in Drupal CMS (via the `drupal_cms_starter` recipe), a site previously installed with only a 404 page and no real starting point. This has been fixed: the recipe now provisions a proper "Home" canvas page at `/home` as the front page, along with configured header and footer regions using the Mercury theme's Navbar and Footer components.

## Impact

Affects module maintainers.

- **New API:** New Canvas component configurations for `sdc.mercury.button`, `sdc.mercury.footer`, `sdc.mercury.hero-billboard`, and `sdc.mercury.navbar` are now shipped by the `drupal_cms_starter` recipe

## Technical details

The fix (landed in [#3558958](https://www.drupal.org/node/3558958), MR !688) adds several things to the `drupal_cms_starter` recipe. New Canvas component configuration YAML files register four Mercury theme SDC components: `sdc.mercury.button`, `sdc.mercury.footer`, `sdc.mercury.hero-billboard`, and `sdc.mercury.navbar`. Two new Canvas layout configs, `mercury.header` and `mercury.footer`, wire up the Mercury Navbar (with site branding block and main navigation menu) and Mercury Footer (with footer menu block) as the default header and footer regions.

The old default content, a `canvas_page` entity titled "Page not found" at `/404`, is replaced by a new `canvas_page` entity titled "Home" at `/home`. The Home page uses a `sdc.mercury.hero-billboard` layout component as its root, with a Heading ("Your content goes here"), a Text block ("Bring your site to life, starting here."), and a Button ("Call to action") nested inside its `hero_slot`. The recipe action that previously set `system.site` `page.404` to `/404` is replaced by one that sets `page.front` to `/home`, making the new page the site's front page.
