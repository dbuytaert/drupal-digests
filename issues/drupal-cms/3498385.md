## Summary

The AI assistant bundled with Drupal CMS now follows a two-phase interaction flow. Before doing anything, it presents a "Preview Step": a plain-language summary of what it plans to do, a detailed bullet-point breakdown, and an explicit "shall I proceed?" prompt. After acting, it follows with a "Recap Step": a short summary of what was completed, with relative links to every item it created or changed. The previous behaviour asked users only once (step 4 of a flat 6-step list) whether to proceed, with no structured before/after distinction. The assistant's opening greeting was also updated to describe its capabilities more concisely.

## Impact

No upgrade or configuration changes required.

## Technical details

All changes land in a single YAML configuration file that defines the AI assistant's `system_prompt`, `instructions`, and per-agent `usage_instructions`. The `instructions` block is restructured with explicit section headings ("Role", "Steps you should take", "Target Audience and Tone", "Formatting Rules") and replaces the former 6-step numbered list with two named phases: "Preview Step" and "Recap Step". The `system_prompt` block is updated to remove the old HTML-output directive and replace it with a Markdown-only rule, add examples of correct relative link formatting (must start with `/`), and clarify how results returned from sub-agents should be presented. The `field_type_agent` `usage_instructions` gains two new guardrails: field groups (details, tabs, fieldsets) cannot be managed by the agent and users must be directed to do this manually; taxonomy fields should default to a select list widget. The chatbot `first_message` is rewritten from a generic greeting to a sentence that describes the assistant's dual capability (instructions and direct actions).
