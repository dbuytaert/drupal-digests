# #3488359: Support retina devices and improve responsive image styles on mobile

## Summary

Drupal CMS's responsive image system was previously tied to Olivero's breakpoints, which lack a sub-500px breakpoint and only define 1x screen resolutions. This caused the fallback image to be underutilised on small phones and images to appear blurry on high-density (retina) screens. The fix introduces a dedicated `custom` breakpoint group for Drupal CMS, decoupled from any theme, and updates all responsive image styles to use it. Sites using the `drupal_cms_site_template_base` recipe now serve appropriately sized images across a wider range of viewport widths and device pixel ratios.

## Impact

No upgrade or configuration changes required.

## Technical details

The breakpoints are defined at runtime, not via a static `.breakpoints.yml` file, using an ECA model at `eca.eca.define_breakpoints.yml` in the `drupal_cms_site_template_base` recipe. The model listens to the `eca_render:breakpoints_alter` event and fires four sequential `eca_render_alter_breakpoint` actions to register `custom.sm` (min-width: 500px), `custom.md` (min-width: 700px), `custom.lg` (min-width: 1000px), and `custom.xl` (min-width: 1300px) under the `custom` breakpoint group. All responsive image styles in `drupal_cms_site_template_base/config/` declare `breakpoint_group: custom` and use `image_mapping_type: sizes` rather than explicit 1x/2x multiplier mappings. The `sizes` approach emits a `srcset` with multiple candidate widths and a `sizes` attribute; browsers then pick the best candidate based on viewport width and device pixel ratio natively, covering retina screens without needing separate 1.5x or 2x image style entries. Viewports below 500px are handled by the `fallback_image_style` property defined on each responsive image style. The chosen approach (option 4 from the issue) keeps the breakpoints fully independent of the active front-end theme.
