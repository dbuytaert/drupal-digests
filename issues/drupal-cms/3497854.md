## Summary

Drupal CMS now surfaces the `drupal_cms_ai`, `drupal_cms_google_analytics`, and `drupal_cms_accessibility_tools` recipes inside the Project Browser UI. Previously the AI recipe was intentionally hidden because the guided API-key setup flow was not yet ready; it could only be applied at the command line. With the setup UI now in place and AI modules reaching stable releases, all three recipes were added to the Project Browser allowed list alongside the existing content-type and tooling recipes. Project Browser itself was updated to alpha7, and a `max_selections: 1` limit was set so users choose one recipe per session.

## Impact

Affects module maintainers.

- **New API:** Three new recipes (`drupal_cms_ai`, `drupal_cms_google_analytics`, `drupal_cms_accessibility_tools`) are now selectable in the Project Browser UI on Drupal CMS sites.

## Technical details

The change is in `drupal_cms_starter`'s recipe YAML, specifically the `project_browser.admin_settings` config block. The `allowed_projects.recipes` list was expanded to include `drupal_cms_ai`, `drupal_cms_google_analytics`, and `drupal_cms_accessibility_tools`, and sorted alphabetically. A new `max_selections: 1` key was added to the same config block. The `drupal/project_browser` dependency constraint in `drupal_cms_starter`'s `composer.json` moved from `^2-alpha6` to `^2-alpha7`. Separately, the AI module version constraints in `drupal_cms_ai`'s `composer.json` were promoted from beta pins (`^1-beta7`, `^1-beta1`, etc.) to stable `^1` ranges for `drupal/ai`, `drupal/ai_agents`, `drupal/ai_image_alt_text`, `drupal/ai_provider_anthropic`, and `drupal/ai_provider_openai`. Two YAML trailing-comma syntax fixes in the AI recipe's input configuration and a scheduler constraint bump (`^2` to `^2.2`) were also included. The project template's `minimum-stability` was raised from `alpha` to `beta`.

### Collaboration

The merge request sparked a substantive debate about readiness. poker10 raised concerns about exposing the AI recipe given quality signals: several AI modules had fewer than 100 installs and lacked security advisory coverage, and early testing revealed hallucination, broken links, and Guzzle timeout issues. pameeela explained the recipe had been hidden specifically because the guided API-key setup UI was not ready, not because of quality concerns, and that the AI team had conducted end-user testing in coordination with UX; the AI modules were tagging stable releases before the merge. The discussion clarified the deliberate product decision to proceed, with the practical constraint that users still need a paid API key from OpenAI or Anthropic to get anything working.
