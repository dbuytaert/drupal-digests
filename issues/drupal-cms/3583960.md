## Summary

The `drupal_cms_anti_spam` and `drupal_cms_starter` recipes now install the [ALTCHA](https://www.drupal.org/project/altcha) module instead of [FriendlyCaptcha](https://www.drupal.org/project/friendlycaptcha). The switch was triggered by FriendlyCaptcha dropping local-only (no external calls) verification support in its 2.x branch, which is the feature Drupal CMS requires. ALTCHA's default mode is fully self-hosted with no third-party calls, keeps the CAPTCHA secret in Drupal's State API (never exported to config), and ships its JavaScript widget as a bundled Web Component that requires no manual library installation. No upgrade path is provided for existing sites; site owners already using the anti-spam recipe will need to manually uninstall FriendlyCaptcha and install ALTCHA.

## Impact

Affects site owners.

- **Config change:** New `altcha.settings` config file added to the recipe; `friendlycaptcha.settings` config action and the `site_uuid` recipe input removed.

## Technical details

The `drupal_cms_anti_spam` recipe removes `friendlycaptcha` from its `install` list and adds `altcha`, updates `captcha.settings` to set `default_challenge: altcha/ALTCHA`, and removes both the `friendlycaptcha.settings` config action and the `input` block that previously injected a `site_uuid` to generate placeholder API keys for FriendlyCaptcha's local endpoint. A new `altcha.settings` config file is included in the recipe with `integration_type: self_hosted` and all other fields left at defaults. The `drupal_cms_starter` recipe likewise swaps `friendlycaptcha` for `altcha` in its install list. A FriendlyCaptcha-specific assertion (`buttonExists('Click to start verification')`) is removed from `ContactFormTest` as ALTCHA's widget does not present a manual trigger button.

## Upgrade

Drupal CMS does not ship upgrade paths between recipe versions. To switch an existing site from FriendlyCaptcha to ALTCHA:

1. Install ALTCHA and configure it via `/admin/config/people/captcha/altcha`.
2. Update the default CAPTCHA challenge at `/admin/config/people/captcha` to `altcha/ALTCHA`.
3. Uninstall FriendlyCaptcha.

### Collaboration

When `poker10` questioned whether ALTCHA was mature and effective enough to replace FriendlyCaptcha, `jurgenhaas` responded with a detailed side-by-side comparison of both modules covering PoW algorithms, difficulty scaling, secret storage, replay protection, and false-positive/negative rates. The analysis concluded that FriendlyCaptcha's local mode offered stronger baseline bot resistance (~200x more computational work per challenge) but that ALTCHA had a cleaner security architecture. `pameeela` cut through the debate by noting that the comparison was beside the point: FriendlyCaptcha no longer meets the project's core requirement of local-only verification, so the choice was ALTCHA or some other module, not ALTCHA vs. FriendlyCaptcha.
