## Summary

The site export command in Drupal CMS now excludes the site name and email address from exported templates. These values are typically provided during installation and should not be bundled into reusable site templates. This change ensures that exported site templates are cleaner and more appropriate for distribution, allowing each installation to collect its own site-specific information during setup rather than inheriting values from the original site.

## Impact

No upgrade or configuration changes required.

## Technical details

The `SiteExporter` class in the `drupal_cms_helper` module now explicitly removes the `name` and `mail` keys from the `system.site` configuration before exporting. This prevents site-specific information collected during the original installation from being included in the exported template.

The implementation uses `unset()` to remove these keys from the `$actions['system.site']['simpleConfigUpdate']` array before the recipe file is written. This is a targeted exclusion that preserves other system.site configuration values that may be legitimately part of the site template.

The change also includes a fix in `RecipeSubscriber` to prevent user 1 from being cached with stale field definitions during command-line installations using monolithic site templates. The subscriber now accepts a `RecipeAppliedEvent` parameter and clears the user entity storage cache for user 1 when installing a Site-type recipe via CLI. The class was also converted to a readonly class.
