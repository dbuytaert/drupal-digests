## Summary

Fresh Drupal CMS installations now allow modules and recipes to be applied directly from the Project Browser UI at `/admin/modules/browse`. Previously, the "Install" button was absent because the `allow_ui_install` flag in Project Browser defaults to `false`. Drupal CMS now overrides that default in its base recipe, so users no longer need Drush to enable content.

## Impact

Affects site owners.

- **Config change:** The Drupal CMS base recipe now sets `project_browser.admin_settings allow_ui_install` to `true`, enabling UI-based installs by default.

## Technical details

Project Browser ships with `project_browser.admin_settings:allow_ui_install` defaulting to `false`, which hides the Install button and shows only a "View Commands" link. The fix adds a single line to the Drupal CMS base recipe's `config:` block, setting `allow_ui_install: true`. This override is applied at install time by the recipe system, so every fresh Drupal CMS site gets the flag enabled without manual configuration.
