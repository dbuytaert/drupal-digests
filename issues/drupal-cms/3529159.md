## Summary

Drupal CMS 2.x now shows a language selector dropdown in the installer. When a language is chosen, the installer reloads with the selected `langcode` in the URL. The selector is hidden while batch installation tasks are running. Note that downloading `.po` translation files at install time is not yet implemented and is deferred to a future change in Recipe Installer Kit; this change covers the UI groundwork only.

## Impact

Affects module maintainers.

- **New API:** New `languages` template variable (keyed by langcode, each with `name` and `direction`) available in `install-page.html.twig` of `drupal_cms_installer_theme`

## Technical details

The work is entirely within the `drupal_cms_installer_theme` sub-theme. The `drupal_cms_installer_theme_preprocess_install_page()` function in `drupal_cms_installer_theme.theme` was added to populate a `$variables['languages']` array. It iterates `LanguageManager::getStandardLanguageList()` and, after the follow-up commits, passes each language's native name (`$language[1]`) and text direction (`$language[2]` falling back to `LanguageInterface::DIRECTION_LTR`) as an associative array. The `$variables['languages']` variable is only populated when `batch_get()` returns empty, suppressing the dropdown during active install tasks.

The `install-page.html.twig` template conditionally renders a `<select name="langcode">` element whose `<option>` tags carry the `dir` attribute for correct RTL rendering. A new `Drupal.behaviors.languageSwitcher` JavaScript behavior (in `js/language-switcher.js`) initialises the dropdown to the current `langcode` URL parameter (defaulting to `en`) and on change redirects to the same page with the updated parameter. CSS positions the selector absolutely at the top-right of the installer wrapper.
