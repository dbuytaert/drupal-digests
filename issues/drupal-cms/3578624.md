## Summary

Drupal CMS 2 sites installed with the Mercury starter template displayed a Twig error on utility pages like the Privacy Policy page. The error occurred because the heading component in the page content template was configured with an invalid default URL value of "No URL" instead of an empty string. This caused rendering to fail when viewing pages that use the Utility page content template. The fix corrects the starter configuration to use a valid empty string value, allowing utility pages to display properly without errors.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue affected sites installed with Drupal CMS 2 using the Mercury starter template. The `canvas.content_template.node.page.full.yml` configuration file contained a heading component with a hardcoded `url` input value set to the string "No URL". When this component was rendered on utility pages, the Mercury heading component's URL validation rejected this value as an invalid URL format, throwing a Twig runtime error.

The fix changes the default `url` value from `"No URL"` to an empty string `""` in the starter configuration. This allows the heading component to render successfully while still preserving the URI format field type, which provides an autocomplete link widget for editors who want to add a URL to the heading.

The change is purely a configuration correction in the `drupal_cms_starter` recipe, affecting only new installations that use this starter template. The fix ensures that the default Utility page content template works out of the box without triggering validation errors.
