## Summary

Drupal CMS now ships Cypress as its end-to-end (browser) testing framework. Contributors who need to write or run UI tests can execute the full suite with `ddev npm test` locally, and a dedicated CI job runs those tests automatically on every push. Spec files belong inside individual recipes, for example at `drupal_cms_blog/tests/e2e/*.cy.js`. PHPUnit remains the tool for unit, kernel, and functional PHP-level tests; JavaScript-driven browser tests should all go through Cypress.

## Impact

Affects module maintainers.

- **New API:** New Cypress end-to-end test runner; place `*.cy.js` spec files under `<recipe>/tests/e2e/` and run with `ddev npm test`

## Technical details

Cypress 13.15.0 is added as an npm dev dependency, with the lockfile committed as `package-lock.json`. DDEV is configured in `.ddev/config.yaml` to run `npm clean-install --foreground-scripts` on startup when `node_modules` is absent, set `DISPLAY=host.docker.internal:0` so Cypress can optionally open its interactive GUI against a host X11 server, and set `CYPRESS_CACHE_FOLDER=/var/www/html/.cache/cypress` so the downloaded binary survives container restarts. The DDEV web image is extended with Cypress' Ubuntu/Debian system dependencies (`libgtk2.0-0`, `libgbm-dev`, `xvfb`, etc.) plus Chromium and Firefox ESR as available browsers.

A new top-level `cypress.config.js` reads `DDEV_PRIMARY_URL` for `baseUrl` and uses `DDEV_DOCROOT + "/**/*.cy.js"` as `specPattern`, so any recipe can contain tests without extra configuration. The GitLab CI pipeline in `.gitlab-ci.yml` gains a new `run end-to-end tests` job that installs Cypress dependencies, starts Apache via an extracted `&start-apache` YAML anchor shared with the renamed `run PHP tests` job, and runs `npm test`; failure artifacts (screenshots, videos) are retained for one week. A minimal proof-of-concept spec at `drupal_cms_blog/tests/e2e/blog.cy.js` confirms the infrastructure runs correctly.
