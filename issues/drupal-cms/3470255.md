## Summary

Drupal CMS now ships with the core Footer menu block placed in the `footer_top` region of the Olivero theme on every page. This gives site owners a standard, manageable location for links such as Privacy Policy, Accessibility Statement, copyright information, and HTML sitemaps without any manual block placement after installation.

## Impact

Affects site owners.

- **Config change:** New default block config `block.block.olivero_footer` places the system Footer menu in the `footer_top` region of the Olivero theme on all pages.

## Technical details

The change is a single new default block configuration file, `block.block.olivero_footer.yml`, added to the Drupal CMS install profile. It wires the `system_menu_block:footer` plugin to the `footer_top` region of the Olivero theme with weight 0, label hidden (`label_display: '0'`), and no visibility conditions, so it appears site-wide. The block depends on the existing `system.menu.footer` config and the `system` module; no new menu or menu items are created by this commit.
