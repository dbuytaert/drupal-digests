## Summary

The Person content type's full-page view in the Olivero theme now matches the official design. Visitors see a two-column layout with the profile photo on the left (33%) and the person's role, phone number, and email on the right (67%). Phone and email fields display with small SVG icons inline. Below that two-column header, the body content and tags appear in a standard full-width section. The demo content path alias also changed from `/profiles/` to `/people/`.

## Impact

No upgrade or configuration changes required.

## Technical details

The Layout Builder display configuration in `node.person.default` was restructured from a single `layout_onecol` section to two sections: a new `layout_twocol_section` (33/67 column widths) containing the featured image (`field_featured_image`, rendered with the `1_1_300x300_focal_point_webp` image style) in the first column, and the role/job title, phone number, and email in the second column; followed by a `layout_onecol` section for `field_content` and tags.

A new CSS component (`css/components/person-full.css`) was added to the `drupal_cms_olivero` sub-theme. It constrains the featured image block's margins and renders `field--name-field-person__phone-number` and `field--name-field-person__email` as bold inline-flex elements with CSS mask-based SVG icons (`phone.svg` and `mail.svg`, both newly added to the theme's `images/` directory) pulled from `var(--color--primary-50)`. The library `person-full` is defined in `drupal_cms_olivero.libraries.yml` and attached via the new `drupal_cms_olivero_preprocess_node__person__full()` hook in `drupal_cms_olivero.module`. A new Twig template renders person name field items inside `<h2>` tags styled with the `h3` class.
