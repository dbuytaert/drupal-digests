# #3557383: Ship local video media type with Drupal CMS for better compatibility with Canvas

## Summary

The `drupal_cms_remote_video` recipe now also ships the local video media type, so Canvas components that reference a `video` schema can use the media library picker instead of falling back to a less reliable file picker. The recipe description has been updated to reflect that it now covers both locally and remotely hosted video. Additionally, whitespace-to-underscore filename sanitization is enabled for uploaded files, and the built-in `files` view is disabled to avoid confusing site editors who may not understand the difference between "Files" and "Media".

## Impact

No upgrade or configuration changes required.

## Technical details

The root problem was that Canvas resolves a `video` field (via `json-schema-definitions://canvas.module/video`) through the local video media type. When that media type was absent, Canvas degraded to a generic file picker that was unreliable. The fix adds `core/recipes/local_video_media_type` to the `recipes` list in `drupal_cms_remote_video/recipe.yml`, alongside the existing `core/recipes/remote_video_media_type`. The recipe also installs the `file` and `views` modules explicitly. A new `core.entity_form_display.media.video.*` config action mirrors the existing remote video form display: it hides `uid`, `langcode`, `path`, and `created`, and enables the `name` field. `file.settings` is updated via `simpleConfigUpdate` to set `filename_sanitization.replace_whitespace: true` in both `drupal_cms_remote_video` and `drupal_cms_media` (the latter backported to the 1.2.x branch of `drupal_cms_image` as a bug fix). `views.view.files` is disabled, and authenticated users receive the `delete own files` permission. The `composer.json` description was updated from "Configures display options for remote video" to "Configures display options for locally and remotely hosted video." A follow-up is planned to explore a Composer package alias so the recipe can also be referenced as `drupal/drupal_cms_local_video`, since the current package name (`drupal_cms_remote_video`) no longer reflects its full scope.
