## Summary

New Drupal CMS projects created via `composer create-project drupal/cms` now include a `.gitignore` file by default. Previously, initializing Git in a fresh project would show thousands of tracked files (vendor directories, compiled assets, OS metadata) because no ignore rules existed. The new file covers Composer-managed directories at the project root and in the web root, along with common OS noise files for macOS and Windows. IDE-specific ignores are being handled separately in [#3489119](https://www.drupal.org/node/3489119).

## Impact

No upgrade or configuration changes required.

## Technical details

The fix adds two `.gitignore` files inside `project_template/`, the directory whose contents are published as the `drupal/cms` Composer package. The root-level `.gitignore` covers `/recipes`, `/vendor`, and OS metadata (macOS `.DS_Store` family, Windows `Thumbs.db` family). The `web/.gitignore` covers the Composer-managed web-root directories (`/core`, `/libraries`, `/modules/contrib`, `/profiles/contrib`, `/themes/contrib`), per-site settings and files directories (`/sites/*/settings*.php`, `/sites/*/files`, etc.), and PHP scripts scaffolded into the web root by Drupal core.

To avoid a conflict with Drupal core's own scaffold file, the `composer.json` in `project_template/` now explicitly disables the `[web-root]/example.gitignore` scaffold mapping (`"[web-root]/example.gitignore": false`). A CI assertion verifies this file is absent after the build.

The cPanel project template is generated from `project_template/` by flattening the `web/` subdirectory into the project root. The cPanel build script (`generate_project_artifacts.sh`) was updated to concatenate `web/.gitignore` into the root `.gitignore` before the `mv web/*` step, so the cPanel template ends up with a single merged file that covers all the same paths.
