## Summary

The `drush site:export` command in Drupal CMS gains a `--base` option that lets site template authors seed an export from an existing recipe directory (such as `drupal_cms_site_template_base`). The base recipe's files are copied to the export destination first, preserving scaffolding like CI configuration, license files, and README documents, then the site's live configuration is exported on top. Three export bugs are also fixed: the command now refuses to overwrite an existing destination directory, the `system.site` front page path is converted from its internal system path to its URL alias in the exported config, and child menu link content entities now correctly record their parent menu link as a dependency in the exported content.

## Impact

No upgrade or configuration changes required.

## Technical details

In `SiteExportCommand`, the new `--base` option triggers a two-step copy before the export. First, `Symfony\Component\Finder\Finder` selects all files from the base recipe path with `ignoreVCS(TRUE)`, `ignoreDotFiles(FALSE)` (so hidden files like `.gitlab-ci.yml` and `.tugboat` are included), and `notPath('content')` (so base recipe content is excluded). `Symfony\Component\Filesystem\Filesystem::mirror()` writes those files to the destination. Second, a follow-up `Finder` pass locates any file whose name matches `*.example` or `.*.example` and renames each by stripping the eight-character `.example` suffix, activating dormant placeholder files such as `.gitattributes.example`. The command also checks for an existing destination directory upfront and returns `Command::FAILURE` immediately if it exists.

`GenericConfigurationListener` is changed from `final readonly` to `final` (mutable) so that `SiteExporter` can toggle a public `$convertFrontPagePathToAlias` flag at runtime. When the flag is `TRUE`, the listener's `__invoke()` method calls `AliasManagerInterface::getAliasByPath()` on the `system.site` `page.front` value before writing it to config storage, replacing the internal node path with its URL alias. `AliasManagerInterface` is injected via the updated constructor. This shim is marked for removal when core issue [#1503146](https://www.drupal.org/node/1503146) is released.

`DefaultContentSubscriber` becomes `final readonly` and gains an `EntityRepositoryInterface` constructor dependency. In `preExport()`, when the entity being exported is a `MenuLinkContentInterface` and its parent ID starts with `menu_link_content:`, the subscriber extracts the UUID, loads the parent entity via `EntityRepositoryInterface::loadEntityByUuid()`, and calls `$event->metadata->addDependency()` to register it. This shim is marked for removal when core issue [#3562072](https://www.drupal.org/node/3562072) is released.
