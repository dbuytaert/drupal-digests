## Summary

Drupal CMS now includes a button on the configuration export page that lets you export your entire site as a recipe template. This makes it easier to package and share site configurations with others. The exported recipe includes configuration, content, and metadata in a downloadable ZIP file. This feature helps teams create reusable starting points for new sites based on existing configurations.

## Impact

No upgrade or configuration changes required.

## Technical details

The `drupal_cms_helper` module adds a new button to the config export form at `/admin/config/development/configuration/full/export` through a `hook_form_config_export_form_alter()` implementation. When clicked, the button triggers the existing `SiteExporter` service to generate a recipe directory structure in the temporary filesystem.

The export includes configuration files, content entities, and recipe metadata (`recipe.yml` and `composer.json`). After export, the `.htaccess` file is removed from the config directory since templates are meant to be shared, not protected. The directory is then compressed into a ZIP archive using PHP's `PharData` class and delivered as a download via `BinaryFileResponse`.

The button is only visible when the `PharData` class is available. The existing "Export" button label changes to "Export configuration" to distinguish it from the new "Export site template" button. Both buttons require the "export configuration" permission.
