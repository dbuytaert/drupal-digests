# #3591409: Unable to add components to any Canvas pages in any site template

- **Project:** Drupal CMS
- **Type:** Critical bug report
- **Status:** Fixed
- **Discussion:** [#3591409](https://git.drupalcode.org/project/drupal_cms/-/work_items/3591409) · 1 contributor · 5 comments

## Summary

On a fresh Drupal CMS installation, attempting to edit any Canvas page produced a fatal "Error 500: A Component is being normalized that belongs in no Canvas UI library." This made Canvas completely unusable on all site templates including Haven and Starter. The fix adds `noUi: true` to the two Single Directory Components (SDCs) provided by the installer theme, so they are correctly excluded from Canvas's component library and no longer cause an error after installation is complete.

## Impact

No upgrade or configuration changes required.

## Technical details

The `drupal_cms_installer_theme` is only present during the installation process and is removed after the site is set up. However, its two SDC components, `dialog` and `language-switcher`, were being discovered by Canvas's component plugin manager after installation. Because their provider theme no longer existed, Canvas could not normalize them and threw a `LogicException`. Canvas respects the `noUi: true` flag in a component's `.component.yml` metadata to exclude that component from its UI library entirely. Adding `noUi: true` to both `drupal_cms_installer/theme/components/dialog/dialog.component.yml` and `drupal_cms_installer/theme/components/language-switcher/language-switcher.component.yml` prevents Canvas from attempting to load or display these installer-only components.
