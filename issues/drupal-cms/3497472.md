## Summary

A new Testimonial component is now available for the Experience Builder (XB) page builder within Drupal CMS's Olivero theme integration. Site builders can add a pull-quote block to any XB-managed page, optionally pairing it with a portrait image, an attributed name, a role or extra detail line, and a call-to-action link. The component is marked experimental and ships with responsive styling based on CSS container queries.

## Impact

Affects module maintainers.

- **New API:** New `Testimonial` XB component in `drupal_cms_olivero` with props: `content` (required), `name` (required), `image`, `role`, `link_text`, `link_url`

## Technical details

The component lives in `drupal_cms_olivero/components/testimonial/` and consists of four new files: a `component.yml` metadata file, a `testimonial.html.twig` template, a `testimonial.css` stylesheet, and an SVG arrow icon used by the CTA link.

The `component.yml` schema defines two required props (`content` typed as an XB `textarea`, and `name`) and four optional props (`image` typed as an XB `image` object with `src`, `alt`, `width`, and `height` sub-keys; `role`; `link_text` with a default of `"Learn more"`; and `link_url` as a `uri-reference`). The component `status` is `experimental`.

The Twig template wraps the quote in a `<blockquote class="testimonial__content">` inside a container div, conditionally renders a `<img class="testimonial__image">` when `image.src` is present, emits a `<cite class="testimonial__source">` block with `<span>` elements for name and role, and appends a `<div class="testimonial__footer">` holding an anchor styled as a button when `link_url` is set. The arrow SVG is inlined via `include`. RTL support is handled in CSS using `:dir(rtl)` selectors with `scale: -1` applied to both the opening quotation-mark pseudo-element and the arrow icon.
