## Summary

The DDEV custom commands for working with Drupal CMS recipes have been renamed for consistency. `apply-recipe` is now `recipe-apply` and `create-recipe` is now `recipe-create`. The old names continue to work as aliases, so existing workflows are not broken. The new naming convention groups all recipe-related commands under a `recipe-` prefix, which allows short aliases (`ra`, `rc`) and console-style aliases (`recipe:apply`, `recipe:create`), and leaves room for future additions like `recipe-unpack` or `recipe-revert`.

## Impact

No upgrade or configuration changes required.

## Technical details

The change updates two DDEV command shell scripts in the Drupal CMS repository. The `## Usage:` and `## Example:` header lines in each script were updated to reflect the new primary names. A new `## Aliases:` header line was added to each script registering the old name and the new short forms. For `recipe-apply` the aliases are `recipe:apply`, `ra`, and `apply-recipe`. For `recipe-create` the aliases are `recipe:create`, `rc`, and `create-recipe`. No logic in the underlying scripts changed.

## Upgrade

If you have documentation, `Makefile` targets, or CI scripts that call `ddev apply-recipe` or `ddev create-recipe`, those still work via the registered aliases, but updating them to the primary names (`ddev recipe-apply`, `ddev recipe-create`) is recommended.
