## Summary

The Drupal CMS project template now bundles `composer/composer:^2.8` as a runtime dependency and raises the minimum Drupal core requirement to 11.2.4. With these two changes together, Package Manager can automatically locate and use the locally-packaged Composer binary instead of relying on a system-wide Composer installation. This significantly lowers the barrier to using Package Manager on shared or managed hosting environments where Composer is not available on the system PATH. Sites already running Drupal 11.2.4 or later will benefit immediately after updating to the corresponding Drupal CMS release.

## Impact

Affects module maintainers and site owners.

- **Requirements:** Minimum Drupal core version raised from ^11.2 to ^11.2.4 in all project template and module constraints.
- **New API:** composer/composer:^2.8 added as a runtime dependency in the project template, bundling Composer locally for use by Package Manager.

## Technical details

The project template `composer.json` gains `composer/composer: ^2.8` in `require` and `drupal/core-vendor-hardening: ^11.2.4`. The vendor hardening plugin is configured via the `drupal-core-vendor-hardening` extra key to exempt `composer/composer` from cleanup (value set to `false`), so the full library survives at `vendor/composer/composer/bin/composer`. However, to avoid leaving an executable Composer binary under `vendor/bin/`, a `post-update-cmd` Composer script runs `@unlink('vendor/bin/composer')` after every `composer install` or `composer update`. The CI pipeline now runs `composer install` without `--no-scripts` and asserts both that `vendor/bin/composer` is absent and that `vendor/composer/composer/bin/composer` is present.

The `drupal_cms_helper` module's minimum `core_version_requirement` is bumped from `^11.2` to `^11.2.4`. As part of the same bump, the `@internal` `AddComponentsToLayout` config action plugin and its registration in `PluginHooks::configActionInfoAlter()` are removed; the equivalent `add_layout_component:addComponentsToLayout` plugin now lives in core (tracked in [#3540305](https://www.drupal.org/node/3540305)). The cPanel template build script also drops its separate `composer require drupal/core-vendor-hardening` step, as `drupal/core-vendor-hardening` is now pulled in through the standard project template.

## Upgrade

The minimum Drupal core version for the project template is now `^11.2.4`. Update your project's `composer.json` constraint accordingly and run `composer update`.

After the update, `vendor/bin/composer` will be automatically removed by the `post-update-cmd` script. The Composer library itself remains at `vendor/composer/composer/bin/composer` for use by Package Manager.

If you maintained a fork or custom copy of the cPanel template that separately required `drupal/core-vendor-hardening`, that explicit requirement can be removed as it is now pulled in by the standard project template.
