## Technical details

The Drupal CMS project template now includes a `CONTRIBUTING.md` file in the `project_template/` directory. This file addresses a recurring problem where contributors would clone the published Composer package repository instead of the development repository.

The new `CONTRIBUTING.md` explains that Drupal CMS uses a monorepo structure that automatically splits into smaller projects. It specifies DDEV version 1.24.0 or later as a requirement and provides three commands to get started: clone the repository from `git@git.drupal.org:project/drupal_cms.git`, change into the directory, and run `ddev launch`.

The file also documents common DDEV commands like `ddev drush` and `ddev composer`, and directs contributors to open issues and merge requests in the Drupal CMS development issue queue. It includes links to the code of conduct, the `#drupal-cms-development` Slack channel, and DDEV documentation.
