## Summary

Drupal CMS previously shipped two custom text formats, `basic_html` and `full_html`, inherited from the Standard profile. Because all content editors in Drupal CMS have the same permissions, the choice between them was unnecessary and confusing. This change removes `basic_html` and the standalone `drupal_cms_html_editors` recipe entirely, folding the Full HTML editor configuration (CKEditor5, Linkit, Media Library) directly into the base recipe. The `full_html` format is now the single editor-facing text format; `restricted_html` remains as a low-level fallback but is not surfaced for content editors. A weight fix also ensures Full HTML is pre-selected by default rather than Restricted HTML.

## Impact

Affects site owners.

- **Config change:** The `basic_html` text format and editor are removed; `full_html` is now the sole CKEditor5-backed format and is part of the base recipe rather than the separate `drupal_cms_html_editors` recipe.
- **Permission change:** The `use text format basic_html` permission is removed; `use text format full_html` and `use text format restricted_html` are now granted by the base recipe instead of `drupal_cms_html_editors`.

## Technical details

The `drupal_cms_html_editors` recipe, which provided both `basic_html` and `full_html` formats with enhanced editors, has been deleted. Its modules (`ckeditor5`, `linkit`, `media_library`) and the `image.style.linkit_result_thumbnail` config import are now part of the base `drupal_cms` recipe's `install` and `config` sections directly. The `full_html` format config's `weight` was changed from `2` to `-1` so it sorts above `restricted_html` and is therefore chosen as the default in field widgets.

The `content_editor` role's text format permissions (`use text format full_html`, `use text format restricted_html`) moved from `drupal_cms_html_editors` into the base recipe. The `basic_html`-specific permission (`use text format basic_html`) is gone entirely, as is the `basic_html` filter format config and its paired CKEditor5 editor config.

All existing config referencing `basic_html` (Views empty-text fields in the blog recipe, and default content in `drupal_cms_content_type_base`) was updated to `restricted_html`, since those are simple static strings that do not require a full editor. The `drupal_cms_seo_tools` recipe, which depended on `drupal_cms_html_editors` for its dashboard blocks, was updated to depend on `core/recipes/full_html_format_editor` and `core/recipes/image_media_type` instead, and a previously missing `handler_settings.target_bundles` constraint on `field_seo_image` was added to ensure it only references image media. The `drupal_cms_dashboard` recipe gained a missing `view welcome dashboard` permission grant for authenticated users.

## Upgrade

If you have a custom recipe that lists `drupal_cms_html_editors` as a dependency, remove that dependency. The modules it provided (`ckeditor5`, `linkit`, `media_library`) and the Full HTML editor are now part of the base Drupal CMS recipe.

If any of your content or configuration references the `basic_html` text format, update it to use `full_html` or `restricted_html` as appropriate:

```yaml
# Before
format: basic_html

# After (for rich-text content fields)
format: full_html

# After (for plain static strings, e.g. Views empty text)
format: restricted_html
```

The `use text format basic_html` permission no longer exists and should be removed from any custom roles.
