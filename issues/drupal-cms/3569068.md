# #3569068: Provide the documentation structure in mkdocs.yml format for parsing into drupal.org

- **Project:** Drupal CMS
- **Type:** Normal feature request
- **Status:** Fixed
- **Discussion:** [#3569068](https://www.drupal.org/node/3569068) · 2 contributors · 6 comments

## Summary

Drupal's documentation infrastructure needed to support both Zensical's preferred TOML format and Drupal.org's mkdocs.yml format for parsing documentation structure. This issue resolved that compatibility problem by setting up automated conversion in the documentation build pipeline. The solution was ultimately superseded when Drupal.org moved away from mirroring documentation locally and instead redirected to the static documentation site built directly from the repository.

## Impact

No upgrade or configuration changes required.

## Technical details

The Drupal documentation repository uses Zensical as its documentation tool, which prefers a `docs/zensical.toml` configuration file. However, Drupal.org's infrastructure was set up to parse `mkdocs.yml` files for documentation structure. To bridge this gap, the team integrated `toml-cli`, a Python utility, into the continuous integration pipeline for documentation publishing. This allowed automated generation and maintenance of an up-to-date `mkdocs.yml` file from the TOML source, ensuring compatibility with Drupal.org's parsing requirements while keeping Zensical's preferred format as the source of truth.

The CI build successfully validated this approach. However, before the solution needed to be maintained long-term, Drupal.org's documentation architecture changed. The organization removed its local documentation mirror and instead configured redirects to point directly to the static documentation site generated from the repository. This architectural shift eliminated the need for Drupal.org to parse any configuration file format, making the conversion pipeline unnecessary.
