## Summary

Drupal CMS now ships with a `drupal_cms_privacy_basic` recipe that is applied to every new installation. It provides out-of-the-box cookie consent management and remote content blocking powered by the [Klaro](https://www.drupal.org/project/klaro) module. The recipe also creates placeholder "Privacy policy" (at `/privacy-policy`) and "Imprint" (at `/imprint`) pages pre-populated with GDPR-aligned template text, and adds both as links in the site footer. No action is needed on new sites; existing Drupal CMS sites must apply the recipe manually to get these features.

## Impact

Affects module maintainers.

## Technical details

This parent issue drove a wide evaluation of Drupal 11-compatible cookie consent modules (COOKiES, EU Cookie Compliance, GDPR, Klaro, and roughly 40 others). The COOKiES module was ruled out because its maintainer was unresponsive about a licensing question for the required CookiesJSR JavaScript library. EU Cookie Compliance was considered for its user base but noted as needing a significant rewrite. The GDPR module was found to lack a consent UI of its own. The Klaro module was selected because it handles both consent management and remote content blocking, ships with a self-hosted JS library (`drupal/klaro_js` on drupal.org, a maintained fork), and was committed to Drupal's security advisory policy coverage.

The concrete implementation landed in [#3483392](https://www.drupal.org/node/3483392). A new `drupal_cms_privacy_basic` recipe package is defined in its own `composer.json`, declaring a dependency on `drupal/klaro` 3.x-dev. The recipe is listed in the base `content_installer_recipe.yml` so it applies to all new sites, and added to the CI matrix in `.gitlab-ci.yml`. It installs a `system_menu_block:footer` block into the `footer_top` region of the Olivero theme (config exported as `block.block.drupal_cms_olivero_footer_menu`), seeds two basic page nodes ("Privacy policy" and "Imprint") via default content YML files, and creates footer menu links pointing to those nodes. A new assertion in `ComponentValidationTest` verifies that both links are visible to anonymous users in the footer after the recipe is applied.

## Upgrade

Existing Drupal CMS sites must apply the recipe manually to get these features:

```bash
drush recipe path/to/drupal_cms_privacy_basic
```

After applying, edit the seeded "Privacy policy" (`/privacy-policy`) and "Imprint" (`/imprint`) pages to replace the placeholder contact details and legal text with your site's real information.
