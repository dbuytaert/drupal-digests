## Summary

Drupal CMS gains an optional `drupal_cms_ai` recipe that adds AI-powered capabilities to a site: automatic image alt text generation, a chatbot assistant for site building guidance, and AI agents that can perform content type and field management tasks on behalf of site builders. During recipe installation, users are prompted to choose an AI provider (amazee.ai, OpenAI, or Anthropic) and supply the relevant API key. amazee.ai is the default and provides a free trial without requiring a key upfront. The recipe is not installed by default and must be explicitly opted into.

## Impact

Affects module maintainers.

- **New API:** New `drupal_cms_ai` recipe installable via `composer require drupal/drupal_cms_ai` that enables AI alt text generation, an AI chatbot, and AI agents for site building.

## Technical details

The new recipe lives at `recipes/drupal_cms_ai/recipe.yml`. It installs the `ai`, `ai_dashboard`, `ai_agents`, `ai_image_alt_text`, `ai_assistant_api`, `ai_chatbot`, and `ai_provider_amazeeio/openai/anthropic` modules, along with the `canvas` theme (required by the chatbot UI) and `klaro` for cookie consent. The recipe declares three `input` values: `provider` (a radio selection), `openai_api_key`, and `anthropic_api_key`. The API key fields use `#states` visibility so only the field matching the selected provider is shown. Keys are stored via the `key` module using the `setupAiProvider` config action, and encryption is initialized via `ensureEasyEncryptionSetup`. The amazee.ai path uses `ensureAmazeeAiAccess` with `no_key_needed: true`.

The recipe ships two config files: `ai_agents.ai_agent.drupal_cms_assistant.yml` and `ai_assistant_api.ai_assistant.drupal_cms_assistant.yml`, which together define the "Drupal CMS Assistant" orchestration agent. The assistant is wired to three sub-agents (content type triage, field triage, taxonomy config) and is configured with an extensive system prompt that shapes its persona and response format for site builders with no Drupal experience. The chatbot block is placed in the admin theme toolbar region via the `placeBlockInAdminTheme` config action. The `content_editor` role is granted the `generate ai alt tags` permission.

An earlier proposal to add an installer step for selecting the AI provider was explicitly rejected during review in favour of delivering the configuration entirely through the recipe's `input` mechanism, keeping the installer unchanged.
