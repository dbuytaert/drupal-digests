## Summary

Drupal CMS now requires Canvas 1.0.0-RC1 instead of the previous beta. The upgrade removes temporary polyfill code that was added to work around a limitation in the beta release, and updates all bundled Canvas component configuration files to match the RC1 schema. For site owners running Drupal CMS, this brings the included page-building theme one step closer to a stable release.

## Impact

No upgrade or configuration changes required.

## Technical details

The Canvas module dependency in both the root `composer.json` and the Drupal CMS starter recipe's `composer.json` was changed from `@beta` / `^1-beta2` to `@rc` / `^1-rc1`.

The main polyfill being removed was a computed `url` base field on `node` entities, which existed solely because Canvas's beta could not resolve entity URLs natively. This consisted of `drupal_cms_helper\UrlField` (a `FieldItemList` using `ComputedItemListTrait`) and the `hook_entity_base_field_info` implementation in `drupal_cms_helper\Hook\EntityHooks`, along with its functional test `UrlFieldTest`. With RC1, Canvas gained a built-in `host-entity-url` sourceType, so the blog card component config was updated to use `sourceType: host-entity-url` instead of a dynamic expression referencing `$node->url`. The blog card's config dependencies were also updated to reference the actual image and media fields directly rather than routing through `drupal_cms_helper`.

All `prop_field_definitions` entries across several component config YAML files (button, hero, card, columns block, text block, and others) gained `required: false`, a new mandatory key in the RC1 schema. The text block component additionally changed its expression from `text_long‚êüvalue` to `text_long‚êüprocessed` and added `format: canvas_html_block` to its default value, ensuring rich text is filtered before rendering. Each affected config file's `active_version` hash was updated to reflect the new schema version.

## Upgrade

If you have custom code that references the `@internal` `UrlField` or `EntityHooks` classes from `drupal_cms_helper`, remove those references. The `url` computed base field on `node` entities added by the old `hook_entity_base_field_info` no longer exists.

If you have custom Canvas component configs derived from the Drupal CMS defaults, add `required: false` to every entry under `prop_field_definitions`:

```yaml
versioned_properties:
  active:
    settings:
      prop_field_definitions:
        my_prop:
          required: false   # add this line
          field_type: string
          ...
```
