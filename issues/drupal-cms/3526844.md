## Summary

Drupal CMS is shipping its first real-world site template, aimed at SaaS and B2B product marketing websites. The template bundles an opinionated set of recipes, content types, layout components, and demo content so a site owner can spin up a product site with a blog, feature highlights, pricing section, FAQ, testimonials, and a contact form. The implementation ultimately landed as the separate `byte` project (drupal.org/project/byte) rather than inside the main Drupal CMS repository.

## Impact

Affects module maintainers.

- **New API:** New `drupal_cms_b2b_template` site recipe (now the `byte` project at drupal.org/project/byte) for bootstrapping SaaS and B2B product marketing sites on Drupal CMS.

## Technical details

The MR scaffolded a `drupal/drupal_cms_b2b_template` recipe (type: `Site`) requiring `drupal/core ^11.2` and `drupal/drupal_cms_blog ^2`. The `recipe.yml` applied `administrator_role`, `content_editor_role`, and `drupal_cms_blog` as recipe dependencies, and imported `system.menu.footer` and `system.menu.main` config. The MR was closed without merging; the full implementation moved to the `byte` project.

The issue body defines the intended content model: Blog Post (delegated to `drupal_cms_blog` as a dependency rather than copied), Feature and FAQ content types as nodes (so editors find all content at `/admin/content`), and a Changelog page built on `drupal_cms_page`. Layout components (Hero, Features, Pricing, Logo grid, Testimonial list, CTA banner) are planned as SDCs, pending the new Drupal CMS design system. Contact form support comes from the existing `drupal_cms_forms` recipe. A `Marketing` role is added alongside the standard `administrator` and `editor` roles from core recipes.

Microcontent reusability (testimonials, FAQs as shared entities) was debated and explicitly excluded from MVP: pameeela noted the reuse case does not meet the 80% threshold. Behavior tracking, CRM integration, and carousels are deferred to future issues. The dependency on the `entity_display_route` contrib module was raised (by thejimbirch) since it lacks a stable release and security coverage; the canonical fix is tracked in Drupal core issue [#3483304](https://www.drupal.org/node/3483304).
