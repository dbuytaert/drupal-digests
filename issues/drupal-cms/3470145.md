# #3470145: To respect user privacy and comply with GDPR, use openstreetmap.de for Leaflet maps

## Summary

Drupal CMS now blocks Leaflet map tiles until a site visitor explicitly gives consent, addressing GDPR and broader privacy compliance concerns out of the box. The default map tile sources (openstreetmap.org via fastly.net, with servers in the United States) transfer the visitor's IP address to a third-party service outside the EU on every page load. Rather than switching tile servers to EU-hosted alternatives, the project resolved the issue by enabling a Klaro consent gate for Leaflet maps whenever the Event recipe is applied. Maps remain invisible until the visitor actively consents, keeping the default state compliant without requiring the site owner to configure anything manually.

## Impact

Affects site owners.

- **Config change:** The Event recipe now enables the `klaro.klaro_app.leaflet` Klaro consent app, blocking Leaflet map tiles by default until the visitor explicitly consents.

## Technical details

The issue originally proposed switching the Leaflet map tile source from `OSM Mapnik` (openstreetmap.org via fastly.net, US-hosted) to `osm-de` (openstreetmap.de, hosted by Hetzner in Germany). After discussion, the privacy track concluded that consent is required before transferring a visitor's IP to any third-party service, regardless of server location, making the tile-server switch insufficient on its own.

The actual fix was implemented in issue [#3491406](https://www.drupal.org/node/3491406): the Event recipe was updated to include a `simpleConfigUpdate` config action that sets `klaro.klaro_app.leaflet` to `status: true`. This activates the Klaro consent app for Leaflet (originally created in the Klaro module via issue [#3491325](https://www.drupal.org/node/3491325)), which blocks tile requests until the user consents through the Klaro consent UI. No map tiles are loaded on page render by default; the Klaro overlay intercepts the Leaflet widget and only permits the external requests after explicit visitor approval.

### Collaboration

The issue sparked a substantive privacy debate. ressa, the reporter, pushed for switching to openstreetmap.de and documented in detail that tile requests resolve to Fastly and Cloudflare infrastructure in the United States. jurgenhaas (privacy track lead) argued that EU-hosted tiles would not eliminate the need for consent, because legal advisors consistently recommend getting user consent before any IP transfer to a third party, and because CDN locations can change silently. That position was later confirmed when ressa cited the January 2025 EU General Court ruling in Case T-354/22, in which the European Commission was fined for transferring a user's IP address to a US-hosted service without consent. The thread ultimately converged on the Klaro consent gate as the legally sounder resolution, with the original tile-server request closed as addressed by the broader approach.
