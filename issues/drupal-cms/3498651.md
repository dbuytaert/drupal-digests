## Summary

The `drupal_cms_person_profiles` recipe now ships a fully styled "People" listing page out of the box. Installing the recipe creates a published "People" basic page at `/people`, adds it to the main navigation, and embeds a new `person_profiles` view block that displays all published person nodes as cards. The URL pattern for individual person profiles also changed from `/profiles/[node:title]` to `/people/[node:title]`. Sites that already had the recipe installed and have person content or links pointing to the old `/profiles/` path will need to update those URLs.

## Impact

Affects module maintainers and site owners.

- **Config change:** Person node URL pattern changed from `/profiles/[node:title]` to `/people/[node:title]`; recipe dependency changed from `drupal_cms_content_type_base` to `drupal_cms_page`.
- **New API:** New `person_profiles` Views config and a default 'People' page node with main-menu link are provisioned by the recipe on install.

## Technical details

Four pieces were added or changed together. First, a new Views configuration (`person_profiles`) with a block display (`all`) lists all published `person` nodes, rendered in the `card` view mode, sorted by creation date descending and paginated at 15 per page. Second, a default `page` node (UUID `43833f92`) at the `/people` alias is exported as content and embedded with a Layout Builder layout that places the `views_block:person_profiles-all` block alongside the standard page fields. A `menu_link_content` entry pointing to that node is also exported, wiring "People" into the main navigation automatically.

Third, CSS styles for `.node--type-person.teaser--card` were added to the Olivero theme layer. The teaser image uses `aspect-ratio: 1/1` with `object-fit: cover` and a square medium view mode (`1_1_square_medium`, changed from `4_3_medium`). On hover a 6 px gray (`--color--gray-65`) left-edge accent stripe appears via a `::before` pseudo-element. The `field--name-field-person__role-job-title` field is added to the existing muted metadata text block.

Fourth, the recipe's dependency was changed from `drupal_cms_content_type_base` to `drupal_cms_page` (required to provision the People listing node), and `menu_link_content` was added to the recipe's `install` list. The pathauto pattern `person_content` was updated from `/profiles/[node:title]` to `/people/[node:title]`, with the corresponding test assertion updated to match.

## Upgrade

If your site already has the Person Profiles recipe installed and person nodes exist, their canonical URLs will have changed from `/profiles/&lt;slug&gt;` to `/people/&lt;slug&gt;`. Update any hardcoded links, redirects, or sitemaps that reference the old `/profiles/` prefix.

No PHP API changes were made; this is a configuration and content-recipe change only.
