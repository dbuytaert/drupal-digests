# #3481715: Target Drupal 11.1 for stable release without core patches applied

## Summary

Drupal CMS's stable release was planned to target Drupal 11.1 with no core patches applied, because the `automatic_updates` module (a key Drupal CMS feature) cannot coexist with `cweagans/composer-patches` in production builds. Including patches silently disables automatic updates for site owners who have no knowledge of, or ability to manage, the underlying patching. The resolution was to eliminate all core patches from stable Drupal CMS builds before the 11.1 release, which was confirmed complete. The follow-on issue [#3488457](https://www.drupal.org/node/3488457) tracks moving `composer-patches` to a dev dependency so it remains available for development but is never present in a deployed stable site.

## Impact

No upgrade or configuration changes required.

## Technical details

The `package_manager` module's `ComposerPatchesValidator` enforces that `cweagans/composer-patches`, if installed, must have been explicitly added as a direct dependency by a technical user at the command line. Because Drupal CMS's project template previously pulled in `composer-patches` automatically (and applied patches inside recipe definitions and `drupal_cms_patches/composer.json`), the validator would either block updates outright or silently leave patches that could fail to apply after a core update. Even in cases where the validator passed, any patch that no longer applied after an upstream fix would break subsequent automatic updates with no path to recovery for a non-technical site owner. Removing core patches from all stable builds, and demoting `composer-patches` to a dev-only dependency, resolves the conflict and allows `automatic_updates` to function as designed for Drupal CMS's target audience.
