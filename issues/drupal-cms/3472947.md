## Summary

Drupal CMS now redirects anonymous users to the login form when they try to access a page they are not allowed to see, instead of showing a plain "Access Denied" error. After logging in, Drupal sends users to the page they originally requested. A related ECA model also sends users back to the login page immediately after they log out.

## Impact

Affects site owners.

- **Config change:** `system.site.page.403` is set to `/user/login` in the `drupal_cms_site_template_base` recipe, replacing the default access-denied page with the login form for all sites built on this recipe.

## Technical details

The original proposal was to add the `r4032login` contrib module as a dependency of the base recipe, but that MR was closed after the same outcome was achieved with core configuration and ECA. The `drupal_cms_site_template_base` recipe sets `system.site.page.403` to `/user/login` via a `simpleConfigUpdate` action in `recipe.yml`. Drupal core's login route already appends the originating URL as a `destination` query parameter when a user is sent there from a 403, so no additional module is needed for the post-login redirect. A separate ECA model (`eca.eca.auth_redirects`, shipped in the same recipe) listens for the `user:logout` event and issues a redirect to `/user/login`, covering the logout path.
