## Summary

Drupal CMS site owners can now install the Google Analytics recipe directly through the Project Browser UI. Previously, the recipe was hidden from Project Browser because it required interactive inputs (a Google Tag Manager ID) that Project Browser could not prompt for. With alpha7 of Project Browser adding recipe input support, `drupal_cms_google_analytics` was added to the allowed recipes list alongside `drupal_cms_ai` and `drupal_cms_accessibility_tools`. Project Browser was also configured to allow only one recipe selection at a time (`max_selections: 1`).

## Impact

Affects module maintainers.

- **New API:** Three Drupal CMS recipes (`drupal_cms_google_analytics`, `drupal_cms_ai`, `drupal_cms_accessibility_tools`) are now listed in the Project Browser allowed set and installable via the UI

## Technical details

The change lands in the `drupal_cms_starter` recipe's `recipe.yml`. The `project_browser.admin_settings` config block's `allowed_projects.recipes` list received three new entries: `drupal_cms_google_analytics`, `drupal_cms_ai`, and `drupal_cms_accessibility_tools`. At the same time, `max_selections: 1` was added to that same config block, and the `drupal/project_browser` dependency in `drupal_cms_starter`'s `composer.json` was bumped from `^2-alpha6` to `^2-alpha7`. The same commit also updated AI-related module dependencies from pre-release beta constraints (`^1-beta*`) to stable `^1` constraints, and raised the project template's `minimum-stability` from `alpha` to `beta`. This was delivered as part of the broader issue [#3498277](https://www.drupal.org/node/3498277), which phenaproxima described as waiting on the Project Browser alpha7 release before merging.
