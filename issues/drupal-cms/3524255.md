## Summary

New Drupal CMS sites created with `composer create-project` will now have their recipes automatically unpacked at project creation time. The recipes are physically placed in the `recipes/` directory so the installer can apply them, but they are removed from `composer.json` immediately afterward. This means that after a project is created and committed to a Git repository, a subsequent `composer install` will not reinstall the recipes. Existing Drupal CMS sites are not affected by this change.

## Impact

Affects site owners.

- **Config change:** New `drupal/core-recipe-unpack` Composer plugin added to the Drupal CMS project template; recipes are no longer retained as Composer dependencies after `composer create-project`.

## Technical details

The change adds `drupal/core-recipe-unpack` (a Composer plugin introduced in Drupal core 11.2) to the Drupal CMS project template's `composer.json`, both as a `require` entry and as an allowed plugin in the `config.allow-plugins` map. The plugin intercepts `composer create-project` and physically copies recipe packages into the `recipes/` directory while stripping them from `composer.json`, making them one-time-use artifacts. A separate bump of `drupal/recipe_installer_kit` from `1-alpha3@alpha` to `^1-alpha4@alpha` is also included in the same commit. The issue body explicitly calls out that deployment workflows that rely on `composer install` at the remote end followed by recipe application will break without either committing the recipes to the codebase or exporting configuration first. The recommended workflow is to apply recipes locally, export configuration, and commit the exported configuration to version control.

## Upgrade

Existing sites are unaffected by default; the plugin only runs during `composer create-project`. If you want to adopt the same behavior on an existing site, manually add the plugin to your top-level `composer.json`:

```json
"require": {
    "drupal/core-recipe-unpack": "^11.2@beta"
},
"config": {
    "allow-plugins": {
        "drupal/core-recipe-unpack": true
    }
}
```

Then run `composer unpack drupal/RECIPE_NAME` for each recipe you want to unpack manually.
