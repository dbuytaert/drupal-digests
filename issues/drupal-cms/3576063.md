## Summary

Drupal CMS installer now displays a curated list of site templates that are not shipped with the core installation. During site setup, users can browse and select from templates hosted remotely, including options like Byte (for SaaS product websites) and Haven (for non-profit sites). The installer retrieves this list from a centralized location, making it easy to expand the available templates without releasing new versions of Drupal CMS. If the project directory is not writable, the installer shows a warning and only displays locally available templates.

## Impact

No upgrade or configuration changes required.

## Technical details

The installer now fetches site template metadata from a remotely hosted YAML file at `https://git.drupalcode.org/api/v4/projects/204857/repository/files/site-templates.yml/raw?ref=HEAD`. This file contains template names, descriptions, package names, and screenshot URLs. The `SiteTemplateForm` class calls `getCuratedList()` to retrieve this data, with caching to minimize HTTP requests.

During development, when the `site-templates.yml` file exists locally, it is read directly from the filesystem. In production releases, this file is excluded via `.gitattributes`, forcing the installer to fetch it remotely. Screenshots can now be either local file paths (converted to base64 data URIs) or remote URLs.

The installer checks if the project root is writable before displaying remote templates. If not writable, it shows a warning and returns an empty list, since users wouldn't be able to install remote templates anyway. Local templates already present in the codebase are always shown and take precedence over remote entries with the same machine name.

The `InternalDevelopmentSubscriber` was updated to use the `DRUPAL_CMS_DEV` environment variable instead of checking the DDEV project name, improving portability.
