## Summary

The Drupal CMS installer now displays a curated list of site templates fetched from a remote source, rather than only showing templates included in the installation package. This allows the installer to present a broader selection of starter sites without requiring all templates to be bundled with every download. The change makes the installer more flexible and marketplace-ready for DrupalCon Chicago, giving site builders more choices during initial setup without increasing download size.

## Impact

No upgrade or configuration changes required.

## Technical details

The installer now retrieves site template listings from a YAML file hosted on drupalcode.org. During development, it reads from a local `site-templates.yml` file in the `drupal_cms_installer` package. In production, this file is excluded via `.gitattributes`, forcing the installer to fetch the list remotely from the canonical URL.

The `SiteTemplateForm::getCuratedList()` method handles this logic. It first checks if the local file exists and reads it directly. If not, it fetches the remote URL and caches the result using the default cache backend. The cache key is an xxh32 hash of the URL. Any HTTP or parse exceptions are caught and displayed as warnings.

The theme layer was updated to handle both local file paths (which get base64-encoded as data URIs) and remote URLs (which are used as-is) for template screenshots. The previous `Settings::get('additional_site_template_list')` mechanism was removed in favor of this more structured approach.

The development environment detection also changed from checking `DDEV_PROJECT` to a simpler `DRUPAL_CMS_DEV` environment variable for internal tooling.
