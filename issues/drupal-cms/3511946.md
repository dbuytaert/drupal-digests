## Summary

Drupal CMS now uses the [Tagify](https://www.drupal.org/project/tagify) module for the Tags field on all built-in content types (Blog, Case Study, Events, News, Pages, and the Portfolio/Projects type). Instead of the standard autocomplete text input, editors see a visual tag-chip component that is more intuitive and consistent with modern CMS interfaces. The change ships with Drupal CMS 1.1.0 and does not affect 1.0.x.

## Impact

Affects module maintainers.

- **New API:** New `tagify_entity_reference_autocomplete_widget` form widget is now the default for `field_tags` in all Drupal CMS built-in content types.

## Technical details

The commit adds `drupal/tagify ^1.2` as a Composer dependency to both the `drupal_cms_admin_ui` and `drupal_cms_content_type_base` recipes, and enables the `tagify_user_list` submodule in each. In the form display config files for every bundled content type, `field_tags` is switched from the `entity_reference_autocomplete_tags` widget type to `tagify_entity_reference_autocomplete_widget`, and the now-irrelevant `size: 60` setting is removed. The `ContentModelTestTrait` gains a new branch to handle the `tagify` input type assertion, since Tagify renders a plain `input[name=field_name]` rather than the array-indexed input the standard autocomplete uses. Making Tagify the automatic default widget for newly created entity reference fields (including base fields like `uid`) was deliberately deferred to a follow-up issue in the Tagify module itself.
