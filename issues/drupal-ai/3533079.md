## Summary

Canvas AI can now generate a complete page layout from a single natural-language prompt. Previously, the AI could only add individual components one at a time. Now a prompt like "Create a homepage for an ice cream shop with a header and footer" assembles an entire page, placing components into the appropriate theme regions. By default the AI populates only the content region; headers and footers are generated only when explicitly requested. Site builders can also navigate to the new "Page Region Descriptions" settings page (under Configuration > AI > Canvas AI) to describe the purpose of each enabled region, guiding the AI when deciding where to place components. The Canvas AI admin routes have also moved from `/admin/config/system/` to `/admin/config/ai/`; a post-update hook rebuilds the route cache automatically on update.

## Impact

Affects module maintainers and site owners.

- **Config change:** New `canvas_ai.theme_region.settings` config object for per-region AI descriptions; Canvas AI admin routes moved from `/admin/config/system/` to `/admin/config/ai/`.
- **Deprecation:** Protected `CanvasAiPageBuilderHelper::processComponentsBelow()` removed with no replacement; footer placement now handled by `canvas_page_builder_agent`.
- **New API:** New public methods `getRegionIndex()`, `getAvailableRegions()`, and `processSetTemplateDataToolInput()` on `CanvasAiPageBuilderHelper`; new `canvas_template_builder_agent` agent plugin and `canvas_ai:set_template_data` function call plugin.

## Technical details

A new `canvas_template_builder_agent` AI sub-agent handles full-page creation. It receives available components via the `canvas_ai:get_component_context` tool, the current page layout via `canvas_ai:get_current_layout`, and region descriptions via the `[canvas_ai:available_regions]` token. The agent produces a YAML structure keyed by region name; a new `canvas_ai:set_template_data` function call plugin validates and passes this YAML to `CanvasAiPageBuilderHelper::processSetTemplateDataToolInput()`, which converts it into `addNewComponentToLayout` operations with calculated `nodePath` values using the new `getRegionIndex()` helper.

Region descriptions are stored in the new `canvas_ai.theme_region.settings` config object and surfaced through `CanvasAiPageBuilderHelper::getAvailableRegions()`, which reads the active theme name from the injected `ThemeHandlerInterface` and merges stored descriptions with the live region list from the page layout. The `[canvas_ai:available_regions]` token is now passed to both the template builder and the page builder agents; the prop-setting logic block in both agents' system prompts was also synchronized to ensure identical image-prop handling.

The orchestrator system prompt gained a new Rule 7 that routes full-page creation requests to `canvas_template_builder_agent` and incremental additions to `canvas_page_builder_agent`. Rule 4 (proactive title and metadata generation) was extended to cover both agents. Polling messages now read "Finding components to place" for the page builder and "Designing the page" for the template builder. The protected `processComponentsBelow()` method was removed from `CanvasAiPageBuilderHelper` as footer placement is now handled by the page builder agent instead. Canvas AI admin local tasks were reorganized into three tabs: General Settings, Component Descriptions, and Page Region Descriptions.

## Upgrade

Canvas AI admin routes have moved. Update any hard-coded links or custom code that references these paths:

```
# Old paths (no longer valid)
/admin/config/system/canvas-ai-settings

# New paths
/admin/config/ai/canvas-ai-settings
/admin/config/ai/canvas-ai-component-description-settings
/admin/config/ai/canvas-ai-theme-region-settings  # new
```

The post-update hook `canvas_ai_post_update_0001_rebuild_router()` rebuilds the route cache automatically when running database updates.

The protected `CanvasAiPageBuilderHelper::processComponentsBelow()` method has been removed. Any custom code extending this class and calling that method must be updated.
