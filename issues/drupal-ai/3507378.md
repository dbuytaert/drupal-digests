## Summary

The AI API Explorer submodule now generates code examples using PHP heredoc syntax instead of string concatenation. This makes the example code displayed in the Explorer interface easier to read and maintain. The change improves the developer experience when learning how to use the AI module's different operation types like chat, embeddings, text-to-image, and speech-to-text.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI API Explorer provides interactive demonstrations of different AI operations and shows developers example code for implementing them. Previously, all code examples were built using string concatenation with `.=` operators and `<br>` tags for line breaks.

The refactoring converts these methods to use PHP heredoc syntax (`<<<END ... END`). This syntax allows multiline strings without concatenation operators, making the code generation logic cleaner and easier to maintain.

Variables that need interpolation are extracted before the heredoc blocks so they can be embedded directly. For example, `$provider_id = $form_state->getValue('chat_ai_provider')` extracts the value once, then `'$provider_id'` is used within the heredoc string.

The change affects seven plugin classes across different operation types: `ChatGenerator`, `EmbeddingsGenerator`, `ModerationGenerator`, `SpeechToTextGenerator`, `TextToImageGenerator`, and `TextToSpeechGenerator`. Each plugin's `normalizeCodeExample()` and related methods were refactored. The generated output remains functionally identical, so end users see the same code examples with cleaner internal implementation.
