## Summary

The AI API Explorer submodule now uses PHP heredoc syntax for generating code examples. This makes the example code easier to read and maintain. When developers explore different AI operations like chat, embeddings, or text-to-image generation, they see cleaner code snippets that demonstrate how to use the API. The change only affects how the code examples are constructed internally and does not change the functionality or output of the explorer.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI API Explorer generates code examples to help developers learn how to use different AI operations. Previously, these examples were built by concatenating many short strings with the `.=` operator, each adding a line of code. This resulted in long, hard-to-read methods where variables had to be manually escaped and the flow was difficult to follow.

The refactoring changes all code example generation methods across seven plugin files (`ChatGenerator`, `EmbeddingsGenerator`, `ModerationGenerator`, `SpeechToTextGenerator`, `TextToImageGenerator`, and `TextToSpeechGenerator`) to use PHP heredoc syntax. With heredoc, multi-line code blocks are written as literal text between `<<<END` and `END` markers, with variables interpolated directly.

This approach extracts form values into variables at the start of each method, then uses those variables within the heredoc blocks. The resulting code is more readable and maintainable because the PHP code closely resembles the actual example code being generated. Variable escaping is simplified since heredoc handles it naturally.
