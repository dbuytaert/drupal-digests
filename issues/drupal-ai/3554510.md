## Technical details

This issue synchronizes the visual design of the AI chatbot toolbar placement with the Canvas AI chatbot, based on months of feedback from AI UI/UX alignment meetings and design work in Figma.

The implementation touches multiple CSS files and templates. The `toolbar-button.css` file replaces the animated SVG approach with static gradient icons (`ai-icon.svg`, `ai-icon-gradient.svg`, `ai-icon-white.svg`), removing the `ai-icon-animated.svg` entirely. The button now uses background images instead of mask images for the toolbar, with a 32px × 32px size and hover/focus states that switch to a white icon on blue background.

The `toolbar-chatbot.css` file adds numerous refinements: the chat container width changes from 21rem to 100vw on small screens, the header height increases to 72px with updated spacing, and the transform-based positioning replaces margin-based displacement on desktop. The sidebar header gets a bottom border and the new gradient icon. Desktop layouts now use `transform: translateX()` on `.admin-toolbar` instead of margin to avoid conflicts with toolbar states.

The `toolbar.yml` DeepChat style configuration extensively updates typography, spacing, and colors. Message bubbles use 16px font size with 26px line height, user messages get `#F1F2F3` background instead of blue tint, and AI messages use `#4B4F58` text color. The chat header grows to 34px font with a gradient effect, the text input increases to 104px height with 8px border radius, and the loading animation switches to a custom `loading.svg` spinner.

The `ChatbotHooks::toolbar()` method was added to create a toolbar item for the legacy toolbar (non-navigation module), ensuring the chatbot button appears in both toolbar systems. The `ai_chatbot_toolbar()` hook implementation calls this service method.

The template `ai-deepchat--toolbar.html.twig` restructures the greeting with line breaks, adds the gradient icon, and updates copy from "Hello 👋" to "Hi there! Welcome to Drupal 👋" with "How can I help you today?" as the subtitle.

The default bot image in `DeepChatFormBlock::defaultConfiguration()` switches from `ai-star-avatar.svg` to `ai-icon-gradient.svg`. The `.cspell-project-words.txt` adds "textcolor" to the dictionary.

Specificity adjustments account for Gin theme changes from [#3560492](https://www.drupal.org/node/3560492) that increased toolbar selector specificity. The `:is(#ai-chatbot, [data-drupal-admin-styles])` wrapper ensures styles apply correctly.

## Collaboration

This issue involved significant cross-team collaboration over several months. Six contributors participated across 43 comments, with extensive testing on Drupal CMS 2.0, vanilla Drupal 11, and multiple Gin toolbar configurations. The design work originated from AI UI/UX alignment meetings and Canvas team feedback, documented in Figma.

The review process caught regressions from upstream Gin theme changes that broke SVG display due to increased CSS specificity, requiring adjustments mid-implementation. Multiple merge request recreations were needed due to rebasing complications between the 1.2.x and 2.0.x branches.

Testers identified issues with Olivero theme compatibility and requested clarification about which admin themes were in scope. The team agreed to limit the initial scope to Gin and Drupal CMS, deferring broader theme support to follow-up work. A follow-up issue [#3568246](https://www.drupal.org/project/ai/issues/3568246) was created for the animation-on-reload problem discovered during final testing.

The issue remained open from October 2025 to January 2026 partly due to evolving requirements and the need to sync with ongoing Gin development. The team prioritized getting the foundational work merged rather than expanding scope further, recognizing that additional design features like file uploads, media support, and detailed tool-call information would require separate issues beyond pure styling work.

## Lessons learned

- When design requirements target a specific environment (admin theme, specific distribution), scope the issue to that context and defer styling for other environments to follow-ups. Expanding scope mid-issue introduces conflicts and delays without delivering the primary goal. *(via bbruno)*
- When upstream frameworks increase CSS specificity, modules that target those selectors must match the new specificity. After framework changes that affect your selectors, update them according to the framework's migration guidance to restore expected behavior. *(via marcus_johansson, bbruno)*
