## Summary

The AI module can now automatically generate summaries from long text fields. When editing content, you can configure a field to automatically create a condensed summary from another field, like creating a teaser from a body field. This uses specialized AI summarization models rather than chat-based AI, providing faster and more focused summaries. The feature works with providers like Hugging Face and Google Chrome's built-in summarization. This is useful for creating article previews, meta descriptions, search snippets, or newsletter excerpts without manual writing.

## Impact

No upgrade or configuration changes required.

## Technical details

This adds two new automator plugins for the summarize operation type introduced in [#3574178](https://www.drupal.org/node/3574178). The `LlmSummarizeToStringLong` plugin targets `string_long` fields, while `LlmSummarizeToTextLong` targets `text_long` fields with text format support.

Both plugins extend a new `Summarize` base class that implements the core summarization logic. Unlike chat-based automators, this uses the AI module's summarize endpoint with dedicated summarization models. The base class overrides `needsPrompt()` and `advancedMode()` to return FALSE since summarization doesn't require custom prompts.

The `generate()` method iterates through base field items, creating a `SummarizationInput` object for each, and calls the provider's `summarize()` method. For `text_long` fields, the plugin adds a form field to select the output text format, ensuring proper format handling for formatted text fields.

Documentation was added covering supported providers (Hugging Face models like `facebook/bart-large-cnn`, `sshleifer/distilbart-cnn-12-6`, `google/pegasus-xsum`, and Google Chrome) and eight common use cases ranging from article summaries to moderation review snippets.
