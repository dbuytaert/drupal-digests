## Summary

The AI module now includes a summarize automator that can automatically generate concise summaries from longer text content. This builds on the summarization operation type added previously by making it accessible for actual use. Site builders can configure fields to automatically generate summaries from other fields, such as creating article teasers from body content or meta descriptions from page text. The automator uses dedicated AI summarization models, which are optimized for extracting key information rather than using chat-based language models.

## Impact

No upgrade or configuration changes required.

## Technical details

This issue implements two new automator plugins for the AI Automators module: `LlmSummarizeToStringLong` for plain text fields and `LlmSummarizeToTextLong` for formatted text fields. Both extend a new `Summarize` base class that handles the core summarization logic.

The `Summarize` base class sets the LLM type to "summarize" and implements the `generate()` method to iterate through base field items, creating `SummarizationInput` objects and calling the AI provider's `summarize()` method. Unlike chat-based automators, summarization does not require prompts or support advanced mode configuration.

The `LlmSummarizeToTextLong` variant adds an extra configuration option for selecting a text format, ensuring the generated summary maintains proper formatting. This is important for cron jobs that run anonymously.

The implementation works with AI summarization providers such as Hugging Face models (BART, DistilBART, Pegasus) and Google Chrome's built-in summarization API. Documentation was added covering field types, requirements, and eight common use cases including article summaries, teaser generation, and meta descriptions.
