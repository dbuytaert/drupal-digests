## Summary

The AI chatbot Assistant button was broken in admin themes other than Gin and incorrectly appeared on frontend themes. This bug was introduced in the AI module version 1.2.x when toolbar integration was added. The button now functions properly in all admin themes (Gin, Claro, etc.) and only appears when an AI chatbot block is actually placed and enabled in the current theme.

## Impact

No upgrade or configuration changes required.

## Technical details

The `ai_chatbot` module was using `hook_toolbar()` to add the Assistant button, but the approach had two problems. First, the toolbar library attachments were not loading correctly outside the Gin theme in Drupal 11.3. Second, the button appeared on all pages regardless of whether a chatbot block was actually configured.

The fix removes `hook_toolbar()` entirely and consolidates on `hook_preprocess_top_bar()` instead. The new `ChatbotHooks::topbar()` method checks two conditions before adding the button: whether the user has the `access deepchat api` permission, and whether any enabled `ai_deepchat_block` exists in the active theme.

A new protected method `hasDeepChatBlock()` uses `ThemeManagerInterface` and `EntityTypeManagerInterface` to query the block storage for enabled `ai_deepchat_block` instances in the current theme. This ensures the Assistant button only appears when there's actually a chatbot to open. The method includes exception handling to fail gracefully if the block storage lookup encounters errors.

## Lessons learned

- When toolbar items depend on JavaScript libraries, using a toolbar hook to load those libraries can fail silently if the libraries don't load in all contexts. Instead, conditionally display toolbar items based on whether the associated functionality (like a block) is present on the current page. *(via marcus_johansson)*
- Using JavaScript to hide UI elements is a poor solution that leads to unstyled, broken elements flashing before removal. Prevent rendering the elements server-side based on context instead. *(via marcus_johansson)*
