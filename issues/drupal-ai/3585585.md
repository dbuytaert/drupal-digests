## Summary

The `ai_best_practices` project now provides explicit, human-readable guidance on how to write Agent Skills for Drupal. A new `docs/skill-authoring.md` file defines when a skill is appropriate (versus documentation, `AGENTS.md`, or MCP resources/tools/prompts), where skills should live (with the project that owns the behavior), and what frontmatter is required. The required Drupal-specific frontmatter fields `status`, `drupal-version`, and `last-reviewed` have been dropped; skills now need only the Agent Skills specification fields `name` and `description`. The eval framework gains a new `agent_skill_frontmatter` check that validates these spec fields.

## Impact

No upgrade or configuration changes required.

## Technical details

The commit does four things. First, it adds `docs/skill-authoring.md` (180 lines), which formalises the boundaries: docs are declarative, skills are procedural; `AGENTS.md` carries always-relevant project context; MCP resources/tools/prompts handle live data and executable actions. The file also defines an admission checklist for skills landing in `ai_best_practices` itself and lists good/bad candidates for that project.

Second, `CONTRIBUTION.md` is updated to reference the new doc, replace the old "one skill = one file" rule with "one skill = one directory" (each skill lives under its own directory with a `SKILL.md` file), and rewrite the frontmatter section. The former required fields `status`, `drupal-version`, and `last-reviewed` are removed; `name` and `description` are the only Drupal-wide required fields going forward.

Third, `evals/run-evals.py` gains two new functions: `parse_frontmatter()`, a lightweight YAML-frontmatter extractor that avoids a YAML library dependency, and `check_agent_skill_frontmatter()`, which validates that `name` is present, is 64 characters or fewer, uses only lowercase letters/numbers/hyphens, and matches the parent directory name, and that `description` is present, is at least 40 characters, and is at most 1024 characters. The existing `frontmatter_fields` check path is updated to use `parse_frontmatter()` as well.

Fourth, the eval JSON files for existing skills (including `drupal-automated-testing`) replace their `frontmatter_fields` assertion with the new `agent_skill_frontmatter` check. The `CONTRIBUTION.md` section on contesting a skill is updated to drop the instruction to set `status: contested`; contributors should instead explain the uncertainty inline in the skill body.

## Upgrade

Skills that previously used `status`, `drupal-version`, or `last-reviewed` frontmatter fields should migrate to the Agent Skills required fields only:

```markdown
# Before
---
status: stable
drupal-version: "11+"
last-reviewed: 2025-04
---

# After
---
name: my-skill-name
description: One sentence describing what this skill helps with and when to use it.
---
```

Eval JSON entries using `"check": "frontmatter_fields"` with the old fields should be updated:

```json
// Before
{ "id": "S02", "check": "frontmatter_fields", "fields": ["status", "drupal-version", "last-reviewed"] }

// After
{ "id": "S02", "check": "agent_skill_frontmatter" }
```
