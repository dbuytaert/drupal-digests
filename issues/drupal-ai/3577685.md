## Summary

The AI File to Text module now supports JSON as a fourth output format alongside plain text, HTML, and Markdown. This is needed for the Document Loader module when storing extracted content in JSON fields. Spreadsheets and CSV files produce structured JSON with column headers as keys, making the data ready for programmatic use. Other document types (PDFs, Word documents, etc.) output a structured JSON DOM tree that preserves document structure like headings, lists, tables, and inline formatting. This maintains semantic information that would be lost in plain text while providing a machine-readable format.

## Impact

No upgrade or configuration changes required.

## Technical details

The `ExtractorInterface` gains a new `extractJson()` method. For spreadsheet and CSV extractors, the method returns native tabular JSON where each row becomes an object keyed by column headers. For other file types, JSON is derived from HTML output using a new `HtmlToJsonTrait` that converts the HTML to a structured DOM tree via `DOMDocument`. Element nodes become associative arrays with `tag`, optional `attributes`, and optional `children` keys. Text nodes become plain strings. Whitespace-only text nodes are filtered out.

The `FileExtractorManager` handles routing: when JSON is requested, it checks if the extractor natively supports JSON (via `getSupportedOutputFormats()`). If so, it calls `extractJson()` directly. If not, but the extractor supports HTML, it extracts HTML with native headings enabled and converts to JSON using the trait. Text-only extractors fall back to plain text.

JSON is included in the aggregated output formats when any extractor supports HTML, since it can be derived. The `PopplerPdfExtractor` declares native JSON support with a fallback to text-wrapped JSON if HTML extraction fails.
