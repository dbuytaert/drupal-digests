## Summary

The AI Chatbot block now offers configurable expansion modes to improve readability when the chatbot is placed in the toolbar. Administrators can choose between three options: none, expand (widens the sidebar and shifts page content), or fullscreen (opens as a modal overlay). This addresses usability concerns with the default sidebar's limited space, giving users more room to interact with longer conversations. The feature includes smooth animations, keyboard support for exiting fullscreen with the Escape key, and conditional loading of assets to avoid unnecessary overhead.

## Impact

No upgrade or configuration changes required.

## Technical details

When the AI Chatbot block is configured with toolbar placement, a new "Expansion method" select field appears in the block settings under styling options. The field offers three values: "none", "expand", and "fullscreen". The expand mode uses CSS custom properties (`--toolbar-chatbot-expand-amount`) to dynamically calculate width adjustments, pushing Drupal's displacement offset and page content to the right by 20rem. The fullscreen mode applies a fade-in animation and renders the chatbot as a modal using pure CSS (fixed positioning with a backdrop overlay), preserving the chatbot's current state without creating a new dialog element. The fullscreen library (`toolbar-fullscreen.css`) is conditionally attached only when the fullscreen method is selected. JavaScript in `toolbar-chatbot.js` handles expand button clicks, toggling body classes (`ai-chatbot-expanded`, `ai-chatbot-fullscreen`), managing `aria-expanded` attributes for accessibility, and listening for the Escape key to exit fullscreen. Exit animations use `animationend` events with double `requestAnimationFrame` to ensure clean state transitions. Both expansion states reset properly when the chatbot is closed. The expand button is omitted from the DOM when set to "none". SVG icons are updated to use `currentColor` for theme consistency.
