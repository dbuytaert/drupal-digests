## Summary

The ai_logging module now provides a `drush ai:logs` command (aliases `ail`, `ai-logs`) to inspect AI LLM call logs from the terminal. The command queries `ai_log` entities and displays prompts, responses, provider/model metadata, and tags. It supports filtering by tag (`--tag=ai_agents`) or provider (`--provider=anthropic`), limiting result count (`--count=20`), and viewing single entries with full text (`--id=42`). This is useful for debugging AI agents, chatbots, or other AI-powered features without opening the Drupal admin UI. The ai_logging module must be enabled and logging must be turned on at /admin/config/ai/logging/settings.

## Technical details

The `AiLogCommands` Drush command class uses `AutowireTrait` for dependency injection and injects `EntityTypeManagerInterface` to query the `ai_log` entity storage. The command has two modes: list mode (default) returns multiple log entries sorted by `created` DESC then `id` DESC, and single-entry mode (`--id=N`) returns one full entry. List mode truncates `prompt` and `output_text` fields to 80 characters using core's `Unicode::truncate()`. Filtering is implemented via entity query conditions on `tags` or `provider` fields. The command returns `RowsOfFields`, so Drush provides `--format=json|yaml|csv|table`, `--fields`, and `--filter` options automatically. The `ValidateModulesEnabled` attribute ensures ai_logging is enabled before execution. Ten kernel tests cover list mode, filtering, single-entry lookup, truncation, empty results, and row structure. The initial implementation was rewritten after maintainer feedback to use Drush's built-in formatters instead of custom output logic, reducing the command from approximately 500 lines to 130 lines and dropping custom token estimation and formatting utilities.
