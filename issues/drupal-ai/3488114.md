## Summary

AI providers can now implement a rerank operation to improve search result quality. Reranking is a two-step process that takes an initial set of documents (like search results) and reorders them based on relevance to a specific query. This helps semantic search systems return more accurate results by using specialized reranking algorithms or AI models as a second pass after initial retrieval. Some providers like Cohere offer dedicated reranking endpoints, while others can use general-purpose language models for this task.

## Impact

Affects module maintainers.

- **New API:** New `rerank` operation type for AI providers that can re-order search results or document lists based on relevance to a query

## Technical details

The AI module now defines a `rerank` operation type alongside existing operations like chat, embedding, and moderation. The operation type consists of three new classes: `ReRankInterface` defines the provider plugin interface with a `rerank()` method, `ReRankInput` accepts a query string, array of input documents, optional top N limit, and metadata, and `ReRankOutput` returns reranked results with ID and metadata.

The `ReRankInput` class uses getter and setter methods for all properties to allow modification after construction. It converts to an array using `toArray()` with `query`, `top_n`, and `inputs` keys. Provider implementations are responsible for mapping the `inputs` property to whatever document format their external API expects.

Configuration schema was updated to support a default rerank provider at `ai.settings.default_providers.rerank`. The AI Search module documentation now references using `hook_ai_search_boost_results_alter` to rerank results before passing them to database or SOLR search. This operation type enables AI Search backends to optionally apply reranking as a post-processing step after vector database retrieval.
