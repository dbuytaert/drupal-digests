# #3601357: GitHub: default to private repo and prompt for public

- **Project:** Drupal AI
- **Type:** Normal feature request
- **Status:** Fixed
- **Discussion:** [#3601357](https://git.drupalcode.org/project/one_line_installer/-/work_items/3601357) · 1 contributor · 10 comments

## Summary

When the one_line_installer prompts you to create a GitHub repository for a new site, it now defaults to **private** visibility. Previously the prompt defaulted to public, meaning a new site could be published to GitHub publicly unless the user explicitly chose otherwise. Now you must opt in to make the repo public.

## Impact

No upgrade or configuration changes required.

## Technical details

The change affects three places in `src/`. The global initializer in `src/setup_site_command.sh` now sets `GH_VISIBILITY="--private"` as the starting default, with a comment noting that repos are created private unless the user explicitly opts into public. In `src/lib/steps/github_setup.sh`, the `github_setup` function's visibility prompt now lists "Private" as the first option (index 0) with `MENU_DEFAULT=0` pre-highlighting it, and "Public" as the second option; selecting index 0 sets `GH_VISIBILITY="--private"`, index 1 sets `--public`. The `--github-setup` CLI flag path in `src/lib/shared/prompts/apply_cli_flags.sh` also sets `GH_VISIBILITY="--private"` (it had no visibility override before; the non-interactive flag path now hard-codes private, consistent with the prompt default).
