## Summary

The Mistral AI provider now supports structured output and tool calling, bringing it up to feature parity with other providers like OpenAI. These capabilities allow AI models to return data in predefined JSON schemas and to call external functions during conversations. This makes it possible to build more reliable integrations where AI responses follow strict data structures or trigger specific actions, such as searching databases or performing calculations.

## Impact

No upgrade or configuration changes required.

## Technical details

This change implements two key features in the `MistralProvider` plugin class: structured output via JSON schema and tool calling (also known as function calling).

For structured output, the provider now checks if a `ChatInput` object includes a JSON schema via `getChatStructuredJsonSchema()`. When present, it adds a `response_format` parameter to the API request with type `json_schema`, instructing the Mistral API to return responses matching the provided schema.

For tool calling, the provider handles three scenarios. First, when sending a request with tools, it serializes the tools configuration using `getChatTools()->renderToolsArray()` and includes it in the API parameters. Second, when processing responses, it extracts tool calls from the Mistral response using `getToolCalls()`, wraps each in a `ToolsFunctionOutput` object with the tool ID and arguments, and attaches them to the `ChatMessage` via `setTools()`. Third, the message construction logic now handles tool response messages (identified by `getToolsId()`) and assistant messages with tool calls from previous conversation turns by calling `addToolMessage()` and `addAssistantMessage()` with the rendered tool calls.

The implementation also standardizes variable naming from camelCase to snake_case throughout the chat operation method for consistency. AI-assisted code generation was used for portions of this implementation.
