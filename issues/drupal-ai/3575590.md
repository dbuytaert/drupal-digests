## Summary

The AI Context module now provides quick access to debug and explore tools directly from the agent settings interface. When the `ai_agents_debugger` or `ai_agents_explorer` modules are enabled, links appear both in the agent list table (as dropdown operations) and on individual agent edit pages. This eliminates the need to manually construct URLs or navigate separately to debugging tools, making agent development and troubleshooting faster.

## Impact

No upgrade or configuration changes required.

## Technical details

The change adds conditional links to two forms in the AI Context module: `AiContextAgentsForm` (the agent list) and `AiContextAgentForm` (individual agent edit page).

In the agent list form, the "Configure context items" link is converted to a dropbutton that can include "Debug agent" and "Explore agent" operations. In the individual agent edit form, debug and explore links are appended inline after the agent description text, styled as subtle bracketed links.

Both forms check whether `ai_agents_debugger` and `ai_agents_explorer` modules are enabled via `ModuleHandlerInterface`, and whether the current user has the required permissions (`debug ai agents` and `use the agent explorer` respectively). The links point to the debug and explorer routes with the `agent_id` passed as a query parameter, and open in new tabs.

CSS styling is added for the inline links to match the Gin admin theme aesthetic. The test suite is expanded to verify link visibility based on module state and user permissions, including exact element counts to ensure links appear only once per context.
