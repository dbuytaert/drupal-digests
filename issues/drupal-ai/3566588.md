## Summary

The AI module now supports AI-assisted generation for Office Hours fields directly within content editing forms. When editing content with Office Hours fields, editors can now trigger AI automators to analyze free-text descriptions of opening times and automatically populate structured office hours data. This makes it easier to convert unstructured schedule information into properly formatted office hours without manual data entry.

## Impact

No upgrade or configuration changes required.

## Technical details

This change adds a new Field Widget Action plugin called `OfficeHours` that integrates the AI automators module with the Office Hours module's field widgets. The plugin supports both `office_hours_default` and `office_hours_list` widget types for the `office_hours` field type.

The `OfficeHours` class extends `AutomatorBaseAction` and implements two key methods. The `aiAutomatorsAjax()` method handles the AJAX callback when editors trigger the AI automator action, extracting the form context and delegating to the base class's `populateAutomatorValues()` method. The `saveFormValues()` method updates the form's default values for the three office hours properties: `day`, `starthours`, and `endhours`. It supports both single-item updates (when a specific key is provided) and bulk updates (when processing all field items).

The implementation follows the pattern established by other Field Widget Action plugins in the `ai_automators` module, such as the Email plugin. The plugin automatically becomes available when the Office Hours module is installed since the widget types it declares only exist when that module is enabled.
