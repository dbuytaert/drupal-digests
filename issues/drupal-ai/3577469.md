## Summary

This is an internal change to the testing infrastructure. FunctionalJavascript tests can now record videos of their execution, making it easier to diagnose test failures. Tests also run selectively in CI: on main branches they skip entirely, on issue branches only tests tagged with the issue number run, and on tagged releases all tests run. This has no effect on sites using the module.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue addressed functional test maintainability and debuggability concerns. The solution added video recording and selective test execution.

The base test class (`BaseClassFunctionalJavascriptTests`) now supports opt-in video recording. When a test sets `$videoRecording = TRUE`, Chrome runs non-headless and ffmpeg captures the Selenium container's X display (`:99.0`) at 25fps with 750k bitrate, saving MP4 files organized by module and test class. The recording lifecycle is managed in `setUp()` and `tearDown()`. The implementation uses `proc_open()` to spawn ffmpeg with x11grab input, and stops gracefully by writing 'q' to stdin (ensuring the MP4 moov atom is finalized) with fallback to SIGTERM/SIGKILL. The `getMinkDriverArgs()` override strips `--headless` from Chrome options when recording is enabled.

A new shell script (`scripts/filter-functionaljavascript-tests.sh`) removes test files before discovery to work around a run-tests.sh limitation where `--directory` ignores `@group` filtering. On main branches (matching `^[0-9]+(\.[0-9]+)?\.x$`), all FunctionalJavascript tests are removed. On issue branches, only tests with `@group {issue_number}` are kept. On tags, all tests run. The script exports `HAS_FJS=1/0` so `.gitlab-ci.yml` can skip ffmpeg installation when no tests will run. The script examines `CI_COMMIT_TAG`, `CI_MERGE_REQUEST_SOURCE_BRANCH_NAME`, and `CI_COMMIT_BRANCH` to determine context, then uses `find` with vendor/node_modules exclusions and `grep -Fq` to check for issue number annotations.

Other base class improvements: changed default theme from `stark` to `claro` for realistic admin UI testing, moved output paths from `sites/default/files/simpletest/` to `sites/simpletest/` to match core conventions, added `setDefaultProvider()` helper to reduce boilerplate, and added strict typing throughout.

The two existing FunctionalJavascript tests were updated to enable video recording, add `@group 3577469`, use strict typing, and use the new `setDefaultProvider()` helper. The `AutoCompleteTagsTaxonomyTest` namespace was corrected to remove an extra "Tests" segment.
