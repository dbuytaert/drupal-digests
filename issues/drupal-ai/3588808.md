# #3588808: [Sprint 8] Add Phase 1.5 (content model setup) using content_type and taxonomy scopes

## Summary

Sites migrated with the `canvas_ai_migrations` migrate-site skill no longer require manually creating Drupal content types and taxonomy vocabularies before running Phase 7 (content composition). Previously, if a source site had dynamic content types (events, blog posts, news items), implementers had to create the corresponding Drupal bundles via the admin UI themselves before Phase 7 would work. The new Phase 1.5 handles this automatically using browser automation against the Drupal admin UI, since the JSON:API endpoints needed for a fully programmatic path are not yet available on Acquia Source instances.

## Impact

Affects module maintainers.

- **New API:** New `content_model_setup_completed` state event documented in `state-events.md` for Phase 1.5 completion tracking.

## Technical details

The change adds a `### Phase 1.5 — Content Model Setup (conditional)` block to `SKILL.md`, inserted between Phase 1 (site-analyzer) and Phase 2 (component-builder). Phase 1.5 is skipped when `plan.md` contains no `content_types:` entries; it executes when Phase 1 detected dynamic content types.

The implementation uses `agent-browser` authenticated via the `.canvas-browser/` session profile to navigate `/admin/structure/types` and `/admin/structure/taxonomy` in the Drupal admin HTML UI. For each content type declared in `plan.md`, the phase checks whether the bundle exists, creates it via the `/admin/structure/types/add` form if missing, and then provisions each declared field. Taxonomy vocabularies are handled similarly via `/admin/structure/taxonomy/add`. The JSON:API routes `/jsonapi/node_type/*` and `/jsonapi/taxonomy_vocabulary/*` remain 404 on all probed Acquia Source releases; the design explicitly documents them as the future API path, gated by `scope-availability.json`, for when Acquia ships those endpoints.

Phase 1.5 writes `docs/migration/workflow/01.5-content-model-checklist.json` per the Phase Gate Convention and appends a `content_model_setup_completed` event to `state.jsonl`. A new precondition in the Phase 7 block requires this checklist to exist with `payload.overall: pass` when any `content_types:` were declared; otherwise Phase 7 halts with a `blocked.md` entry. Five new scope probe entries (`canvas:brand_kit`, `content_type:administer`, `taxonomy:administer`, `page_template:administer`, `site_settings:administer`) are added additively to `scripts/preflight/lib/scope-manifest.json`, bringing the total to 14 scopes. The five historical `acquia_source:*` entries are preserved unchanged. Nine tests in `tests/skills/sprint8-3588808-content-model-phase.test.mjs` cover the new section, scope-manifest contents, state-events documentation, and the cross-file checklist filename consistency.
