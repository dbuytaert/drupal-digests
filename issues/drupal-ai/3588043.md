## Technical details

The MR adds two files under `skills/drupal-gitlab/`: `SKILL.md` (the agent-facing instruction file) and `README.md` (the human-facing setup guide).

`SKILL.md` addresses several Drupal-specific constraints that vanilla `glab` guidance does not cover. Cross-project MRs (from an issue fork to the upstream project) are not supported by `glab mr create`, so the skill instructs agents to use the GitLab REST API directly, fetching both the source (`issue/<project>-<issue-id>`) and target (`project/<repo>`) project IDs before constructing the payload. Work item status is a widget, not a label, so reading or updating it requires a GraphQL `workItemUpdate` mutation rather than a label command. Issue forks cannot be created by pushing or via the API because Drupal's CDN intercepts those routes and returns a 301 to `drupal.org/git-error`; the skill instructs the agent to direct the user to the Drupal.org management page (`https://new.drupal.org/drupalorg/issue-fork/management?source_link=...`) instead. The `GITLAB_HOST` environment variable must be passed to every `glab` command, and `--hostname` must additionally be passed to `glab api` and `glab auth` subcommands. Commit messages must follow Drupal's Conventional Commits format including `By:` lines with Drupal.org usernames.
