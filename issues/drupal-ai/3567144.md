# #3567144: Chat configuration select lists use machine names not labels

## Summary

This issue tracked a user interface problem where chatbot configuration forms displayed machine names (like "system_main" or "content") instead of human-readable labels (like "Main content" or "Content") in dropdown menus. This made the configuration interface difficult to understand for site administrators. The fix was delivered as part of a larger migration to the latest FlowDrop UI version.

## Impact

No upgrade or configuration changes required.

## Technical details

This was a regression in the chatbot configuration forms where select lists were displaying machine names instead of labels. The affected fields included Theme and Region dropdowns in Block Placement settings, Style and Placement selects in Styling Settings, and Toggle State in Advanced Settings.

The issue was resolved by migrating to the latest version of FlowDrop UI as documented in [#3572310](https://www.drupal.org/project/flowdrop_ui_agents/issues/3572310). No separate patch was committed for this specific issue because the fix was incorporated into that broader migration work.

The root cause was that the form API elements were using raw machine names as option values and labels, rather than mapping machine names to their corresponding human-readable labels. The migration to the updated FlowDrop UI version corrected this by properly translating machine names to labels throughout the chatbot configuration interface.
