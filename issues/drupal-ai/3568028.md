## Summary

The AI module's provider settings page at `/admin/config/ai/settings` gains a new Info column in the capability table that shows documentation and pricing links for each AI provider and model. Provider and model links update dynamically via Ajax when a selection changes. Providers without metadata defined degrade gracefully. This is a UX improvement for site builders who need to compare AI providers before configuring them.

## Impact

Affects module maintainers.

- **New API:** New `resources/ai.provider_registry.yml` file in AI Core provides a static, YAML-based registry of provider metadata (documentation links, capabilities) consumed by the settings form.

## Technical details

This issue builds on the table layout introduced in [#3568026](https://www.drupal.org/node/3568026). The core addition is a new `resources/ai.provider_registry.yml` file in the AI Core module that acts as a static registry of provider metadata (documentation links, capabilities, pricing references) for providers maintained by the Drupal AI Initiative. A parallel YAML file was added for vector database providers.

The settings form was updated to read this YAML once per request (fixing an earlier regression where `file_get_contents` was called on every model selection, loading the file multiple times per page). The `buildEmbeddingProviderLinks` method was refactored to derive its link list from the YAML registry rather than hardcoding provider data in PHP, removing data duplication.

An empty `$form['default_providers']` array guard was added to prevent a white-screen-of-death (WSOD) when a third-party provider module (such as Groq) attempts to alter a form key that no longer exists after the table layout restructure.

Ajax error handling was improved: API key validation errors that occur when fetching models for a selected provider are now surfaced inline next to the model select element using Drupal's `MessageCommand` Ajax response, rather than via the global messenger (which requires a page reload to appear). A separate status-message check on form render prevents those same errors from duplicating in non-Ajax contexts.

New CSS library assets and icon files accompany the UI additions. Providers without a corresponding entry in the YAML registry are handled gracefully, leaving the Info cell empty without triggering errors.
