## Summary

The `drupal-devkit` tool now supports harness-native plugin installation for Codex CLI users. Prior to this fix, the `drupal-devkit harness-plugin install` command had no available backend for Codex users, even though Codex was listed as a supported skill-install harness in the README. A new `CodexPluginBackend` was added alongside the existing `ClaudeCodePluginBackend`, closing the gap between the two harnesses.

## Impact

Affects module maintainers.

- **New API:** New `CodexPluginBackend` added to `drupal_devkit` harness plugin registry, enabling `drupal-devkit harness-plugin install` for Codex CLI users

## Technical details

The `HarnessPluginBackend` Protocol in `src/drupal_devkit/harness_plugins/protocol.py` defines the contract that all backends must implement. The fix adds a `CodexPluginBackend` class implementing `is_available`, `install`, `uninstall`, `list_installed`, and `info`, and registers it in `harness_plugins/__init__.py` alongside the Claude Code backend. The plugin registry resolves backends by `harness` + `type` values from `default_marketplaces.json`, so the new backend also required documenting the Codex `type` string in the marketplace schema. Test coverage was added under `tests/harness_plugins/` to match the pattern established by the Claude Code backend tests.
