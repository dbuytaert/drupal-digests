## Summary

The `ai_validations` module gains a seventh field validation rule: **AI audio constraint**. Site builders can now attach this rule to a File field that accepts audio uploads. When an editor saves a node, the constraint first transcribes the uploaded audio through a configurable speech-to-text provider, then evaluates that transcript with a separate chat provider using the same `XTRUE`/`XFALSE` prompt protocol the existing text and image constraints already use. If the chat provider returns `XFALSE`, the node is rejected and the configured error message is shown.

## Impact

Affects module maintainers.

- **New API:** New `transcribeAudio()` and `buildProviderDefaultValue(string $configKey)` helpers on `AiConstraintValidatorBase` and `AiConstraintFieldValidationRuleBase`; `loadImageFile()` renamed to `loadFile()` on `AiConstraintValidatorBase`.

## Technical details

Three new files implement the feature: `AiAudioConstraintFieldValidationRule` (the `@FieldValidationRule` plugin with id `ai_audio_constraint_rule`), `AiAudioConstraint` (the Symfony `@Constraint` with id `AiAudioPrompt`), and `AiAudioConstraintValidator` (the validator). The validator resolves both provider options, calls `transcribeAudio()` to get a string, then passes it straight into `runBooleanChat()` to obtain a boolean verdict.

Two base-class changes support the new plugin and clean up the existing ones. In `AiConstraintFieldValidationRuleBase`, the private logic formerly in `buildModelDefaultValue()` was extracted into a new `buildProviderDefaultValue(string $configKey)` method. `buildModelDefaultValue()` now delegates to it, so subclasses that need a second provider selector (as `AiAudioConstraintFieldValidationRule` does for its `chat_provider` key) can call `buildProviderDefaultValue()` with any config key without duplicating the decode logic. In `AiConstraintValidatorBase`, `loadImageFile()` was renamed to `loadFile()` to reflect that it handles any file type, and a new `transcribeAudio(mixed $data, string $option)` helper was added alongside the existing `loadFile()` and `runBooleanChat()` helpers. All three existing image-constraint validators (`AiImageConstraintValidator`, `AiImageClassificationConstraintValidator`, `AiObjectDetectionConstraintValidator`) were updated to call `loadFile()` instead of `loadImageFile()`.

## Upgrade

Custom validators that extend `AiConstraintValidatorBase` and call `loadImageFile()` must rename the call to `loadFile()`:

```php
// Before
$file = $this->loadImageFile($data);

// After
$file = $this->loadFile($data);
```
