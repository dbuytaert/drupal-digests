# #3565750: Make it easier to drag and drop agents and tools onto nodes

## Summary

This issue improves the user experience for building AI agent workflows by making it easier to connect components through drag-and-drop. Previously, connecting tools to agents, agents to assistants, and chatbots to agents required clicking through multiple steps and manually drawing connection lines. Now users can simply drag components onto each other to automatically create connections. The system enforces logical rules about which components can attach to each other, making the interface more intuitive for site builders creating AI-powered features.

## Impact

No upgrade or configuration changes required.

## Technical details

This change affects the FlowDrop UI Agents module, which provides a visual workflow builder for AI agents. The enhancement implements automatic connection behavior when dragging workflow components onto each other in the canvas interface.

The connection rules implement a hierarchy: tools can attach to agents but not assistants, agents can attach to assistants and other agents, chatbots can only attach to agents, and assistants cannot attach to anything. Components dragged onto empty canvas space simply place themselves without creating connections. Tools already on the canvas can be dragged onto agents or assistants to create connections.

The fix was resolved during a migration to a newer version of the FlowDrop UI library as documented in [#3572310](https://www.drupal.org/project/flowdrop_ui_agents/issues/3572310). An initial implementation experienced a regression that was later addressed with an improved approach to the drag-and-drop attachment behavior.
