## Summary

This change makes it easier to build agent workflows by allowing you to drag and drop components directly onto each other instead of manually connecting them with lines. Tools can now snap directly to agents, agents to assistants, and chatbots to agents when you drag them onto the canvas. This streamlines the visual workflow builder and reduces the number of clicks needed to assemble agent configurations.

## Impact

No upgrade or configuration changes required.

## Technical details

This issue improves the drag-and-drop interaction model in the FlowDrop UI Agents visual workflow builder. The change implements automatic snapping and attachment when dragging workflow components onto compatible targets.

The interaction rules define a hierarchy: tools attach to agents but not assistants, agents attach to assistants and other agents, chatbots only attach to agents, and assistants cannot attach to anything. When a component is dragged onto an incompatible target or empty canvas space, it remains as a standalone node that can be repositioned later.

The implementation was completed as part of migrating to the latest version of FlowDrop UI in [#3572310](https://www.drupal.org/project/flowdrop_ui_agents/issues/3572310). The feature addresses a usability issue where creating workflow connections required multiple manual steps to draw connection lines between nodes. The new snap-to-attach behavior provides immediate visual feedback and reduces the interaction cost for building agent workflows.
