## Summary

The AI module now supports AI-assisted generation of Metatag values through the Field Widget Actions (FWA) framework when using the `metatag_firehose` sidebar widget. Site builders can configure an AI Automator that reads node content (title, body) and generates a set of metatags, then apply or preview the result via an action button directly in the Metatag sidebar UI, without having to save the full node first.

## Impact

Affects module maintainers.

- **New API:** New FWA plugin for the Metatag `metatag_firehose` widget enabling AI Automator action buttons in the Metatag sidebar UI (requires the Metatag module)

## Technical details

The challenge was that the `metatag_firehose` widget does not render inline with other form elements; it is placed in the sidebar via Drupal's `#group` property (typically the `advanced` vertical tab group). Standard FWA plugins attach their Ajax action buttons to the widget wrapper and return the re-rendered widget, but when the parent element uses `#group`, the Ajax response was returning an empty result because the render system did not include the widget inside the `advanced` element's context.

`a.dmitriiev` identified the `#group` rendering issue as the core blocker (comment [#16472723](https://www.drupal.org/node/16472723)) and implemented a new FWA plugin for Metatag alongside a companion fix in the standalone `field_widget_actions` project (merge request 12 at `https://git.drupalcode.org/project/field_widget_actions/-/merge_requests/12`) to handle elements that rely on `#group`. A documentation file with testing instructions was added in the same MR for this issue.

An additional edge case surfaced during review: when the advanced field configuration for a metatag automator is left incomplete, the AI provider returns an array of values rather than a string. The FWA plugin was written to accept both arrays and strings rather than strictly validating the shape, keeping the integration permissive for users who skip optional configuration.

The Metatag module is a hard requirement for this plugin; the plugin is guarded so it only activates when Metatag is present. `marcus_johansson` merged MR !1146 and forward-ported the change.
