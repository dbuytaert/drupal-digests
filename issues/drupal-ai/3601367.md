# #3601367: Fix the viewport issue

- **Project:** Drupal AI
- **Type:** Major bug report
- **Status:** Fixed
- **Discussion:** [#3601367](https://git.drupalcode.org/project/one_line_installer/-/work_items/3601367) · 1 contributor · 2 comments

## Summary

The `one_line_installer`'s interactive menus (`select_one` for single-choice and `select_many` for checkbox selections) would corrupt the terminal display whenever the option list was taller than the terminal window. Each keypress would shove the entire display further up the screen instead of moving just the cursor. This is now fixed: menus clamp themselves to a scrollable viewport sized to the terminal height, showing only a window of options at a time with "↑ N more" / "↓ N more" indicators when the list is clipped. A related wrapping bug is also fixed: labels longer than the terminal width would wrap onto a second physical line, throwing off the repaint's cursor-up jump; labels are now truncated with an ellipsis and terminal line-wrap is disabled for the menu's duration.

## Impact

No upgrade or configuration changes required.

## Technical details

Both menus repaint by issuing `\033[<block>A` to jump the cursor back up over the block they last drew, counting option rows. When the full option list exceeds the terminal height, printing the last row scrolls the buffer: the list no longer starts `<block>` rows above the cursor, so the jump lands on the wrong row and each subsequent repaint pushes the window further up. The fix is in `src/lib/core/_menu_viewport.sh`, which introduces five helpers used by both selectors.

`_menu_term_height` and `_menu_term_width` read terminal dimensions via `stty size 2>/dev/null </dev/tty`, falling back to 24 and 80 when there is no terminal. `_menu_viewport <n>` computes three globals: `MENU_VIEW` (option rows to paint), `MENU_CLIPPED` (1 when the list is clipped), and `MENU_OFFSET` (index of the first visible option). Available rows are `terminal_height - MENU_CHROME_LINES` where `MENU_CHROME_LINES=3` accounts for the prompt, the key-hint line, and one spare row so the block never ends on the very last screen line. When clipped, `MENU_VIEW` is reduced by 2 to leave room for the indicator lines, which are part of the counted block. `_menu_scroll <cur> <n>` is called before every repaint and slides `MENU_OFFSET` so the highlighted row stays inside `[MENU_OFFSET, MENU_OFFSET + MENU_VIEW)`. `_menu_more_line` draws the "↑/↓ N more" line, printing a blank-but-cleared line when nothing is hidden in that direction, so the block height remains constant and the cursor-up jump stays exact.

The second bug — a label wider than the terminal wrapping onto a second physical row — is addressed by `_menu_fit <text> <reserve>`, which truncates the label to `_menu_term_width` minus the prefix width and appends `…`. Both `select_one` and `select_many` additionally disable terminal line-wrap with `\033[?7l` for the duration of the menu and restore it with `\033[?7h` on exit, as a second safeguard. The skills menu was the concrete trigger: its option labels ran up to 111 columns on an 80-column terminal.
