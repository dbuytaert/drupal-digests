## Summary

The AI LiteLLM provider module now supports the rerank operation, which lets AI models rearrange search results or documents based on their relevance to a query. Reranking is a popular AI capability that helps improve search quality by using machine learning to score and reorder results. This feature works with AI providers like Together AI that offer reranking models through the LiteLLM proxy. The implementation demonstrates the new rerank operation introduced in the parent AI module.

## Impact

No upgrade or configuration changes required.

## Technical details

This change adds rerank operation support to the AI LiteLLM provider module by implementing the `ReRankInterface` from the AI module. The implementation adds a `rerank()` method that sends requests to the LiteLLM proxy's `/rerank` endpoint, which follows Cohere's rerank API structure.

The `Model` DTO was updated to include a `$supportsRerank` property that checks if the model's mode is "rerank". The `getModels()` method now filters models for the "rerank" operation type, checking both the capability flag and whether the model name contains "rerank" to handle cases where capabilities may not be correctly reported.

The rerank payload includes the model ID, query string, input documents, and optional `top_n` parameter to limit results. The API response is mapped into a `ReRankOutput` object containing reranked documents with scores and metadata. If the API only returns indices, the original document text is re-hydrated from the input.

The module's dependency was updated from AI module 1.2.0 to 1.3.0-rc2, which includes the base rerank operation support. Project-specific spelling words were added for code quality checks.
