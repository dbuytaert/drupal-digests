## Summary

The LiteLLM provider now supports the Rerank operation, which helps improve search results by reordering documents based on relevance to a query. This operation is commonly used in search and retrieval systems to refine initial results using AI models. While reranking is not part of the standard OpenAI API, it is widely supported by AI platforms through LiteLLM's proxy. Sites using LiteLLM can now leverage reranking models to enhance their search and content discovery features.

## Impact

Affects module maintainers.

- **New API:** New `rerank()` method available in LiteLLM provider for document reranking operations.

## Technical details

This change implements the `ReRankInterface` in the `LiteLlmAiProvider` plugin class, adding a new `rerank()` method that follows Cohere's rerank API structure. The implementation sends queries and document lists to LiteLLM's `/rerank` endpoint via HTTP POST, since the OpenAI SDK does not natively support reranking operations.

The model discovery logic was enhanced to detect rerank-capable models by checking both the model's reported capabilities (`supportsRerank` property) and by examining model names that contain "rerank" as a fallback. This dual approach ensures compatibility even when models don't report their capabilities correctly.

The rerank payload includes the model ID, query string, document array, and optional `top_n` parameter for limiting results. The response is parsed and transformed into a `ReRankOutput` object containing reranked documents with relevance scores, result IDs, and metadata. If the API returns only document indices, the implementation rehydrates the results with original text from the input.

Error handling follows the existing pattern using the provider's `handleApiException()` method. The composer dependency was updated to require AI module version 1.3.0-rc2 or higher, which includes the Rerank operation interface.

## Upgrade

No code changes are required for existing implementations. The rerank functionality is available as a new operation type for sites that wish to use it. To use reranking:

1. Ensure the AI module is version 1.3.0-rc2 or higher
2. Configure a LiteLLM provider pointing to a proxy with rerank-capable models
3. Use the Rerank operation through the AI module's standard interfaces

Example models that support reranking include Together AI's mxbai-rerank-large-v2 and similar models available through LiteLLM.
