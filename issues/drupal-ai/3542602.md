## Summary

The "LLM: Entity reference" AI automator type can now reuse existing entities instead of always creating new ones. When configuring the automator, editors can tick a new "Use existing entity" checkbox beneath each enabled field. If all checked fields match an existing entity in the database, that entity is referenced rather than a duplicate being created. If no match is found, the automator falls back to creating a new entity as before.

## Impact

No upgrade or configuration changes required.

## Technical details

The change lives entirely in `modules/ai_automators/src/PluginBaseClasses/EntityReference.php`, which is the abstract base class for the "LLM: Entity reference" automator plugin.

**Form layer.** `extraFormFields()` now renders an `automator_entity_field_use_existing_{field}` checkbox for every enabled target-bundle field, alongside the existing enable/generate pair. The checkbox is shown only when its corresponding `automator_entity_field_enable_{field}` checkbox is checked (via `#states`). The config key is saved with the rest of the automator configuration.

**Storage layer.** `storeValues()` was extended to perform an entity lookup before deciding whether to create. For each LLM-generated value set, the method:
1. Builds an entity query filtered to the configured bundle.
2. Iterates the generated field values; for any field where `automator_config['entity_field_use_existing_{field}']` is non-empty, it adds a `condition()` to the query and sets a `$hasMatchFields` flag.
3. If at least one match field is configured and the query returns results, the found entity IDs are collected and `$useExisting` is set to true.
4. When `$useExisting` is true the existing IDs are appended to `$targets`; otherwise the original entity-creation path runs unchanged.

The lookup uses AND semantics: all fields marked "use existing" must match simultaneously. This was an explicit design decision raised by `marcus_johansson` during DrupalCon review; the initial patch used ANY (OR) matching and was reworked to AND before merge.
