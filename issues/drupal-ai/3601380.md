# #3601380: Separate composer and recipe

- **Project:** Drupal AI
- **Type:** Major unknown (0)
- **Status:** Fixed
- **Discussion:** [#3601380](https://git.drupalcode.org/project/one_line_installer/-/work_items/3601380) · 1 contributor · 2 comments

## Summary

The `recipes` key in `.drupalaibp.json` no longer fetches packages from Composer. It is now apply-only: the installer runs `drush recipe` for each entry but never calls `composer require`. Contrib recipes (e.g. `drupal/drupal_cms_blog`) must be listed in both `composer_projects` (to fetch) and `recipes` (to apply). Core or already-present recipes (e.g. `core/recipes/standard`) still work in `recipes` alone because no fetch is needed. A new boolean key `composer_with_all_dependencies` passes `--with-all-dependencies` (`-W`) to the `composer_projects` require step for packages that pull in a broader dependency tree.

## Impact

Affects site owners.

- **Config change:** The `recipes` key is now apply-only; contrib recipes must also be listed in `composer_projects` for the fetch step. New `composer_with_all_dependencies` boolean key adds `--with-all-dependencies` to the `composer_projects` require call.

## Technical details

Previously `step_provision_from_config` handled the `RECIPES` array by calling `composer require` followed by `drush recipe` in one loop, conflating fetch and apply. The refactor splits them into two strictly ordered phases: `COMPOSER_PROJECTS` are `composer require --no-interaction`'d (no apply); `RECIPES` are `drush recipe`'d (no fetch). A `drupal/<name>` entry in `RECIPES` maps to `../recipes/${name#drupal/}` because `ddev drush` runs from the `web/` docroot and Composer installs contrib recipes one level up in `recipes/`; any other recipe locator (e.g. `core/recipes/standard`) is passed to drush verbatim. The `COMPOSER_WITH_ALL_DEPS` global, loaded from the new `composer_with_all_dependencies` config key, appends `--with-all-dependencies` to the single `composer require` call when a recipe or package requires a broader dependency-tree update. The `provision_from_config.sh` source file and the AGENTS.md, README, and `docs/drupalaibp-json.md` documentation were all updated to reflect the separation.

## Upgrade

Any `.drupalaibp.json` that listed a contrib recipe solely in `recipes` must be updated to also list it in `composer_projects`. Core or built-in recipes that need no download are unaffected.

Before:
```json
{
  "recipes": ["drupal/drupal_cms_blog"]
}
```

After:
```json
{
  "composer_projects": ["drupal/drupal_cms_blog"],
  "composer_with_all_dependencies": true,
  "recipes": ["drupal/drupal_cms_blog"]
}
```

`composer_with_all_dependencies` is optional; set it to `true` when the recipe pulls in packages that require their shared dependencies to be updated together.
