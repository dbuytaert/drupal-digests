## Summary

Chat messages with attached images can now be properly serialized and restored. Previously, attempting to convert a chat message with an image to an array and back would fail because the system tried to store binary image data directly. The module now uses base64 encoding for image serialization, allowing chat conversations with images to be saved, cached, or transmitted reliably. This fix enables features like storing chat history with images in databases or passing image-containing messages between systems.

## Impact

No upgrade or configuration changes required.

## Technical details

The `AbstractFileBase::toArray()` method previously stored image data using the `binary` key, which contained raw binary data. This made serialization fragile and incompatible with many storage backends. The fix changes the serialization to use base64 encoding instead.

In `AbstractFileBase::toArray()`, the method now returns `base64` instead of `binary`, calling `getAsBase64EncodedString()` which produces a data URL format like `data:image/png;base64,<encoded_data>`. The corresponding `fromArray()` method now looks for the `base64` key and calls the new `setFromBase64EncodedString()` method.

The `GenerateBase64Trait` gained a new `setFromBase64EncodedString()` method that handles both plain base64 strings and data URL formats. It uses a regex to detect and parse the data URL scheme, extracting the MIME type automatically before decoding the base64 string back to binary.

The `ChatMessage::toArray()` method was updated to call `toArray()` on each file object instead of `getBinary()`, ensuring the full serialization chain uses base64 encoding.

Comprehensive unit tests verify round-trip serialization for single images, multiple images, and full chat input objects with images.
