## Summary

The AI module now correctly serializes chat messages that contain images. Previously, attempting to convert a `ChatInput` with attached images to an array and back would fail because the system tried to store image data as raw binary. This made it impossible to cache, queue, or otherwise persist conversations that included images. The fix switches to base64 encoding for image serialization, enabling round-trip conversion while preserving all image data including MIME type and filename.

## Impact

No upgrade or configuration changes required.

## Technical details

The `AbstractFileBase::toArray()` method previously stored image binary data directly under a `binary` key. This caused serialization failures when `ChatMessage` objects with attached images were converted to arrays for storage or transmission.

The fix changes the serialization format to use base64 encoding instead. The `toArray()` method now stores images under a `base64` key using the existing `getAsBase64EncodedString()` method, which generates data URL format strings like `data:image/png;base64,<encoded_data>`.

A new `setFromBase64EncodedString()` method in `GenerateBase64Trait` handles deserialization. It extracts the MIME type from data URL schemes if present, strips the prefix, and decodes the base64 string back to binary. The `fromArray()` method now calls this new method instead of `setBinary()`.

The `ChatMessage::toArray()` method was updated to call `toArray()` on each file object rather than `getBinary()`, ensuring the base64 serialization format is used throughout the stack.

Five comprehensive unit tests verify round-trip serialization for single images, multiple images, and complete `ChatInput` objects with image messages.
