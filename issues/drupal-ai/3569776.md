## Technical details

This issue migrated the Context Control Center (CCC) module from custom UI components to the shared CSS library provided by AI Core. The work included replacing custom tooltip implementation with the new AI Tooltip component using the native Popover API, adopting shared `.ai-pill` and `.ai-description` classes, and standardizing icon usage with AI Core's SVG library.

A security review identified XSS vulnerabilities where user input was rendered without escaping. The fix adds `Html::escape()` calls throughout `AiContextScopeManager`, form classes, and templates to sanitize labels, descriptions, and plugin names before rendering.

The pill component received accessibility improvements: the "light" variant now uses a light blue background with dark text to meet WCAG contrast requirements, replacing the previous light grey that failed accessibility checks. Pills are limited to 17 characters width with tooltips showing full text on longer values.

The external link icon switched from a Unicode character to an SVG mask from AI Core's shared assets, ensuring consistent rendering. CSS cleanup removed duplicate styles now handled by the shared library. The diff shows proper escaping in multiple locations including scope summaries, agent forms, and overview templates.

## Lessons learned

- When rendering user-provided data in markup, escape all output to prevent XSS vulnerabilities. Use Drupal's render API with proper escaping mechanisms rather than raw #markup with unescaped strings. *(via kristen pol)*
- When referencing assets from external modules, avoid hardcoded absolute paths. Use dynamic path resolution so the code remains portable across different installations and survives module reorganization. *(via kristen pol)*
- When implementing UI updates that depend on unreleased upstream changes, document the dependency clearly and provide instructions for reviewers to test against the correct branch. This prevents confusion and enables parallel development. *(via bbruno)*
