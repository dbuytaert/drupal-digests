## Summary

The `ai_best_practices` module renames its AI agent skill directories from gerund-prefixed names to a `drupal-<task-or-topic>` kebab-case pattern. `writing-automated-tests` becomes `drupal-automated-testing` and `how-to-write-documentation` becomes `drupal-writing-documentation`. Any project that references these skill file paths in an `AGENTS.md`, `CLAUDE.md`, or automation script must update those references after upgrading the package.

## Impact

Affects module maintainers.

- **API change:** Skill directory names changed: `writing-automated-tests` → `drupal-automated-testing`, `how-to-write-documentation` → `drupal-writing-documentation`; all `vendor/` paths shift accordingly.

## Technical details

The previous convention used action-gerund prefixes (`writing-`, `how-to-`), which an analysis found to be out of step with broader skill-naming practice. The replacement pattern is `<optional-domain>-<task-or-topic>` in kebab-case. Within this repository, the `drupal-` domain prefix is mandatory for every skill.

The two shipped skills were renamed: `skills/writing-automated-tests/` → `skills/drupal-automated-testing/` and `skills/how-to-write-documentation/` → `skills/drupal-writing-documentation/`. The `name` field in each skill's YAML frontmatter must match the directory name exactly, as the eval runner derives paths from it: `skills/drupal-automated-testing/SKILL.md` corresponds to `evals/drupal-automated-testing/`. All eval JSON configs (`evals.json`, `static-checks.json`), eval `README.md` files, the `evals/compare.py` script, and the repository's contributing-guidance document were updated to reflect the new names. The contributing docs also gained explicit prose documenting the pattern and a note that upgrading the package changes paths under `vendor/`.

## Upgrade

Update any `AGENTS.md`, `CLAUDE.md`, or automation that references old skill paths:

```
# Old
./vendor/drupal/ai_best_practices/skills/writing-automated-tests/SKILL.md
./vendor/drupal/ai_best_practices/skills/how-to-write-documentation/SKILL.md

# New
./vendor/drupal/ai_best_practices/skills/drupal-automated-testing/SKILL.md
./vendor/drupal/ai_best_practices/skills/drupal-writing-documentation/SKILL.md
```

Eval runner commands also change accordingly:

```bash
# Old
python3 evals/run-evals.py --behavioral --skill writing-automated-tests

# New
python3 evals/run-evals.py --behavioral --skill drupal-automated-testing
```
