## Summary

The AI Logs view now displays log entries in the order they were created, fixing a problem where multiple log entries from the same agent request appeared out of order. Previously, the view sorted by creation timestamp, but when multiple logs were created in quick succession with identical timestamps, they appeared in unpredictable order. The view now sorts by log ID instead, ensuring entries always appear in the correct sequence.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI Logs view (`views.view.ai_logs`) was configured to sort by the `created` field, which caused ordering issues when multiple log entries were generated with identical timestamps during a single agent request. The view configuration has been updated to sort by the `id` field in descending order instead.

The change modifies `modules/ai_logging/config/install/views.view.ai_logs.yml` to replace the `created` sort field with `id` in the default display options. The sort uses the `standard` plugin instead of the `date` plugin, and maintains descending order (DESC) so newest entries appear first.

A new update hook `ai_logging_update_10306()` ensures existing installations receive this fix. The update function loads the view configuration, checks if the old `created` sort exists, removes it, and adds the new `id` sort with proper configuration including entity type, field, and plugin ID settings.
