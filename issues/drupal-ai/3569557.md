## Technical details

This change adds a new documentation file `docs/contribute/issue_review_guidelines.md` to the AI Initiative project. The file provides structured guidance for issue reviewers at multiple levels.

The "Before You Start" section directs reviewers to existing Drupal.org documentation on issue etiquette and emphasizes reading the issue summary and recent comments for context. The "Quick Review Checklist" covers three essential verification points: issue summary completeness, scope control to prevent scope creep, and AI usage disclosure.

The documentation links to comprehensive external resources including the full merge request review guide and maintainer response templates. The most practical addition is a review comment template that structures feedback into three parts: thanking the contributor with specific positive feedback, documenting exactly what was reviewed using a checklist format, and providing specific findings or approval with actionable next steps.

The template also clarifies when to set each issue status: "Needs Work" when issues are found, "Needs Review" for partial reviews or after changes, and "RTBC" (Reviewed and Tested by the Community) when everything passes a complete review. This standardization helps both new and experienced reviewers provide consistent, constructive feedback.
