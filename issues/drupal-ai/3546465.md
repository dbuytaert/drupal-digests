## Summary

Sites using the Chosen module's enhanced select widget for taxonomy fields can now benefit from AI-powered classification automation. The `ClassificationOptionsSelect` Field Widget Action in the AI Automators module previously only supported the standard `options_select` widget. It now also supports `chosen_select`, so content editors on sites that use Chosen for large vocabulary selection will see the AI classification button on those fields.

## Impact

No upgrade or configuration changes required.

## Technical details

The `ClassificationOptionsSelect` plugin lives in `modules/ai_automators/src/Plugin/FieldWidgetAction/ClassificationOptionsSelect.php` and is declared via a PHP attribute `#[FieldWidgetAction]`. Before this change its `widget_types` array contained only `['options_select']`, so the plugin was invisible to any form field using the Chosen module's `chosen_select` widget, even though both widgets render a select element for `entity_reference` fields targeting the same `target_id` form property.

Three changes were made to the attribute in the same file:

1. `widget_types` was expanded from `['options_select']` to `['options_select', 'chosen_select']`, making the plugin available to both widget types.
2. `multiple: FALSE` was added explicitly. During testing, ezeedub discovered that the default `multiple: TRUE` prevented the automator button from appearing at all on select widgets. Select widgets require the button to be placed on the complete form element rather than on individual sub-elements; setting `multiple: FALSE` ensures `actionButton()` is invoked through `completeFormAlter()` rather than the per-item path.
3. The plugin label was updated from `'Classification (Options Select)'` to `'Classification (Options/Chosen Select)'` to reflect the broader widget coverage.

The identical diff was committed to both the `1.3.x` and `2.0.x` branches of the `ai` project.
