## Summary

The AI Agents Test module now automatically discovers and displays all available parameter test plugins when creating agent tests. Previously, the interactive test form hardcoded a fixed list of parameter validation rules, meaning custom plugins couldn't be used even though the system supported them. This change makes the module properly extensible, allowing developers to create custom parameter validation plugins that immediately appear in the testing interface.

## Impact

No upgrade or configuration changes required.

## Technical details

The `InteractiveAgentTestForm` class was building parameter test options using a hardcoded array of ten validation rules (empty, isset, not_set, equals, etc.), even though `AgentsParameterTest` items were already implemented as plugins with a discovery system. This meant custom plugins existed but couldn't be selected in the UI.

The fix injects the `AgentsParameterTestPluginManager` service into the form and replaces the hardcoded options array with a new `getParameterTestOptions()` method. This method queries the plugin manager for all available definitions and builds the options list dynamically from their labels, preserving the 'empty' (No rules) option as the first entry.

The form states logic for showing/hiding the value field was also refactored. Instead of listing all checks that should show the field, it now uses an inverse approach, hiding the field only for 'empty', 'isset', and 'not_set'. This makes the visibility rules work automatically for new plugins without requiring code changes.
