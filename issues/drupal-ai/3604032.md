# #3604032: Add static_list completion providers to example prompt argument configs and cover them with tests

- **Project:** Drupal AI

## Summary

The `mcp_server_examples` module now ships example prompt argument configurations that expose `static_list` completion providers, enabling MCP clients to autocomplete valid values for arguments such as `skill_level`, `court_surface`, `focus_area`, and others. Previously these arguments described their valid values only in prose; they now carry machine-readable completion data that MCP clients can query. Functional test coverage was added at the same time to verify each configured argument through the `completion/complete` endpoint.

## Impact

No upgrade or configuration changes required.

## Technical details

Each affected prompt config YAML received one or more `completion_providers` blocks using `plugin_id: static_list`, listing the exact accepted values. The affected arguments span six prompt configs: `comprehensive_lesson.lesson_focus`, `match_strategy.opponent_type` and `court_surface`, `multi_provider_prompt.opponent_type` (which was also missing three of its four values), `tennis_resources.topic`, `tennis_tip_basic.skill_level` and `focus_area`, and `test_with_args.skill_level` and `surface`. No schema changes were needed because `mcp_server.mcp_prompt_config.*` already declared `completion_providers` as a supported key.

The new `assertPromptArgumentCompletions()` method in `McpServerExamplesFunctionalTest` iterates over nine argument cases, calls the `completion/complete` MCP endpoint with an empty argument value, and asserts that `completion.values`, `completion.total`, and `completion.hasMore` all match the configured static list. The method is called from the existing `testMcpServerExamples()` entry point, so it runs as part of the standard functional test suite.
