## Summary

The AI Validations module now supports validating uploaded images based on what objects appear in them. You can configure validation rules to require or forbid specific objects like "person," "backpack," or "apple" in images. This uses object detection AI models that run quickly, often in milliseconds, and can work on standard webserver hardware or free services like Huggingface. You can combine multiple object requirements with AND or OR logic, set minimum confidence thresholds, and choose whether validation passes when objects are found or when they are missing. This gives site owners fast, automated content moderation based on image content.

## Impact

No upgrade or configuration changes required.

## Technical details

This feature adds a fourth Field Validation plugin to the AI Validations module: `AiObjectDetectionConstraintFieldValidationRule`. It integrates with the AI module's object detection operation introduced in [#3570609](https://www.drupal.org/node/3570609).

The plugin provides a configuration form where administrators select an AI provider and model supporting object detection, enter keywords for object labels to detect, choose a matching strategy (exact, contains, or case-insensitive substring), and set a minimum confidence threshold. The keyword filter supports AND (all keywords must match) or OR (any keyword must match) logic. The rule mode determines whether validation passes or fails when objects are found.

The validator (`AiObjectDetectionConstraintValidator`) loads the uploaded file, creates an `ObjectDetectionInput` with the image, and calls the AI provider's `objectDetection()` method. It filters detections by confidence score, matches labels against configured keywords using the selected finder strategy, and applies the keyword filter logic. Based on the rule mode (approve or disapprove when found), it adds a constraint violation if the condition is met. When the model is unavailable, the validator can skip validation or fail based on configuration.

The `composer.json` now requires `drupal/ai` version `^1.3.x-dev` to ensure object detection support is available.
