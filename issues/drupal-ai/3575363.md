## Summary

The AI Validations module now includes object detection validation for image fields. Site owners can configure automated checks that require or forbid specific objects in uploaded images, like requiring a person in profile photos or preventing images with weapons. The validation runs in milliseconds using the AI module's object detection feature, which can analyze images locally on web servers or through free services like Huggingface. This provides a faster and more cost-effective alternative to full vision AI for common content moderation scenarios.

## Impact

No upgrade or configuration changes required.

## Technical details

The new `AiObjectDetectionConstraintFieldValidationRule` plugin adds a fourth validation type to the AI Validations module. It uses the AI module's object detection operation to identify common objects in uploaded images and validate them against configured rules.

The plugin configuration includes a model selector for object detection providers, a list of keywords to match against detected object labels, a finder mode (exact, contains, or case-insensitive substring), and a minimum confidence threshold. Site builders can choose AND or OR logic for multiple keywords and decide whether to approve or disapprove when objects are found.

The validator (`AiObjectDetectionConstraintValidator`) loads the file entity, creates an `ObjectDetectionInput` with the image, and calls the provider's `objectDetection()` method. It then filters detections by confidence score and compares labels against keywords using the configured finder mode. Based on the keyword filter (AND/OR) and rule mode (approve/disapprove when found), it determines whether to add a validation violation.

The module's `composer.json` now requires `drupal/ai: ^1.3.x-dev` to ensure the object detection operation is available.
