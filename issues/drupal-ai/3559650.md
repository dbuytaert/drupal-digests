## Summary

The AI Chatbot module now lets site administrators customize the message displayed while the chatbot is processing a response. Instead of just showing an animated ellipsis ("..."), you can configure a more helpful message like "Give me a moment to find the best answer..." This improves the user experience by providing more context and reassurance while visitors wait for AI responses. The feature only works when Verbose Mode is disabled, and leaving the field blank preserves the original ellipsis behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

The `DeepChatFormBlock` block plugin now includes a `loading_message` configuration field that administrators can set in the block settings under Message Settings. This field has a form state that automatically disables it when Verbose Mode is checked, since verbose mode has its own loading feedback mechanism.

The loading message is passed to the frontend through `drupalSettings.ai_deepchat.loading_message`. In `deepchat-init.js`, when a message is sent and Verbose Mode is disabled, the code checks if a custom loading message exists and displays it using DeepChat's `addMessage()` method with a temporary HTML wrapper using the `deep-chat-temporary-message` class.

The toolbar style sheet (`toolbar.yml`) was updated to apply the same avatar loading animation styles to both the existing `.loading-wrapper` selector and the new `.deep-chat-temporary-message` selector. This ensures the spinning avatar animation appears consistently alongside custom loading text. Minor padding adjustments were made to ensure proper vertical alignment of the message.

Configuration schema was updated to include the `loading_message` field as a text type. The field defaults to an empty string, which preserves backward compatibility with the original ellipsis behavior when no custom message is configured.
