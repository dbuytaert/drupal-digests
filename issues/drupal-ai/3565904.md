# #3565904: When loading an Assistant for the first time, it jumps to the right

- **Discussion:** [#3565904](https://www.drupal.org/node/3565904) · 4 contributors · 8 comments

## Summary

A bug caused the Drupal CMS Assistant interface element to appear incorrectly positioned on the right side of the screen when first loaded after a cache clear or hard refresh. A page reload would sometimes correct the position, but the behavior was inconsistent across environments. This positioning issue affected the initial user experience when interacting with the AI assistant feature.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue was fixed as part of migrating to a newer version of the FlowDrop UI components in [#3572310](https://www.drupal.org/project/flowdrop_ui_agents/issues/3572310). The positioning bug occurred during the initial render of the assistant interface, before stylesheets or JavaScript fully initialized the component's layout. The problem was environment-dependent, with different behavior on local development versus hosted environments like DrupalForge, suggesting a timing issue with asset loading or component initialization. The upgrade to the latest FlowDrop UI resolved the underlying rendering issue.
