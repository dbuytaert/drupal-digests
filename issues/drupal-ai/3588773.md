## Summary

The `canvas_ai_migrations` menu-builder agent now uses a hybrid pattern for menu operations: it reads existing menu state via the Acquia Source REST API before each write, but still performs writes through browser automation. This makes Phase 6 menu-building re-runs safe, because the agent checks whether each `(label, url)` tuple already exists via `GET /api/menu_items/<menu>` and skips items that are already present. The change also deduplicates items within a single run. Full API-first writes remain blocked because the `menu:administer` OAuth scope exposes only read endpoints on the current Acquia Source release (POST/PATCH return HTTP 405).

## Impact

No upgrade or configuration changes required.

## Technical details

An empirical probe on 2026-05-01 confirmed that `GET /api/menu_items/main`, `/footer`, and `/social-media` all return HTTP 200 with current menu data, but `POST /api/menu_items/main` returns HTTP 405 (`Allow: GET, HEAD`) and `POST /api/menu_link_content/menu_link_content` returns HTTP 404. The `menu:administer` OAuth scope successfully mints tokens but grants only read access on this release.

The fix adds a `## HYBRID PATTERN — read-API, write-browser` section to `.claude/agents/menu-builder.md`. The section specifies three behaviors: (1) an idempotency check via `GET /api/menu_items/<menu>` before every write, skipping items matched by `(label, url)` tuple; (2) in-memory deduplication of tuples across the current Phase 6 dispatch; (3) writes still go through browser automation against `/admin/structure/menu/manage/<menu>` via `scripts/create-menus.sh` (see issue [#3588768](https://www.drupal.org/node/3588768)). The documentation instructs the agent to re-probe POST support via the Phase 0.5 preflight (Sprint 1 [#3588763](https://www.drupal.org/node/3588763)) on each new Acquia Source release.

A second commit adds a Vitest prose-sanity suite (`test/sprint2-3588773-hybrid-menu.test.js`) that asserts the `HYBRID PATTERN` block is present in `menu-builder.md`, names all three menu bundle endpoints, references the idempotency tuple, confirms browser writes, and checks the 2026-05-11 verification date. The tests are intentionally written against the prose so that future drift in the agent file causes a visible CI failure.
