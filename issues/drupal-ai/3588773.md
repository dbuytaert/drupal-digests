## Summary

The `canvas_ai_migrations` menu-builder agent (Phase 6) now uses a hybrid approach for menu creation: it reads existing menu state via the Acquia Source OAuth read API before performing any writes, making Phase 6 re-runs idempotent. Writes still go through browser automation against the Drupal admin UI because the upstream `menu:administer` write endpoints are not yet available. A vitest prose-sanity test suite was added to lock in this documented pattern.

## Impact

No upgrade or configuration changes required.

## Technical details

An empirical probe on 2026-05-01 confirmed that the `menu:administer` OAuth scope mints valid tokens and `GET /api/menu_items/<menu>` returns HTTP 200, but `POST /api/menu_items/main` returns 405 (`Allow: GET, HEAD`) and `POST /api/menu_link_content/menu_link_content` returns 404. Write endpoints are not shipped on the current Acquia Source release, so a full API-first replacement of browser automation remains blocked.

The resolution codifies a hybrid pattern in `.claude/agents/menu-builder.md` under a new `HYBRID PATTERN` section. Before each write, the agent calls `GET /api/menu_items/<menu>` and skips any item whose `(label, url)` tuple is already present in the response. An in-memory set tracks the same tuples within a single Phase 6 dispatch to prevent duplicates even when `menu-map.json` lists an item under multiple menus. Actual writes still go through `scripts/create-menus.sh` browser automation against `/admin/structure/menu/manage/<menu>` (introduced in [#3588768](https://www.drupal.org/node/3588768)). The hybrid does not replace the deterministic script; it adds an idempotency lookup that the script performs before each write.

A companion Vitest file (`2b12f91a`) adds five prose-sanity assertions that verify the `HYBRID PATTERN` block is present in `menu-builder.md`, that all three menu bundle endpoints are named, that the `(label, url)` deduplication tuple is documented, that browser automation is still referenced for writes, and that the 2026-05-11 verification date is recorded. The Phase 0.5 preflight (Sprint 1 [#3588763](https://www.drupal.org/node/3588763)) is expected to re-probe POST support on each new Acquia Source release; when write endpoints land, the hybrid prose will be rewritten to API-first.
