## Summary

When using AI to generate multiple FAQ items from text, users previously had to click a separate button for each individual FAQ entry, resulting in multiple AI inference jobs even though the AI generated all answers at once. The FAQ field automation now uses a single button that fetches and populates all FAQ items in one operation, saving time and reducing unnecessary API calls.

## Impact

No upgrade or configuration changes required.

## Technical details

The `FaqField` Field Widget Action plugin now sets `multiple: FALSE` in its annotation to create a single button for the entire field rather than one button per item. The fix introduces a new `runAutomatorSubmit()` method that acts as a submit handler before the AJAX callback.

This submit handler runs the automator once for the entire field, counts the resulting items, and updates the widget state by calling `WidgetBase::setWidgetState()` to increase `items_count` to match. It also populates `$form_state` user input with all generated FAQ values so the form rebuild creates the correct number of delta slots with proper subfield structure.

The `aiAutomatorsAjax()` method was simplified to just return the field element and clear the rebuild flag. The old `saveFormValues()` method that manually populated form element values was removed entirely, replaced by the state-based approach that lets Drupal's form API handle the rebuild correctly. This ensures the cached form structure matches the automator output so subsequent Save operations work properly.
