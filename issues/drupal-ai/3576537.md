# #3576537: FAQ Field FWA should fetch multiple

## Summary

The FAQ Field integration with Field Widget Actions previously created a separate button for each FAQ item when using AI automators. This meant users had to click multiple times to generate multiple FAQs, triggering separate AI inference jobs for each item even though the AI could generate all FAQs in one request. The Field Widget Action now wraps the entire FAQ field with a single button that fetches all requested FAQ items at once, reducing both user clicks and AI processing overhead.

## Impact

No upgrade or configuration changes required.

## Technical details

The FAQ Field widget action was configured to operate at the individual item level rather than the field level. When using the "LLM: FAQ From Text" automator, this caused the Field Widget Action button to appear once per FAQ item delta. Each button click would trigger a separate AI inference request, even though the prompt was designed to generate multiple FAQs simultaneously.

The fix moves the Field Widget Action trigger to wrap the entire field widget instead of individual items. This required handling multivalue field state properly so the widget knows how many items to populate when the AI returns multiple FAQ pairs. The solution aligns with how other multivalue fields work in the Field Widget Actions system, where a single action can populate multiple field deltas based on the AI response.

The change ensures that clicking the action button once generates all requested FAQs in a single AI call and populates them into the appropriate number of field items automatically.
