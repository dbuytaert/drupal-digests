## Summary

The FAQ Field integration with AI Automators previously created a separate button for each FAQ item in a multi-value field. This meant users had to click multiple times and trigger separate AI inference jobs for each item, even though they wanted to generate all FAQs at once from the same source text. The field widget action now places a single button that generates all FAQ items in one operation, populating the entire field with multiple questions and answers from a single AI call.

## Impact

No upgrade or configuration changes required.

## Technical details

The `FaqField` plugin was treating the FAQ field as if each item needed separate processing. The fix adds `multiple: FALSE` to the plugin annotation to indicate the button should wrap the entire field rather than each delta.

The plugin now implements a dedicated `runAutomatorSubmit()` submit handler that runs before the AJAX callback. This handler runs the automator once for the entire field, counts the returned FAQ items, and updates the form state accordingly. It manipulates the user input array to include all generated items and uses `WidgetBase::getWidgetState()` and `WidgetBase::setWidgetState()` to update the `items_count`, ensuring the form rebuilds with enough delta slots to display all generated FAQs.

The `aiAutomatorsAjax()` method was simplified to just return the field element after clearing the rebuild flag. The old `saveFormValues()` method that manually set form element values was replaced with the state-based approach, which properly integrates with Drupal's form API and ensures the cached form structure matches the actual data.
