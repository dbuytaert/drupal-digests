## Summary

The AI module updated its code to avoid using a deprecated Symfony method that was flagged by PHPStan static analysis. The old shortcut method `Request::get()` has been replaced with explicit calls to check query parameters and request body parameters separately. This keeps the module compatible with newer versions of Symfony while maintaining the same functionality. The change is purely internal and does not affect how the module works for site owners or other developers.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI module was using `Request::get()` from Symfony's HTTP Foundation component, which was deprecated in Symfony 7.4. PHPStan static analysis flagged two occurrences in `AiPromptForm` and `AiSetupForm`.

The fix replaces `$request->get()` calls with explicit checks of `$request->query->get()` and `$request->request->get()`. In `AiPromptForm`, the code now calls `$request->query->all('prompt_types')` first, then falls back to `$request->request->all('prompt_types')` if empty. In `AiSetupForm::getStage()`, the code checks `$request->query->get('stage')` first, then `$request->request->get('stage')` as a fallback.

Both changes include a `@todo` comment referencing [#3555532](https://www.drupal.org/project/drupal/issues/3555532), which tracks a broader solution in Drupal core for handling this Symfony deprecation. The fix maintains existing behavior while eliminating the deprecation warnings from PHPStan.
