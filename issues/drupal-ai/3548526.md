## Summary

The OpenAI provider module for Drupal now supports the OpenAI Files API through the normalized AI module file-handling interface. Developers can upload files to OpenAI, attach them by remote ID to chat messages, download file content, and delete remote files. This enables use cases like document Q&A and fine-tuning data management without leaving the Drupal AI abstraction layer. Requires `drupal/ai` 1.4.2 or higher.

## Impact

Affects module maintainers and site owners.

- **Requirements:** Minimum drupal/ai version raised from ^1.4.0 to ^1.4.2
- **New API:** OpenAiProvider now implements AiFileProviderInterface, adding uploadFile(), deleteFile(), downloadFile(), and supportsMimeType() methods for interacting with the OpenAI Files API.

## Technical details

`OpenAiProvider` now implements `AiFileProviderInterface` (from `drupal/ai` 1.4.2+) and uses the `FileApiTrait` (`Drupal\ai\Traits\OpenAi\FileApiTrait`) to fulfil that interface. The trait provides `uploadFile()`, `deleteFile()`, `downloadFile()`, and `supportsMimeType()` methods backed by the OpenAI PHP SDK. MIME type eligibility is gated by purpose: `batch` and `fine_tune` accept text and JSON only, `vision` accepts image types, and `user_data` is unrestricted.

On the chat side, `OpenAiProvider::chat()` was extended to detect `getRemoteFiles()` on each `ChatMessage` and append `{'type': 'file', 'file': {'file_id': $id}}` content blocks to the outgoing API payload. This allows previously uploaded files to be referenced in a conversation turn without re-sending file bytes.

Error handling in `uploadFile()` maps "Request too large" API errors to `AiRateLimitException` and quota errors to `AiQuotaException`. The minimum `drupal/ai` constraint in `composer.json` was tightened from `^1.4.0` to `^1.4.2`, which is the release that introduced `AiFileProviderInterface` and `FileApiTrait` in the base module (see [#3546954](https://www.drupal.org/node/3546954)).

## Upgrade

No changes required for existing code. To use file handling, obtain the provider as `AiFileProviderInterface` and call its methods:

```php
use Drupal\ai\AiFileProviderInterface;

// Upload a file (returns the AiFile entity with remote ID set).
/** @var AiFileProviderInterface $provider */
$aiFile = $this->aiFileManager->upload('public://docs/report.pdf', 1, 'openai');

// Reference the uploaded file in a chat message.
$message = new ChatMessage('user', 'Summarize this document.');
$message->addRemoteFile($aiFile->getRemoteId());
$input = new ChatInput([$message]);
$response = $provider->chat($input, 'gpt-4o')->getNormalized();
```
