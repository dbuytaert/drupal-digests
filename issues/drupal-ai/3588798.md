# #3588798: [Sprint 8] Promote PASS/FAIL table format across all sub-agents

## Summary

The `canvas_ai_migrations` module's AI-assisted migration workflow has standardized how sub-agents report their results. Five agents (`storybook-qa`, `phase-verifier`, `visual-verifier`, `upload-verifier`, `component-builder`) now produce a structured PASS/FAIL table in their chat reply AND write a JSON checklist file to `docs/migration/workflow/` before returning. The orchestrator reads those JSON files to decide whether a phase passed, instead of parsing the agent's prose output. This makes phase gating deterministic across long sessions that trigger context compaction, where prose summaries were previously paraphrased and lost specific evidence values.

## Impact

No upgrade or configuration changes required.

## Technical details

The motivation was a concrete compaction-survival problem: the LLM compaction algorithm's "Files and Artifacts" section quotes table rows verbatim but paraphrases prose paragraphs, so agents that narrated results like "fixed styling differences" left the orchestrator unable to confirm specific values in later iterations. Agents that used tables (the existing `artifact-checker`) preserved exact values across compaction boundaries.

Each of the five updated agent definition files (`.claude/agents/*.md`) gained a `## Output format` section specifying: a `| Check | Status | Details |` return table capped at 20 lines, and a mandatory side-effect of writing a JSON checklist to `docs/migration/workflow/<zero-padded-phase>-<agent>-checklist.json`. The checklist schema nests results under a `payload` key (with `overall`, `items[]`, and `gaps[]`) so the pre-existing `checklist-gate.sh` Stop hook, which reads `.payload.overall` via `jq`, handles the new checklists without modification.

`SKILL.md` gained a `## Phase Gate Convention` section formalizing three steps: (1) verify the checklist file exists at the predicted path, (2) read `payload.overall` and proceed or re-dispatch the agent listed in `payload.gaps[].rerun`, (3) if the file is missing, dispatch `artifact-checker` in `phase-N` mode as a fallback synthesizer. The `state-events.md` reference file gained an `agent_checklist_written` event definition so the orchestrator can short-circuit existence checks when the event is present in `state.jsonl`. Fourteen Vitest tests in `tests/skills/sprint8-3588798-checklist-convention.test.mjs` verify the presence of the `## Output format` section, the correct `payload`-nested schema, and the zero-padded filename convention in every affected agent file.

## Upgrade

No external code changes are required. If you have custom orchestration scripts that parse sub-agent chat replies to determine phase success, update them to read `payload.overall` from the corresponding `docs/migration/workflow/<phase>-<agent>-checklist.json` file instead. Example for `upload-verifier` after Phase 4:

```bash
overall=$(jq -r '.payload.overall' docs/migration/workflow/04-upload-verifier-checklist.json)
if [ "$overall" = "pass" ]; then
  # proceed
fi
```
