## Summary

When comparing revisions of AI context items, the Split Fields and Unified Fields diff views were missing scope plugin information. Only the Visual Inline view showed all changes. This made it difficult to track what changed between versions when using the more commonly used diff formats. The issue has been fixed so all three comparison formats now properly display scope changes with human-readable labels.

## Impact

No upgrade or configuration changes required.

## Technical details

The Diff module's `CoreFieldBuilder` handles map field types but checks for a `value` key in its `build()` method. Map items store data as key-value pairs without a `value` key, so the scope field was silently skipped in Split Fields and Unified Fields views.

The fix adds a new `ScopeMapFieldDiffBuilder` plugin specifically for the `map` field type used by the scope field. The plugin iterates over map key-value pairs and uses `AiContextScopeManager` to resolve human-readable labels for both scope types and their values. For example, it converts `use_case: writing_words` to "Use Cases: Writing Words".

The install file was updated to configure the scope field to use `ai_context_scope_map_field_diff_builder` instead of `core_field_diff_builder`. The `MapFormatter` was also updated to handle optional configuration keys with default values, preventing errors when used in diff contexts.

Five new kernel tests verify the plugin works with various scope configurations. A comprehensive functional test confirms that scope changes appear correctly in the revision comparison UI for all diff formats.
