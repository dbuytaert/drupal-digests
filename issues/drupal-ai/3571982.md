## Summary

When an AI Assistant is configured to use an agent (via the `ai_agents` module), the contextual data sent by DeepChat with each request (such as the current route) was silently dropped before reaching the agent. This meant the chatbot could not answer context-aware questions like "what is the current route?" even though that information was present in the API payload. The fix forwards the DeepChat context into the agent execution pipeline and fires a new Symfony event so that any module can inspect or enrich that context before the agent starts its reasoning.

## Impact

Affects module maintainers.

- **New API:** New `AiAssistantPassContextToAgentEvent` (event name `ai_assistant.pass_context_to_agent`) dispatched before agent `determineSolvability()`; provides mutable access to the agent instance and DeepChat context array.

## Technical details

When an assistant is backed by an agent, `AiAssistantApiRunner::process()` delegates to `AgentRunner::runAsAgent()`. Previously that method had no way to receive the DeepChat context array that `AiAssistantApiRunner` already held in `$this->context`.

Three coordinated changes fix this:

1. **`AgentRunner::runAsAgent()` gains an optional `array $context = []` parameter.** `AiAssistantApiRunner::process()` now passes `$this->getContext()` as the sixth argument, closing the gap between the two services.

2. **New event `AiAssistantPassContextToAgentEvent`** (at `modules/ai_assistant_api/src/Event/AiAssistantPassContextToAgentEvent.php`) is dispatched inside `runAsAgent()` immediately after the agent is configured but before `$agent->determineSolvability()` is called. The event carries both the live `AiAgentInterface` agent instance and the context array. It is mutable: `getContext()` and `setContext()` are both present, so a subscriber can enrich or replace the context. The event is dispatched unconditionally (no `if (!empty($context))` guard), allowing subscribers to inject default context even when DeepChat sent nothing.

3. **`AgentRunner`'s constructor type-hint uses `EventDispatcherInterface`** (from `Symfony\Component\EventDispatcher\EventDispatcherInterface`) rather than the concrete `EventDispatcher` class, correctly accommodating Drupal's `ContainerAwareEventDispatcher` and any decorators.

Modules that want to act on this context (for example, converting `current_route` into agent tokens) can subscribe to `AiAssistantPassContextToAgentEvent::EVENT_NAME` (`'ai_assistant.pass_context_to_agent'`), call `$event->getAgent()` and `$event->getContext()`, and optionally call `$event->setContext()` to modify what downstream subscribers or the agent will see.

## Upgrade

Modules that need to act on DeepChat context when an assistant runs as an agent should create an event subscriber:

```php
use Drupal\ai_assistant_api\Event\AiAssistantPassContextToAgentEvent;

class MyContextSubscriber implements EventSubscriberInterface {
  public static function getSubscribedEvents(): array {
    return [AiAssistantPassContextToAgentEvent::EVENT_NAME => 'onPassContext'];
  }

  public function onPassContext(AiAssistantPassContextToAgentEvent $event): void {
    $agent = $event->getAgent();
    $context = $event->getContext(); // e.g. ['current_route' => '/admin/content']
    // Optionally enrich or replace the context for later subscribers:
    $event->setContext($context + ['my_key' => 'my_value']);
  }
}
```
