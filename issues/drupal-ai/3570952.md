## Summary

A continuous integration configuration issue was fixed where the JavaScript build process for CKEditor was running unnecessarily on every pipeline execution, including merge requests and feature branches. The build now runs only on main version branches like 1.x and 2.x, reducing pipeline execution time and computational overhead for routine development work.

## Impact

No upgrade or configuration changes required.

## Technical details

The `js_build` job in the GitLab CI pipeline was executing on every branch and merge request, rebuilding CKEditor JavaScript assets each time. This was inefficient because the JavaScript build only needs to run on main version branches where release artifacts are prepared.

The pipeline configuration was updated to restrict the `js_build` job to main branches only (version branches like 1.1.x, 1.3.x, and 2.0.x). Merge request pipelines no longer include this job, reducing unnecessary build overhead during development and code review processes.

The fix was applied to the pipeline configuration and cherry-picked to all active main branches to ensure consistent behavior across all supported versions.
