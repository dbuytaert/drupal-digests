# #3570952: js_build is run in every branch

- **Project:** Drupal AI
- **Type:** Major bug report
- **Version:** 1.2.x-dev
- **Status:** Fixed
- **Discussion:** [#3570952](https://www.drupal.org/node/3570952) · 3 contributors · 9 comments

## Summary

The AI module's continuous integration pipeline was unnecessarily rebuilding CKEditor JavaScript assets on every pipeline run, including feature branches and merge requests. This wasted computing resources and slowed down automated testing. The change ensures the JavaScript build step only runs on main release branches like 1.x and 2.x, making development faster and more efficient.

## Impact

No upgrade or configuration changes required.

## Technical details

The `js_build` job in the AI module's GitLab CI pipeline was configured to run on all branches, causing CKEditor JavaScript assets to be rebuilt every time a pipeline executed. This was inefficient because JavaScript builds are only needed when changes are ready to be deployed from main release branches.

The fix modifies the pipeline configuration to exclude the `js_build` job from merge request pipelines. The job now only executes on the main version branches (1.1.x, 1.3.x, 2.0.x, etc.) where JavaScript builds are actually needed for releases. This change was applied to all active release branches to ensure consistent behavior across the project.
