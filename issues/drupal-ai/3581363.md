## Summary

The AI module's MDX markdown editor widget now supports being filled programmatically from JavaScript. Any code on the page can dispatch a `drupal:mdx-fill` custom DOM event on the underlying textarea element, passing the desired markdown string in `event.detail.content`, and the editor will update its visual state and keep Drupal's form state in sync automatically. This makes it straightforward to integrate the MDX editor with AI-assist features or other tools that need to inject generated content into the field.

## Impact

Affects module maintainers.

- **New API:** New `drupal:mdx-fill` custom DOM event on the textarea element lets external JavaScript set the MDX editor content programmatically via `event.detail.content`.

## Technical details

The MDX editor is a React component (`Editor`) mounted by `initTextareaEditors()` in `main.jsx`. Previously there was no way for code outside the React tree to call methods on the editor instance.

The fix introduces two cooperating pieces. First, the `Editor` component gains an `onRef` prop. A `useEffect` hook fires once the component mounts and calls `onRef(editorRef.current)`, exposing the MDXEditor instance (which carries `setMarkdown`) to the caller. Second, in `initTextareaEditors()` an `editorMethods` object (`{ current: null }`) is created before `createRoot().render()`. The `onRef` callback stores the live ref there. After rendering, a `drupal:mdx-fill` event listener is attached to the original textarea. When that event fires, the listener reads `event.detail.content`, guards that it is a string, calls `editorMethods.current.setMarkdown(content)` to update the React visual editor, writes the value to `textarea.value`, and dispatches synthetic `input` and `change` events so that Drupal's form handling sees the update. The commit was authored by ahmad khader and cherry-picked to the `1.x`, `1.4.x`, `2.x`, and `2.0.x` branches.
