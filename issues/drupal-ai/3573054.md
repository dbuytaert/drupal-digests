## Summary

The AI File to Text module now provides a Document Loader plugin that allows other modules to convert documents into text, HTML, or Markdown without knowing which underlying service is being used. This makes it easier to build portable AI agents and automators that work with documents. The module ships with support for Word, PDF, spreadsheets, plain text, and Markdown files out of the box. Other modules can add support for new file types by registering extractor services without modifying the core module.

## Impact

Affects module maintainers.

- **New API:** New `document_loader:file` plugin for document conversion; extractors can be registered via `ai_file_to_text.extractor` service tag.

## Technical details

The module now integrates with the Document Loader plugin system. A single `FileDocumentLoader` plugin serves as the entry point for all file conversions, delegating to the existing `FileExtractorManager` layer. Each extractor now declares its `document_loader_type` (e.g. `document_loader_type:pdf`) and supported output formats via `getDocumentLoaderType()` and `getSupportedOutputFormats()` interface methods.

The `FileExtractorManager` collects all extractors tagged with `ai_file_to_text.extractor`, builds an extension-to-extractor map, and aggregates document loader types and output formats. A `hook_document_loader_info_alter()` implementation patches the cached plugin definition so `getLoaderByType()` can discover the plugin for any type, including types added by third-party modules.

When extractors have different output capabilities (e.g. PDF supports HTML but PowerPoint supports only text), the alter hook splits the plugin definition into multiple entries, each with accurate `output_types`. This prevents false positives when filtering by output format. The `FileExtractorManager` validates per-extension format support and falls back to plain text when a requested format is unsupported.

The existing AI automators and function calls now use the unified Document Loader plugin instead of calling extractors directly. The `CsvExtractor` and `OdsExtractor` classes were merged into a single `SpreadsheetExtractor` using PHPOffice/PhpSpreadsheet for all tabular formats.

## Upgrade

To use the Document Loader plugin in custom code, create a `FileInput` object and call `load()`:

```php
use Drupal\ai_file_to_text\DocumentLoaderType\Input\FileInput;

$input = new FileInput('public://docs/report.pdf', 'markdown');
$loader = \Drupal::service('plugin.manager.document_loader')
  ->createInstance('document_loader:file');
$output = $loader->load($input);
$markdown = $output->getContent();
```

Or discover by type:

```php
$loader = \Drupal::service('plugin.manager.document_loader')
  ->getLoaderByType('document_loader_type:spreadsheet', 'html');
```

To add support for a new file type, implement `ExtractorInterface` with methods `isAvailable()`, `getDocumentLoaderType()`, `getSupportedOutputFormats()`, `getSupportedExtensions()`, `extractText()`, and `extractHtml()`. Register it as a service tagged with `ai_file_to_text.extractor`:

```yaml
services:
  your_module.extractor.pptx:
    class: Drupal\your_module\Extractor\PptxExtractor
    tags:
      - { name: ai_file_to_text.extractor }
```
