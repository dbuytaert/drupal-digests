## Summary

The AI File to Text module now integrates with the [Document Loader](https://www.drupal.org/project/document_loader) plugin system. A single `document_loader:file` plugin is exposed so that AI automators, AI agent function calls, and any third-party code can convert documents to text, HTML, or Markdown without caring which underlying extractor handles the file. Other modules can add support for new file types by implementing `ExtractorInterface` and registering a tagged service, with no changes to this module needed. Spreadsheet handling (`.xlsx`, `.xls`, `.ods`, `.csv`) is consolidated into a new `SpreadsheetExtractor` backed by PHPOffice/PhpSpreadsheet, replacing the former separate `CsvExtractor` and `OdsExtractor`. The module now requires `drupal/document_loader` and `phpoffice/phpspreadsheet`.

## Impact

Affects module maintainers and site owners.

- **Requirements:** New required dependencies: `drupal/document_loader ^1.0` and `phpoffice/phpspreadsheet ^2.0`.
- **API change:** `ExtractorInterface` gains two required methods: `getDocumentLoaderType()` and `getSupportedOutputFormats()`; `CsvExtractor` and `OdsExtractor` classes removed; `ai_file_to_text.extractor.csv` and `ai_file_to_text.extractor.ods` service IDs removed.
- **New API:** New `document_loader:file` plugin; third-party modules can register file extractors via the `ai_file_to_text.extractor` service tag.

## Technical details

The central change is the introduction of `FileDocumentLoader`, a `document_loader:file` plugin that delegates all conversion work to `FileExtractorManager`. A `FileInput` DTO carrying the file URI, desired output format, and options is the single input type for all file formats.

`ExtractorInterface` gains two new required methods: `getDocumentLoaderType()` (e.g. `'document_loader_type:pdf'`) and `getSupportedOutputFormats()` (returns `['text']` or `['text', 'html']`; `markdown` is always derived from `html` by the manager and must not be listed). `FileExtractorManager` now accepts an `iterable` of tagged services via `!tagged_iterator ai_file_to_text.extractor` instead of explicit constructor arguments. At construction time it builds an extension-to-extractor map, collects unique `document_loader_type` strings, and aggregates output formats per extension and per type.

The Document Loader plugin system filters by `output_types` at the plugin level, not per type, which would cause false positives when extractors have different output capabilities. `ai_file_to_text_document_loader_info_alter()` in `ai_file_to_text.module` addresses this: it calls `FileExtractorManager::getCapabilityGroups()` to group `document_loader_type` values by their output format sets. When all extractors share the same capabilities the single `document_loader:file` definition is patched in place; when capabilities differ the definition is split into variant entries (`document_loader:file`, `document_loader:file__1`, etc.) each with accurate `document_loader_types` and `output_types`. This ensures `getLoaderByType('document_loader_type:presentation', 'html')` returns `NULL` rather than a false positive for a text-only extractor. At extract time, `FileExtractorManager::extract()` validates per-extension format support and silently falls back to plain text when the requested format is unsupported.

`CsvExtractor` (PHP `fgetcsv`) and `OdsExtractor` (ZipArchive + DOMDocument) are removed and replaced by `SpreadsheetExtractor`, which uses `PhpOffice\PhpSpreadsheet\IOFactory::load()` for all tabular formats (`.xlsx`, `.xls`, `.ods`, `.csv`). The new extractor adds richer HTML output: bold, italic, underline, strikethrough, and font color from cell styles; rich-text spans within a single cell; automatic heading-row detection; and leading/trailing empty row and column trimming. All built-in extractors now carry the `ai_file_to_text.extractor` service tag.

## Upgrade

Any module that implements `ExtractorInterface` must add two methods:

```php
public function getDocumentLoaderType(): string {
  // Return the document_loader_type string for this extractor.
  return 'document_loader_type:your_type';
}

public function getSupportedOutputFormats(): array {
  // Always include 'text'. Include 'html' only if extractHtml() produces
  // meaningful output. Do NOT include 'markdown' — it is derived from html.
  return ['text', 'html'];
}
```

Register the extractor as a tagged service (required for auto-discovery):

```yaml
# your_module.services.yml
services:
  your_module.extractor.custom:
    class: Drupal\your_module\Extractor\CustomExtractor
    tags:
      - { name: ai_file_to_text.extractor }
```

If your code referenced `CsvExtractor` or `OdsExtractor` directly, replace them with `SpreadsheetExtractor`. The `ai_file_to_text.extractor.csv` and `ai_file_to_text.extractor.ods` service IDs no longer exist; use `ai_file_to_text.extractor.spreadsheet`.

The `FileExtractorManager` constructor no longer accepts individual extractor arguments; it now takes a single `iterable`. Any code that constructed it manually must be updated to pass an array or iterator of `ExtractorInterface` objects.
