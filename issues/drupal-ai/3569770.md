Tooltips automatically reposition themselves when near screen edges, improving usability on smaller viewports. ## Technical details

The tooltip component uses the HTML Popover API with `popover="hint"` and CSS Anchor Positioning with `position-try-fallbacks: flip-inline` to automatically flip tooltips when they would extend beyond the viewport edge. The implementation lives in `ai_global.js` as a Drupal behavior that finds elements with a `data-ai-tooltip` attribute and creates associated popover elements.

The JavaScript handles delayed showing (500ms) and immediate hiding with timer management to prevent race conditions. It attaches both mouse and keyboard event handlers for accessibility. The `ai_global` library is now attached globally via `hook_page_attachments()` instead of only on forms, enabling tooltips anywhere in the module. All classes use the `ai-` prefix to avoid conflicts.

The native Popover API is supported by all major browsers since 2025. The `hint` value provides additional tooltip priority features but degrades gracefully in browsers that don't support it. No polyfill was added since core functionality works universally. The CSS uses Gin admin theme variables for consistent styling with borders, shadows, and proper spacing.

## Lessons learned

- When adding UI components that may be reused across a module, use data attributes (e.g., `data-*`) to declaratively attach functionality to elements. This separates configuration from implementation and allows developers to add features like tooltips without writing JavaScript, making the component API simple and intuitive for other developers to adopt. *(via bbruno, ahmad khader)*
- When building tooltip or popover components, leverage native browser APIs (like the Popover API) even when optional features (like `popover='hint'`) aren't universally supported. If the core functionality works across all major browsers and the unsupported parts provide only minor enhancements (like tooltip priority), the simplicity of avoiding polyfills outweighs the loss of those features. *(via bbruno)*
