## Summary

The AI module now includes a reusable tooltip component built on native browser features. Tooltips display helpful information when users hover over or focus on elements, and automatically reposition themselves when near the edge of the screen to remain visible. The first implementation appears in the AI settings dashboard, where provider and model information links now show tooltips instead of basic browser title attributes. This provides a better user experience with more consistent styling and smarter positioning.

## Impact

No upgrade or configuration changes required.

## Technical details

The implementation uses the HTML Popover API with `popover="hint"` attribute for native browser tooltip behavior. CSS Anchor Positioning with `position-try-fallbacks: flip-inline` enables automatic viewport-aware repositioning. The tooltip flips from right to left when near screen edges without requiring JavaScript calculations.

A new `ai_global.js` file implements a Drupal behavior that finds elements with the `data-ai-tooltip` attribute. For each element, it creates a corresponding popover div, assigns unique IDs, sets up ARIA relationships, and attaches mouse and keyboard event listeners. Tooltips show after a 500ms delay on mouseenter or focusin, and hide on mouseleave or focusout. Timers prevent race conditions when users quickly move between elements.

The `ai_global` library now attaches globally via `hook_page_attachments()` instead of only on forms. All styles use the `ai-` prefix to avoid conflicts. The settings form replaces the native `title` attribute on provider and model info links with the new `data-ai-tooltip` attribute. Styling uses Gin design tokens for consistent appearance across the admin interface.

## Lessons learned

- When building reusable UI components with modern browser APIs, avoid polyfills for features that gracefully degrade. If the native popover attribute works across all browsers and only minor enhancements (like the hint value) are unsupported, ship the component without polyfills—the core functionality works everywhere while accepting that edge cases will have slightly different behavior. *(via bbruno)*
- When building shared UI components, use data attributes for the public API rather than requiring manual markup. A simple attribute like data-ai-tooltip="content" lets developers add functionality to any element without learning complex HTML structures or JavaScript initialization, making the component easier to adopt across a codebase. *(via bbruno, ahmad khader)*
