# #3601375: Post-install hand-off: open the project shell on inactivity instead of on Enter

- **Project:** Drupal AI

## Summary

The post-install shell hand-off in the `one_line_installer` now opens the project shell by default on inactivity instead of requiring an explicit keypress. Previously, the 30-second countdown required the user to press Enter to open a shell in the new project directory, and timing out simply printed the directory path. The prompt is now inverted: the shell opens automatically after 30 seconds of inactivity, and pressing Enter is what skips it. This eliminates an extra keypress for the common case where a human just finished an install and wants to start working immediately. Automated or agent-driven runs that reach this prompt without `--yolo` will now drop into an interactive shell rather than timing out safely; such callers should pass `--yolo`, which bypasses the prompt entirely.

## Impact

No upgrade or configuration changes required.

## Technical details

The change is in `src/lib/steps/agent_handoff.sh` inside `step_agent_handoff`. The `read -t 30` call and the branch logic are inverted: `read -r -t 30` now returns zero on Enter (skip) and non-zero on timeout (open shell). The timeout path falls through to `exec "${SHELL:-bash}" </dev/tty`, and the Enter path returns after printing the `cd` hint. The prompt copy changed from "Press Enter within 30s to open a shell…" to "Opening a shell in your project in 30s — press Enter to skip…". The guard for `--yolo` and non-tty environments (`YOLO=1` or `/dev/tty` unavailable) is unchanged: those paths print the project path and return without any countdown. The `agent_handoff.bats` test assertion was updated to check for "press Enter to skip" absence under `--yolo`, confirming the new prompt text. The generated `drupalaibp` script must be regenerated after this `src/` change, per the project's standard workflow.
