## Summary

Canvas AI agents that generate YAML-formatted tool inputs (like the Page Builder and Template Builder agents) can now be tested with two new validation plugins. `YAML Path Exists` checks whether a specific path is present in the YAML output. `YAML Path Equals` checks whether a value at a specific path matches an expected value. Test authors can use these plugins when writing automated tests for Canvas AI agents using the AI Agents Test module. This is a testing-only change with no user-facing effects.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas AI agents generate tool parameters in YAML format. The AI Agents Test module provides an `AgentsParameterTest` plugin system for validating those parameters, but it lacked plugins specifically for YAML validation.

This change adds a new test module `canvas_ai_agents_test` with two new `AgentsParameterTest` plugins. Both plugins validate only the `component_structure` parameter.

The `YamlPathExistsTest` plugin checks whether one or more dot-notated paths exist in the YAML. The `YamlPathEqualsTest` plugin checks whether values at specific paths match expected values using `path=value` syntax. Both plugins support multiple checks separated by newlines and support escaping literal dots in keys with a backslash (e.g., `sdc\.canvas_test_sdc\.heading`).

A new `YamlPathHelper` service handles the shared YAML parsing and path traversal logic. It uses Symfony's YAML component to parse the string and walks the parsed array using a regex-split path that preserves escaped dots. The service returns an array indicating whether the path exists, the value if found, and any parse errors.

Both plugins inject the helper service, split the rule value by newlines, and call `getValueAtPath()` for each path. They record detailed results for each check using the base plugin's `createDetailedResultRow()` method.
