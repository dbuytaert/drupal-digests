The Drupal AI module now includes comprehensive documentation for testing custom AI providers during development. The new guide covers how to test all operation types, including chat, embeddings, image classification, moderation, speech-to-text, text-to-speech, and text-to-image. Developers can use the AI API Explorer module to verify features like streaming, vision, structured data output, and tool use. ## Technical details

The change adds a new developer documentation file at `docs/developers/testing_an_ai_provider.md` with step-by-step test scenarios for AI providers. The documentation is organized by operation type, with subsections for specialized features within each type.

For chat operations, the guide covers basic chat, system prompts, streaming responses, vision input, structured output using JSON schemas, PDF parsing, and function calling with tools. For embeddings, it covers both text and image input. Additional operation types documented include image classification, content moderation, speech-to-text, text-to-speech, and text-to-image generation.

Each test scenario provides the exact AI API Explorer path to use (for example `/admin/config/ai/explorers/chat_generator`), the inputs to provide, and the expected outputs. The guide explains how to enable the `ai_api_explorer` module and configure providers before testing. For tool use testing, it describes enabling test mode with `$settings['extension_discovery_scan_tests']` and using the `ai_test` module's calculator tool.

The documentation structure was also reorganized in `mkdocs.yml` to create a dedicated "Develop an AI Provider" section containing both the existing getting started guide and the new testing guide.
