## Summary

This discussion issue for the `ai_context` contributed module proposed decoupling AI Context items from AI Agents so that context (such as a company writing style guide) could be used by any AI-powered integration, not only agent-based ones. Use cases like automators that generate newsletters or AI Translate tools that produce culturally appropriate translations would benefit from shared context even without an agent. The discussion concluded that the approach was viable and the concrete implementation was folded into the companion issue [#3574359](https://www.drupal.org/node/3574359) (Refactor context selection logic), which is now closed fixed.

## Impact

No upgrade or configuration changes required.

## Technical details

The `ai_context` module's context request pipeline was initially scoped around agents. The key class cited in the thread is `AiContextRequestFactory`, whose `fromParameters` factory method was shown to construct an `AiContextRequest` without requiring an `agent_id`, supporting agent-free callers such as automators and translation workflows. The `AiContextRequest` value object collects parameters including `task`, `scopeSubscriptions`, `alwaysInclude`, `neverInclude`, `consumerId`, `entityType`, `entityId`, `maxItems`, `maxTokens`, and `selectionMode`. The discussion also raised smart context selection: tagging context items by category (Design System, Content, Brand, etc.) so a relevance selector could automatically include only appropriate items, reducing the need for manual agent-to-context wiring. Concrete code changes, including refactoring `fromParameters` and the selection logic, were tracked and resolved in [#3574359](https://www.drupal.org/node/3574359).
