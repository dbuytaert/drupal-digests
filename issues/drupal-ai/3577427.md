## Summary

The AI Context module now makes the Diff module optional for comparing revisions of context items. Previously, Diff was a required dependency even if revision comparison wasn't needed. The module now works without Diff installed, displaying a helpful message on the revisions page that invites users to install Diff if they want to enable the comparison feature. When Diff is enabled, the full revision comparison functionality appears automatically. This change reduces the installation footprint for sites that only need basic context item management without revision comparison capabilities.

## Impact

No upgrade or configuration changes required.

## Technical details

The Diff module was moved from `require` to `require-dev` in `composer.json` and added as a suggested dependency. The `ai_context_item` entity type now uses the core `RevisionHtmlRouteProvider` by default instead of always loading Diff's `DiffRouteProvider`. A new `DiffHooks` class implements hooks to conditionally swap in Diff's route provider and configure field settings only when the module is installed.

The revision overview form (`AiContextItemRevisionOverviewForm`) was updated to check whether Diff is enabled. When absent, it disables the radio buttons for selecting revisions, hides the compare button, and displays a status message linking to the Diff project page. When Diff is present, the full comparison UI appears as before.

Two new functional test classes cover both states: `AiContextItemRevisionUiWithoutDiffTest` confirms the revisions tab remains accessible and the install message appears, while `AiContextItemRevisionUiWithDiffTest` validates that the compare workflow reaches the diff route. The `.gitlab-ci.yml` file was updated to test against both Drupal 10.5 and 11.

## Lessons learned

- When adding optional integrations with external modules, declare them as suggested dependencies rather than required dependencies, and conditionally load features only when the external module is present. Use service definitions and attribute-based discovery (e.g., route providers in entity annotations) to defer registration until runtime availability checks confirm the dependency is installed. *(via a.dmitriiev, scott falconer)*
- When a framework feature can be dynamically provided by an optional dependency, provide graceful degradation in the UI: disable interactive controls, display informational messages explaining what capability is missing, and guide users toward enabling the optional module. This maintains core functionality while communicating upgrade paths. *(via kristen pol)*
