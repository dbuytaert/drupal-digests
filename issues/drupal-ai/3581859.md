## Summary

The `ai_best_practices` project now validates contributed Agent Skills against the official Agent Skills specification on every merge request. Skills must follow a subdirectory layout (`skills/{name}/SKILL.md`) rather than a flat file (`skills/{name}.md`), and each `SKILL.md` must include `name` and `description` frontmatter fields alongside the existing `status`, `drupal-version`, and `last-reviewed` metadata. A GitLab CI job runs the validator automatically, catching non-compliant skills before they are merged. Existing skills in the repository were restructured to match the new layout as part of this change.

## Impact

No upgrade or configuration changes required.

## Technical details

A new `.gitlab-ci.yml` was added with a `validate-agent-skills` job that triggers on merge request events when files under `skills/**/*` change. It runs `composer install` using the `composer:2` Docker image and then invokes `vendor/bin/agent-skills-validator skills`, a CLI tool from the new dev dependency `ronaldtebrake/agent-skills-validator: ^1.0` added to `composer.json`.

The two existing skills were migrated from flat files (`skills/writing-automated-tests.md`, `skills/how_to_write_documentation.md`) to the specification-compliant subdirectory format (`skills/writing-automated-tests/SKILL.md`, `skills/how-to-write-documentation/SKILL.md`). Frontmatter in each `SKILL.md` was updated to include `name` and `description` at the top level, with `status`, `drupal-version`, and `last-reviewed` nested under a `metadata:` key. The old `skills/writing-automated-tests.md` stub was deleted.

All references in `CONTRIBUTING.md`, `CLAUDE.md`, eval configuration JSON files, and eval README files were updated to reflect the new path convention. The eval static assertion for `S01` (file exists check) was also updated to point to `skills/writing-automated-tests/SKILL.md`.

## Upgrade

Any skill file previously placed at `skills/{name}.md` must be moved to `skills/{name}/SKILL.md` and its frontmatter must be updated to the new shape:

```yaml
---
name: your-skill-name
description: One-line description of what this skill covers.
metadata:
  status: draft
  drupal-version: "10.3+"
  last-reviewed: 2026-04
---
```

References to the old flat path in eval JSON files, README files, or agent context files (e.g., `CLAUDE.md`, `GEMINI.md`) must be updated to the new `skills/{name}/SKILL.md` path.
