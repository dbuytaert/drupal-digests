## Summary

The Huggingface AI provider now supports text summarization. This allows site owners to automatically generate summaries of longer texts using Huggingface's summarization models. The feature integrates with the AI module's new summarization operation type, making it available through the AI framework's standard interfaces and explorer tools. Sites using Huggingface as their AI provider can now leverage summarization capabilities alongside existing features like chat, embeddings, and image classification.

## Technical details

This change implements the `SummarizationInterface` in the `HuggingfaceProvider` class, adding support for the summarization operation type introduced in the AI module ([#3574178](https://www.drupal.org/node/3574178)).

The implementation adds a `summarize()` method that accepts text input (as a string, array, or `SummarizationInput` object), calls the Huggingface API's summarization endpoint, and returns a `SummarizationOutput` object containing the generated summary. The method normalizes input by extracting the text property when a `SummarizationInput` object is provided, validates that the text is not empty, and makes the API request through the existing `HuggingfaceApi::summarization()` client method.

The provider's capabilities configuration is updated to include `summarize` in both the `MODEL_MAPPING` array (which maps the operation to Huggingface's `summarization` filter for model discovery) and the `getSupportedOperationTypes()` method. This enables the AI module to discover compatible Huggingface models and present summarization functionality through the explorer interface.

Minor cleanup includes fixing an unused parameter issue in the existing text generation method and adding a content type parameter to image classification requests.
