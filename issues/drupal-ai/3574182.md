## Summary

The Huggingface AI provider now supports text summarization. Sites using Huggingface models can now automatically generate summaries of longer text content. This matches the new summarization operation type added to the AI module, allowing consistent summarization features across different AI providers. The feature works through Huggingface's summarization inference endpoints.

## Impact

No upgrade or configuration changes required.

## Technical details

The `HuggingfaceProvider` class now implements the `SummarizationInterface` from the AI module core. The new `summarize()` method accepts text input (either as a string, array, or `SummarizationInput` object), validates that a Huggingface endpoint is configured for the model, and calls the API's summarization endpoint.

The implementation normalizes different input formats into a consistent array with `text` and `prompt` keys. It validates that input text is not empty before making the API call. The response is decoded from JSON and validated to ensure it contains the expected `summary_text` field in the first array element. Results are wrapped in a `SummarizationOutput` object.

The provider's supported operations list and model filters were updated to include 'summarize' alongside existing operations like chat, embeddings, and image classification. A minor fix was also included to explicitly set the content type for image classification requests.
