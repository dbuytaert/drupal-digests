## Summary

The OpenAI provider module was sending configuration parameters to GPT-5.x and o-series models that those models do not support, causing API requests to fail with an error. The module now automatically removes unsupported parameters like `top_p`, `temperature`, `frequency_penalty`, and `presence_penalty` when using these newer model families. This allows sites to successfully use GPT-5.1, GPT-5.2, and o-series models without manual configuration changes.

## Impact

No upgrade or configuration changes required.

## Technical details

The `OpenAiProvider` plugin sends various configuration parameters to the OpenAI API based on user settings and defaults. These include sampling parameters like `top_p` (nucleus sampling), `temperature` (randomness), `frequency_penalty`, and `presence_penalty` (repetition control).

GPT-5.x models and o-series models have different API constraints than earlier GPT models. They reject requests that include these sampling parameters, even though some API documentation incorrectly marks them as optional. When these parameters were sent, the API returned an error: "Unsupported parameter: 'top_p' is not supported with this model."

The fix adds logic in the `OpenAiProvider::chat()` method to detect when a GPT-5.x or o-series model is being used by checking the model ID prefix. For these models, the code now explicitly removes all four sampling parameters from the configuration array before sending the API request. This is applied in the same conditional block that already handles converting `max_tokens` to `max_completion_tokens` for these newer models.
