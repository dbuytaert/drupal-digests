## Summary

The AI module's OpenAI provider was sending unsupported parameters when using GPT-5.x and o-series models, causing errors that prevented these newer models from working. The provider now automatically filters out unsupported parameters like `top_p`, `temperature`, `frequency_penalty`, and `presence_penalty` for these model families. This fix allows sites to use the latest GPT models without encountering parameter validation errors.

## Impact

No upgrade or configuration changes required.

## Technical details

The OpenAI API has different parameter requirements for different model families. The GPT-5.x and o-series models reject parameters like `top_p`, `temperature`, `frequency_penalty`, and `presence_penalty` that are accepted by earlier models like GPT-4.

The `OpenAiProvider` class was sending these parameters regardless of the model being used, which caused API errors when invoking GPT-5.1, GPT-5.2, or o-series models. The fix adds logic to detect these newer model families and strip out the unsupported parameters before sending the request.

The detection uses the existing model ID check for o-series models (checking if the model ID starts with 'o') and extends it to cover GPT-5.x models as well. When these models are detected, the provider now explicitly unsets the four unsupported configuration parameters from the request payload. This ensures compatibility across different OpenAI model generations without requiring configuration changes.
