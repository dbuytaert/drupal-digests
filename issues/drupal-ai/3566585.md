## Summary

The AI Automators module now supports Field Widget Actions (FWA) for the content moderation state field. Site owners who use Drupal's Content Moderation module can configure an AI Automator on the `moderation_state` field and have the AI suggest a transition state (for example, Draft or Published) directly in the node edit form via an action button on the widget. Note that enabling the `Base Field Override UI` module is required to expose the `moderation_state` base field in the form display configuration so the FWA can be attached to it.

## Impact

Affects module maintainers.

- **New API:** New `FieldWidgetAction` plugin `automator_moderation_state` targeting the `moderation_state_default` widget, enabling AI-suggested moderation state transitions in the node edit form.

## Technical details

The feature adds a new `FieldWidgetAction` plugin class `ModerationState` (plugin id `automator_moderation_state`) in `modules/ai_automators/src/Plugin/FieldWidgetAction/ModerationState.php`. The class extends `AutomatorBaseAction` and is annotated via `#[FieldWidgetAction]` to target `widget_types: ['moderation_state_default']` with `field_types: ['string']`, making it active only when the Content Moderation widget is in use.

Two methods are overridden to accommodate the non-standard form structure of the `moderation_state_default` widget, compared to the reference implementation in `Email.php`:

- `aiAutomatorsAjax()`: Uses two consecutive `array_pop()` calls on `$array_parents` to walk up the form tree to the field level. Sibling plugins like `Email` use one pop followed by appending `$this->formElementProperty`, but the moderation widget's button sits one level deeper, requiring the extra pop and no property re-append.
- `saveFormValues()`: Writes to the `['state']` sub-element of the widget rather than to `[$this->formElementProperty]` (which is `'value'`). For the null-key path (all items), it sets `#default_value`; for the specific-item path it sets `#value`. This matches the internal form structure of the `moderation_state_default` widget, which exposes a `state` select element rather than a generic `value` input.

No `content_moderation` dependency was added to `ai_automators.info.yml`; the plugin is naturally scoped by the `widget_types` annotation to situations where that widget is in use.
