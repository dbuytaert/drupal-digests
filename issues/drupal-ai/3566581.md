## Summary

The AI Automators module can now work with JSON fields that use the textarea widget. This lets site builders trigger AI-powered actions directly from JSON field editing forms, such as generating valid JSON from a text description or normalizing existing JSON data. The feature requires the JSON Field module to be installed and supports the three JSON field types it provides.

## Impact

No upgrade or configuration changes required.

## Technical details

This change adds a new Field Widget Action plugin `Json` to the AI Automators module at `modules/ai_automators/src/Plugin/FieldWidgetAction/Json.php`. The plugin extends `AutomatorBaseAction` and registers support for the `json_textarea` widget with three field types: `json`, `json_native`, and `json_native_binary`.

The plugin follows the same pattern as the existing `Email` Field Widget Action. It implements `aiAutomatorsAjax()` to handle AJAX requests when users trigger an automator action from the field widget. The method extracts the field name and delta from the triggering element, then calls `populateAutomatorValues()` to run the configured AI automator and update the form.

The `saveFormValues()` method handles writing automator results back into the JSON field's form structure. It includes logic to handle both standard textarea structures where value is a child element and flattened widget structures where the textarea is at the top level. The method works with both single delta updates and bulk updates across all field deltas.

The plugin is declared using PHP attributes and specifies the `json_textarea` widget type and all three JSON field types in its annotation. This makes the "Automator JSON" option automatically available in the field widget configuration UI when the JSON Field module provides these field types.
