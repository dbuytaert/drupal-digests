## Summary

JSON API Extras previously required the broad `administer site configuration` permission to access its configuration forms and manage resource config entities. This made it impossible to delegate JSON:API Extras configuration to a role without granting that role unrestricted access to every configuration on the site. A new, dedicated permission called `administer json api extra configuration` now controls access to these forms. An automatic database update (`jsonapi_extras_update_9002`) grants the new permission to every role that already held `administer site configuration`, so existing setups will continue working after running updates.

## Impact

Affects site owners.

- **Permission change:** New `administer json api extra configuration` permission replaces `administer site configuration` for all JSON API Extras configuration forms and resource config entities; run `update.php` or `drush updb` to migrate existing roles automatically.

## Technical details

The change touches three areas in the codebase. First, `jsonapi_extras.permissions.yml` declares the new `administer json api extra configuration` permission with `restrict access: true`. Second, the route requirement in the routing YAML and the `admin_permission` annotation on the `JsonapiResourceConfig` config entity both switch from `administer site configuration` to the new permission, which gates both the settings form (`/admin/config/services/jsonapi/extras`) and all CRUD operations on the resource config entities. Third, `jsonapi_extras_update_9002()` in `jsonapi_extras.install` iterates over all user roles and calls `grantPermission('administer json api extra configuration')` on any role that already has `administer site configuration`, ensuring backward compatibility without manual intervention. A new kernel test `JsonapiExtrasUpdateTest` verifies that the update hook grants the permission only to qualifying roles.

## Upgrade

Run `drush updb` (or visit `/update.php`) after updating the module. The update hook `jsonapi_extras_update_9002` automatically grants the new permission to all roles that had `administer site configuration`.

If you have roles that should be able to manage JSON:API Extras configuration but do **not** have `administer site configuration`, grant the new permission to those roles manually:

```php
// Via Drush or a one-time script:
$role = \Drupal\user\Entity\Role::load('your_role_id');
$role->grantPermission('administer json api extra configuration')->save();
```

Or grant it through the admin UI at `/admin/people/permissions`.
