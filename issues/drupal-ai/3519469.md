## Summary

The Gemini AI provider now supports tools calling, allowing AI models to execute functions like calculators or weather lookups during conversations. This brings Gemini up to parity with other providers like OpenAI and Anthropic that already support this feature. The underlying library was also upgraded from version 1.0 to 2.7, which provides access to newer Gemini models including Gemma and Nano variants.

## Impact

No upgrade or configuration changes required.

## Technical details

The `GeminiProvider` class now implements tools calling by building `FunctionDeclaration` objects from `ChatInput` tool definitions and passing them to the Gemini API via the `withTool()` method. When the AI response includes function calls, the provider executes them through the `FunctionCallPluginManager` and returns the readable output.

The implementation maps Drupal's `ToolsPropertyInputInterface` properties to Gemini's `Schema` objects, handling nested objects and arrays recursively in the `buildSchema()` method. It supports all property types including objects with sub-properties, arrays with typed items, and validation constraints like min/max length and patterns.

Function execution happens in `handleFunctionCall()`, which retrieves the function plugin instance, sets context values from the function call arguments, executes the function if it implements `ExecutableInterface`, and returns the output for functions implementing `ExecutableFunctionCallInterface`.

The `composer.json` dependency on `google-gemini-php/client` was updated from `^1.0` to `^2.5`, and the AI module dependency was updated from `^1` to `^1.1` to ensure tools support is available. A type cast was added for the `topK` configuration parameter to match the library's int type requirement.
