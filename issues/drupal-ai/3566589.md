## Summary

Content editors can now generate audio files directly from text using AI within file upload fields. When editing content with a file field, editors can trigger an AI Automator configured for speech generation, and the resulting audio file automatically attaches to the field. This streamlines workflows where AI-generated audio needs to be stored alongside other content, such as podcasts, accessibility features, or multimedia content.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI Automators module now includes a new Field Widget Action plugin for file fields. The `File` class in `modules/ai_automators/src/Plugin/FieldWidgetAction/File.php` extends `AutomatorBaseAction` and registers itself for the `file_generic` and `file_default` widget types on `file` field types.

The plugin provides an `aiAutomatorsAjax()` method that handles the AJAX callback when an editor triggers the action from the widget. It extracts the form structure and delegates to `populateAutomatorValues()` to run the configured automator and update the form state.

The `saveFormValues()` method ensures generated files are properly attached to the form element by setting the `#value` property with the file ID from the entity's field item. It handles both single-item and multi-value field scenarios by iterating through items when no specific key is provided, or targeting a specific index when a key is given.
