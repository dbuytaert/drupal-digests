## Summary

The AI module now supports text summarization through a dedicated operation type. This allows sites to condense long content into shorter summaries using specialized AI models. These summarization models are often smaller and faster than general-purpose language models, making them efficient for running locally or through services like Huggingface and Ollama. The feature includes a testing interface where administrators can try different summarization models and optionally provide custom instructions.

## Impact

Affects module maintainers.

- **New API:** New `summarize` operation type with `SummarizationInterface`, `SummarizationInput`, and `SummarizationOutput` classes for text summarization.

## Technical details

The change introduces a new `summarize` operation type following the module's plugin architecture pattern. The `SummarizationInterface` defines a `summarize()` method that accepts text input and returns a summary. The `SummarizationInput` class holds the text to summarize plus an optional prompt for guiding the summarization style. The `SummarizationOutput` class wraps the summary string along with raw provider output and metadata.

A new `SummarizationGenerator` explorer plugin provides an admin UI at `/admin/config/ai/explorers/summarize_generator`. The form accepts text input, an optional summarization prompt, and provider/model selection. It handles empty input validation and catches both `TypeError` for configuration issues and generic exceptions for API errors.

The summarization API is designed to work with dedicated summarization models like facebook/bart-large-cnn from Huggingface, browser-based models in Chrome's built-in AI, and Ollama summarization models. These specialized models are typically smaller and more efficient than general chat models for this specific task.

## Upgrade

AI provider modules that want to support summarization should implement `SummarizationInterface` and the `summarize()` method. The method receives a `SummarizationInput` object (or string/array that gets normalized), a model ID, and optional tags. It must return a `SummarizationOutput` object.

Example implementation pattern:

```php
public function summarize(string|array|SummarizationInput $input, string $model_id, array $tags = []): SummarizationOutput {
  if (!$input instanceof SummarizationInput) {
    $input = new SummarizationInput((string) $input);
  }
  
  $text = $input->getText();
  $prompt = $input->getPrompt();
  
  // Call your provider's summarization API
  $result = $this->apiClient->summarize($text, $prompt);
  
  return new SummarizationOutput(
    $result['summary'],
    $result,
    $result['metadata'] ?? []
  );
}
```

Consumers can call the API through the provider:

```php
$input = new SummarizationInput($long_text, 'Summarize in 3 bullet points');
$output = $provider->summarize($input, 'facebook/bart-large-cnn');
$summary = $output->getSummary();
```
