## Summary

The AI module now supports dedicated text summarization models. Some AI models are specifically trained for summarization tasks and can run more efficiently than general-purpose chat models. This matters for sites using on-device models like Chrome's built-in AI or specialized Huggingface and Ollama summarization models. A new explorer interface lets site builders test summarization with different providers and models before deploying them in production features.

## Impact

No upgrade or configuration changes required.

## Technical details

The module introduces a new `summarize` operation type alongside existing operations like chat and text generation. The architecture follows the established operation type pattern with three components: `SummarizationInterface` for providers to implement, `SummarizationInput` for text plus optional guidance prompts, and `SummarizationOutput` for the resulting summary.

The `SummarizationGenerator` explorer plugin provides a testing interface at `/admin/config/ai/explorers/summarize_generator`. Users enter text to summarize, optionally add instructions like "summarize in bullet points," and select a provider and model. The explorer handles empty input validation, configuration errors, and API exceptions with clear error messages.

The input object accepts an optional prompt parameter because some summarization models support guidance while others do not. This flexibility accommodates both specialized single-purpose models and general LLMs that implement summarization. The output normalizes to a plain string summary, consistent with other operation types.

Providers like Huggingface can now offer summarization-specific models (such as `facebook/bart-large-cnn`) separately from their chat models. This enables sites to use smaller, faster, or more accurate models for summarization tasks.
