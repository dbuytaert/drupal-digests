## Summary

The `ai_provider_huggingface` module now supports the `text_classification` operation type introduced by the AI core module. Site builders can add a Hugging Face text-classification endpoint (such as `distilbert/distilbert-base-uncased-finetuned-sst-2-english`) on the provider settings page and then use the Text Classification Explorer at `/admin/config/ai/explorers/text_classification_generator`. The module's `drupal/ai` dependency has been raised to `^1.4.x-dev@dev`, which is the version that ships the text-classification operation type classes.

## Impact

Affects module maintainers and site owners.

- **Requirements:** Minimum `drupal/ai` version raised from `^1.3.x-dev@dev` to `^1.4.x-dev@dev`.
- **New API:** New `text_classification` operation type available in `HuggingfaceProvider`, implementing `TextClassificationInterface` from `drupal/ai` 1.4.x.

## Technical details

The `HuggingfaceProvider` class now implements `TextClassificationInterface` and lists `text_classification` in both its `$supportedTypes` map (with Hugging Face pipeline filter `text-classification`) and its `getSupportedOperationTypes()` return value. The new `textClassification()` method accepts a string or `TextClassificationInput`, calls `HuggingfaceApi::textClassification()`, decodes the JSON response, handles both single-item and array responses from the Inference API, and wraps each result in a `TextClassificationItem` before returning a `TextClassificationOutput`.

A pre-existing bug in `HuggingfaceApi::textClassification()` was fixed at the same time: the method always included `'parameters' => []` in the request payload. PHP's `json_encode` serializes an empty array as `[]` (a JSON list), which the Hugging Face Inference API rejects with a `400` error and the message `'list' object has no attribute 'setdefault'`. The fix omits the `parameters` key entirely when the array is empty; when parameters are present they are cast to `(object)` so they serialize as a JSON object `{}`.

The `composer.json` dependency on `drupal/ai` was bumped from `^1.3.x-dev@dev` to `^1.4.x-dev@dev` because `TextClassificationInterface`, `TextClassificationInput`, `TextClassificationItem`, and `TextClassificationOutput` were added to AI core in `1.4.x` via [#3456768](https://www.drupal.org/node/3456768).

## Upgrade
