## Summary

Drupal CMS now offers a free trial AI provider called amazee.ai that works immediately after installation without requiring an API key. This makes it easier to try AI-powered features like automatic alt text generation and the AI assistant chatbot during setup. Site builders can still choose OpenAI or Anthropic by providing their own API keys, but amazee.ai is now the default option and provides anonymous usage with a limited free trial that can be extended by registering for an account.

## Impact

No upgrade or configuration changes required.

## Technical details

The `drupal_cms_ai` recipe now installs and configures the `ai_provider_amazeeio` module alongside the existing OpenAI and Anthropic providers. The recipe dependency versions were updated to require `drupal/ai` version 1.2.6 or higher and `drupal/ai_provider_amazeeio` version 1.2.6 or higher to ensure compatibility.

The recipe's input configuration was restructured to make amazee.ai the default provider instead of OpenAI. The provider selection radio buttons now list amazee.ai first, with updated help text explaining that no API key is required for the free trial. The prompts and form descriptions for the OpenAI and Anthropic API key fields were reworded to clarify that these fields can be left empty if using amazee.ai.

Configuration actions were added under `ai_provider_amazeeio.settings` to call the `setupAiProvider` action with `provider: amazeeio` and a new `no_key_needed: true` flag, plus `ensureAmazeeAiAccess: ~` to trigger the anonymous trial access setup. This pattern differs from OpenAI and Anthropic configuration, which still require `key_value`, `key_name`, and `key_label` parameters.

The changes required corresponding updates in the AI module ([#3567649](https://www.drupal.org/project/ai/issues/3567649)) to support configuring providers without API keys and in the amazee.io provider module ([#3566607](https://www.drupal.org/project/ai_provider_amazeeio/issues/3566607)) to set default model mappings.
