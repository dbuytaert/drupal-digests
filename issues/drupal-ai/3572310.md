## Summary

The FlowDrop module now uses the latest version of the standalone FlowDrop UI library instead of an outdated version with custom modifications. This update brings better visual workflow editing with improved automatic node placement, more intuitive drag-and-drop connections, and cleaner spacing between workflow components. The module now relies on the external library's built-in features rather than maintaining custom code, making future updates easier and more reliable.

## Impact

No upgrade or configuration changes required.

## Technical details

The FlowDrop UI library was extracted into a standalone project, and the Drupal module was updated to consume it as an external dependency. This required removing local patches and custom code that is now handled by the library itself.

The module migrated from deprecated API methods to current ones: `updateNode` replaced `updateNodePosition`, and `updateEdges` replaced `addEdge`. Schema definitions moved from the non-standard `enum`/`enumNames` pattern to JSON Schema's standard `oneOf` with `const`/`title` pairs. Textarea fields now use `format = "multiline"` as the proper schema attribute.

The custom auto-attach and drag-over-highlight system was replaced with FlowDrop UI's native proximity connect feature. The sidebar agent definitions were updated to include a `tools` output port so the proximity system can detect tool-compatible connections. A new `/api/flowdrop-agents/categories` endpoint provides category grouping for the sidebar.

Bidirectional mapping was added between FlowDrop's `instanceDescription` variable and Drupal's `description_override` field, affecting the PHP loader, mapper, save controller, and workflow parser. Layout spacing was increased and a staggered column system prevents accidental port alignment. The proximity connect distance was reduced from 200px to 50px to avoid unintended connections during repositioning.
