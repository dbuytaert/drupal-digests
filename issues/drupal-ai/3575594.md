## Summary

The MDXEditor rich-text field in the AI module was too short when empty, making it difficult to work with. A `min-height: 240px` CSS rule (roughly 8 lines of text) has been added so the editor presents a usable amount of space before any content is entered. This is a visual-only change with no functional impact.

## Impact

No upgrade or configuration changes required.

## Technical details

The MDXEditor component rendered its `[contenteditable]` area with no minimum height, so the editor collapsed to nearly nothing when the field was empty. The fix adds one CSS rule to the AI Core module's MDXEditor stylesheet targeting `.mdxeditor .mdxeditor-rich-text-editor [contenteditable]` and setting `min-height: 240px`. The selector scope (`.mdxeditor` parent plus `.mdxeditor-rich-text-editor` wrapper) ensures the rule applies only inside the rich-text editing surface and does not affect other contenteditable elements on the page. The change was committed to the 1.x, 1.3.x, 1.4.x, 2.0.x, and 2.x branches.
