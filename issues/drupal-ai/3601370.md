## Summary

The one_line_installer now offers the Agent-Ready Drupal Build Kit as an optional extra during setup. When selected, the installer adds a pre-packaged agent skill to every coding agent the user chose (Claude Code, Codex, OpenCode), giving the agent a defined workflow for content modelling, verification, and review-packet generation when rebuilding an existing website as a Drupal CMS site. The extra is opt-in (not installed by default) and requires the Drupal CMS profile; choosing it on a `standard` or `minimal` profile skips installation and prints a prominent warning.

## Impact

Affects module maintainers.

- **New API:** New `agent_ready_drupal_build_kit` installer extra: selectable via `--extras agent_ready_drupal_build_kit` or the interactive menu; requires the Drupal CMS profile (`cms` or `template`)

## Technical details

The extra is delivered as `src/lib/extras/extra_agent_ready_drupal_build_kit.sh`. The installer discovers extras at runtime by scanning the function namespace for `extra_<id>_meta` symbols (see `_extras_discover.sh`), so no prompt, global, or pipeline change was needed beyond dropping in the new file.

Unlike the other extras (`dtk`, `droost`, `tool_api`), this one installs no Drupal module. It calls `npx --yes skills add 'https://github.com/scottfalconer/agent-ready-drupal-build-kit'` inside the DDEV web container, iterating over `_skill_agent_targets` to register the skill for each chosen agent. After installation it appends a note to the managed block in `AGENTS.md`.

The `extra_agent_ready_drupal_build_kit_meta()` function omits the `default` flag so the extra starts unticked in the interactive menu and is not installed under `--yolo` / unattended mode. A profile check gates execution: `PROFILE` must be `cms` or `template` (both scaffold `drupal/cms`); any other profile causes `_build_kit_skipped_banner` to print a bold-red ASCII block and returns early. The extra can also be selected non-interactively with `--extras agent_ready_drupal_build_kit` or added on top of the defaults with `--add-extras agent_ready_drupal_build_kit`.

## Upgrade

No upgrade steps needed for existing sites. To use the new extra on a fresh install:

```bash
# Interactive menu — the extra appears as an opt-in choice
bash <(curl -fsSL https://project.pages.drupalcode.org/one_line_installer/drupalaibp)

# Or select it non-interactively (Drupal CMS profile is required)
drupalaibp --profile cms --extras agent_ready_drupal_build_kit
```
