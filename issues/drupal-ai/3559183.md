## Summary

The add/edit Agent form in the AI Agents module has been redesigned to be significantly less overwhelming. Tool configuration is now shown in lightweight MicroModal pop-ups rather than expanding inline on the main form. The system prompt textarea now uses the MDXEditor component with token typeahead. The structured output schema field now uses the dedicated `ai_json_schema` element. Advanced settings (max loops, default information tools, orchestration/triage classification flags, and structured output) are collapsed into an "Advanced settings" section by default. Per-property tool restrictions are displayed as a compact table. The minimum required version of the `drupal/ai` base module is now `^1.3.0`.

## Technical details

The agent entity form class `AiAgentForm` (`src/Form/AiAgentForm.php`) was substantially restructured. The most visible change is that each selected tool's configuration is rendered inside a MicroModal dialog: `createToolUsageForm()` now wraps the tool container in modal markup using namespaced CSS classes (`.ai-modal__overlay`, `.ai-modal__container`, `.ai-modal__header`, `.ai-modal__content`) to avoid collisions with themes or other modules. A "Configure" button is injected into each tool card in the tool library picker via a new `hook_preprocess_ai_tools_library_item()` implementation in `ai_agents.module`. The button carries a `data-tool` attribute and the `.dynamic-tool-modal` class that the JS uses to call `MicroModal.show(id)`.

MicroModal 0.4.10 is bundled as `js/micromodal.min.js`, sourced via npm (`package.json`) and committed to the repository. This was preferred over a `libraries.yml` CDN reference or Composer asset packagist in order to keep the library and its license metadata available locally without external CDN dependencies or privacy concerns. A companion `css/micromodal.css` provides the `.ai-modal__*` styles, with z-index 1300 for the overlay and 1310 for the fixed header (calibrated against Drupal core toolbar and Gin theme layers), plus Gin dark-mode overrides using `.gin--dark-mode` and Gin CSS custom properties.

`js/agents_form.js` was rewritten from jQuery to vanilla JS using Drupal's `once()` API under two `Drupal.behaviors`. `openToolsModal` initialises MicroModal and attaches click handlers to `.dynamic-tool-modal` buttons. `aiAgentsToolUsageRefresh` uses a `MutationObserver` on `document.body` to detect when the jQuery UI tool-selection dialog is removed from the DOM, then dispatches a `mousedown` event on the `[data-ai-tools-library-form-element-update]` element to trigger an AJAX refresh. This indirection is necessary because the jQuery UI `dialog:afterclose` event is unavailable without loading jQuery. The library definition in `ai_agents.libraries.yml` now declares `core/drupal` and `core/once` as explicit dependencies.

Form field reorganisation: `max_loops` (default raised from 3 to 10), `default_information_tools`, the `orchestration_agent` and `triage_agent` checkboxes (now placed under a `classification` sub-details), and the structured output block are moved under a new collapsed `prompt_detail > advanced` details element. Per-property restrictions changed from individual fieldsets to a `#type table` element with the `.ai-agent-property-restrictions-table` class. The structured output schema field changed from `#type textarea` to `#type ai_json_schema`. Tool settings (`return_directly`, `require_usage`, `description_enabled`, `description_override`, `progress_message`, `use_artifacts`) are now nested under `tool_usage[$tool_id][tool_settings]`; property restriction values are at `tool_usage[$tool_id][property_restrictions][table][$property_name][restrictions]`. The `save()` method was updated to read from these new paths.

A static `afterBuildToolsLibrary()` `#after_build` callback sets `#limit_validation_errors = []` on the AJAX `update_widget` button so the form can rebuild after tool selection without requiring all fields to pass validation first.

The `system_prompt` textarea now carries a `data-mdxeditor` attribute and passes `drupalSettings` config that includes a typeahead plugin populated from `\Drupal::service('token')->getInfo()`, enabling inline token autocomplete in the MDXEditor component. The `Token` service is injected into `AiAgentForm` via its constructor. The `AiAgent` entity properties `$orchestration_agent` and `$triage_agent` were changed from `bool` to `?bool = NULL` to prevent PHP errors on agents that pre-date those fields.

The minimum `drupal/ai` requirement in `composer.json` was bumped from `^1.2.0` to `^1.3.0` to cover dependencies on newer AI module form elements and the MDXEditor component.

Three follow-up issues were filed for work delegated to the `drupal/ai` upstream: sticky category tabs in the tool library modal ([#3579424](https://www.drupal.org/node/3579424)), a trash-icon replacement for the tool removal button ([#3579423](https://www.drupal.org/node/3579423)), MDXEditor dark-mode rendering ([#3582474](https://www.drupal.org/node/3582474)), and Tool Explorer dark-mode rendering ([#3582473](https://www.drupal.org/node/3582473)).
