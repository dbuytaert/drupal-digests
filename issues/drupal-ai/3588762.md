## Summary

Long AI-assisted migration sessions trigger context compaction, which strips specific technical facts from the agent's working memory. After compaction the agent would re-attempt already-known broken patterns, such as passing multiple `-c` flags to `canvas:upload`, or calling `ToolSearch` dozens of times to reload tool schemas it had already resolved. This change introduces a `gotchas.md` artifact and two new protocol sections in `SKILL.md` so that hard-won facts survive compaction and are available at the start of every session.

## Impact

Affects module maintainers.

- **New API:** New `docs/migration/gotchas.md` runtime artifact and `Session Start` / `Compaction Protocol` sections in `SKILL.md` define a persistent memory contract for the migrate-site skill.

## Technical details

Three coordinated changes landed in commit `44d2d74f`:

A versioned template at `.claude/skills/migrate-site/references/gotchas.md` ships with the skill, pre-populated with known facts across four sections: Canvas CLI (e.g. `canvas:upload` takes one `-c` flag at a time; `canvas:download` overwrites local files without warning), Runtime (e.g. `getNodePath()` can return the truthy string `'#'`; Tailwind directives only work in `global.css`), Acquia Platform (e.g. menu JSON:API responses are CDN-cached; media style URLs return thumbnails rather than originals), and OAuth scope vs. endpoint mismatches (e.g. `canvas_oauth:menu_link_content` mints a token but `POST /jsonapi/menu_link_content` is 404 on Acquia Source). The template is the stable, version-controlled baseline; per-migration discoveries are appended only to the runtime copy at `docs/migration/gotchas.md`.

A "Session Start" preflight block is inserted into `SKILL.md` before Phase 0. It defines four mandatory steps: (1) copy the template to `docs/migration/gotchas.md` if the file does not yet exist, or preserve the existing copy if the session is being resumed; (2) read the runtime copy fully into context; (3) pre-load tool schemas via `ToolSearch select:<tool_name>` so `mcp__chrome-devtools__*` and other frequently used schemas are ready without reactive reloading; (4) run an orphan safeguard that detects a `migration_completed` event in `state.jsonl` without a matching PASS record in `09-migration-complete-checklist.json` and routes the session into `RECOVERY_REQUIRED` rather than re-emitting a false completion signal. The orphan safeguard implementation lives in `.claude/agents/artifact-checker.md` under mode `session-start`.

A "Compaction Protocol" section in `SKILL.md` requires that every compaction summary embed the full, verbatim contents of `docs/migration/gotchas.md` inside a `HARD-WON FACTS (preserve verbatim)` delimited block. After compaction the agent must re-execute the Session Start procedure before resuming any phase. Commit `d031b00c` adds a Vitest prose-sanity suite that asserts structural properties of both the template and the updated `SKILL.md`.
