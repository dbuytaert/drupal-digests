## Summary

Long AI migration sessions that trigger context compaction were causing the agent to lose hard-won technical facts: correct `canvas:upload` syntax, runtime API quirks, Acquia Platform gotchas. After compaction, the agent would re-discover and re-debug the same issues from scratch. This change introduces a persistent `gotchas.md` artifact and a formal Session Start preflight protocol so that accumulated knowledge survives compaction boundaries.

## Impact

Affects module maintainers.

- **New API:** New `Session Start` preflight protocol and `Compaction Protocol` section in `SKILL.md`; new versioned template at `.claude/skills/migrate-site/references/gotchas.md`

## Technical details

Two new sections are added to `SKILL.md`. The **Session Start** preflight runs before Phase 0 and after every compaction: (1) it copies `.claude/skills/migrate-site/references/gotchas.md` to `docs/migration/gotchas.md` if the runtime copy does not yet exist, or preserves the existing runtime copy if the session is resumed; (2) it reads the runtime copy fully into working context; (3) it pre-loads tool schemas via `ToolSearch select:<tool_name>` so schemas like `mcp__chrome-devtools__*` are available without reactive re-loading; (4) it runs an orphan safeguard that detects a `migration_completed` event in `state.jsonl` without a passing `09-migration-complete-checklist.json` and marks the migration `RECOVERY_REQUIRED`, dispatching `artifact-checker` with `mode: session-start`.

The **Compaction Protocol** section requires that any compaction summary embed the full, verbatim contents of `docs/migration/gotchas.md` inside a `HARD-WON FACTS (preserve verbatim)` delimited block. This redundantly preserves the facts in the compaction artifact itself. After compaction, the Session Start procedure is re-executed before the prior phase resumes.

The new template file at `.claude/skills/migrate-site/references/gotchas.md` ships pre-populated with four sections: Canvas CLI (e.g., `canvas:upload` runs no validation before upload; `canvas:download` overwrites local edits silently), Runtime (e.g., `getNodePath()` returns the truthy string `'#'`; Tailwind directives only work in `global.css`), Acquia Platform (e.g., menu JSON:API responses are CDN-cached; media style URLs return thumbnails rather than originals), and OAuth scope vs. endpoint mismatches (e.g., `canvas_oauth:menu_link_content` mints a token but `POST /jsonapi/menu_link_content` is 404 on Acquia Source). The runtime copy under `docs/migration/` is the per-migration working memory; the skill-dir file is the versioned template. Phase 0.5 (verify-scopes preflight) appends session-discovered mismatches to the runtime copy only.

A Vitest test suite (commit `d031b00c`) validates that the template file exists with the four required sections and at least one bullet per section, and that `SKILL.md` contains the Session Start and Compaction Protocol blocks with the expected prose patterns.

## Upgrade

No changes to existing site or developer code are required. For maintainers of the `canvas_ai_migrations` skill: the new session initialization contract means any custom orchestrator that wraps `SKILL.md` must execute the four Session Start steps before dispatching agents. The runtime artifact layout gains a new file:

```
docs/migration/gotchas.md   ← runtime working copy, seeded from template
.claude/skills/migrate-site/references/gotchas.md   ← versioned template (do not append here)
```

When a new gotcha is discovered during a session, append it to `docs/migration/gotchas.md` (the runtime copy), never to the template.
