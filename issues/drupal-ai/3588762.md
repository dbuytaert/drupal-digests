## Summary

The `canvas_ai_migrations` migrate-site skill now ships a persistent `gotchas.md` knowledge file and a mandatory Session Start preflight procedure. Before these changes, AI agent context compaction silently discarded hard-won technical facts (correct CLI syntax, API quirks, OAuth scope gaps) so every new session re-discovered and re-debugged the same issues from scratch. The fix introduces a versioned template at `.claude/skills/migrate-site/references/gotchas.md`, pre-populated with known gotchas, and a per-migration working copy at `docs/migration/gotchas.md` that accumulates session-discovered facts. The agent now reads this file before any work begins and after every compaction event.

## Technical details

Three coordinated additions landed in `SKILL.md` and `references/gotchas.md`:

**`gotchas.md` template.** Added at `.claude/skills/migrate-site/references/gotchas.md`. Organized into four sections: Canvas CLI (upload/download pitfalls), Runtime (`getNodePath()` truthy-`'#'` trap, Tailwind `@apply` rules), Acquia Platform (CDN caching, media style URL thumbnails), and OAuth scope/endpoint mismatches (`canvas_oauth:menu_link_content` mints a token but `POST /jsonapi/menu_link_content` is 404). Each entry is a one-line fact plus a one-line symptom description. Phase 0.5 (verify-scopes preflight) appends additional scope mismatches to the per-migration runtime copy; the skill template stays minimal.

**Session Start preflight block in `SKILL.md`.** Runs once at session start and again after every compaction. Four ordered steps: (1) copy the template to `docs/migration/gotchas.md` if the runtime copy is absent, otherwise preserve it (session-appended facts must not be overwritten); (2) read `docs/migration/gotchas.md` fully into context; (3) pre-load tool schemas via `ToolSearch select:<tool_name>` for at least `mcp__chrome-devtools__*` tools; (4) run the orphan safeguard (dispatches `artifact-checker` with `mode: session-start` to detect a `migration_completed` event in `state.jsonl` that lacks a corresponding `PASS` in `09-migration-complete-checklist.json`, and enters `RECOVERY_REQUIRED` if so).

**Compaction Protocol section in `SKILL.md`.** Requires any post-compaction summary to include a verbatim `HARD-WON FACTS (preserve verbatim)` delimited block containing the full unabridged contents of `docs/migration/gotchas.md`. After compaction the Session Start procedure re-runs in full to restore context.

Vitest prose-sanity tests in the second commit verify file existence, the four required sections each containing at least one bullet, and that `SKILL.md` carries all required headings and key phrases for the Session Start and Compaction Protocol blocks.

## Upgrade

No external API changed. Developers using the migrate-site skill should be aware of the new session-start contract:

- At session start (and post-compaction) the agent will now copy or read `docs/migration/gotchas.md`.
- Append session-specific gotchas to the **runtime copy** (`docs/migration/gotchas.md`), never to the skill template.
- The skill template lives at `.claude/skills/migrate-site/references/gotchas.md` and should only be updated via versioned commits to the skill itself.
