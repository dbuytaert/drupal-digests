## Summary

The AI Chatbot block now shows a better user interface when configuring placement outside the toolbar. When you choose "Bottom right" or "Bottom left" placement, the configuration form now hides options that only apply to toolbar placement and sets sensible defaults. The chatbot displays at 500px width and height instead of the previous "auto" and "100%" values that caused overlapping elements and a cramped appearance in the Olivero theme.

## Impact

No upgrade or configuration changes required.

## Technical details

The `DeepChatFormBlock` class in the `ai_chatbot` module had default width and height values of "auto" and "100%" that worked well for toolbar placement but caused rendering issues when the chatbot was placed in the bottom-right or bottom-left corners. These defaults made the chatbot appear small and caused overlapping elements in themes like Olivero.

The fix changes the default width and height from "auto" and "100%" to "500px" for both dimensions. The block configuration form now uses Form API `#states` to conditionally hide the Style, Width, Height, and Collapsed minimal fields when Placement is set to "Toolbar". The Placement field was moved to the top of the Styling fieldset to make this dependency clear.

When rendering the block, the `build()` method now checks the placement configuration. If placement is "toolbar", it overrides the configured width and height to use "100%" and "auto" respectively. For bottom-right and bottom-left placements, it uses the configured pixel values. This conditional logic applies to both the drupalSettings and inline styles.
