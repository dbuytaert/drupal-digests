## Summary

The Context Control Center module now works with Drupal CMS 2.0. Previously, installing the module failed due to conflicting version requirements for the ECA module. The dependency conflict has been resolved by removing the ECA dependency and updating version constraints to support both Drupal 10.5+ and Drupal 11.2+. This enables site owners using Drupal CMS 2.0 to install and use the Context Control Center to manage AI context used in agent prompts.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue originated from a version conflict where the module required `drupal/eca` ^2.1, but Drupal CMS 2.0 required `drupal/eca` ^3.0.8. The upstream AI module (1.3.x) removed its ECA dependency, which allowed this module to do the same via [#3577426](https://www.drupal.org/node/3577426).

The module's `composer.json` was updated to broaden compatibility. The `drupal/core` requirement changed from `^11.2` to `^10.5 || ^11.2`, adding Drupal 10.5+ support. The `drupal/ai` requirement changed from `^1.3` to `>=1.3.0-rc2@RC || ^2` to ensure compatibility with AI Core RC2+ (needed for styling and icon work) while also allowing version 2.x.

The `ai_context.info.yml` file was updated to reflect the new core version requirement: `^10.5 || ^11.2` instead of `^10.4 || ^11`. This change ensures the module properly declares its compatibility with both Drupal 10.5+ and 11.2+, enabling installation on Drupal CMS 2.0 environments.
