## Summary

This is an internal tooling change. The `scripts/tag-release.sh` script in the Drupal AI module used bash/zsh-specific constructs (`pushd`/`popd`, `[[ ]]`, `read -p`) despite its `#!/bin/sh` shebang. On Linux, where `/bin/sh` is typically a minimal POSIX shell like dash, this caused errors such as "pushd: not found". The script was rewritten to use only POSIX-compliant shell constructs so it works on both Linux and macOS. The required Node.js version was also updated from v20 to v24+.

## Impact

No upgrade or configuration changes required.

## Technical details

The `tag-release.sh` script automates creating tagged releases of the Drupal AI module. It clones the repo, builds JS UIs in three subdirectories (`modules/ai_ckeditor`, `ui/mdxeditor`, `ui/json-schema-editor`), commits the build artifacts, creates a git tag, removes the artifacts, and pushes.

The original script declared `#!/bin/sh` but relied on bash/zsh features. On Linux where `/bin/sh` is typically dash, `pushd`/`popd` are unavailable, causing the npm commands to run in the wrong directory and fail with "ENOENT: no such file or directory" errors.

The rewrite makes the script POSIX-compliant. `pushd`/`popd` for directory changes during builds are replaced with `npm --prefix "$DIR"` in the new `build_dir()` helper function, which avoids directory changes entirely. The `reset_version()` function uses `cd`/`cd -` instead. Bash-specific `[[ ]]` conditionals become `[ ]`, `read -p` becomes `printf` followed by `read`, and `echo` with escape sequences becomes `printf`. Color variables are assigned via `$(printf '\033[...')` subshells instead of bare escape-sequence strings.

Other improvements bundled in: a `check_dependency()` function validates node and npm versions up front using POSIX `case` pattern matching. The minimum Node.js version was raised from v20 to v24, and an npm >= v11 check was added. The macOS-only `open` command now falls back to `xdg-open` on Linux. `git tag "$TAG" HEAD` was changed to `git tag -a "$TAG"` to create annotated tags that open an editor for the tag message. Error handling was added with `|| exit 1` throughout the script.

The `ui/json-schema-editor/package.json` dependency keys were reordered alphabetically. The `package-lock.json` was regenerated, adding `resolved`/`integrity` fields and picking up minor dependency version bumps (rollup 4.57.1 to 4.59.0, etc.).
