## Summary

The OpenAI AI provider module now supports the `image_to_image` operation type, enabling developers to use OpenAI's image editing endpoint (used by `gpt-image-1` and `dall-e-2`) to modify existing images with a text prompt and an optional mask. Previously the module only supported generating images from text (`text_to_image`). Sites that want to offer image editing functionality through the AI module ecosystem can now wire up these models without building a custom provider.

## Impact

Affects module maintainers.

- **New API:** New `image_to_image` operation type on `OpenAiProvider`, implementing `ImageToImageInterface` with support for prompt-driven image editing and optional mask via OpenAI's images-edit endpoint.

## Technical details

`OpenAiProvider` now implements `ImageToImageInterface` and uses `ImageToImageTrait`. The new `imageToImage()` method only accepts an `ImageToImageInput` object (plain string or array inputs throw `AiResponseErrorException`), which carries the source image binary, an optional mask `ImageFile`, and the edit prompt. Because the OpenAI PHP SDK uploads files as multipart form data, the method writes the image (and mask, if supplied) to `temporary://` paths via `fileSystem->saveData()`, opens them as PHP file resources, and passes them to `$this->client->images()->edit()`. A `finally` block guarantees the temporary files and open handles are cleaned up regardless of outcome. The response bodies are decoded from base64 JSON; output MIME type defaults to `image/png` but switches to `image/jpeg` or `image/webp` when `output_format` is set in the configuration payload. The provider declares `requiresImageToImagePrompt()` as `TRUE` (the edits endpoint always requires a prompt) and `hasImageToImageMask()` as `TRUE`. Model filtering in the provider's model-listing logic restricts `image_to_image` to models matching `dall-e-2` or `gpt-image-*`, explicitly excluding `dall-e-3` which cannot edit images. The default model recommendation for this operation type is `gpt-image-1`. `api_defaults.yml` gained an `image_to_image` section covering `n` (1–10) and `size` configuration keys.
