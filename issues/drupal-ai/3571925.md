# #3571925: Output streaming is handled inconsistently, fails to work in API AI-explorer

## Summary

Output streaming for AI responses was failing in certain contexts like the API AI Explorer when using nginx web servers. The module was setting HTTP headers inconsistently across different integration points, missing critical headers like X-Accel-Buffering that control server-side buffering. This caused streamed responses to buffer completely before displaying, defeating the purpose of streaming. A new centralized response class now ensures all AI streaming responses use consistent headers and configuration, making streaming work reliably across all integration points regardless of web server setup.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI module had four different locations implementing streamed responses, each with different header configurations. Some set X-Accel-Buffering: no (required for nginx), others did not. Cache-Control and Content-Type headers varied, and output buffer handling was inconsistent. The ai_ckeditor integration worked because it set the nginx header, while ai_api_explorer and some ai_chatbot contexts failed.

The fix introduces a centralized AiStreamedResponse class extending Symfony's StreamedResponse. This class sets all required headers by default: X-Accel-Buffering: no for nginx compatibility, Cache-Control: no-cache to prevent caching, and appropriate Content-Type headers. It also handles output buffer cleanup with ob_end_clean. All four streaming locations now use this single class, ensuring consistent behavior.

The Content-Type handling was also corrected. CKEditor was incorrectly using text/event-stream without actually implementing Server-Sent Events, while DeepChat was sending SSE data without the proper header. The standardized class resolves these inconsistencies.
