# #3571925: Output streaming is handled inconsistently, fails to work in API AI-explorer

- **Project:** Drupal AI

- **Version:** 1.3.x-dev
- **Status:** Fixed
- **Discussion:** [#3571925](https://www.drupal.org/node/3571925) · 4 contributors · 13 comments

## Summary

Output streaming for AI responses was failing in the API AI Explorer and handled inconsistently across different AI module integrations when using web servers like nginx. The four different places where streaming responses were generated each set different HTTP headers or none at all, causing streaming to fail in some contexts. A new centralized `AiStreamedResponse` class now ensures all streaming responses include the necessary headers to work reliably across different server configurations and integrations.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI module generates streaming responses in four locations: CKEditor integration, DeepChat API, ChatForm, and API Explorer. Each implemented streaming differently with inconsistent HTTP headers. Some set `X-Accel-Buffering: no` for nginx compatibility, others did not. Cache control headers varied between `no-cache,must-revalidate`, `no-cache`, or missing entirely. Content-type headers were also inconsistent, with some incorrectly setting `text/event-stream` when not actually using Server-Sent Events.

The fix introduces a centralized `AiStreamedResponse` class that extends Symfony's `StreamedResponse`. This class sets all required headers by default: `X-Accel-Buffering: no` for nginx/Apache streaming support, `Cache-Control: no-cache` to prevent response caching, and appropriate content-type headers. All four streaming locations now use this unified class, ensuring consistent behavior across all AI integrations. The `X-Accel-Buffering` header was the critical missing piece that prevented streaming from working in nginx environments, which is why API Explorer failed while CKEditor worked.
