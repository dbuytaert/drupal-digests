## Summary

Multi-line text fields in AI module configuration now export cleanly to YAML files. Previously, long prompts and instructions stored in configuration appeared as single lines with `\r\n` line breaks, making them difficult to review in version control. The module now normalizes all line endings to `\n`, so exported YAML displays multi-line text as readable blocks. This makes configuration changes easier to review in pull requests and configuration diffs, especially for AI assistant prompts and instructions that can span many lines.

## Impact

No upgrade or configuration changes required.

## Technical details

The change adds `#normalize_newlines` and a custom `#value_callback` to every textarea field across the AI module that stores configuration. The callback is implemented in a new `Textarea` utility class that converts `\r\n` and `\r` line endings to `\n` before form values are processed. This ensures consistent line endings regardless of the browser or operating system used to edit the configuration.

The implementation affects textarea fields in `AiAssistantForm`, `AiGuardrailForm`, `AiPromptSubform`, automator plugins, CKEditor plugins, and guardrail plugins. Each textarea receives both the `#normalize_newlines` property and the `#value_callback` pointing to `Textarea::valueCallback()`.

The custom callback is a temporary measure copied from the pending Drupal core issue [#3202631](https://www.drupal.org/project/drupal/issues/3202631). Once that core feature lands and the AI module's minimum Drupal version includes native `#normalize_newlines` support, the custom `#value_callback` can be removed while keeping the property declaration. The code includes extensive comments explaining this transition plan at each usage site.

The change also fixes a typo where `$typehead_config` was renamed to `$typeahead_config` in `AiPromptSubform`.
