## Summary

Multi-line text in exported configuration files is now easier to read and review. When you configure AI prompts, assistant instructions, or automator settings, the exported YAML files will properly preserve line breaks instead of showing everything on a single line. This makes it much easier to track changes in version control and review updates to long pieces of text like prompts and instructions.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI module now normalizes newlines in all textarea form elements across assistant configuration, automators, CKEditor AI plugins, guardrails, and prompt entities. Each textarea receives the `#normalize_newlines` property set to `TRUE` and a custom `#value_callback` pointing to `Textarea::valueCallback()`.

The `Textarea` utility class provides a custom value callback that converts all Windows-style (`\r\n`) and Mac-style (`\r`) line endings to Unix-style (`\n`) before saving. This ensures exported YAML configuration files use consistent newline characters that YAML interprets as multi-line strings rather than escaped literals.

The implementation is temporary. The code includes extensive comments explaining that `#normalize_newlines` support is landing in Drupal core via [#3202631](https://www.drupal.org/project/drupal/issues/3202631). Once the minimum supported Drupal version includes this property natively, the custom `#value_callback` can be removed while keeping the `#normalize_newlines` property.

Changes span multiple submodules including `ai_assistant_api`, `ai_automators`, `ai_ckeditor`, and core forms for guardrails and prompts. The diff also includes minor fixes like a typo correction from `$typehead_config` to `$typeahead_config` in `AiPromptSubform` and adding "mdxeditor" to the spell-check ignore list.
