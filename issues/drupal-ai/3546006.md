## Summary

Multi-line text fields in AI module configuration now export to YAML files in a human-readable format with proper line breaks. Previously, long prompts and instructions saved with Windows-style line endings appeared as single lines in exported configuration files, making it difficult to review changes in version control or configuration management tools. The module now normalizes all line endings to Unix-style format before saving, ensuring that multi-line text displays properly in YAML exports.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI module adds a custom `Textarea` utility class with a `valueCallback()` method that normalizes line endings from `\r\n` (Windows) and `\r` (classic Mac) to `\n` (Unix) when the `#normalize_newlines` property is set to `TRUE`. This callback is applied to all textarea form elements that store configuration data, including assistant instructions, prompts, error messages, guardrail patterns, and CKEditor AI plugin prompts.

The implementation uses both `#normalize_newlines` and `#value_callback` properties on textarea elements. The normalization property is being added to Drupal core in [#3202631](https://www.drupal.org/project/drupal/issues/3202631). Once that lands and becomes the minimum required Drupal version, the custom value callback can be removed and only the property will remain.

The changes affect approximately 30 textarea fields across multiple submodules: `ai_assistant_api` (assistant instructions and error messages), `ai_automators` (tool descriptions and field prompts), `ai_ckeditor` (plugin prompts), and core AI forms (guardrail configurations and prompt entities). The `Textarea::valueCallback()` method is copied from the core issue until the feature becomes available in Drupal core.
