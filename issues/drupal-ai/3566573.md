## Summary

The AI Automators module gained support for the Address field type when rendered with the `address_default` widget. Site owners can now configure an AI Automator to parse free-text input containing an address and populate all sub-fields of an Address field automatically. The Address module must be enabled for this feature to be available. The change landed in both 1.3.x and 2.0.x.

## Impact

No upgrade or configuration changes required.

## Technical details

AI Automators exposes a Field Widget Actions (FWA) plugin system that lets AI-derived values be written directly into a widget's sub-fields. Before this change, no FWA plugin existed for the `address` field type and its `address_default` widget, so AI Automators could not populate Address fields inline.

The fix adds a new FWA plugin for that field/widget combination, modeled after the existing `modules/ai_automators/src/Plugin/FieldWidgetAction/Email.php` plugin. The plugin declares a dependency on the Address module (which provides both the `address` field type and the `address_default` widget) and is only active when that combination is in use. Example documentation following the conventions of `docs/modules/ai_automators/examples/` was added alongside the plugin so developers can see a working automator configuration. The change was merged to 1.3.x and cherry-picked to 2.0.x.
