## Summary

The `ai_recipe_validations_image_safety` recipe previously required the `drupal_cms_media` recipe as a dependency, which pulled in many unrelated packages and had a broken Composer version constraint (`^1.0` while the package is now at 2.x). This made the recipe uninstallable via Composer. The dependency is now replaced with the leaner `core/recipes/image_media_type` recipe, which is what the recipe actually needs. The change also broadens compatibility so the recipe works on any Drupal site that uses the core image media type, not just Drupal CMS environments.

## Impact

No upgrade or configuration changes required.

## Technical details

In `composer.json`, the `drupal/drupal_cms_media: ^1.0` require entry is removed entirely. In the recipe YAML, the `recipes` list entry `drupal_cms_media` is replaced with `core/recipes/image_media_type`. The `drupal/core` and other direct dependencies (`drupal/ai`, `drupal/field_validation`) are unchanged. The `core/recipes/image_media_type` recipe sets up the Media: Image bundle that this recipe's field validations target, so it satisfies the actual runtime requirement without the overhead of the full Drupal CMS media stack. A secondary effect noted in the thread is that the old `^1.0` constraint had no matching release (drupal_cms_media skipped 1.x entirely and went straight to 2.x), causing a hard Composer resolution failure before this fix.
