## Summary

Installing `drupal/ai_best_practices` now automatically sets up your Drupal project's AI agent context. The Composer plugin creates `AGENTS.md`, `CLAUDE.md`, `GEMINI.md`, and `ai_best_practices.yaml` at your project root, discovers Agent Skills from installed Drupal modules and Composer packages, and copies them into `.agents/skills/`. Two commands are available after install: `composer drupal-ai skills-sync` refreshes skills and `composer drupal-ai skills-list` shows what would be installed. Running `composer remove drupal/ai_best_practices` reverses the `composer.json` changes and removes `ai_best_practices.yaml`. This migrates the orchestration engine from the now-sunsetted Surge project directly into this module so feedback and iteration can happen in one place.

## Impact

Affects module maintainers.

## Technical details

The MR adds a Composer plugin (`AiBestPracticesPlugin`) implementing `PluginInterface` and `EventSubscriberInterface`. To avoid running before autoload is dumped, it uses a two-phase deferred-sync pattern: `ComposerProjectSynchronizer::markSyncPendingFromPackageEvent()` writes a marker at `vendor/.ai_bp_sync_pending` during `POST_PACKAGE_INSTALL` / `POST_PACKAGE_UPDATE`, and `runDeferredSyncIfPending()` acts on it in the later `post-install-cmd` / `post-update-cmd` phase. The sync is skipped entirely when the `AI_BEST_PRACTICES_SKIP_PACKAGE_SYNC` or `AI_BEST_PRACTICES_SKIP_COMPOSER_MUTATION` environment variables are set.

`AgentsMdSync` manages a region in `AGENTS.md` bounded by `<!-- ai-best-practices:start -->` and `<!-- ai-best-practices:end -->` HTML comment markers. If the file does not yet exist, the full template from `.agents/AGENTS.md.template` is written; if it exists without markers, the managed block is prepended; if it exists with markers, only the inner region is replaced (idempotent: returns `null` and skips the write when the content is unchanged). `AgentRedirectFiles` copies `CLAUDE.md` and `GEMINI.md` from the package's `.agents/` directory to the project root only when those files are absent, printing a Composer warning if they already exist.

The skills subsystem (`SkillsScanner`, `SkillsAggregator`, `SkillValidator`) discovers skills from three priority tiers: project-level (`.agents/skills/`, `.claude/skills/`, `skills/`), Drupal modules under `modules/*` or `web/modules/*`, and Composer packages under `vendor/*`. Each skill directory is validated for a `SKILL.md` file with valid YAML frontmatter per the Agent Skills spec before being copied to the configured output directory (default `.agents/skills/`). The `SkillsListCommand` supports both `table` and `xml` output formats; the XML path is consumed by file-based AI agents that cannot call tools directly.

On `composer remove`, `ComposerConsumerRollback::stripPluginFromComposerData()` removes the `drupal-ai` script key (and the legacy `abp` key), the package's `config.allow-plugins` entry, and prunes now-empty `scripts`/`config` objects from `composer.json`; `ai_best_practices.yaml` is deleted separately. A known open issue noted during review: module-contributed skills are correctly copied to `.agents/skills/` but are not listed in the `AGENTS.md` managed region, so agents reading that file will not discover them.

## Upgrade

No existing code needs updating; this is all new. To add the plugin to an existing project:

```bash
composer config allow-plugins.drupal/ai_best_practices true
composer require --dev drupal/ai_best_practices
```

After install, these files appear at your project root and are safe to commit:

```
.agents/
AGENTS.md
CLAUDE.md
GEMINI.md
ai_best_practices.yaml
```

The `drupal-ai` script is also registered in your `composer.json` automatically. To refresh skills after adding or updating modules:

```bash
composer drupal-ai skills-sync
```

To remove everything the plugin added:

```bash
composer remove drupal/ai_best_practices
```
