# #3601369: Add the possibility to add one skill or extra to the default

## Summary

The `one_line_installer` tool now supports `--add-skills` and `--add-extras` flags that extend the default agent skills and extra Drupal installations rather than replacing them. Previously, using `--skills` or `--extras` required listing every value you wanted, which would discard the defaults. The new additive flags let you say "give me the defaults plus these specific additions" without having to repeat the full default list. Both flags also compose with a loaded config: `--add-skills` extends a config's `skills` list, and `--add-extras` extends a config's extras rather than overwriting them.

## Impact

Affects module maintainers.

- **New API:** New `--add-skills` and `--add-extras` CLI flags added to `setup-site`; they extend defaults (or a loaded config) rather than replacing them.

## Technical details

Two new flags are declared in `src/bashly.yml` and processed in `apply_cli_flags()` inside `src/lib/shared/prompts/apply_cli_flags.sh`.

`--add-skills` accepts a comma-separated list of `'repo|skill'` specifiers. When applied, it reads the default skill specs via `_default_skill_specs()` (or the config's `SKILLS_SPEC` array when `USE_CONFIG=1`), normalizes each added entry through `_skill_spec_normalize()`, then merges them with the base list using the new `_merge_added()` helper. The result is stored back in `SKILLS_SPEC` and `SKILLS_SET=1` is set, which causes `choose_skills` to skip the interactive prompt.

`--add-extras` accepts a comma-separated list of extra IDs. It selects the base from whichever is already set: if `--extras` was also passed (or a config is active), the existing `EXTRAS` array is the base; otherwise `_extras_default_ids()` supplies the defaults. `_merge_added()` appends the new entries, deduplicating against the base in insertion order, and stores the result in `EXTRAS` with `EXTRAS_SET=1`.

`_merge_added()` is a small helper that takes a CSV string plus a list of base items, walks both in order, and prints each unique item exactly once. It handles skill specs that contain `|` and `*` without word-splitting or glob expansion by never passing them through bare variable expansion.
