## Summary

The toolbar chatbot feature in the AI Chatbot module was unusable when viewed with the Olivero theme. The site header appeared on top of the chatbot window, hiding the close button and top portion of the conversation. This made it impossible to properly interact with or close the chatbot. The fix adjusts CSS z-index values to ensure the chatbot appears above all other page elements when opened, including the Olivero header and toolbar.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue occurred because Olivero's site header and layout wrapper elements used high z-index values that conflicted with the chatbot overlay. When the chatbot was opened, Olivero's `.site-header__initial` and `.layout-main-wrapper` elements remained stacked above the chatbot block.

The fix modifies `toolbar-chatbot.css` to target specific Olivero elements when the chatbot is open. When the `body.ai-chatbot-opened` class is present, the CSS now sets z-index to `unset` on `.site-header__initial` and `.layout-main-wrapper`, and reduces `.toolbar-bar` to `z-index: 1`. This allows the `.block-ai-deepchat-block` element to appear above these elements with its existing calculated z-index.

The patch also adds `cursor: pointer` to the `.sidebar-header--icon` button element for better user experience, indicating the element is clickable. The team considered using Web Components with shadow DOM for better CSS isolation across themes, but decided against it to avoid complexity of recreating Twig template elements in JavaScript.

Instead, they opted for targeted CSS overrides specific to Olivero, following patterns from issue [#3560492](https://www.drupal.org/node/3560492). This pragmatic choice prioritized maintainability over architectural purity, though the team noted that future work may revisit CSS isolation strategies as the Navigation module and Drupal CMS evolve.

## Lessons learned

- When injecting admin UI components into arbitrary frontend themes, z-index conflicts and style leakage are inevitable. Use CSS isolation techniques (shadow DOM, namespaced resets, or targeted overrides) rather than attempting to coordinate z-index values across theme boundaries. *(via marcus_johansson, bbruno, thamas)*
- When weighing architectural changes against complexity, prefer simpler solutions that solve the immediate problem unless the pattern recurs frequently. Shadow DOM provides perfect CSS isolation but requires duplicating template logic in JavaScript, making maintenance harder when simple CSS overrides suffice. *(via bbruno, thamas)*
