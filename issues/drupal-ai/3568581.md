## Summary

When using the Olivero theme, the toolbar chatbot was unusable because the site header's z-index caused it to appear on top of the chatbot interface. Users could not close the chatbot or see messages properly. This fix adjusts the CSS z-index values so the chatbot displays correctly above all page elements when opened, while maintaining proper toolbar functionality on mobile devices.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue occurred because Olivero's `.site-header__initial` and `.layout-main-wrapper` elements had z-index values that positioned them above the chatbot when it was opened. The fix modifies `toolbar-chatbot.css` to add specific CSS overrides when the `body.ai-chatbot-opened` class is present. 

The changes unset the z-index on `.site-header__initial` and `.layout-main-wrapper` when the chatbot is open, preventing these elements from overlapping the chat interface. For the toolbar itself, the z-index of `.toolbar-bar` is set to 1 to ensure proper stacking. The fix handles both placement scenarios: when the chatbot block is in the header region and when it's in a region within the main content area.

The patch also adds `cursor: pointer` to `.sidebar-header--icon` to improve the user experience by indicating the button is clickable. Contributors discussed using web components with shadow DOM for better CSS isolation across different themes, but decided this added unnecessary complexity since the chatbot UI comes from Twig templates.
