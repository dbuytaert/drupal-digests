## Summary

The Huggingface AI provider now supports object detection, allowing sites to identify and locate objects within images. This enables new use cases like automatic image cropping based on detected subjects, content analysis, and accessibility improvements. Object detection works by analyzing an image and returning information about what objects are present and where they appear in the image. This feature leverages Huggingface's pre-trained object detection models through their inference API.

## Impact

No upgrade or configuration changes required.

## Technical details

This change adds object detection as a new operation type to the Huggingface provider module. The implementation follows the same pattern as existing operations like image classification and embeddings.

The `HuggingfaceProvider` class now implements `ObjectDetectionInterface` and includes `object_detection` in its list of supported operation types. A new `objectDetection()` method handles the operation by accepting an image input (either binary data or an `ObjectDetectionInput` object), saving it to a temporary file, and sending it to the Huggingface inference API endpoint.

The underlying `HuggingfaceApi::objectDetection()` method was updated to pass the correct content type (`image/jpeg`) when making POST requests with file uploads. The API base URL was also updated from `api-inference.huggingface.co` to `router.huggingface.co/hf-inference` to align with Huggingface's current infrastructure.

Response data from Huggingface includes detected objects with labels, bounding box coordinates, and confidence scores. These are parsed into `ObjectDetectionItem` objects and returned as an `ObjectDetectionOutput`. This change requires the corresponding AI Core module changes from [#3570609](https://www.drupal.org/node/3570609) to define the operation type interfaces and data structures.
