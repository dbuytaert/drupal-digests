## Summary

The `ai_best_practices` project (a Composer plugin that copies AI guidance files into Drupal projects) established its canonical information architecture for skill files. Skills live under `skills/`, each in its own subdirectory named with a `drupal-` prefix in kebab-case (for example `skills/drupal-automated-testing/`), with a `SKILL.md` file as the required entry point. Every skill must have a matching `evals/<skill-name>/` directory containing static structural checks and behavioral test cases. The `AGENTS.md` index that consumer projects receive is defined in `.agents/AGENTS.md.template` and merged into project roots by the Composer plugin. These conventions govern all current and future skill contributions to the project.

## Technical details

The architecture aligns with the [Agent Skills specification](https://agentskills.io/specification). Each `SKILL.md` requires only `name` (must match the directory name) and `description` frontmatter; the proposal to add Drupal-specific keys such as `status`, `drupal-version`, and `last-reviewed` was explicitly rejected. The `description` field serves the "when to use" cue that agents use to decide whether to load a skill for a given task.

Skill directory naming follows the pattern `<domain>-<task-or-topic>` in kebab-case. All skills in this repo carry the `drupal-` domain prefix to avoid collision with skills from other packages. The gerund-first naming convention originally proposed was relaxed: the final rule is plain-English kebab-case, not a required gerund prefix.

Each skill's `evals/<name>/` directory contains `static-checks.json` (structural assertions run in CI with no API calls, using check types `file_exists`, `agent_skill_frontmatter`, `contains`, `order`, `max_lines`), `evals.json` (behavioral prompt/assertion pairs that require a live AI CLI), and `README.md`. The `compare.py` script supports A/B testing of skill edits across multiple AI providers. Skills are auto-discovered from the `skills/` directory tree; no manual registration in `.agents/AGENTS.md.template` is required for the skill to ship to consumers, though an entry there is still recommended so agents know when to load it.
