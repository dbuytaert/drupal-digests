## Summary

Sites running Drupal CMS that apply the AI recipe now get the AI Dashboard module (`drupal/ai_dashboard`) installed automatically. The dashboard provides a unified overview at `/admin/config/ai` showing which AI providers and features are configured, which ones are available to install via Project Browser, and how to proceed with AI setup. This replaces the previous situation where AI configuration was scattered across multiple locations with no central starting point.

## Impact

Affects module maintainers.

- **New API:** The `drupal_cms_ai` recipe now installs and configures `drupal/ai_dashboard` ^1.0 automatically.

## Technical details

The `drupal_cms_ai` recipe (in `recipes/drupal_cms_ai/`) was extended with three additions, all in the same commit authored by a.dmitriiev and cherry-picked to both `2.x` and `2.0.x`. In `composer.json`, `"drupal/ai_dashboard": "^1.0"` was added as a Composer dependency. In `recipe.yml`, `ai_dashboard` was added to the `install` list, and `ai_dashboard: '*'` was added under `config.import` so the module's default configuration is imported when the recipe is applied.

During review, marcus_johansson found that the AI Dashboard was overriding the default Dashboard at `/admin/dashboard`. a.dmitriiev resolved this by setting the dashboard weight to 100 instead of 0 (which conflicted with Drupal CMS's own "Welcome to the site" dashboard at weight 0). A separate schema validation failure for `project_browser.admin_settings:enabled_sources.ai_dashboard_recommended` was fixed upstream in the `ai_dashboard` module itself (see issue [#3567688](https://www.drupal.org/node/3567688) in that project).

Dries provided product-level feedback noting the dashboard feels more generic than Drupal CMS-specific, flagging the "Extensions" section and large feature cards as potentially unnecessary for the opinionated Drupal CMS experience. pameeela clarified this feedback was not a blocker for the merge, and follow-up issues were filed in the `ai_dashboard` project to address those UX concerns separately.
