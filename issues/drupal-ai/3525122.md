## Summary

This research issue evaluated JavaScript library options for adding @mentions-style autocomplete to the AI Prompt Management UI, where users currently have to type variables and tokens manually. After testing several editors and autocomplete libraries, the team chose MDXEditor. The actual implementation was completed in [#3570097](https://www.drupal.org/node/3570097), which added the rich text editor with autocomplete support for tokens and context variables in prompt fields.

## Impact

No upgrade or configuration changes required.

## Technical details

The AI Prompt Management subform (`src/Form/AiPromptSubform.php`) supports two types of placeholder: variables (like those used in `t()`) and Drupal tokens. Any chosen library therefore needed to support at least two distinct trigger characters and multiple independent editor instances on the same page. These requirements, added to the evaluation criteria by scott_euser, ruled out the CKEditor Mentions contrib module, which provides only global configuration rather than per-instance configuration.

marcus_johansson built live demos of five candidates: Ace Editor, CKEditor, EasyMDE + CodeMirror Show Hint, Overtype + Tribute, and Quill with quill-mention. MDXEditor was added after initial round. afoster and emma horrell performed a UX review against three criteria: WYSIWYG markdown editing, autocomplete for tokens, and a togglable raw-markdown mode with syntax highlighting. breidert produced a detailed feature matrix covering editor experience, markdown/rich-text support, token preview, undo/diff, and paste-from-Word behaviour.

MDXEditor scored best overall: it supports both a WYSIWYG and a raw-markdown diff view in the same editor, has an official mentions/autocomplete extension (`@mdx-editor/extras`), supports multiple trigger characters, and can be packaged independently of Drupal core's CKEditor version. The identified risk is that the upstream project relies heavily on a single lead contributor. The decision to use MDXEditor was reached by rough consensus and implementation was tracked in [#3570097](https://www.drupal.org/node/3570097).
