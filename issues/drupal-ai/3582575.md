## Summary

The Drupal AI module now ships a Drush code generator and an AI agent skill to help developers create custom guardrail plugins. Before this change, creating a guardrail required knowing the full plugin API: the `#[AiGuardrail]` attribute, base class hierarchy, four result types, streaming interfaces, and config schema requirements. The new `drush generate plugin:ai:guardrail` command (alias: `ai-guardrail`) asks five questions and writes a ready-to-edit plugin stub. The agent skill at `.agents/skills/create-guardrail-plugin/SKILL.md` gives AI coding assistants (such as Claude or GitHub Copilot) a structured guide to scaffold guardrails from a natural-language description.

## Impact

Affects module maintainers.

## Technical details

Guardrail plugins (`#[AiGuardrail]`) run before and after AI generation and return one of four result types from `Drupal\ai\Guardrail\Result\`: `PassResult`, `StopResult`, `RewriteInputResult`, or `RewriteOutputResult`. Two distinct plugin patterns exist: deterministic (fixed rules, extends `AiGuardrailPluginBase`, implements `ConfigurableInterface` + `PluginFormInterface`) and non-deterministic (calls an AI provider internally, additionally implements `NonDeterministicGuardrailInterface`, `NonStreamableGuardrailInterface`, and `ContainerFactoryPluginInterface`, uses `NeedsAiPluginManagerTrait`).

**Drush generator** (`src/Drush/Generators/GuardrailGenerator.php`) extends `DrupalCodeGenerator\Command\BaseGenerator` and is registered via the `#[Generator]` attribute with name `plugin:ai:guardrail` and alias `ai-guardrail`. The `generate()` method interviews for module machine name, label, plugin ID (auto-derived as snake_case from the label), description, and deterministic/non-deterministic choice, then writes `src/Plugin/AiGuardrail/{ClassName}.php` from one of two Twig templates at `templates/Plugin/_ai-guardrail/`. The deterministic template produces `processInput()` with a `ChatInput` type check, last-message extraction via `end($messages)`, and `@todo` markers for custom validation logic, plus full `ConfigurableInterface` boilerplate. The non-deterministic template adds the `NeedsAiPluginManagerTrait`, a `create()` method injecting `AiProviderFormHelper` via `ai.form_helper`, provider/model configuration in `buildConfigurationForm()` using `AiProviderFormHelper::generateAiProvidersForm()`, and `submitConfigurationForm()` that extracts and type-casts LLM config keys using `CastUtility::typeCast()`.

**Agent skill** (`.agents/skills/create-guardrail-plugin/SKILL.md`) is a structured prompt document consumed by AI coding assistants via the `/create-guardrail-plugin` slash command. It walks through four steps: determine guardrail type, gather requirements, run `drush generate plugin:ai:guardrail`, add config schema to `config/schema/ai.schema.yml` under the key `ai.guardrail.settings.{plugin_id}`, and optionally generate a kernel test using the EchoAI provider. Config schema guidance was added during review (comment #8) after a reviewer found it missing when testing the skill against the `InputLengthLimit` guardrail from [#3582856](https://www.drupal.org/node/3582856).

The `#[Generator]` attribute sets `templatePath` to `__DIR__ . '/../../../templates/Plugin/_ai-guardrail'`, which is the actual location of the Twig files. The SKILL.md reference section lists `templates/guardrail/deterministic.twig` (the older path discussed in review), but the authoritative path used by the generator is `templates/Plugin/_ai-guardrail/`.
