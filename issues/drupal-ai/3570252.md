# #3570252: Cannot select all models for "Chat with Complex JSON"

- **Project:** Drupal AI
- **Type:** Major bug report
- **Version:** 1.2.0
- **Status:** Fixed
- **Discussion:** [#3570252](https://www.drupal.org/node/3570252) · 4 contributors · 11 comments

## Summary

When configuring OpenAI models in the AI module, newer GPT-5 models were not available for selection in the "Chat with Complex JSON" operation type, even though they appeared in the regular "Chat" operation and other configuration screens. This prevented site administrators from using the latest GPT-5 models for structured JSON output, which these models support according to OpenAI's documentation. The fix ensures GPT-5 models now appear in the selection list for complex JSON chat operations.

## Impact

No upgrade or configuration changes required.

## Technical details

The OpenAI provider module filtered available models for the "Chat with Complex JSON" operation type based on model name patterns. The filter included GPT-3 and GPT-4 variants but did not include the GPT-5 pattern, causing newer GPT-5 models (gpt-5.1 and gpt-5.2) to be excluded from the selection list at `/admin/config/ai/settings`.

GPT-5 models support structured output according to OpenAI's model documentation, making them suitable for JSON-based chat operations. The fix adds the GPT-5 pattern to the model name filter, allowing these models to appear alongside existing options. This brings the "Chat with Complex JSON" operation type in line with other operation types that already displayed all available models.

The change follows confirmation from maintainers that GPT-5 models work with structured output, as documented in OpenAI's platform documentation for both gpt-5.1 and gpt-5.2 models.
