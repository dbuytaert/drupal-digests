## Summary

The AI module now includes a rich-text prompt editor built with MDXEditor. This editor allows users to write AI prompts with formatting options and special auto-complete features for tokens (triggered by `[`) and custom variables (triggered by `{`). The editor appears wherever prompts are created or edited, such as in prompt libraries, AI agents, and context control centers. It gracefully falls back to a plain textarea when JavaScript is unavailable.

## Impact

No upgrade or configuration changes required.

## Technical details

The implementation adds MDXEditor, a React-based rich-text editor, to provide a better user experience for writing AI prompts. The build process uses Vite and is located in the `mdxeditor` directory with sources in `mdxeditor/src` and built artifacts in `mdxeditor/dist`.

The editor integrates with Drupal through a form element hook. The new `Drupal\ai\Hook\FormElement` service alters the `textarea` element info to attach the editor JavaScript and CSS via the `mdx_editor` library. The editor attaches to textareas with a specific data attribute (`data-mdxeditor`) and configuration is exposed through `drupalSettings.mdxeditor`, keyed by unique IDs matching the textarea's data attribute.

The `AiPromptSubform` class was updated to expose token and variable definitions from prompt types to the editor configuration. When users type `[` in the prompt field, they see available tokens; typing `{` shows available variables.

The GitLab CI pipeline was updated to build both CKEditor and MDXEditor artifacts. The `js_build` job now builds both `modules/ai_ckeditor/js/build` and `mdxeditor/dist`. For tagged releases, a new `scripts/tag-release.sh` script (similar to Canvas module's approach) temporarily forces built files into Git despite them being in `.gitignore`, commits and tags the release, then removes the forced files. This ensures users installing from drupal.org get pre-built assets without bloating the Git history with build artifacts from every commit.
