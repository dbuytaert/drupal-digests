## Summary

The AI module now includes MDXEditor, a rich text editor with autocomplete support for tokens and custom context variables. This gives site builders a modern editing experience when creating AI prompts in places like Context Control Center, AI Agents, Automators, and the Prompt Library. The editor provides markdown formatting tools and intelligently suggests available tokens (triggered with `[`) and variables (triggered with `{`) based on the prompt type configuration.

## Impact

No upgrade or configuration changes required.

## Technical details

This change adds a JavaScript build system under `mdxeditor/` that compiles MDXEditor with the `@mdxeditor/typeahead-plugin` for custom mention functionality. The build uses Vite and outputs to `mdxeditor/dist/`.

The editor attaches to textareas with a `data-mdxeditor` attribute. Configuration is exposed through `drupalSettings.mdxeditor` with unique IDs matching the textarea's data attribute. For prompt forms, `AiPromptSubform` provides editor config containing tokens and variables defined in the prompt type entity.

A new `Drupal\ai\Hook\FormElement` service alters form elements to integrate the editor. The `mdx_editor` library in `ai.libraries.yml` loads the built JavaScript bundle and CSS from `dist/assets/`.

Build artifacts are excluded from the repository via `.gitignore`. The `.gitlab-ci.yml` includes a `js_build` job that compiles both CKEditor and MDXEditor assets. A `scripts/tag-release.sh` script temporarily forces built files into tagged releases, following the same pattern used by Canvas AI module. This means the compiled JavaScript is only committed for official releases, keeping the git history clean while ensuring built assets are available to users installing via Composer or Drupal.org.
