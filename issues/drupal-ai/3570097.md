## Technical details

This change adds MDXEditor with typeahead support to the AI module. The editor is built using Vite and bundled with the `@mdxeditor/typeahead-plugin` for mention-style autocomplete. The build artifacts are located in `mdxeditor/dist/` and loaded via the `mdx_editor` library.

The editor attaches to textareas with specific data attributes and supports graceful degradation to plain textareas when JavaScript is unavailable. Configuration is exposed through `drupalSettings.mdxeditor`, keyed by unique IDs matching the textarea's `data-mdxeditor` attribute. For prompt forms, `AiPromptSubform` populates tokens and variables from the prompt type into this configuration.

A new form element hook service `Drupal\ai\Hook\FormElement` was added to integrate the editor into the form API. The build process was integrated into GitLab CI (`js_build` job) to compile both MDXEditor and CKEditor assets. The built files are added to `.gitignore` and only included in tagged releases via the `scripts/tag-release.sh` script, following the Canvas project's pattern. This keeps the repository clean while ensuring production releases contain all necessary assets.

The editor supports reading and writing to the underlying textarea from external JavaScript, enabling integration with other form behaviors. Toolbar items include rich text formatting options demonstrated in the proof-of-concept at workflows-of-ai.com.
