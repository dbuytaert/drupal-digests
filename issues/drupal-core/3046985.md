## Summary

Drupal now adds translation context to ordinal numbers and time units that were previously ambiguous. Words like "Second" can mean either a unit of time or an ordering position. In languages like German, these require completely different translations. Without context, the word "Sekunde" (time unit) might be mistranslated as "zweiter" (ordinal number), or vice versa. Additionally, ordinal numbers are adjectives in many languages and require different forms based on grammatical gender. Layout region labels, date format options, and similar interface text now include context strings so translators can provide accurate translations based on how each word is actually used.

## Impact

No upgrade or configuration changes required.

## Technical details

The change adds translation context to all bare occurrences of ordinal numbers in core's user interface strings. Two contexts are used: "PHP date format" for time-related instances of "Second" in date element dropdowns and Views sort options, and "Layout region" for ordinal position labels in layout configuration files.

In PHP code, context is added as the third parameter to the `t()` function: `t('Second', [], ['context' => 'PHP date format'])`. For YAML layout definition files, a new `translation context` key is added alongside each region label.

The issue focused on instances where a bare word appears without surrounding context that would clarify its meaning. Strings like "Second item" were already clear and required no changes. Only standalone words like "Second" needed disambiguation.

The change affects layout files in `layout_builder.layouts.yml`, `layout_discovery.layouts.yml`, and `umami.layouts.yml`, the `Datelist` element class, and the Views Date sort plugin. Context was added to all ordinal numbers (First through Fifth) found in layout regions, not just "Second", because these also require gender-specific translations in many languages.

### Collaboration

This issue remained open for nearly seven years before resolution. The initial report in 2019 clearly identified the problem for German translation, but determining the correct approach took considerable discussion. Early questions centered on whether YAML layout files could even support translation context, leading to discoveries about how the POTX extraction process works.

The biggest challenge was defining appropriate context strings. Initial suggestions for generic contexts like "ordinal" proved insufficient because adjectives require gender matching in many languages. The solution evolved to use specific contexts describing what each ordinal refers to, such as "Layout region."

The issue was reserved for mentored contribution during DrupalCon Europe 2020. Stefan Auditor created the initial patch in late 2020 that established the pattern used in the final solution. After conversion to a merge request in 2024, the change received final review and was committed in March 2026. A follow-up issue was noted to potentially add automated checks for bare words requiring translation context.

## Lessons learned

- When a translatable string has multiple meanings that translate differently across languages, provide translation context using the context parameter in t(). Words like 'Second' (time unit vs. ordinal number) and ordinal numbers that require gender agreement ('first', 'second', 'third') need context specifying what they refer to (e.g., 'ordinal for footer region' or 'datetime') so translators can choose the grammatically correct translation for each use. *(via jhodgdon, joachim namyslo)*
