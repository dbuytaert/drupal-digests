## Summary

Drupal now adds translation context to ordinal numbers and time units that were previously ambiguous. Words like "Second" can mean either a unit of time or an ordering position. In languages like German these require completely different translations: "Sekunde" for the time unit, and "zweite/r/s" for an ordinal number. Without context, translators could produce strings like "Eine zweiter zuvor" instead of "Eine Sekunde zuvor." Ordinal numbers are also adjectives in many languages and require different forms based on grammatical gender. Layout region labels, date format dropdowns, and Views sort options now include context strings so translators can provide accurate translations based on how each word is actually used.

## Impact

No upgrade or configuration changes required.

## Technical details

The change adds `translation context` to all bare occurrences of standalone ordinal numbers in core UI strings, targeting the two distinct meanings of "Second": a unit of time and a positional ordinal.

Two context strings are used throughout the change. In PHP code, the context is passed as the third argument to `t()`:

```php
$title = t('Second', [], ['context' => 'PHP date format']);
```

This appears in two places: the `processDatelist()` method of `Datelist` (`core/lib/Drupal/Core/Datetime/Element/Datelist.php`) for the second-selector in date element dropdowns, and `buildOptionsForm()` in the Views `Date` sort plugin (`core/modules/views/src/Plugin/views/sort/Date.php`) for granularity options.

For YAML layout definition files, a `translation context` key is added directly under each region entry:

```yaml
regions:
  first:
    label: First
    translation context: 'Layout region'
  second:
    label: Second
    translation context: 'Layout region'
```

This change is applied to every ordinal region label (First through Fifth as applicable) in three files: `core/modules/layout_builder/layout_builder.layouts.yml`, `core/modules/layout_discovery/layout_discovery.layouts.yml`, and `core/profiles/demo_umami/themes/umami/umami.layouts.yml`.

The issue notes that strings like `t('Second item')` did not need changes because the surrounding words make the meaning clear. Only standalone words were ambiguous. The plural form "Seconds" was also left unchanged as it is unambiguous. Following the commit, `longwave` suggested adding an automated check (a Coder or PHPStan rule) to catch bare single-word translatables missing context; `berdir` pointed to the related issue [#3108111](https://www.drupal.org/node/3108111).

### Collaboration

The issue ran from April 2019 to March 2026. Early discussion focused on whether YAML layout files could carry translation context at all. `jhodgdon` investigated the POTX extraction process and discovered layout label strings were not even being extracted for localize.drupal.org, opening a separate tracking issue ([#3048032](https://www.drupal.org/node/3048032)). She also clarified why a generic context string like "ordinal" (suggested by `dpi`) was insufficient: because ordinal numbers are adjectives in gendered languages, the context must describe what the ordinal refers to, so that different occurrences can receive different gender-inflected translations in different languages.
