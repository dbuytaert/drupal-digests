## Summary

Drupal 12 removes a batch of deprecated APIs from the Extension, Form, Layout, Plugin, and Site subsystems that had been carrying deprecation notices since versions 10.3 through 11.3. Site owners are not directly affected, but contributed and custom modules that called any of the removed APIs will break on Drupal 12 and must be updated to use the replacements that have been available since those deprecations were introduced.

## Impact

Affects module maintainers.

- **API change:** Removed `ModuleInstallerInterface::addUninstallValidator()`, `ThemeHandlerInterface::rebuildThemeData()`, `ThemeHandlerInterface::getBaseThemes()`, `ThemeExtensionList::getBaseThemes()`, `CachedDiscoveryClearer::addCachedDiscovery()`, `PluginManagerBase::getModuleHandler()`, and the protected `DrupalKernel` helpers `getModulesParameter()`, `getModuleFileNames()`, and `getModuleNamespacesPsr4()`, along with the `module_installer.uninstall_validators` service and the `ExtensionDiscovery::$fileCache` property.

## Technical details

This task removes code that was deprecated across several Drupal 11.x minor releases and is now enforced for Drupal 12.

**Extension subsystem**

`ModuleInstaller::addUninstallValidator()` and the matching declaration in `ModuleInstallerInterface` are removed (deprecated in 11.1.0, [#3432595](https://www.drupal.org/node/3432595)). The `module_installer.uninstall_validators` service definition is removed from `core.services.yml` at the same time. The `$uninstallValidators` constructor parameter in `ModuleInstaller` is now non-nullable: callers must inject the tagged iterator directly rather than relying on the removed service.

Three protected `DrupalKernel` helpers are removed: `getModulesParameter()`, `getModuleFileNames()`, and `getModuleNamespacesPsr4()` (all deprecated in 11.3.0, [#3551652](https://www.drupal.org/node/3551652)). Their replacements `getExtensionsParameter()`, `getExtensionFileNames()`, and `getExtensionNamespacesPsr4()` remain. A stale `@see` comment pointing to the old `getModuleNamespacesPsr4()` is corrected to `getExtensionNamespacesPsr4()`.

The deprecated `$fileCache` property and its `FileCacheFactory::get()` initializer are removed from `ExtensionDiscovery` (deprecated in 11.2.0, [#3490431](https://www.drupal.org/node/3490431)).

`ThemeExtensionList::getBaseThemes()` and `ThemeHandler::getBaseThemes()` / `ThemeHandler::rebuildThemeData()` along with their interface declarations in `ThemeHandlerInterface` are removed (deprecated in 10.3.0, [#3413187](https://www.drupal.org/node/3413187) and [#3413196](https://www.drupal.org/node/3413196)).

**Theme and Form subsystems**

`ThemeManager::__construct()` now requires `$kernel` (formerly optional/nullable, deprecated in 11.3.0). Similarly `$keyValueFactory` and `$cache` are now required parameters in the same constructor.

`FormAjaxResponseBuilder`, `FormSubmitter`, and `FormValidator` all had `$callableResolver` introduced as an optional parameter in 11.3.0 ([#3548821](https://www.drupal.org/node/3548821)); it is now a required parameter and all three classes use constructor promotion, removing the separate property declarations.

**Layout subsystem**

`LayoutDefinition` previously triggered `E_USER_DEPRECATED` when a layout plugin had neither a label nor a deriver; it now throws `\InvalidArgumentException`. `LayoutPluginManager::processDefinition()` previously triggered `E_USER_DEPRECATED` when a definition lacked a label; it now throws `InvalidPluginDefinitionException`.

**Plugin subsystem**

`CachedDiscoveryClearer::addCachedDiscovery()` and its backing `$legacyCachedDiscoveries` property are removed (deprecated in 11.1.0, [#3442229](https://www.drupal.org/node/3442229)). `PluginManagerBase::getModuleHandler()` is removed (deprecated in 10.3.0, [#3310017](https://www.drupal.org/node/3310017)).

`DirectoryWithMetadataDiscovery` and `DirectoryWithMetadataPluginDiscovery` no longer accept a `$fileSystem` parameter (deprecated in 11.3.0, [#3530869](https://www.drupal.org/node/3530869)).

**Settings**

The `$deprecatedSettings` map in `Settings` is emptied: entries for `state_cache` (deprecated in 11.0.0) and `allow_authorize_operations` (deprecated in 12.0.0) are removed.

**PHPStan baseline**

Corresponding `$ignoreErrors` entries for `addUninstallValidator()` (missing return type and deprecated service), `CachedDiscoveryClearer::addCachedDiscovery()` (missing return type), and `ModuleInstallerInterface::addUninstallValidator()` are removed from the PHPStan baseline.

The issue explicitly deferred `DefaultPluginManager::__construct()` to a separate discussion; berdir opened [#3577900](https://www.drupal.org/node/3577900) for that.

## Upgrade

If your module or custom code uses any of the following, update it before upgrading to Drupal 12:

**ModuleInstaller uninstall validators** â€” stop calling `addUninstallValidator()` and stop using the `module_installer.uninstall_validators` service. Tag your validator with `module_install.uninstall_validator` and let it be injected automatically:
```yaml
# your_module.services.yml
services:
  your_module.uninstall_validator:
    class: Drupal\your_module\YourUninstallValidator
    tags:
      - { name: module_install.uninstall_validator }
```

**DrupalKernel protected methods** â€” replace calls to the removed helpers:
```php
// Before
$this->getModulesParameter();
$this->getModuleFileNames();
$this->getModuleNamespacesPsr4($filenames);

// After
$this->getExtensionsParameter($this->moduleList, [$this, 'moduleData']);
$this->getExtensionFileNames($this->moduleList, [$this, 'moduleData']);
$this->getExtensionNamespacesPsr4($filenames);
```

**ThemeHandlerInterface methods** â€” replace removed calls:
```php
// Before
$theme_handler->rebuildThemeData();
$theme_handler->getBaseThemes($themes, $theme);

// After (rebuildThemeData)
\Drupal::service('extension.list.theme')->reset()->getList();

// getBaseThemes has no direct replacement
```

**CachedDiscoveryClearer** â€” stop calling `addCachedDiscovery()`. Register your plugin manager as a tagged service instead:
```yaml
tags:
  - { name: plugin_manager_cache_clear }
```

**PluginManagerBase::getModuleHandler()** â€” inject `ModuleHandlerInterface` directly into your plugin manager constructor; there is no replacement method.

**DirectoryWithMetadataDiscovery / DirectoryWithMetadataPluginDiscovery** â€” remove the `$fileSystem` argument from constructor calls:
```php
// Before
new DirectoryWithMetadataDiscovery($dirs, $suffix, $file_system);

// After
new DirectoryWithMetadataDiscovery($dirs, $suffix);
```

**FormAjaxResponseBuilder / FormSubmitter / FormValidator** â€” `$callableResolver` is now a required argument. If you instantiate these classes directly (outside the service container), pass a `CallableResolver` instance.

**Layout plugins** â€” plugins that lack both `label` and `deriver` now throw `\InvalidArgumentException` at container compile time rather than a deprecation notice. Ensure every layout plugin definition has at least one of those properties.
