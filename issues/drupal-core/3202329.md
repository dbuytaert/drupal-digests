## Summary

Outbound path processors can now access route names and parameters when generating URLs. Previously, processors received only the route object, which forced some modules to use workarounds like parsing URLs or injecting services that could create circular dependencies. This enhancement enables modules like Domain to optimize path processing without performance penalties or architectural problems.

## Impact

Affects module maintainers.

- **New API:** Outbound path processors now receive `route_name` and `route_parameters` in the `$options` array passed by `UrlGenerator::generateFromRoute()`

## Technical details

`UrlGenerator::generateFromRoute()` now adds two new keys to the `$options` array passed to outbound path processors: `route_name` (the string route name) and `route_parameters` (the array of parameters). These join the existing `route` key that provides the route object.

This change addresses a gap where processors needed route information but had no clean way to access it. The route object alone does not expose the route name or the specific parameters used to generate the current URL. Previously, processors had to call `Url::fromUserInput()` to extract this information, which is expensive, or inject the route provider or router, which creates circular dependency issues.

The new data is added directly in `UrlGenerator::generateFromRoute()` in `core/lib/Drupal/Core/Routing/UrlGenerator.php`, right before calling `processPath()`. The information is already available at that point in the generation flow, so passing it to processors is straightforward. The `OutboundPathProcessorInterface` documentation in `core/lib/Drupal/Core/PathProcessor/OutboundPathProcessorInterface.php` now documents these two new options alongside the existing `route` option.

Test coverage in `UrlGeneratorTest::testGenerateWithRouteNameInOptions()` verifies that a mock path processor receives both `route_name` and `route_parameters` with the expected values.

## Upgrade

Custom outbound path processors can now access `$options['route_name']` and `$options['route_parameters']` in their `processOutbound()` method. These keys are automatically populated when URLs are generated from routes.

```php
public function processOutbound($path, &$options = [], Request $request = NULL, BubbleableMetadata $bubbleable_metadata = NULL) {
  if (isset($options['route_name']) && $options['route_name'] === 'entity.node.canonical') {
    $node_id = $options['route_parameters']['node'];
    // Process path based on node ID without expensive lookups.
  }
  return $path;
}
```

No code changes are required. This is a backward-compatible addition of new optional data.
