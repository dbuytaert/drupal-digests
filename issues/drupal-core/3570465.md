## Summary

When running `run-tests.sh` without a `--sqlite` argument, test results are now stored in a lightweight PHP in-memory implementation instead of being written to the installed Drupal database. This removes a SQLite dependency for throwaway CI runs and eliminates unnecessary database round-trips. A new `--results-on-installed-db` option is provided for the rare case where results must persist to the installed Drupal database. CI pipelines can simply drop the `--sqlite :memory:` argument they previously needed to pass.

## Impact

No upgrade or configuration changes required.

## Technical details

The fix introduces `MemoryTestRunResultsStorage` in `Drupal\TestTools\TestRunner` (marked `@internal`), a pure PHP implementation of `TestRunResultsStorageInterface` that keeps test run data in process-local arrays keyed by run ID. The `simpletest_script_setup_test_run_results_storage()` function in `core/scripts/run-tests.sh` now checks `Config::get('sqlite')`: when it is `NULL` (option absent), it instantiates `MemoryTestRunResultsStorage`; when a path or `:memory:` is supplied, it falls through to `SimpletestTestRunResultsStorage` backed by SQLite as before. The new `--results-on-installed-db` option is registered in `Configuration::commandLineDefinition()` and validated at parse time so that it and `--sqlite` are mutually exclusive. GitLab CI job definitions in `.gitlab-ci.yml` drop the explicit `--sqlite :memory:` argument across all three job templates that previously carried it, since in-memory storage is now the default. This builds on [#3515347](https://www.drupal.org/node/3515347), which already moved all inter-process communication into the master process, making external persistence of test results unnecessary for standard runs.

## Upgrade

If you invoke `run-tests.sh` without `--sqlite` and previously relied on test results being stored in the installed Drupal database, pass `--results-on-installed-db` explicitly to restore that behavior:

```bash
# Before (no --sqlite meant results went to the installed DB)
php ./core/scripts/run-tests.sh --dburl $SIMPLETEST_DB ...

# After: to write results to the installed DB, pass the new option
php ./core/scripts/run-tests.sh --results-on-installed-db --dburl $SIMPLETEST_DB ...
```

If you were passing `--sqlite :memory:` in CI pipelines, that argument can be removed entirely:

```bash
# Before
php ./core/scripts/run-tests.sh --sqlite ":memory:" --dburl $SIMPLETEST_DB ...

# After
php ./core/scripts/run-tests.sh --dburl $SIMPLETEST_DB ...
```
