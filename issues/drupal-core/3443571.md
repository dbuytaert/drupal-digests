## Summary

When the Navigation module's mobile sidebar is open, keyboard and screen reader users could tab through elements obscured by the sidebar, violating the WCAG 2.4.11 Focus Not Obscured criterion. The sidebar now implements a focus trap using the HTML `inert` attribute: while the sidebar is expanded on screens narrower than 1024px, all other direct children of `body` become inert and unreachable by keyboard or assistive technology. The trap is lifted when the sidebar closes or the viewport widens to desktop size, and reapplied if the sidebar is still open when the viewport narrows back to mobile. A related followup ([#3572169](https://www.drupal.org/node/3572169)) also put the close button itself back into the tab sequence.

## Impact

No upgrade or configuration changes required.

## Technical details

The change lives in `admin-toolbar-wrapper.js` inside the navigation module. A module-level `mobileMediaQuery` constant (`window.matchMedia('(max-width: 1023px)')`) replaces an earlier inline `window.matchMedia` call and is reused in two places.

The new `toggleFocusTrap(newState)` function queries `body > *:not(:is(.admin-toolbar, .admin-toolbar-overlay))` and calls `el.toggleAttribute('inert', newState)` on each result, simultaneously setting or removing a `data-admin-toolbar-inert` marker attribute so the set of affected elements can be identified later. The function is called in two contexts: from the existing `HTML_TRIGGER_EVENT` listener when the sidebar state changes (gated by `mobileMediaQuery.matches`), and from a new `change` listener on `mobileMediaQuery` that removes `inert` on resize to desktop and reapplies it on resize back to mobile if the sidebar is already expanded.

The `ScriptBytes` assertion values in `AssetAggregationAcrossPagesTest`, `UmamiIntegrationTest`, and the Umami-specific asset test were updated in both commits to reflect the larger script payload. Three new Nightwatch tests in `navigationMobileFocusTrapTest.js` verify that `inert` and `data-admin-toolbar-inert` attributes are present on non-toolbar `body` children when the sidebar opens on mobile, absent on desktop widths, and removed when the sidebar closes via the close button or overlay click.
