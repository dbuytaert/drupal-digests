## Summary

The section headings on the `/admin/help` page now carry HTML `id` attributes derived from each help section's plugin ID. This means sections can be linked to directly using URL fragments, for example `/admin/help#help-topics` or `/admin/help#hook-help`. The "See the Help page" link shown at the top of individual help-topic pages was also updated to anchor directly to the `#help-topics` section.

## Impact

Affects front-end developers.

- **Theme change:** The `help_section` theme hook gains a `plugin_id` variable; `help-section.html.twig` (and its overrides in Claro, Stable9, and Umami) now render `<h2 id="{{ plugin_id|clean_id }}">` instead of a bare `<h2>`. Custom theme overrides of this template should add the `id` attribute accordingly.

## Technical details

In `HelpController::helpMain()`, the plugin ID is now passed to the `help_section` render array as `#plugin_id`. The `help_section` theme hook definition (declared in the same controller's `theme()` method) gains `plugin_id` as a new variable defaulting to `NULL`. The `help-section.html.twig` template was updated in four locations: the canonical template in the Help module, Claro, Stable9, and the Umami installation profile. Each now renders `<h2 id="{{ plugin_id|clean_id }}">` instead of a bare `<h2>`. The `clean_id` Twig filter converts underscores to hyphens and strips characters that are invalid in HTML `id` values, so `hook_help` becomes `hook-help` and `help_topics` becomes `help-topics`. The `help.help_topic` route handler's back-link was updated to call `->setOption('fragment', 'help-topics')` on the URL so it lands on the correct anchor. Test assertions in `HelpTest` were updated to expect the new `id` attributes on the `<h2>` elements.

## Upgrade

If your custom theme overrides `help-section.html.twig`, update the `<h2>` element to include the new `id` attribute:

```twig
{# Before #}
<h2>{{ title }}</h2>

{# After #}
<h2 id="{{ plugin_id|clean_id }}">{{ title }}</h2>
```

The `plugin_id` variable is now available in all `help_section` template contexts.
