## Summary

Every call to `EntityRepository::loadEntityByUuid()` previously issued a SQL query to look up the entity ID from the UUID before loading the entity by ID. On pages that look up the same entity by UUID multiple times, or that resolve many entity references (as JSON:API does), this produced a large number of redundant queries. A static in-memory cache now stores UUID-to-ID pairs so that only the first lookup for a given UUID requires a query. The cache is also primed whenever any entity is loaded by ID, so UUID lookups that follow a normal entity load are free. The JSON:API performance tests confirm the improvement: one scenario dropped from 25 queries to 17, another from 14 to 6.

## Impact

No upgrade or configuration changes required.

## Technical details

`EntityStorageBase` gains a new `$uuidMemoryCacheTag` property (`entity.memory_cache:uuid:{entityTypeId}`) that allows the UUID lookup cache to be invalidated independently from the main entity memory cache.

`loadByProperties()` now intercepts calls where the only filter is a single UUID string. It checks a `uuid_lookup:{entityTypeId}:{uuid}` entry in the memory cache before running the EntityQuery. Both positive results (an array of entity IDs) and empty results (no entity found) are stored, so repeated calls with the same UUID are served entirely from memory. `setStaticCache()` also writes these `uuid_lookup` entries whenever entities are stored by ID, priming the UUID cache as a side effect of ordinary entity loads.

`resetCache()` invalidates the entire UUID lookup cache tag (`$uuidMemoryCacheTag`) whenever any entity ID's cache is cleared, because there is no way to reverse-map a given ID back to its UUID without an additional query. When resetting all caches both the entity and UUID tags are invalidated together.

A required adjustment was made to `LruMemoryCache::invalidateTags()`: it now removes items from the cache immediately when their tags are invalidated, instead of marking them as invalid and keeping them in memory. This is necessary for the UUID lookup cache to work correctly, since empty-result entries must be evictable via tag invalidation when an entity is saved or deleted. The change also means `LruMemoryCache` no longer returns tag-invalidated items when `$allow_invalid = TRUE` is passed to `get()`.

The `UninstallDefaultContentTest` was updated to obtain fresh storage instances after module install and uninstall rather than calling `resetCache()` on stale instances, which the more aggressive UUID cache invalidation exposed.
