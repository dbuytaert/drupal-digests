## Summary

The deprecation notice for `HtmxRequestInfoTrait::getHtmxTrigger()` incorrectly stated "There is no replacement" when the method was deprecated in Drupal 11.5.0. The correct replacement is `HtmxRequestInfoTrait::getHtmxSource()`. This fix updates both the PHPDoc `@deprecated` tag and the runtime `@trigger_error()` message to name that replacement, so developers who encounter the warning know exactly what to use instead.

## Impact

No upgrade or configuration changes required.

## Technical details

The `getHtmxTrigger()` method in `\Drupal\Core\Htmx\HtmxRequestInfoTrait` was deprecated as part of [#3607711](https://www.drupal.org/node/3607711) (which deprecates several HTMX-related methods in 11.5.0). That original commit mistakenly wrote "There is no replacement" in both the PHPDoc `@deprecated` annotation and the `@trigger_error()` call inside the method body. The actual replacement is `HtmxRequestInfoTrait::getHtmxSource()`.

The fix updates the PHPDoc tag to read `Use HtmxRequestInfoTrait::getHtmxSource() instead.` and updates the runtime error string to match. The unit test assertion in `HtmxRequestInfoTraitTest::testGetHtmxTrigger()` is updated to expect the corrected string. The change is in `11.x` only, as the deprecations do not exist on `main`.

## Upgrade

If your code calls `HtmxRequestInfoTrait::getHtmxTrigger()`, migrate to `HtmxRequestInfoTrait::getHtmxSource()` before Drupal 12.0.0.
