Drupal core bumps `mglaman/phpstan-drupal` (to 2.0.15) and `phpstan/phpstan` (to 2.1.54) along with related static-analysis packages, and applies the code fixes those newer versions revealed. ## Technical details

The functional code change is in `AutowireInstanceTrait::getAutowireArguments()`. The old call `$parameter->allowsNull()` directly on a `ReflectionParameter` instance is deprecated in PHP 8.4; it is replaced with `$parameter->getType()?->allowsNull() || $parameter->getType() === NULL`. The null-safe operator handles parameters that carry no type hint at all (where `getType()` returns `null`), preserving the original behaviour of setting such arguments to `null`.

Two link module field-formatter classes (`LinkFormatter` and a sibling formatter) gain `@phpstan-param \Drupal\Core\Field\FieldItemListInterface<\Drupal\link\LinkItemInterface> $items` docblock annotations on their `viewElements()` methods. These PHPStan generics annotation let the analyser infer the concrete item type without requiring runtime `assert()` calls, resolving unknown-variable-type warnings introduced by the stricter PHPStan 2.x series.

The PHPStan baseline file `core/.phpstan-baseline.php` loses a large number of now-resolved suppressions: `missingType.return` entries for generated `trait*()` proxy methods across `BlockBase`, `BlockManager`, `ContentEntityDeleteForm`, `FieldTypePluginManager`, `ModerationStateFieldItemList`, workspaces `Query`/`QueryAggregate`, and others; and `variable.undefined` entries across several modules and tests that the upgraded analyser can now prove are always defined. A small number of new baseline entries are added where the stricter analysis flags previously unchecked patterns (for example, an `isset.offset` check on a known array shape in `BlockListBuilder` and a `variable.undefined` for `$fiber` in `RendererTest`).

### Collaboration

During review, xjm flagged that adding `assert()` calls for link items could disrupt CI pipelines if non-link items had ever been permitted there. mondrake explained that the downstream `buildUrl()` call already enforces the type natively, making a runtime failure on non-link items inevitable regardless. This led mondrake to switch from assertions to PHPStan generics annotations, which satisfied the type-analysis requirement without adding runtime overhead. longwave separately identified the `ReflectionParameter::allowsNull()` deprecation warning spreading through anything using `AutowireTrait` and proposed the one-line fix that was then incorporated.
