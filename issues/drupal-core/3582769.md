## Summary

This is an internal testing infrastructure change. A new `clickLink()` method was added to the `HttpKernelUiHelperTrait` test trait to support converting functional tests to kernel tests. Site owners are not affected.

## Impact

Affects module maintainers.

- **New API:** New `HttpKernelUiHelperTrait::clickLink()` method for kernel tests that make HTTP requests

## Technical details

The `HttpKernelUiHelperTrait` provides testing methods for kernel tests that make HTTP requests through the HTTP kernel. It previously had `drupalGet()` and `assertSession()` to match the functional test API. This issue adds `clickLink()` to complete the basic navigation API.

The `clickLink()` method finds links by their text label using Mink's named selector. It accepts an optional index parameter to handle cases where multiple links have the same text. The method uses `drupalGet()` internally rather than Mink's native `click()` method because `drupalGet()` produces HTML debug output when tests fail.

The test coverage uses the `test_page_test` module which provides a page with multiple links having identical text, verifying that both default behavior (index 0) and explicit indexing work correctly.

This is needed for converting functional tests in the help module to kernel tests ([#3517299](https://www.drupal.org/node/3517299)), part of the broader effort to convert suitable functional tests to kernel tests ([#3582242](https://www.drupal.org/node/3582242)).
