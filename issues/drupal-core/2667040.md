## Summary

Entity types can now specify URLs for viewing, editing, and deleting entities using either link templates or route providers. An older method using `uri_callback` was still documented and partially functional, but rarely used. This change formally deprecates `uri_callback` and updates documentation to guide developers toward the modern approaches. The feature will be removed in Drupal 13, giving developers time to migrate any custom entity types still using this legacy pattern.

## Impact

Affects module maintainers.

- **Deprecation:** The `uri_callback` property in entity type annotations/attributes is deprecated in Drupal 11.4.0 and removed in Drupal 13.0.0. Use link templates or route providers instead.

## Technical details

The `uri_callback` property allowed entity types to specify a callable that would return routing information for an entity. This approach was largely replaced by link templates and route providers years ago, but the old API remained in place without formal deprecation.

The change adds `@deprecated` tags to the `uri_callback` property across all entity type attribute classes (`EntityType`, `ContentEntityType`, `ConfigEntityType`), the `EntityType` class property, and the getter/setter methods in `EntityTypeInterface` and `EntityType`. When `uri_callback` is set in an attribute constructor or accessed via `getUriCallback()`, a deprecation notice is triggered.

Documentation in `entity.api.php` was updated to move `uri_callback` information to a less prominent position, clearly marking it as deprecated and only used as a fallback when link templates are not provided. The outdated documentation referencing non-existent functions like `node_uri()` was corrected.

The functionality itself remains unchanged to maintain backward compatibility. The deprecation targets Drupal 13.0.0 for removal, giving custom entity types time to migrate to link templates or route providers.

## Upgrade

Custom entity types still using `uri_callback` should migrate to link templates or a route provider.

For simple cases, replace `uri_callback` with link templates in your entity type annotation or attribute:

```php
// Old (deprecated):
uri_callback = "my_entity_uri",

// New:
links = {
  "canonical" = "/my-entity/{my_entity}",
  "edit-form" = "/my-entity/{my_entity}/edit",
  "delete-form" = "/my-entity/{my_entity}/delete",
}
```

For more complex routing needs, implement a route provider by specifying a `handlers` entry and creating a class that implements `EntityRouteProviderInterface`.

See the change record at [https://www.drupal.org/node/3575062](https://www.drupal.org/node/3575062) for complete migration guidance.

## Lessons learned

- When removing or changing a feature that developers configure through annotations or attributes, add deprecation warnings at the point where the configuration is consumed, not just in property definitions. Detecting deprecated configuration during entity info collection ensures developers receive actionable feedback. *(via catch)*
- When deprecating entity configuration options like uri_callback, direct developers to both the consumer API (toUrl()) and the provider pattern (link templates or route handlers). Developers creating entities need guidance on how to structure their entity type, not just how to consume it. *(via jhodgdon, longwave)*
