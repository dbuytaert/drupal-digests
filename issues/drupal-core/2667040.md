## Summary

Entity types can now specify URLs for viewing, editing, and deleting entities using either link templates or route providers. An older method using `uri_callback` was still documented and partially functional, but rarely used. This change formally deprecates `uri_callback` and updates documentation to guide developers toward the modern approaches. The feature will be removed in Drupal 13, giving developers time to migrate any custom entity types still using this legacy pattern.

## Impact

Affects module maintainers.

- **Deprecation:** `uri_callback` on entity type annotations/attributes and the `getUriCallback()`/`setUriCallback()` methods on `EntityTypeInterface` and `EntityType` are deprecated in drupal:11.4.0 and removed in drupal:13.0.0. Use link templates or a route provider instead.

## Technical details

The `uri_callback` property allowed entity types to specify a callable that would return routing information for an entity. This approach was largely replaced by link templates and route providers years ago, but the old API remained in place without a formal deprecation.

The change adds `@deprecated` tags to the `uri_callback` property across all entity type attribute classes (`EntityType`, `ContentEntityType`, `ConfigEntityType` in `core/lib/Drupal/Core/Entity/Attribute/`), the `$uri_callback` property in `EntityType`, and the `getUriCallback()` and `setUriCallback()` methods in `EntityTypeInterface` and `EntityType`. When `uri_callback` is set in an attribute constructor, or when `getUriCallback()` or `setUriCallback()` is called, a runtime `@trigger_error()` with `E_USER_DEPRECATED` is fired.

Documentation in `entity.api.php` was rewritten to lead with link templates and route providers as the recommended approach. The old text, which referenced the non-existent `node_uri()` as an example and presented `uri_callback` as the primary pattern, was removed. A new subordinate note describes `uri_callback` accurately as a deprecated fallback used only when no matching link template is found. The `uri_callback` documentation entry for `hook_entity_bundle_info()` was removed entirely. In `EntityInterface.php`, inline code examples were annotated with "(deprecated)" markers.

The functionality itself remains unchanged to maintain backward compatibility. The deprecation targets Drupal 13.0.0 for removal, giving custom entity types time to migrate to link templates or route providers. See the change record at https://www.drupal.org/node/3575062.

## Upgrade

Custom entity types still using `uri_callback` should migrate to link templates or a route provider.

For simple cases, replace `uri_callback` with link templates in your entity type annotation or attribute:

```php
// Old (deprecated):
uri_callback: 'my_entity_uri',

// New — use link templates in the #[EntityType] attribute:
links: [
  'canonical'    => '/my-entity/{my_entity}',
  'edit-form'    => '/my-entity/{my_entity}/edit',
  'delete-form'  => '/my-entity/{my_entity}/delete',
],
```

For more complex routing needs, implement a route provider by adding a `handlers` entry pointing to a class that implements `EntityRouteProviderInterface`.

See the change record at https://www.drupal.org/node/3575062 for complete migration guidance.

The issue then stalled for years before longwave moved the work into a merge request in 2026. godotislate's review caught two remaining gaps: the absence of a change record URL in the deprecation messages, and an undeprecated `uri_callback` reference in the `hook_entity_bundle_info()` documentation. longwave addressed both, removing the bundle-info reference entirely rather than adding a deprecated note to it.
