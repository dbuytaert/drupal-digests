# #3583795: Change the new CLI entry point to “dr”

## Summary

The new built-in Drupal CLI tool, being developed in [#3453474](https://www.drupal.org/node/3453474), was originally named `drupal` (invoked as `vendor/bin/drupal`). This issue reached consensus to rename the official command to `dr` before the CLI becomes widely adopted. The shorter name avoids overloading the project name, reduces cognitive overhead, requires no aliasing, and follows the pattern established by two-character CLIs like `wp`, `gh`, `ng`, and `az`. The rename was applied in the merge request for [#3453474](https://www.drupal.org/node/3453474), with the old `drupal` entry point kept as a deprecated wrapper pointing to `dr`.

## Impact

No upgrade or configuration changes required.

## Technical details

The CLI entry point lives in `core/scripts/` and is declared as a Composer `bin`, so it is copied into `vendor/bin/` under the same filename. Because Composer provides no mechanism to alias or rename binned scripts, the rename requires renaming `core/scripts/drupal` to `core/scripts/dr` and providing a separate `core/scripts/drupal` shim that emits a deprecation notice and delegates to `dr` via `return include __DIR__ . '/dr'`. The deprecation window proposed in the thread is `drupal:11.4.0` removed in `drupal:13.0.0`. The actual file changes are tracked in the parent issue [#3453474](https://www.drupal.org/node/3453474); this issue served purely as the naming decision record. A minor trade-off noted in the thread is that Drupal.org issue search requires at least three characters, so `dr` cannot be searched directly, though that limitation is expected to disappear as core issues migrate to GitLab.
