# #3135100: [policy and meta] Provide a proper mechanism for deprecating modules and themes

## Summary

Drupal now has a formal, standardized mechanism for deprecating and retiring modules and themes in core (and contrib). Before this work, deprecating a module was ad hoc and labor-intensive, requiring manually deprecating every symbol inside it. The new system introduces a `lifecycle` key in `.info.yml` that can be set to `experimental`, `stable`, `deprecated`, or `obsolete`. Setting it to `deprecated` triggers PHP deprecation notices when the extension is installed, shows a labeled warning in the admin modules/appearance pages, and surfaces an alert on the status report. Setting it to `obsolete` hides the extension from the admin UI entirely and blocks any new installation with an exception. This gives site owners clear, consistent signals about which extensions are being phased out and where to find migration guidance.

## Impact

Affects module maintainers.

- **New API:** New `lifecycle` key for `.info.yml` files with values `experimental`, `stable`, `deprecated`, `obsolete`; new `ExtensionLifecycle` class with constants `LIFECYCLE_IDENTIFIER`, `LIFECYCLE_LINK_IDENTIFIER`, `EXPERIMENTAL`, `STABLE`, `DEPRECATED`, and `OBSOLETE`; new `ObsoleteExtensionException` thrown when installation of an obsolete extension is attempted.

## Technical details

The problem this meta issue coordinated was that Drupal had no formal API for signaling an extension's maturity or retirement status. The previous approach for removing a module (seen in [#3062281](https://www.drupal.org/node/3062281) for Place Block) required deprecating every individual PHP symbol, which was tedious and inconsistent.

The solution introduces a lifecycle progression tracked in each extension's `.info.yml`: `experimental` → `stable` → `deprecated` → `obsolete`. The `stable` value is the default, applied automatically by `InfoParserDynamic` when the key is absent. For the `deprecated` and `obsolete` values a companion `lifecycle_link` URL is required; `InfoParserDynamic` throws an `InfoParserException` if either it is missing or not a valid URL.

All valid values and the two constant identifiers (`LIFECYCLE_IDENTIFIER`, `LIFECYCLE_LINK_IDENTIFIER`) are defined in `ExtensionLifecycle`, a final class at `core/lib/Drupal/Core/Extension/ExtensionLifecycle.php`.

The runtime enforcement is spread across several integration points coordinated by the child issues:

- **Installation blocking and notices** ([#3257127](https://www.drupal.org/node/3257127)): `ModuleInstaller::install()` reads the lifecycle field. An `obsolete` module throws an `ObsoleteExtensionException`; a `deprecated` module triggers `@trigger_error()` with `E_USER_DEPRECATED`. `ThemeInstaller` does the same for themes.
- **Admin modules page** ([#3215043](https://www.drupal.org/node/3215043)): `ModulesListForm` hides obsolete modules from the listing entirely and appends a linked `(Deprecated)` / `(Experimental)` label for non-stable extensions; `ModulesListNonStableConfirmForm` presents a confirmation step when a user attempts to install a deprecated module.
- **Admin appearance page** ([#3215044](https://www.drupal.org/node/3215044)): Parallel treatment for themes in `SystemController`.
- **Status report** ([#3250585](https://www.drupal.org/node/3250585)): `SystemRequirementsHooks` collects all currently-installed deprecated modules, deprecated themes, and obsolete extensions (modules or themes) and surfaces them as distinct requirements warnings at `admin/reports/status`, each linking to the `lifecycle_link` URL.
- **Upgrade Status module** ([#3223453](https://www.drupal.org/node/3223453), upgrade_status project): The contrib Upgrade Status module was updated to detect deprecated and obsolete projects based on this key.

The remaining open work is Composer namespacing when a core extension moves to contrib ([#3188544](https://www.drupal.org/node/3188544)), since a core and contrib module with the same machine name conflict under Composer even if the core version carries `lifecycle: deprecated`.

## Upgrade

To deprecate a module or theme, add the following to its `.info.yml`:

```yaml
lifecycle: deprecated
lifecycle_link: 'https://www.drupal.org/node/YOUR_CHANGE_RECORD'
```

To mark an extension as fully obsolete (blocks installation and hides it from the UI):

```yaml
lifecycle: obsolete
lifecycle_link: 'https://www.drupal.org/node/YOUR_CHANGE_RECORD'
```

The `lifecycle_link` value must be a valid URL; `InfoParserDynamic` throws an `InfoParserException` at parse time if it is absent or malformed. Omitting the `lifecycle` key entirely defaults to `stable`.
