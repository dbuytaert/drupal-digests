## Summary

A test fixture that simulates uninstalling the Contact module was incomplete, leaving behind form and view display configuration for contact messages. This caused any update tests using the `ConfigEntityUpdater` to update display configurations to crash with a "contact_message entity type does not exist" error. The fixture now properly removes all display-related configuration when uninstalling Contact, preventing these test failures.

## Impact

No upgrade or configuration changes required.

## Technical details

The `uninstall-contact.php` test fixture simulates uninstalling the Contact module by removing its database entries and configuration. However, it only deleted config entries matching `contact.%`, which missed the entity form and view display configurations stored as `core.entity_form_display.contact%` and `core.entity_view_display.contact%`.

When tests using `ConfigEntityUpdater` tried to process all form or view displays, they attempted to load display configs for the contact_message entity type. Since the entity type had been removed but the displays remained, this caused a `PluginNotFoundException`.

The fix updates the delete query to use an OR condition group that matches all three config name patterns: the original `contact.%` pattern plus the two display patterns. This ensures the fixture completely removes all Contact-related configuration, matching what happens during actual module uninstallation.
