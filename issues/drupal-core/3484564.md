## Summary

The navigation module's Top Bar now has three formally defined regions: Tools (left), Context (center), and Actions (right). Previously the bar had no structural contract and local tasks were injected directly in an ad-hoc way. A new plugin system now controls what content appears in each region. The bar itself is hidden by CSS when all regions are empty, and shows automatically when any region has content. This lays the groundwork for contrib modules and future Drupal core features to add items to the appropriate Top Bar region in an orderly, extensible way.

## Impact

Affects module maintainers and front-end developers.

- **Theme change:** The `top_bar` theme hook changed from a variables-based definition to a render element; `top-bar.html.twig` now exposes `tools`, `context`, and `actions` variables instead of `local_tasks`, and always renders three region wrapper divs.
- **New API:** New `TopBarItem` plugin type with `TopBarItemPluginInterface`, `TopBarItemBase`, `TopBarItemManagerInterface`, and `hook_top_bar_item_alter()` for adding and altering items in the navigation Top Bar's three regions (Tools, Context, Actions).

## Technical details

The issue replaces an ad-hoc local-tasks injection approach with a proper plugin system for Top Bar content, spread across several new types.

**`TopBarRegion` enum** (`src/TopBarRegion.php`): A string-backed PHP enum with cases `Tools` ('tools'), `Context` ('context'), and `Actions` ('actions'). It is used throughout the system to reference regions in a type-safe way. Because Twig enum support requires Twig 3.15 and core ships Twig 3.14, the enum is used only on the PHP side.

**`TopBarItem` plugin type**: The `#[TopBarItem]` attribute (`src/Attribute/TopBarItem.php`) declares plugin metadata: `id`, `region` (a `TopBarRegion`), optional `label`, and optional `deriver`. `TopBarItemPluginInterface` (`src/TopBarItemPluginInterface.php`) requires `label()`, `region()`, and `build()`. `TopBarItemBase` (`src/TopBarItemBase.php`) extends `PluginBase` and implements `label()` and `region()`, leaving `build()` abstract. `TopBarItemManager` (`src/TopBarItemManager.php`) extends `DefaultPluginManager` to discover plugins under `Plugin/TopBarItem`, calls `alterInfo('top_bar_item')` to expose `hook_top_bar_item_alter()`, caches with the key `top_bar_item_plugins`, and adds `getDefinitionsByRegion(TopBarRegion)` and `getRenderedTopBarItemsByRegion(TopBarRegion)` methods. The service is registered as `plugin.manager.top_bar_item` in `navigation.services.yml` and aliased to `TopBarItemManagerInterface`.

**`TopBar` render element** (`src/Element/TopBar.php`): A new `#[RenderElement('top_bar')]` element whose `preRenderTopBar()` iterates over all `TopBarRegion` cases, calls `getRenderedTopBarItemsByRegion()` for each, and merges the results into the element array keyed by the region's string value.

**`NavigationRenderer` changes**: `buildTopBar()` now emits `['#type' => 'top_bar']` instead of `['#theme' => 'top_bar']`. The inline local-tasks block is removed from the renderer and the cache metadata is simplified to `['keys' => ['top_bar'], 'contexts' => ['user.permissions']]`. An `#access` guard is added using the newly injected `AccountInterface`. `getLocalTasks()` and `hasLocalTasks()` are promoted from `private` to `public` so plugins can consume them.

**`PageActions` plugin** (`src/Plugin/TopBarItem/PageActions.php`): The first built-in plugin, placed in `TopBarRegion::Actions`. Its `build()` delegates to `NavigationRenderer::hasLocalTasks()` and `getLocalTasks()` to reproduce the previous local-tasks display inside the new system.

**Template and CSS**: `top-bar.html.twig` is rewritten to unconditionally render three wrapper divs (`top-bar__tools`, `top-bar__context`, `top-bar__actions`). Visibility is CSS-driven: `.top-bar` defaults to `display: none` and is revealed by a `:has(.top-bar__tools:not(:empty), .top-bar__context:not(:empty), .top-bar__actions:not(:empty))` selector. `inset-block-start` is updated to respect `var(--drupal-displace-offset-top, 0)` so the bar positions correctly below the admin toolbar. New flex layout classes are added for all three regions. The preprocessing function (`preprocessTopBar()` in `NavigationThemeHooks`) maps the three region values from the render element into Twig template variables.

## Upgrade

Modules that previously relied on `local_tasks` being available as a template variable in `top-bar.html.twig` must update to use the plugin system instead.

To add a custom item to the Top Bar, create a plugin class in your module's `Plugin/TopBarItem` namespace:

```php
<?php

namespace Drupal\my_module\Plugin\TopBarItem;

use Drupal\Core\StringTranslation\TranslatableMarkup;
use Drupal\navigation\Attribute\TopBarItem;
use Drupal\navigation\TopBarItemBase;
use Drupal\navigation\TopBarRegion;

#[TopBarItem(
  id: 'my_module_item',
  region: TopBarRegion::Actions,
  label: new TranslatableMarkup('My Item'),
)]
class MyItem extends TopBarItemBase {

  public function build(): array {
    return ['#markup' => 'Hello from the Actions region'];
  }

}
```

To alter existing plugin definitions, implement `hook_top_bar_item_alter()`:

```php
function my_module_top_bar_item_alter(array &$definitions): void {
  // Modify $definitions keyed by plugin ID.
}
```
