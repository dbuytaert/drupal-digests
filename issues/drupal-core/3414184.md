## Summary

Drupal now includes a test to verify that every PHP version used in continuous integration has a known end-of-life date recorded in the codebase. This helps prevent a situation where Drupal tests run on a PHP version but the system doesn't know when that version will stop receiving security updates. The test catches missing data early, ensuring Drupal can accurately warn site owners when they're running on PHP versions approaching or past their support deadline.

## Impact

No upgrade or configuration changes required.

## Technical details

The `PhpRequirements` class maintains an internal array of PHP end-of-life dates that powers status reports and warnings to site administrators. Previously, when a new PHP version was added to the GitLab CI testing matrix, developers could forget to add the corresponding EOL date to this array.

The new test assertion in `PhpRequirementsTest::testPhpEolDates()` checks that the currently running PHP version (determined by `PHP_MAJOR_VERSION` and `PHP_MINOR_VERSION`) exists as a key in the `$phpEolDates` static property. This means that whenever CI runs tests on a new PHP version, the test will fail until someone adds that version's EOL date to `PhpRequirements`.

This change was validated when PHP 8.5 was added to CI. The test initially failed, then passed after the EOL date was added in a related issue. The test serves as an automated safeguard against incomplete EOL tracking.
