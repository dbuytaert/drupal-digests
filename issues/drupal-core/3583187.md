## Summary

This is an internal test-only fix with no user-facing effect. It resolves a randomly failing CI test in `DuplicateContextualLinksTest::testSameContextualLinks` that was breaking roughly 50% of the time.

## Impact

No upgrade or configuration changes required.

## Technical details

The test originally placed two identical `views_block:contextual_recent-block_1` blocks and hardcoded `#block-first` as the container for the node contextual link. `Block::sort()` falls back to `strcmp()` on block labels when weights are equal, and `drupalPlaceBlock()` generates labels via `randomMachineName()`, so which block received the rendered node teaser was non-deterministic. When the node landed in `#block-second`, the jQuery selector matched nothing, the click was silent, and `waitForElementVisible` timed out returning `null`, causing `Call to a member function isVisible() on null`.

As amateescu noted in the thread, a second factor compounded this: [#2940605](https://www.drupal.org/node/2940605) ("Can only intentionally re-render an entity with references 20 times") caused the node to only appear in one of the two blocks rather than both, making the test design structurally unsound.

The fix rewrites the test to remove all dependency on `block`, `node`, `views`, and `views_ui`. A new route `contextual_test.duplicate_links` is added to `contextual_test`, backed by a new `duplicateContextualLinks()` method on `TestController`. That method uses `ContextualLinksSerializer::linksToId()` to build a contextual ID and renders two deterministic `<div>` regions (with IDs `region-first` and `region-second`) via inline templates, each carrying an identical `#type => contextual_links_placeholder`. The test now also strengthens the assertion: it clicks each region's trigger in turn and verifies that only the targeted region's links open while the other's remain hidden, which is the actual behavior originally proven by [#2891603](https://www.drupal.org/node/2891603). The `contextual_recent` view config file shipped with the `contextual_test` module is deleted as it is no longer needed.
