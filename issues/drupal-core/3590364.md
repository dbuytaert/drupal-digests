## Summary

This issue completes the second phase of CSS consolidation for the new `default_admin` admin theme (see [#3582833](https://www.drupal.org/node/3582833) for Part 1). The remaining migration CSS files inherited from Gin and Claro have been merged into the theme's permanent stylesheet locations and the migration directory stubs deleted. Along the way, several visual improvements and bug fixes landed: rounded corner radii are now sharper (`--base-border-radius` changed from `0.5rem` to `2px`), the header background shifts to a bluer tone, media library and dialog layouts were repaired, and dark-mode support was added or fixed in several components. The autocomplete loading indicator was redesigned from a background-image SVG to a CSS animation spinner. Any site using the `default_admin` theme will see these visual changes automatically after updating to 11.4.

Affects front-end developers.

## Technical details

The migration directory `core/themes/default_admin/migration/css/` held Gin-origin stylesheet stubs for components, layout, and theme overrides. This issue merged all remaining stubs into the canonical `css/` tree and removed the stubs.

Key mechanical changes visible in the diff: `--base-border-radius` was corrected from `0.5rem` to `2px` and `--gin-bg-header` changed from `#eeeff3` to `#e1eafc` in `variables.pcss.css`. `--vertical-tabs-border-color` was fixed to reference `--gin-border-color` instead of the nonexistent `--details-border-color`. The `.button` rule switched from `display: inline-block` to `display: inline-flex; align-items: center; box-sizing: border-box`, and the `.project__action_button` selector was dropped from that rule entirely. In `autocomplete-loading.module.pcss.css`, the Claro-origin wrapper class `.claro-autocomplete` and its sibling `.claro-autocomplete__message` were renamed to `.gin-autocomplete` and `.gin-autocomplete__message`; the SVG spinner background-image was replaced with a CSS `@keyframes gin-throbber` animation. The CKEditor 5 integration file, which previously contained only a single `--ck-color-base-border` override, now maps the full suite of `--ck-*` custom properties to `--gin-*` design tokens. New standalone files were created for components that had no Claro equivalent: `description-toggle`, `contextual-links`, `edit-form`, `revisions`, `settings`, `sidebar`, and `status`. Details and dialog components gained dark-mode rules and logical-property rewrites. Toolbar integration CSS was removed because the toolbar module is being deprecated for Drupal 12.

## Upgrade

Contrib modules or custom CSS that target `.claro-autocomplete` or `.claro-autocomplete__message` must be updated:

```css
/* Before */
.claro-autocomplete { … }
.claro-autocomplete__message { … }

/* After */
.gin-autocomplete { … }
.gin-autocomplete__message { … }
```

The `.project__action_button` selector no longer inherits `.button` styles from `default_admin`. If a module relied on this for button appearance in the admin theme, add the `.button` class directly to the element or target `.project__action_button` explicitly in your own stylesheet.

Custom CSS that references `--base-border-radius` expecting `0.5rem` rounded corners will now receive `2px`. Override the variable in your sub-theme if the old value is required.
