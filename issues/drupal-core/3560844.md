## Summary

The starter kit theme now uses object-oriented PHP hooks instead of procedural hooks in a `.theme` file. When developers generate a new theme from the starter kit, they will receive a modern OOP structure with hook implementations in a class file. This change brings the starter kit in line with Drupal's current best practices for hook implementations and ensures the theme generation system properly handles this new pattern.

## Impact

No upgrade or configuration changes required.

## Technical details

The starterkit_theme now implements hooks using the OOP pattern introduced in Drupal 11. The procedural `starterkit_theme_preprocess_image_widget()` function in `starterkit_theme.theme` was converted to a method `preprocessImageWidget()` in the `StarterkitThemeHooks` class, located in `src/Hook/StarterkitThemeHooks.php`. The method uses the `#[Hook('preprocess_image_widget')]` attribute to declare the hook implementation.

The `GenerateThemeTest` was updated to reflect this structural change. Tests that previously checked for the existence of a `.theme` file now check for the `src/Hook/[ThemeName]Hooks.php` class file. Tests that verified the generated theme structure now look for OOP method signatures (like `public function preprocessImageWidget()`) instead of procedural function signatures (like `function theme_name_preprocess_image_widget()`).

One assertion was removed: the test no longer verifies that the `src` directory does not exist, since generated themes now include this directory by default for hook classes.
