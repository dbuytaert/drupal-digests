## Summary

The starterkit theme has been converted from procedural hooks to object-oriented hook implementations. This affects the theme generation system, which uses starterkit as a template when developers create new custom themes. The change aligns the starter theme with Drupal's modern OOP hook architecture. Tests were updated to expect the new file structure with hooks in class files rather than a traditional .theme file.

## Impact

No upgrade or configuration changes required.

## Technical details

The starterkit theme's `hook_preprocess_image_widget()` implementation was migrated from a procedural function in `starterkit_theme.theme` to an OOP method `preprocessImageWidget()` in the new `StarterkitThemeHooks` class. This class lives in `src/Hook/StarterkitThemeHooks.php` and uses the `#[Hook('preprocess_image_widget')]` attribute.

The `GenerateThemeTest` was updated to reflect this structural change. The test previously asserted that generated themes would have a `.theme` file and no `src` directory. Now it expects a `src/Hook/` directory structure with a hooks class file. Test assertions were updated to check for the hooks class file instead of the `.theme` file, and to verify the new OOP method signature rather than the procedural function signature.

One assertion that checked `src` does not exist was removed since generated themes now include the `src` directory. The theme generation command correctly copies and renames the OOP hook structure when creating new themes from the starterkit.
