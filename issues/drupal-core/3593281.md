## Summary

The long-standing "Promoted to front page" label on nodes has been renamed to simply "Promoted" across the entire admin UI. The old name was misleading because the `/node` listing is not the front page on most sites and, per the related initiative ([#3572350](https://www.drupal.org/node/3572350)), will no longer be enabled by default. Site editors will see "Promoted" in the publishing options when creating or editing content, "Promote content" and "Unpromote content" in bulk-action menus, and updated descriptions in the Frontpage view. No database schema, field machine names, or PHP class/method names changed.

## Impact

Affects site owners.

- **Config change:** Labels in `system.action.node_promote_action.yml`, `system.action.node_unpromote_action.yml`, `views.view.frontpage.yml`, and `field.field.node.*.promote.yml` default config updated to use the new terminology.

## Technical details

The diff touches labels and human-readable strings only; the underlying `promote` field, `NodeInterface::PROMOTED`/`NOT_PROMOTED` constants, and all method names (`isPromoted()`, etc.) are unchanged. Updated locations include: the base field definition label in `Node::baseFieldDefinitions()`, the `NodeTypeForm` publishing-options checkbox, the `#[Action]` annotations on `PromoteNode` and `DemoteNode` (and their corresponding `system.action.node_promote_action.yml` / `node_unpromote_action.yml` config files), Views data strings in `NodeViewsData::getViewsData()`, the `views.view.frontpage.yml` description and empty-state text, `field.field.node.article.promote.yml` and `field.field.node.recipe.promote.yml` default config, the `get-started.html.twig` onboarding template, `node.module` help text, and `node.html.twig` docblock comments in all core themes. Config schema labels in `node.schema.yml` for `action.configuration.node_unpromote_action` and `action.configuration.node_promote_action` are also updated. Because many of the renamed strings are translatable, existing interface translations (on localize.drupal.org or in custom translation files) for the old strings will no longer match and will fall back to English until updated. The full list of changed strings was documented in a change record at https://www.drupal.org/node/3593641.

## Upgrade
