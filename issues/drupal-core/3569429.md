## Summary

A batch of unit and kernel test files across ten Drupal core modules was updated to replace expectation-less mocks with PHPUnit stubs. The change has no effect on site behavior; it is purely an internal test-quality improvement that brings the test suite into alignment with PHPUnit best practices and clears related PHPStan warnings.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit distinguishes between a mock object, which can assert that specific methods are called a certain number of times, and a stub, which simply returns canned values with no call-count expectations. When a test creates a mock with `createMock()` or `getMockBuilder()` and immediately sets `expects($this->any())`, it gains no assertion value from the mock machinery: the `any()` matcher never fails regardless of how many times the method is called. PHPUnit 12.5 deprecated `any()` to push developers toward stubs for this pattern.

This issue converted those cases across the modules `block_content`, `editor`, `file`, `help`, `image`, `media`, `path_alias`, `rest`, `search`, and `update`. The mechanics of each change follow the same pattern: `createMock()` or `getMockBuilder()->disableOriginalConstructor()->getMock()` paired with `expects($this->any())` is replaced with `createStub()`, dropping the `expects()` call entirely. PHPDoc type annotations on test-class properties were updated from `MockObject` to `Stub` to match. Where a method genuinely must be called, the unconstrained `any()` was tightened to `atLeastOnce()` or `once()` so a real assertion remains.

Affected test files include `BlockContentLocalTasksTest.php`, `EditorConfigEntityUnitTest.php`, `EditorXssFilter/StandardTest.php`, `FileItemTest.php`, `HelpTopicTwigLoaderTest.php`, `HelpTopicTwigTest.php`, `ImageItemTest.php`, `ImageStyleTest.php`, `IFrameUrlHelperTest.php`, `EndpointTest.php`, `ResourceFetcherTest.php`, `AliasManagerTest.php`, `AliasPathProcessorTest.php`, `CollectRoutesTest.php`, `SerializerTest.php`, `SearchPageRepositoryTest.php`, `SearchPluginCollectionTest.php`, `UpdateFetcherTest.php`, and `UpdateMailTest.php`.

The corresponding `any()` suppression entries were removed from `core/.phpstan-baseline.php`, meaning PHPStan will now flag any future regression that reintroduces `any()` in these files. Changes originally drafted for the `contact` module were intentionally omitted because that module was moved to contrib before this issue landed.
