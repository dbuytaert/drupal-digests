## Summary

This is an internal test infrastructure change with no user-facing effect. As part of the planned deprecation of the Shortcut module, test coverage that exercises shortcut-specific behavior has been relocated from several other modules into the Shortcut module's own test suite. No production code was changed.

## Impact

No upgrade or configuration changes required.

## Technical details

The Shortcut module is being prepared for deprecation and eventual move to contrib. As a prerequisite, tests that are genuinely about shortcut behavior need to live in the shortcut module's test suite so they travel with it, rather than being scattered across other modules.

The commit touches ten files across seven modules, all test files:

**Files moved into `core/modules/shortcut/tests/`:**
- `navigation/tests/src/Functional/NavigationShortcutsBlockTest.php` was renamed to `shortcut/tests/src/Functional/NavigationShortcutsBlockTest.php` and its namespace updated from `Drupal\Tests\navigation\Functional` to `Drupal\Tests\shortcut\Functional`. Its `#[Group]` attribute was changed from `navigation` to `shortcut`.
- `jsonapi/tests/src/Functional/ShortcutTest.php` was renamed to `shortcut/tests/src/Functional/Jsonapi/ShortcutTest.php` with namespace updated to `Drupal\Tests\shortcut\Functional\Jsonapi` and `#[Group]` changed to `shortcut`. An unneeded `comment` module dependency was also removed.
- `jsonapi/tests/src/Functional/ShortcutSetTest.php` was renamed to `shortcut/tests/src/Functional/Jsonapi/ShortcutSetTest.php` with namespace and group updated similarly.

**New test files created under `shortcut/tests/`:**
- `shortcut/tests/src/Functional/ConfigTranslationListUiTest.php`: A focused test for the shortcut set translate-operation link, extracted from `config_translation/tests/src/Functional/ConfigTranslationListUiTest.php`.
- `shortcut/tests/src/Functional/TrustedHostsTest.php`: A standalone test verifying shortcut module compatibility with trusted-host verification, extracted from `system/tests/src/Functional/System/TrustedHostsTest.php`.
- `shortcut/tests/src/FunctionalJavascript/NavigationBlockUiTest.php`: A new JS functional test for the shortcuts block in the Navigation block admin UI (`testNavigationBlockAdminUiPage`), extracted from `navigation/tests/src/FunctionalJavascript/NavigationBlockUiTest.php`.
- `shortcut/tests/src/Kernel/ShortcutEntityLinksTest.php`: A kernel test for `ShortcutLinkTarget` and the `entity_links` filter, extracted from `filter/tests/src/Kernel/EntityLinksTest.php`.

**Tests cleaned from other modules:**
- `block/tests/src/Functional/BlockInstallTest.php`: `shortcut` removed from the module list installed during the cache-tag test; shortcut was not necessary for that assertion.
- `config_translation/tests/src/Functional/ConfigTranslationListUiTest.php`: The `doShortcutListTest()` method and the `shortcut` module dependency removed.
- `filter/tests/src/Kernel/EntityLinksTest.php`: `testShortcutEntity()` and the `shortcut` module dependency removed.
- `jsonapi/tests/src/Functional/JsonApiFilterRegressionTest.php`: The `testFilterByIdFromIssue3015759` method was rewritten to use `MenuLinkContent` instead of `Shortcut` entities, since the test is about JSON:API filter behavior and does not need to be shortcut-specific.
- `navigation/tests/src/FunctionalJavascript/NavigationBlockUiTest.php`: `testNavigationBlockAdminUiPage()` and the `shortcut` module dependency removed.
- `system/tests/src/Functional/System/TrustedHostsTest.php`: `testShortcut()` removed.
- `system/tests/src/Functional/Theme/ToolbarClaroOverridesTest.php`: `shortcut` module and shortcut-related permissions removed from the test setup.
- `system/tests/src/Kernel/Entity/EntityOperationsCacheabilityTest.php`: The `shortcut_set` data-provider row removed.

`core/.phpstan-baseline.php` was updated to reflect the new test namespaces and file paths for `NavigationShortcutsBlockTest` and `NavigationBlockUiTest` (both moved from `navigation` to `shortcut`).

The issue notes that Claro theme tests and a handful of other tests that pull in shortcut only as a side-dependency were intentionally left in place; those will be addressed in a separate deprecation pass.
