## Summary

Several previously deprecated classes and methods in Drupal's testing framework have been fully removed in preparation for Drupal 12.0.0. Developers who write automated tests for custom modules or contrib projects must update any code that extends `InstallerExistingConfigTestBase`, uses `WebDriverCurlService`, calls `TestDiscovery::getTestClasses()` or related discovery methods, or calls removed `AssertContentTrait` assertion methods such as `assertNoLinkByHref()` and `assertUniqueText()`. This change has no effect on production site behavior; only test code is affected.

## Impact

Affects module maintainers.

- **Deprecation:** Testing framework classes `InstallerExistingConfigTestBase`, `WebDriverCurlService`, `TestFileParser`, and `TestDiscovery` methods `getTestClasses()`, `findAllClassFiles()`, `scanDirectory()`, `getTestInfo()`, `parseTestClassSummary()`, `getPhpunitTestSuite()` removed; `AssertContentTrait` methods `assertNoLinkByHref()`, `assertNoLinkByHrefInMainRegion()`, `assertUniqueText()`, `assertNoUniqueText()` removed

## Technical details

`TestDiscovery` is heavily reduced and marked `@final`. The methods `getTestClasses()`, `findAllClassFiles()`, `scanDirectory()`, `getTestInfo()`, and `parseTestClassSummary()` are deleted. Only `registerTestNamespaces()` and `getExtensions()` remain. `getPhpunitTestSuite()` is moved to `PhpUnitTestDiscovery` as a public static method, and `parseTestClassSummary()` is moved there as a private static method. `PhpUnitTestDiscovery` is also marked `@final`. `TestFileParser` (`@internal`, deprecated 11.2.0) and `InstallerExistingConfigTestBase` (deprecated 10.4.0) are deleted in their entirety. `WebDriverCurlService` (deprecated 11.1.0) is deleted with no replacement. The `AssertContentTrait` methods `assertNoLinkByHref()`, `assertNoLinkByHrefInMainRegion()`, `assertUniqueText()`, and `assertNoUniqueText()` (all deprecated 11.1.0) are removed with no replacement. `ExpectDeprecationTrait` is removed from `KernelTestBase`, `BrowserTestBase`, and `UnitTestCase` (see related [#3552827](https://www.drupal.org/node/3552827)).

A bug fix lands inside `PhpUnitTestDiscovery::getTestClassInfo()`. The old code aggregated group names via `TestCase::groups()` and then checked if the result was empty, but `TestCase::groups()` always returns at least one synthetic group, so the `MissingGroupException` could never be thrown. The fixed code calls `$test->valueObjectForEvents()->metadata()->isGroup()->isEmpty()` on each individual `Test` object (and each `DataProviderTestSuite` child) before collecting groups, throwing `MissingGroupException` immediately when the PHPUnit `#[Group]` attribute is absent.

Native return types are also added throughout: `installParameters(): array`, `drupalGet(): string`, `visit(): Mink`, `instantiateServer(): Process`, and `getMinkDriverArgs(): string|false`, across installer, functional, and build-test base classes. PHPStan baseline entries for all deleted code are cleaned up in tandem.

## Upgrade

Replace `InstallerExistingConfigTestBase` with `InstallerConfigDirectoryTestBase`:

```php
// Before
use Drupal\FunctionalTests\Installer\InstallerExistingConfigTestBase;
class MyInstallerTest extends InstallerExistingConfigTestBase { ... }

// After
use Drupal\FunctionalTests\Installer\InstallerConfigDirectoryTestBase;
class MyInstallerTest extends InstallerConfigDirectoryTestBase { ... }
```

Replace `TestDiscovery::getPhpunitTestSuite()` with the equivalent on `PhpUnitTestDiscovery`:

```php
// Before
use Drupal\Core\Test\TestDiscovery;
$suite = TestDiscovery::getPhpunitTestSuite($classname);

// After
use Drupal\Core\Test\PhpUnitTestDiscovery;
$suite = PhpUnitTestDiscovery::getPhpunitTestSuite($classname);
```

Replace `WebDriverCurlService` with the base `WebDriver\Service\CurlService` directly. There is no feature-equivalent replacement.

Replace removed `AssertContentTrait` methods with `WebAssert` equivalents or direct XPath queries. There is no direct replacement for `assertNoLinkByHref()`, `assertUniqueText()`, or `assertNoUniqueText()`.

Replace `ExpectDeprecationTrait` usage with `$this->expectUserDeprecationMessage()` or `$this->expectUserDeprecationMessageMatches()` (see [#3552827](https://www.drupal.org/node/3552827)).
