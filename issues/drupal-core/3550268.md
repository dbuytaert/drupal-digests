## Summary

This change removes deprecated code from Drupal's testing framework. Several deprecated classes and methods that were previously marked for removal in Drupal 12 have been deleted, including test discovery classes, installer test base classes, and various assertion methods. The change also moves some functionality from `TestDiscovery` to `PhpUnitTestDiscovery` and adds native return type declarations to testing methods. This is an internal change to the testing framework and does not affect site owners.

## Impact

Affects module maintainers.

- **Deprecation:** Several testing framework classes and methods removed: `TestFileParser`, `InstallerExistingConfigTestBase`, `WebDriverCurlService`, and assertion methods like `assertNoLinkByHref()`, `assertUniqueText()`, `assertFieldById()`, `assertOptionByText()`

## Technical details

The change removes several deprecated components from the testing framework. The main deletions are `TestFileParser` (deprecated in 11.2.0), `InstallerExistingConfigTestBase` (deprecated in 10.4.0), `WebDriverCurlService` (deprecated in 11.1.0), and multiple deprecated assertion methods from `AssertContentTrait` (deprecated in 11.1.0).

The `TestDiscovery` class is substantially reduced, with most methods removed. Methods like `getTestClasses()`, `findAllClassFiles()`, `scanDirectory()`, `getTestInfo()`, and `parseTestClassSummary()` are deleted. The class now only handles registering test namespaces. The `parseTestClassSummary()` and `getPhpunitTestSuite()` methods are moved to `PhpUnitTestDiscovery` where they are now private.

A bug fix is included for test group detection. Previously `TestCase::groups()` always returns at least one group, making empty checks ineffective. The new code checks explicitly if the group attribute is defined using `$test->valueObjectForEvents()->metadata()->isGroup()->isEmpty()` from PHPUnit's API. If a test has no groups defined, a `MissingGroupException` is thrown.

Native return type declarations are added to several methods across the testing framework, including `installParameters()`, `drupalGet()`, and `visit()`. PHPStan baseline entries and deprecation ignore patterns are cleaned up. Test files are updated to reflect the new APIs.

## Upgrade

If extending `InstallerExistingConfigTestBase`, switch to `InstallerConfigDirectoryTestBase`:

```php
// Before
use Drupal\FunctionalTests\Installer\InstallerExistingConfigTestBase;

class MyTest extends InstallerExistingConfigTestBase {
}

// After
use Drupal\FunctionalTests\Installer\InstallerConfigDirectoryTestBase;

class MyTest extends InstallerConfigDirectoryTestBase {
}
```

If using `TestDiscovery` methods, switch to `PhpUnitTestDiscovery`:

```php
// Before
$discovery = new TestDiscovery($root, $class_loader);
$tests = $discovery->getTestClasses('my_module');

// After
$discovery = PhpUnitTestDiscovery::instance()
  ->setConfigurationFilePath($root . '/core');
$tests = $discovery->getTestClasses('my_module');
```
