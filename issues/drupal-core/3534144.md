## Summary

This meta/plan issue established the official platform requirements for Drupal 12. PHP 8.5 is now required (following the existing policy of requiring the highest stable PHP version at the first beta). Database minimums are: MySQL 8.0 (unchanged from Drupal 11), MariaDB 10.11 (up from 10.6), PostgreSQL 18 (up from 16), and SQLite 3.45 (unchanged). Composer 2.9.3 is recommended but no programmatic minimum is enforced. Browser requirements are unchanged. An official announcement was published on drupal.org and the system-requirements documentation pages were updated accordingly.

## Impact

Affects site owners.

- **Requirements:** Drupal 12 requires PHP 8.5+, MariaDB 10.11+ (up from 10.6), and PostgreSQL 18+ (up from 16); MySQL 8.0 and SQLite 3.45 minimums are unchanged from Drupal 11.

## Technical details

The issue tracks the policy decisions and their rationale rather than a single code change; the actual enforcement changes were carried in child issues ([#3537713](https://www.drupal.org/node/3537713) for PHP 8.5 and [#3569115](https://www.drupal.org/node/3569115) for MariaDB 10.11).

PHP 8.5 was required by the policy formalized in [#3406215](https://www.drupal.org/node/3406215) (require the highest stable version available at first beta). The constraint is enforced via `"php": ">=8.5.0"` in `core/composer.json`.

Database minimums are enforced through constants in each driver's `Install/Tasks.php` class and checked by `Tasks::minimumVersion()` in `core/lib/Drupal/Core/Database/Install/Tasks.php`. The final values in the repo are:

- `mysql`: `MYSQL_MINIMUM_VERSION = '8.0'` and `MARIADB_MINIMUM_VERSION = '10.11'` in `core/modules/mysql/src/Driver/Database/mysql/Install/Tasks.php`
- `pgsql`: `PGSQL_MINIMUM_VERSION = '18'` in `core/modules/pgsql/src/Driver/Database/pgsql/Install/Tasks.php`
- `sqlite`: `SQLITE_MINIMUM_VERSION = '3.45'` in `core/modules/sqlite/src/Driver/Database/sqlite/Install/Tasks.php`

MySQL was kept at 8.0 despite it reaching EOL before Drupal 12 releases; the reasoning (from catch and gábor hojtsy) was that no new MySQL-specific features are needed and that raising the database floor makes adoption harder for hosts. PostgreSQL was raised to 18 per the draft policy in [#3363102](https://www.drupal.org/node/3363102), which calls for the most recent PostgreSQL major version available six months before the expected release date (PostgreSQL always releases in September, so the six-month and three-month-before-beta heuristics converge on the same release). MariaDB was raised from 10.6 to 10.11 to align with their LTS lifecycle.

## Upgrade

Before upgrading to Drupal 12, verify your server meets the following platform requirements (changes from Drupal 11 are noted):

- PHP 8.5 or later (raised from 8.3)
- MySQL 8.0 or later (unchanged)
- MariaDB 10.11 or later (raised from 10.6)
- PostgreSQL 18 or later (raised from 16)
- SQLite 3.45 or later (unchanged)
- Composer 2.9.3 or later is recommended
