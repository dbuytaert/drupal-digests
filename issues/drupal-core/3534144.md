## Summary

Drupal 12 platform requirements have been finalized to ensure compatibility with modern hosting environments while balancing upgrade feasibility. The requirements follow established policies that select versions based on availability, stability, and support lifecycles. PHP 8.5 will be required following the policy to use the highest stable version available. Database requirements mostly remain conservative, keeping MySQL at 8.0 despite it approaching end of life, because hosts upgrade databases more slowly than other components and the security impact is low. MariaDB increases to 10.11 to follow LTS policies, and PostgreSQL jumps to version 18 released six months before the beta window.

## Impact

Affects site owners.

- **Requirements:** Drupal 12 will require PHP 8.5, MySQL 8.0 (no change), MariaDB 10.11 (up from 10.6), PostgreSQL 18 (up from 16), and SQLite 3.45 (no change), with Composer 2.9.3 recommended.

## Technical details

This meta issue coordinated the selection of platform requirements for Drupal 12.0.0 by applying policies defined in separate issues. The PHP requirement of 8.5 follows [#3406215](https://www.drupal.org/node/3406215), which established that new major versions should require the latest stable PHP release available at beta. Database version selection attempted to follow the draft policy in [#3363102](https://www.drupal.org/node/3363102), though that policy is still being finalized.

For MySQL, the team decided to keep 8.0 as the minimum despite it being EOL before Drupal 12 releases. This reflects that database upgrades are harder for hosts than PHP upgrades, and security risks from database versions are much lower than from PHP versions. Supporting newer versions is more important than requiring them. MariaDB increased to 10.11 to align with their LTS cycle. PostgreSQL moved to version 18 based on the six-month-before-release window. SQLite stayed at 3.45 as no compelling features required a bump.

Composer 2.9.3 is recommended but not strictly required, as enforcement is difficult and Package Manager now uses local Composer when available. Browser requirements remained unchanged per existing policy.
