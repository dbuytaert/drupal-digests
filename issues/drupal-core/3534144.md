## Summary

Drupal 12 platform requirements have been established to ensure sites run on secure, well-supported infrastructure. The most significant change is requiring PHP 8.5, following the new policy to adopt the latest stable PHP version at beta release. Database requirements reflect conservative increases that balance access to modern features with practical hosting constraints. Most database versions remain the same or receive modest bumps, recognizing that database upgrades are harder for hosting providers than PHP updates. These requirements align with Ubuntu LTS releases and vendor support cycles to maximize compatibility.

## Impact

Affects site owners.

- **Requirements:** Drupal 12 will require PHP 8.5, MySQL 8.0 or higher, MariaDB 10.11 or higher, PostgreSQL 18 or higher, and SQLite 3.45 or higher.

## Technical details

This meta issue coordinated the platform requirement decisions for Drupal 12 across multiple subsystems. The PHP 8.5 requirement implements the recently approved policy to require the highest stable PHP release available when a major version reaches beta.

For databases, the team applied draft policies that consider vendor LTS cycles, Ubuntu availability, and end-of-life dates. MySQL stays at 8.0 despite being near EOL because hosts struggle with database upgrades and the security surface is low. MariaDB increases from 10.6 to 10.11 to follow their LTS policy more closely. PostgreSQL bumps from 16 to 18, adopting the version available six months before release. SQLite remains at 3.45 since no pressing features require a newer version.

The Composer requirement recommends version 2.9.3 for security but doesn't enforce it strictly since Package Manager now uses local Composer when available. The decisions prioritize making Drupal 12 adoption easier for hosts while ensuring sites run on secure platforms. Multiple documentation pages were updated following the announcement.
