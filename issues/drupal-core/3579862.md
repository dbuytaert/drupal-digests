## Summary

This is a purely internal test-infrastructure change with no effect on production code or site behavior. It prepares Drupal's unit test suite for compatibility with PHPUnit 12.5.0, which will issue notices (and eventually errors) when mock objects are created without any expectations configured on them.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces a notice for any mock object created with `createMock()` that never configures an expectation, because expectation-less mocks are functionally identical to stubs. PHPUnit also deprecates `$this->any()` (equivalent to having no expectation) and `with()` on stubs.

The fix converts qualifying `createMock()` calls to `createStub()` across 14 test files in `core/tests/Drupal/Tests/Core/`: the `Datetime/` directory (`DateHelperTest`, `DateTest`, `DrupalDateTimeTest`, `TimeZoneFormHelperTest`), the `Utility/` directory (`CallableResolverTest`, `ErrorTest`, `LinkGeneratorTest`, `PhpRequirementsTest`, `TokenTest`, `UnroutedUrlAssemblerTest`), and standalone files `DrupalTest.php`, `PrivateKeyTest.php`, `UnroutedUrlTest.php`, and `UrlTest.php`. Where appropriate, `@var` docblock type hints are updated from `\PHPUnit\Framework\MockObject\MockObject` to `\PHPUnit\Framework\MockObject\Stub`. Objects that still configure real expectations (via `->expects()`) retain `createMock()`. This work is part of a broader effort across the codebase; a sibling issue [#3578904](https://www.drupal.org/node/3578904) applied the same pattern to `UnitTestCase` itself.
