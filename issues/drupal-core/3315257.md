## Summary

This meta issue tracked all the work needed to deprecate the `migrate_drupal` and `migrate_drupal_ui` modules in Drupal core following Drupal 7's end-of-life. The policy decision ([#3371229](https://www.drupal.org/node/3371229)) was to maintain both modules through Drupal 11, deprecate them in an 11.x minor release, and remove them entirely in Drupal 12. Sites that still need to migrate from Drupal 6 or 7 will have until the end of Drupal 11's support window (approximately mid-to-late 2028) to do so against a supported release of Drupal core.

## Impact

Affects module maintainers.

- **Deprecation:** `migrate_drupal` and `migrate_drupal_ui` deprecated in Drupal 11 for removal in Drupal 12; all D6/D7 source plugins (via `DrupalSqlBase`), MigrateField plugins, and D6/D7-specific destination and process plugins deprecated with no replacement; `ContentEntity` and `d8_config` source plugins moved to the `migrate` module with updated namespaces

## Technical details

The `migrate_drupal` module provides source plugins, field plugins, destination plugins, and process plugins specifically for upgrading Drupal 6 and Drupal 7 sites. With both of those versions EOL, continuing to maintain them in core is no longer justified. The deprecation work proceeded in several coordinated child issues.

Before deprecating, version-agnostic pieces were extracted to the `migrate` module so they would not be lost. The `content_entity` source plugin (`\Drupal\migrate_drupal\Plugin\migrate\source\ContentEntity`) was moved to `migrate` ([#3498915](https://www.drupal.org/node/3498915)), since it is useful for in-site data transformations unrelated to legacy Drupal versions. The `d8_config` source plugin was likewise moved to `migrate` ([#3506605](https://www.drupal.org/node/3506605)).

With those rescues in place, the remaining D6/D7-specific code was deprecated for removal in Drupal 12:

- Source plugins were covered by deprecating `DrupalSqlBase` (the common base class for approximately 107 source plugins), along with `EmptySource` and `I18nQueryTrait`, rather than annotating every individual class ([#3507572](https://www.drupal.org/node/3507572)).
- MigrateField plugins were deprecated ([#3502749](https://www.drupal.org/node/3502749)).
- Migrate destination plugins required only for site upgrades were deprecated ([#3502752](https://www.drupal.org/node/3502752)).
- Migrate process plugins required only for site upgrades were deprecated ([#3502755](https://www.drupal.org/node/3502755)).
- An earlier annotations-to-attributes conversion for `migrate_drupal` source plugins was reverted as part of cleanup ([#3009349](https://www.drupal.org/node/3009349)).
- Test integrations in the `migrate` module that depended on `migrate_drupal` were cleaned up ([#3573895](https://www.drupal.org/node/3573895)).

The `migrate_drupal` module itself is marked deprecated in Drupal 11 and will become obsolete in Drupal 12. The `migrate_drupal_ui` module follows the same schedule.

## Upgrade

Code that extends or directly references `\Drupal\migrate_drupal\Plugin\migrate\source\ContentEntity` should be updated to use the new class in the `migrate` module. Code that extends `DrupalSqlBase` or other deprecated source, field, destination, or process plugins in `migrate_drupal` will need to be updated before Drupal 12.
