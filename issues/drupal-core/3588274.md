## Summary

New Drupal installations using the Standard profile or the standard recipe no longer include a "Tags" taxonomy vocabulary. The `content_editor` role also no longer receives the `create terms in tags` and `edit terms in tags` permissions. Sites that need a Tags vocabulary must now create one manually after installation. This is part of a broader effort to pare down the Standard profile to a minimal baseline, following the earlier removal of the Article and Basic page content types ([#3587118](https://www.drupal.org/node/3587118)) and the Comment module dependency ([#3588276](https://www.drupal.org/node/3588276)).

## Impact

Affects site owners.

- **Config change:** The `taxonomy.vocabulary.tags` config object and the `create terms in tags` / `edit terms in tags` permissions on `content_editor` are no longer included in the Standard profile or standard recipe.

## Technical details

The change deletes `taxonomy.vocabulary.tags.yml` from the standard recipe config directory. The `content_editor` role config (`content_editor.yml`) is updated to drop its dependency on both `taxonomy.vocabulary.tags` config and the `taxonomy` module itself, and to remove the `create terms in tags` and `edit terms in tags` permissions. The same permission removals are mirrored in the standard recipe's own role config snapshot. The `StandardTest` functional test is updated to remove `taxonomy.vocabulary.tags` from the expected list of installed config objects. The change was motivated by the fact that after the removal of the Article content type (which previously used the Tags field), the vocabulary was left floating with nothing referencing it.

## Upgrade

If your site's deployment scripts, migration code, or tests assume the `tags` vocabulary exists in a fresh Standard install, you must either create the vocabulary explicitly or add it via a custom recipe/install hook. The `content_editor` role will also no longer have taxonomy term permissions, so any code granting that role and expecting tag editing rights must be updated.
