## Summary

Drupal core's root `composer.json` now includes two convenience scripts for running PHPStan. Contributors can run `composer phpstan` to analyse files and `composer phpstan-baseline` to regenerate `core/.phpstan-baseline.php`, instead of having to look up or remember the full command strings each time.

## Impact

No upgrade or configuration changes required.

## Technical details

The change adds two entries to the `scripts` section of the root `composer.json`. The `phpstan` script runs `phpstan analyze --memory-limit=-1 --configuration=core/phpstan.neon.dist --`, accepting additional file arguments after the `--` separator. The `phpstan-baseline` script adds `--generate-baseline=core/.phpstan-baseline.php` to regenerate the baseline file. The `--memory-limit=-1` flag was added after mstrelan noted it was required on their setup. The scripts mirror the manual commands documented at drupal.org/docs/develop/development-tools/phpstan/phpstan-in-drupal-core and follow the existing pattern of `phpcs` and `phpcbf` scripts already present in `composer.json`.
