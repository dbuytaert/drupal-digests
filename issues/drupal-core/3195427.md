## Summary

The Olivero theme was missing CSS rules for Drupal core's responsive tables API, which uses `priority-low` and `priority-medium` classes on table headers and cells to hide lower-priority columns on narrow viewports and surface a "Show all columns" toggle button. Because those rules were absent, the toggle button stayed hidden and all columns remained visible regardless of screen width. This fix adds the missing CSS to Olivero so responsive tables behave correctly: medium-priority columns are hidden below a medium breakpoint and low-priority columns below a large breakpoint, while a styled toggle button lets users reveal them. All columns are intentionally shown when printing.

## Impact

No upgrade or configuration changes required.

## Technical details

The `drupal.tableresponsive` JS library (in `core/misc/tableresponsive.js`) detects responsive table support by checking whether `priority-medium` or `priority-low` cells are hidden via CSS. If the rules are not present in the active theme, the library never renders the toggle button and never hides any columns. Olivero's `table.pcss.css` lacked these rules entirely, while Claro had them in its `css/components/tables.pcss.css`.

The fix adds two blocks to Olivero's `core/themes/olivero/css/components/table.pcss.css`: `th.priority-medium, td.priority-medium` default to `display: none` and are shown at the `--md` custom media query, and `th.priority-low, td.priority-low` default to `display: none` and are shown at the `--lg` custom media query. Both media queries carry the `screen` keyword so that `display: none` is not applied in print context, ensuring all columns appear when the page is printed.

A companion addition to `core/themes/olivero/css/components/button.pcss.css` styles `button.link` as a link-like element (transparent background, no border, `var(--color-text-primary-medium)` color, hover and focus states) so the toggle button inserted by `tableresponsive.js` matches the Olivero design instead of appearing as an unstyled browser button.
