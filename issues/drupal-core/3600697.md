## Summary

Sites using the JSON:API Resources or JSON:API Extras contrib modules were logging `[warning] Array to string conversion AttributeRouteDiscovery.php:41` on every cache rebuild. This caused functional tests with `failOnWarning=true` to fail unexpectedly. The warning is now gone and attribute-based route discovery works correctly alongside those modules.

## Impact

No upgrade or configuration changes required.

## Technical details

`AttributeRouteDiscovery::collectRoutes()` (introduced in [#3311365](https://www.drupal.org/node/3311365) to support PHP `#[Route]` attributes on controllers) iterated over `container.namespaces` and assumed every namespace value was a plain string directory path. It appended `/Controller` directly to that value. PSR-4 namespace registrations are allowed to map a single namespace to an array of base directories, and both JSON:API Resources and JSON:API Extras do exactly that: they register extra paths under the `Drupal\jsonapi\Normalizer\ImpostorFrom\*` namespace by pushing onto an existing `container.namespaces` key, producing an array. Concatenating a string onto an array triggered PHP's `Array to string conversion` notice and effectively made the scanner check a nonsense path, silently skipping attribute route discovery for those namespaces.

The fix wraps the per-namespace value in `(array)` and adds a middle loop, so a string value becomes a one-element array and an existing array is iterated as-is. The `@param` docblock on the constructor was updated to document the value type as `string|array<int, string>`. A new test covers the array-of-directories case using `router_test_directory`, including a deliberately missing path to verify the scanner skips non-existent directories gracefully.
