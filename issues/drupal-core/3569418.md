## Summary

Unit tests in the Block module have been updated to replace expectation-less mock objects with stubs, a prerequisite for compatibility with PHPUnit 12.5+. The change has no user-facing effect; it only affects the test suite.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5+ introduces notices (and eventually errors) when a mock object created with `createMock()` or `getMockBuilder()->getMock()` has no configured expectations, because such an object should be a plain stub instead. Additionally, `$this->any()` is deprecated as it is equivalent to having no expectation, and `with()` is deprecated for stubs. This issue is part of the broader meta effort tracked in [#3561671](https://www.drupal.org/node/3561671) to make Drupal core's test suite fully compatible with PHPUnit 12.5+.

Six test files in `core/modules/block/tests/src/Unit/` were updated:

- `BlockConfigEntityUnitTest.php`: `$this->entityType` and `$this->uuid` converted from `createMock()` to `createStub()`. The `$this->any()` expectation on `entityTypeManager->getDefinition()` was tightened to `$this->once()`.
- `BlockFormTest.php`: Seven objects (`$this->conditionManager`, `$this->language`, `$this->contextRepository`, `$this->entityTypeManager`, `$this->themeHandler`, `$this->themeManager`, `$this->contextHandler`) converted to `createStub()`. The `$this->any()` call on `entityTypeManager->getStorage()` was removed (stubs do not use `expects()`). In `getBlockMockWithMachineName()`, `getMockBuilder()->getMock()` with `$this->any()` was replaced by `createStub()` for both `$plugin` and `$block`.
- `BlockRepositoryTest.php`: `$this->contextHandler` and the local `$entity_type_manager` converted to stubs; `$this->any()` removed from `getStorage()`, `label()`, and `getWeight()` method stubs.
- `CategoryAutocompleteTest.php`: `$block_manager` converted to `createStub(BlockManagerInterface::class)` with `$this->any()` removed.
- `Menu/BlockLocalTasksTest.php`: `$theme_handler` converted to `createStub(ThemeHandlerInterface::class)` with `$this->any()` removed from `listInfo()` and `hasUi()`.
- `Plugin/DisplayVariant/BlockPageVariantTest.php`: `CacheContextsManager` converted from `getMockBuilder()->onlyMethods()->getMock()` to `createStub()`; four block plugin objects (`$block_plugin`, `$main_content_block_plugin`, `$messages_block_plugin`, `$title_block_plugin`) converted to stubs. Two tests (`testBuildWithoutMainContent()` and `testCacheMetadataFromPlugin()`) gained explicit `$this->never()` expectations on `$this->blockViewBuilder->method('view')` to document that the view builder is not called in those code paths. `$route_match` converted to `createStub(RouteMatchInterface::class)`.

String-form class names in `createMock()` and `createStub()` calls were also modernised to `ClassName::class` constants throughout.
