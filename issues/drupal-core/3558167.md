## Summary

This is a purely internal test-infrastructure fix with no effect on site behavior. It corrects PHPStan type errors in several unit test classes by updating mock object property declarations to use PHP intersection types (`FooInterface&MockObject`) instead of imprecise union-type docblocks or bare untyped properties. One additional bug was fixed where a `MockBuilder` instance was being passed where a `MockObject` was expected.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPStan, when `checkFunctionArgumentTypes: true` is enabled, reported 17 violations of the form "expects FooInterface, MockObject given" in unit test classes. The root cause is that `TestCase::createMock()` returns `MockObject&RealInstanceType` (a PHP intersection type), but many test class properties were declared with a union-type `@var` docblock such as `FooInterface|MockObject` or just `@var MockObject` with no interface at all. A union type tells PHPStan the value is one or the other, not both simultaneously, so passing such a property to a method that expects `FooInterface` triggers a type mismatch.

The fix upgrades property declarations in six test files to use proper PHP 8.1 intersection type syntax directly on the property, dropping the `@var` docblock entirely. The affected files and their changed properties are:

- `core/modules/image/tests/src/Unit/ImageStyleTest.php`: The `$image_effect` local variable was assigned a bare `MockBuilder` instance (the result of `getMockBuilder()` alone, without calling `->getMock()`). The fix chains `.disableOriginalConstructor()->getMock()` to produce the actual `MockObject`.
- `core/modules/locale/tests/src/Unit/LocaleTranslationTest.php`: Properties `$storage`, `$lock`, `$cache`, `$languageManager` updated to intersection types (`StringStorageInterface&MockObject`, `LockBackendInterface&MockObject`, etc.) with the respective interface `use` statements added.
- `core/modules/migrate_drupal/tests/src/Unit/source/DrupalSqlBaseTest.php`: Properties `$state` and `$entityTypeManager` changed to `StateInterface&MockObject` and `EntityTypeManagerInterface&MockObject`.
- `core/tests/Drupal/Tests/Core/Access/AccessManagerTest.php`: Properties `$routeProvider`, `$paramConverter`, `$account`, `$argumentsResolverFactory`, and `$currentUser` updated to intersection types; missing `@var` description on `$currentUser` also filled in.
- `core/tests/Drupal/Tests/Core/Cache/CacheCollectorTest.php`: Properties `$cacheBackend`, `$cacheTagsInvalidator`, and `$lock` updated; missing interface `use` statements added.
- `core/tests/Drupal/Tests/Core/Menu/LocalTaskManagerTest.php`: Properties `$argumentResolver`, `$routeProvider`, `$pluginDiscovery`, `$factory`, `$accessManager`, `$routeMatch`, and `$account` updated to intersection types with the required `use` statements.

This issue is part of the broader [#3404246](https://www.drupal.org/node/3404246) meta to enable strict phpstan function-argument type checking across Drupal core. Reviewer mondrake noted that some of these mocks could potentially be stubs instead (tracked in [#3561671](https://www.drupal.org/node/3561671)), but that refactoring was kept out of scope here to avoid scope creep.
