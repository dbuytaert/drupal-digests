## Summary

Drupal's `FormBuilder` class and its interface now formally declare a variadic `mixed ...$args` parameter on `getForm()` and `submitForm()`, replacing the old `func_get_args()` workaround that had been in place while the interface change was being phased in. This completes a migration prepared in [#3355839](https://www.drupal.org/node/3355839). The change is internal to the forms system and has no effect on sites that do not implement `FormBuilderInterface` directly.

## Impact

Affects module maintainers.

- **API change:** `FormBuilderInterface::getForm()` and `FormBuilderInterface::submitForm()` now formally declare `mixed ...$args`; custom implementations of the interface must add this parameter.

## Technical details

The Drupal forms system lets callers pass extra arguments to `FormBuilderInterface::getForm()` and `FormBuilderInterface::submitForm()` beyond the required `$form_arg` and `$form_state`. These extra arguments end up in `$form_state->getBuildInfo()['args']` and are available to form constructors and `hook_form_alter()` implementations.

Before this change, the interface methods had the variadic parameter commented out (e.g. `getForm($form_arg /* , mixed ...$args */)`), held back by a `@todo` referencing issue [#3354672](https://www.drupal.org/node/3354672) that tracked safely introducing the interface change. The implementations in `FormBuilder` used `func_get_args()` to collect the extra arguments at runtime, then manually stripped the leading positional parameters with `unset($args[0])` or `unset($args[0], $args[1])` before calling `$form_state->addBuildInfo('args', array_values($args))`.

The commit activates the parameter on both the interface (`FormBuilderInterface.php`) and the implementation (`FormBuilder.php`). `getForm()` becomes `getForm($form_arg, mixed ...$args)` and `submitForm()` becomes `submitForm($form_arg, FormStateInterface &$form_state, mixed ...$args)`. Both implementations now pass `$args` directly to `addBuildInfo()`, eliminating the `func_get_args()` calls and the manual index-removal logic.

Two suppression patterns that silenced the PHP deprecation warnings about these missing variadic parameters were also removed from `core/.deprecation-ignore.txt`, since the warnings can no longer fire once the parameters are declared. The phpcs:disable/enable comment pairs and `@todo` annotations in the interface docblocks were cleaned up at the same time.

## Upgrade

Any custom class that directly implements `FormBuilderInterface` must update its method signatures to include the variadic parameter:

```php
// Before
public function getForm($form_arg) { ... }
public function submitForm($form_arg, FormStateInterface &$form_state) { ... }

// After
public function getForm($form_arg, mixed ...$args) { ... }
public function submitForm($form_arg, FormStateInterface &$form_state, mixed ...$args) { ... }
```

Classes that extend `FormBuilder` rather than implementing the interface directly are unaffected unless they override these two methods without declaring the variadic parameter.
