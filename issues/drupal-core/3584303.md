## Summary

Several internal identifiers in the `default_admin` core theme still carried their original `gin_` prefix from the Gin theme they were derived from. This change renames those identifiers so that form element keys, Twig template variables, and HTML element IDs consistently use the `default_admin_` prefix, removing residual Gin branding from the codebase.

## Impact

Affects module maintainers.

- **API change:** Form element keys (`gin_sticky_actions`, `gin_sidebar`, etc.) and preprocess template variables (`gin_form_actions`, `gin_breadcrumbs`, etc.) renamed to use the `default_admin_` prefix; HTML element ID `#gin_sidebar` renamed to `#default_admin_sidebar`.

## Technical details

The change is part of the broader effort to remove Gin references from the `default_admin` theme (see the meta issue [#3576494](https://www.drupal.org/node/3576494)). The scope was explicitly limited to `gin_`-prefixed form element keys and template variables, not PHP local variables such as `$gin_info`.

In `FormAlter.php`, the form element keys injected by `stickyActionButtonsAndSidebar()` and `formAfterBuild()` were renamed: `gin_sticky_actions` to `default_admin_sticky_actions`, `gin_sidebar` to `default_admin_sidebar`, `gin_sidebar_toggle` to `default_admin_sidebar_toggle`, `gin_sidebar_close` to `default_admin_sidebar_close`, `gin_sidebar_overlay` to `default_admin_sidebar_overlay`, and `gin_move_focus_to_sticky_bar` to `default_admin_move_focus_to_sticky_bar`.

In `PreprocessHooks.php`, the template variables set by `preprocessPage()` and `preprocessTopBar()` were renamed: `gin_form_actions` to `default_admin_form_actions`, `gin_form_actions_class` to `default_admin_form_actions_class`, `gin_local_actions` to `default_admin_local_actions`, and `gin_breadcrumbs` to `default_admin_breadcrumbs`.

The HTML element ID `gin_sidebar` used in two layout templates (`navigation/content-sidebar.html.twig` and `node-edit-form.html.twig`), referenced in CSS (`gin.css`) and in the JS sidebar resize handler, was likewise renamed to `default_admin_sidebar`. The `once()` call that initialises the sidebar also targets the new ID. The corresponding Twig templates (`top-bar--gin.html.twig`, `page--admin.html.twig`, `node-edit-form.html.twig`) were updated to consume the renamed variables.

## Upgrade

Any contributed module or theme that references the old `gin_`-prefixed form element keys or template variables from the `default_admin` theme will need to update those references:

```php
// Before
$form['gin_sticky_actions']
$form['gin_sidebar']
$form['gin_sidebar_toggle']
$variables['gin_form_actions']
$variables['gin_form_actions_class']
$variables['gin_local_actions']
$variables['gin_breadcrumbs']

// After
$form['default_admin_sticky_actions']
$form['default_admin_sidebar']
$form['default_admin_sidebar_toggle']
$variables['default_admin_form_actions']
$variables['default_admin_form_actions_class']
$variables['default_admin_local_actions']
$variables['default_admin_breadcrumbs']
```

Custom templates or CSS/JS targeting the HTML element ID `#gin_sidebar` must change it to `#default_admin_sidebar`.
