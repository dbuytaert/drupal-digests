## Summary

Batch processing large numbers of entities is now faster. When processing thousands of entities and accessing multiple field values, the system was repeatedly calculating the same data type information. This optimization stores that information after the first calculation instead of recalculating it every time. In one test case processing 4,000 entities, this eliminated 800,000 unnecessary function calls. Sites that process large numbers of entities in batch operations will see performance improvements.

## Impact

No upgrade or configuration changes required.

## Technical details

The `EntityDataDefinition::getDataType()` method constructs a data type string based on entity type and bundle information. This method was being called repeatedly during entity processing, particularly when accessing field values on multiple entities.

The optimization adds a `$dataType` property that caches the result after the first call. Subsequent calls to `getDataType()` return the cached value instead of rebuilding the string.

To prevent the cached value from being stored when the object is serialized (such as when stored in cache), a `__sleep()` method was added that clears the `$dataType` property before serialization. This keeps serialized objects smaller and ensures the cache is always reconstructed from the source data when needed.

The initial test failure occurred because a test was serializing two objects where only one had called `getDataType()`, making the serialized versions different. Clearing the property during serialization resolved this.
