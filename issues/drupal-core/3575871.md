## Summary

Several core tests attempt to install all stable modules to verify functionality like configuration import, REST coverage, and library definitions. These tests were failing when modules were marked as obsolete because the tests only filtered out deprecated modules. This change updates seven test classes to skip obsolete modules in addition to deprecated ones, preventing installation failures and unblocking the removal of obsolete modules from core.

## Impact

No upgrade or configuration changes required.

## Technical details

Tests that install all stable modules need to filter out modules that are not suitable for installation. Previously these tests already skipped experimental and deprecated modules, but not obsolete ones. When modules began being marked as `ExtensionLifecycle::OBSOLETE`, these tests started failing.

The fix adds checks for `ExtensionLifecycle::OBSOLETE` alongside existing `ExtensionLifecycle::DEPRECATED` checks in seven test files: `ConfigImportAllTest`, `TestCoverageTest` (JSON:API), `EntityResourceRestTestCoverageTest` (REST), `ResolvedLibraryDefinitionsFilesMatchTest`, `ModuleConfigureRouteTest`, `Stable9TemplateOverrideTest`, and `StableLibraryOverrideTestBase`.

Each test uses filtering logic to determine which modules to install. The change adds a new condition to exclude modules with the obsolete lifecycle status. This prevents the module installer from attempting to enable modules that are no longer supported, which would cause test failures. This pattern matches the existing approach for deprecated modules, ensuring consistency across lifecycle states.
