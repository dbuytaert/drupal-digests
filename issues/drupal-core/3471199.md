## Summary

The deprecation-warning code for the `sid_length` and `sid_bits_per_character` session parameters has been removed from Drupal core. These parameters had already stopped having any effect in Drupal 11.1.0, when PHP 8.4 deprecated the underlying ini settings and Symfony's `NativeSessionStorage` dropped support for them. From this point on, Drupal no longer warns about them. Sites that still declare these parameters in their services YAML files or settings should remove them, as they are silently ignored.

## Impact

Affects module maintainers.

- **Deprecation:** The `sid_length` and `sid_bits_per_character` keys in `session.storage.options` were deprecated in drupal:11.1.0; the deprecation-check code that issued those warnings has now been removed entirely.

## Technical details

The parent issue [#3465836](https://www.drupal.org/node/3465836) first addressed PHP 8.4's deprecation of the `session.sid_length` and `session.sid_bits_per_character` ini settings. As part of that work, `CoreServiceProvider::register()` in `core/lib/Drupal/Core/CoreServiceProvider.php` gained a runtime check: if `session.storage.options` contained a `sid_length` or `sid_bits_per_character` key, Drupal would call `@trigger_error()` with an `E_USER_DEPRECATED` notice pointing to the drupal:11.1.0 deprecation and the drupal:12.0.0 removal target. Entries suppressing the corresponding Symfony 7.2 `NativeSessionStorage` deprecation notices were also added to `core/.deprecation-ignore.txt`.

This commit removes both `@trigger_error()` calls from `CoreServiceProvider::register()`, eliminating the entire "Deprecated parameters" block. The underlying `session.storage.options` parameter is still reachable in the container, but Drupal no longer inspects it for these keys. The `.deprecation-ignore.txt` suppression entries for the Symfony-side warnings are also gone from the current tree, leaving no remaining references to either setting anywhere in core. The fix required Symfony 7.2 to be the minimum (available in Drupal 11.1) before the runtime check could be safely dropped, which is why the issue sat open for several months after creation.

## Upgrade

Remove `sid_length` and `sid_bits_per_character` from any `session.storage.options` block in your services YAML files (for example, `services.yml`, `services.pantheon.production.yml`, or similar hosting-provider overrides):

```yaml
# Before (remove these lines):
parameters:
  session.storage.options:
    sid_length: 48
    sid_bits_per_character: 6

# After: simply omit sid_length and sid_bits_per_character entirely.
```

These keys have had no effect since Drupal 11.1.0. Leaving them in place no longer triggers a deprecation notice, but they remain dead configuration.
