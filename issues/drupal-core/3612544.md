## Summary

Drupal core now ships with Guzzle 8.0, upgraded from 7.x. The `composer.json` constraint is broadened to `^7.10 || ^8.0` (and `guzzlehttp/psr7` to `^2.8.0 || ^3.0`), so sites can still resolve Guzzle 7 when other dependencies force it. The installed lock file pins `guzzlehttp/guzzle` 8.0.0, `guzzlehttp/psr7` 3.0.0, and `guzzlehttp/promises` 3.0.0. The dev dependency `php-http/guzzle7-adapter` is removed because no Guzzle 8 compatible release exists for it.

## Impact

Affects site owners.

- **Requirements:** Guzzle 8.0.0, guzzlehttp/psr7 3.0.0, and guzzlehttp/promises 3.0.0 are now locked; core composer.json allows ^7.10 || ^8.0 for guzzle and ^2.8.0 || ^3.0 for psr7.

## Technical details

The `composer.json` version constraints for `guzzlehttp/guzzle` and `guzzlehttp/psr7` are widened to accept both major versions. `ralouphie/getallheaders` drops out of the lock file because `guzzlehttp/psr7` 3.x no longer depends on it.

Guzzle 8 removes `ClientInterface::getConfig()` entirely, so three PHPStan `$ignoreErrors` entries suppressing that deprecation warning (in `BrowserTestBase`, `ClientFactoryTest`, and `DrupalTestBrowser`) are deleted. In `DrupalTestBrowser::doRequest()`, the catch block previously caught `RequestException` and had to guard against a null response; Guzzle 8's `BadResponseException` (covering 4xx and 5xx responses) always carries a response, so the null-response branch and the old `RequestException` import are removed.

Guzzle 8 requires handler callables to return a `PromiseInterface` rather than a `ResponseInterface` directly. A mock handler in `LocaleTranslationTest` is updated to wrap its `Response` in `Create::promiseFor()` and declare the correct return type signature `function(RequestInterface $request, array $options): PromiseInterface`.

Guzzle 8's `CookieJar` no longer stores a `Secure`-flagged cookie received over a non-HTTPS connection (the HTTPS mock used in browser tests is not a real TLS connection). `LoginHttpsTest::loginHttps()` is updated to parse the `Set-Cookie` response header directly with `SetCookie::fromString()` instead of calling `CookieJar::getCookieByName()`. A companion fix in `BrowserTestBase::getSessionCookies()` guards against a `null` session ID before building the cookie jar.

Guzzle 8 also trims the trailing newline that earlier versions appended to the response body excerpt in exception messages, so several test string assertions drop the trailing `\n`.

## Upgrade

**Custom Guzzle handlers** must return a `PromiseInterface` instead of a `ResponseInterface`:

```php
// Before (Guzzle 7)
$handler = function (RequestInterface $request, array $options): ResponseInterface {
    return new Response(200, [], 'body');
};

// After (Guzzle 8)
use GuzzleHttp\Promise\Create;
use GuzzleHttp\Promise\PromiseInterface;

$handler = function (RequestInterface $request, array $options): PromiseInterface {
    return Create::promiseFor(new Response(200, [], 'body'));
};
```

**Exception handling** for 4xx/5xx responses should use `BadResponseException` instead of `RequestException`:

```php
// Before (Guzzle 7)
use GuzzleHttp\Exception\RequestException;
catch (RequestException $e) {
    if (!$e->hasResponse()) {
        throw $e;
    }
    $response = $e->getResponse();
}

// After (Guzzle 8)
use GuzzleHttp\Exception\BadResponseException;
catch (BadResponseException $e) {
    $response = $e->getResponse();
}
```

**Remove any calls to `getConfig()`** on `GuzzleHttp\Client` or `GuzzleHttp\ClientInterface`; that method is removed in Guzzle 8. Store configuration values your code needs at construction time instead of reading them back from the client.
