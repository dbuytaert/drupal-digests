## Summary

Drupal core's `composer.json` now includes a `funding` key pointing to `https://www.drupal.org/composer-fund`. When Drupal is installed as a dependency of another project, running `composer fund` will surface a link showing how to support Drupal financially. Previously, Drupal was absent from that list even though the Composer tooling is designed to surface exactly this kind of information.

## Impact

No upgrade or configuration changes required.

## Technical details

The `funding` key is a standard Composer schema field (documented at https://getcomposer.org/doc/04-schema.md#funding). The change adds it to `core/composer.json` with `{"type": "other", "url": "https://www.drupal.org/composer-fund"}`. The `composer.lock` entry for the `drupal/core` package is updated in parallel to reflect the same `funding` block and a new `reference` hash. The type was deliberately kept as `"other"` rather than `"community"` because `"other"` is the value documented by Composer and no rationale was offered for diverging from that. The URL was updated from the originally proposed `https://www.drupal.org/association/donate` by drumm following a Slack discussion among drumm, longwave, and hestenet. Template `composer.json` files under `composer/Template/` were intentionally left unchanged: `composer fund` only surfaces funding info for packages listed as dependencies, not for the root project itself, so adding the key to project templates would have no effect.
