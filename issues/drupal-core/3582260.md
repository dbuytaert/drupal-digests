## Summary

A PHP fatal error was caused by an incorrect namespace declaration in the `NodeSearchTest` kernel test class. The file at `core/modules/search/tests/src/Kernel/node/NodeSearchTest.php` declared the namespace `Drupal\Tests\search\Kernel\node` when it should have been `Drupal\Tests\search\Kernel`, causing PHP to attempt to redeclare the class and crashing the entire `PhpUnitApiGetTestClassesTest` test suite. This is an internal test infrastructure fix with no impact on production site behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

The file `core/modules/search/tests/src/Kernel/node/NodeSearchTest.php` contained the namespace declaration `Drupal\Tests\search\Kernel\node`. The PSR-4 autoloader resolved the class as `Drupal\Tests\search\Kernel\NodeSearchTest`, which conflicted with the declared namespace and caused PHP to raise a fatal "Cannot redeclare class" error when `PhpUnitApiGetTestClassesTest::testEquality` attempted to enumerate all test classes. The one-line fix corrects the `namespace` statement to `Drupal\Tests\search\Kernel`, matching what the autoloader derives from the project's PSR-4 configuration. Notably, the fix was committed to both `11.x` and `main` by catch, then reverted on both branches shortly after; the issue was subsequently auto-closed after two weeks of inactivity.
