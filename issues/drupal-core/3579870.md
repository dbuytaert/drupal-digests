## Summary

Unit tests in Drupal's Plugin library were updated to align with PHPUnit 12.5.0's stricter distinction between mocks and stubs. This is a test-code-only change with no effect on production behavior or the public API.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 deprecates `$this->any()` and emits a notice when a mock object has no configured expectations, requiring that expectation-free mocks be converted to stubs. The fix touches eleven test files under `core/tests/Drupal/Tests/Core/Plugin/` and cleans up the PHPStan baseline in `core/.phpstan-baseline.php`.

The concrete changes follow two patterns:

1. **Mock-to-stub conversion.** Objects that existed only to stub return values and carried no assertion about how many times they would be called were converted from `$this->createMock(Interface::class)` (with `->expects($this->any())`) to `$this->createStub(Interface::class)` (no expectation at all). Affected classes include `ModuleHandlerInterface`, `ModuleExtensionList`, `ContextDefinitionInterface`, `TypedDataInterface`, `DataDefinitionInterface`, `ContextInterface`, and `DiscoveryInterface`.

2. **Expectation tightening.** Where the original code called `->expects($this->any())` but the test body actually implied a specific call count, the expectation was replaced with a more precise matcher. For example, `DefaultLazyPluginCollectionTest` changed `$this->any()` to `$this->never()` for plugin-manager creation in sort-helper tests (the sort code should not instantiate plugins), and `DefaultSingleLazyPluginCollectionTest` changed `$this->any()` to `$this->atLeastOnce()` for tests that do exercise the factory.

A special case appears in `ContextTest`: the shared `$this->typedDataManager` stub created in `setUp()` is insufficient for `setUpDefaultValue()`, which needs an exact `once()` expectation on `->create()`; that method therefore re-creates the manager as a mock with a precise expectation, while other test methods continue to use the shared stub.

`HookDiscoveryTest` was refactored to construct `HookDiscovery` inline per test rather than once in `setUp()`, letting each test choose a stub or a mock independently.

Ten `$ignoreErrors` entries in `core/.phpstan-baseline.php` covering the `any()` deprecation across these files were removed because the violation no longer exists.

dcam noted in comment [#16516892](https://www.drupal.org/node/16516892) that four residual notices in this area depended on the companion fix in [#3578904](https://www.drupal.org/node/3578904) (Convert expectation-less test mocks to stubs, UnitTestCase).
