## Summary

Drupal's rendering system has been modernized to use Symfony's service locator pattern instead of an older compiler pass approach. This internal refactoring improves code quality and reduces complexity without changing how Drupal renders content. The change removes approximately 55 lines of code while making the dependency injection pattern clearer and more maintainable. Site owners and developers will not notice any functional difference in how their sites work.

## Impact

No upgrade or configuration changes required.

## Technical details

The `MainContentViewSubscriber` previously used a custom compiler pass (`MainContentRenderersPass`) to collect all services tagged with `render.main_content_renderer` into a container parameter. The subscriber then used the class resolver to instantiate the appropriate renderer based on format.

This has been replaced with Symfony's autowired service locator pattern. The `#[AutowireLocator]` attribute automatically creates a service locator containing all tagged services, indexed by the `format` attribute. The subscriber now receives this `ServiceLocator` directly and calls `get()` to retrieve renderers.

The constructor signature changed from accepting `ClassResolverInterface`, `RouteMatchInterface`, and an array parameter to accepting only `RouteMatchInterface` and `ServiceLocator`. The service is now autowired in `core.services.yml` instead of having explicit arguments.

The `MainContentRenderersPass` compiler pass and the `main_content_renderers` parameter were deleted entirely. The documentation in `core.api.php` was updated to reference `TaggedHandlersPass` as an example instead.

This modernizes the code to use Symfony best practices and reduces the amount of custom Drupal-specific dependency injection code.
