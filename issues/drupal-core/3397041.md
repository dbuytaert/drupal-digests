## Summary

Drupal core modules can now automatically find and inject most of the dependencies their services need, without requiring manual configuration. This reduces boilerplate in service definition files and makes it easier for developers to work with dependency injection. The change affects dozens of core modules, removing hundreds of lines of repetitive dependency wiring code while maintaining the same runtime behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

This issue enables autowiring for core module services by adding `autowire: true` to the `_defaults` section of each module's `*.services.yml` file. Autowiring allows the dependency injection container to automatically resolve constructor arguments based on type hints, eliminating the need to manually list each dependency in the `arguments` key.

The change removes explicit `arguments` declarations from services where all constructor parameters can be resolved by autowiring. Services that require explicit configuration (like those with scalar parameters, specific service configurations, or complex setups) retain their manual arguments.

A new test in `AutowireTest::testCoreAutowiring()` validates that services are properly configured for autowiring. The test parses all core service files, checks which services have constructor parameters that can be autowired, and ensures developers remove manual arguments when autowiring is possible.

This builds on the autowiring infrastructure introduced in earlier versions and follows [#3295751](https://www.drupal.org/node/3295751), which enabled autowiring for `core/core.services.yml`. Module services follow the same pattern, making the codebase more consistent and reducing maintenance overhead when service dependencies change.
