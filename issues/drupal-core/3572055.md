## Summary

A PHPUnit runner change broke the efficient test-skipping strategy for REST functional tests that do not support XML deserialization. Before this fix, 116 tests were each taking roughly 7 seconds of CPU time to skip because `setUp()` was running in full before the skip was applied. After the fix the same tests skip in under a quarter of a second each, saving around 13 minutes of total CI time across the suite. The change is internal to the test infrastructure and has no effect on production sites.

## Impact

No upgrade or configuration changes required.

## Technical details

Issue [#3462264](https://www.drupal.org/node/3462264) introduced `#[Before]` attribute methods in several REST entity resource test base classes and `XmlEntityNormalizationQuirksTrait` to skip tests for formats that cannot be deserialized (chiefly XML) before any expensive setup work ran. A subsequent PHPUnit runner change altered the execution order so that `setUp()` is now called before `#[Before]` hooks fire, meaning the skip no longer short-circuits the costly bootstrap.

The fix removes every `#[Before]`-attributed method and moves the `markTestSkipped()` calls to `setUp()` overrides instead, which is the correct hook for pre-test skipping in PHPUnit. The affected classes are `CommentResourceTestBase`, `FileResourceTestBase`, `ImageStyleResourceTestBase`, `NodeResourceTestBase`, `TermResourceTestBase`, and `UserResourceTestBase`. In `XmlEntityNormalizationQuirksTrait` the entire `xmlEntityNormalizationQuirksTraitSkipTests()` method is deleted; its logic (skipping `testPatch` and `testPost` for XML) is absorbed into the existing `setUp()` of `EntityResourceTestBase` directly. All new `setUp()` overrides call `parent::setUp()` after the skip guard, matching Drupal test conventions. The `PHPUnit\Framework\Attributes\Before` import is removed from every affected file.
