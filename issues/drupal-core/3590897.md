## Summary

When using the default admin theme and clicking the media browser on a media reference field, the sidebar toggle button was incorrectly rendered inside the media library dialog. This allowed users to trigger main-page navigation from within the overlay. The fix removes the spurious button from all AJAX-driven dialogs, including media library popups opened via media reference fields.

## Impact

No upgrade or configuration changes required.

## Technical details

The problem was in `FormHooks::stickyActionButtonsAndSidebar()` in the default admin theme. The early-return guard that prevents injecting the sidebar toggle and sticky action button markup into dialogs used a private `isModalOrOffcanvas()` helper. That helper checked the `_wrapper_format` request parameter for `drupal_modal` or `drupal_dialog` substrings. However, the media library opened from a media reference field sends `_wrapper_format=drupal_ajax`, which does not match either string, so the guard never fired and the sidebar toggle was included in the form output.

The fix replaces the call to the now-deleted `isModalOrOffcanvas()` with a call to the existing `isAjax()` helper. Because the default admin theme does not support sticky action buttons in any AJAX context, the broader `isAjax()` guard is the correct check. The `isModalOrOffcanvas()` private method was removed entirely.
