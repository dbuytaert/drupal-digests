## Summary

Drupal core has adopted `vincentlanglet/twig-cs-fixer` as a `require-dev` dependency to enforce Twig coding standards, closing the last gap in core's automated linting suite alongside PHP_CodeSniffer, ESLint, and Stylelint. As part of the change, approximately 176 existing core Twig files were reformatted to comply with the new rules. Developers contributing Twig templates to core (or wanting to follow core conventions in contrib or custom themes) now have an official tool and a reference configuration to work from. Site owners are not affected; the reformatted templates produce identical rendered output.

## Impact

Affects module maintainers.

- **New API:** New Composer scripts `composer twigcs` and `composer twigcs-fix` for checking and auto-fixing Twig coding standard violations.

## Technical details

The new `core/.twig-cs-fixer.php` configuration file registers Drupal's `TwigTransTokenParser` so the linter understands the `{% trans %}` tag, excludes `tests/` directories from linting, and applies the `TwigCsFixer` standard (which implements the official Twig coding standards at twig.symfony.com). Two rules are overridden for Drupal: `IndentRule(spaceRatio: 2)` for 2-space indentation and `CompactHashRule(TRUE)` to require no spaces around inline hash literals (e.g. `{key: value}` rather than `{ 'key': value }`). `allowNonFixableRules()` is called so the config works in both lint and fix modes.

Two Composer scripts are added: `composer twigcs` runs a lint check and `composer twigcs-fix` auto-fixes violations. A new `🧹 Twig Coding standards (twig-cs-fixer)` job is added to the GitLab CI Lint stage; on merge-request pipelines it only triggers when `.twig` files, `core/.twig-cs-fixer.php`, or `composer.json`/`composer.lock` change. Cache artifacts use the file `core/.twigcsfixercache`, which is added to `.gitignore` and wired into the existing CI cache download/upload pattern alongside the PHP and JS caches. The `commit-code-check.sh` pre-commit script is also updated to run the linter against any staged `.twig` files, and to re-lint all files when the config itself changes.

The bulk reformatting of the ~176 existing core files normalized four categories of style: `{% include 'tmpl' %}` tags became `{{ include('tmpl') }}` function calls; `{% include 'tmpl' with {...} only %}` became `{{ include('tmpl', {...}, with_context: false) }}`; double-quoted string literals became single-quoted; and hash array keys were converted from quoted form (`{'key': value}`) to unquoted form (`{key: value}`) where the key is a valid identifier. Trailing commas were also added to multi-line array and argument lists.

## Upgrade

No changes are required for existing sites. Developers contributing Twig templates to Drupal core can check their files with:

```bash
composer twigcs
```

And auto-fix violations with:

```bash
composer twigcs-fix
```

The configuration lives in `core/.twig-cs-fixer.php`. Custom modules or themes that want to adopt the same standards can reference that file as a starting point, though the `TwigTransTokenParser` registration it contains is only needed when linting templates that use Drupal's `{% trans %}` tag.
