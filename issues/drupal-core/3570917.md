## Summary

The `editor_image_upload_settings_form()` function has been deprecated and replaced with a new service-based approach. This change is part of a broader effort to eliminate procedural code from core `.module` files. Text editor plugins that provide image upload configuration can now use the `EditorImageUploadSettings` service to generate the image upload settings form. The underlying functionality remains the same, but the code is now organized using modern object-oriented patterns with proper dependency injection.

## Impact

Affects module maintainers.

- **Deprecation:** `editor_image_upload_settings_form()` and `editor.admin.inc` deprecated in favor of `EditorImageUploadSettings` service

## Technical details

The procedural function `editor_image_upload_settings_form()` has been deprecated and its logic moved to the new `EditorImageUploadSettings` service. The service provides a `getForm()` method that returns the same form render array previously generated by the procedural function.

The service accepts `ConfigFactoryInterface` and `StreamWrapperManagerInterface` dependencies via constructor injection. It uses `StringTranslationTrait` for translatable strings and follows modern Drupal coding standards with camelCase variable naming.

Core editor plugins that previously called the deprecated function (`Image` and `UnicornEditor`) now inject the `EditorImageUploadSettings` service and call its `getForm()` method. These plugins implement `ContainerFactoryPluginInterface` to support dependency injection and include backward compatibility layers in their constructors for sites that may extend these plugins without passing the new dependency.

The deprecated `editor.admin.inc` file now triggers a deprecation error when loaded and simply wraps the new service. All `@see` references throughout core have been updated to point to the new service method.

## Upgrade

If you maintain a text editor plugin that uses `editor_image_upload_settings_form()`, update it to use the `EditorImageUploadSettings` service instead.

Before:
```php
public function buildConfigurationForm(array $form, FormStateInterface $form_state) {
  $form_state->loadInclude('editor', 'admin.inc');
  $form['image_upload'] = editor_image_upload_settings_form($form_state->get('editor'));
  return $form;
}
```

After:
```php
use Drupal\Core\Plugin\ContainerFactoryPluginInterface;
use Drupal\editor\EditorImageUploadSettings;
use Symfony\Component\DependencyInjection\Attribute\Autowire;

class MyEditor extends EditorBase implements ContainerFactoryPluginInterface {

  public function __construct(
    array $configuration,
    $plugin_id,
    $plugin_definition,
    #[Autowire(service: EditorImageUploadSettings::class)]
    protected EditorImageUploadSettings $editorImageUploadSettings,
  ) {
    parent::__construct($configuration, $plugin_id, $plugin_definition);
  }

  public function buildConfigurationForm(array $form, FormStateInterface $form_state) {
    $form['image_upload'] = $this->editorImageUploadSettings->getForm($form_state->get('editor'));
    return $form;
  }
}
```
