## Summary

Drupal's typed data list API now has a `last()` method to complement the existing `first()` method. Developers working with field item lists and other typed data lists can now retrieve the final item in a list with a single direct call instead of manually computing the index.

## Impact

Affects module maintainers.

- **API change:** `ListInterface` gains a new required method `last(): ?TypedDataInterface`; any custom class implementing this interface must add the method.
- **New API:** New `last(): ?TypedDataInterface` method on `ListInterface` and `ItemList` returns the final item in a typed data list, or `null` for an empty list.

## Technical details

The typed data system represents multi-valued fields and other ordered collections through `ListInterface` and its default implementation `ItemList`. `ListInterface` already had a `first()` method but had no symmetric `last()` counterpart, requiring callers to write `$list->get($list->count() - 1)` themselves.

The fix adds `last(): ?TypedDataInterface` to `ListInterface` (in `core/lib/Drupal/Core/TypedData/ListInterface.php`) and implements it in `ItemList` (in `core/lib/Drupal/Core/TypedData/Plugin/DataType/ItemList.php`) as exactly that one-liner: `return $this->get($this->count() - 1);`. The implementation delegates to the existing `get()` method, which already returns `null` when the index is out of range, so an empty list correctly returns `null`.

A kernel test `testTypedDataListsLast()` was added to `core/tests/Drupal/KernelTests/Core/TypedData/TypedDataTest.php`. It covers a two-item list (verifying the second item is returned), appending a third item (verifying the new last item is returned), and an empty list (verifying `null` is returned).

## Upgrade

If you implement `ListInterface` in custom code, you must add a `last(): ?TypedDataInterface` method. The simplest implementation matches core:

```php
public function last(): ?\Drupal\Core\TypedData\TypedDataInterface {
  return $this->get($this->count() - 1);
}
```
