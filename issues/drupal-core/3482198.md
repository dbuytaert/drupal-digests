## Summary

The Ban module, which lets administrators block visits from specific IP addresses, has been deprecated in Drupal 11.x and will be removed in a future major release. A contrib replacement is available at drupal.org/project/ban, with a stable 1.0.0 release produced via a subtree split of the core module. Sites currently using Ban should add the contributed version as a Composer dependency before upgrading to a version of Drupal core that removes the module entirely.

## Impact

Affects module maintainers.

- **Deprecation:** The `ban` core module is deprecated in 11.x; install `drupal/ban` from contrib as a replacement.

## Technical details

This meta issue (Plan [#3482198](https://www.drupal.org/node/3482198)) coordinated all tasks required by Drupal's core module removal policy, following the approval in [#1570102](https://www.drupal.org/node/1570102). The concrete work was split across several child issues:

- **[#3488838](https://www.drupal.org/node/3488838)**: Replaced Ban with a test module in profile-level tests (`testing_install_profile_all_dependencies`), removing the profile's dependency on Ban.
- **[#3488834](https://www.drupal.org/node/3488834)**: Removed Ban-related migration tests that lived outside the module itself.
- **[#3488836](https://www.drupal.org/node/3488836)**: Removed Ban help topics from core.
- **[#3499865](https://www.drupal.org/node/3499865)** (the deprecation commit `2d16d314`): Added `lifecycle: deprecated` and `lifecycle_link: https://www.drupal.org/node/3223395#s-ban` to `core/modules/ban/ban.info.yml`, and added `@group legacy` to all six test classes in the module (`GenericTest`, `BanIpTest`, `MigrateBlockedIpsTest`, `BlockedIpsTest`, `BanAdminTest`, `BanMiddlewareTest`).

Creating the contrib project required resolving a drupal.org infrastructure hurdle: the short name `ban` was reserved by the Drupal Association for core module names. An infrastructure queue issue ([#3537047](https://www.drupal.org/node/3537047)) unblocked the name, after which mstrelan published Ban 1.0.0 from the subtree split. A follow-up ([#3539672](https://www.drupal.org/node/3539672) in the `project_composer` queue) was opened to correct the duplicated namespace (`drupal/ban-ban`) that results from the reserved-name workaround.

One remaining task noted in the issue body is removing references to Ban from core database dumps, which will be handled in a single combined issue covering all extensions deprecated in this cycle. The `ban` word in comments inside `\Drupal\Core\Config\ConfigImporter::createExtensionChangelist` was explicitly identified as unrelated to the Ban module (it describes sort behavior) and intentionally left unchanged.

## Upgrade

Sites using the core Ban module should require the contributed version before the core module is removed:

```bash
composer require drupal/ban
```

After adding the dependency, rebuild caches. Module discovery checks contrib paths before `core/modules`, so the contributed version will take precedence automatically without deleting the core copy. Run `drush updatedb` afterward in case the contrib release includes any schema updates.
