## Summary

The Locale module's translation fetching functions have been modernized by moving their logic into a new service class. This continues the project's ongoing effort to replace procedural code with object-oriented patterns. The old functions still work but will be removed in Drupal 13, giving site owners and module developers time to update. This change makes the code easier to test, maintain, and extend for future improvements to translation management.

## Impact

Affects module maintainers.

- **Deprecation:** Three procedural functions in `locale.fetch.inc` are deprecated: `locale_translation_batch_update_build()`, `locale_translation_batch_fetch_build()`, and `_locale_translation_fetch_operations()`, replaced by the new `LocaleFetch` service.

## Technical details

Three procedural functions in `locale.fetch.inc` have been deprecated and their logic moved to a new `LocaleFetch` service class. The service provides two public methods: `batchUpdateBuild()` (which checks status, downloads, and imports translations) and `batchFetchBuild()` (which only downloads and imports). A protected `fetchOperations()` method handles the common logic of building batch operations.

The deprecated procedural functions now trigger deprecation notices and delegate to the service via `\Drupal::service(LocaleFetch::class)`. The service is registered in `locale.services.yml` with autowiring enabled, injecting `LocaleProjectStorageInterface` and `ModuleExtensionList`.

All internal callers throughout core (in `install.core.inc`, `locale.module`, `locale.translation.inc`, `LocaleHooks.php`, and `TranslationStatusForm.php`) have been updated to use the new service. The `TranslationStatusForm` now injects `LocaleFetch` as a constructor dependency. The file `locale.fetch.inc` remains in place for backward compatibility and will be removed in Drupal 13 along with the deprecated functions.

## Upgrade

If your code calls `locale_translation_batch_update_build()` or `locale_translation_batch_fetch_build()`, replace them with the `LocaleFetch` service:

```php
// Old:
$batch = locale_translation_batch_update_build($projects, $langcodes, $options);

// New:
$batch = \Drupal::service(LocaleFetch::class)->batchUpdateBuild($projects, $langcodes, $options);
```

For `locale_translation_batch_fetch_build()`:

```php
// Old:
$batch = locale_translation_batch_fetch_build($projects, $langcodes, $options);

// New:
$batch = \Drupal::service(LocaleFetch::class)->batchFetchBuild($projects, $langcodes, $options);
```

The internal helper `_locale_translation_fetch_operations()` is deprecated with no replacement; use the service methods instead. See [#3572345](https://www.drupal.org/node/3572345) for the change record.
