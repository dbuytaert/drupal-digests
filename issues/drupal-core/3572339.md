## Summary

The Locale module now provides translation fetching functionality through a proper service instead of procedural functions. This change modernizes the codebase by moving batch building operations for downloading and importing translations into the new `LocaleFetch` service. Sites using the standard translation update workflows will see no change in behavior. This cleanup is part of an ongoing effort to modernize the Locale module by converting procedural code to object-oriented services.

## Impact

Affects module maintainers.

- **Deprecation:** Three procedural functions in `locale.fetch.inc` are deprecated: `locale_translation_batch_update_build()`, `locale_translation_batch_fetch_build()`, and `_locale_translation_fetch_operations()`; use the new `LocaleFetch` service methods instead.

## Technical details

The change introduces a new `LocaleFetch` service that encapsulates the logic previously in three procedural functions from `locale.fetch.inc`. The service provides two public methods: `batchUpdateBuild()` checks translation status then downloads and imports updates, while `batchFetchBuild()` downloads and imports without checking status first. Both methods return batch definition arrays for the Batch API. A protected `fetchOperations()` method builds the operation arrays.

The deprecated procedural functions now trigger deprecation errors and delegate to the service. The service uses dependency injection to access `LocaleProjectStorageInterface` and `ModuleExtensionList`. All internal Drupal code that called the old functions has been updated to use the service instead, including installation routines, form handlers, cron implementations, and hook implementations.

The `_locale_translation_fetch_operations()` function was marked internal (leading underscore) and has no public replacement since it was an implementation detail. The file `locale.fetch.inc` remains for backward compatibility in Drupal 11.x but will be removed in Drupal 13.0.

## Upgrade

If your custom code calls the deprecated functions, update to use the `LocaleFetch` service:

```php
// Before:
$batch = locale_translation_batch_update_build($projects, $langcodes, $options);

// After:
$batch = \Drupal::service(LocaleFetch::class)->batchUpdateBuild($projects, $langcodes, $options);
```

```php
// Before:
$batch = locale_translation_batch_fetch_build($projects, $langcodes, $options);

// After:
$batch = \Drupal::service(LocaleFetch::class)->batchFetchBuild($projects, $langcodes, $options);
```

If you called `_locale_translation_fetch_operations()`, refactor to call one of the two public methods instead. See [#3572345](https://www.drupal.org/node/3572345) for details.
