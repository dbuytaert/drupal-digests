## Summary

The Locale module is modernizing its codebase by converting procedural functions to object-oriented services. Three functions in `locale.fetch.inc` that build batch operations for downloading and importing translations have been deprecated and moved to a new `LocaleFetch` service. This change improves code organization and testability while maintaining backward compatibility. The deprecated functions continue to work in Drupal 11.x by calling the service methods, but will be removed in Drupal 13.0. Custom code using these functions should update to use the service instead.

## Impact

Affects module maintainers.

- **Deprecation:** Functions `locale_translation_batch_update_build()`, `locale_translation_batch_fetch_build()`, and `_locale_translation_fetch_operations()` deprecated in favor of `LocaleFetch` service methods

## Technical details

This modernization creates a new `LocaleFetch` service that encapsulates translation fetching logic previously in procedural functions. The service provides two public methods: `batchUpdateBuild()` checks status then downloads and imports translations, while `batchFetchBuild()` only downloads and imports. A protected `fetchOperations()` method constructs the batch operations array.

The deprecated functions `locale_translation_batch_update_build()` and `locale_translation_batch_fetch_build()` now trigger deprecation warnings and delegate to the service. The private `_locale_translation_fetch_operations()` function is also deprecated with no replacement since it becomes the protected service method.

Core uses of these functions throughout the installer, module system, language forms, cron hooks, and translation status form have been updated to call the service. The `locale.fetch.inc` file remains for backward compatibility in 11.x but will be removed in 13.0 when the deprecated functions are deleted.

The service uses constructor property promotion and autowiring, depending on `LocaleProjectStorageInterface` and `ModuleExtensionList`. It employs `StringTranslationTrait` for translatable strings.

## Upgrade

Replace calls to deprecated functions with the `LocaleFetch` service:

```php
// Before:
$batch = locale_translation_batch_update_build($projects, $langcodes, $options);

// After:
$batch = \Drupal::service(LocaleFetch::class)->batchUpdateBuild($projects, $langcodes, $options);
```

```php
// Before:
$batch = locale_translation_batch_fetch_build($projects, $langcodes, $options);

// After:
$batch = \Drupal::service(LocaleFetch::class)->batchFetchBuild($projects, $langcodes, $options);
```

For `_locale_translation_fetch_operations()`, there is no public replacement. If you need this functionality, you must call one of the batch build methods instead.

See the change record at [#3572345](https://www.drupal.org/node/3572345) for complete details.
