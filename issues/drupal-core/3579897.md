## Summary

This is a test-only cleanup with no effect on production behaviour. Tests in the Cache library were updated to be compatible with PHPUnit 12.5.0+, which tightens the distinction between mock objects (which must configure expectations) and stubs (which do not). The change has no impact on end users or site owners.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces notices and deprecations that require test code to distinguish between mocks and stubs more precisely. Three rules triggered the changes: (1) mock objects with no configured expectations now emit a notice and should be replaced with `createStub()` calls, (2) `$this->any()` is deprecated because configuring no expectation is equivalent, and (3) `with()` on a stub is deprecated because argument constraints are pointless without expectations. The fix converts expectation-less mock objects in the cache system's unit tests (`Tests/Core/Cache`) from `createMock()` to `createStub()`, removes `$this->any()` usages, and removes `with()` calls from stubs. Reviewer smustgrave confirmed all 213 tests in `Tests/Core/Cache` passed after the change with no remaining `any()` instances.
