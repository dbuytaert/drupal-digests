## Summary

This change finalises the cacheability API for entity list builders that was prepared in [#2473873](https://www.drupal.org/node/2473873). The `$cacheability` parameter in `EntityListBuilderInterface::getOperations` and `EntityListBuilder::getDefaultOperations` was previously present only as a commented-out placeholder to avoid PHP deprecation notices. It is now a real, enforced parameter across all core list builder implementations. Site owners are not directly affected, but developers who have written custom entity list builders will need to update their method signatures.

## Impact

Affects module maintainers.

- **API change:** `EntityListBuilder::getDefaultOperations()` now requires a non-nullable `CacheableMetadata $cacheability` argument; `EntityListBuilderInterface::getOperations()` now explicitly declares `?CacheableMetadata $cacheability = NULL`.

## Technical details

Background: [#2473873](https://www.drupal.org/node/2473873) added cacheability support to entity operations so that dropbuttons can be cached correctly. To avoid a PHP "missing argument" deprecation at the time, the `$cacheability` parameter was commented out in signatures and implementations used `func_get_args()` to silently consume it when passed.

This issue completes the transition for the `main` branch (Drupal 12+). The approach taken distinguishes between the two method visibility levels:

- `getDefaultOperations(EntityInterface $entity, CacheableMetadata $cacheability)` (protected): the parameter is now required and non-nullable. Subclasses must accept it. All core overrides in `ConfigEntityListBuilder`, `BlockListBuilder`, `BlockContentTypeListBuilder`, `CommentTypeListBuilder`, `FieldConfigListBuilder`, `FilterFormatListBuilder`, `ImageStyleListBuilder`, `MenuLinkListBuilder`, `MenuListBuilder`, `NodeTypeListBuilder`, `ResponsiveImageStyleListBuilder`, `SearchPageListBuilder`, `ShortcutSetListBuilder`, `VocabularyListBuilder`, `RoleListBuilder`, `ViewListBuilder`, and `WorkspaceListBuilder` were updated.

- `getOperations(EntityInterface $entity, ?CacheableMetadata $cacheability = NULL)` (public interface): the parameter is nullable with a default of `NULL`. This preserves backward compatibility with any code that calls `getOperations()` without the second argument. The base `EntityListBuilder::getOperations()` immediately resolves a `NULL` value with `$cacheability ??= new CacheableMetadata()` before delegating to `getDefaultOperations()`. The same nullable-optional signature was applied to all public overrides in `BlockContentListBuilder`, `ConfigTranslationEntityListBuilder`, `EntityDisplayModeListBuilder`, and `UserListBuilder`.

The `func_get_args()` workaround and the commented-out parameter syntax were removed from every affected method. The phpcs suppression comments around the `@todo` blocks were also cleaned up. Finally, the two suppression regexes for the PHP deprecation warnings ("will require a new argument") were removed from `core/.deprecation-ignore.txt`, since the deprecation no longer fires.

## Upgrade

If you have a custom entity list builder that overrides `getDefaultOperations()`, add the required `CacheableMetadata` parameter and remove any `func_get_args()` workaround:

```php
// Before
protected function getDefaultOperations(EntityInterface $entity/* , ?CacheableMetadata $cacheability = NULL */) {
  $args = func_get_args();
  $cacheability = $args[1] ?? new CacheableMetadata();
  // ...
}

// After
protected function getDefaultOperations(EntityInterface $entity, CacheableMetadata $cacheability) {
  // ...
}
```

If you override the public `getOperations()`, update its signature to the nullable form:

```php
// Before
public function getOperations(EntityInterface $entity/* , ?CacheableMetadata $cacheability = NULL */) {
  $args = func_get_args();
  $cacheability = $args[1] ?? new CacheableMetadata();
  // ...
}

// After
public function getOperations(EntityInterface $entity, ?CacheableMetadata $cacheability = NULL) {
  $cacheability ??= new CacheableMetadata();
  // ...
}
```
