## Summary

Drupal's image and responsive image formatters now use the standard `#attributes` property instead of the special-case `#item_attributes` property. This change aligns these formatters with the rest of Drupal's render array system, where `#attributes` is the expected way to pass HTML attributes to elements. The old property still works but triggers a deprecation warning and will be removed in Drupal 12.

## Impact

- **Deprecation:** The `#item_attributes` property in `image_formatter` and `responsive_image_formatter` theme hooks is deprecated in favor of `#attributes`.

## Technical details

Most render arrays in Drupal use `#attributes` to pass HTML attributes to theme functions, but the `image_formatter` and `responsive_image_formatter` theme hooks used `#item_attributes` instead. This inconsistency created a special case that complicated the style utility API being developed in #3517033.

The change adds `#attributes` as a new variable to the `image_formatter` theme hook definition in `ImageThemeHooks::theme()`. The `preprocessImageFormatter()` method now checks for `#attributes` first and falls back to `#item_attributes` if not present. When `#item_attributes` is used, a deprecation warning is triggered pointing to the change record.

Core's image and responsive image field formatters (`ImageFormatter`, `ResponsiveImageFormatter`, and `MediaThumbnailFormatter`) now pass `#attributes` instead of `#item_attributes` when building render arrays. The responsive image formatter already had partial support for `#attributes` from a previous fix in #3522087.

All documentation in docblocks has been updated to reference `#attributes` instead of `#item_attributes`. Test coverage was added to verify the deprecation warning fires correctly when the old property is used.

## Upgrade

If you have custom code that builds `image_formatter` or `responsive_image_formatter` render arrays, change `#item_attributes` to `#attributes`:

```php
// Before:
$element = [
  '#theme' => 'image_formatter',
  '#item' => $item,
  '#item_attributes' => ['class' => ['my-class']],
];

// After:
$element = [
  '#theme' => 'image_formatter',
  '#item' => $item,
  '#attributes' => ['class' => ['my-class']],
];
```

The same applies to `responsive_image_formatter`. Custom field formatters extending `ImageFormatter` or `ResponsiveImageFormatter` should also be updated. See the change record at [#3554585](https://www.drupal.org/node/3554585) for full details.
