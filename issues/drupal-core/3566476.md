## Summary

This is a purely internal test-code change with no user-facing effect. Unit tests in the Link module were updated to comply with PHPUnit 12.5+ requirements, which distinguish between mock objects (objects that verify call expectations) and stubs (objects that simply return canned values). No behavior of the Link module itself changed.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5+ introduced two related changes that affect how test doubles are written. First, a mock object that configures no expectations emits a deprecation notice and will eventually throw; such objects must be converted to stubs using `$this->createStub()` instead of `$this->createMock()`. Second, calling `$this->any()` is deprecated because it is equivalent to having no expectation, and `with()` on a stub is deprecated because stubs do not verify call arguments.

Six test files in `core/modules/link/tests/src/Unit/` were updated:

- `LinkFormatterTest.php`: `FieldDefinitionInterface` and `PathValidatorInterface` doubles converted from `createMock()` to `createStub()`. An `EntityInterface` mock and its associated `getParent`/`__get` expectations (all using `$this->any()`) were removed entirely as they were not exercising any meaningful behavior. The inline exception objects are now constructed directly in `willThrowException()`.

- `LinkAccessConstraintValidatorTest.php`: The `LinkItemInterface` double converted to a stub; `$user` and `$context` doubles converted to stubs in the `testUnexpectedValue` and `testEmptyField` methods. The surviving `$user->expects($this->any())` on `hasPermission` was tightened to `$this->once()`. The `$constraintViolationBuilder->method('atPath')` stub call gained `$this->never()` or `$this->once()` expectations, making it a proper mock.

- `LinkExternalProtocolsConstraintValidatorTest.php`: Multiple `LinkItemInterface` doubles converted to stubs using `$this->createStub(LinkItemInterface::class)` and the `with('uri')` argument removed from the stub's `atPath` call. `ExecutionContextInterface` doubles converted to stubs in guard-clause tests.

- `LinkNotExistingInternalConstraintValidatorTest.php`: `LinkItemInterface` doubles converted to stubs. `UrlGeneratorInterface` expectations tightened from `$this->any()` to `$this->once()`. The private `validate()` helper's parameter type changed from `LinkItemInterface&MockObject` to `LinkItemInterface&Stub`. The `PHPUnit\Framework\MockObject\Stub` import was added.

- `LinkTitleRequiredConstraintValidatorTest.php`: `ExecutionContextInterface` double in `testUnexpectedValue` converted to a stub. `$constraintViolationBuilder->method('atPath')` converted to a proper mock expectation with `$this->once()`. The `doValidate()` helper's type hint changed from `ExecutionContextInterface&MockObject` to `ExecutionContextInterface`.

- `LinkTypeConstraintValidatorTest.php`: Two test methods (`testInternal`, `testBadExternal`) had their `UrlGeneratorInterface` mock and all its setup removed because the URL generator was never actually invoked by those test paths. `getMockFieldDefinition()` was converted to use `createStub()` and its return type updated to `FieldDefinitionInterface&Stub`. Several surviving `$this->any()` expectations were replaced with `$this->once()`. The `$constraintViolationBuilder` mocks in `testBadExternal` and `testBadUrl` had their `atPath` calls given explicit `$this->once()` expectations, and `testBadExternal` received a missing `willReturn($constraintViolationBuilder)` on `buildViolation` that was previously silently unchecked.

This issue is part of the broader PHPUnit 12.5 compatibility effort tracked in [#3561671](https://www.drupal.org/node/3561671).
