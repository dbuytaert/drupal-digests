## Summary

The last two remaining functions in `field.module` were moved to appropriate classes and deprecated. `_field_create_entity_from_ids()` is now an internal method on `ContentEntityStorageBase`, and `field_form_field_config_edit_form_entity_builder()` is now a method on `FieldConfigEditForm`. Site owners are not affected unless custom code directly calls these functions, which is unlikely given their specialized nature.

## Impact

Affects module maintainers.

- **API change:** `FieldConfigEditForm::__construct()` accepts optional `$field_type_plugin_manager` parameter, required in drupal:12.0.0
- **Deprecation:** `_field_create_entity_from_ids()` and `field_form_field_config_edit_form_entity_builder()` deprecated with no replacement

## Technical details

`_field_create_entity_from_ids()` was a procedural helper that assembled a partial entity object from IDs (entity_id, revision_id, bundle). It is now `ContentEntityStorageBase::createEntityFromIds()`, marked `@internal`, and changed from accepting an object parameter to an array. The method is used in `SqlContentEntityStorage::readFieldItemsToPurge()` and in the field UI edit forms to create arbitrary entity objects for rendering field widgets.

`field_form_field_config_edit_form_entity_builder()` was a form entity builder callback that updated handler settings when the target_type changed on entity reference fields. It is now `FieldConfigEditForm::entityFormEntityBuild()`. The form registration changed from appending a function name string to the `#entity_builders` array to using an associative array with a method reference.

To support the moved entity builder logic, `FieldConfigEditForm` gained a new optional constructor parameter for the field type plugin manager. When not provided, a deprecation is triggered and the service is fetched via `\Drupal::service()`. Property promotion was deliberately not used for this parameter to make backward compatibility easier for contributed modules that extend the form.

## Upgrade

If extending `FieldConfigEditForm`, pass the field type plugin manager to the constructor:

```php
// Before
new FieldConfigEditForm(
  $entityTypeBundleInfo, 
  $typedDataManager, 
  $entityDisplayRepository, 
  $tempStore, 
  $elementInfo
);

// After
new FieldConfigEditForm(
  $entityTypeBundleInfo, 
  $typedDataManager, 
  $entityDisplayRepository, 
  $tempStore, 
  $elementInfo,
  $fieldTypePluginManager
);
```
