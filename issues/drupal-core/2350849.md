## Summary

The procedural function `image_filter_keyword()` in `image.module` has been deprecated in Drupal 11.1.0 and will be removed in Drupal 12.0.0. Its logic now lives in the properly typed static method `\Drupal\Component\Utility\Image::getKeywordOffset()`. Sites or contributed modules that call `image_filter_keyword()` directly must update their code before upgrading to Drupal 12.

## Impact

Affects module maintainers.

- **Deprecation:** `image_filter_keyword()` deprecated in drupal:11.1.0 and removed in drupal:12.0.0; use `\Drupal\Component\Utility\Image::getKeywordOffset()` instead.
- **New API:** New `\Drupal\Component\Utility\Image::getKeywordOffset(string $anchor, int $current_size, int $new_size): int` static method added to `core/lib/Drupal/Component/Utility/Image.php`.

## Technical details

`image_filter_keyword()` was a procedural function in `core/modules/image/image.module` that translated positioning keywords ('top', 'left', 'bottom', 'right', 'center') into pixel offsets for image crop and scale-and-crop operations. It had two problems: its declared return type was `int|string` (silently passing through unrecognized anchor values unchanged), and the 'center' calculation returned a float due to division without rounding. The float return caused PHP 8.1 deprecations when those values were passed to GD functions expecting integers.

The fix adds `Image::getKeywordOffset(string $anchor, int $current_size, int $new_size): int` as a static method to the existing `\Drupal\Component\Utility\Image` class in `core/lib/Drupal/Component/Utility/Image.php`. It uses a `match` expression with strict types: 'top' and 'left' return `0`, 'bottom' and 'right' return `$current_size - $new_size`, and 'center' returns `(int) round($current_size / 2 - $new_size / 2)` to guarantee an integer. Unrecognized anchors now throw `\InvalidArgumentException` rather than returning the original value — a deliberate behavior change from the old function.

The two built-in image effects that called the old function were updated to use the new method directly: `CropImageEffect::applyEffect()` in `core/modules/image/src/Plugin/ImageEffect/CropImageEffect.php` and `ScaleAndCropImageEffect::applyEffect()` in `core/modules/image/src/Plugin/ImageEffect/ScaleAndCropImageEffect.php`. Both now import `Drupal\Component\Utility\Image` and apply explicit `(int)` casts and `round()` to scaled dimensions before passing them in. The old `image_filter_keyword()` is kept as a shim with a `@trigger_error()` deprecation notice pointing to change record node/3268441.

Test coverage was added in three places: `core/tests/Drupal/Tests/Component/Utility/ImageTest.php` gained `testGetKeywordOffset()` (data-provider driven, covering all five keywords including both odd- and even-remainder 'center' cases) and `testInvalidGetKeywordOffset()` (asserts `\InvalidArgumentException`); a new `core/modules/image/tests/src/Unit/ImageDeprecationTest.php` class (`ImageDeprecationTest`) verifies the `@trigger_error` shim fires; and `core/modules/image/tests/src/Kernel/ImageEffectsTest.php` was corrected to use valid keyword anchors instead of the numeric pass-through values that the old function accepted.

## Upgrade

```php
// Before (deprecated in drupal:11.1.0, removed in drupal:12.0.0)
$x = image_filter_keyword($x, $image->getWidth(), $width);
$y = image_filter_keyword($y, $image->getHeight(), $height);

// After
use Drupal\Component\Utility\Image;
$x = Image::getKeywordOffset($x, $image->getWidth(), $width);
$y = Image::getKeywordOffset($y, $image->getHeight(), $height);
```

Note: `Image::getKeywordOffset()` throws `\InvalidArgumentException` for any anchor value other than 'top', 'left', 'bottom', 'right', or 'center'. The old function silently returned the unrecognized value as-is. If your code passed arbitrary strings, add validation before calling the new method.
