## Summary

Fresh Drupal installations using the Standard or Umami demo profile no longer have the Search module enabled. The `search content` permission is no longer granted to anonymous or authenticated users on those installs. The `core/recipes/content_search` recipe has been deleted from core entirely. Sites that need search functionality must now enable the module manually or apply the contributed recipe at drupal.org/project/content_search. This is part of the ongoing deprecation of the core Search module.

## Impact

Affects module maintainers and site owners.

- **Permission change:** The `search content` permission is no longer granted to anonymous or authenticated users in fresh Standard and Umami profile installations.
- **API change:** The `core/recipes/content_search` recipe is removed from Drupal core; its equivalent is now the contributed recipe at drupal.org/project/content_search.

## Technical details

Both `standard.info.yml` and `demo_umami.info.yml` drop `search` from their `install` lists. The `search` module dependency and `search content` permission are removed from `user.role.anonymous.yml` and `user.role.authenticated.yml` in both profiles. The Umami-specific `block.block.umami_search.yml` (a `search_form_block` block in the `pre_header` region) is deleted.

On the recipe side, `core/recipes/content_search/recipe.yml` and its accompanying `GenericTest.php` are deleted. The `core/recipes/standard/recipe.yml` drops `content_search` from its recipe dependency list and removes the following search-tied config imports: `search.page.help_search`, `block.block.claro_help_search`, `block.block.olivero_search_form_narrow`, `block.block.olivero_search_form_wide`, and `search.page.user_search`.

`core.entity_view_mode.node.search_index` and `core.entity_view_mode.node.search_result` are intentionally left in core for now, as catch noted they are also used by Search API-based setups (including Drupal CMS). `StandardRecipeTest` is updated to expect these two view modes in the `create` changelist after applying the standard recipe. Performance test baselines in `StandardPerformanceTest` and the Umami `OpenTelemetry*PerformanceTest` classes reflect reduced cache-get and cache-tag-lookup counts, including the removal of the `config:search.settings` and `CACHE_MISS_IF_UNCACHEABLE_HTTP_METHOD:form` grouped cache tag lookups.

## Upgrade

If your site provisioning or CI references the `core/recipes/content_search` recipe, replace it with the contributed equivalent:

```yaml
# Before (core recipe, now removed):
recipes:
  - content_search

# After: use the contributed content_search recipe
# See https://www.drupal.org/project/content_search
```

If you rely on a fresh Standard or Umami install having Search enabled, add it explicitly to your install profile or apply the contributed recipe after installation:

```bash
php core/scripts/drupal recipe path/to/content_search
```
