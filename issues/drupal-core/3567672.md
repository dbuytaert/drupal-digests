## Summary

New Drupal installations using the Standard or Umami demo profile will no longer have the Search module enabled by default. The `content_search` core recipe has been removed entirely. The `search content` permission is no longer granted to anonymous or authenticated users on fresh installs of these profiles. Sites that need Search functionality should enable the module manually; the recipe formerly shipped as `core/recipes/content_search` is now available as a contributed project at drupal.org/project/content_search. This is part of the broader deprecation of the core Search module.

## Impact

Affects module maintainers and site owners.

- **Permission change:** The `search content` permission is no longer granted to anonymous or authenticated users in fresh Standard and Umami profile installations.
- **API change:** The `core/recipes/content_search` recipe is removed from Drupal core; its equivalent is now the contributed recipe at drupal.org/project/content_search.

## Technical details

The Search module is under active deprecation in Drupal core (referenced in the issue as a "parent issue"). This task removes all Search wiring from the Standard profile, the Umami demo profile, and the core recipe layer.

**Profile changes (Standard and Umami, both commits):**
- `search` is removed from the `install` list in `standard.info.yml` and `demo_umami.info.yml`.
- The `search` module dependency and the `search content` permission are removed from `user.role.anonymous.yml` and `user.role.authenticated.yml` in both profiles.
- `block.block.umami_search.yml` (the Umami search form block configuration, using `plugin: search_form_block`) is deleted.

**Recipe changes:**
- `core/recipes/content_search/recipe.yml` and its accompanying test `core/recipes/content_search/tests/src/Functional/GenericTest.php` are deleted. The recipe had installed the `search` module, imported `core.entity_view_mode.node.search_index`, `core.entity_view_mode.node.search_result`, and `search.page.node_search`, and granted the `search content` permission to anonymous and authenticated roles.
- `core/recipes/standard/recipe.yml`: the `content_search` recipe is removed from the recipe dependency list, and the following search-related config imports are dropped: `search.page.help_search`, `block.block.claro_help_search`, `block.block.olivero_search_form_narrow`, `block.block.olivero_search_form_wide`, and `search.page.user_search`.

**View mode note:** `core.entity_view_mode.node.search_index` and `core.entity_view_mode.node.search_result` are not removed from core in this issue (catch noted they should be handled separately). `StandardRecipeTest` is updated to expect these two configs in the `create` changelist when comparing sync storage to active config after applying the standard recipe.

**Performance test baselines:** `StandardPerformanceTest` and the Umami `OpenTelemetry*PerformanceTest` classes have reduced cache-get counts and cache-tag-lookup counts reflecting Search no longer being installed. The `config:search.settings` and `CACHE_MISS_IF_UNCACHEABLE_HTTP_METHOD:form` grouped cache tag lookups are removed from the expected baseline in `StandardPerformanceTest`, and a `search.page.*` config query is eliminated from the `testLoginBlock` scenario.

**Migrate UI tests (11.x backport only, commit `03f40ce5`):** The D6 and D7 `migrate_drupal_ui` upgrade review-page tests across both `migrate_drupal_ui` and `contact` modules move Search from `getAvailablePaths()` to `getMissingPaths()`, since Standard no longer installs it. `search_page` entity counts (previously 3) and search-related block counts are reduced in the `Upgrade6Test` and `Upgrade7Test` fixtures.

## Upgrade

If your site provisioning or CI references the `core/recipes/content_search` recipe, replace it with the contributed equivalent:

```yaml
# Before (core recipe, now removed)
recipes:
  - content_search

# After: install the contributed content_search recipe
# See https://www.drupal.org/project/content_search
```

If you rely on a fresh Standard or Umami install having Search enabled, add it explicitly to your installation profile or apply the contributed recipe after installation:

```bash
php core/scripts/drupal recipe path/to/content_search
```
