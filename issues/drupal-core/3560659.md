## Summary

When using the `core/drupal.htmx` library with HTMX's boost feature (placing `hx-boost="true"` on the `<body>` element), Drupal behaviors were silently not applied after each boosted page navigation. The fix corrects this so that behaviors are reliably attached after any HTMX swap, including full-body swaps.

## Impact

No upgrade or configuration changes required.

## Technical details

The `core/drupal.htmx` library provides two scripts: `htmx-assets.js` fires a custom `htmx:drupal:load` event after assets are ready, and `htmx-behaviors.js` listens for that event and calls `Drupal.attachBehaviors()`. The bug was in the event-dispatch logic inside `htmx-assets.js`.

After an `htmx:afterSettle` event, the code was unconditionally calling `htmx.trigger(detail.elt.parentNode, 'htmx:drupal:load')`. This pattern is intentional for most swap modes: because HTMX swaps can insert the new element before or after `detail.elt`, firing the event on the parent ensures the freshly inserted nodes are within scope for `attachBehaviors`. However, when `detail.elt` is `document.body`, its parent is `<html>`, and the listener in `htmx-behaviors.js` receives `<html>` as the context element rather than `<body>`, so behaviors are not applied to the swapped content.

The fix in `core/misc/htmx/htmx-assets.js` introduces a `processTarget` variable, defaulting to `detail.elt.parentElement`. A guard is then added: if `detail.elt === document.body` or `processTarget` is `null` (no parent at all), `processTarget` is set to `detail.elt` itself before triggering `htmx:drupal:load`. This mirrors the same exception that HTMX itself applies for `outerHTML` swaps on the body.

Test coverage was added across several files in the `test_htmx` test module: a new `selectBody()` controller method in `HtmxTestAttachmentsController`, a new `test_htmx.attachments.body` route, a new `TestHtmxHooks` class that implements `hook_preprocess_html` to attach `core/drupal.htmx` and set `data-hx-boost="true"` on the body for that route, and a new "Boosted Body" Nightwatch test case in `core/tests/Drupal/Nightwatch/Tests/htmx/htmxTest.js` that exercises the full click-and-swap flow.
