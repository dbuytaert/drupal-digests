# #3554220: Claro's libraries don't enforce the variables.css dependency

## Summary

When contributed modules like `gin_toolbar` used Claro's component libraries (such as dialogs or autocomplete) on sites with a non-Claro frontend theme, those components appeared broken. CSS custom properties like `--jui-dropdown-fg-color` failed to resolve because Claro's `variables.css` file was not loaded. This fix extracts the variables into a separate library and makes all Claro component libraries depend on it, so components work correctly even when used individually outside the full Claro theme.

## Impact

No upgrade or configuration changes required.

## Technical details

Claro's `variables.css` defines CSS custom properties used throughout the theme's components. Previously, this file was bundled into the `claro/global-styling` library. When contrib modules like `gin_toolbar` referenced individual Claro libraries such as `claro/claro.drupal.dialog` or `claro/claro.jquery.ui` without loading the full global styling, CSS rules that used variables like `var(--jui-dropdown-border-color)` would fail to resolve.

The fix creates a new `claro/variables` library containing only `css/base/variables.css`. The file was removed from `claro/global-styling` and replaced with a dependency on `claro/variables`. Then dependencies on `claro/variables` were added to all component libraries that use CSS custom properties: `claro.jquery.ui`, `claro.drupal.dialog`, `drupal.tablesort`, `system.admin`, `icon-link`, `drupal.shortcut`, `status.report`, `views`, `views_ui`, `card`, `file`, `media_library.theme`, `media_library.ui`, `progress`, `filter`, `form-two-columns`, and `image.admin`.

This ensures that any component library can be used standalone and will automatically load the variables it needs. The change is non-breaking because `claro/global-styling` still loads the variables through its new dependency, maintaining the existing behavior for themes that use Claro normally.

## Lessons learned

- When creating reusable component libraries that reference CSS custom properties (variables), explicitly declare the variables file as a dependency. Components that reference undefined variables will silently fail in CSS, breaking the UI when used in isolation by other modules or themes. *(via codebymikey)*
