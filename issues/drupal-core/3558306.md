## Summary

Controllers and plugins can now inject additional services using setter methods marked with the `#[Required]` attribute instead of extending parent constructors. This makes it easier for child classes to add their own dependencies when the parent class already has a long constructor signature. The setter methods are automatically called during object creation, similar to how Symfony handles autowired service definitions. This reduces boilerplate code and makes dependency management more flexible for complex class hierarchies.

## Impact

Affects module maintainers.

- **New API:** Controllers and plugins can now use the `#[Required]` attribute on public setter methods to automatically inject services without extending constructors.

## Technical details

The `AutowiredInstanceTrait::createInstanceAutowired()` method now scans for public methods with the `#[Required]` attribute after instantiating the object. It automatically resolves and injects parameters into these setter methods using the same autowiring logic as the constructor.

The implementation refactored the existing autowiring logic into a new `getAutowireArguments()` helper method that both the constructor and setter injection paths share. This method resolves service parameters by type hint or `#[Autowire]` attribute, handles nullable parameters, and throws `AutowiringFailedException` when resolution fails.

After the instance is created with constructor arguments, reflection iterates through all public methods. For each method with the `#[Required]` attribute, it invokes the method with autowired arguments. This matches Symfony's service definition behavior but applies it to `ContainerInjectionInterface` and `ContainerFactoryPluginInterface` classes.

The exception message was improved to provide better guidance when autowiring fails, suggesting that developers check their type hints or `#[Autowire]` attributes. Test coverage was added via `AutowiredInstanceTraitTest` to verify setter injection works correctly.

## Upgrade

Classes using `AutowireTrait` or `AutowiredInstanceTrait` can now add public setter methods with the `#[Required]` attribute to inject services:

```php
use Symfony\Contracts\Service\Attribute\Required;

class ExamplePlugin extends PluginBase {

  protected MyServiceInterface $myService;

  #[Required]
  public function setMyService(MyServiceInterface $myService): void {
    $this->myService = $myService;
  }
}
```

Multiple setter methods can be used, or one setter can accept multiple parameters. The `#[Autowire]` attribute can be used on parameters if needed. Setter methods are called automatically during instantiation when using the autowiring system.
