## Summary

This is a purely internal test-code cleanup with no effect on production behaviour. A batch of unit tests across the Ajax, Authentication, Block, Breadcrumb, Condition, Controller, Cron, and DependencyInjection test directories were updated to replace expectation-less mock objects with stubs and to remove deprecated PHPUnit constructs, keeping Drupal's test suite compatible with PHPUnit 12.5 and later.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces a notice for any mock object created with `createMock()` that never has expectations configured, because such an object should be a stub (`createStub()`) instead. It also deprecates `$this->any()` (which is equivalent to having no expectation) and deprecates `with()` on stubs (where argument matching is pointless). This issue fixes 10 affected test classes by replacing `createMock()` calls with `createStub()` where no expectations are set, removing `$this->any()` matchers, and removing `with()` calls from stub configurations. During review, smustgrave noted that residual notices in `BlockManagerTest` and `ControllerBaseTest` were not caused by this patch but by helper methods in `UnitTestCase` itself, which are addressed separately in [#3578904](https://www.drupal.org/node/3578904). The fix is restricted to `core/tests/Drupal/Tests/Core/` subdirectories: `Ajax/`, `Authentication/`, `Block/`, `Breadcrumb/`, `Condition/`, `Controller/`, `Cron/`, and `DependencyInjection/`.
