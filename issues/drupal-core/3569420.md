## Summary

Unit tests in the `field` and `field_ui` modules were updated to replace expectation-less mock objects with stubs, and to remove uses of the deprecated `$this->any()` matcher. These changes prepare Drupal's test suite for PHPUnit 12.5 and later, which emits notices for mocks that configure no expectations and deprecates `$this->any()` and `with()` on stubs. The change landed on `main` only; the 11.x commit was reverted due to compatibility issues on that branch.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced notices for mock objects that configure no expectations, deprecating `$this->any()` (which was equivalent to no expectation), and later deprecated calling `with()` on stubs. The fix touches seven test files across the `field` and `field_ui` modules.

The core pattern throughout is replacing `createMock()` with `createStub()` for objects that only need to return values and carry no call-count assertions, and dropping the `->expects($this->any())` call that previously prefixed every `->method()` chain on those objects. Where a mock genuinely needed to enforce a call count, the expectation was tightened to `$this->once()`, `$this->never()`, or `$this->atLeastOnce()` instead of being deleted.

**`FieldConfigEntityUnitTest`** (`core/modules/field/tests/src/Unit/FieldConfigEntityUnitTest.php`): `$entityTypeManager`, `$uuid`, and `$fieldTypePluginManager` converted to stubs. The shared `$entityType` property was removed; a local `createStub(ConfigEntityTypeInterface::class)` is now created inline per test. Precise expectations (`once()`, `never()`) were added where call counts are meaningful, and `with()` constraints removed from stub method chains. `$entityFieldManager` remains a mock because `testCalculateDependencies` adds a `once()` expectation to it.

**`FieldStorageConfigAccessControlHandlerTest`** (`core/modules/field/tests/src/Unit/FieldStorageConfigAccessControlHandlerTest.php`): All objects created in `setUp()` with no call-count expectations (`$anon`, `$member`, `$storageType`, `$entityType`, `$moduleHandler`, `$entity_type_manager`, and the `uuid` service) converted to stubs.

**`FieldStorageConfigEntityUnitTest`** (`core/modules/field/tests/src/Unit/FieldStorageConfigEntityUnitTest.php`): `$entityTypeManager` and `$uuid` converted to stubs. Three `$fieldTypeManager` expectations that used `$this->any()` were tightened to `$this->atLeastOnce()` because the object stays a mock (it carries other expectations).

**`StringFormatterTest`** (`core/modules/field/tests/src/Unit/Plugin/Field/FieldFormatter/StringFormatterTest.php`): A `getMockBuilder(StringItem::class)->disableOriginalConstructor()->onlyMethods([])->getMock()` pattern was replaced by constructing a real `StringItem` with a `createStub(ComplexDataDefinitionInterface::class)` whose `getPropertyDefinitions()` returns an empty array.

**`FieldConfigEditFormTest`** (`core/modules/field_ui/tests/src/Unit/FieldConfigEditFormTest.php`): Five `createMock()` calls in `setUp()` replaced with `createStub()`.

**`FieldUiTest`** (`core/modules/field_ui/tests/src/Unit/FieldUiTest.php`): The `$pathValidator` property (a mock stored on the class) was dropped; the service is now created inline with `createStub(PathValidatorInterface::class)`.

Three PHPStan baseline suppression entries in `core/.phpstan-baseline.php` for disallowed `any()` calls in `FieldConfigEntityUnitTest`, `FieldStorageConfigAccessControlHandlerTest`, and `FieldStorageConfigEntityUnitTest` were removed as the violations no longer exist.

Residual PHPUnit notices observed during review from `UnitTestCase::getStringTranslationStub()` are out of scope here and are addressed separately in [#3578904](https://www.drupal.org/node/3578904).
