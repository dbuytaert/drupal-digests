## Summary

Drupal core's PHPStan configuration now forbids calling `md5()`, `sha1()`, `crc32()`, and `hash()` with the weak algorithm strings `'md5'`, `'sha1'`, `'crc32'`, or `'crc32b'`. Any new use of these functions in core will cause a PHPStan error during CI, pointing developers to use `hash()` with an xxHash algorithm for non-cryptographic purposes, or `sha256()` for cryptographic contexts. The existing core usages are added to the PHPStan baseline (rather than converted immediately) and are tagged for follow-up replacement in [#3609780](https://www.drupal.org/node/3609780). The `uniqid()` function was considered during this issue but excluded, as `bin2hex(random_bytes())` is not a mechanical drop-in and its handling is tracked separately in [#2972100](https://www.drupal.org/node/2972100).

No upgrade or configuration changes required.

## Technical details

The change adds a `disallowedFunctionCalls` block to `phpstan.neon` using the `spaze/phpstan-disallowed-calls` extension (introduced as a dev dependency in [#3573259](https://www.drupal.org/node/3573259)). Four rules are configured: `crc32()` is unconditionally forbidden; `md5()` is unconditionally forbidden; `sha1()` is unconditionally forbidden; and `hash()` is forbidden specifically when its first `algo` parameter is one of `'md5'`, `'sha1'`, `'crc32'`, or `'crc32b'` (using `allowExceptCaseInsensitiveParams` to match only those literal string arguments). Each rule's `errorTip` points to the change record at `https://www.drupal.org/node/3581605`. The recommended replacements are `hash()` with an xxHash algorithm for non-cryptographic hashing, and `sha256()` (or a stronger algorithm via the `Crypt` class) for cryptographic needs. All existing core files that still call these functions are added to `phpstan-ignore-errors.php` as baseline suppressions, covering files including `GenerateTheme.php`, `CheckpointStorage.php`, `DrupalKernel.php`, `PhpassHashedPasswordBase.php`, `RouteProvider.php`, and several SQLite driver and test infrastructure files.

## Upgrade

If you contribute to Drupal core or use core's `phpstan.neon` as a base for your own project's static analysis, any call to `md5()`, `sha1()`, `crc32()`, or `hash()` with a weak algorithm will now produce a PHPStan error. Replace them as follows:

```php
// Non-cryptographic hashing (speed/deduplication use-cases):
// Before:
$hash = md5($value);
$hash = crc32($value);
$hash = hash('md5', $value);

// After (xxHash is fast and low-collision):
$hash = hash('xxh3', $value);   // or 'xxh32', 'xxh64', 'xxh128'

// Cryptographic hashing:
// Before:
$hash = sha1($value);

// After:
$hash = hash('sha256', $value);
// Or use Drupal's Crypt class for common patterns:
use Drupal\Component\Utility\Crypt;
$hash = Crypt::hashBase64($value);
```

Existing usages already in core are suppressed in `phpstan-ignore-errors.php` and will be converted in follow-up issue [#3609780](https://www.drupal.org/node/3609780).
