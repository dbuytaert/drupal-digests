## Technical details

PHPUnit 12.5.0 introduces two breaking changes for test doubles: mock objects that set no call expectations must be replaced with stubs (otherwise a deprecation notice is emitted), and `$this->any()` is deprecated because it is semantically equivalent to no expectation. The Access library tests in `core/tests/Drupal/Tests/Core/Access/` used both patterns extensively.

The commit makes the following mechanical changes across `AccessManagerTest`, `AccessResultTest`, `CsrfAccessCheckTest`, `CsrfTokenGeneratorTest`, `CustomAccessCheckTest`, `DefaultAccessCheckTest`, `RoutePathGenerationTraitTest`, and `RouteProcessorCsrfTest`:

- `$this->createMock()` is replaced with `$this->createStub()` wherever the object's call count is never asserted, and the intersection type annotations on properties are updated from `MockObject` to `Stub` accordingly.
- `->expects($this->any())` is removed from stubs (the `willReturn*` chain is kept) and replaced with `->expects($this->atLeastOnce())` on the mocks in `CsrfTokenGeneratorTest` and `RoutePathGenerationTraitTest` where actual invocation is meaningful.
- Several test classes (`CustomAccessCheckTest`, `AccessManagerTest`) had their `setUp()` method simplified or stripped: objects that needed precise per-test expectations (such as `AccessArgumentsResolverFactoryInterface`, `ParamConverterManagerInterface`, and `AccessManager` itself) are now instantiated inside each test method rather than shared via properties, which makes the expectations local and unambiguous.
- The `RouteProcessorCsrfTest` separates the stub used for the happy-path request stack from the mock created inside `testProcessOutboundNoRequirement` and `testProcessOutboundJsonFormat` where exact call counts are verified.
- PHPStan baseline suppressions for `any()` calls in `AccessManagerTest`, `AccessResultTest`, `CsrfTokenGeneratorTest`, `CustomAccessCheckTest`, and `RouteProcessorCsrfTest` are removed from `core/.phpstan-baseline.php` because the offending calls no longer exist.
