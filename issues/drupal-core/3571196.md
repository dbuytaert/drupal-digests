## Summary

Drupal core now requires a patched version of the Symfony Process component to fix a security vulnerability affecting Windows users. The vulnerability could allow incorrect command argument escaping when using MSYS2 or Git Bash on Windows, potentially leading to unintended file operations. Sites running on Windows with these tools should update immediately. Sites on other platforms are not directly affected but should still update to maintain security best practices.

## Impact

Affects site owners.

- **Security:** Fixes CVE-2026-24739, a medium severity vulnerability where incorrect argument escaping under MSYS2/Git Bash on Windows could lead to destructive file operations.

## Technical details

This change updates the `symfony/process` dependency constraint in Drupal core to require patched versions that fix CVE-2026-24739. For Drupal 11.x and 11.3.x running Symfony 7.4, the constraint changes from `^7.4` to `^7.4.5`. For Drupal 10.6.x running Symfony 6.4, it changes from `^6.4` to `^6.4.33`.

The vulnerability affects the Symfony Process component when running under MSYS2 or Git Bash environments on Windows. These environments provide Unix-like shells on Windows, and improper argument escaping in the Process component could allow command arguments to be interpreted incorrectly, potentially leading to destructive file operations.

The fix was applied to the 10.6.x, 11.3.x, and 11.x branches. The main development branch was intentionally skipped since it is waiting for the full Symfony 8 update tracked in [#3570077](https://www.drupal.org/project/drupal/issues/3570077).

The diff shows updates to `core/composer.json` and `composer/Metapackage/CoreRecommended/composer.json` with the new version constraints, along with the corresponding `composer.lock` updates reflecting the upgraded package versions.

## Upgrade

Run `composer update` to pull in the patched Symfony Process component. No code changes are required in your site or custom modules.

If you maintain a custom composer.json file that pins specific Symfony component versions, ensure your constraints allow the updated versions: `symfony/process:^7.4.5` for Drupal 11.x or `^6.4.33` for Drupal 10.x.
