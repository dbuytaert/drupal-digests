## Summary

The `layout_builder_expose_all_field_blocks` feature-flag module, previously marked deprecated, is now marked obsolete in Drupal 12. Running database updates will automatically uninstall it on any site where it is still installed. The module allowed all fields across all entity bundles to be exposed as Layout Builder field blocks regardless of whether Layout Builder was enabled on those bundles. That behavior is now gone: field blocks are always scoped only to bundles that have a Layout Builder display configured. Sites relying on field blocks from bundles that do not have Layout Builder enabled may lose those blocks after the update runs.

## Impact

Affects site owners.

- **Database update:** New `layout_builder_update_12001()` uninstalls `layout_builder_expose_all_field_blocks` automatically on upgrade.

## Technical details

The module's `lifecycle` key in its `.info.yml` was changed from `deprecated` to `obsolete`. The `LayoutBuilderExposeAllFieldBlocksHooks` class, which contained the sole `hook_help` implementation, was deleted entirely, as was its generic functional test (`GenericTest`).

Integration code in `LayoutBuilderEntityViewDisplay::save()` previously skipped calling `\Drupal::service('plugin.manager.block')->clearCachedDefinitions()` when the module was installed; that conditional was removed and the cache invalidation now runs unconditionally on every save.

In `FieldBlockDeriver::getDerivativeDefinitions()` and `getFieldMap()`, the `$expose_all_fields` branch that bypassed bundle-level filtering when the module was present was removed. Both methods now always filter entity types and bundles to only those with configured Layout Builder displays.

A new update hook, `layout_builder_update_12001()`, calls `module_installer->uninstall(['layout_builder_expose_all_field_blocks'])` if the module is still installed, ensuring it is removed on upgrade. Tests that previously relied on the module to expose user entity field blocks were updated to instead explicitly enable Layout Builder on the user entity's default display in their `setUp()`.

## Upgrade

Sites running `layout_builder_expose_all_field_blocks` will have it uninstalled automatically by the update hook. No manual action is required to trigger uninstallation. However, you should audit any Layout Builder displays that used field blocks from entity bundles that do not themselves have Layout Builder enabled: those field blocks will no longer be available after the module is uninstalled. To preserve access to such fields, enable Layout Builder on the relevant bundle's display before running updates.
