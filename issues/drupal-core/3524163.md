## Summary

SDC (Single Directory Components) slots with empty definitions were giving a false validation error since [#3517317](https://www.drupal.org/node/3517317) landed in Drupal 11.1.7. An empty slot, written as `minimal: {}` in a component's YAML, is valid per the official documentation, but `ComponentValidator` was reporting "Array value found, but an object is required" for it. This bug is now fixed.

## Impact

No upgrade or configuration changes required.

## Technical details

`ComponentValidator::validateDefinition()` calls `Validator::arrayToObjectRecursive()` to convert PHP arrays to objects before JSON schema validation. This helper only converts non-empty associative arrays; a truly empty array stays as a PHP array `[]`. The JSON schema expects slot definitions to be objects, so an empty slot definition (YAML `{}`, which YAML parses to an empty PHP array) would fail with "Array value found, but an object is required".

The fix mirrors existing code that already handled the same edge case for the props `properties` key. A loop was added in `ComponentValidator::validateDefinition()` that iterates over `$definition['slots']` and replaces any empty array with `new \stdClass()` before the schema validator runs. A `minimal: { }` slot was added to the `my-banner` test fixture in `sdc_test` to provide regression coverage; smustgrave confirmed in comment #14 that the test fails without the fix and passes with it.
