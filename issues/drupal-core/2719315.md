## Summary

This is a pure internal cleanup with no user-facing effect. Two explicit calls that manually primed the system module's file path in the extension list during installation have been removed, along with a stale `@todo` comment. A new unit test confirms that pathnames registered via `setPathname()` survive an `ExtensionList::reset()` call, which had been the intended behavior all along.

## Impact

No upgrade or configuration changes required.

## Technical details

During Drupal installation the system module must be discoverable by the extension system before the normal module discovery process has run. Historically, `install_begin_request()` in `core/includes/install.core.inc` and `drupal_install_system()` in `core/includes/install.inc` each called `setPathname('system', 'core/modules/system/system.info.yml')` explicitly on the `extension.list.module` service (`ModuleExtensionList`) to prime its internal `$addedPathNames` cache. These calls became redundant after [#3481778](https://www.drupal.org/node/3481778) moved the priming of the system module entry to `InstallerKernel::getExtensions()`, which now unconditionally ensures `system` is present in the extensions array before container compilation. The alexpott comment (comment 16535910) confirmed that "the calls being removed here are currently pointless" due to that earlier change.

This issue removes both now-dead `setPathname('system', ...)` calls from the installer functions. It also removes the `@todo` comment in `ExtensionList::reset()` in `core/lib/Drupal/Core/Extension/ExtensionList.php` that pointed here; the comment suggested that manually-added pathnames could not yet survive a `reset()` call, but `reset()` already intentionally preserves `$addedPathNames` (clearing only `$this->pathNames` and related caches). The `reset()` docblock already documents this: "We don't reset statically added filenames, as it is a static cache which logically can't change."

A new unit test `testSetPathnameSurvivesReset()` was added to `core/tests/Drupal/Tests/Core/Extension/ExtensionListTest.php` to formally prove that a path registered with `setPathname()` is still returned by `getPathname()` and `getPath()` after `reset()` is called, providing regression coverage for this invariant.
