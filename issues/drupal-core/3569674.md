## Summary

Drupal's PHPStan baseline file previously suppressed 17 `missingType.generics` errors across several core interfaces and classes. This change resolves those suppressions by adding proper generic type parameters directly to PHPDoc return-type annotations. No runtime behavior changes. Developers running PHPStan at level 1 or above on contrib or custom modules that implement or extend the affected public interfaces may now encounter new `missingType.generics` warnings in their own code, because PHPStan will expect implementing classes to declare matching generic types.

## Impact

Affects module maintainers.

- **API change:** Generic type parameters added to PHPDoc on FormatterInterface (@template T), FieldableEntityInterface::get/getFields/getTranslatableFields, TypedDataInterface::getParent/getRoot, TypedConfigManagerInterface::get/createFromNameAndData, and several concrete classes; implementing code may require matching annotations to satisfy PHPStan.

## Technical details

In `phpstan-baseline.php`, 17 `missingType.generics` suppression entries are removed across files in `Component`, `Core`, `layout_builder`, `views`, and test directories.

The affected public interfaces and classes receive explicit generic parameters in their PHPDoc. `FormatterInterface` gains a `@template T of FieldItemListInterface` declaration; its `prepareView()`, `view()`, and `viewElements()` methods use `@phpstan-param T $items`. `FieldableEntityInterface::get()` now returns `FieldItemListInterface<FieldItemInterface>`, and `getFields()`/`getTranslatableFields()` return `array<string, FieldItemListInterface<FieldItemInterface>>`. `TypedConfigManagerInterface::get()` and `createFromNameAndData()` return `TraversableTypedDataInterface<int|string, TypedDataInterface>`. `TypedDataInterface::getParent()` and `getRoot()` gain the same parameterization, as do the corresponding methods in `TypedData`. `FormatterPluginManager::getInstance()`, `FieldBlock::getFormatter()` (layout_builder), and `views\EntityField::getFormatterInstance()` all return `FormatterInterface<FieldItemListInterface<FieldItemInterface>>`. `ItemList::$list` is typed `array<int, TypedDataInterface>` and `getIterator()` returns `ArrayIterator<int, TypedDataInterface>`. `DirectoryWithMetadataDiscovery::getDirectoryIterator()` returns `RecursiveIteratorIterator<SplFileInfo>`, and `ParamConversionEnhancer::copyRawVariables()` returns `InputBag<string|int|float|bool|null>`.

`StaticReflectionClass`'s PHP reflection stubs (`getInterfaces()`, `getParentClass()`, `getTraits()`) instead receive inline `// @phpstan-ignore missingType.generics` comments because those methods implement PHP's built-in reflection API and cannot have their signatures changed to carry generic parameters.

## Upgrade

If you implement or extend any of the following interfaces in a contrib or custom module and run PHPStan, you may now see `missingType.generics` errors that were previously suppressed only in core's baseline:

- `FormatterInterface` — add a `@template` or `@extends` annotation, e.g.:
  ```php
  /**
   * @extends \Drupal\Core\Field\FormatterInterface<\Drupal\Core\Field\FieldItemListInterface<\Drupal\Core\Field\FieldItemInterface>>
   */
  class MyFormatter extends FormatterBase { ... }
  ```
- `TypedDataInterface` / `TypedData` — `getParent()` and `getRoot()` now return `TraversableTypedDataInterface<int|string, TypedDataInterface>`.
- `FieldableEntityInterface` — `getFields()` and `getTranslatableFields()` now return `array<string, FieldItemListInterface<FieldItemInterface>>`.
- `TypedConfigManagerInterface` — `get()` and `createFromNameAndData()` now return `TraversableTypedDataInterface<int|string, TypedDataInterface>`.

If your module already suppresses `missingType.generics` for these methods in its own PHPStan baseline, you can remove those suppressions after adding the proper generic annotations.
