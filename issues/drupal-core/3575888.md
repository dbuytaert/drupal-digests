## Summary

This is a test infrastructure change with no effect on live Drupal sites. The REST functional test suite in core had 143 entity test classes, each running GET, POST, PATCH, and DELETE as four separate PHPUnit test methods. Because every test method triggers a full Drupal install, this was extremely expensive: roughly 572 installs just for those four operations per class. By merging all four operations into a single `testCrud()` method, the suite now needs only one install per class, saving approximately 429 installs across a full run and cutting individual test class runtime by about 40%.

## Impact

Affects module maintainers.

- **API change:** `EntityResourceTestBase::testGet/testPost/testPatch/testDelete` renamed to `doTestGet/doTestPost/doTestPatch/doTestDelete` and made protected; a single `testCrud()` now orchestrates all four.

## Technical details

The core change is in `EntityResourceTestBase`: the four public test methods `testGet()`, `testPost()`, `testPatch()`, and `testDelete()` are renamed to `protected` helpers `doTestGet()`, `doTestPost()`, `doTestPatch()`, and `doTestDelete()`. A new public `testCrud()` method calls all four in sequence. To make chaining possible without a reinstall between operations, each `do*` method now cleans up after itself: it deletes the REST resource config via `$this->resourceConfigStorage->load(...)->delete()`, calls `$this->refreshTestStateAfterRestConfigChange()`, reloads the entity via `$this->entityStorage->loadUnchanged()`, and calls the new helper `revokeAllPermissions()`. That helper strips all permissions from both the anonymous and authenticated roles by loading them via `Role::load()` and calling `revokePermission()` on each, so each subsequent operation starts from a clean authorization state.

Subclasses required corresponding changes. `ConfigEntityResourceTestBase` previously used a `SKIP_METHODS` constant and skipped `testPost`, `testPatch`, and `testDelete` in `setUp()`; that constant and logic are removed, replaced by an overridden `testCrud()` that only calls `doTestGet()`. `FileResourceTestBase` previously used a PHPUnit `#[Before]` hook to skip `testPost` by name; it now overrides `doTestPost()` with an empty body. `XmlEntityNormalizationQuirksTrait` similarly replaces its `#[Before]` skip logic for POST and PATCH with empty `doTestPost()` and `doTestPatch()` overrides. `ImageStyleResourceTestBase` moves its `#[Before]` skip for `testGet`+XML into `setUp()`, updated to check for `testCrud` by name. `ContactMessageResourceTestBase` renames its three public `test*` overrides to `do*`. The `MediaResourceTestBase` override of `testPost()` is renamed to `doTestPost()` with its `parent::doTestPost()` call updated to match.

Any contrib or custom code that extends `EntityResourceTestBase` and overrides any of the four renamed public methods will silently stop running those overrides under `testCrud()`. A change record was created at https://www.drupal.org/node/3578858. The related issue [#3572055](https://www.drupal.org/node/3572055) addressed a prior breakage in XML REST test skipping that this refactor also supersedes.

## Upgrade

If you maintain a module with tests that extend `EntityResourceTestBase` (or its subclasses such as `ConfigEntityResourceTestBase`) and override any of the following public methods, rename them to the listed protected equivalents:

```php
// Before
public function testGet(): void { ... }
public function testPost(): void { ... }
public function testPatch(): void { ... }
public function testDelete(): void { ... }

// After
protected function doTestGet(): void { ... }
protected function doTestPost(): void { ... }
protected function doTestPatch(): void { ... }
protected function doTestDelete(): void { ... }
```

If you previously called `parent::testPost()` (or similar) from an override, update those calls to `parent::doTestPost()` etc. If your override used `markTestSkipped()` to suppress an operation, replace it with an empty method body instead.
