## Summary

Two integer constants that controlled where the comment form appears on a page have been replaced with a proper PHP enum. The constants `CommentItemInterface::FORM_SEPARATE_PAGE` and `CommentItemInterface::FORM_BELOW` are now deprecated; the equivalent values live in the new `FormLocation` enum. This is a purely internal refactor with no change to behavior or stored configuration data.

## Impact

Affects module maintainers.

- **Deprecation:** `CommentItemInterface::FORM_SEPARATE_PAGE` and `CommentItemInterface::FORM_BELOW` deprecated in drupal:11.4.0, removed in drupal:13.0.0; use `FormLocation::SeparatePage` and `FormLocation::Below` instead.
- **New API:** New `Drupal\comment\FormLocation` backed int enum with cases `SeparatePage = 0` and `Below = 1`.

## Technical details

The change introduces `Drupal\comment\FormLocation`, a backed `int` enum in `core/modules/comment/src/FormLocation.php` with two cases: `SeparatePage = 0` and `Below = 1`, mirroring the integer values of the old constants. The constants `FORM_SEPARATE_PAGE` and `FORM_BELOW` in `CommentItemInterface` are kept but marked deprecated in `drupal:11.4.0`, to be removed in `drupal:13.0.0`, with a pointer to change record node 3550055.

All internal call sites were updated: `CommentLinkBuilder` and `CommentManager` now call `FormLocation::tryFrom((int) $field_definition->getSetting('form_location'))` to convert the stored integer setting to an enum case before comparing. `CommentFormatter` similarly uses `FormLocation::tryFrom($comment_settings['form_location'])`. The `CommentItem` field type's `defaultFieldSettings()` now stores `FormLocation::Below->value` instead of the bare integer. Unit and functional tests in `CommentLinkBuilderTest`, `CommentTestTrait`, and `CommentLinksAlterTest` are updated to pass `FormLocation` enum cases rather than raw integers.

This issue is part of the broader effort tracked by [#3518993](https://www.drupal.org/node/3518993) to replace all comment-module integer constants with enums. The companion issue [#3547353](https://www.drupal.org/node/3547353) previously converted the `HIDDEN`, `CLOSED`, and `OPEN` status constants to the `CommentingStatus` enum.

## Upgrade

Replace any direct use of the deprecated constants with the corresponding `FormLocation` enum case:

```php
// Before
use Drupal\comment\Plugin\Field\FieldType\CommentItemInterface;

$location = CommentItemInterface::FORM_BELOW;        // 1
$location = CommentItemInterface::FORM_SEPARATE_PAGE; // 0

// After
use Drupal\comment\FormLocation;

$location = FormLocation::Below;        // FormLocation enum case
$location = FormLocation::SeparatePage; // FormLocation enum case
```

When storing the value in configuration or comparing against a raw integer retrieved from settings, use `->value`:

```php
// Storing / writing to settings
'form_location' => FormLocation::Below->value,

// Reading from settings and comparing
$form_location = FormLocation::tryFrom((int) $field_definition->getSetting('form_location'));
if ($form_location === FormLocation::SeparatePage) { ... }
```
