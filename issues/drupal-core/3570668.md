## Summary

The History module, which tracked which content authenticated users had viewed and marked nodes as new or updated, has been removed from Drupal core. The module was already deprecated and is now available as a contributed module. Sites that relied on it, or on the "new comment" and "new content" indicators it powered, will need to install the contrib version to preserve that functionality.

## Impact

Affects module maintainers.

- **API change:** Deprecated `CommentManagerInterface::getCountNewComments()`, `CommentController::renderNewCommentsNodeLinks()`, the `comment.new_comments_node_links` route, `comment\Plugin\views\field\NodeNewComments`, and the `drupal.comment-new-indicator`/`drupal.node-new-comments-link` Comment library aliases are all removed.
- **Deprecation:** The entire History module is removed from core; it is available as a contributed module at drupal/history.

## Technical details

The History module was marked `lifecycle: deprecated` in its `info.yml` and this commit removes it entirely from core along with all deprecated bridge code that was added in earlier releases to ease the transition.

**History module removed.** All module files are deleted: `history.install` (with `history_schema()` and `history_update_last_removed()`), `history.module` (with the `HISTORY_READ_LIMIT` constant and the procedural functions `history_read()`, `history_read_multiple()`, and `history_write()`), `history.routing.yml` (routes `history.get_last_node_view`, `history.read_node`, and `history.new_comments_node_links`), `history.services.yml` (services `history.comment_link_builder` and `Drupal\history\HistoryManager`), `history.libraries.yml` (libraries `api`, `mark-as-read`, `drupal.comment-new-indicator`, and `drupal.node-new-comments-link`), and the associated JavaScript files (`history.js`, `mark-as-read.js`, `comment-new-indicator.js`, `node-new-comments-link.js`).

**Deprecated Comment module shims removed.** During the deprecation period, the Comment module provided backward-compatibility bridges pointing to their History equivalents. All of these are now deleted: the `CommentManagerInterface::getCountNewComments()` method and its `CommentManager` implementation; `CommentController::renderNewCommentsNodeLinks()`; the `comment.new_comments_node_links` route in `comment.routing.yml`; the `comment\Plugin\views\field\NodeNewComments` proxy class; and the `drupal.comment-new-indicator` and `drupal.node-new-comments-link` deprecated library aliases in `comment.libraries.yml`.

**PHPStan baseline cleaned up.** Eighteen `$ignoreErrors[]` suppressions for `missingType.return` and `return.missing` errors across `HistoryController`, `HistoryUserTimestamp` (field and filter plugins), `NodeNewComments`, and `HistoryTest` are removed from `phpstan-baseline.neon`.

**Update path test fixtures.** An `uninstall-history.php` fixture was added and is now referenced in six update path tests (`BlockWeightUpdateTest`, `MenuBlockDepthTest`, `BlockContentRemoveListingEmptyUpdateTest`, `BlockContentStatusInfoUpdatePathTest`, `AddListPluginStylesTest`, `ContentModerationStateIndexUpdatePathTest`). These tests start from 10.3.0 database dumps where History was installed; the fixture uninstalls it before running assertions. Related fixture work is tracked in [#3569127](https://www.drupal.org/node/3569127).

**MAINTAINERS.txt.** The History maintainer entry is removed.

## Upgrade

If your site has the History module installed, you must install the contributed version before upgrading to Drupal 12.0.0:

```
composer require drupal/history
```

If your custom code calls any of the following removed APIs, migrate to the contrib module's equivalents or remove the calls:

- `CommentManagerInterface::getCountNewComments()` → `\Drupal\history\HistoryManager::getCountNewComments()`
- `CommentController::renderNewCommentsNodeLinks()` → `\Drupal\history\Controller\HistoryController::renderNewCommentsNodeLinks()`
- Route `comment.new_comments_node_links` → route `history.new_comments_node_links`
- Libraries `comment/drupal.comment-new-indicator`, `comment/drupal.node-new-comments-link` → `history/drupal.comment-new-indicator`, `history/drupal.node-new-comments-link`
- `Drupal\comment\Plugin\views\field\NodeNewComments` → `\Drupal\history\Plugin\views\field\NodeNewComments`
- Procedural functions `history_read()`, `history_read_multiple()`, `history_write()` → methods on `\Drupal\history\HistoryManager`
