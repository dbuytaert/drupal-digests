## Summary

Drupal 11 upgraded its test infrastructure from PHPUnit 9 to PHPUnit 10. PHPUnit 10 is a major release with sweeping breaking changes, so this meta issue coordinated more than 160 child issues over roughly three years to get Drupal's test suite fully compatible. The end result is that Drupal 11 ships with PHPUnit 10, drops the `symfony/phpunit-bridge` dependency, and uses PHPUnit's modern extension and event system throughout. Contrib modules and custom code that run against Drupal 11's test infrastructure will need to ensure their own tests are also PHPUnit 10 compatible. A follow-up issue [#3418267](https://www.drupal.org/node/3418267) tracks upgrading to PHPUnit 11.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 10 introduced several architectural breaks that required coordinated changes across Drupal's test infrastructure.

**Listener architecture replaced by extensions and events.** PHPUnit 10 removed the listener API entirely. Drupal's HTML output printer was rewritten as a proper PHPUnit extension: `HtmlOutputLogger` and its event subscribers now live under `core/tests/Drupal/TestTools/Extension/HtmlLogging/`. `DrupalComponentTestListenerTrait`, which used the old listener mechanism, was removed and its checks replaced by a PHPStan rule ([#3400366](https://www.drupal.org/node/3400366)).

**`symfony/phpunit-bridge` dropped.** The bridge was not made compatible with PHPUnit 10 in time. Drupal replaced its deprecation-tracking functionality with a custom `DeprecationHandler` extension at `core/tests/Drupal/TestTools/Extension/DeprecationBridge/DeprecationHandler.php`, which still reads the `SYMFONY_DEPRECATIONS_HELPER` environment variable for backwards-compatible configuration ([#3417066](https://www.drupal.org/node/3417066)).

**Data providers must be static.** PHPUnit 10 requires all `@dataProvider` methods to be declared `static public`. The bulk of Drupal's data providers were converted in [#3353210](https://www.drupal.org/node/3353210) and remaining cases in [#3422383](https://www.drupal.org/node/3422383). Data providers that relied on mock objects instantiated in the test class needed special handling ([#3368713](https://www.drupal.org/node/3368713)).

**`withConsecutive()` removed.** `InvocationMocker::withConsecutive()` was deprecated in PHPUnit 9.6 and removed in 10. All usages across core tests were refactored ([#3406006](https://www.drupal.org/node/3406006)).

**`TestCase::getName()` renamed.** The method was renamed to `name()` and the old form removed. Drupal introduced a forward-compatibility shim so tests could use `name()` before the PHPUnit 10 upgrade ([#3405353](https://www.drupal.org/node/3405353)).

**`expect*()` error/warning methods removed.** `expectError*()` and `expectWarning*()` calls were deprecated in 9.6 and removed in 10; all usages were replaced ([#3417650](https://www.drupal.org/node/3417650)).

**`Test::parseTestMethodAnnotations` removed.** Drupal's use of this internal method was replaced ([#3355795](https://www.drupal.org/node/3355795)). The `@requires externalCommand` annotation, which relied on PHPUnit's annotation parser, was also no longer parsed and required a workaround ([#3362083](https://www.drupal.org/node/3362083)).

**`MarkupInterfaceComparator` signature fix.** PHPUnit 10 added typed return declarations to its `Comparator` base class. `MarkupInterfaceComparator` at `core/tests/Drupal/TestTools/Comparator/MarkupInterfaceComparator.php` was updated to match ([#3405351](https://www.drupal.org/node/3405351)).

**Test suite static discovery removed.** `TestSuite::static()` no longer exists in PHPUnit 10. Drupal's test suites were deprecated ([#3402444](https://www.drupal.org/node/3402444)), since core's actual test runner (`run-tests.sh`) uses `TestDiscovery::getPhpunitTestSuite()` rather than PHPUnit's own suite discovery.

**`TestCase` constructor arguments changed.** A handful of tests that directly instantiated `TestCase` subclasses were rewritten to avoid the constructor ([#3405360](https://www.drupal.org/node/3405360)).

**`phpunit.xml.dist` migrated.** The `<listeners>` section was removed and replaced by an `<extensions>` block listing the new PHPUnit extension classes. The schema declaration now points to `https://schema.phpunit.de/10.5/phpunit.xsd`.

**`phpspec/prophecy-phpunit` updated.** Version 2.1.0 of the package is required for PHPUnit 10 compatibility.

**`TestRequirementsTrait::checkRequirements` deprecated.** The trait's `checkRequirements` and `checkModuleRequirements` methods were deprecated as part of cleaning up annotation-based test requirements ([#3418379](https://www.drupal.org/node/3418379)).

**Symfony `Process` for subprocesses.** `PhpUnitTestRunner` was updated to use Symfony's `Process` component when spawning PHPUnit subprocesses ([#3417805](https://www.drupal.org/node/3417805)), replacing direct shell invocation and addressing changed CLI options in PHPUnit 10.

## Upgrade

Contrib modules and custom code targeting Drupal 11 must ensure PHPUnit 10 compatibility. Key changes required:

1. Make all `@dataProvider` methods `static public`:
```php
// Before
public function myDataProvider(): array { ... }

// After
public static function myDataProvider(): array { ... }
```

2. Replace `withConsecutive()` on mock expectations (removed in PHPUnit 10). Use separate `willReturnCallback` or multiple `expects` calls instead.

3. Replace deprecated error/warning expectation methods:
```php
// Before
$this->expectError();
$this->expectWarning();

// After: use set_error_handler() or refactor the code under test
```

4. Use `name()` instead of `getName()` on `TestCase`:
```php
// Before
$this->getName();

// After
$this->name();
```

5. If your module defines a PHPUnit listener in `phpunit.xml.dist`, migrate it to the PHPUnit 10 extension API.

6. If your tests use `Test::parseTestMethodAnnotations()`, replace it with PHPUnit 10's metadata API.

See the GitLab CI template adjustments in [#3444792](https://www.drupal.org/node/3444792) if you maintain a GitLab CI pipeline for a contrib project.
