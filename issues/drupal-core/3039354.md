## Summary

The "User ID from logged in user" contextual filter default argument in Views was fetching the current user via the static `\Drupal::currentUser()` call rather than through proper dependency injection. This has been corrected so the `CurrentUser` plugin now receives an `AccountInterface` service through its constructor, consistent with the rest of Drupal core. The change is purely internal and has no effect on how the filter behaves for site owners.

## Impact

Affects module maintainers.

- **Deprecation:** Calling `CurrentUser::__construct()` without the `$currentUser` argument is deprecated in drupal:11.2.0 and will be required in drupal:12.0.0; see change record https://www.drupal.org/node/3347878

## Technical details

The `CurrentUser` class at `core/modules/user/src/Plugin/views/argument_default/CurrentUser.php` is the Views argument-default plugin that provides the "User ID from logged in user" option for contextual filters. It extends `ArgumentDefaultPluginBase` and implements `CacheableDependencyInterface`.

Before this change, `getArgument()` called `\Drupal::currentUser()->id()` directly, bypassing the service container. The fix adds a proper constructor with a promoted optional property `?AccountInterface $currentUser = NULL`, a `create()` factory that pulls `current_user` from the container, and updates `getArgument()` to use `$this->currentUser->id()`. For backwards compatibility with any code instantiating the class without passing `$currentUser`, a deprecation is triggered: calling `CurrentUser::__construct()` without `$currentUser` is deprecated in `drupal:11.2.0` and will be required in `drupal:12.0.0` (change record: https://www.drupal.org/node/3347878).

A same-day follow-up commit (`91de3017`) corrected the argument name in the deprecation message from `$current_user` to `$currentUser` to match the actual parameter name, spotted immediately by nikolay shapovalov after the merge.

## Upgrade

If you subclass `Drupal\user\Plugin\views\argument_default\CurrentUser` and override `__construct()`, add `AccountInterface $currentUser` as a parameter and pass it to the parent constructor:

```php
use Drupal\Core\Session\AccountInterface;
use Symfony\Component\DependencyInjection\ContainerInterface;

public function __construct(array $configuration, $plugin_id, $plugin_definition, AccountInterface $currentUser) {
  parent::__construct($configuration, $plugin_id, $plugin_definition, $currentUser);
}

public static function create(ContainerInterface $container, array $configuration, $plugin_id, $plugin_definition) {
  return new static($configuration, $plugin_id, $plugin_definition, $container->get('current_user'));
}
```

Calling the constructor without `$currentUser` triggers a deprecation in drupal:11.2.0 and the argument becomes required in drupal:12.0.0.
