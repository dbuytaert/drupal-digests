## Summary

This is a test-only change with no effect on production behavior. Nine unit test files in the Config library were updated to replace expectation-less PHPUnit mock objects with stubs, and two PHPStan baseline suppressions were removed now that the underlying code is clean.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces a deprecation notice when a mock object is created via `createMock()` but no expectations are configured on it, because such objects should be stubs instead. Additionally, `$this->any()` is deprecated (it was equivalent to having no expectation), and `with()` is deprecated on stubs.

The fix across ten files in `core/tests/Drupal/Tests/Core/Config/`:

- `createMock()` calls were replaced with `createStub()` wherever no `expects()` calls follow. Affected classes include `StorageInterface`, `EventDispatcherInterface`, `TypedConfigManagerInterface`, `CacheTagsInvalidatorInterface`, `UuidInterface`, `LanguageManagerInterface`, `EntityTypeManagerInterface`, `ConfigEntityTypeInterface`, `QueryInterface`, and `ConfigEntityStorageInterface`.
- `$this->any()` matchers were removed from the few places they were used.
- `with()` argument constraints were removed from stub method configurations.
- For the handful of tests that *do* set expectations, a pattern of lazy re-initialization was introduced: a default stub is set up in `setUp()`, and dedicated helper methods (`setUpMockCacheTagsInvalidator()`, `setUpMockEntityType()`, `setUpMockTypedConfigManager()`, `setUpMockUuidService()`, `setUpMockEntityTypeManger()`) replace only that dependency with a real mock at the start of the tests that need it. This avoids penalizing the many tests that never invoke an assertion on those objects.
- `getMockBuilder(StubConfigEntity::class)->onlyMethods([])->getMock()` patterns were replaced with direct `new StubConfigEntity(...)` instantiation, and the helper `getMockEntity()` in `ConfigEntityStorageTest` was removed entirely.
- `EntityDisplayBaseMockableClass` received an explicit no-op constructor so it can be directly instantiated without `disableOriginalConstructor()`.
- Two `$ignoreErrors[]` entries suppressing `any()` deprecation warnings for `ConfigEntityBaseUnitTest` and `EntityDisplayModeBaseUnitTest` were removed from `core/.phpstan-baseline.php`.
