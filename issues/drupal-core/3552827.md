## Summary

Drupal's test suite now uses PHPUnit's native method for expecting deprecation messages. The change replaces all instances of the deprecated `expectDeprecation()` method with `expectUserDeprecationMessage()` throughout core tests. This internal cleanup ensures Drupal's testing framework stays aligned with modern PHPUnit standards and allows contributed modules to receive proper deprecation warnings when testing with Drupal 11.4.

## Impact

No upgrade or configuration changes required.

## Technical details

In [#3497124](https://www.drupal.org/project/drupal/issues/3497124), the `expectDeprecation()` method was deprecated in favor of PHPUnit's native `expectUserDeprecationMessage()`. This issue completes the transition by replacing all remaining calls across Drupal core's test suite.

The change touches over 100 test files across all subsystems, performing a straightforward method name replacement. No test logic changes were required. The deprecation ignore rule for `expectDeprecation()` was also removed from `core/.deprecation-ignore.txt`, allowing Drupal to now report usage of the deprecated method.

The merge requests targeted both the `main` and `11.x` branches to ensure the cleanup reaches Drupal 11.4, where contrib modules testing against that version will start seeing deprecation warnings if they still use `expectDeprecation()`. This helps the contributed module ecosystem migrate to the new PHPUnit standard before the old method is fully removed in Drupal 12.
