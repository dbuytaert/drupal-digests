## Summary

Drupal's coding standards include a rule that flags arrays with long declarations, but this rule was not enforced across the entire codebase. This change fixes all remaining violations of the `LongLineDeclaration` standard by splitting long array declarations and function calls across multiple lines for better readability. The code now consistently follows the rule that lines should not exceed a certain length, making the codebase easier to read and maintain. All changes preserve existing functionality while improving code formatting.

## Technical details

The `Drupal.Array.Array.LongLineDeclaration` PHPCS sniff was previously enabled only for specific directories through include patterns in `phpcs.xml.dist`. This issue removed those directory restrictions and enabled the sniff globally across the entire codebase.

The changes address violations by splitting long lines in several ways: breaking array declarations across multiple lines with each element on its own line, splitting function calls with multiple parameters, and wrapping chained method calls. In `batch.inc`, an unnecessary `array_merge()` call was removed and the parameters were passed directly instead. One line in `responsive_image` was marked with a PHPCS ignore comment because reformatting would harm readability.

The formatting changes follow existing patterns in the codebase, prioritizing readability and consistency. No API behavior changes occurred. The diff touches approximately 170 files across core, affecting function calls with long parameter lists, array declarations, and method chaining patterns.
