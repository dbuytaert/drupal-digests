## Summary

Views cache plugins included an outdated method from Drupal 7 that retroactively checked cache expiration by comparing timestamps. This approach is unnecessary because Drupal's cache system already handles expiration when cache items are stored. The method has been deprecated and is no longer called internally. This simplifies the Views caching logic and removes confusion about how cache expiration works in Views.

## Impact

Affects module maintainers.

- **Deprecation:** `CachePluginBase::cacheExpire()` deprecated with no replacement; custom Views cache plugins should remove overrides of this method.

## Technical details

The `CachePluginBase::cacheExpire()` method was a holdover from Drupal 7 caching that attempted to validate cache freshness by comparing `$cache->created` against a calculated cutoff time during cache retrieval. This was redundant because Drupal's cache backend already enforces expiration based on the `expire` parameter set during `cache->set()`.

The `cacheGet()` method previously called `cacheExpire()` to get a cutoff timestamp, then conditionally restored cached data only if `!$cutoff || $cache->created > $cutoff`. Since core implementations either returned `FALSE` (Tag plugin) or calculated a cutoff based on lifespan (Time plugin), this logic duplicated what the cache backend already handles through proper max-age and expiration settings via `cacheSetMaxAge()`.

The method is now deprecated with a trigger error, and the call has been removed from `cacheGet()`. Cached results are now restored directly when found, relying on the cache backend to return only valid entries. Core implementations in the Time and Tag plugins also add the deprecation trigger while maintaining their return values for backward compatibility.

## Upgrade

Custom Views cache plugins that override `CachePluginBase::cacheExpire()` should remove their implementation. If your plugin needs time-based expiration, implement it by returning the appropriate value from `cacheSetMaxAge()` instead, which sets the cache `expire` parameter when storing cache entries.

Before:
```php
protected function cacheExpire($type) {
  return $this->time->getRequestTime() - 3600;
}
```

After: Remove the method entirely. Use `cacheSetMaxAge()` to control expiration:
```php
protected function cacheSetMaxAge($type) {
  return 3600;
}
```

See the change record at [#3576855](https://www.drupal.org/node/3576855) for details.
