## Summary

Drupal core now ships the `twig/html-extra` package, exposing three new functions in every Twig template: `html_cva` for Class Variance Authority (CVA) patterns, `html_attr` for building HTML attribute strings, and `html_classes` for conditional CSS class management. Theme and component developers can use these functions without installing a contrib module.

## Impact

Affects module maintainers and site owners.

- **Requirements:** New required Composer dependency: `twig/html-extra ^3.23`.
- **New API:** Three new Twig functions available in all templates: `html_cva` for CVA-style component variant styling, `html_attr` for HTML attributes, and `html_classes` for conditional class lists.

## Technical details

The change adds `twig/html-extra: ^3.23` as a required dependency in `composer.json`. Because `twig/html-extra` is a subtree split of the main Twig repository and follows the same release cycle and security policies as `twig/twig`, no extended dependency evaluation was required beyond framework-manager and release-manager sign-off.

`Twig\Extra\Html\HtmlExtension` is registered in `core.services.yml` as a Twig service tagged with `twig.extension` at priority 101, which is the same priority used by other core Twig extensions. The `Twig\Extra\Html\Cva` class is added to the allowed-classes list in `TwigSandboxPolicy`, because its sole public method (`apply()`) is meant to be called directly from templates and would otherwise be blocked by the Twig sandbox.

A new Single Directory Component (`sdc_theme_test:cva`) and a `CvaTest` kernel test verify that calling `html_cva()` with a base class and a `variants` map, then invoking `cva.apply()` with a prop value, produces the correct combined CSS class string in rendered output. A follow-up issue ([#3578007](https://www.drupal.org/node/3578007)) tracks adopting `html_classes` in existing core Twig templates.
