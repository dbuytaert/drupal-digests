## Summary

Core now includes the `twig/html-extra` package, which provides three new Twig functions for theme developers. The `html_cva` function implements Class Variance Authority (CVA) patterns for managing component styling with variants. The `html_attr` and `html_classes` functions help manage HTML attributes and CSS classes. These functions are now available in all Twig templates.

## Impact

Affects module maintainers and site owners.

- **Requirements:** New required dependency: `twig/html-extra` package
- **New API:** Three Twig functions: `html_cva` for CVA patterns, `html_attr` for HTML attributes, and `html_classes` for class management

## Technical details

The `twig/html-extra` package is a subtree split of the main Twig repository, maintained by the Twig team with the same release and security policies. The change adds it as a core dependency in `composer.json` and registers `Twig\Extra\Html\HtmlExtension` as a Twig extension in `core.services.yml` with priority 101.

The `Cva` class is added to `TwigSandboxPolicy`'s allowed classes list because its only public method is meant to be called from templates. Without this, calling methods on CVA objects would be blocked by the sandbox.

The test creates a Single Directory Component that uses `html_cva` to define base classes and a `friendliness` variant with two states. The kernel test verifies that calling `cva.apply()` with different prop values produces the correct CSS classes in the rendered output.
