## Summary

Drupal's render system now accepts `page_top` and `page_bottom` as valid keys inside `#attached` render arrays. This lets a controller or build function inject content directly into those page regions without implementing `hook_page_top` or `hook_page_bottom`. The immediate beneficiary is the node preview feature: the "Back to content editing" toolbar that previously ran a route check on every page request via `NodeThemeHooks::pageTop()` is now attached directly by `NodePreviewController::view()`, so it only executes when the node preview route is actually visited.

## Impact

Affects module maintainers.

- **API change:** `HtmlRenderer::buildPageTopAndBottom()` gains optional `$page_top` and `$page_bottom` array parameters (see change record #3558617); `NodePreviewController::__construct()` gains an optional `$formBuilder` parameter, required from Drupal 12.0.
- **Deprecation:** `NodePreviewController::__construct()` without a `$formBuilder` argument is deprecated in drupal:11.4.0 and the argument will be required in drupal:12.0.0.
- **New API:** New `page_top` and `page_bottom` keys in `#attached` render arrays let any render array inject content into those page regions without a hook.

## Technical details

The node preview toolbar was injected via `NodeThemeHooks::pageTop()`, a `hook_page_top` implementation that checked `$this->routeMatch->getRouteName() == 'entity.node.preview'` on every request before building the form render array. The fix moves this logic into the controller where it belongs and opens the `#attached` mechanism to support the pattern broadly.

**Render pipeline changes (`HtmlRenderer` and `HtmlResponseAttachmentsProcessor`)**

`HtmlRenderer::buildPageTopAndBottom()` gains two optional parameters: `array $page_top = []` and `array $page_bottom = []`. These seed the local variables that `hook_page_top` and `hook_page_bottom` implementations then augment. The caller, `HtmlRenderer::renderResponse()`, extracts the values from `$main_content['#attached']['page_top']` and `$main_content['#attached']['page_bottom']` (defaulting to empty arrays) and passes them in. Previously, those variables were always initialised to `[]` inside the method, ignoring any `#attached` values entirely.

`HtmlResponseAttachmentsProcessor::processAttachments()` adds `page_top` and `page_bottom` to its allow-list of recognised attachment types, so they are no longer rejected as unsupported. `AttachmentsInterface`'s docblock is updated to document both new keys.

Two PHPStan baseline suppressions for "variable always falsy in empty()" are removed because `$page_top` and `$page_bottom` are now method parameters and no longer always empty at initialisation time.

**Node module changes (`NodePreviewController` and `NodeThemeHooks`)**

`NodePreviewController::view()` now builds the preview toolbar inline and attaches it as `$build['#attached']['page_top']['node_preview']`, the same render array structure that `NodeThemeHooks::pageTop()` formerly produced. To do this, the controller receives `FormBuilderInterface` as a new fourth constructor argument. It is optional (defaulting to `NULL`) with a `@trigger_error` deprecation notice, so existing code that instantiates the controller directly without passing `form_builder` gets a deprecation in 11.4 and will break in 12.0. `NodePreviewController::create()` is updated to inject `form_builder` from the container.

`NodeThemeHooks::pageTop()`, its `#[Hook('page_top')]` attribute, and the `#[AutowireServiceClosure('form_builder')]` dependency are removed entirely from `NodeThemeHooks`.

The public signature change on `buildPageTopAndBottom()` is tracked as a change record in [#3558617](https://www.drupal.org/node/3558617). A follow-up issue [#3571143](https://www.drupal.org/node/3571143) was filed to apply the same pattern to `BlockHooks::pageTop()`.

## Upgrade

If you call `HtmlRenderer::buildPageTopAndBottom()` directly (it is a public method), update the call site to pass the new optional parameters if needed:

```php
// Before (still works but misses #attached page_top/page_bottom content):
$this->buildPageTopAndBottom($html);

// After (to seed from an existing $main_content render array):
$this->buildPageTopAndBottom(
  $html,
  $main_content['#attached']['page_top'] ?? [],
  $main_content['#attached']['page_bottom'] ?? [],
);
```

If you instantiate `NodePreviewController` manually, add `form_builder` as the fourth argument to avoid the deprecation notice in 11.4:

```php
// Before (triggers deprecation in 11.4, fatal in 12.0):
new NodePreviewController($entityTypeManager, $renderer, $entityRepository);

// After:
new NodePreviewController($entityTypeManager, $renderer, $entityRepository, $formBuilder);
```

To inject content into `page_top` or `page_bottom` from a render array without implementing a hook:

```php
$build['#attached']['page_top']['my_element'] = [
  '#markup' => '<p>Above everything</p>',
];
```

### Collaboration

longwave proposed the key design insight in comment [#14917380](https://www.drupal.org/node/14917380): since `['#attached']['html_head']` already allows injecting into `<head>`, the same pattern could be extended to `page_top` and `page_bottom`. This became the foundation for acbramley's implementation. During review, alexpott identified that documentation for the new attachment keys was missing from the correct location and requested a follow-up to convert `BlockHooks::pageTop()` the same way; acbramley filed that follow-up as [#3571143](https://www.drupal.org/node/3571143).
