## Summary

This issue moves procedural code from `content_translation.module` into services and hook classes as part of the initiative to eliminate core `.module` files. Several procedural functions for field installation, access checking, and form configuration were deprecated and replaced with service methods in `ContentTranslationHooks`, `ContentTranslationEnableTranslationPerBundle`, and `ContentTranslationManager`.

## Impact

Affects module maintainers.

- **API change:** Constructor signatures changed for `ContentTranslationManager`, `ContentTranslationOverviewAccess`, `CommentAdminOverview`, and `AccountSettingsForm` (backward compatible)
- **Deprecation:** `_content_translation_install_field_storage_definitions()`, `content_translation_translate_access()`, `content_translation_enable_widget()`, `content_translation_language_configuration_element_process()`, `content_translation_language_configuration_element_validate()`, and `content_translation_language_configuration_element_submit()` functions replaced by service methods
- **New API:** New `ContentTranslationManagerInterface::access()` method and `ContentTranslationEnableTranslationPerBundle` service

## Technical details

The change consolidates content translation procedural code into object-oriented services. The underscore-prefixed `_content_translation_install_field_storage_definitions()` function was moved to `ContentTranslationHooks::installFieldStorageDefinitions()`. The `content_translation_translate_access()` function was replaced by a new `ContentTranslationManager::access()` method. A new `ContentTranslationEnableTranslationPerBundle` service was created to handle the form widget for enabling translation per bundle, replacing `content_translation_enable_widget()`, `content_translation_language_configuration_element_process()`, `content_translation_language_configuration_element_validate()`, and `content_translation_language_configuration_element_submit()`. Form alter implementations were moved from `AccountSettingsForm` and `CommentAdminOverview` into `ContentTranslationFormHooks`. Comment link building was moved from `CommentLazyBuilders` to a `hook_comment_links_alter()` implementation in `ContentTranslationHooks`. The entity type definition access callback was changed from the function name `'content_translation_translate_access'` to service method notation `'content_translation.manager:access'`. `ContentTranslationOverviewAccess` now uses `CallableResolver` to invoke this. Several service constructors were updated to inject dependencies that were previously obtained via `\Drupal::service()`, with backward compatibility layer for old signatures using `func_get_arg()` and union types.
