## Summary

This change adds a functional test to confirm that a moderated node's title appears exactly once in the page heading on the `latest-version` route, preventing regressions where the title could be rendered twice. It is a pure test addition with no user-facing behavior changes.

## Impact

No upgrade or configuration changes required.

## Technical details

The `latest-version` route in `content_moderation` can display a node's draft revision to users with the appropriate permission. A related cleanup effort ([#2877924](https://www.drupal.org/node/2877924), removing `ContentPreprocess` and associated `is_page` logic) needed regression coverage before it could safely proceed, so this test was split into its own issue.

The single change is a new test method `testNodeLatestVersionContent()` added to `ModerationStateNodeTest` in `core/modules/content_moderation/tests/src/Functional/ModerationStateNodeTest.php`. The method:

1. Creates a `moderated_content` node in the `published` state.
2. Saves a new draft revision with a distinct title (`'Test moderated node content - draft'`).
3. Navigates to the node's `latest-version` URL as an admin user.
4. Asserts a 200 response, then uses `pageTextMatchesCount(2, ...)` to confirm the draft title appears exactly twice (once in the `<title>` tag and once in the `<h1>`), and `elementNotExists('css', 'article h2')` to confirm no duplicate title is rendered inside the article body.

Reviewer berdir noted that this coverage is what enables the `is_page` code removal in `content_moderation` to proceed without regression risk. Committer catch observed the test is also a future candidate for simplification once `KernelTestBase` gains HTTP request support via [#3390193](https://www.drupal.org/node/3390193).
