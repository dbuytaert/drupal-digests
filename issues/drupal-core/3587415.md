## Summary

On a completely cold cache, Drupal was querying the key/value store for all entity last-installed schema definitions and then immediately querying it again to fetch the field storage definitions for each entity type individually. This double-work is now eliminated. The fix reduces database queries and cache operations on every cold-cache page load, with performance tests showing roughly 9 fewer queries and 9 fewer cache gets and sets per request on the Umami demo profile.

## Impact

No upgrade or configuration changes required.

## Technical details

The problem was in `EntityLastInstalledSchemaRepository::getLastInstalledDefinitions()`. When the method had to load all entity type definitions from the key/value store (a cache miss), it already had the raw `$all_definitions` array in hand, which contains both entity type definitions and per-entity-type field storage definitions keyed as `{entity_type}.field_storage_definitions`. However, it was discarding the field storage data and only caching the entity type definitions. A subsequent call to get field storage definitions for any entity type would then trigger another key/value lookup.

The fix replaces the single `$this->cacheBackend->set('entity_type_definitions.installed', ...)` call with a `$this->cacheBackend->setMultiple($items)` call that writes both the entity type definitions cache entry (`entity_type_definitions.installed`) and a `{entity_type}.field_storage_definitions.installed` cache entry for every entity type in one shot. It also populates the in-process static cache `$this->entityFieldStorageDefinitions[$key]` for each entity type at the same time.

This is a follow-up in the same area as [#3131585](https://www.drupal.org/node/3131585), which had previously addressed a performance regression from using last-installed entity definitions. The improvement is neutral for "cool cache" requests (individual field storage definitions are still cached separately so not everything is loaded when only some are needed) but reduces both key/value queries and cache set operations on the first completely cold request.
