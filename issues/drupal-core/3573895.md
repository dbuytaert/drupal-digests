## Summary

This is a purely internal test infrastructure change with no effect on production behavior. Two tests in the `migrate` module had undeclared dependencies on `migrate_drupal`, which would break once `migrate_drupal` is removed from core. The fix eliminates those dependencies so the migrate module's tests are self-contained.

## Impact

No upgrade or configuration changes required.

## Technical details

The change is part of the broader removal of `migrate_drupal` from Drupal core (tracked in [#3572280](https://www.drupal.org/node/3572280)). Two problems were resolved:

1. `MigrateSourcePluginTest` used `migrate_drupal` as a fixture module to test discovery of source plugins with multiple providers. A new dedicated test module, `migrate_multiple_provider_test`, is introduced with its own `EmptySource` source plugin (at `modules/migrate/tests/modules/migrate_multiple_provider_test/src/Plugin/migrate/source/EmptySource.php`) that extends `Drupal\migrate\Plugin\migrate\source\EmptySource` and implements `ContainerFactoryPluginInterface` and `DependentPluginInterface`. This plugin uses `DependencyTrait` and resolves an entity-type-based module dependency in `calculateDependencies()`. All references to `migrate_drupal` in `MigrateSourcePluginTest` are replaced with `migrate_multiple_provider_test`, and the expected discovered plugins from `migrate_drupal` (e.g. `d6_file`, `d7_file`) are dropped from assertions.

2. `MigrateEntityContentBaseTest` imported `StubTestTrait` from `Drupal\migrate_drupal\Tests\StubTestTrait`. That trait is now moved into the `migrate` module itself at `modules/migrate/tests/src/StubTestTrait.php` under the namespace `Drupal\Tests\migrate`. The moved trait contains `performStubTest()`, `createEntityStub()`, and `validateStub()` unchanged in logic; the use statement in `MigrateEntityContentBaseTest` is updated to the new namespace.

`migrate_source_annotation_bc_test` previously extended `Drupal\migrate_drupal\Plugin\migrate\source\EmptySource`; it is updated to extend the new `Drupal\migrate_multiple_provider_test\Plugin\migrate\source\EmptySource` instead.

The PHPStan baseline (`phpstan-baseline.neon`) is updated to remove the now-resolved `class.extendsDeprecatedClass` suppression for the old annotation BC test class and to add a `missingType.return` suppression for `addDependencies()` in the new test module's `EmptySource`. A previously suppressed `missingType.return` for `MigrateEntityContentBaseTest::performStubTest()` is removed because that method now has a `void` return type in the moved trait.
