## Summary

Several hook implementation methods in the Olivero theme's `OliveroHooks` class were missing explicit `array` type declarations on their by-reference parameters. This is an internal code quality improvement with no user-facing effect, aligning Olivero with Drupal's move toward stricter typing in PHP. The scope was expanded during review to include all affected parameters in the class rather than fixing them one hook at a time.

## Impact

Affects module maintainers.

- **API change:** 27 method signatures in `OliveroHooks` now declare `array` on their by-reference parameters (e.g. `array &$variables`); code that extends `OliveroHooks` and overrides these methods must add matching type declarations.

## Technical details

In `core/themes/olivero/src/Hook/OliveroHooks.php`, 27 hook implementation methods had their leading by-reference parameter declared without an explicit type, for example `&$variables`, `&$form`, `&$suggestions`, and `&$info`. The fix adds the `array` type hint to each, producing signatures such as `array &$variables`, `array &$form`, `array &$suggestions`, and `array &$info`. Affected methods include all `preprocess_*` implementations, both `form_alter` and `form_node_preview_form_select_alter`, `element_info_alter`, `theme_suggestions_menu_alter`, `theme_suggestions_block_alter`, `theme_suggestions_user_alter`, and `form_views_exposed_form_alter`. The change was originally scoped to Olivero alone; a companion issue [#3579925](https://www.drupal.org/node/3579925) (ClaroHooks.php) was closed as a duplicate once its changes were cherry-picked into this branch. The issue was blocked briefly on [#3579903](https://www.drupal.org/node/3579903) (eliminating Olivero's legacy `.theme` file), which needed to land first so the hook implementations existed only in the OOP `OliveroHooks` class.
