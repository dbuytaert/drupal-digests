## Summary

Drupal continues to remove dependencies on jQuery UI Dialog as part of its modernization effort. This change replaces jQuery UI's internal focus management methods with vanilla JavaScript when dialogs are opened via Ajax. The focus behavior remains identical for end users, but the code no longer relies on jQuery UI's private `_focusedElement` and `_focusTabbable` methods. This is another incremental step toward fully replacing jQuery UI Dialog with a native HTML dialog element in the future.

## Impact

No upgrade or configuration changes required.

## Technical details

The `dialog.ajax.js` file previously relied on jQuery UI Dialog's private methods `_focusedElement` and `_focusTabbable` to manage focus when Ajax responses opened or updated dialogs. This created tight coupling to jQuery UI internals.

The patch replaces these two method calls with vanilla JavaScript that replicates the same focus behavior. The new code follows jQuery UI Dialog's focus priority: first check for an autofocus attribute, then find the first focusable element in the dialog content, then check the button pane, then the close button, and finally focus the dialog wrapper itself.

The `focusable()` utility function, already used elsewhere in Drupal core, handles finding focusable elements. This change maintains identical user-facing behavior while removing dependency on jQuery UI private APIs. The timeout wrapper remains to allow dialog buttons to update before focus moves.

This is part of the broader effort tracked in multiple issues to replace jQuery UI Dialog completely. Once all such conversions are done, `dialog.jquery-ui.js` can be removed entirely.
