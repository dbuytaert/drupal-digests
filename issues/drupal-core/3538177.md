## Summary

Drupal's ongoing effort to replace jQuery UI Dialog takes another step forward. This change removes the last jQuery UI-specific focus management code from AJAX dialog handling and replaces it with native browser methods. The dialog system now handles keyboard navigation and focus management using standard JavaScript instead of jQuery UI's internal methods. This brings Drupal closer to removing the jQuery UI Dialog library entirely, reducing page weight and improving maintainability. The focus behavior remains identical for site visitors and developers.

## Impact

No upgrade or configuration changes required.

## Technical details

When AJAX commands open or update dialogs, Drupal needs to move keyboard focus into the dialog for accessibility. Previously this relied on jQuery UI Dialog's internal `_focusedElement` and `_focusTabbable()` methods.

The new implementation uses vanilla JavaScript and Drupal's existing `focusable()` utility (which wraps the tabbable library) to replicate the exact same focus behavior. It follows a priority order: first try elements with the `autofocus` attribute, then the first focusable element in the dialog content, then focusable buttons in the button pane, then the close button, and finally the dialog container itself.

This change only affects `dialog.ajax.js`, which handles AJAX responses that manipulate dialogs. The jQuery UI Dialog widget itself remains in place for now. The focus logic is now entirely self-contained instead of calling into jQuery UI's private API, making it easier to eventually swap in a native HTML `<dialog>` element or alternative widget without changing the AJAX command layer.
