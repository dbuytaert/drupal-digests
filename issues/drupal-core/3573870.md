## Summary

Drupal 12 removes code from the User and Basic Auth modules that was deprecated in Drupal 10.3 through 11.3. This cleanup includes the old user authentication interface, username validation functions, and several route providers and theme hooks. Sites that have already updated their code to use the replacement APIs will see no changes. This is part of the routine process of removing deprecated code before a major version release.

## Impact

No upgrade or configuration changes required.

## Technical details

The removed code includes the `UserAuthInterface` and `UserAuth` class, replaced by `UserAuthenticationInterface` and `UserAuthentication` in 10.3. The `user_validate_name()` function was replaced by `UserNameValidator::validateName()`. The `user_is_blocked()` function was replaced by `UserInterface::isBlocked()`. The `EntityPermissionsRouteProviderWithCheck` class and `EntityPermissionsForm::access()` method were replaced by standard permission checks in route definitions.

Several deprecated preprocess functions (`template_preprocess_username()` and `template_preprocess_user()`) were removed in favor of direct `hook_theme()` registrations. Constructor parameters across multiple classes were updated to require the new interfaces rather than accept union types for backward compatibility.

The test module `user_auth_decorator_test` and its associated tests were removed, as they existed only to verify the deprecated BC layer worked correctly. The change also removed several Views plugin constructors that accepted optional parameters with fallbacks to `\Drupal::service()` calls.

The Basic Auth module had deprecation trigger code removed from its constructor, where it previously checked if the injected service implemented the old interface.
