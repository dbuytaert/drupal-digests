## Summary

PHPUnit 12.5 began emitting notices when mock objects were created without any configured expectations, and deprecated the `any()` expectation matcher entirely. This change updates the unit tests in Drupal's user module to suppress those notices and align with PHPUnit 12.5 best practices. The fix is purely internal to the test suite and has no effect on production behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 emits a notice for every `createMock()` object that has no expectations configured via `expects()`, and PHPUnit 12.5.5 deprecates `$this->any()` because it is semantically identical to having no expectation at all. The combination means that the pattern `$obj->expects($this->any())->method(…)->willReturn(…)` produces both kinds of noise.

The fix across `core/modules/user/tests/src/Unit/` covers twelve test files:

**`createMock` to `createStub` conversions.** Wherever a doubled object only needed to return a canned value with no call-count assertion, `createMock(SomeInterface::class)` was replaced with `createStub(SomeInterface::class)`. `createStub` is PHPUnit's dedicated API for objects that carry no expectations. Affected files include `UserLocalTasksTest.php`, `PermissionAccessCheckTest.php`, `PermissionHandlerTest.php`, `RoleUserTestBase.php`, `ProtectedUserFieldConstraintValidatorTest.php`, `PermissionsTest.php`, `UserBulkFormTest.php`, `UserAccessControlHandlerTest.php`, and `RolesRidTest.php`. String class names passed to `createMock()` were also modernized to `ClassName::class` constants, and corresponding `use` statements were added.

**`expects($this->any())` removal.** In cases where a mock needed to stay as a mock (because it retained other `expects()` calls elsewhere in the same test), the `expects($this->any())` line was simply dropped from individual `->method()` chains, leaving `willReturn()` in place. Examples appear in `AddRoleUserTest.php`, `RemoveRoleUserTest.php`, and `ProtectedUserFieldConstraintValidatorTest.php`.

**`getMockBuilder` with `disableOriginalConstructor` converted to `createStub`.** Several tests used the more verbose `getMockBuilder(…)->disableOriginalConstructor()->getMock()` pattern purely to avoid instantiation. These were collapsed to `createStub()`, which also skips the constructor by default. Affected: `UserBulkFormTest.php` for `ViewsData`, `ViewExecutable`, and `DisplayPluginBase`; `UserAccessControlHandlerTest.php` for `FieldItemList`.

**`UserAuthTest.php` (now `UserAuthenticationTest.php`).** The most complex change. The shared `$testUser` property (a mock of `Drupal\user\Entity\User` built with `getMockBuilder`) was removed. Tests that need to assert call counts on `id()`, `setPassword()`, or `save()` now create a per-test `$testUser` via `createPartialMock(User::class, ['id', 'setPassword', 'save', 'getPassword'])`, keeping precise expectations only where the test genuinely cares about them. Tests that only verify a failed password check use `$this->createStub(User::class)` directly inline. Two tests that previously bypassed the `Cookie` provider by building it with `getMockBuilder(Cookie::class)->onlyMethods([])->getMock()` were refactored to instantiate `Cookie` directly with stubs for `SessionConfigurationInterface`, `Connection`, and `MessengerInterface`, making it clear that no real Cookie behavior is being bypassed.

**`UserTest.php`** (in `Plugin/Core/Entity`) changed its parent class from `Drupal\Tests\Core\Session\UserSessionTest` to `UnitTestCase`, and tightened expectations on `id()` from `$this->any()` to `$this->once()` and on `get('roles')` from `$this->any()` to `$this->atLeastOnce()`.

**`PermissionHandlerTest.php`** gained a missing `$this->callableResolver->expects($this->never())->method('getCallableFromDefinition')` assertion in `testBuildPermissionsSortPerModule()`, which had previously been unchecked.
