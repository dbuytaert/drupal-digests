## Summary

Drupal core now tests against modern browsers using W3C-compliant webdriver commands with the `selenium/standalone-chrome:latest` image. This ensures Drupal is tested on up-to-date browsers without maintaining a separate chromedriver image that quickly becomes outdated. The change enables future browser testing improvements and allows sites to run JavaScript tests using the same modern browser standards that Drupal core uses. Both W3C and legacy non-W3C testing modes are supported during the transition period.

## Impact

Affects module maintainers and site owners.

- **Config change:** `MINK_DRIVER_ARGS_WEBDRIVER` environment variable now requires `"w3c": true` in options to enable W3C mode for WebDriverTestBase tests.
- **Deprecation:** `\Drupal\FunctionalJavascriptTests\WebDriverCurlService` class deprecated with no replacement—use the base class instead.
- **New API:** New `DRUPAL_TEST_WEBDRIVER_W3C` environment variable enables W3C-compliant webdriver testing for Nightwatch tests.

## Technical details

The core test infrastructure now uses `selenium/standalone-chrome:latest` instead of the custom `drupalci/chromedriver` image for all functional JavaScript tests. This shift enables W3C webdriver protocol compliance, which is the modern standard for browser automation.

The GitLab CI configuration was updated to use Selenium containers with W3C mode enabled via the `"w3c": true` capability in `MINK_DRIVER_ARGS_WEBDRIVER`. Nightwatch tests enable W3C mode by setting `DRUPAL_TEST_WEBDRIVER_W3C=true`. The `DrupalSelenium2Driver` class no longer needs to override `setCookie()` or inject a custom curl service, simplifying the codebase.

Legacy WebDriver methods were replaced throughout the test suite. Alert handling changed from `getAlert_text()` and `accept_alert()` to `alert()->getText()` and `alert()->accept()`. Mouse movement code that used `moveto()` directly now uses `mouseOver()` with XPath selectors. The deprecated `WebDriverCurlService` class, which worked around race conditions in the old protocol, is no longer needed.

Core maintains a limited non-W3C test job during the deprecation period to ensure backward compatibility. The Selenium container configuration includes `SE_NODE_MAX_SESSIONS: "5"` to control concurrency and avoid stampede issues. Test parallelism increased from 2 to 4 runners for functional JavaScript tests to maintain similar pipeline times despite slightly slower Selenium execution.

## Upgrade

To enable W3C-compliant webdriver testing in your own environment, update your test configuration:

For WebDriverTestBase tests, set the `MINK_DRIVER_ARGS_WEBDRIVER` environment variable with `"w3c": true`:

```bash
MINK_DRIVER_ARGS_WEBDRIVER='["chrome", {"browserName":"chrome", "goog:chromeOptions":{"w3c": true, "args":["--no-sandbox","--headless"]}}, "http://selenium:4444"]'
```

For Nightwatch tests, add to your `core/.env` file:

```bash
DRUPAL_TEST_WEBDRIVER_W3C=true
DRUPAL_TEST_WEBDRIVER_HOSTNAME='selenium'
DRUPAL_TEST_WEBDRIVER_PORT='4444'
```

If you use `selenium/standalone-chrome:latest` as your browser container, these settings enable modern W3C webdriver testing. The drupalci/chromedriver image still works with W3C disabled for backward compatibility.

If your tests extend `LayoutBuilderDisableInteractionsTest::movePointerTo()`, replace calls with `$this->getSession()->getDriver()->mouseOver()` as the helper method is deprecated.

## Lessons learned

- When migrating testing infrastructure to new standards, run tests in both legacy and new modes during a transition period before deprecating the old mode. This allows projects to test compatibility while maintaining stability. *(via alexpott, catch)*
- When tests skip silently due to missing dependencies (like browser drivers), the lack of visible failure messages can mask infrastructure problems. Always check test reports for skipped tests to distinguish between intentional skips and configuration failures. *(via alexpott, andypost)*
- When switching to slower but more accurate testing infrastructure, increasing test parallelism (more runners) can maintain acceptable CI times while gaining the benefits of testing against production-like environments. *(via alexpott, catch)*
