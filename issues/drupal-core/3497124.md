## Summary

Drupal's homegrown deprecation testing mechanism, which was introduced when Drupal moved to PHPUnit 10 and dropped the Symfony PHPUnit bridge, has been deprecated and superseded by PHPUnit's own built-in deprecation assertion methods. Tests that previously called `expectDeprecation()` or used `ExpectDeprecationTrait` should now use PHPUnit's `expectUserDeprecationMessage()` or `expectUserDeprecationMessageMatches()` directly. This aligns Drupal's test infrastructure with standard PHPUnit conventions. A follow-up issue ([#3552827](https://www.drupal.org/node/3552827)) handles updating all remaining `expectDeprecation()` calls in core tests.

## Impact

Affects module maintainers.

- **Deprecation:** `ExpectDeprecationTrait` and its `expectDeprecation()` method deprecated in drupal:11.4.0, removed in drupal:12.0.0; use PHPUnit's `expectUserDeprecationMessage()` or `expectUserDeprecationMessageMatches()` instead

## Technical details

When Drupal adopted PHPUnit 10 and dropped `symfony/phpunit-bridge`, it built its own `ExpectDeprecationTrait` and a two-tier error handler system (`BootstrapErrorHandler` and `TestErrorHandler`) to replicate the Symfony bridge's deprecation-testing workflow. PHPUnit 11.0 introduced `expectUserDeprecationMessage()` and `expectUserDeprecationMessageMatches()` as native equivalents, making Drupal's custom implementation redundant.

**`TestErrorHandler` deleted.** The entire `core/tests/Drupal/TestTools/ErrorHandler/TestErrorHandler.php` class is removed. It had been swapped in per-test to capture deprecations into `DeprecationHandler`'s static arrays.

**`DeprecationHandler` stripped down.** The static `$expectedDeprecations` and `$collectedDeprecations` arrays and the methods that managed them (`reset()`, `expectDeprecation()`, `getExpectedDeprecations()`, `collectActualDeprecation()`, `getCollectedDeprecations()`, `isDeprecationTest()`, `isTestInLegacyGroup()`) are all removed from `core/tests/Drupal/TestTools/Extension/DeprecationBridge/DeprecationHandler.php`. The class now only handles the deprecation ignore list.

**`BootstrapErrorHandler` simplified.** The block in `__invoke()` that prefixed "Unsilenced deprecation: " and forwarded to `DeprecationHandler::collectActualDeprecation()` is removed; the handler now just consults the ignore list and delegates.

**`ExpectDeprecationTrait` deprecated.** The trait at `core/tests/Drupal/TestTools/Extension/DeprecationBridge/ExpectDeprecationTrait.php` now fires a file-level `@trigger_error()` on inclusion, marking itself deprecated in drupal:11.4.0, removed in drupal:12.0.0. The `setUpErrorHandler()` and `tearDownErrorHandler()` `#[Before]`/`#[After]` methods are removed. The `expectDeprecation()` method is kept as a shim: it fires its own deprecation and delegates to `expectUserDeprecationMessageMatches()` using a private `regularExpressionForFormatDescription()` helper that translates the format-string wildcards (`%A`, `%s`, etc.) used by the old Symfony bridge into a proper regex.

**`DrupalTestCaseTrait` added.** A new `core/tests/Drupal/Tests/DrupalTestCaseTrait.php` replaces the error-handler-checking responsibility previously held by `ExpectDeprecationTrait::tearDownErrorHandler()`. Its single `#[After]` method `checkErrorHandlerOnTearDown()` verifies the active handler is still a `BootstrapErrorHandler` after each test, catching any test that installs its own error handler without cleaning up. The trait is used by `UnitTestCase`, `BrowserTestBase`, `KernelTestBase`, and `BuildTestBase`.

**JavaScript deprecations rescheduled.** The code that replayed JavaScript deprecations from `sessionStorage` was previously in `WebDriverTestBase::tearDown()`. Because PHPUnit stops collecting deprecations during teardown (meaning `expectUserDeprecationMessage*()` cannot catch them there), the logic was moved: `DrupalSelenium2Driver` gains a public `processJavascriptDeprecations()` method and an `executeScript()` override that calls it immediately after each script execution, and `WebDriverTestBase` overrides `drupalGet()` to also call it after each page navigation.

**`#[Group('legacy')]` replaced.** Several core tests still used `#[Group('legacy')]` as a marker for deprecation tests; these are corrected to `#[IgnoreDeprecations]`, which is the actual PHPUnit attribute that tells PHPUnit to collect deprecations for the test. The `#[Group('legacy')]` annotation was a Symfony bridge convention that no longer has meaning.

**Known behavioral difference.** As documented in the change record, deprecations triggered inside `::tearDown()` or `#[After]` methods can no longer be captured with `expectUserDeprecationMessage*()`, because PHPUnit's internal deprecation collection stops during test finalization. This also blocked the issue for several months: PHPUnit had a separate bug (phpunit/phpunit[#6102](https://www.drupal.org/node/6102)) where `expectUserDeprecationMessage*()` failed in tests run in separate processes; this was fixed in PHPUnit 11.5.33 before the issue could proceed. Committing this change also required [#3537104](https://www.drupal.org/node/3537104) (Remove support for PHPUnit 10) to land first.

## Upgrade

If your custom module or test uses `ExpectDeprecationTrait` or calls `expectDeprecation()`, update as follows:

```php
// Before (deprecated in drupal:11.4.0, removed in drupal:12.0.0):
use Drupal\TestTools\Extension\DeprecationBridge\ExpectDeprecationTrait;

class MyTest extends TestCase {
  use ExpectDeprecationTrait;

  #[Group('legacy')]
  public function testSomething(): void {
    $this->expectDeprecation('Some message about a deprecation.');
    trigger_error('Some message about a deprecation.', E_USER_DEPRECATED);
  }
}

// After:
use PHPUnit\Framework\Attributes\IgnoreDeprecations;

class MyTest extends TestCase {
  // No trait needed; expectUserDeprecationMessage() is a native PHPUnit method.

  #[IgnoreDeprecations]
  public function testSomething(): void {
    $this->expectUserDeprecationMessage('Some message about a deprecation.');
    trigger_error('Some message about a deprecation.', E_USER_DEPRECATED);
  }
}
```

If you need substring or pattern matching, use `expectUserDeprecationMessageMatches('/regex/')` instead of `expectUserDeprecationMessage()`.

Note: deprecations triggered inside `::tearDown()` or `#[After]` methods can no longer be asserted with `expectUserDeprecationMessage*()`.
