## Summary

Sites installed with the Standard profile (or the standard recipe) no longer receive the "Article" and "Basic page" content types out of the box. A fresh Standard install now starts with no node types at all. Existing sites are unaffected; this only changes what a brand-new installation contains. The change is part of the broader effort ([#3159848](https://www.drupal.org/node/3159848)) to pare the Standard profile down to a generic starting point rather than a use-case-specific demo.

## Impact

Affects site owners.

- **Config change:** Standard profile and recipe no longer install Article or Basic page content types, and the content_editor role loses the corresponding create/edit/delete permissions on a fresh install.

## Technical details

The four core recipes that composed the Standard content-type setup (`article_content_type`, `page_content_type`, `article_tags`, and `article_comment`) are removed from `core/recipes/`. Their YAML and PHP files are deleted entirely from that directory. The standard recipe's `recipe.yml` no longer lists those four recipes as dependencies, so none of their config is shipped on install.

Alongside the content-type removal, the `content_editor` role loses the permissions that were specific to those types (`create article content`, `create page content`, `delete article revisions`, etc.), and the Tags vocabulary description is updated from "articles on similar topics" to "content on similar topics" to be type-agnostic.

Tests that previously relied on the `article` or `page` content types being present under Standard now call `drupalCreateContentType()` directly to create a local `test_content` type. The `StandardPerformanceTest` reflects a net reduction of three queries and two cache get/set operations per anonymous front-page request, because the article-specific fields (`node__comment`, `node__field_image`, `node__field_tags`) are no longer joined in and the `comment_entity_statistics` query is gone. The four removed recipes survive only as test fixtures under `core/tests/fixtures/recipes/` for internal use by recipe infrastructure tests.

## Upgrade

The four core recipes listed below no longer exist in `core/recipes/`. Any distribution or custom code that referenced them by name must be updated.

```yaml
# Before (these references will now fail):
recipes:
  - article_content_type
  - page_content_type
  - article_tags
  - article_comment

# After: copy the recipe definitions into your own distribution or module,
# or create the content types programmatically / via your own recipe.
```

No action is needed for existing installed sites. Only fresh Standard installs are affected.
