## Summary

Three functions originally intended to suppress contextual links during Views preview have been deprecated. These functions have not worked correctly for several years due to changes in how contextual links are rendered. Investigation revealed they never prevented contextual links from appearing in the preview. Since no contributed modules actively use them and they serve no purpose, they are being removed without replacement.

## Impact

Affects module maintainers.

- **Deprecation:** Three `views_ui` procedural functions (`views_ui_contextual_links_suppress()`, `views_ui_contextual_links_suppress_push()`, `views_ui_contextual_links_suppress_pop()`) deprecated with no replacement.

## Technical details

The `views_ui_contextual_links_suppress()`, `views_ui_contextual_links_suppress_push()`, and `views_ui_contextual_links_suppress_pop()` functions were deprecated as part of the effort to remove `drupal_static()` usage from core. Originally these functions were meant to suppress contextual links of entities within Views preview results.

During review, developers discovered these functions stopped working when contextual links switched from using `#contextual_links` render elements to `#contextual_links_placeholder`. The suppression check in `ViewsUiHooks::contextualLinksViewAlter()` was never called because the placeholder rendering path doesn't invoke the alter hook. Additionally, the push/pop calls wrapped around building the render array, but contextual links are rendered later during prerender, after suppression was already popped.

A search of contributed modules found only one module calling these functions, which has no stable release. The functions now only trigger deprecation warnings and no longer attempt to suppress contextual links. All related code has been removed from `ViewUI::renderPreview()` and the alter hook.

## Upgrade

Remove any calls to `views_ui_contextual_links_suppress()`, `views_ui_contextual_links_suppress_push()`, or `views_ui_contextual_links_suppress_pop()` from your code. There is no replacement, as these functions have not worked correctly for several years and contextual links cannot be suppressed this way.

### Collaboration

This issue was opened in 2019 as part of the effort to remove `drupal_static()` from core. After several early attempts to refactor the functions into a service, longwave investigated in comment #26 and discovered the code hadn't worked since contextual links rendering changed. This shifted the approach from refactoring to deprecating without replacement. The discovery that these functions were non-functional simplified the path forward, allowing the team to deprecate them cleanly rather than maintaining broken code. The issue took seven years from opening to commit, but the final resolution was straightforward once the non-functionality was confirmed.

## Lessons learned

- When deprecating broken functionality that no longer works as intended and has minimal contrib usage, deprecate with no replacement and schedule for removal in the next major version. Document why the code stopped working rather than attempting to fix unmaintained features. *(via longwave, nicxvan)*
