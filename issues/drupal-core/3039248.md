## Summary

Three `views_ui` procedural functions intended to suppress contextual links during the Views UI preview have been deprecated in Drupal 11.4.0 and will be removed in Drupal 12.0.0. Investigation confirmed the functions have not worked for years: contextual links in the Views preview still appear regardless. There is no replacement. Module authors should remove any calls to these functions.

## Impact

Affects module maintainers.

- **Deprecation:** Three `views_ui` procedural functions (`views_ui_contextual_links_suppress()`, `views_ui_contextual_links_suppress_push()`, `views_ui_contextual_links_suppress_pop()`) deprecated in `drupal:11.4.0` and removed in `drupal:12.0.0` with no replacement.

## Technical details

The three functions `views_ui_contextual_links_suppress()`, `views_ui_contextual_links_suppress_push()`, and `views_ui_contextual_links_suppress_pop()` in `core/modules/views_ui/views_ui.module` were originally designed to suppress contextual links on entities shown within a Views UI preview. `views_ui_contextual_links_suppress()` stored its state via `drupal_static()`, and `ViewUI::renderPreview()` wrapped the preview build with push/pop calls, while `ViewsUiHooks::contextualLinksViewAlter()` was supposed to clear `$element['#links']` when suppression was active.

Two reasons explain why the code stopped working. First, contextual links now use a `#contextual_links_placeholder` render element rather than `#contextual_links`. The placeholder rendering path does not invoke `hook_contextual_links_view_alter`, so the suppression check in `contextualLinksViewAlter()` is never reached. Second, even if the alter hook were invoked, the push/pop calls only wrap the render-array build phase; the prerender hooks run later, after suppression has already been popped.

A contrib search found only one module calling these functions and it has no stable release.

The commit removes the `views_ui_contextual_links_suppress()` guard block from `ViewsUiHooks::contextualLinksViewAlter()` (leaving only the `entityviewedit-form` link rewrite), removes the push/pop calls from `ViewUI::renderPreview()`, and converts the three functions in `views_ui.module` to stubs that only emit a `@trigger_error` deprecation notice. This issue is part of the broader [#1577902](https://www.drupal.org/node/1577902) effort to eliminate `drupal_static()` from Drupal core.

## Upgrade

Remove any calls to `views_ui_contextual_links_suppress()`, `views_ui_contextual_links_suppress_push()`, or `views_ui_contextual_links_suppress_pop()` from your code. There is no replacement, as these functions have not worked for years and contextual links in Views preview cannot be suppressed this way.

### Collaboration

The issue was opened in 2019 as part of the [#1577902](https://www.drupal.org/node/1577902) effort to remove `drupal_static()` from core. Early patches proposed a refactoring into a `ViewsUiContextualLinks` service with methods like `isSuppressed()` and `suppress()`. In 2021, longwave first noted the functions appeared broken, finding that the push/pop calls were reached but `contextualLinksViewAlter()` was never triggered. In 2026, longwave returned with a detailed analysis confirming the mechanism was fully broken: contextual links use `#contextual_links_placeholder` rather than `#contextual_links`, so the alter hook is never invoked on that path, and even if it were, the timing of push/pop versus prerender meant suppression could not work. nicxvan independently confirmed this and verified that only one contrib module (with no stable release) called the functions. This analysis shifted the resolution from a planned service refactor to a straightforward no-replacement deprecation.
