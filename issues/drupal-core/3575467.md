## Summary

Drupal has completed the modernization of theme engines by removing the old extension-based system. Theme engines like Twig are now implemented as tagged services rather than scanned from `.engine` files. This cleanup removes deprecated code that was replaced in Drupal 11.3, making the theme system simpler and more consistent with other Drupal subsystems. The change is purely internal and does not affect how themes work for site owners or developers.

## Impact

No upgrade or configuration changes required.

## Technical details

Theme engines were previously discovered by scanning for `.engine` files and treated as a separate extension type alongside modules, themes, and profiles. In Drupal 11.3, theme engines were converted to tagged services (see [#1685492](https://www.drupal.org/project/drupal/issues/1685492)), but the old system remained as a deprecated backward compatibility layer.

This issue removes that BC layer completely. The changes include removing `ThemeEngineExtensionList` and `InstallerThemeEngineExtensionList` classes, removing the `extension.list.theme_engine` service, removing the `.engine` file format, and cleaning up theme engine scanning throughout the codebase. The `ExtensionPathResolver` no longer needs to inject the theme engine list, and `ThemeExtensionList` no longer depends on it.

The `ActiveTheme::getOwner()` method is removed since theme engines are no longer extensions with file paths. Extension discovery no longer scans for `theme_engine` types. Theme-engine-specific preprocess function scanning was already removed in a prior issue, so this completes the removal of engine-as-extension concepts.

All references to the old `twig_render_template()`, `twig_theme()`, and `twig_extension()` functions are removed, with documentation updated to reference `TwigThemeEngine::renderTemplate()` instead. The test theme engine and related tests for the deprecated `.engine` file format are removed.
