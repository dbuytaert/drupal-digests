## Summary

Drupal has removed the legacy theme engine extension system that allowed themes to use different rendering engines. Theme engines are now exclusively implemented as tagged services rather than `.engine` files. This change completes the conversion started in earlier versions and removes all deprecated code related to scanning, loading, and processing `.engine` files. The Twig theme engine continues to work as before, but is now registered as a service instead of discovered as a file-based extension.

## Impact

No upgrade or configuration changes required.

## Technical details

Theme engines were previously treated as a special extension type alongside modules, themes, and profiles. They had their own discovery mechanism through `ThemeEngineExtensionList`, `.engine` files, and `.info.yml` files in the `core/themes/engines/` directory. This system has been fully removed.

The conversion removes `ThemeEngineExtensionList` and `InstallerThemeEngineExtensionList` classes entirely, along with the `extension.list.theme_engine` service. The `ExtensionPathResolver` no longer accepts or resolves theme engine paths. Theme engine discovery has been removed from `ExtensionDiscovery`.

The `ActiveTheme` class no longer stores or provides the `owner` property (which pointed to the theme engine's `.engine` file path). The `ThemeExtensionList` no longer depends on `ThemeEngineExtensionList` and uses autowiring for its dependencies. Theme initialization no longer includes `.engine` files.

All legacy Twig engine functions (`twig_theme()`, `twig_extension()`, `twig_render_template()`) and the `twig.engine` file have been removed. The theme registry and theme manager now call theme engine services directly through the tagged service system. The test theme engine and its associated tests have been removed.

Performance test assertions were updated to reflect reduced cache lookups now that theme engines are no longer scanned at runtime.
