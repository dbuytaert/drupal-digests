## Summary

The `drupal generate-theme` command would produce garbled file names and content when the starterkit theme's machine name was short enough to appear as a substring in its own derived naming variants. For example, generating a new theme named `galactus_ta` from a starterkit named `galactus` resulted in files named `galactusTaTa_ta.info.yml` instead of `galactus_ta.info.yml`, ultimately crashing with a PHP fatal error. This fix corrects the replacement logic so theme generation works correctly regardless of the starterkit's machine name length or complexity.

## Impact

No upgrade or configuration changes required.

## Technical details

The root cause was in `GenerateTheme::execute()`, which called PHP's `str_replace()` with two parallel arrays built by `namePatterns()`. That method produces eight variants per name (plain machine name, camelCase, PascalCase, and title case, for both machine name and label). PHP processes array-based `str_replace` sequentially, so when the starterkit name was short and lowercase (e.g. `galactus`), its camelCase variant was identical to the plain name. The first substitution correctly turned `galactus` into `galactus_ta`, but the very next pass then found `galactus` again inside the already-substituted string and transformed it into `galactusTa_ta`, corrupting every file name and piece of content in the generated theme.

The fix introduces a two-pass placeholder strategy in both the file-content loop and the file-rename loop. Pass 1 replaces every old pattern with a unique, collision-free sentinel string (`__MACHINE_NAME__`, `__MACHINE_NAME_CAMEL__`, etc.); Pass 2 replaces those sentinels with the new pattern values. Because no sentinel matches another old pattern, cascading substitutions cannot occur.

Alongside the two-pass change, the diff introduces explicit inline marker comments for starterkit template files: `#@starterkit:machine_name`, `#@starterkit:label`, `#@starterkit:machine_class_name`, and `#@starterkit:label_class_name`. A `preg_replace` pass after the generic substitution finds these markers and replaces the line immediately following each one with the correct new value, providing an unambiguous escape hatch when short or identical-looking label fragments would still be ambiguous after the placeholder pass. Existing test fixtures are updated from the old `# machine_name` comment style to the new `#@starterkit:` format. A new test `testSimpleStarterkitTheme()` in `GenerateThemeTest` covers the short-name scenario end-to-end using a starterkit named `simple`. The pattern collision was introduced in [#3364885](https://www.drupal.org/node/3364885), which added `namePatterns()` and the array-based `str_replace`.
