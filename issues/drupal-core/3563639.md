## Summary

Drupal's key-value store API gains a new `getAllKeys()` method on `KeyValueStoreInterface`. Developers working with large key-value collections can now retrieve only the collection's keys without pulling all values into memory, making chunked or batch-processing workflows practical where calling `getAll()` would previously cause memory exhaustion.

## Impact

Affects module maintainers.

- **New API:** New `getAllKeys(): iterable` method on `KeyValueStoreInterface` returns only collection keys without loading values.

## Technical details

The new `getAllKeys(): iterable` method is declared on `KeyValueStoreInterface` and implemented in four concrete classes. `DatabaseStorage` issues a lightweight `SELECT [name]` query against the key-value table, returning only key names. `DatabaseStorageExpirable` does the same but adds the `AND [expire] > :now` clause so only non-expired keys are returned. A null storage implementation returns an empty array. The abstract `StorageBase` base class receives a default implementation via `array_keys($this->getAll())`, which means any contrib or custom class that extends `StorageBase` automatically satisfies the interface without requiring code changes. Adding a method to an interface is permitted under Drupal's BC policy when a base class covers the default; catch requested the `StorageBase` implementation specifically for this compatibility reason. `EntitySchemaTest` was updated to use `getAllKeys()` in place of `getAll()` followed by `array_keys()`, demonstrating the intended usage. Related issue [#3576212](https://www.drupal.org/node/3576212) (unbuffered MySQL queries) was noted as a complementary approach to the same memory-pressure problem, though that is considered far from a general recommendation.

## Upgrade

Any class that **directly implements** `KeyValueStoreInterface` (without extending `StorageBase`) must add the new method:

```php
public function getAllKeys(): iterable {
    // Return an iterable of key strings for this collection.
    return array_keys($this->getAll());
}
```

Classes that extend `StorageBase` inherit this default implementation and require no changes.
