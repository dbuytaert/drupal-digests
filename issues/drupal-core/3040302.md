## Summary

Text fields with summary display have an "Edit summary" or "Hide summary" button that toggles the summary field visibility. Previously, the markup for this button was hardcoded in JavaScript and could not be customized by themes or modules. This made it difficult to adjust the button's appearance or add accessibility features without patching core. The button markup is now generated through the themeable `Drupal.theme` system, allowing developers to override it using standard JavaScript theming patterns. This provides flexibility for sites with custom design systems or accessibility requirements.

## Impact

Affects module maintainers.

- **New API:** New `Drupal.theme.textEditSummaryButton(title)` JavaScript theme function allows themes and modules to override the markup of the Edit/Hide summary button on text_with_summary fields.

## Technical details

The text module's JavaScript previously created the Edit/Hide summary button markup using a hardcoded template literal in `core/modules/text/js/text.js`. The inline string included the wrapping span, button element, and CSS classes, but was not wrapped in a `Drupal.theme()` call.

The change extracts this markup into a new `Drupal.theme.textEditSummaryButton(title)` function registered via `$.extend(Drupal.theme, {...})` at the bottom of `text.js`. The `title` parameter receives the translated "Hide summary" or "Edit summary" text. The original construction site now calls `Drupal.theme('textEditSummaryButton', Drupal.t('Hide summary'))` instead of building the markup inline.

The default theme function returns the exact same markup as before, maintaining backward compatibility. Themes or modules can now override this function to customize the button markup, add extra classes, modify the structure, or enhance accessibility.

A new test module (`text_summary_test`) at `core/modules/text/tests/modules/text_summary_test/` demonstrates the override pattern: its `text-summary-override.js` library replaces `Drupal.theme.textEditSummaryButton` with a version that adds a `text-test-edit-link` class and a visually-hidden prefix span. The module attaches this library via `hook_page_attachments_alter()` implemented in `TextSummaryTestHooks`. The existing functional JavaScript test class `TextareaWithSummaryTest` gains a new `testSummaryButtonOverride()` method that enables this test module and asserts the custom class is present on the rendered button.

## Upgrade

To customize the Edit/Hide summary button markup, override the `Drupal.theme.textEditSummaryButton` function in your theme or module JavaScript:

```javascript
Drupal.theme.textEditSummaryButton = function (title) {
  return `<span class="field-edit-link custom-class">
    (<button type="button" class="link link-edit-summary">${title}</button>)
  </span>`;
};
```

The `title` parameter contains the translated "Hide summary" or "Edit summary" text. Your override should return an HTML string containing a button element that the text module can attach event handlers to. Ensure your library declares a dependency on `core/drupal.text` so the override loads after the default implementation.
