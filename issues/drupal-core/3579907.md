## Summary

This is a purely internal test housekeeping change. No production code or user-facing behavior is affected. The Entity library's unit tests are updated to be compatible with PHPUnit 12.5.0, which deprecates certain mock-object patterns that were previously common in the test suite.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces three breaking changes for test doubles. First, mock objects that configure no expectations emit a deprecation notice: objects that only stub return values should be created with `createStub()` rather than `createMock()`. Second, `$this->any()` is deprecated because it is equivalent to having no expectation at all. Third, `with()` is deprecated on stubs because argument matching is only meaningful on mocks that verify calls.

The commit touches 23 test files under `core/tests/Drupal/Tests/Core/Entity/` and the shared `UnitTestCase` base class. The main mechanical changes are:

- `createMock()` calls whose resulting object never had `expects(...)` are replaced with `createStub()`. Type hints and `@var` docblocks are updated from `MockObject` to `Stub` accordingly.
- `$this->any()` expectations are removed. Where the object genuinely needs call-count verification (e.g., `$this->fieldTypePluginManager`), the constraint is tightened to `$this->atLeastOnce()` or `$this->exactly(N)`.
- `with()` matchers are dropped from stub chains, since they have no effect on stubs.
- In `ContentEntityBaseUnitTest`, the shared `setUp()` method previously configured several collaborators (language manager, entity type, typed data manager) as fully-specified mocks with `any()` chains. Because individual tests override those configurations, the setup is refactored into opt-in helper methods (`setUpMockEntityType()`, `setUpMockLanguageManager()`, `setUpMockTypedDataManager()`) that tests call explicitly when they need the richer mock rather than a plain stub.
- `UnitTestCase::setupMockIterator()` is updated to accept and return `Stub&\Iterator` instead of `MockObject&\Iterator`, since iterator stubs have no expectations.
- 126 PHPStan baseline suppressions for the now-resolved `any()` violations across the Entity test files are removed from `core/.phpstan-baseline.php`.
