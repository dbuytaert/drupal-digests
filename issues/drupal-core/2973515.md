## Summary

When visiting a page whose only matching menu link was disabled, Drupal would still mark ancestor menu items with an active-trail CSS class and could generate incorrect breadcrumbs in modules like menu_breadcrumb. This has been fixed: disabled menu links are now excluded when determining the active menu trail. Sites that deliberately used disabled menu links as breadcrumb anchors (a pattern noted by at least one large production site) will be affected and should consult the change record for this issue.

## Impact

No upgrade or configuration changes required.

## Technical details

The `MenuActiveTrail` service in `core/lib/Drupal/Core/Menu/MenuActiveTrail.php` computes the active trail for the current request via `getActiveLink()`. Before this fix, that method called `reset($links)` to blindly return the first route-matching link from the menu link manager, without consulting `MenuLinkInterface::isEnabled()`. A disabled link was therefore treated identically to an enabled one, causing its ancestors to receive active-trail treatment.

The fix replaces the `reset()` call with a `foreach` loop that calls `$link->isEnabled()` on each candidate and returns the first enabled match, or `NULL` if no enabled match exists. When `NULL` is returned the caller receives an empty active trail, so no ancestor link is marked active and no spurious breadcrumb segment is generated.

Two new data-provider cases were added to `core/tests/Drupal/Tests/Core/Menu/MenuActiveTrailTest.php`: one confirms that a disabled intermediate (parent) link is skipped and the enabled child is still surfaced; the other confirms that when the only matching link is disabled, `getActiveLink()` returns `NULL` and the active trail is empty.

The change was committed to both `main` and `11.x` branches. A change record was added because at least one known usage pattern relies on the previous behavior: placing 1500+ disabled links in a menu so that menu_breadcrumb can match disabled links to their enabled parent items. After discussion, the decision was made not to provide a BC layer or deprecated sub-module to restore the old behavior; the change record is a heads-up only.

## Upgrade

No code changes are required for most sites. If your site deliberately placed disabled menu links in a menu so that menu_breadcrumb or similar modules would match them to an active trail, those disabled links will now be ignored. You will need to enable the relevant links, restructure your menu hierarchy, or find an alternative breadcrumb strategy.
