## Summary

This is round 5 of a multi-issue effort to add PHP return types and explicit property type declarations to Drupal core's Kernel test code, removing entries from the PHPStan baseline as types are properly declared. The bulk of the changes are test-only. One runtime class also changed: `LocaleDefaultConfigStorage` now accepts the broader `LanguageManagerInterface` as its `$language_manager` constructor argument instead of the narrower `ConfigurableLanguageManagerInterface`.

## Impact

Affects module maintainers.

- **API change:** `LocaleDefaultConfigStorage::__construct()` second parameter type widened from `ConfigurableLanguageManagerInterface` to `LanguageManagerInterface`; existing callers require no changes.

## Technical details

The primary work is typing the `core/tests/` Kernel test layer. `KernelTestBase` received typed property declarations (`?ClassLoader $classLoader`, `?ContainerBuilder $container`, `?ConfigImporter $configImporter`, `?vfsStreamDirectory $vfsRoot`, `bool $strictConfigSchema`, `static array $configSchemaCheckerExclusions`, etc.) and explicit `void`/typed return types on its many helper methods (`bootEnvironment()`, `setUpFilesystem()`, `bootKernel()`, `config()`, `getDatabaseConnectionInfo()`, `enableModules()`, `disableModules()`, `render()`, and others). `AssertContentTrait` likewise had all its properties typed and methods annotated with return types. Several individual test classes had their locally-duplicated `protected $configImporter` declarations removed since `KernelTestBase` already declares it. Abstract method contracts in test base classes (`GenericCacheBackendUnitTestBase`, `ConfigStorageTestBase`) now carry explicit return types.

The one non-test change is in `core/modules/locale/src/LocaleDefaultConfigStorage`. Its constructor parameter `$language_manager` was typed as `ConfigurableLanguageManagerInterface`, but the only method the class calls on it is `getStandardLanguageList()`, which lives on the parent `LanguageManagerInterface`. PHPStan flagged a baseline entry because `\Drupal::languageManager()` declares a return type of `LanguageManagerInterface`. The fix widens the constructor type to `LanguageManagerInterface`, which is backward compatible and removes the need for the `assertInstanceOf` narrowing workaround in the locale test.

Additional minor fixes in test code include initializing variables before loop-entry to satisfy PHPStan (`$found = FALSE`, `$expected_driver = NULL`, `$new_field_schema_data = []`), adding `assert(isset($e))` before accessing an exception variable in a `finally` block, casting `$user->passRaw` to `(string)` where a string argument was expected, fixing a PHP error-handler callback to return `bool` (required by PHP's error handler contract), and replacing some `assertSame` calls with `assertEquals` where object identity could not be guaranteed.

## Upgrade

If your code constructs `LocaleDefaultConfigStorage` directly, the second constructor argument has been widened:

```php
// Before: required ConfigurableLanguageManagerInterface
new LocaleDefaultConfigStorage($storage, $configurableLanguageManager, $profile);

// After: accepts any LanguageManagerInterface (existing code is unaffected)
new LocaleDefaultConfigStorage($storage, $languageManager, $profile);
```

No changes are required for existing callers; the widened type is fully backward compatible.
