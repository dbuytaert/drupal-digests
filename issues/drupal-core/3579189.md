## Summary

This is round 5 of an ongoing series adding return type declarations and fixing PHPStan baseline suppression entries across Drupal core's test code. It clears a large batch of baseline entries that were suppressing static analysis warnings in Kernel test classes, and resolves the underlying issues by adding explicit type hints, initializing variables before use, and tightening a few assertions. One runtime class was also updated: `LocaleDefaultConfigStorage` now accepts the broader `LanguageManagerInterface` instead of `ConfigurableLanguageManagerInterface` for its language manager dependency, since only methods on the parent interface are actually used.

## Impact

Affects module maintainers.

- **API change:** `LocaleDefaultConfigStorage::__construct()` `$language_manager` parameter type widened from `ConfigurableLanguageManagerInterface` to `LanguageManagerInterface`; `AssertContentTrait` and `KernelTestBase` methods gain explicit return types that subclasses overriding these methods must now match.

## Technical details

The bulk of the diff removes `$ignoreErrors[]` entries from `core/.phpstan-baseline.php` that were suppressing `missingType.return`, `argument.type`, `variable.undefined`, and related identifiers across dozens of files in `core/tests/Drupal/KernelTests/` and `core/tests/Drupal/Tests/`. The corresponding fixes land in the test files themselves.

Key test infrastructure changes: `AssertContentTrait` gains explicit PHP return type declarations on `getRawContent(): string`, `setRawContent(string|\Stringable $content): void`, `getTextContent(): string`, `removeWhiteSpace(): void`, `getDrupalSettings(): array`, `setDrupalSettings(array $settings): void`, `buildXPathQuery(string $xpath, array $args = []): string`, `xpath(...): array|false`, `cssSelect(string $selector): array`, `assertTitle(...): void`, and `assertThemeOutput(...): void`. `KernelTestBase` gains `void` return types on many lifecycle helpers such as `bootEnvironment()`, `bootKernel()`, `enableModules()`, `disableModules()`, `installConfig()`, `installSchema()`, `installEntitySchema()`, `copyConfig()`, and others. `GenericCacheBackendUnitTestBase` changes `$cacheBackends` to `protected ?array $cacheBackends = NULL` and replaces `unset($this->cacheBackends)` with `$this->cacheBackends = NULL` to satisfy PHPStan's `unset.possiblyHookedProperty` rule.

The one runtime change is in `core/modules/locale/src/LocaleDefaultConfigStorage.php`: the constructor's `$language_manager` parameter type was narrowed from `ConfigurableLanguageManagerInterface` to `LanguageManagerInterface` (the parent interface), because only `getStandardLanguageList()` is called on the stored property, and that method lives on the parent interface. This widens what callers may pass, removing a baseline entry for `LocaleDefaultConfigStorageTest`.

Variable initialization fixes include `$found = FALSE` before a `foreach` in `ApcuBackendTest`, `$expected_driver = NULL` before a loop in `DriverSpecificKernelTestBase`, and similar patterns. A handful of `assertSame()` calls in `ConfigStorageTestBase` were replaced with `assertEquals()` because `assertSame()` raised `argument.unresolvableType` from PHPStan given the abstract return type of `read()`.

## Upgrade

If you have a custom class that extends `LocaleDefaultConfigStorage` and overrides `__construct()`, you should update the `$language_manager` parameter type from `ConfigurableLanguageManagerInterface` to `LanguageManagerInterface`:

```php
// Before
public function __construct(StorageInterface $config_storage, ConfigurableLanguageManagerInterface $language_manager, string $install_profile) { ... }

// After
public function __construct(StorageInterface $config_storage, LanguageManagerInterface $language_manager, string $install_profile) { ... }
```

If you have custom test classes that override methods from `AssertContentTrait` or `KernelTestBase`, you must ensure your method signatures match the newly declared return types. For example:

```php
// Before (no return type, now causes a signature mismatch)
protected function getRawContent() { ... }

// After
protected function getRawContent(): string { ... }
```
