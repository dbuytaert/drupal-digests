## Summary

Layout Builder checks all section storage plugins when rendering entities to determine if layouts are enabled. A previous performance improvement ([#3060985](https://www.drupal.org/project/drupal/issues/3060985)) added a way for plugins to quickly indicate they do not support certain entities. However, the Navigation module did not implement this interface, which undid the performance gains whenever Navigation was enabled. This change makes Navigation and test plugins implement the interface and deprecates not implementing it, restoring the performance improvement.

## Impact

Affects module maintainers.

- **Deprecation:** Section storage plugins not implementing `\Drupal\layout_builder\SupportAwareSectionStorageInterface` now trigger a deprecation warning; required in Drupal 13.

## Technical details

The `SupportAwareSectionStorageInterface` was introduced in [#3060985](https://www.drupal.org/project/drupal/issues/3060985) to let section storage plugins quickly declare whether they support a given entity type, bundle, and view mode combination. This avoids expensive context checks when Layout Builder is not actually enabled for the entity being rendered.

The `SectionStorageManager::notSupported()` method now explicitly checks if each section storage plugin implements the interface. If not, it triggers a deprecation warning and returns `FALSE` (assumes the plugin might be supported, taking the safer but slower path).

The Navigation module's `NavigationSectionStorage` plugin now implements the interface and returns `FALSE` from `isSupported()` because navigation layouts are not used for entity rendering. Test plugins `SimpleConfigSectionStorage` and `TestStateBasedSectionStorage` also implement the interface with appropriate logic.

Performance test expectations are updated to reflect the restored optimization: fewer database queries and cache operations occur because Layout Builder correctly skips checking the navigation section storage when rendering entities.

## Upgrade

If you maintain a custom section storage plugin, implement `\Drupal\layout_builder\SupportAwareSectionStorageInterface` and add the `isSupported()` method. Return `TRUE` if your plugin supports the given entity type, bundle, and view mode; return `FALSE` otherwise.

Example from `NavigationSectionStorage`:

```php
public function isSupported(string $entity_type_id, string $bundle, string $view_mode): bool {
  // Navigation section storage does not support any entity type.
  return FALSE;
}
```
