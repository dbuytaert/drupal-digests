## Summary

Layout Builder had a performance optimization from [#3060985](https://www.drupal.org/project/drupal/issues/3060985) that prevented unnecessary work when checking if Layout Builder is enabled for an entity. However, the optimization only worked when all section storage plugins properly implemented a support-checking interface. The Navigation module's section storage did not implement this interface, which caused the optimization to fail completely whenever Navigation was enabled. This change adds the interface to Navigation and deprecates section storage plugins that do not implement it, restoring the performance improvements.

## Impact

Affects module maintainers.

- **Deprecation:** Section storage plugins not implementing `\Drupal\layout_builder\SupportAwareSectionStorageInterface` is deprecated in drupal:11.4.0 and required from drupal:13.0.0

## Technical details

The `SectionStorageManager::notSupported()` method checks whether Layout Builder should be skipped for a given entity type, bundle, and view mode. A previous optimization introduced `SupportAwareSectionStorageInterface` to allow section storage plugins to declare whether they support specific contexts without expensive setup work.

The original logic short-circuited if *any* plugin either didn't implement the interface or returned TRUE from `isSupported()`. This meant that a single plugin not implementing the interface would cause the optimization to fail for all requests, triggering expensive context resolution unnecessarily.

The fix splits the conditional logic. Now if a plugin doesn't implement the interface, a deprecation error is triggered and the method returns FALSE (not supported), but this only affects that specific check. The Navigation module's `NavigationSectionStorage` now implements the interface and returns FALSE from `isSupported()` since it does not support entity-based layouts.

Test section storage plugins (`SimpleConfigSectionStorage`, `TestStateBasedSectionStorage`) also received the interface to avoid triggering deprecations during testing. Performance test expectations were updated to reflect the restored optimization, showing fewer queries and cache operations.

## Upgrade

If you maintain a custom section storage plugin, implement `\Drupal\layout_builder\SupportAwareSectionStorageInterface` and add the `isSupported()` method:

```php
use Drupal\layout_builder\SupportAwareSectionStorageInterface;

class MyCustomSectionStorage extends PluginBase implements SectionStorageInterface, SupportAwareSectionStorageInterface {
  
  public function isSupported(string $entity_type_id, string $bundle, string $view_mode): bool {
    // Return TRUE if this storage applies to the given context.
    // Return FALSE if it does not (e.g., Navigation returns FALSE for all entity types).
    return FALSE;
  }
}
```

See [#3574738](https://www.drupal.org/node/3574738) for full details.
