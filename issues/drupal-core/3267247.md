## Summary

This is an internal test infrastructure meta issue with no user-facing effect. A batch of JavaScript functional tests were temporarily skipped in [#3267124](https://www.drupal.org/node/3267124) after random failures began surfacing, many triggered by a ChromeDriver update that exposed pre-existing race conditions in browser-based tests. This meta issue tracked the systematic effort to debug each skipped test, measure its baseline failure rate, apply fixes, and re-enable it. All 15+ child issues tracked here are now closed.

## Impact

No upgrade or configuration changes required.

## Technical details

The problem was a cluster of intermittently failing `FunctionalJavascript` tests spread across Layout Builder (`LayoutBuilderUiTest`, `LayoutBuilderTest`, `LayoutBuilderDisableInteractionsTest`, `LayoutBuilderNestedFormUiTest`, `MoveBlockFormTest`, `AjaxBlockTest`, `ContentPreviewToggleTest`, `BlockFormMessagesTest`, `ContextualLinksTest`), QuickEdit (`QuickEditIntegrationTest`), CKEditor5 (`MediaLibraryTest`), Media (`MediaSourceFileTest`), Settings Tray (`SettingsTrayBlockFormTest`), and Media Library (`EntityReferenceWidgetTest`). The root cause in the majority of cases was race conditions: tests were not using the correct Mink/WebDriver waiting APIs to block until async UI state (dialog opening, off-canvas tray animation, AJAX responses) had settled before making assertions.

The resolution strategy was: (1) open a child issue per test, (2) submit a patch that merely un-skips the test and run it 500–1000 times against MySQL/MariaDB to get a baseline fail rate, (3) submit the actual fix alongside baseline runs so the improvement is measurable, and (4) skip other methods in the same test class if they also became unstable. The fix for `LayoutBuilderDisableInteractionsTest::testFormsLinksDisabled` ([#3268680](https://www.drupal.org/node/3268680)) introduced a synchronization helper for the off-canvas area that was then reused to stabilize other Layout Builder tests.

In the current `main` branch, `MoveBlockFormTest::setUp()` and `LayoutBuilderUiTest::setUp()` (for the `testAddHighlights` method) still carry `markTestSkipped("Skipped temporarily for random fails.")` calls, indicating those two cases were not yet fully resolved before the meta was closed.
