## Summary

Drupal 11.5 deprecates a set of methods on the `\Drupal\Core\Htmx\Htmx` builder class and the `HtmxRequestInfoTrait` trait. These methods correspond to features that htmx v4 removes or restructures, and they will be removed entirely in Drupal 12.0. Modules or custom code that call any of the deprecated methods will receive PHP deprecation notices on Drupal 11.5 and will break on Drupal 12.

## Impact

Affects module maintainers.

- **Deprecation:** Eleven methods across \Drupal\Core\Htmx\Htmx and HtmxRequestInfoTrait deprecated in drupal:11.5.0 and removed from drupal:12.0.0; triggerAfterSettleHeader() and triggerAfterSwapHeader() replaced by triggerHeader(), the rest have no PHP-level replacement.

## Technical details

The commit adds `@trigger_error()` calls with `E_USER_DEPRECATED` and corresponding `@deprecated` PHPDoc blocks to eleven methods across two classes in `core/lib/Drupal/Core/Htmx/`.

In `\Drupal\Core\Htmx\Htmx`: `triggerAfterSettleHeader()` and `triggerAfterSwapHeader()` are deprecated in favour of `triggerHeader()`. The methods `disabledElt()`, `disinherit()`, `ext()`, `history()`, `inherit()`, `params()`, `prompt()`, and `request()` are deprecated with no direct PHP replacement (the change record at https://www.drupal.org/node/3583674 documents the htmx-v4-compatible approaches). `request()` is notable because the underlying `hx-request` attribute was removed outright in htmx v4.

In `\Drupal\Core\Htmx\HtmxRequestInfoTrait`: `getHtmxPrompt()` and `getHtmxTrigger()` are deprecated with no replacement, because htmx v4 no longer sends the corresponding `HX-Prompt` and `HX-Trigger` headers in the same way.

All deprecations target `drupal:11.5.0` and removal at `drupal:12.0.0`. Existing unit tests in `HtmxTest`, `HtmxHeaderTest`, and `HtmxRequestInfoTest` were updated to use `#[IgnoreDeprecations]` and `expectUserDeprecationMessage()` so they continue to exercise the deprecated code paths without failing. This work is a prerequisite for [#3555916](https://www.drupal.org/node/3555916), which will upgrade Drupal core to htmx v4 on the `main` (Drupal 12) branch and remove these methods.

## Upgrade

Replace deprecated `Htmx` method calls as follows:

```php
// Before (deprecated in 11.5, removed in 12.0):
$htmx->triggerAfterSettleHeader('myEvent');
$htmx->triggerAfterSwapHeader('myEvent');

// After:
$htmx->triggerHeader('myEvent');
```

For `disabledElt()`, `disinherit()`, `ext()`, `history()`, `inherit()`, `params()`, `prompt()`, and `request()`, no equivalent method exists in the PHP API. Follow the htmx v4 migration guidance in the change record at https://www.drupal.org/node/3583674.

For `getHtmxPrompt()` and `getHtmxTrigger()` (from `HtmxRequestInfoTrait`): there is no replacement. Remove calls to these methods.

To write code that works on both Drupal 11 and 12, you can use `DeprecationHelper`:

```php
use Drupal\Component\Utility\DeprecationHelper;

DeprecationHelper::backwardsCompatibleCall(
  currentVersion: \Drupal::VERSION,
  deprecatedVersion: '11.5',
  currentCallable: fn() => /* new approach */,
  deprecatedCallable: fn() => $htmx->triggerAfterSettleHeader('myEvent'),
);
```
