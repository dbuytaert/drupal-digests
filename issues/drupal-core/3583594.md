## Summary

Two helper methods have been added to `HttpKernelUiHelperTrait` (`\Drupal\Tests`) to reduce friction when converting Browser tests to Kernel tests. `getNodeElementsByXpath()` performs an XPath search against the current page and returns Mink `NodeElement` objects, matching the behaviour of `BrowserTestBase::xpath()`. `getUrl()` returns the current URL from the Mink session. Both methods are now available in any Kernel test class that uses this trait, including `KernelTestBase` subclasses.

## Impact

Affects module maintainers.

- **New API:** Two methods added to `HttpKernelUiHelperTrait`: `getNodeElementsByXpath(string $xpath, array $arguments = []): array` returning `\Behat\Mink\Element\NodeElement[]`, and `getUrl(): string` returning the current Mink session URL.

## Technical details

Kernel tests already include `AssertContentTrait::xpath()` (via `KernelTestBase`), which returns `\SimpleXMLElement[]|false`. `BrowserTestBase::xpath()` returns `\Behat\Mink\Element\NodeElement[]`. Adding a second `xpath()` directly to `HttpKernelUiHelperTrait` causes a fatal PHP trait method collision in `KernelTestBase`. To avoid this, the Mink-based XPath helper is added under the distinct name `getNodeElementsByXpath(string $xpath, array $arguments = []): array`. It delegates to `$this->assertSession()->buildXPathQuery()` for argument substitution and then calls `$this->getSession()->getPage()->findAll('xpath', $xpath)`, exactly mirroring the `BrowserTestBase` implementation. `getUrl()` simply wraps `$this->getSession()->getCurrentUrl()`, matching the contract of `UiHelperTrait::getUrl()`. A follow-on issue ([#3585774](https://www.drupal.org/node/3585774)) is filed to standardise and disambiguate the names of all `xpath()` variants across the test suite.
