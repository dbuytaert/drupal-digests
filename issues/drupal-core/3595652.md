## Summary

The two procedural functions in `module.inc` that manage module weights, `module_set_weight()` and `module_config_sort()`, are deprecated in Drupal 11.5.0 and will be removed in Drupal 13.0.0. Their logic has been moved into a new `Drupal\Core\Extension\ModuleWeight` service, which provides equivalent `set()` and `sort()` methods. Modules that call either procedural function directly need to switch to the service before upgrading to Drupal 13.

## Impact

Affects module maintainers.

- **API change:** `ModuleInstaller::__construct()` gained a new nullable `?ModuleWeight $moduleWeight` parameter; it will be required in drupal:12.0.0.
- **Deprecation:** `module_set_weight()` and `module_config_sort()` in `module.inc` deprecated in drupal:11.5.0 and removed in drupal:13.0.0; replaced by `Drupal\Core\Extension\ModuleWeight::set()` and `::sort()`.
- **New API:** New `Drupal\Core\Extension\ModuleWeight` service with `set(string $module, int $weight): void` and `sort(array $data): array` methods.

## Technical details

The new `Drupal\Core\Extension\ModuleWeight` class lives in `core/lib/Drupal/Core/Extension/ModuleWeight.php` and is registered as an autowired service. It accepts `ModuleHandlerInterface` and `ConfigFactoryInterface` via constructor injection. `ModuleWeight::set(string $module, int $weight): void` replicates the logic previously in `module_set_weight()`, including writing to `core.extension` config and updating the live module list in the module handler. `ModuleWeight::sort(array $data): array` replicates the sign-prefix plus zero-padded integer plus name string-sort algorithm from `module_config_sort()`.

The procedural functions in `module.inc` now contain only a `@trigger_error` deprecation notice followed by a delegation call to `\Drupal::service(ModuleWeight::class)`. All internal callers across Drupal core (including `ModuleInstaller`, `ExcludedModulesEventSubscriber`, the installer's `install_finished()`, `content_translation.install`, `views.install`, and several test classes) have been updated to use the service directly.

`ModuleInstaller::__construct()` gained a new nullable `?ModuleWeight $moduleWeight` parameter at the end of its signature; a deprecation notice is triggered if the argument is absent when the class is constructed, and the argument will be required in Drupal 12.0.0. `ExcludedModulesEventSubscriber` similarly gained `ModuleWeight` as a new constructor argument and its service definition in `core.services.yml` was updated accordingly.

## Upgrade

Replace calls to `module_set_weight()` with the `ModuleWeight` service:

```php
// Before (deprecated since 11.5.0):
module_set_weight('mymodule', 10);

// After:
use Drupal\Core\Extension\ModuleWeight;
\Drupal::service(ModuleWeight::class)->set('mymodule', 10);
```

Replace calls to `module_config_sort()` with the `ModuleWeight` service:

```php
// Before (deprecated since 11.5.0):
$sorted = module_config_sort($modules);

// After:
use Drupal\Core\Extension\ModuleWeight;
$sorted = \Drupal::service(ModuleWeight::class)->sort($modules);
```

If you extend or directly instantiate `ModuleInstaller`, add `ModuleWeight` as the final constructor argument. See the change record at https://www.drupal.org/node/3595653.
