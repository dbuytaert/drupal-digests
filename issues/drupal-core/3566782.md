## Summary

Drupal automatically copies block placements from the default theme when a new theme is installed. This behavior is triggered by a procedural function that existed in the block module file. As part of ongoing work to eliminate module files in core, this function has been deprecated and its logic moved into the block module's hook class. The function will be removed in Drupal 13. This change makes it easier to potentially remove this automatic block copying behavior in the future if core maintainers decide it is no longer needed.

## Impact

Affects module maintainers.

- **Deprecation:** `block_theme_initialize()` deprecated with no replacement; internal logic moved to protected method in `BlockHooks` class

## Technical details

The `block_theme_initialize()` function was a procedural function in `block.module` that copied block placements from the default theme to newly installed themes. This automatic copying behavior dates back to earlier versions of Drupal and has been questioned in related issues like [#3105597](https://www.drupal.org/project/drupal/issues/3105597).

This issue is part of the broader initiative tracked in [#3566536](https://www.drupal.org/project/drupal/issues/3566536) to eliminate `.module` files from core by moving procedural code into services and hook classes.

The implementation moves the function's logic into a new protected method `themeInitialize()` in the existing `BlockHooks` class. The original function remains but now triggers a deprecation error and is marked for removal in Drupal 13.0.0. Internal calls within `BlockHooks::modulesInstalled()` and `BlockHooks::themesInstalled()` now call the protected method directly. The API documentation for `hook_themes_installed()` was updated to remove the example usage of the deprecated function.

Because the replacement method is protected rather than a public service method, there is no direct replacement for external callers. This allows core to maintain the functionality internally while signaling that external code should not rely on it.

## Upgrade

If your code calls `block_theme_initialize()`, remove those calls. There is no direct replacement provided. The function triggered deprecation warnings starting in Drupal 11.4 and will be removed in Drupal 13.0.0.

If you need to initialize blocks for a theme programmatically, you will need to implement your own logic or reconsider whether this is necessary for your use case. The automatic copying of blocks between themes is under evaluation for potential removal.

See the change record at [https://www.drupal.org/node/3566783](https://www.drupal.org/node/3566783) for details.
