## Summary

This is a testing infrastructure change with no user-facing effect. It tracks the removal of all remaining uses of `withConsecutive()` in Drupal core unit tests, completing the migration away from a PHPUnit API that was deprecated in PHPUnit 9.6 and removed in PHPUnit 10. Each affected test was refactored to use cleaner assertion patterns, and the PHPStan baseline suppressions for the deprecated method were removed file by file as tests were fixed.

## Impact

No upgrade or configuration changes required.

## Technical details

Background: `PHPUnit\Framework\MockObject\Builder\InvocationMocker::withConsecutive()` was deprecated in PHPUnit 9.6 and removed in PHPUnit 10. As a bridge, Drupal added `MockTrait::consecutiveCalls()` in [#3306554](https://www.drupal.org/node/3306554) to mimic the removed API. That shim was always intended as a temporary stop-gap because `withConsecutive()` was considered code smell for tests that were over-complicated and mocking too much.

This META issue coordinated replacing all remaining direct and indirect uses of `withConsecutive()` across core unit tests. The files identified in the thread included `RouteBuilderTest.php`, `UrlTest.php`, `PrivateTempStoreTest.php`, `SharedTempStoreTest.php`, and `MigrateExecutableMemoryExceededTest.php`, plus a suppressed entry in `MigrateExecutableMemoryExceededTest.php` that PHPStan had not caught.

The approach taken in the committed child issue [#3419288](https://www.drupal.org/node/3419288) (`RouteBuilderTest.php`) illustrates the pattern used: the mock was switched from PHPUnit's `createMock()` to Prophecy's `prophesize()`, and the single chained `withConsecutive()` call with multiple argument sets was replaced by separate per-call Prophecy assertions using `shouldBeCalled()`. The `$dispatcher` property type annotation was updated to reflect `\Prophecy\Prophecy\ObjectProphecy`, and `->reveal()` was passed where the concrete interface was needed. Corresponding PHPStan baseline entries suppressing `Call to deprecated method withConsecutive()` were removed for each fixed test file.

The original plan also included formally deprecating `MockTrait::consecutiveCalls()`, but this was dropped before closure of the issue.
