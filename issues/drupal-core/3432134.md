## Summary

The Actions UI module (`action`) has been completely removed from Drupal 11 core and moved to a contributed module. Sites that had the module enabled need to either uninstall it before upgrading or install the standalone contributed `action` module to keep the admin interface for listing and configuring actions.

## Impact

Affects module maintainers.

- **Deprecation:** The entire `action` module (Actions UI) removed from core; UI and its form/plugin classes are no longer available. The `system` module still provides the Action entity type and plugin infrastructure.

## Technical details

The `core/modules/action` module provided the administrative UI for the action entity system: routes at `/admin/config/system/actions`, `ActionListBuilder`, form classes (`ActionFormBase`, `ActionAddForm`, `ActionEditForm`, `ActionDeleteForm`, `ActionAdminManageForm`), and three configurable action plugins (`AssignOwnerNode`, `UnpublishByKeywordComment`, `UnpublishByKeywordNode`). The commit deleted every file under `core/modules/action/` including the module file, routing, menu and task link definitions, config schema, help topics, and all functional and unit tests. A PHPStan baseline suppression for `ActionFormBase::save()` was also cleaned up from `core/.phpstan-baseline.php`, and the "Actions UI" entry was removed from `core/MAINTAINERS.txt`.

The underlying infrastructure was not touched: the `Action` config entity type in `system\Entity\Action`, the `plugin.manager.action` service, and the `administer actions` permission defined in `system.permissions.yml` all remain in the `system` module.

The module's `action_migration_plugins_alter()` hook, which added migration plugin mappings for `comment_unpublish_by_keyword_action` and `node_unpublish_by_keyword_action` during Drupal 7 upgrades, was also removed with the module.

Work on this issue was blocked until [#3437589](https://www.drupal.org/node/3437589) moved permission, JSON:API, and REST tests that lived inside the action module's test directory to more appropriate locations, so they would not be lost with the removal. The issue initially explored keeping an obsolete stub `action.info.yml` to ease upgrade path testing, but this approach was dropped in favour of a clean full removal, which forces sites to make an explicit choice (see collaboration note).

## Upgrade

Sites running Drupal 10 with the Actions UI module enabled should take one of these steps before or after upgrading to Drupal 11:

1. Uninstall the module if the Actions UI is not needed:
   ```
   drush pm:uninstall action
   ```

2. Or install the contributed replacement to retain the UI:
   ```
   composer require drupal/action
   drush en action
   ```
