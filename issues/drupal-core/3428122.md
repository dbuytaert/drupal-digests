## Summary

A `packageManager` field has been added to `core/package.json`, declaring `yarn@1.22.22` as the project's package manager. Drupal core has recommended using corepack to manage the Yarn version since September 2022, but without this field, running any yarn command inside `/core` caused corepack to write the field itself, dirtying the working tree for every developer using corepack. The fix pins the version in the standard location so corepack reads it rather than writing it. Developers who install Node.js via a system package manager and do not already have corepack may need to install and enable it explicitly before running yarn commands in `/core`.

## Impact

Affects site owners.

- **Requirements:** Core contributors need corepack installed and enabled (`corepack enable`) before running yarn commands in `/core`; some Node.js distributions do not bundle corepack and require a manual `npm install -g corepack` first.

## Technical details

Node.js's corepack tool reads the `packageManager` field in `package.json` to activate the correct package manager version. When that field was absent from `core/package.json`, corepack would inject it on first use, producing an unintended modification to a tracked file and a permanently dirty working tree for corepack users. The fix adds `"packageManager": "yarn@1.22.22"` as a top-level key in `core/package.json`. The same one-line change was cherry-picked identically to 10.2.x, 10.3.x, and 11.x.

A secondary benefit noted by longwave: Yarn will warn users who have the `packageManager` field set but have not yet run `corepack enable`, nudging them toward the correct setup. The flip side, reported by xjm after the fix landed, is that some Node.js distributions (especially those installed from a system package manager) do not bundle corepack at all. On those systems, running yarn in `/core` after this change fails until `npm install -g corepack && corepack enable` is run explicitly. xjm updated the change record at drupal.org/node/3428571 to document this requirement.

This issue is related to [#3109556](https://www.drupal.org/node/3109556), which upgraded Drupal core's build tooling from Yarn 1 to Yarn 4 on 11.x.

## Upgrade

Core contributors using Node.js installed via a system package manager (Linux distro packages, some CI images) may need to install and enable corepack before yarn commands in `/core` will work:

```sh
npm install -g corepack
corepack enable
```

This is a one-time setup step. Once done, corepack will automatically use the Yarn version declared in `core/package.json`.
