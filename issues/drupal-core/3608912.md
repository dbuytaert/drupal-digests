## Summary

Sites upgrading from Drupal 11.3.x to 11.4.x were hitting fatal errors during `drush updb` because the `help_search` and `node_search` search plugins could not be found. This was caused by the upgrade logic (installing `search_help` and `search_node` modules) living in `system_update_11400/11401()` instead of in the search module itself, creating a race condition where plugin discovery ran before the new modules were in place. The fix moves the update hooks into the search module and adds temporary deprecated stub plugin classes in the `help` and `node` modules so they are always resolvable during the upgrade window. This was backported to 11.4.x and released in 11.4.2.

## Impact

Affects module maintainers and site owners.

- **Database update:** New `search_update_11401()` and `search_update_11402()` hooks install `search_help` and `search_node`; `help_update_11400()` drops the `help_search_items` table when search is not installed.
- **Deprecation:** `\Drupal\help\Plugin\Search\HelpSearch` and `\Drupal\node\Plugin\Search\NodeSearch` are now deprecated stubs; use `\Drupal\search_help\Plugin\Search\HelpSearch` and `\Drupal\search_node\Plugin\Search\SearchNode` respectively.

## Technical details

The original `system_update_11400()` and `system_update_11401()` installed `search_help` and `search_node` and dropped the `help_search_items` table. Because those updates lived in `system`, any future removal of search from core required all sites to have already run them, effectively blocking Drupal 12 from dropping search. Moving the logic to `search_update_11401()` (installs `search_help` if `help` is enabled) and `search_update_11402()` (installs `search_node` if `node` is enabled) means the updates only matter when the search module exists, which is the same contract as any other module.

A separate problem caused "plugin does not exist" errors during the update: router rebuilds triggered during the update process load `search_page` config entities, which instantiate search plugins by ID. If `search_help`/`search_node` were not yet installed when the router rebuilt, the `help_search` and `node_search` plugin IDs could not be resolved. The fix adds deprecated stub classes: `\Drupal\help\Plugin\Search\HelpSearch` (new file in `help` module) and a refactored `\Drupal\node\Plugin\Search\NodeSearch` that now extends `ConfigurableSearchPluginBase` directly instead of the `search_node` class. Both fire a `@trigger_error` deprecation in their constructors and return empty/forbidden results from all methods, providing just enough surface for plugin discovery to succeed without executing any real search logic.

The `help_search_items` table management was also restructured: `hook_schema()` was removed from `search_help.install` so the table is no longer created on module install. Instead, `\Drupal\search_help\Plugin\Search\HelpSearch::updateTopicList()` and `findResults()` now wrap DB queries in try/catch blocks and call `ensureTableExists()` (backed by `schemaDefinition()`) to create the table on first use. `search_help_uninstall()` drops the table on uninstall. A new `help_update_11400()` handles the case where a site has the `help_search_items` table but search is not installed. The `help_post_update_search_help_dependencies()` function is emptied and its logic moved to `search_post_update_search_help_dependencies()`, so it travels with the search module into contrib. Multiple update path test classes were annotated with `#[IgnoreDeprecations]` to suppress noise from the stub constructors during testing, and `HelpSearchHelpUpgradeTest` was relocated from the `Tests\help\Functional` namespace to `Tests\search\Functional`.

## Upgrade

No manual changes are required for sites upgrading to 11.4.2 or later. Running `drush updb` will execute the new `search_update_11401()` and `search_update_11402()` hooks automatically.

If your custom code extends or directly instantiates the deprecated stubs, update the references:

```php
// Deprecated (drupal:11.4.0, removed drupal:12.0.0):
\Drupal\help\Plugin\Search\HelpSearch
\Drupal\node\Plugin\Search\NodeSearch

// Use instead:
\Drupal\search_help\Plugin\Search\HelpSearch   // requires search_help module
\Drupal\search_node\Plugin\Search\SearchNode   // requires search_node module
```
