## Summary

When using the Olivero theme on mobile devices with the Navigation module enabled, the close button for the mobile menu overlapped with the search bar, making it difficult to use either element. This fix adds padding to the mobile navigation panel when the Navigation module's top bar is present, ensuring the close button and search bar no longer overlap and remain properly accessible.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue occurred when the Navigation module was enabled alongside the Olivero theme on mobile viewports. The mobile navigation panel's close button would overlay the search bar, creating a usability problem.

The fix adds a CSS rule to `header-navigation.css` and `header-navigation.pcss.css` that targets the mobile navigation panel when the Navigation module's control bar is present. Using the `:has()` selector to detect `body:has(> .admin-toolbar-control-bar)`, the rule adds `padding-top: 3rem` (48px) to the navigation panel on viewports narrower than 63.9375rem (1023px). This pushes the navigation content down, preventing overlap with the mobile top bar elements.
