## Summary

This is a purely internal test-infrastructure change with no user-facing effect. It removes all uses of the `stable9` theme from Drupal core's test suite in preparation for the broader Stable 9 deprecation ([#3567667](https://www.drupal.org/node/3567667)).

## Impact

No upgrade or configuration changes required.

## Technical details

Stable 9 is being deprecated and its use in tests needed to be eliminated before the deprecation is completed. The changes fall into several groups.

**Test theme `.info.yml` files**: Seventeen test theme fixtures across `big_pipe`, `block`, `config_translation`, `help`, `system`, `update`, `user`, `views`, and profile test directories had `base theme: stable9` replaced with `base theme: false` (no base theme). Affected fixtures include `big_pipe_test_theme`, `block_test_theme`, `block_test_special_chars_theme`, `config_translation_test_theme`, `help_topics_test_theme`, `test_core_semver`, `test_invalid_core_semver`, `test_invalid_region`, `test_theme_having_veery_long_name_which_is_too_long`, `test_theme_settings_features`, `test_theme_twig_registry_loader`, `update_test_base_theme`, `user_test_theme`, `views_test_checkboxes_theme`, `views_test_theme`, `testing_config_import_theme`, and `test_stable`.

**`starterkit_theme`**: Its own `base theme` declaration was changed from `stable9` to `false`, removing its runtime dependency on Stable 9.

**`test_base_theme` expansion**: The `test_base_theme` fixture previously relied on `libraries-extend: starterkit_theme/base` to inherit base styles. That mechanism was removed and replaced with self-owned library definitions. New `base`, `messages`, and `dialog` library entries were added to `test_base_theme.libraries.yml`, backed by new CSS stub files (`css/components/button.css`, `container-inline.css`, `details.css`, `messages.css`, `dialog.css`). The fixture's `.info.yml` now lists `test_base_theme/base` and `test_base_theme/messages` directly.

**`test_theme` rework**: `test_theme` changed its `base theme` from `starterkit_theme` to `false`. It now declares `test_theme/base` and `test_theme/messages` as its own libraries, and its `libraries-override` block was updated to reference `test_base_theme` asset paths (under `core/modules/system/tests/themes/test_base_theme/css/components/`) instead of `starterkit_theme` or `stable9` paths. Path overrides for `core/drupal.dropbutton` and `core/drupal.vertical-tabs` were updated to use `misc/` source paths rather than `/core/themes/stable9/css/` paths.

**`test_subtheme`**: Its `libraries-extend` block was updated from `starterkit_theme/base` to `test_base_theme/base`.

**Functional and kernel test updates**: Tests that installed `stable9` to represent a hidden theme (in `BlockAdminThemeTest`, `SystemThemeTest`, `AjaxTest`, and views tests) now install `test_base_theme` or `stark` instead. `ResolvedLibraryDefinitionsFilesMatchTest` removes `stable9` from its `$allThemes` list. `LibraryDiscoveryIntegrationTest` assertions for library overrides and extends are updated to reference `test_base_theme` instead of `starterkit_theme`. `ModuleRequiredByThemesUninstallValidatorTest` replaces `stable9` fixture data with `stark`. The `OffCanvasTest::getTestThemes()` list drops `stable9`. Views pager and empty-row tests drop `stable9` from the theme iteration lists. A comment in `NodeRssCacheTest` about the `<article>` tag no longer attributes it to the `stable9` theme. A comment in `MediaPreviewTest` about the `media-embed-error.html.twig` template is corrected to point to the canonical module template location rather than `stable9`.
