## Summary

Drupal core's bundled Underscore.js library was updated from version 1.13.7 to 1.13.8 to address a security vulnerability. The previous version contained a denial-of-service (DoS) flaw where deeply nested data structures could trigger stack overflow errors in the `.flatten` and `.isEqual` functions due to uncontrolled recursion. The vulnerability was rated high severity (CVSS 7.5–8.2) and is fixed in the upstream 1.13.8 release. This update protects sites from potential service disruptions caused by malicious input to these functions.

Affects site owners.

## Technical details

The vulnerability (CVE-2026-27601, CWE-770) existed in Underscore.js versions prior to 1.13.8. The `.flatten` and `.isEqual` functions performed recursion without depth limits. An attacker could craft deeply nested structures (e.g., via `JSON.parse` without validation) and pass them to these functions, causing stack overflow and crashing the JavaScript runtime.

The 1.13.8 release addresses this by implementing depth controls in the affected functions. The update was applied to Drupal core by running `yarn vendor-update` to pull the latest minified library and its source map. The fix is entirely within the vendored JavaScript library; no Drupal code changes were required.

The vulnerability affects the `.flatten` function when called without a finite depth parameter and the `.isEqual` function when comparing attacker-controlled deeply nested objects. Sites using these functions on untrusted input were at risk of DoS attacks. The update mitigates the risk by preventing unbounded recursion.
