## Summary

Following the CKEditor 5 v48 upgrade in [#3587182](https://www.drupal.org/node/3587182), the old sub-path import style for CKEditor 5 (such as `ckeditor5/src/core`) was sunset by the library. This issue updates all of Drupal core's built-in CKEditor 5 plugin source files to use the new unified `ckeditor5` package import and rebuilds the compiled bundles, keeping the editor fully functional on Drupal 12 (`main`). This change does not affect Drupal 10 or 11.

## Impact

No upgrade or configuration changes required.

## Technical details

CKEditor 5's new installation method requires importing from the single unified `ckeditor5` package rather than from deep sub-paths like `ckeditor5/src/core`. The compiled bundles for the five core CKEditor 5 plugins, `drupalEmphasis`, `drupalHtmlEngine`, `drupalImage`, `drupalMedia`, and `drupalEntityLinkSuggestions`, all had their webpack external binding updated from `"ckeditor5/src/core"` to `"ckeditor5"` in the minified output, reflecting matching source-file changes that were rebuilt with webpack.

During review, the question arose whether to also remove the now-dead externals patterns in `webpack.config.js` (the `request.startsWith('ckeditor5/')` and `request.startsWith('@ckeditor/')` branches that previously mapped old sub-path imports to the `CKEDITOR` UMD global). Testing confirmed these branches are dead code after the migration, but the cleanup was intentionally deferred to a follow-up to keep this patch focused.

## Upgrade

Contrib or custom module developers who maintain CKEditor 5 plugins for Drupal 12 and use the old sub-path import style must update their source files:

```js
// Before (old installation method, sunset in CKEditor 5 v48)
import { Plugin } from 'ckeditor5/src/core';
import { ButtonView } from 'ckeditor5/src/ui';

// After (new installation method)
import { Plugin, ButtonView } from 'ckeditor5';
```

After updating the source files, rebuild the plugin bundles with `yarn build` (or the equivalent). See the [CKEditor 5 migration guide](https://ckeditor.com/docs/ckeditor5/latest/updating/nim-migration/migration-to-new-installation-methods.html) for full details.
