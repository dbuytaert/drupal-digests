## Summary

This is a test-only change with no user-facing effect. The unit tests in the Comment module were updated to comply with PHPUnit 12.5.0+ rules that prohibit mock objects from being created without any expectations configured. Tests that only needed to control return values now use stubs instead of mocks, and the deprecated `$this->any()` and stub-level `with()` constraints have been removed throughout.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced a deprecation notice whenever a mock object created via `createMock()` is used without configuring any expectations. PHPUnit's position is that such objects should be stubs (`createStub()`) instead, because they never assert anything about how the collaborator is called. Additionally, `$this->any()` is deprecated (it was always equivalent to having no expectation), and `with()` is deprecated for stubs because argument matching is meaningless when no expectation is asserted.

Eight unit test files under `core/modules/comment/tests/src/Unit/` were updated:

- **`CommentLinkBuilderTest.php`**: `$this->commentManager` and `$this->currentUser` switched from `createMock()` to `createStub()`. All `expects($this->any())` chains removed from their `setUp()` and from the per-test `hasPermission`, `isAuthenticated`, and `isAnonymous` stubs. The helper `getMockNode()` method similarly switches the node, a `FieldDefinitionInterface` stub, and related method chains to `createStub()`. PHPDoc type hints updated from `MockObject\MockObject` to `MockObject\Stub`.
- **`CommentManagerTest.php`**: `EntityTypeManagerInterface`, `ConfigFactoryInterface`, `TranslationInterface`, `ModuleHandlerInterface`, `AccountInterface`, and `EntityDisplayRepositoryInterface` switched to `createStub()`. A `prophesize(EntityDisplayRepositoryInterface::class)->reveal()` call replaced with `createStub(EntityDisplayRepositoryInterface::class)`. An unused `getClass()` expectation removed; the `entityClassImplements` expectation tightened from `$this->any()` to `$this->once()`.
- **`CommentStatisticsUnitTest.php`**: `Statement` and `Select` objects refactored from `getMockBuilder(...)->disableOriginalConstructor()->getMock()` to `createStub()`. `AccountInterface`, `EntityTypeManagerInterface`, and `StateInterface` constructor arguments switched to `createStub()`.
- **`Entity/CommentLockTest.php`**: Container services `module_handler`, `current_user`, `cache.test`, and `comment.statistics` switched to `createStub()`. Inline `CacheTagsInvalidator`, `$anon_user`, `EntityTypeInterface`, and `CommentStorageInterface` switched similarly. Remaining `expects($this->any())` on the stub-style calls removed.
- **`Plugin/views/field/CommentBulkFormTest.php`**: `ActionConfigEntityInterface`, `EntityStorageInterface`, `EntityRepositoryInterface`, `LanguageManagerInterface`, `MessengerInterface`, `ResettableStackedRouteMatchInterface`, `ViewExecutable`, and `DisplayPluginBase` all switched to `createStub()`. `getMockBuilder()->disableOriginalConstructor()->getMock()` pattern eliminated for `ViewExecutable` and `DisplayPluginBase`. Leftover `$this->any()` expectations on `ViewsData::get` and `ViewEntityInterface::get` tightened to `$this->once()`.
- **`Plugin/views/field/EntityLinkTest.php`**: `ViewExecutable` and `DisplayPluginBase` switched to `createStub()`.
- **`Plugin/views/field/LinkApproveTest.php`**: Constructor arguments for `AccessManagerInterface`, `EntityTypeManagerInterface`, `EntityRepositoryInterface`, `LanguageManagerInterface`, `ViewExecutable`, and `DisplayPluginBase` switched to `createStub()`.
- **`Plugin/views/field/LinkReplyTest.php`**: Same as `LinkApproveTest`. Additionally, the `string_translation` container service replaced with `$this->getStringTranslationStub()` (the UnitTestCase helper) rather than a bare `createStub(TranslationInterface::class)`.

Where an object still needs a real expectation (e.g., `$entity_field_manager->expects($this->once())->method('getFieldMapByFieldType')`), `createMock()` is intentionally retained.
