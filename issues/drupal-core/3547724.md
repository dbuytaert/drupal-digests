## Summary

Support for the `access callback` key in Views handler plugin definitions has been removed in Drupal 12.0.0. Any custom Views handler that still declares an `access callback` in its definition array will now throw an `AccessException` when `access()` is called, rather than silently invoking the procedural function. Developers must replace procedural access callbacks with a custom `access()` method on their handler class.

## Impact

Affects module maintainers.

- **API change:** `HandlerBase::access()` now throws `AccessException` (rather than invoking the callback or returning `TRUE`) when `access callback` is present in the handler definition; the key is no longer supported.

## Technical details

In `HandlerBase::access()` (located in `core/modules/views/src/Plugin/views/HandlerBase.php`), the old code checked `function_exists($this->definition['access callback'])`, triggered a `@trigger_error` deprecation, and then called the procedural function. The new code replaces all of that with a single unconditional `throw new AccessException(...)` whenever the `access callback` key is present in the definition at all, regardless of whether the function exists. The check for `function_exists` is gone, meaning there is no longer any execution path that runs the callback.

The two test helper functions `views_test_data_handler_test_access_callback()` and `views_test_data_handler_test_access_callback_argument()` have been deleted from `core/modules/views/tests/modules/views_test_data/views_test_data.module`, and the corresponding functional `testAccess()` test in `HandlerTest` has been removed. A new unit test `testAccessThrowsExceptionWithAccessCallback()` in `HandlerTest` (unit variant) covers the exception path. The original deprecation was introduced in drupal:11.3.0 referencing [#3539918](https://www.drupal.org/node/3539918).

## Upgrade

Any custom Views handler definition that sets `access callback` must be updated. Instead of:

```php
// Old: in hook_views_data() or a derived views data class
$data['my_table']['my_field']['field']['access callback'] = 'my_module_access_callback';
```

Remove the `access callback` key entirely and override `access()` on the handler class:

```php
// New: in the handler plugin class
public function access(AccountInterface $account) {
  // Your access logic here.
  return $account->hasPermission('my permission');
}
```
