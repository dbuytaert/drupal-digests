## Summary

A regression introduced in Drupal 10.5.x caused `FileSystem::deleteRecursive()` to silently skip all deletions when the target path uses a remote stream wrapper such as S3fs, Azure Blob Storage, or Flysystem. The method returned `TRUE` claiming success while leaving all files and directories untouched in remote storage. This broke image style flushing (`drush image-flush`), cache management for derivative files, and any code that uses `deleteRecursive()` on a remote URI, and posed a risk of sensitive private files remaining in storage after deletion was requested. The fix is now committed to 10.6.x, 11.x, 11.3.x, and main.

## Impact

Affects site owners.

- **Security:** Files targeted by `deleteRecursive()` on remote stream wrapper URIs (S3fs, Azure, Flysystem) were silently not deleted and falsely reported as deleted, potentially leaving sensitive private files in storage.

## Technical details

The root cause was a change made in [#3482449](https://www.drupal.org/node/3482449) to prevent symlink traversal attacks. That fix added a `realpath()` call at the top of `deleteRecursive()` to resolve stream wrapper URIs to absolute local paths, so that `is_link()` checks would work correctly. However, `FileSystemInterface::realpath()` is documented to return `FALSE` for remote URIs, and the code unconditionally assigned the result back to `$path`. With `$path` set to `FALSE`, the subsequent `file_exists(FALSE)` check returned `FALSE`, the method hit its early return, and exited without deleting anything.

The fix tightens the assignment to only apply when `realpath()` returns a non-false value: `$realpath = $this->realpath($path); if ($realpath !== FALSE) { $path = $realpath; }`. Local stream wrappers that return a real path still get resolved to local filesystem paths, preserving the symlink traversal protection from [#3482449](https://www.drupal.org/node/3482449). Remote stream wrappers that return `FALSE` keep their original URI, which is sufficient because `is_link()` cannot detect symlinks through those wrappers anyway, and the underlying remote stream wrapper handles directory traversal correctly.

A new kernel test class `FileDeleteRecursiveRemoteTest` was added using a `DummyRemoteStreamWrapper` fixture to cover single-file deletion, empty directories, directories with files, and nested subdirectories via a remote-style wrapper that returns `FALSE` from `realpath()`.
