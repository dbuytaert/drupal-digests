## Summary

On the node create and edit form, the Save and Preview action buttons were rendering immediately below the Title field instead of at the bottom of the page when using Drupal's new default admin theme (introduced in 11.x). This has been fixed; the buttons now appear at the bottom of the form as expected.

## Impact

No upgrade or configuration changes required.

## Technical details

The template responsible for the node edit form layout in the new default admin theme was passing the wrong key names to the `form|without(...)` Twig filter. It excluded `gin_actions` and `gin_sidebar_toggle`, which are Gin-theme identifiers that do not exist in Drupal core's node form, so the actual `actions` element was never suppressed from the main region. It was therefore rendered inline, right after the title field.

The fix has two parts. First, the exclusion list in `form|without(...)` is corrected: `gin_actions` becomes `actions` and `gin_sidebar_toggle` becomes `default_admin_sidebar_toggle`. Second, a new `<div class="layout-region layout-region-node-footer">` block is added after the secondary (sidebar) region and explicitly renders `{{ form.actions }}` there, placing the buttons at the bottom of the form where they belong.
