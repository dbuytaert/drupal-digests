## Summary

PHPUnit 12.5 introduced notices when mock objects are created without configuring expectations, and deprecated the `any()` expectation matcher. This change updates the unit tests in the `views` and `views_ui` modules to eliminate all such notices, making the test suite compatible with current PHPUnit 12 behavior. There is no effect on production code or site functionality.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 began emitting notices for mocks that have no expectations configured, and PHPUnit 12.5.5 deprecated `expects($this->any())` because it is equivalent to having no expectation. The goal of this issue was to silence those notices across the `views` and `views_ui` module test suites.

The fix applies two complementary strategies across a large set of unit test files under `core/modules/views/tests/src/Unit/` and `core/modules/views_ui/tests/src/Unit/`:

1. **Convert expectation-less mocks to stubs.** Wherever `createMock()` or `getMockBuilder()->getMock()` was used and no expectations were actually asserted, the call is replaced with `createStub()`. `createStub()` returns a test double that returns default values (null, empty arrays, etc.) for all methods without enforcing call counts. Affected classes include `ViewExecutable`, `View`, `DisplayPluginCollection`, `ViewExecutableFactory`, `EntityStorageInterface`, `CurrentPathStack`, `RedirectDestinationInterface`, `PathValidatorInterface`, `RouteProviderInterface`, `StateInterface`, `AccountInterface`, `CacheableMetadata`, and others.

2. **Replace `any()` with precise expectations or remove it.** Where a mock genuinely needs an expectation, `expects($this->any())` is replaced with `$this->once()`, `$this->atLeastOnce()`, or `$this->atMost(1)` as appropriate. For stubs, the `expects(...)` call is dropped entirely and only `->method()->willReturn()` is retained.

3. **Introduce `setUpMock*()` helper methods.** Several test classes (e.g., `ViewAjaxControllerTest`, `ViewsLocalTaskTest`) use shared dependencies in `setUp()` that are now initialized as stubs. For individual tests that actually assert call counts, new protected helper methods such as `setUpMockCurrentPath()`, `setUpMockViewStorage()`, and `setUpMockState()` create a real mock and inject it via `ReflectionProperty`, upgrading the stub for that specific test.

4. **Refactor `HandlerBaseTest`** to remove its use of the `HandlerTestTrait` shared setup in favor of per-test, locally scoped mock/stub creation, eliminating the need for shared mutable state that required broad `any()` expectations.

5. **Clean up `core/.phpstan-baseline.php`.** All `ignoreErrors` entries suppressing PHPStan `disallowed.method` warnings for `any()` calls across the affected Views test files are removed, since the underlying calls no longer exist. Related `missingType.return` suppressions for now-removed helper methods in `HandlerBaseTest` are also dropped.

The `HandlerTestTrait` shared trait (`setupExecutableAndView()`, `setupViewsData()`, `setupDisplay()`) is updated to use `createStub()` throughout, benefiting all tests that still use it.
