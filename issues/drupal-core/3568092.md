## Summary

The Editor module moved four procedural form callbacks from `editor.module` into methods within the `EditorHooks` class. These callbacks handle configuration, validation, submission, and AJAX behavior for the text editor settings form. The procedural functions remain as thin wrappers that trigger deprecation notices and delegate to the new methods. This change aligns with Drupal's broader effort to eliminate procedural code from `.module` files in favor of object-oriented patterns.

## Impact

Affects module maintainers.

- **Deprecation:** Four procedural form callbacks in `editor.module` replaced with methods in `EditorHooks` class

## Technical details

The change relocates four functions that were part of `filter_format_form()` into the `EditorHooks` service class: `editor_form_filter_admin_format_editor_configure()`, `editor_form_filter_admin_form_ajax()`, `editor_form_filter_admin_format_validate()`, and `editor_form_filter_admin_format_submit()`. Each procedural function now triggers a deprecation error and forwards to the corresponding method.

The new methods are: `editorFormFilterAdminFormatEditorConfigure()`, `editorFormFilterAdminFormAjax()`, `editorFormFilterAdminFormatValidate()`, and `editorFormFilterAdminFormatSubmit()`. All form alter logic in `EditorHooks::formFilterFormatFormAlter()` was updated to reference these methods using `self::class . ':methodName'` syntax for callbacks.

References to the old function names in `ckeditor5.module` and `CKEditor5.php` were updated with `@see` tags pointing to the new methods. The conversion also allowed removal of a PHPStan baseline entry for a missing return type. The functions handle editor selection changes, AJAX updates, validation of editor configuration, and saving editor entities when the filter format form is submitted.

## Upgrade

The four deprecated procedural functions still work but will be removed in Drupal 12. If your custom code references these functions directly, update to use the `EditorHooks` service methods instead:

```php
// Before:
editor_form_filter_admin_format_editor_configure($form, $form_state);

// After:
\Drupal::service(\Drupal\editor\Hook\EditorHooks::class)
  ->editorFormFilterAdminFormatEditorConfigure($form, $form_state);
```

Similarly for `editorFormFilterAdminFormAjax()`, `editorFormFilterAdminFormatValidate()`, and `editorFormFilterAdminFormatSubmit()`. If you reference these functions as form callbacks, use `EditorHooks::class . ':methodName'` format.
