## Summary

This internal refactoring moves Settings Tray-specific block metadata and form classes out of core modules (system, help, page title) and into the Settings Tray module itself, as part of the ongoing effort to move Settings Tray to contributed module status. There is no user-facing behavior change.

## Impact

Affects module maintainers.

- **API change:** `SystemBrandingOffCanvasForm` and `SystemMenuOffCanvasForm` moved from `Drupal\system\Form` to `Drupal\settings_tray\Form`; callers must update their `use` statements.

## Technical details

Five core block plugins previously carried `forms: ['settings_tray' => ...]` entries in their `#[Block]` attributes: `PageTitleBlock`, `HelpBlock`, `SystemBrandingBlock`, `SystemMainBlock`, and `SystemMenuBlock`. Those attribute entries were removed. The `blockAlter()` method in the Settings Tray module's hook class now uses a `match` expression on `$definition['id']` to inject the same values for those specific block IDs, centralizing all Settings Tray awareness inside the module itself.

`SystemBrandingOffCanvasForm` and `SystemMenuOffCanvasForm` were moved from `Drupal\system\Form` (in `core/modules/system/src/Form/`) to `Drupal\settings_tray\Form` (in `core/modules/settings_tray/src/Form/`). longwave observed during review that `SystemMenuOffCanvasForm` is exclusively used by the Settings Tray feature and so should live there; the move of both form classes was a direct result of that review comment. The PHPStan baseline (`parameters.ignoreErrors`) was updated to reflect the new namespaces. The hardcoded block ID list in the `match` expression is still alterable by any `hook_block_alter()` implementation that runs after the Settings Tray one.

## Upgrade

If custom or contributed code references the old form class namespaces, update the fully-qualified class names:

```php
// Before
use Drupal\system\Form\SystemBrandingOffCanvasForm;
use Drupal\system\Form\SystemMenuOffCanvasForm;

// After
use Drupal\settings_tray\Form\SystemBrandingOffCanvasForm;
use Drupal\settings_tray\Form\SystemMenuOffCanvasForm;
```
