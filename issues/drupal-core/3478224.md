## Summary

Site builders can now use Drupal recipes to programmatically add blocks to the Navigation module's block layout. A new `addNavigationBlock` config action lets a recipe insert a block at a specific position in the navigation layout by specifying the block plugin ID, label, and other configuration, without requiring custom code.

## Impact

Affects module maintainers.

- **New API:** New `addNavigationBlock` config action plugin in `navigation.module` for inserting blocks into the Navigation layout from recipes.

## Technical details

The Navigation module stores its Layout Builder configuration in the simple config object `navigation.block_layout`, loaded through the `navigation` section storage type via `SectionStorageManagerInterface`. Unlike standard Layout Builder usage on content types (which stores sections in config entities), this simple-config storage model meant the generic component-insertion trait approach introduced in [#3475115](https://www.drupal.org/node/3475115) could not be reused: that trait assumes a config entity target.

The new `AddNavigationBlock` plugin (`core/modules/navigation/src/Plugin/ConfigAction/AddNavigationBlock.php`) implements `ConfigActionPluginInterface` and `ContainerFactoryPluginInterface`, and is registered via the PHP 8 attribute `#[ConfigAction(id: 'addNavigationBlock')]`. Its `apply()` method is locked to `navigation.block_layout`, throwing a `ConfigActionException` for any other config name. It loads the navigation section storage, retrieves section 0, and inserts a new `SectionComponent` at the caller-supplied `delta`. If the delta is out of bounds it falls back to `appendComponent()`. The plugin is marked `@internal` as experimental API.

The same commit corrects a minor bug in `navigation.layouts.yml`: `default_region` was changed from `first` to `content` to match the actual region name used by the navigation layout.

A new kernel test class `AddNavigationBlockConfigActionTest` covers both valid block insertion at various delta values (including out-of-bounds clamping) and the guard that rejects non-navigation config names.

## Upgrade

Recipes targeting Navigation can now use the new action:

```yaml
config:
  actions:
    navigation.block_layout:
      addNavigationBlock:
        delta: 1
        configuration:
          id: 'navigation_menu:content'
          label: 'My Block'
          label_display: 1
          provider: navigation
          level: 1
          depth: 2
```

No changes are needed for existing code. The action is only available when the `navigation` module is installed.

### Collaboration

penyaskito raised the possibility of reusing the generic component-insertion approach from [#3475115](https://www.drupal.org/node/3475115) to avoid duplication. plopesc and m4olivei investigated and found it was not applicable: Navigation's Layout Builder state lives in simple config (`navigation.block_layout`) rather than a config entity, whereas the work in [#3475115](https://www.drupal.org/node/3475115) requires a config entity target. This confirmed that a dedicated custom plugin for Navigation was the correct path forward.
