## Summary

This is a test-only change with no user-facing effect. It adds a new `procedural_hook_theme` test theme and two accompanying test classes to verify that Drupal's hook collection system correctly handles procedural hooks declared in `.theme` files. The tests ensure the BC layer for theme hooks continues to work as Drupal transitions toward OOP hook implementations.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue was raised after berdir noticed that procedural hooks in themes lacked explicit test coverage, which matters because those hooks are still collected and executed during the ongoing migration to OOP hooks.

Three test artifacts were added inside `core/modules/system/tests/themes/HookCollector/procedural_hook_theme/`:

1. `procedural_hook_theme.theme`: Declares four procedural functions. `procedural_hook_theme_procedural_alter()` is a plain procedural hook that should be collected and invoked. `procedural_hook_theme_procedural_legacy_alter()` carries the `#[LegacyHook]` attribute, which signals that an OOP implementation in `ProceduralHookThemeHooks::proceduralLegacyAlter()` (decorated with `#[Hook('procedural_legacy_alter')]`) should take precedence and the procedural version should be skipped. `procedural_hook_theme_preprocess_node()` and `procedural_hook_theme_preprocess_node__article()` are preprocess hooks that override the node title variable.

2. `ThemeHookTest` (kernel test in `core/modules/system/tests/src/Kernel/Theme/`): Installs the theme, activates it, and asserts that `\Drupal::theme()->alter('procedural', ...)` picks up only the plain procedural hook, and that `alter('procedural_legacy', ...)` yields only the OOP result (confirming `#[LegacyHook]` suppression works in themes).

3. `ThemeProceduralTest` (functional browser test in `core/modules/system/tests/src/Functional/Theme/`): Uses `procedural_hook_theme` as the default theme, creates article and basic node types, and asserts that `hook_preprocess_node__article()` sets the title to `'Procedural Article Node Preprocess'` for articles while `hook_preprocess_node()` sets it to `'Procedural Node Preprocess'` for other types. A kernel-test approach was attempted but abandoned; the browser test provides real rendering coverage.

As berdir noted in the thread, these tests form the BC guarantee that the procedural theme hook pipeline works until `.theme` files are deprecated ([#3581218](https://www.drupal.org/node/3581218)), and are designed to be convertible into deprecation tests at that point.
