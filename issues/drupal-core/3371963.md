## Summary

Drupal core's Nightwatch browser-testing library has been updated from version 2.4.2 to 3.7.0, targeting Drupal 11.1.x. This keeps the JavaScript testing toolchain current and allows the test suite to run against a modern Selenium and Chrome stack. The update was blocked for over a year by W3C WebDriver compatibility issues that required fixes to existing tests and a prerequisite switch to a W3C-compliant Selenium environment before the version bump could land. This change has no effect on site behaviour; it only affects developers who write or run Nightwatch tests.

## Impact

No upgrade or configuration changes required.

## Technical details

Nightwatch 3.x enables W3C WebDriver mode by default, which introduced two concrete incompatibilities with existing Drupal Nightwatch helpers.

First, the `domain` property in cookie objects is rejected in W3C mode. The `drupalInstall.js` custom command sets a `SIMPLETEST_USER_AGENT` cookie during test installation; the `domain: url.host` field was removed from that cookie object to make cookie creation succeed under the new driver.

Second, the `api.execute()` method signature changed: in Nightwatch 3.x the callback must be the third argument, with an explicit arguments array as the second. Two custom assertion files, `core/tests/Drupal/Nightwatch/Assertions/deprecationErrorExists.js` and `core/tests/Drupal/Nightwatch/Assertions/noDeprecationErrors.js`, called `this.api.execute(fn, callback)` (two arguments), which silently caused tests to hang because Nightwatch treated the callback as the arguments array. Both were updated to `this.api.execute(fn, [], callback)`.

In `core/package.json` the `nightwatch` dependency moved from the pinned `2.4.2` to `^3.7.0`, and the `resolutions` section that previously pinned `glob/jackspeak` to `2.1.1` and `nightwatch/semver` to `~7.5.2` as workarounds for dependency conflicts was removed entirely. Notable transitive dependency updates in `core/yarn.lock`: `selenium-webdriver` from 4.3.1 to 4.22.0, `mocha` from 9.2.2 to 10.3.0, `nightwatch-axe-verbose` from 2.0.3 to 2.3.1 (pulling in `axe-core` 4.10.0), and `dotenv` from 10.0.0 to 16.3.1. A prerequisite for landing was [#3421202](https://www.drupal.org/node/3421202), which switched the GitLab CI environment to a W3C-compliant Selenium and Chrome setup.

## Upgrade

If your contrib module or custom code has Nightwatch tests that call `browser.execute()` or `api.execute()` with the callback as the second argument, update those calls to pass an explicit arguments array as the second argument and move the callback to the third position:

```js
// Before (Nightwatch 2.x)
this.api.execute(function () {
  return someValue;
}, callback);

// After (Nightwatch 3.x)
this.api.execute(function () {
  return someValue;
}, [], callback);
```

If you set cookies with a `domain` property in Nightwatch tests, remove the `domain` field or ensure it conforms to W3C WebDriver requirements. See the [Nightwatch 3.x upgrade guide](https://nightwatchjs.org/guide/overview/whats-new-in-v3.html) for the full list of breaking changes.
