## Summary

Drupal 12 removed code that was deprecated in Drupal 11 from the Views and Views UI modules. This cleanup effort removes outdated functions, methods, and patterns that were marked for removal in the next major version. The change affects multiple areas including template preprocessing functions, deprecated service methods, hook implementations, and internal utility functions. Site owners using Drupal 12 will benefit from a cleaner codebase with less technical debt, though any custom code still calling deprecated functions will need updates.

## Impact

Affects module maintainers.

- **API change:** Removed deprecated functions `views_entity_field_label()`, `views_field_default_views_data()`, `_views_field_get_entity_type_storage()`, and `template_preprocess_*()` functions for Views templates
- **Deprecation:** Removed previously deprecated methods including `EntityViewsData::getFieldStorageDefinitions()`, `HandlerBase::defineExtraOptions()`, and `views_ui_contextual_links_suppress()` functions

## Technical details

The cleanup removes code deprecated across Drupal 11.x releases. Major removals include standalone `template_preprocess_*()` functions that were moved to hook classes like `ViewsThemeHooks` and `ViewsUiThemeHooks`. These preprocessing functions are now registered directly in `hook_theme()` implementations.

The change removes utility functions like `views_entity_field_label()` (replaced by the entity field manager service), `views_field_default_views_data()` (replaced by the field data provider service), and contextual links suppression functions that are no longer needed.

Constructor parameters that were previously optional in `ViewExecutable` and `ViewExecutableFactory` are now required, specifically the display plugin manager. The code also removes support for the deprecated `theme_file` definition in plugin definitions and removes deprecated template files `views.theme.inc` and `views_ui.theme.inc`.

Filter operators now throw a `LogicException` instead of triggering a deprecation when a filter doesn't implement `FilterOperatorsInterface`. Template documentation updates replace references to deprecated functions with their hook class equivalents. The config updater no longer triggers deprecations for automated fixes that should have been applied manually. One test that verified deprecated behavior was removed entirely.

## Upgrade

If custom code calls removed functions, replace them with the documented alternatives:

```php
// Old: Deprecated in Drupal 11.2
$label = views_entity_field_label($entity_type, $field_name);

// New: Use the service
$label = \Drupal::service('entity_field.manager')->getFieldLabels($entity_type, $field_name);
```

```php
// Old: Deprecated in Drupal 11.2
$data = views_field_default_views_data($field_storage);

// New: Use the service
$data = \Drupal::service('views.field_data_provider')->defaultFieldImplementation($field_storage);
```

For Views plugins, remove any `theme_file` definitions from plugin annotations. Ensure filter plugins implement `FilterOperatorsInterface` if they define custom operators.

Template documentation should reference hook class methods instead of removed functions (e.g., `\Drupal\views\Hook\ViewsThemeHooks::preprocessViewsView()` instead of `template_preprocess_views_view()`).
