## Summary

Drupal core's unit test suite has been systematically refactored to replace expectation-less mock objects with stubs, preparing the codebase for PHPUnit 12.5+ compatibility. In PHPUnit 12.5, a mock object that never has expectations configured issues a notice and is slated to become an error in future versions. All occurrences of the deprecated `$this->any()` expectation were also removed. This is a test-only change with no effect on runtime behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces a notice for any `MockObject` that has no expectations configured: the object should instead be a `TestStub` created via `createStub()`. Additionally, `->expects($this->any())` is deprecated (it was equivalent to having no expectation), and `->with()` on a stub is deprecated. This meta issue ([#3561671](https://www.drupal.org/node/3561671)) tracked roughly 30 child issues that collectively swept all of Drupal core's unit tests.

The primary remediation strategies, documented by dcam in comment [#16441146](https://www.drupal.org/node/16441146), were:

1. Replace `createMock()` or `getMockBuilder()` calls that set no expectations with `createStub()`.
2. Replace `->expects($this->any())` with a precise expectation such as `->expects($this->atLeastOnce())`, or drop the expectation entirely and convert the object to a stub.
3. Remove `->with()` calls from stubs, or upgrade those stubs to mocks with a proper `->expects()`.
4. Where mocks were used purely to instantiate abstract base classes, replace them with concrete stub subclasses.

The `#[AllowMockObjectsWithoutExpectations]` attribute (available in PHPUnit 12.5) was intentionally not used during this effort, since Drupal core had not yet adopted PHPUnit 12; the one remaining use in `RouteBuilderTest.php` reflects a case where a full conversion was not yet feasible.

After all child issues landed, the repo shows 376 test files using `createStub()` and zero remaining uses of `->expects($this->any())`. The related issue [#3583606](https://www.drupal.org/node/3583606) tracks a remaining class of violations: `with*()` called on mocks that lack a corresponding `expects()`.
