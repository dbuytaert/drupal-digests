## Summary

Drupal core's base `UnitTestCase` class was causing hundreds of PHPUnit notices when running tests against PHPUnit 12.5.0+, which deprecates mock objects that have no call-count expectations and the `$this->any()` matcher. This change converts expectation-less mock objects in `UnitTestCase` to proper stubs, eliminating 556 notices traced back to that class. The fix has no effect on production behavior; it is purely a test infrastructure modernization needed for forward compatibility with PHPUnit 12.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced two related deprecations: (1) mock objects created with `createMock()` that never configure a call-count expectation emit a notice saying to use a stub instead, and (2) `$this->any()` is deprecated because it is semantically equivalent to having no expectation at all.

`UnitTestCase::getConfigFactoryStub()` replaced `getMockBuilder(...)->getMock()` + `expects($this->any())` with `createStub()` for `ImmutableConfig`, `Config`, and `ConfigFactoryInterface`. `getStringTranslationStub()` did the same for `TranslationInterface`. `getClassResolverStub()` converted `createMock(ClassResolverInterface::class)` + `expects($this->any())` to `createStub(ClassResolverInterface::class)`. In all three cases the `->method()->willReturn*()` chaining is retained; only the object creation call and the removed `expects()` call change.

`getContainerWithCacheTagsInvalidator()` is a different case: the container mock does need a verifiable expectation because the container's `get()` call is meaningful. The fix changes `expects($this->any())` to `expects($this->atLeastOnce())`. As a side effect, `ViewsDataTest::setUp()` had called this helper unconditionally even though only two test methods actually exercise the cache tags code path. Those two calls were moved out of `setUp()` and into the individual test methods `testFullAndTableGetCache()` and `testClearResetsFullyLoaded()` to avoid "expected at least once but not called" failures for the other tests in that class.

The PHPStan baseline (`phpstan-baseline.neon`) entry that suppressed nine `any()` deprecation warnings in `UnitTestCase.php` was also removed, since `$this->any()` no longer appears there. The docblock return types for `getConfigFactoryStub()` and `getStringTranslationStub()` were updated from `MockObject` to `Stub` to reflect the actual PHPUnit type returned.

## Upgrade

If a contrib or custom module's unit test calls `getContainerWithCacheTagsInvalidator()` inside `setUp()` (where it runs before every test method), tests that do not actually invoke `cache_tags.invalidator` will now fail with an "expected to be called at least once, but was not called" error. Move the call into the specific test methods that exercise the cache tags invalidator:

```php
// Before – called unconditionally in setUp():
protected function setUp(): void {
  parent::setUp();
  $this->cacheTagsInvalidator = $this->createMock(CacheTagsInvalidatorInterface::class);
  $this->getContainerWithCacheTagsInvalidator($this->cacheTagsInvalidator);
}

// After – called only in test methods that actually use it:
protected function setUp(): void {
  parent::setUp();
  $this->cacheTagsInvalidator = $this->createMock(CacheTagsInvalidatorInterface::class);
}

public function testSomethingThatUsesCacheTags(): void {
  $this->getContainerWithCacheTagsInvalidator($this->cacheTagsInvalidator);
  // ... test body ...
}
```

If your code type-hints the return value of `getStringTranslationStub()` or `getConfigFactoryStub()` as `\PHPUnit\Framework\MockObject\MockObject`, update those type hints to `\PHPUnit\Framework\MockObject\Stub`, which is now the actual return type.
