## Summary

This is a pure test infrastructure cleanup with no effect on production Drupal sites. It prepares the test suite for PHPUnit 12.5.0+, which deprecates `$this->any()` and emits notices for mock objects that have no configured expectations. The changes affect only test files across several core modules.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 changes mock object semantics in three ways: mock objects without any configured expectations emit a notice recommending stubs instead, `$this->any()` is deprecated (it was equivalent to no expectation), and `with()` on stubs is deprecated. This issue cleans up the remaining occurrences that crept in after [#3573259](https://www.drupal.org/node/3573259) was committed, plus all kernel-test violations that had not been checked before.

The diff removes eight `$ignoreErrors[]` entries from the PHPStan baseline file (`core/.phpstan-baseline.php`) that suppressed the `disallowed.method` identifier for `any()` calls in `CKEditor5Test.php`, `InstallerRedirectTraitTest.php`, `InputTest.php`, `VendorHardeningPluginTest.php`, `ParamConversionEnhancerTest.php`, `PrimitiveTypeConstraintValidatorTest.php`, and `SerializedConstraintValidatorTest.php`. Removing these entries means PHPStan will no longer silently accept those deprecated calls.

In the test files themselves the changes follow consistent patterns:

- `$this->any()` replaced with `$this->atLeastOnce()` when an expectation is genuinely needed (e.g. `CKEditor5Test`, `LanguageSwitcherTest`, `AliasManagerTest`, `ViewsBulkFormTest`).
- `createMock()` converted to `createStub()` for double objects whose methods are configured only with `willReturn` and no call-count expectation (e.g. `LanguageSwitcherTest`, `MysqlDriverTest`, `SqlBaseTest`, `DownloadTest`, `FileCopyTest`, `SyslogTest`, `StageCommitExceptionTest`, `StageEventsTest`).
- `with()` constraint removed from a stub's `createInstance` call in `ImageStyleTest`, because `with()` is deprecated on stubs.
- `getMockBuilder(...)->disableOriginalConstructor()->getMock()` replaced with `createStub()` in `ImageStyleTest`.
- Several `setUp()` methods were refactored to use a lightweight `createStub()` by default, with new helper methods (`setUpMockRouteProvider()`, `setUpMockAliasPrefixList()`, `setUpMockConfigEntityStorage()`, `setUpMockEntityTypeManager()`) that re-initialize the property as a proper `createMock()` with expectations only in the specific test methods that interact with those collaborators (e.g. `ConfigNamesMapperTest`, `AliasManagerTest`, `ContentLanguageSettingsUnitTest`). This avoids "no expectations configured" notices for tests that do not call the mocked method.
- `createStub()` also replaces `createMock()` for objects such as `EditorManager` in `EditorConfigEntityUnitTest`, which is changed from `createStub` back to `createMock` so that `expects($this->atLeastOnce())` can be attached to `createInstance`.

One test, `TransactionTest::testMockTransaction`, cannot be fully fixed in the current PHPUnit version because its sole purpose is to verify that mock transaction objects can be constructed; fixing it requires the `#[AllowMockObjectsWithoutExpectations]` attribute only available in PHPUnit 12, so it is deferred to issue [#3527936](https://www.drupal.org/node/3527936).
