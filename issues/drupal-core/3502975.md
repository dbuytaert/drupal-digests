## Summary

Drupal 12 removes the entire legacy "Update Manager" subsystem that was deprecated throughout the Drupal 11 cycle. This means `authorize.php`, the `FileTransfer` class hierarchy (FTP, SSH, Local), the `Updater` classes, all related hooks, system module helper functions, and every test covering this code are gone. The feature allowed site administrators to install and update modules and themes via FTP or SSH directly through the Drupal admin UI. It has been effectively non-functional for years and Composer is the correct tool for managing site code. Sites already running Drupal 11 should have no user-facing change since the UI and main code paths were removed in earlier issues; this cleans up the remaining PHP classes, hooks, and tests that were carrying deprecation notices.

## Impact

Affects module maintainers and site owners.

- **Database update:** New `update_post_update_clear_disk_cache_again()` post-update clears any leftover Update Manager disk-cache directories on upgrade to Drupal 12
- **API change:** `authorize.php`, the entire `Drupal\Core\FileTransfer` and `Drupal\Core\Updater` namespaces, `drupal_get_updaters()`, `drupal_get_filetransfer_info()`, `hook_updater_info()`, `hook_updater_info_alter()`, `hook_filetransfer_info()`, `hook_filetransfer_info_alter()`, `hook_archiver_info_alter()`, and `system_authorized_run()` / `system_authorized_batch_process()` are all removed; all were deprecated in drupal:11.2.0

## Technical details

This issue is the final cleanup step for the Update Manager removal, split from the meta issue [#3491731](https://www.drupal.org/node/3491731). The deprecations it removes were introduced across three earlier issues: [#3417136](https://www.drupal.org/node/3417136) (removing URL-based extension installation), [#3502974](https://www.drupal.org/node/3502974) (deprecating `authorize.php`, `FileTransfer`, and `Updater`), and [#3521059](https://www.drupal.org/node/3521059) (deprecating remaining Update Status functions).

The single commit (09985a50) deletes 4,863 lines across 53 files and adds 22 lines. The deletions cover:

**Core entry point**: `core/authorize.php` is deleted. This was the standalone PHP script that ran outside the normal Drupal bootstrap to perform privileged file operations.

**`core/includes/common.inc`**: `drupal_get_updaters()` and `drupal_get_filetransfer_info()` are removed. These assembled the registries of `Updater` and `FileTransfer` classes by invoking the now-removed hooks.

**`Drupal\Core\FileTransfer` namespace** (`core/lib/Drupal/Core/FileTransfer/`): The entire directory is removed, including the abstract `FileTransfer` base class, `FTP`, `FTPExtension`, `Local`, `SSH`, `ChmodInterface`, `FileTransferException`, and `FileTransferAuthorizeForm`.

**`Drupal\Core\Updater` namespace** (`core/lib/Drupal/Core/Updater/`): `Updater`, `Module`, `Theme`, and `UpdaterInterface` are all removed.

**Hooks** in `core/lib/Drupal/Core/Extension/module.api.php` and `core/lib/Drupal/Core/File/file.api.php`: `hook_updater_info()`, `hook_updater_info_alter()`, `hook_filetransfer_info()`, `hook_filetransfer_info_alter()`, and `hook_archiver_info_alter()` are removed.

**System module**: `system_authorized_run()` and `system_authorized_batch_process()` are removed from `system.module`. A stale `updaterInfo()` method implementing `hook_updater_info` in `SystemHooks` was caught during review and also removed.

**Tests**: All `FileTransfer` and `Updater` functional tests, test modules (`MockFileTransfer`, `TestFileTransferWithSettingsForm`, `UpdateTestArchiver`), and related fixtures are removed, including two orphaned `system.authorize.yml` config fixture files in `core/tests/fixtures/config_install/` that were missed when the config schema itself was removed in [#2715145](https://www.drupal.org/node/2715145).

**The only addition** is `update_post_update_clear_disk_cache_again()` in the `update` module. A related post-update (`update_post_update_clear_disk_cache()`) already ran in Drupal 11 to remove files written by the old code, but because the deprecated `update_clear_update_disk_cache()` function could still be called directly by third-party code in Drupal 11, a second post-update runs in Drupal 12 after all those code paths are fully gone.

The `core/lib/Drupal/Core/Archiver` namespace is intentionally kept; it is still used by the Config module's `ConfigImportForm` and `ConfigController`.

## Upgrade

Any custom or contributed code that extends or calls the removed classes or functions must be updated. There is no replacement; use Composer to manage module and theme code.

```php
// REMOVED - do not use in Drupal 12:

drupal_get_filetransfer_info();
system_authorized_run($callback, $operations_file, $url);
system_authorized_batch_process();

// REMOVED classes (Drupal\Core\FileTransfer\*):
// FileTransfer, FTP, FTPExtension, Local, SSH, ChmodInterface, FileTransferException
// FileTransferAuthorizeForm

// REMOVED classes (Drupal\Core\Updater\*):
// Updater, Module, Theme, UpdaterInterface

// REMOVED hooks (implement these no longer):

// hook_filetransfer_info(), hook_filetransfer_info_alter()
// hook_archiver_info_alter()
```

All of the above were deprecated in `drupal:11.2.0`. Remove any implementations or callers before upgrading to Drupal 12.
