## Summary

The `$password` parameter in `InstallCommand::install()` is now marked with the PHP `#[SensitiveParameter]` attribute. This prevents the password value from appearing in exception stack traces if that method throws an error during site installation, reducing the risk of accidental credential exposure in logs or error output.

## Impact

No upgrade or configuration changes required.

## Technical details

The `$password` argument of `InstallCommand::install()` in `core/lib/Drupal/Core/Command/InstallCommand.php` was missing the `#[\SensitiveParameter]` PHP 8.2 attribute. Without it, any unhandled exception thrown inside that method would include the raw password value in the stack trace. The fix adds `#[\SensitiveParameter]` directly to the parameter declaration, so PHP's exception handler redacts the value and replaces it with a `SensitiveParameterValue` wrapper object. The issue author (prudloff) also noted that the password is subsequently passed into `install_drupal()` inside the `$parameters` array, and flagged that array as a potential further hardening candidate, though that was left out of scope for this change.
