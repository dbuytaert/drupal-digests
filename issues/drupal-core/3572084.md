## Summary

Dotfiles listed under the `ignore:` key in a starterkit theme's `.starterkit.yml` were silently not excluded when generating a new theme. For example, placing `.npmrc` or `.gitlab-ci.yml` in the ignore list had no effect and those files were still copied into the generated theme. This bug is now fixed, and VCS directories such as `.git` inside a starterkit source theme are also excluded automatically.

## Impact

No upgrade or configuration changes required.

## Technical details

The bug lived in `GenerateTheme::loadStarterKitConfig()` in `core/lib/Drupal/Core/Command/GenerateTheme.php`. That method uses a Symfony `Finder` instance (from `self::createFilesFinder()`) to resolve the file lists declared in each key of the `.starterkit.yml` config, including the `ignore:` key. Symfony `Finder` silently skips dotfiles by default (`ignoreDotFiles(TRUE)`), so when a theme author listed `.npmrc` in `ignore:`, the Finder never matched it. The resulting ignore list was empty for those entries, and the copy step proceeded to include the dotfiles.

The fix chains two additional calls onto the `Finder` instance: `->ignoreDotFiles(FALSE)` so that dotfiles are visible to the Finder and can be matched against the ignore patterns, and `->ignoreVCS(TRUE)` to explicitly exclude `.git` directories, which would otherwise be picked up now that dotfile discovery is enabled.

Two new functional tests are added to cover the corrected behaviour: `testIgnoredDotFiles()` verifies that a `.npmrc` file listed under `ignore:` does not appear in the generated theme, and `testExcludedGitFolder()` verifies that a `.git` directory present in the starterkit source is not copied to the generated theme.
