## Summary

When the Navigation module's sidebar is collapsed on mobile-width viewports, all its menu items remained reachable via keyboard Tab and were still listed in screen reader rotors such as VoiceOver. Focus would visually disappear as it moved into the hidden sidebar, violating WCAG 2.1.1 (Keyboard), 2.4.11 (Focus Not Obscured), 1.3.1, and 1.3.2. After this fix, tabbing past the "Expand sidebar" button skips the closed sidebar entirely, and collapsed items no longer appear in the aural interface. The desktop sidebar is unaffected.

## Impact

No upgrade or configuration changes required.

## Technical details

The root cause was that the collapsed mobile sidebar used only a CSS `transform` to slide it off screen, leaving its contents fully in the accessibility tree and tab order.

The fix adds `visibility: hidden` to `.admin-toolbar` at non-desktop breakpoints (`@media not (min-width: 64rem)`) by default, and restores `visibility: visible` when the `[data-admin-toolbar="expanded"]` attribute is present. Crucially, `visibility` is added alongside `transform` in the element's `transition` declaration, so the open/close slide animation is preserved: CSS allows transitioning `visibility` (it snaps at the transition boundary, so the sidebar becomes hidden only after the close animation completes and becomes visible immediately when opening).

A secondary bug was also corrected in the same change: `--admin-toolbar-sidebar-width` was previously defined only inside `[data-admin-toolbar="expanded"]`, meaning the collapsed sidebar had the wrong width for its breakpoint. The custom property is now set unconditionally per breakpoint (80vw at narrow, `calc(16.5 * var(--admin-toolbar-rem))` at tablet, `calc(4.5 * var(--admin-toolbar-rem))` at desktop), with the expanded-state override (`calc(16.5 * var(--admin-toolbar-rem))`) applied only on desktop. These changes are in `navigation.module.css` (the PostCSS source) and the compiled output.

### Collaboration

The thread went through two distinct approaches before landing. After the issue was filed by kentr, finnsky pointed out that a naive `visibility: hidden` or `display: none` would break the collapse animation, and suggested using the HTML `inert` attribute instead, citing a prior use in the module. sandip implemented the `inert`-via-JS approach, but mherchel set it back to Needs Work after discovering that at wide widths the sidebar was incorrectly becoming inert and inoperable; nod_ noted that the existing test suite caught the regression. kentr then proposed transitioning `visibility` in CSS, shared a working devtools snippet demonstrating it preserved the animation, and opened MR !14200 with that approach after mherchel encouraged them to do so. During testing of that MR, finnsky caught and fixed the separate `--admin-toolbar-sidebar-width` width calculation bug. rkoller, who had originally reported the aural interface symptom as a duplicate issue ([#3543541](https://www.drupal.org/node/3543541)), tested the final result across Safari, Firefox, and Edge on macOS including VoiceOver, confirming the fix resolved both the keyboard and screen reader problems.
