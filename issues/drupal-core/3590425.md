## Summary

In Drupal's PHPUnit test infrastructure, the `$root` property on `DrupalTestCaseTrait` was previously initialized via a PHP 8.4 property hook. That hook has been replaced with a plain property initialized by a new `#[Before(100)]` method called `setUpRoot()`. The change makes the code backportable to 11.x (where property hooks are not available) and ensures `$root` is reliably populated before any other `#[Before]` hooks or `setUp()` calls execute. This is a `main`-only change; backporting is tracked in [#3590435](https://www.drupal.org/node/3590435).

## Impact

No upgrade or configuration changes required.

## Technical details

The previous implementation used a PHP 8.4 hooked property: `protected string $root { get { ... } }`, which initialized lazily on first read. This required a `phpcs:ignoreFile` directive because PHP_CodeSniffer did not yet support property hook syntax. The replacement uses a plain `protected string $root;` declaration combined with a `final #[Before(100)]` method `setUpRoot()` that performs eager initialization before any other test setup runs. The high priority value `100` ensures `setUpRoot()` fires before other `#[Before]` hooks and before `setUp()`.

The method is declared `final` to block subclasses from overriding it, which would bypass the single intended initialization. If `$root` is already set when `setUpRoot()` is invoked, it throws a `\LogicException` with the message "setUpRoot should be called exactly once by PHPUnit's Before test hook and root overrides should happen after." This guards against pre-assignment of `$root` in a subclass constructor or property initializer. The method is marked `@internal`.

This change is part of a broader consolidation of test base-class logic into `DrupalTestCaseTrait` ([#3573954](https://www.drupal.org/node/3573954)) and the deprecation of `TestRequirementsTrait::getDrupalRoot()` ([#3589047](https://www.drupal.org/node/3589047)). The plan documented in the issue body is to re-introduce a property hook once backporting to 11.x is no longer needed, at which point `setUpRoot()` can be removed.

## Upgrade

If a custom or contrib test class that uses `DrupalTestCaseTrait` (or a base class that uses it) assigns `$this->root` or declares `protected string $root = '...'` before PHPUnit runs the `#[Before]` hooks, that test will now throw a `\LogicException`. Remove any such pre-assignment; `$root` is initialized automatically by `setUpRoot()` before `setUp()` runs.

```php
// Before (would now cause a LogicException at runtime):
class MyTest extends BrowserTestBase {
  protected string $root = '/custom/path'; // pre-assignment
}

// After: do not pre-set $root; override it in setUp() after parent::setUp() if needed.
```
