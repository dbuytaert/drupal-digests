## Summary

Drupal's installer and installation documentation no longer mention Percona Server as a supported MySQL-compatible database. The database driver label displayed during installation has been simplified to "MySQL, MariaDB, or equivalent" (or "MySQL or equivalent" when a live MariaDB connection is detected). Percona Server was removed because Drupal core does not test against it in CI; MySQL and MariaDB remain fully supported.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue originally requested that MySQL be de-recommended in favour of MariaDB, backed by informal benchmarks. After discussion among release managers (quietone, catch, longwave), the consensus was narrower: Drupal cannot drop MySQL because it is still widely used and is exercised in core CI, but Percona Server should be removed because it is not tested in CI and has known issues (e.g. lock problems on Percona XtraDB cluster, referenced in [#3143562](https://www.drupal.org/node/3143562)). The "recommendation" in the heading was understood to mean the MySQL-family versus PostgreSQL or SQLite, not MySQL specifically over MariaDB.

Three files were changed in the commit:

`core/INSTALL.txt`: The Percona Server entry (along with its "backwards-compatible replacement for MySQL" description) was removed from the database requirements list. The MariaDB entry's "fully compatible drop-in replacement" description was also stripped, leaving only the bare version requirement line.

`core/modules/mysql/src/Driver/Database/mysql/Install/Tasks.php`: The `name()` method previously returned `'MySQL, Percona Server, or equivalent'` when a live non-MariaDB connection was active, and `'MySQL, MariaDB, Percona Server, or equivalent'` when no connection was available (e.g. during pre-install). Both strings were updated to remove "Percona Server", yielding `'MySQL or equivalent'` and `'MySQL, MariaDB, or equivalent'` respectively.

`core/modules/mysql/tests/src/Unit/InstallTasksTest.php` and `core/tests/Drupal/FunctionalTests/Installer/InstallerTest.php`: Assertions were updated to match the new strings.
