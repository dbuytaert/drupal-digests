## Summary

Drupal continues modernizing its codebase by moving form submission logic out of procedural functions and into object-oriented hook classes. The Contact module now handles form submissions for user profile and account settings through methods in its hooks class. This makes the code more maintainable and follows current best practices. Site functionality remains unchanged, but custom code calling the old functions should update to use the new class methods.

## Impact

- **Deprecation:** Two procedural functions `contact_user_profile_form_submit()` and `contact_form_user_admin_settings_submit()` deprecated in favor of `ContactFormHooks` class methods.

## Technical details

This change moves two form submission handlers from `contact.module` to the `ContactFormHooks` class. The `contact_user_profile_form_submit()` function becomes `ContactFormHooks::profileFormSubmit()`, and `contact_form_user_admin_settings_submit()` becomes `ContactFormHooks::userAdminSettingsSubmit()`.

The old procedural functions are now deprecated but remain in place as wrappers. They trigger a deprecation notice and delegate to the hook class methods via service instantiation using `\Drupal::service(ContactFormHooks::class)`. The actual form alter hooks now register the class methods as submit handlers using `self::class . ':profileFormSubmit'` instead of the function name strings.

The implementation also includes a detailed comment explaining why `#config_target` cannot be used for the user admin settings form. During validation, `#config_target` triggers validation of the entire `contact.settings` config object, but the default `default_form` value of 'feedback' references a contact form entity that does not ship with the module, causing validation failures.

A minor bug was also fixed: the `ContactFormHooks` constructor had `configFactoryInterface` (lowercase 'c') corrected to `ConfigFactoryInterface`.

## Upgrade

If your custom code directly calls `contact_user_profile_form_submit()` or `contact_form_user_admin_settings_submit()`, update to use the `ContactFormHooks` service methods instead.

```php
// Old:
contact_user_profile_form_submit($form, $form_state);

// New:
\Drupal::service(ContactFormHooks::class)->profileFormSubmit($form, $form_state);
```
