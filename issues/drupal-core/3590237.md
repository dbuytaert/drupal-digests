## Summary

Drupal core now ships `symfony/polyfill-php86` (v1.37), making two PHP 8.6 additions available on sites running older PHP versions: the `clamp()` function and the `SortDirection` enum. Contrib module authors and custom code developers can use these features today without gating on the PHP version of the host environment.

## Impact

Affects module maintainers.

- **New API:** PHP 8.6 `clamp()` function and `SortDirection` enum are now available on all supported PHP versions via `symfony/polyfill-php86` v1.37

## Technical details

The diff adds `"symfony/polyfill-php86": "^1.37"` to `core/composer.json` and pins it to `~v1.37.0` in the root `composer/composer.json`. The package's `bootstrap.php` is autoloaded via Composer's `files` mechanism, so the polyfill is always active regardless of the PHP version. On PHP 8.6+ the package is a no-op; on earlier versions it provides the `clamp()` global function and the `SortDirection` enum stub under `Symfony\Polyfill\Php86`. No existing core code was changed to use these symbols in this commit; the dependency was added to unblock contrib and to simplify backporting MRs that target `main` (or `11.x`) to older branches.
