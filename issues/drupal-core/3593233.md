## Summary

Cloning an aggregate entity query now correctly isolates the clone's aggregate condition tree from the original. Before this fix, both the original query and its clone shared the same `ConditionAggregate` object, so adding conditions to either one silently changed both, and the cloned query could execute SQL built against the wrong query object. The fix mirrors the existing clone handling for regular conditions and also extends `ConditionAggregateInterface` with `setQuery()` and `getQuery()` methods.

## Impact

Affects module maintainers.

- **API change:** `ConditionAggregateInterface` gains two new required methods: `setQuery(QueryInterface $query): static` and `getQuery(): QueryInterface`.

## Technical details

The root cause was in `QueryBase::__clone()`, which cloned `$condition` but left `$conditionAggregate` pointing at the same instance. The SQL-layer `Query::__clone()` then called `$this->condition->setQuery($this)` to re-parent the cloned regular condition, but did nothing equivalent for `$conditionAggregate`.

The fix adds a symmetric `clone $this->conditionAggregate` call in `QueryBase::__clone()` (guarded by `isset()` since non-aggregate queries never set the property), and a matching `$this->conditionAggregate->setQuery($this)` call in the SQL `Query::__clone()` override. Because `setQuery()` was previously absent from `ConditionAggregateInterface`, the fix also formally adds `setQuery(QueryInterface $query): static` and `getQuery(): QueryInterface` to that interface. Both methods are already implemented in `ConditionFundamentals`, so existing classes that extend it are unaffected at runtime, but any class that implements `ConditionAggregateInterface` directly without extending `ConditionFundamentals` must now provide these two methods. A new functional test `testCloneAggregateIsolation()` in the SQL aggregate query test class covers the before/after behavior.

## Upgrade

If you have a custom class that implements `ConditionAggregateInterface` directly (without extending `ConditionFundamentals`), add the two new required methods:

```php
public function setQuery(QueryInterface $query): static {
    $this->query = $query;
    return $this;
}

public function getQuery(): QueryInterface {
    return $this->query;
}
```
Classes that already extend `ConditionFundamentals` inherit these implementations and need no changes.
