## Summary

PHPDoc annotations on `\Drupal::classResolver()` and `ClassResolverInterface::getInstanceFromDefinition()` now use PHPStan generics and conditional return types, so static analysis tools and IDEs can infer the concrete class type when a class-string literal is passed. Before this change, both methods returned an untyped `object` (or `ClassResolverInterface`) regardless of argument, forcing callers to add manual `/** @var MyClass $x */` casts. After this change those casts are unnecessary. This mirrors the equivalent fix applied to `\Drupal::service()` in [#3490713](https://www.drupal.org/node/3490713).

## Impact

Affects module maintainers.

- **API change:** `\Drupal::classResolver()` and `ClassResolverInterface::getInstanceFromDefinition()` now carry PHPStan generic/conditional return-type annotations; callers passing a class-string literal get the concrete class inferred instead of `object`.

## Technical details

In `Drupal::classResolver()`, the `@param` is now typed `class-string<T>|null $class` with a `@template T of object` declaration, and the `@return` uses the conditional form `($class is class-string<T> ? T : \Drupal\Core\DependencyInjection\ClassResolverInterface)`. In `ClassResolverInterface::getInstanceFromDefinition()`, the same pattern is applied with `class-string<T>|string $definition` (no null, since that method does not accept null) and `@return ($definition is class-string<T> ? T : object)`.

As a side-effect, four call sites in `layout_builder`'s `LayoutBuilderHooks` that previously needed explicit `@var` cast annotations (for `InlineBlockEntityOperations` and `LayoutOverrideFieldHelper`) were simplified to direct method chaining, because PHPStan now resolves the type automatically from the `class-string` literal passed to `classResolver()`.

## Upgrade

No runtime behavior changed. If your code uses `\Drupal::classResolver(MyClass::class)` or `ClassResolverInterface::getInstanceFromDefinition(MyClass::class)` and carries an inline `@var` cast to suppress PHPStan errors, that cast can now be removed:

```php
// Before
/** @var \My\Module\MyClass $instance */
$instance = \Drupal::classResolver(\My\Module\MyClass::class);
$instance->doSomething();

// After (PHPStan infers \My\Module\MyClass automatically)
\Drupal::classResolver(\My\Module\MyClass::class)->doSomething();
```
