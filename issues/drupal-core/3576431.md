## Summary

The Admin theme in Drupal core was incorrectly marked as experimental using an outdated metadata format. The theme used `experimental: true` in its configuration file, but the correct format is `lifecycle: experimental`. This change updates the Admin theme to use the proper lifecycle metadata syntax. The visual result on the Appearance page remains the same, showing the experimental status correctly. This is a metadata correction that ensures the theme follows Drupal's current standards for marking experimental features.

## Impact

No upgrade or configuration changes required.

## Technical details

The Admin theme's `admin.info.yml` file contained `experimental: true`, which is an obsolete way to mark a theme as experimental. Drupal's current standard uses the `lifecycle` key with a value of `experimental` instead.

This change updates the single line in `admin.info.yml` from `experimental: true` to `lifecycle: experimental`. Both formats display the experimental status on the Appearance page, but the lifecycle format is the current standard used across Drupal core.

The lifecycle metadata system is used to communicate the maturity level of themes and modules. Using the correct format ensures consistency with other experimental components in core and makes the status machine-readable in the expected format.

This is a metadata-only change with no functional impact. The theme continues to work exactly the same way, and the experimental warning continues to appear on the Appearance page. The change simply corrects the metadata format to match current Drupal standards.
