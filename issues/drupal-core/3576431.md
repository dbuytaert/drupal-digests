## Summary

The admin theme's metadata file incorrectly used an outdated `experimental: true` flag instead of the current `lifecycle: experimental` standard. This caused the theme to display incorrectly on the Appearance page. The fix updates the theme metadata to use the proper lifecycle indicator, ensuring the experimental status appears correctly in the user interface. This change only affects how the admin theme's experimental status is labeled on the Appearance page.

## Impact

No upgrade or configuration changes required.

## Technical details

Drupal's extension metadata format evolved to use a standardized `lifecycle` key for indicating development status rather than the older `experimental: true` boolean flag. The admin theme's `admin.info.yml` file still used the deprecated format.

The fix updates the single line in the theme info file from `experimental: true` to `lifecycle: experimental`. This aligns with the current metadata standard used across Drupal core extensions. The lifecycle key supports multiple values (experimental, deprecated, obsolete) and integrates with the extension system's display logic.

The visual difference is subtle but important for consistency. The before screenshot showed the experimental label rendered using the old system, while the after screenshot shows it using the modern lifecycle-aware rendering. This ensures the admin theme follows the same conventions as other core themes and modules.
