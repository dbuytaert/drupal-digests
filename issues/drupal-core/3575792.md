## Summary

Drupal core has updated its PHPStan static analysis tool and related extensions to their latest versions. PHPStan is a development tool that helps find bugs in code before running it. The update includes support for property hooks, a new PHP 8.4 feature. This change improves the quality of automated code analysis during Drupal development. The update also revealed additional code issues that were added to the baseline for future cleanup.

## Impact

No upgrade or configuration changes required.

## Technical details

This issue updates PHPStan from version 2.1.38 to 2.1.40 in the main branch (11.x received a similar update). It also updates `mglaman/phpstan-drupal` to 2.0.11, which adds support for parsing PHP 8.4 property hooks, and `phpstan/phpstan-phpunit` to 2.0.16. The `phpstan-deprecation-rules` extension was updated to 2.0.4 as well.

The update exposed new errors that PHPStan now detects. These were added to the PHPStan baseline file rather than being fixed immediately. The new errors include warnings about variables in `empty()` checks that always exist and are always falsy, missing return type specifications on several methods, and missing generic type specifications for certain interfaces.

One quirk encountered in the 11.x branch: PHPStan incorrectly reported that non-capturing catch blocks were unsupported, even though Drupal requires PHP 8.3+. This was worked around by adding a variable to the catch block and suppressing the corresponding PHPCS error.
