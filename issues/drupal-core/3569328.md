## Summary

The Locale module now uses modern services instead of procedural functions in its translation management system. Functions that handled translation source files, project data, and update options have been reorganized into reusable service classes. This change reduces circular dependencies in the module and makes the code easier to maintain. Site owners who use the Locale module should see no visible changes in functionality.

## Impact

- **Deprecation:** Functions `locale_translation_get_projects()`, `locale_translation_clear_cache_projects()`, `locale_translation_load_sources()`, `locale_translation_build_sources()`, `locale_translation_source_check_file()`, `locale_translation_source_build()`, `locale_translation_build_server_pattern()`, `locale_cron_fill_queue()`, `_locale_translation_file_is_remote()`, `_locale_translation_source_compare()`, and `_locale_translation_default_update_options()` deprecated. Constants `LOCALE_TRANSLATION_SOURCE_COMPARE_LT`, `LOCALE_TRANSLATION_SOURCE_COMPARE_EQ`, and `LOCALE_TRANSLATION_SOURCE_COMPARE_GT` deprecated without replacement. File `locale.translation.inc` deprecated.
- **New API:** New `LocaleSource` service and `LocaleDefaultOptions` class; new `LocaleProjectStorageInterface::getProjects()` method.

## Technical details

The issue converted 11 procedural functions from `locale.translation.inc` into service-based architecture. The main changes organize code into three areas. First, a new `LocaleSource` service encapsulates functions related to building and checking translation source files: `loadSources()`, `buildSources()`, `sourceCheckFile()`, `sourceBuild()`, and `buildServerPattern()`. Second, the existing `LocaleProjectStorage` service gained a new `getProjects()` method that replaces `locale_translation_get_projects()` and handles its own caching through a `$projectCache` property. Third, a new `LocaleDefaultOptions` class provides a static `updateOptions()` method for translation import settings.

The cron queue population logic from `locale_cron_fill_queue()` moved directly into `LocaleHooks::cron()`. The function `_locale_translation_source_compare()` was inlined in `TranslationStatusForm::prepareUpdateData()` as a simple timestamp comparison. Helper function `_locale_translation_file_is_remote()` became a protected method in `LocaleSource`. Three comparison constants (`LOCALE_TRANSLATION_SOURCE_COMPARE_LT`, `EQ`, `GT`) were deprecated without replacement since they are only used in the inlined comparison logic.

All deprecated functions now trigger deprecation errors and proxy to the new service methods. The deprecated functions targeting removal in 12.0.0 are prefixed with underscore (internal use). Functions in the public API target removal in 13.0.0. The entire `locale.translation.inc` file triggers a deprecation notice when included. Calls to `drupal_static_reset('locale_translation_get_projects')` now trigger deprecation and call `LocaleProjectStorage::resetCache()`.

The `LocaleProjectStorage::resetCache()` method now clears both the existing `$cache` property and the new `$projectCache` property. Multiple files that previously required `locale.translation.inc` now use dependency injection or service locator calls instead.

## Upgrade

Module maintainers who call deprecated functions should update to the new services:

Replace `locale_translation_get_projects()` with:
```php
\Drupal::service('locale.project')->getProjects($project_names);
```

Replace `locale_translation_clear_cache_projects()` with:
```php
\Drupal::service('locale.project')->resetCache();
```

Replace `locale_translation_build_sources()` with:
```php
\Drupal::service(LocaleSource::class)->buildSources($projects, $langcodes);
```

Replace `locale_translation_source_build()` with:
```php
\Drupal::service(LocaleSource::class)->sourceBuild($project, $langcode, $filename);
```

Replace `_locale_translation_default_update_options()` with:
```php
LocaleDefaultOptions::updateOptions();
```

See the change record at https://www.drupal.org/node/3569330 for the complete list of replacements.
