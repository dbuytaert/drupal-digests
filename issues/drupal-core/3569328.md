## Summary

The Locale module's translation functions have been converted to modern services. This modernization reduces circular dependencies in the codebase and follows Drupal's architectural direction of moving procedural code into object-oriented services. The change affects how translation sources are built and managed internally but does not change how translations work for site owners.

## Impact

Affects module maintainers.

- **Deprecation:** Functions `locale_translation_get_projects()`, `locale_translation_clear_cache_projects()`, `locale_translation_load_sources()`, `locale_translation_build_sources()`, `locale_translation_source_check_file()`, `locale_translation_source_build()`, `locale_translation_build_server_pattern()` deprecated; underscore functions `_locale_translation_file_is_remote()`, `_locale_translation_source_compare()`, `_locale_translation_default_update_options()` and `locale_cron_fill_queue()` deprecated; constants `LOCALE_TRANSLATION_SOURCE_COMPARE_LT`, `LOCALE_TRANSLATION_SOURCE_COMPARE_EQ`, `LOCALE_TRANSLATION_SOURCE_COMPARE_GT` deprecated; entire `locale.translation.inc` file deprecated
- **New API:** New `LocaleSource` service provides translation source building methods; new `LocaleDefaultOptions` class provides translation update options; `LocaleProjectStorageInterface::getProjects()` added

## Technical details

The locale.translation.inc file contained 11 procedural functions at the bottom of a chain of circular dependencies. This change converts them to two new services and deprecates the entire file.

The new `LocaleSource` service provides methods for building and checking translation sources: `loadSources()`, `buildSources()`, `sourceCheckFile()`, `sourceBuild()`, and `buildServerPattern()`. The `LocaleProjectStorageInterface` gains a new `getProjects()` method that replaces the static-cached `locale_translation_get_projects()` function. The `LocaleDefaultOptions` class provides a static `updateOptions()` method for translation import options.

The `locale_cron_fill_queue()` function's logic was moved directly into `LocaleHooks::cron()` to eliminate an unnecessary layer. The `_locale_translation_source_compare()` function was inlined in `TranslationStatusForm` where it had only one usage, simplifying from a comparison constant system to direct timestamp comparison.

All deprecated functions in locale.translation.inc forward to the new service methods. Non-API functions (those prefixed with underscore) are removed in Drupal 12, while public API functions are removed in Drupal 13. The `LocaleProjectStorage` service now maintains its own `projectCache` property separate from the drupal_static system. A backward compatibility layer in `drupal_static_reset()` forwards calls for `locale_translation_get_projects` to the new service method.

## Upgrade

Module developers using deprecated locale translation functions should update to the new services:

```php
// Old:
$projects = locale_translation_get_projects();
locale_translation_clear_cache_projects();
$sources = locale_translation_build_sources($projects, $langcodes);

// New:
$projects = \Drupal::service('locale.project')->getProjects();
\Drupal::service('locale.project')->resetCache();
$sources = \Drupal::service(LocaleSource::class)->buildSources($projects, $langcodes);
```

For translation update options:
```php
// Old:
$options = _locale_translation_default_update_options();

// New:
use Drupal\locale\LocaleDefaultOptions;
$options = LocaleDefaultOptions::updateOptions();
```

See the change record at [https://www.drupal.org/node/3569330](https://www.drupal.org/node/3569330) for complete details.
