## Summary

The core admin theme was renamed from "Admin" to "Default Admin" with machine name `default_admin`. The word "admin" appears over 20,000 times across nearly 2,000 files in core, making it difficult to grep for theme-specific code. The new name `default_admin` has only 2 hits in core and 45 across contrib, making it much easier to find references during development and debugging.

## Impact

Affects module maintainers.

- **API change:** Theme machine name changed from `admin` to `default_admin`; template namespace changed from `@admin` to `@default_admin`; config schema changed from `admin.settings` to `default_admin.settings`

## Technical details

The theme machine name changed from `admin` to `default_admin`. This affected the theme directory path (`themes/admin` to `themes/default_admin`), all config files (`admin.settings.yml` to `default_admin.settings.yml`), block IDs (e.g., `admin_breadcrumbs` to `default_admin_breadcrumbs`), and template namespaces (`@admin` to `@default_admin` in extends statements). The user-facing theme name changed from "Admin" to "Default Admin" throughout the UI and config schema labels.

The primary motivation was developer experience. Searching for "admin" returns 20,000 results across 1,982 files, making it nearly impossible to find references to the theme itself. The compound name `default_admin` was chosen because it returns only 2 hits in core and 45 in contrib, making code searches and mass renaming operations practical.

## Upgrade
