## Summary

Drupal previously checked every page request to determine whether to display a "back" link on the block region demonstration page. This created unnecessary overhead for all page loads. The change moves this logic directly into the block demonstration controller, so the check only runs when actually viewing that specific page. This improves performance by eliminating a route check on every request.

## Impact

No upgrade or configuration changes required.

## Technical details

The `BlockHooks::pageTop()` method was being invoked on every page request via `hook_page_top()`. Inside this hook, it checked whether the current route was `block.admin_demo` before conditionally adding a "back" link to the page. This meant every request performed a route match check even though the vast majority of requests were not for the block demo page.

The change removes this hook implementation entirely. Instead, the logic now lives directly in `BlockController::demo()`, which only executes when visiting the block region demonstration page. The backlink is now attached using `#attached['page_top']`, a capability added in [#3339905](https://www.drupal.org/node/3339905).

The controller constructor was modernized to use promoted readonly properties and now injects `ThemeManagerInterface` in addition to `ThemeHandlerInterface`. The backlink building logic remains functionally identical but is now triggered only when needed.
