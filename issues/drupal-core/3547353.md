## Summary

The comment system now uses a modern enum to represent whether comments are open, closed, or hidden. This replaces older numeric constants with clearer, type-safe code that is easier to understand and maintain. The change makes comment status values consistent with how other parts of Drupal handle options. Sites and modules can continue using the old constants during the transition period, but should update to the enum for future compatibility.

## Impact

Affects module maintainers.

- **Deprecation:** Constants `CommentItemInterface::HIDDEN`, `CommentItemInterface::CLOSED`, and `CommentItemInterface::OPEN` deprecated in favor of `CommentingStatus` enum cases.
- **New API:** New `CommentingStatus` enum with `Hidden`, `Closed`, and `Open` cases provides type-safe comment field status values.

## Technical details

Three class constants on `CommentItemInterface` (`HIDDEN`, `CLOSED`, `OPEN`) have been deprecated and replaced with a new `CommentingStatus` enum. The enum defines three cases: `Hidden` (0), `Closed` (1), and `Open` (2), matching the original constant values for backward compatibility.

Throughout core, status checks now compare against enum cases rather than integer constants. The enum uses `OptionsEnumTrait` to provide translatable labels and an `asOptions()` method for form element options. Widget and filter classes use `CommentingStatus::asOptions()` instead of manually building option arrays.

Most code reads the status value from the field and converts it to an enum case using `CommentingStatus::tryFrom()`, then compares that case. When writing values, code uses `->value` to get the integer representation. The `CommentTestTrait::addDefaultCommentField()` method signature now accepts either a `CommentingStatus` enum case or an integer for the default value parameter, checking the type and converting integers to enum cases.

The interface constants remain but are marked deprecated with `@see` tags pointing to the change record. They will be removed in Drupal 13.0.0.

## Upgrade

Replace references to `CommentItemInterface::OPEN`, `CommentItemInterface::CLOSED`, and `CommentItemInterface::HIDDEN` with the new enum:

```php
// Before
use Drupal\comment\Plugin\Field\FieldType\CommentItemInterface;
$status = CommentItemInterface::OPEN;
if ($entity->comment->status == CommentItemInterface::CLOSED) { }

// After
use Drupal\comment\CommentingStatus;
$status = CommentingStatus::Open->value;
if (CommentingStatus::tryFrom((int) $entity->comment->status) == CommentingStatus::Closed) { }
```

For form element options:

```php
// Before
'#options' => [
  CommentItemInterface::OPEN => $this->t('Open'),
  CommentItemInterface::CLOSED => $this->t('Closed'),
  CommentItemInterface::HIDDEN => $this->t('Hidden'),
],

// After
'#options' => CommentingStatus::asOptions(),
```

When creating entities or fields with a default status value, pass the enum case's value property or use the enum directly where supported.
