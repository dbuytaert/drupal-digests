## Summary

The mobile Navigation sidebar can now be closed by pressing the Escape key, meeting accessibility requirements. Previously, keyboard users had no way to close the mobile sidebar with the Escape key, creating an inequitable experience compared to mouse users. The fix implements a hierarchical behavior: Escape closes submenus first, and if no submenu is open, it closes the entire sidebar and returns focus to the "Expand sidebar" button. This completes a critical accessibility gap identified in the Navigation module's accessibility review.

## Impact

No upgrade or configuration changes required.

## Technical details

This change adds two key behaviors to the Navigation module's mobile sidebar. First, in `admin-toolbar-wrapper.js`, a new `keydown` event listener detects the Escape key when the sidebar is expanded and triggers the mobile toggle button to close it. When the sidebar closes via user input, focus moves to the "Expand sidebar" button, following WCAG best practices for focus management in slide-out navigation.

Second, the `arrow-navigation.js` file now implements hierarchical Escape key handling. The code checks if the user is inside a popover submenu before closing. If they are inside a popover that is open or on a non-trigger element, Escape closes only that submenu using `stopPropagation()` to prevent the event from bubbling to the parent handler. If no submenu is open, the Escape key bubbles to the document-level handler and closes the entire sidebar.

The template adds `data-drupal-selector="admin-toolbar-mobile-trigger"` to identify the expand button for focus management. New Nightwatch tests verify both wide and narrow viewport behaviors, including focus return to the trigger button. This incorporates code from issue [#3541741](https://www.drupal.org/project/drupal/issues/3541741) that fixed submenu Escape handling.
