## Summary

This change moves XSS filtering logic for text editors from a procedural function into a service method. The existing `editor_filter_xss()` function is deprecated in favor of `Element::filterXss()` accessed through the `element.editor` service. This is part of the ongoing effort to eliminate procedural code from Drupal core's `.module` files and adopt modern object-oriented patterns. The filtering behavior remains identical, but now uses dependency injection instead of static service calls.

## Impact

Affects module maintainers.

- **Deprecation:** `editor_filter_xss()` deprecated; use `Element::filterXss()` from the `element.editor` service instead.

## Technical details

The `editor_filter_xss()` function was converted to a public method `filterXss()` on the `Drupal\editor\Element` service class. The entire filtering logic was moved from `editor.module` into the service, including checking whether XSS filtering is needed, determining if the text editor is XSS-safe, verifying filter types, and applying the `editor_xss_filter` alter hook.

The deprecated function now wraps the service method with a deprecation notice. The `element.editor` service constructor gained a second parameter for `ModuleHandlerInterface`, enabling the alter hook to be called through dependency injection instead of `\Drupal::moduleHandler()`.

`EditorController` was updated to accept the `element.editor` service via constructor injection and call `$this->editorElement->filterXss()` instead of the procedural function. Both `Element` and `EditorController` constructors include backward compatibility fallbacks that trigger deprecation notices when the new parameters are missing, ensuring existing code continues to work during the deprecation period.

The change preserves all existing logic, including the security checks, format switching behavior, and the ability for modules to alter the XSS filter class through `hook_editor_xss_filter_alter()`.

## Upgrade

Replace calls to `editor_filter_xss()` with the service method:

```php
// Before
$filtered = editor_filter_xss($html, $format, $original_format);

// After
$filtered = \Drupal::service('element.editor')->filterXss($html, $format, $original_format);
```

If you extend `EditorController`, update your constructor to accept the new parameter:

```php
public function __construct(protected ?Element $editorElement = NULL) {
  parent::__construct($editorElement);
}
```
