## Summary

The standard install profile and its accompanying recipe no longer depend on or install the comment module. Sites installed fresh from the standard profile will not have commenting functionality enabled by default. Site builders who want comments on their content must now explicitly install the comment module and configure it themselves. This is a follow-up to [#3587118](https://www.drupal.org/node/3587118), which removed the built-in Article and Basic page content types from the standard profile as part of a broader effort to slim down what the profile installs.

## Impact

Affects site owners.

- **Config change:** Standard profile no longer installs the comment module or any comment configuration; comment-related permissions are removed from the anonymous, authenticated, and content_editor roles.

## Technical details

The commit deletes all comment-related configuration that the standard profile previously shipped: `comment.type.comment`, `core.entity_form_display.comment.comment.default`, `core.entity_view_display.comment.comment.default`, and `field.field.comment.comment.comment_body`. The `comment` module entry is removed from the `install` list in `standard.info.yml`.

Role configuration is also cleaned up. The `user.role.anonymous` role loses the `comment` module dependency and the `access comments` permission. The `user.role.authenticated` role loses the `comment` dependency along with `access comments`, `post comments`, and `skip comment approval` permissions. The `user.role.content_editor` role loses the `comment` dependency and `edit own comments`.

Test files are updated accordingly: `BigPipeTest` drops the `post comments` and `skip comment approval` permissions from its test user, and a stale parenthetical referring to "no comment form on test content type" is removed from a comment in that test. Performance baseline counts in `NodePageCacheTest` are adjusted downward to reflect the removal of the comment type config query and reduced cache lookups. The standard profile config synchronisation test in `StandardTest` removes expected comment-related config entries (`comment.settings`, `comment.type.comment`, `views.view.comment`, `views.view.comments_recent`, related field and display config, and comment system actions) from its changelist assertions.
