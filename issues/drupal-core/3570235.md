## Summary

The Syslog module's two remaining procedural functions are now deprecated because they were only used internally by the module itself. The module has been refactored to use modern dependency injection and config targets, making the configuration form simpler and more maintainable. This cleanup removes unnecessary public API surface while keeping all functionality working exactly as before. No contributed modules were found to be using these functions, so the deprecation has no known external impact.

## Impact

Affects module maintainers.

- **Deprecation:** `syslog_facility_list()` and `syslog_logging_settings_submit()` deprecated without replacement; functionality moved to protected methods in `SyslogHooks` class

## Technical details

Two functions in `syslog.module` have been deprecated: `syslog_facility_list()` and `syslog_logging_settings_submit()`. Both were only used internally by the Syslog module to build and process its configuration form.

The functionality has been moved into the `SyslogHooks` class. The facility list is now a protected `facilityList()` method called directly from the form alter hook. The submit handler is no longer needed because the form now uses config targets (`#config_target`) to automatically save values to `syslog.settings` configuration.

Additional improvements include proper dependency injection of `ModuleHandlerInterface` into the hooks class, and removal of manual configuration loading and default value setting that config targets handle automatically. The form now relies on the Form API's built-in config target system to handle both populating default values and saving submitted values.

These functions are deprecated without replacement since they were never intended as public API and no known external usage exists.

## Upgrade

If you are calling `syslog_facility_list()` or `syslog_logging_settings_submit()`, you must update your code. These functions are deprecated in Drupal 11.4.0 and will be removed in Drupal 12.0.0 and 13.0.0 respectively.

There is no direct replacement. If you need a list of syslog facilities, implement your own array:

```php
$facilities = [
  LOG_LOCAL0 => 'LOG_LOCAL0',
  LOG_LOCAL1 => 'LOG_LOCAL1',
  // ... etc
];
```

If you were relying on the submit handler, use config targets in your form instead:

```php
$form['my_field'] = [
  '#type' => 'textfield',
  '#config_target' => 'my_module.settings:my_key',
];
```

See [#3566774](https://www.drupal.org/node/3566774) for details.
