## Summary

This is a test-only cleanup with no effect on production behavior. It updates unit tests in the Theme library to stay compatible with PHPUnit 12.5, which will deprecate mock objects that carry no expectations and will deprecate `$this->any()` as an expectation matcher.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces two related changes: mock objects that configure no expectations must be replaced with stubs (or PHPUnit will emit a notice), and `$this->any()` is deprecated because it is semantically equivalent to having no expectation. The commit addresses both in eleven test files under `core/tests/Drupal/Tests/Core/Theme/`.

The main mechanical changes are:

1. **`createMock()` replaced with `createStub()`** for all double objects that are injected but never have explicit `expects(...)` calls. Affected files include `RegistryTest.php` (eight properties: `$cache`, `$lock`, `$moduleHandler`, `$themeHandler`, `$runtimeCache`, `$themeManager`, `$moduleList`, `$kernel`), as well as various Icon subsystem tests (`IconExtractorSettingsFormTest`, `IconExtractorTest`, `IconFinderTest`, `IconPackManagerTest`, `ExtractorTest`, `PathExtractorTest`, `SvgExtractorTest`, `SvgSpriteExtractorTest`).

2. **`$this->any()` replaced with `$this->once()`** in `ThemeNegotiatorTest`, where the expectation on `$this->themeAccessCheck` was genuinely called exactly once per test case.

3. **Shared `setUp()` stubs converted to per-test locals** in `IconFinderTest`. Because some tests need a plain stub while others need a mock with `expects()`, the class-level `$fileUrlGenerator`, `$logger`, and `$iconFinder` properties and `setUp()` method were removed entirely. Each test now creates its own local `$iconFinder` with either `createStub()` or `createMock()` as appropriate, and injects concrete local collaborators.

4. **Helper re-initializers added to `RegistryTest`**: Three protected methods (`setUpMockModuleHandler()`, `setUpMockModuleList()`, `setUpMockThemeManager()`) are introduced to allow individual tests that do configure expectations to swap in a real mock object using `ReflectionProperty`, while the global `setUp()` continues to use cheaper stubs.

5. **PHPStan baseline cleaned up**: Two `disallowed.method` suppression entries for `any()` calls in `IconFinderTest` and `ThemeNegotiatorTest` are removed from `core/.phpstan-baseline.php` because the violations no longer exist.
