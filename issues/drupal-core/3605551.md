## Summary

Install profiles can now declare `keep_profile: false` in their `.info.yml` file to be automatically uninstalled at the end of the site installation process. Previously, a profile that existed only to customize the installer had no clean built-in way to remove itself: either the profile had to include explicit post-install uninstall code, or the site builder had to know to run `drush pmu profile_name` manually. With this change, setting `keep_profile: false` in the profile's info file causes Drupal's `install_finished` task to call `module_installer->uninstall()` on the profile automatically. The default behavior (profile remains installed) is unchanged when the key is absent or set to `true`.

## Impact

Affects module maintainers.

- **New API:** New `keep_profile` key in install profile `.info.yml` files; when set to `false` the profile is automatically uninstalled at the end of `install_finished`.

## Technical details

The change is a small addition to the `install_finished()` function in the install API. It reads `$install_state['profile_info']['keep_profile']` (defaulting to `true` when absent). When the value is `true`, the existing `module_set_weight($profile, 1000)` call runs as before. When it is `false`, `\Drupal::service('module_installer')->uninstall([$profile])` is called instead.

If the profile declares `keep_profile: false` but also installs modules that are depended on by other installed code, the standard module uninstall checks apply and throw an exception, aborting the install. This is tested by `ProfileAutoUninstallFailureTest`, which uses a fixture profile (`testing_auto_uninstall_has_dependencies`) that lists `testing_in_use_dependency` under `install:`. The success path is covered by `ProfileAutoUninstallTest` using the `testing_auto_uninstall` fixture profile, which verifies the profile key is absent from `core.extension` after install completes.

The naming follows the existing `keep_english` pattern already used in core profile info files.

## Upgrade

Profile authors who want their profile to self-uninstall after site setup add one line to the profile's `.info.yml`:

```yaml
keep_profile: false
```

No code changes are needed. The profile must not provide any modules that other installed extensions depend on, or the install will fail with an exception.
