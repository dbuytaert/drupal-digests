## Summary

Drupal 12 now requires MariaDB 10.11 as the minimum supported version, up from 10.6. This aligns with the platform requirements set for Drupal 12 and ensures sites run on databases that receive active vendor support. Sites using older MariaDB versions will need to upgrade their database server before upgrading to Drupal 12.

## Impact

Affects site owners.

- **Requirements:** Minimum MariaDB version raised from 10.6 to 10.11

## Technical details

The minimum MariaDB version constant in the MySQL driver install tasks was updated from 10.6 to 10.11. This change affects the version check performed during installation and updates, which validates that the database server meets minimum requirements.

The GitLab CI configuration was updated to test against MariaDB 10.11 instead of 10.6, ensuring core development and testing use the new minimum version. This includes both the default test matrix and the daily test jobs.

## Upgrade

Sites running MariaDB versions older than 10.11 must upgrade their database server before upgrading to Drupal 12. The upgrade path depends on your hosting environment:

- Managed hosting: Contact your hosting provider to upgrade MariaDB
- Self-hosted: Follow your operating system's package manager procedures to upgrade MariaDB

No code changes are required for contributed modules or custom code.
