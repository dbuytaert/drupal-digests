## Summary

The Help module's internal code structure has been modernized by moving a helper function into a class method and simplifying duplicated hook implementations. The change removes the procedural function `_help_search_update()` from `help.module` and consolidates four similar hook implementations into three. This is part of ongoing work to align Drupal core with modern PHP practices and make code easier to maintain. Site behavior remains identical when installing or uninstalling modules and themes.

## Impact

No upgrade or configuration changes required.

## Technical details

The `_help_search_update()` procedural function has been removed from `help.module` and converted to a protected `searchUpdate()` method within the `HelpHooks` class. This function ensures the help search index is updated when extensions are installed or uninstalled.

Four hook implementations now call `searchUpdate()` instead of the old function: `modulesInstalled()`, `modulesUninstalled()`, and a new combined method. The `themesInstalled()` and `themesUninstalled()` hook implementations were identical except for their hook attribute, so they have been merged into a single `themesInstallOrUninstall()` method that uses both `#[Hook('themes_installed')]` and `#[Hook('themes_uninstalled')]` attributes.

The entire `help.module` file has been deleted since it only contained the helper function. All hook logic now lives in the `HelpHooks` class, following Drupal's modern hook system architecture. The functionality remains unchanged: when modules or themes are installed or uninstalled, the help search plugin instance updates its topic list and index state.
