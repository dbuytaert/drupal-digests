## Summary

The procedural `node_access_grants()` function, a holdover from Drupal 7 kept in `node.module`, has been moved into a new injectable service `NodeGrantsHelper`. The function itself is now deprecated and will be removed in Drupal 13. This is an internal refactoring with no change in behavior for site owners, but module authors who call `node_access_grants()` directly need to update their code.

## Impact

Affects module maintainers.

- **API change:** `NodeGrantDatabaseStorage` and `NodeAccessGrantsCacheContext` each gain a new optional `?NodeGrantsHelper $nodeGrantsHelper` constructor argument, deprecated without it in 11.4.0 and required in 12.0.0
- **Deprecation:** `node_access_grants()` procedural function deprecated in drupal:11.4.0 and removed in drupal:13.0.0; use `\Drupal::service('Drupal\node\NodeGrantsHelper')->nodeAccessGrants()` instead
- **New API:** New `NodeGrantsHelper` service (`Drupal\node\NodeGrantsHelper`) with `nodeAccessGrants(string $operation, AccountInterface $account): array` method

## Technical details

The core problem was that `node_access_grants()` â€” which invokes `hook_node_grants` and `hook_node_grants_alter` and returns the merged grants array â€” was a procedural function in `node.module` with no OOP equivalent. Moving it to `NodeAccessControlHandler` was the original plan, but that would have introduced a circular dependency: `NodeGrantDatabaseStorage` (which calls `node_access_grants()` three times internally) is itself a dependency of `NodeAccessControlHandler`.

The resolution was to introduce a dedicated `NodeGrantsHelper` class (`core/modules/node/src/NodeGrantsHelper.php`) with a single public method `nodeAccessGrants(string $operation, AccountInterface $account): array`. This class takes `ModuleHandlerInterface` as its only constructor dependency and contains the exact logic previously in `node_access_grants()`. It is registered as the service `Drupal\node\NodeGrantsHelper` in `node.services.yml`.

With the new service available, all internal callers were updated:
- `NodeGrantDatabaseStorage` gains a new `?NodeGrantsHelper $nodeGrantsHelper` constructor parameter (deprecated without it in 11.4.0, required in 12.0.0), replacing three direct calls to `node_access_grants()`.
- `NodeAccessGrantsCacheContext` gains the same new optional constructor parameter under the same deprecation schedule, replacing one call in `checkNodeGrants()`.
- The Views `NodeStatus` filter's `query()` method uses `\Drupal::service(NodeGrantsHelper::class)->nodeAccessGrants()` as a static call since it cannot easily be injected there.
- `node_access_grants()` itself is deprecated in drupal:11.4.0, removed in drupal:13.0.0, and now simply delegates to the new service.

The related issue [#3566536](https://www.drupal.org/node/3566536) was a prior prerequisite that was resolved before this one could proceed.

## Upgrade

If your code calls `node_access_grants()` directly, replace it with the new service method:

```php
// Before
$grants = node_access_grants($operation, $account);

// After
$grants = \Drupal::service(\Drupal\node\NodeGrantsHelper::class)->nodeAccessGrants($operation, $account);

// Or with proper injection
public function __construct(
  protected readonly \Drupal\node\NodeGrantsHelper $nodeGrantsHelper,

```
