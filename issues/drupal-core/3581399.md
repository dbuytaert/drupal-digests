## Summary

Drupal's test runner `run-tests.sh` historically used legacy names for PHPUnit test suites (such as `PHPUnit-Unit`, `PHPUnit-Kernel`, `PHPUnit-Functional`) that did not match the names defined in `phpunit.xml.dist` (such as `unit`, `kernel`, `functional`). This mismatch required internal mapping and reverse-mapping logic. This change removes that mapping and standardizes everything on the PHPUnit names. Site owners or teams running custom CI pipelines that invoke `run-tests.sh --types` with the legacy names will need to update their scripts. This change targets Drupal 12 (main) only.

## Impact

Affects module maintainers and site owners.

- **Config change:** The `--types` argument to `run-tests.sh` must now use phpunit.xml suite names (`unit`, `kernel`, `functional`, `functional-javascript`, `build`) instead of the legacy `PHPUnit-*` prefixed names.
- **API change:** `PhpUnitTestDiscovery::getPhpunitTestSuite()` now returns lowercase hyphenated suite names (`unit`, `kernel`, `functional`, `functional-javascript`, `unit-component`, `build`) instead of the legacy mixed-case values (`Unit`, `Kernel`, `Functional`, `FunctionalJavascript`, etc.).

## Technical details

The core problem was a dual naming system maintained inside `PhpUnitTestDiscovery`: a `$map` array translated legacy names (e.g. `PHPUnit-Unit`) to phpunit.xml names (e.g. `unit`) on input, and a `$reverseMap` (computed via `array_flip`) translated them back when building the test list. `getPhpunitTestSuite()` returned mixed-case legacy suffixes like `Unit`, `Kernel`, `Functional`, `FunctionalJavascript` which callers prefixed with `PHPUnit-`.

The fix removes `$map`, `$reverseMap`, and all translation calls. `getPhpunitTestSuite()` now returns the lowercase phpunit.xml suite names directly (`unit`, `kernel`, `functional`, `functional-javascript`, `unit-component`, `build`), using `match` expressions to handle the `FunctionalJavascript` namespace segment correctly. `getTestClasses()` passes `$testSuites` values straight to `--testsuite=` without conversion. The `.gitlab-ci.yml` CI pipeline variables were updated from `PHPUnit-Build`, `PHPUnit-FunctionalJavascript`, `PHPUnit-Functional`, and `PHPUnit-Kernel` to `build`, `functional-javascript`, `functional`, and `kernel`. The `--types` help text in `run-tests.sh` was updated accordingly. Test data providers in `PhpUnitTestDiscoveryTest` and `TestInfoParserTest` were also converted from array-returning methods to generators, and updated to use the new names.

## Upgrade

Any custom CI script or tooling that passes legacy suite names to `run-tests.sh --types` must be updated:

```bash
# Before
php ./core/scripts/run-tests.sh --types "PHPUnit-Unit,PHPUnit-Kernel"

# After
php ./core/scripts/run-tests.sh --types "unit,kernel"
```

The full mapping of old names to new names is:

| Old (legacy) | New (phpunit.xml) |
|---|---|
| `PHPUnit-Unit` | `unit` |
| `PHPUnit-Unit-Component` | `unit-component` |
| `PHPUnit-Kernel` | `kernel` |
| `PHPUnit-Functional` | `functional` |
| `PHPUnit-FunctionalJavascript` | `functional-javascript` |
| `PHPUnit-Build` | `build` |

Any code that calls `PhpUnitTestDiscovery::getPhpunitTestSuite()` and expects the old mixed-case return values must also be updated to expect the new lowercase hyphenated names.
