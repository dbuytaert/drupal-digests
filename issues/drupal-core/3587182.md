## Summary

The bundled CKEditor 5 library in Drupal core `main` (the Drupal 12 track) has been updated from version 47.x to 48.0.1. This upgrade resolves longstanding table accessibility problems: CKEditor 5 previously collapsed multiple `<tbody>` sections and discarded `<thead>` structure when editing HTML tables, producing markup that hindered screen readers. Version 48 preserves correct table structure, adds support for `<tfoot>` (table footer rows), and supports `<th scope>` attributes on header cells. Drupal 11 is intentionally staying on the CKEditor 5 v47 LTS release and does not receive this upgrade.

## Impact

Affects module maintainers.

- **New API:** New CKEditor 5 table UI strings: 'Footer row', 'Column header', 'Row header', exposed through the updated bundled translation files.

## Technical details

CKEditor 5 v48 dropped its DLL (Dynamic Link Library) build format entirely, which Drupal previously relied on for its plugin installation method. This upgrade was therefore blocked until [#3527914](https://www.drupal.org/node/3527914) landed to switch the integration to the UMD (Universal Module Definition) installation method. Once that prerequisite merged into `main`, this issue could proceed for Drupal 12 only.

The commit updates the bundled CKEditor 5 UMD-format JS files under `core/assets/vendor/ckeditor5/`. The translated language bundles gain new UI strings corresponding to the improved table feature set: `"Footer row"`, `"Column header"`, and `"Row header"`. Some previously present strings (e.g. the standalone `"Alignment"` label in certain table property contexts) were reorganized or removed in line with upstream changes to table property panels.

The `jsdoc` extraction script, which generates API documentation from the CKEditor 5 package, was also updated in this commit: its glob and regex patterns were adjusted to look at TypeScript definition files rather than the old JS source layout, fixing a breakage introduced by the v48 package restructuring. `yarn vendor-update` no longer automatically updates the deprecated-version entries in the vendor metadata, so those were updated manually. The `AssetAggregationAcrossPagesTest` expected byte-size bounds were also recalibrated to reflect the new bundle size.
