## Summary

Claro's `file-link.html.twig` template always rendered the file size, even when there was nothing to show, producing empty parentheses in contexts like the `file_table` formatter where file size is already displayed in a separate column. The template now only renders the file size span when `file_size` is non-empty. This fixes both the spurious empty parentheses case and the duplicate-display case handled on the file module side by [#3482024](https://www.drupal.org/node/3482024).

## Impact

Affects front-end developers.

- **Theme change:** Claro's `file-link.html.twig` now conditionally renders the file size span; contributed or custom themes that copied Claro's unconditional markup should add a `{% if file_size %}` guard around the `<span class="file__size">` element.

## Technical details

Claro provides its own override of `core/themes/claro/templates/field/file-link.html.twig`. Before this fix the template unconditionally output the file size span: `{{ link }} <span class="file__size">({{ file_size }})</span>`. This caused two problems: empty parentheses when `file_size` was not set, and duplicate size display when used with the `file_table` formatter, which already renders size in its own column. The core file module's own `file-link.html.twig` already guarded against this, but Claro's override did not. The one-line fix wraps the span in a `{% if file_size %}` Twig conditional, so the span is suppressed whenever `file_size` is empty or has been unset by a preprocess function. The companion issue [#3482024](https://www.drupal.org/node/3482024) adds the mechanism for formatters such as `file_table` to unset `file_size` when they handle it themselves. Contributed or custom themes that maintain their own `file-link.html.twig` (such as the now-contributed Seven theme) should apply the same conditional if they copied Claro's previous unconditional markup.
