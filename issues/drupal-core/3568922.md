## Summary

When Drupal's oEmbed system tries to discover an oEmbed endpoint URL for a given media URL, it makes an HTTP GET request to that URL and parses the response body as HTML to find a link tag pointing to the oEmbed endpoint. Before this fix, it never checked the `Content-Type` response header, so it would attempt to parse JSON, binary, or any other non-HTML response as HTML. The fix makes the method return early with no match whenever the response is not `text/html`, preventing erroneous parsing of non-HTML content.

## Impact

No upgrade or configuration changes required.

## Technical details

The oEmbed subsystem uses `UrlResolver::discoverResourceUrl()` (in `core/modules/media/src/OEmbed/UrlResolver.php`) to perform HTML link-discovery, which is the fallback path when an oEmbed provider is not matched by a known URL pattern. The method issues an HTTP GET via `$this->httpClient->get($url)` and then immediately calls `Html::load()` on the response body, followed by a `DOMXPath` search for `<link rel="alternate">` tags carrying the oEmbed endpoint URL.

The problem was that no guard existed between the HTTP response and the HTML parse step. A URL returning JSON, XML, or any other content type would be fed into `Html::load()` without error, but the XPath search would find nothing and the discovery would silently fail. In edge cases where a non-HTML body happened to contain the right string, it could also produce incorrect results.

The fix inserts a six-line guard immediately after the successful HTTP response: it reads the `Content-Type` header with `$response->getHeaderLine('Content-Type')`, lowercases it with `strtolower()`, and returns `FALSE` early if the value does not contain `'text/html'`. The existing HTML parsing and XPath logic is otherwise unchanged.

A new unit test class `UrlResolverTest` was added at `core/modules/media/tests/src/Unit/UrlResolverTest.php`. Because `discoverResourceUrl()` is `protected`, the test uses an anonymous subclass that promotes the method to `public`. Two test cases cover the two branches: `testDiscoverResourceUrlParsesHtml` provides a mock HTTP response with `Content-Type: text/html` and asserts the oEmbed URL is extracted correctly; `testDiscoverResourceUrlSkipsNonHtml` provides a mock response with `Content-Type: application/json` and asserts the method returns `false`.
