## Summary

This is a pure internal test maintenance change with no user-facing effects. Several unit test classes were updated to be compatible with PHPUnit 12.5.0, which tightens the rules around mock objects: mocks without configured expectations must now be stubs, `$this->any()` is deprecated, and `with()` on stubs is deprecated.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces notices and deprecations for three patterns that were widespread in Drupal's test suite. First, a mock object created via `createMock()` that never has `expects()` called on it now emits a notice saying it should be a stub instead. Second, `$this->any()` (which is equivalent to no expectation) is deprecated. Third, `with()` on stubs is deprecated because argument matching on an object with no expectation is pointless. This issue addresses the six affected test classes in `core/tests/Drupal/Tests/Core/Image/`, `core/tests/Drupal/Tests/Core/ParamConverter/`, and `core/tests/Drupal/Tests/Core/Test/`. The fix is mechanical: calls to `createMock()` where no `expects()` was ever configured are replaced with `createStub()`, and any matching `$this->any()` or `with()` calls on those objects are removed. Mock objects that do have real expectations (using `$this->once()`, `$this->exactly(N)`, `$this->never()`, etc.) are correctly left as mocks. The issue notes that `TestDiscoveryTest::testTestInfoParser()` will fail under PHPUnit 12 for a separate, unrelated reason involving a removed PHPUnit function used in test annotation discovery, and that problem is explicitly out of scope here.
