## Summary

This is a purely internal test-code cleanup with no effect on site behavior or public APIs. Twelve unit test files across the `config_translation`, `content_translation`, `language`, and `locale` modules were updated to remove PHPUnit 12.5 deprecation notices.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 emits a notice for every mock object that has no configured expectations, and PHPUnit 12.5.5 will deprecate `$this->any()` (which is logically equivalent to having no expectation at all). This issue is one installment in the parent meta task [#3561671](https://www.drupal.org/node/3561671) to systematically fix these warnings across Drupal core.

The changes follow two related patterns. First, `createMock()` calls (and `getMockBuilder(...)->disableOriginalConstructor()->getMock()` chains) for dependencies that are only stubbed out and never verified with expectations were replaced with `createStub()`. Affected classes include `TypedConfigManagerInterface`, `LocaleConfigManager`, `ConfigMapperManagerInterface`, `RouteProviderInterface`, `LanguageManagerInterface`, `EventDispatcherInterface`, `CacheBackendInterface`, `AccountInterface`, `EntityTypeManagerInterface`, `FieldConfigInterface`, `FieldStorageConfigInterface`, `ModuleHandlerInterface`, `ThemeHandlerInterface`, `CacheContextsManager`, and others. PHPDoc `@var` annotations were updated from `MockObject` to `Stub` to match. Second, every `->expects($this->any())` call was removed from method chains; where a stub already provides the return value via `willReturn`, no expectation annotation is needed.

In `ConfigNamesMapperTest`, the container/url-generator setup was extracted from `setUp()` into a new `buildContainer()` helper method and called only from the two tests that actually invoke URL generation, preventing unnecessary mock setup from bleeding across unrelated tests. In `LocaleLookupTest`, the class-level `$cache` property was removed; each test method that needs a `CacheBackendInterface` now creates a scoped `createMock` or `createStub` so that the `get()` expectation applies only where relevant. A few tests also gained explicit `expects($this->never())` assertions (e.g., `testGetType()` and `testGetOperations()` in `ConfigEntityMapperTest`) to make it clear that certain methods must not be called in those paths. Several `expects($this->any())` calls on partial mocks were replaced with `expects($this->atLeastOnce())` where at least one invocation is guaranteed. Corresponding PHPStan baseline suppressions for the `any()` calls were removed from `core/.phpstan-baseline.php`.
