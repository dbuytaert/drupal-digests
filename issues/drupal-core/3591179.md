## Summary

Following the Drupal 11.3.10 security release, the `main` and `11.x` development branches have been updated to the same patched versions of Twig and Symfony that were shipped in the stable security release. The update raises `twig/twig` from `^3.21.0` to `^3.26.0` and tightens the minimum constraint for several Symfony 8.0 packages (including `http-kernel`, `mailer`, `mime`, `routing`, and `yaml`) to `^8.0.12`. Sites on a stable release were already covered by 11.3.10; this change keeps the development branches aligned with those security fixes so contributors are not working against vulnerable dependencies.

## Impact

Affects site owners.

- **Security:** Twig and Symfony packages updated to security-patched versions matching Drupal 11.3.10; twig/twig minimum raised to 3.26.0 and several symfony/* packages pinned to 8.0.12

## Technical details

The commit updates both `composer.json` version constraints and the resolved `composer.lock` entries. On the constraint side, `twig/twig` is moved to `^3.26.0` and five Symfony packages (`symfony/http-kernel`, `symfony/mailer`, `symfony/mime`, `symfony/routing`, `symfony/yaml`, and the test-only `symfony/dom-crawler`) are pinned to `^8.0.12`. The remaining Symfony packages receive lock-file-only bumps without constraint changes. Symfony polyfills advance from `~v1.33.0` to `~v1.37.0`. The locked `symfony/serializer` `v8.0.10` entry gains a new conflict against `symfony/property-access <7.4.2|>=8.0,<8.0.2`, and `symfony/validator` `v8.0.10` gains a conflict against `symfony/expression-language <7.4`, reflecting upstream dependency refinements present in those releases. PHPStan baseline adjustments were also needed to resolve `ignore.unmatchedIdentifier` errors against the MySQL, PostgreSQL, and SQLite database driver files that appeared after the Symfony upgrade; those changes are part of the same commit.
