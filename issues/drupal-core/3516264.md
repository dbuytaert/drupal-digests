## Summary

Sites using CKEditor 5 with multiple text formats, a secondary interface language, and JS aggregation disabled could trigger HTTP 429 "Too many requests" errors. When an AJAX operation loaded a second editor instance after the first had already set up translations on the initial page load, CKEditor 5 skipped its translation-filtering logic and included translation files for every installed language rather than just the active interface language. This fix ensures the filtering always runs correctly, even when the triggering placeholder library was already loaded by the initial request.

## Impact

No upgrade or configuration changes required.

## Technical details

The filtering logic lives in `Ckeditor5Hooks::jsAlter()`. It detects that CKEditor 5 translations are needed by checking for the `core/ckeditor5.translations` library and then removes all per-language translation JS files except the one matching the current interface language.

The original code checked `$assets->getLibraries()` and `$assets->getAlreadyLoadedLibraries()` directly for `core/ckeditor5.translations`. This failed during AJAX requests because that library is typically a transitive dependency of a language-specific library (e.g. `core/ckeditor5.translations.en`) rather than being listed directly in the assets. Calling `LibraryDependencyResolverInterface::getLibrariesWithDependencies()` on both lists fully expands the dependency tree, so the check correctly detects the library in all loading scenarios.

As a secondary fix, `unset($javascript[$placeholder_file])` is moved outside the conditional block so the placeholder file (`core/assets/vendor/ckeditor5/translation.js`) is always removed from the response regardless of whether filtering ran. `Ckeditor5Hooks` now receives `ModuleHandlerInterface` and `LibraryDependencyResolverInterface` as injected constructor dependencies instead of using `\Drupal::moduleHandler()` static calls. A unit test `testJsAlterHook()` was added to `CKEditor5Test` covering both the placeholder-present and placeholder-already-loaded cases, and a new performance test `testNodeAddPagesAuthor()` in `AssetAggregationAcrossPagesTest` guards against regressions by asserting script counts across multiple node-add pages.

## Upgrade

If you have custom code that instantiates `Ckeditor5Hooks` directly (e.g. in a test or a custom service definition override), you must now pass the two additional constructor arguments:

```php
// Before:
$hooks = new Ckeditor5Hooks($languageMapper);

// After:
$hooks = new Ckeditor5Hooks(
  $languageMapper,
  \Drupal::service('module_handler'),        // ModuleHandlerInterface
  \Drupal::service('library.dependency_resolver'), // LibraryDependencyResolverInterface
);
```

Service container wiring in core is updated automatically; only custom instantiations are affected.
