## Summary

Drupal core now prepares for Symfony 8 compatibility by replacing the deprecated `Request::get()` method. Symfony deprecated this method because it blurred the source of input data, creating potential security risks. Drupal now explicitly checks the correct parameter bags (`query`, `request`, or `attributes`) for each use case. The change also consolidates handling of AJAX page state by storing it in request attributes early in the request lifecycle, making the code clearer and more maintainable. All tests pass and the behavior remains the same for site owners and developers.

## Impact

No upgrade or configuration changes required.

## Technical details

The Symfony `Request::get()` method is deprecated in Symfony 7.4 and will be removed in Symfony 8.0. This method checked multiple parameter bags in sequence (`attributes`, `query`, `request`), which made the origin of data unclear and potentially unsafe.

Drupal core replaced all uses of `Request::get()` with explicit calls to the appropriate parameter bag. For query string parameters, code now calls `$request->query->get()`. For POST data, code now calls `$request->request->get()`. For values that could come from either source, code checks both explicitly. The `AjaxPageState` middleware was enhanced to copy the final AJAX page state into `$request->attributes`, making it available as a request attribute throughout the request lifecycle.

The most significant changes are in `ViewAjaxController::ajaxView()`, which now checks both query and request bags for view parameters, and in `AjaxBasePageNegotiator`, `AjaxResponseAttachmentsProcessor`, `HtmlResponseAttachmentsProcessor`, and CSS/JS asset optimizers, which now read `ajax_page_state` from request attributes instead of using the ambiguous `get()` method. The `FormBuilder` now explicitly checks both bags when validating form IDs for AJAX requests.

Tests in `AjaxPageStateTest` and `AjaxBasePageNegotiatorTest` were updated to set values in the attributes bag to match the new behavior. The workaround code in `ViewAjaxController` that temporarily copied theme information to request attributes is now removed since the middleware handles this consistently.

## Lessons learned

- When accepting user input via HTTP requests, explicitly check the correct parameter bag (query, request, or attributes) rather than using generic getters that check multiple sources.
