## Summary

Drupal's PostCSS build configuration in `core/scripts/css/compile.js` was simplified by removing the now-obsolete explicit options block passed to the `postcss-preset-env` plugin, replacing it with a bare `postcssPresetEnv()` call. The `postcssPixelsToRem` options block was also reformatted for consistent indentation. There is no change to site behavior or compiled output.

## Impact

No upgrade or configuration changes required.

## Technical details

Drupal's CSS compilation pipeline lives in `core/scripts/css/compile.js` and chains several PostCSS plugins. The `postcss-preset-env` plugin was previously invoked with an explicit `stage: 1`, `preserve: false`, an `autoprefixer` options block (with `cascade: false` and `grid: 'no-autoplace'`), and a `features` map that individually disabled seven specific transforms (`blank-pseudo-class`, `focus-visible-pseudo-class`, `focus-within-pseudo-class`, `has-pseudo-class`, `image-set-function`, `prefers-color-scheme-query`, `content-alt-text`). As `postcss-preset-env` has evolved, these options were no longer needed: the features were either already off by default in current versions or no longer applicable, making the configuration dead weight. The fix replaces the entire options object with a plain `postcssPresetEnv()` call. The `postcssPixelsToRem` block was left functionally unchanged but its indentation was corrected to match the surrounding code style.
