## Summary

Olivero's dropdown navigation menus stopped working for logged-in users in Drupal 11.3. When authenticated with BigPipe enabled (which is the default), hovering over menu items with nested children would not open the submenu. This affected site navigation for administrators and other authenticated users, making nested menu items inaccessible. The issue was introduced when changes to block placeholdering caused the main navigation to be rendered via BigPipe on cache misses, but Olivero's navigation JavaScript was not designed to reinitialize when content is dynamically added to the page.

## Impact

No upgrade or configuration changes required.

## Technical details

The root cause was that `second-level-navigation.js` in Olivero ran its initialization code immediately on page load using `document.querySelectorAll()`, rather than using Drupal's behavior system. When BigPipe dynamically replaced the navigation menu after the initial page load, the JavaScript event listeners were never attached to the new DOM elements.

The fix refactored `second-level-navigation.js` to use `Drupal.behaviors.secondLevelNav`. The initialization code was moved into an `init()` function that accepts a navigation element parameter. The behavior's `attach()` method uses `once()` to find the primary navigation menu and calls `init()` for each instance. This ensures event listeners are properly attached whenever the navigation is added to the DOM, whether during initial page load or via BigPipe replacement.

The code also wraps document-level event listeners (for Escape key and outside clicks) with `once('olivero-second-level-nav-document', document)` to prevent duplicate handlers when the behavior runs multiple times. The `secondLevelNavMenus` variable was changed from a const initialized at load time to a let variable that gets populated during `init()`.

Investigation revealed this was actually a regression introduced in Drupal 11.2 via [#3437499](https://www.drupal.org/project/drupal/issues/3437499), which made the main navigation block use placeholdering. On cache misses, the navigation now renders via BigPipe, exposing this latent bug in Olivero's JavaScript.

## Lessons learned

- When JavaScript initializes UI components on page load, it must use Drupal behaviors instead of direct DOM manipulation. Big Pipe and other lazy-loading systems inject markup after DOMContentLoaded, causing initialization code that runs once at page load to miss dynamically added elements. *(via mherchel)*
- When adding document-level event listeners inside Drupal behaviors that can attach to multiple elements, wrap them with once() to prevent duplicate handlers. Without this, each behavior invocation adds redundant global listeners that waste resources and may cause bugs. *(via lauriii)*
