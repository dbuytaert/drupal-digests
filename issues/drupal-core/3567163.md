## Summary

Drupal is modernizing its codebase by organizing form callbacks into dedicated hook classes. The Field UI module now uses an object-oriented approach for handling the "Save and manage fields" button submission instead of a standalone procedural function. This is part of a broader effort to make Drupal's code more consistent and maintainable, but does not change how the Field UI works or looks.

## Impact

Affects module maintainers.

- **Deprecation:** Procedural function `field_ui_form_manage_field_form_submit()` deprecated in favor of `FieldUiHooks::manageFieldFormSubmit()` method

## Technical details

The `field_ui_form_manage_field_form_submit()` procedural function has been moved into the `FieldUiHooks` class as the `manageFieldFormSubmit()` method. The logic remains identical: it checks if the triggering element is the `save_continue` button and redirects to the field overview page if appropriate.

The `FieldUiHooks::formAlter()` method now references the new method using the `self::class . ':manageFieldFormSubmit'` syntax instead of the string function name. The old procedural function remains as a deprecated wrapper that triggers an `E_USER_DEPRECATED` error and delegates to the new method through the service container.

This change aligns with Drupal's ongoing migration to hook classes, consolidating related hook implementations and form callbacks into cohesive class structures. The form submission handler logic itself is unchanged.

## Upgrade

If your custom code references `field_ui_form_manage_field_form_submit()` directly as a form submit handler, update it to use the class method instead:

```php
// Old:
$form['#submit'][] = 'field_ui_form_manage_field_form_submit';

// New:
$form['#submit'][] = \Drupal\field_ui\Hook\FieldUiHooks::class . ':manageFieldFormSubmit';
```

See [#3566774](https://www.drupal.org/node/3566774) for full details.
