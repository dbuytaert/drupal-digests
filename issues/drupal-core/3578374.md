## Summary

A batch of Drupal core service classes across many modules has been converted to use Symfony's `#[Autowire]` attribute for dependency injection, removing the corresponding explicit `arguments:` arrays from their `.services.yml` definitions. Several service files also gained `autowire: true` in their `_defaults` block. This is a purely internal refactoring with no user-facing behavior change.

## Impact

No upgrade or configuration changes required.

## Technical details

The predecessor issue [#3397041](https://www.drupal.org/node/3397041) autowired core services whose constructor parameter types were sufficient for Symfony to resolve the dependency unambiguously. This issue covers the remaining cases where type alone is ambiguous: for example, a constructor that accepts `CacheBackendInterface` could legally receive `cache.default`, `cache.discovery`, or `cache.data`, and a `RouterInterface` parameter might need `router.no_access_checks` rather than the standard `router`. For those parameters, `#[Autowire(service: 'explicit.service.id')]` attributes are added directly on the constructor parameter.

Affected classes include `LayoutPluginManager`, `BigPipeResponseAttachmentsProcessor`, `BlockContentUuidLookup`, `BreakpointManager`, `SmartDefaultSettings` (ckeditor5), `ConfigMapperManager`, `editor\Element`, `FilterUninstallValidator`, `EntityAccessChecker`, `EntityResource`, `ResourceResponseSubscriber` and `ResourceObjectNormalizer` (jsonapi), `ResourceTypeRepository`, `LanguageNegotiationMethodManager`, `LanguageNegotiator`, `LocaleConfigManager`, `LocaleTranslation`, `media\OEmbed\ResourceFetcher`, `media\OEmbed\UrlResolver`, `navigation\EntityRouteHelper`, `path_alias\AliasManager`, `rest\ResourcePluginManager`, and `rest\ResourceResponseSubscriber`, among others.

As a secondary change within the same commit, classes that previously assigned injected services to instance properties inside the constructor body were converted to constructor property promotion, and the associated `@var` docblocks and constructor `@param` docblocks were removed. The service YAML `arguments:` arrays for all affected services were dropped, and `autowire: true` was added to the `_defaults` block of several modules' service files (`breakpoint`, `language`, `layout_discovery`, `responsive_image`). The notation for autowired-only service aliases (e.g., `Drupal\ckeditor5\LanguageMapper: ~` replacing `autowire: true`) was also normalized.
