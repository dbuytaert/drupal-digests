## Summary

Core module services that could not be fully autowired because their constructors accept interfaces with multiple registered implementations now carry `#[Autowire(service: '...')]` PHP 8 attributes to identify the correct service to inject. The corresponding explicit `arguments:` lines are removed from `.services.yml` definitions. Constructor property promotion is applied throughout, and verbose constructor docblocks are removed. This is a pure code-modernization change with no behavioral effect for site owners.

## Impact

No upgrade or configuration changes required.

## Technical details

The predecessor issue [#3397041](https://www.drupal.org/node/3397041) handled services where all constructor parameters could be resolved by type alone. This batch covers services where at least one parameter has a type that maps to multiple registered services, requiring an explicit service ID. For example, `CacheBackendInterface` is ambiguous among `cache.discovery`, `cache.default`, `cache.data`, `cache.bootstrap`, and others; `SerializerInterface` must distinguish `serializer` from `jsonapi.serializer`; and `RouterInterface` must distinguish `router.no_access_checks` from `router`. The `#[Autowire(service: 'service.id')]` attribute on each ambiguous constructor parameter resolves this without re-adding a YAML `arguments:` block.

Affected classes span roughly 20 classes across `big_pipe` (`BigPipeResponseAttachmentsProcessor`), `block_content` (`BlockContentUuidLookup`), `breakpoint` (`BreakpointManager`), `ckeditor5` (`SmartDefaultSettings`), `comment` (`CommentStatistics`), `config_translation` (`ConfigMapperManager`), `editor` (`Element`), `filter` (`FilterUninstallValidator`), `jsonapi` (`EntityAccessChecker`, `EntityResource`, `ResourceResponseSubscriber`, `ResourceObjectNormalizer`, `ResourceTypeRepository`), `language` (`LanguageNegotiationMethodManager`, `LanguageNegotiator`), `locale` (`LocaleConfigManager`, `LocaleTranslation`), `media` (`ResourceFetcher`, `UrlResolver`), `navigation` (`EntityRouteHelper`), `path_alias` (`AliasManager`), `rest` (`ResourcePluginManager`, `ResourceResponseSubscriber`), and `layout_discovery` (`LayoutPluginManager`). Several `.services.yml` files also gain `autowire: true` in their `_defaults` block (notably `breakpoint`, `language`, `layout_discovery`, and `responsive_image`), covering all services in those files. PHP 8 constructor property promotion replaces both manual `$this->property = $argument` assignments and now-redundant `@var` property docblocks. Broader constructor promotion across core is tracked in [#3278431](https://www.drupal.org/node/3278431) via a Rector rule.
