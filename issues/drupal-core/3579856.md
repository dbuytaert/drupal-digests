## Summary

This is a purely internal test maintenance change with no user-facing effects. Unit tests in eight core subsystems (Display, Extension, Field, File, Flood, HTMX, HTTP, and Language) were updated to stay compatible with PHPUnit 12.5.0+, which now warns or deprecates certain mock usage patterns.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced a notice when a mock object is created but no expectations are configured: the object should instead be a stub created with `createStub()` rather than `createMock()`. Separately, `$this->any()` was deprecated (it was equivalent to configuring no expectation, so the object should either be a stub or carry a precise expectation count), and `with()` was deprecated for stubs. The fix converts expectation-free `createMock()` calls to `createStub()` and removes `willReturn()` / `$this->any()` / `with()` patterns that are no longer appropriate, across the unit test classes under `core/tests/Drupal/Tests/Core/` for the Display, Extension, Field, File, Flood, Htmx, Http, and Language namespaces. A related companion change covering `UnitTestCase` itself was made in [#3578904](https://www.drupal.org/node/3578904) and was merged into this MR before commit.
