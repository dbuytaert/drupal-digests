## Summary

The Navigation module's logo link previously gave screen reader users no useful destination label: the only text in the link was the image alt attribute "Navigation logo," which describes the image rather than where the link goes. This fix makes the logo image decorative and adds a visually-hidden "Home page" label inside the anchor, so assistive technologies now announce the link's purpose clearly. The change applies to both the default Drupal SVG logo and any custom image logo configured for the navigation toolbar.

## Impact

No upgrade or configuration changes required.

## Technical details

The Navigation module renders its admin toolbar logo as an `<a class="admin-toolbar__logo">` anchor pointing to `path('<front>')` in `core/modules/navigation/layouts/navigation.html.twig`. Two template files were changed.

In `navigation.html.twig`, the `<img>` element previously carried `alt="{{ 'Navigation logo'|t }}"` as its accessible name. That attribute was removed and replaced with `role="presentation" aria-hidden="true" focusable="false"`, marking the image as decorative. The `{% include '@navigation/logo.svg.twig' with { label: 'Navigation logo'|t } only %}` call was simplified to `{% include '@navigation/logo.svg.twig' only %}`, removing the label variable entirely. A `<span class="visually-hidden">{{ 'Home page'|t }}</span>` was then added inside the `<a>` tag so that the link itself has a meaningful accessible name regardless of whether the logo is an SVG or a custom image.

In `core/modules/navigation/templates/logo.svg.twig`, the `<svg>` element previously used `role="img" aria-label="{{ label }}"` to expose itself as a labelled image. That was replaced with `role="presentation" aria-hidden="true" focusable="false"`, consistent with the decorative treatment applied to the `<img>` path.

The label text "Home page" (two words with a space) was deliberately chosen over the single-word "Homepage" after contributors confirmed that the spaced form is announced more reliably by screen readers. "Back to site," which was used in the old admin toolbar, was considered but rejected because it becomes misleading when the user is already on the front page.
