## Summary

A randomly failing Layout Builder test has been fixed by moving it out of the JavaScript test runner and into a plain browser test. The test `testInlineBlocksRevisioning`, which verifies that inline block content revisions are correctly tracked when a node is revised and reverted, was sitting inside a `FunctionalJavascript` test class even though it never relied on JavaScript. Running it under WebDriver caused intermittent `TypeError` crashes. The test now lives in a standard `BrowserTestBase` class and runs reliably. This change has no effect on site behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

The random failure manifested as `TypeError: str_starts_with(): Argument #1 ($haystack) must be of type string, array given` at `WebDriverTestBase::115`, caused by the test executing inside a WebDriver context it did not need. The fix moves `testInlineBlocksRevisioning` out of `InlineBlockTest` (a `FunctionalJavascript` class extending `WebDriverTestBase`) and into `LayoutBuilderRevisionTest` (a `Functional` class extending `BrowserTestBase`), both under `core/modules/layout_builder/tests/src/`.

Because `LayoutBuilderRevisionTest` previously had no `block_content` dependency, several additions were needed to support the moved test: the `BlockContentCreationTrait` (from `core/modules/block_content/tests/src/Traits/BlockContentCreationTrait.php`) was mixed into the class, the modules `field_ui`, `block`, and `block_content` were added to `$modules`, and `setUp()` was extended to call `createBlockContentType()` to create a revisable "basic" block content type.

The test logic was also reworked to avoid JS-dependent helpers. The old version used `InlineBlockTestBase`'s `addInlineBlockToLayout()` and `configureInlineBlock()` helper methods, which rely on WebDriver interactions. The rewritten version drives the same user flow through plain `clickLink()` and `submitForm()` calls, and looks up the block component UUID directly from the entity via `OverridesSectionStorage::FIELD_NAME` to build the `layout_builder/update/block/...` URL for updating the inline block without a JavaScript dialog.
