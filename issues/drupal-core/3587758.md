# #3587758: [PP-1] Labels for accent color choices on settings form are hidden from accessible technology

## Summary

Radio button labels for the accent color picker on the `default_admin` theme settings form were hidden using `display: none`, which completely removes them from the accessibility tree. This caused twelve WCAG 4.1.2 (Name, Role, Value) violations rated Critical by Deque axe-core, reported by tools such as Accessibility Insights and axe. The issue affected both the admin appearance settings page and the per-user settings form when users are allowed to override admin theme settings. The fix was delivered as part of a broader CSS refactor in [#3599680](https://www.drupal.org/node/3599680), which removed the `display: none` rule and replaced it with a visually-hidden approach that keeps labels available to assistive technology.

## Impact

No upgrade or configuration changes required.

## Technical details

The accent color radio buttons in `Settings::getSettingsForm()` had associated labels that were hidden by custom CSS in the `default_admin` theme using `display: none`. That property removes elements from the accessibility tree entirely, so screen readers and other assistive technology could not read the labels. The fix landed as part of [#3599680](https://www.drupal.org/node/3599680), which consolidated and refactored the theme's CSS custom properties. The `display: none` rule on those labels was removed during that cleanup; the labels remain visually suppressed through a visually-hidden CSS pattern instead. A follow-up to switch to core's `.visually-hidden` class (which would also benefit from the improvements proposed in [#3591112](https://www.drupal.org/node/3591112)) was noted as a future task.
