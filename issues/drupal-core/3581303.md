## Summary

All batch-related procedural functions in the locale module's `.inc` files are now deprecated in Drupal 11.4.0 and will be removed in 13.0.0. The logic has been moved into three injectable services: the existing `LocaleFetch` and `LocaleProjectChecker` services (which gained new methods) and two new services, `LocaleConfigBatch` and `LocaleImportBatch`. Custom modules, install profiles, or any code that calls the old procedural callbacks directly will trigger deprecation warnings and must be updated before upgrading to Drupal 13.

## Impact

Affects module maintainers.

- **Deprecation:** Nineteen procedural batch callback functions across `locale.batch.inc`, `locale.bulk.inc`, and `locale.compare.inc` deprecated in 11.4.0, removed in 13.0.0; replacements are methods on `LocaleFetch`, `LocaleProjectChecker`, `LocaleConfigBatch`, and `LocaleImportBatch` services.
- **New API:** Two new services added: `Drupal\locale\LocaleConfigBatch` and `Drupal\locale\LocaleImportBatch`.

## Technical details

The change spans three `.inc` files (`locale.batch.inc`, `locale.bulk.inc`, `locale.compare.inc`) and affects nineteen procedural functions. Each function now contains only a `@trigger_error()` deprecation notice and a delegation call to the equivalent service method. Two new service classes are introduced: `LocaleConfigBatch` (handles config translation batch operations previously in `locale_config_batch_*` functions) and `LocaleImportBatch` (handles translation file import batch operations previously in `locale_translate_batch_*` functions). Both are registered in `locale.services.yml` without autowiring on 11.x. The existing `LocaleFetch` service gained `batchVersionCheck()`, `batchStatusCheck()`, `batchDownload()`, `batchImport()`, `batchFinished()`, and `getStatusOperations()` methods. `LocaleProjectChecker` gained `triggerBatch()` and `batchFinished()`.

Two methods on `LocaleFetch` were also renamed as part of this commit: `batchUpdateBuild()` → `buildUpdateBatch()` and `batchFetchBuild()` → `buildFetchBatch()`. The deprecated procedural wrappers `locale_translation_batch_update_build()` and `locale_translation_batch_fetch_build()` (from a prior issue, [#3577671](https://www.drupal.org/node/3577671)) had their deprecation notice text updated to reflect the new names. Internal callers in `locale.module`, `install.core.inc`, `ImportForm`, and `TranslationStatusForm` are updated to use the service methods directly. `ImportForm` and `TranslationStatusForm` were also modernized to use constructor property promotion and inject the new services. `locale_config_batch_build()` is deprecated but retains a partially delegated implementation that uses `LocaleConfigBatch::class . ':batchUpdateConfigTranslations'` as batch operation strings, enabled by the `BatchBuilder` accepting plain strings (see [#3575096](https://www.drupal.org/node/3575096)).

## Upgrade

Replace calls to the procedural functions with calls to the equivalent service methods. The change record URL is https://www.drupal.org/node/3589759.

**locale.batch.inc functions:**
```php
// Before
locale_translation_batch_version_check($project, $langcode, $context);
locale_translation_batch_status_check($project, $langcode, $options, $context);
locale_translation_batch_fetch_download($project, $langcode, $context);
locale_translation_batch_fetch_import($project, $langcode, $options, $context);
locale_translation_batch_fetch_finished($success, $results);
locale_translation_batch_status_finished($success, $results);
_locale_translation_batch_status_operations($projects, $langcodes, $options);

// After
\Drupal::service(\Drupal\locale\LocaleFetch::class)->batchVersionCheck($project, $langcode, $context);
\Drupal::service(\Drupal\locale\LocaleFetch::class)->batchStatusCheck($project, $langcode, $options, $context);
\Drupal::service(\Drupal\locale\LocaleFetch::class)->batchDownload($project, $langcode, $context);
\Drupal::service(\Drupal\locale\LocaleFetch::class)->batchImport($project, $langcode, $options, $context);
\Drupal::service(\Drupal\locale\LocaleFetch::class)->batchFinished($success, $results);
\Drupal::service(\Drupal\locale\LocaleProjectChecker::class)->batchFinished($success, $results);
\Drupal::service(\Drupal\locale\LocaleFetch::class)->getStatusOperations($projects, $langcodes, $options);
```

**locale.bulk.inc functions:**
```php
// Before
locale_translate_batch_build($files, $options);
locale_translate_batch_import($file, $options, $context);
locale_translate_batch_import_save($context);
locale_translate_batch_refresh($context);
locale_translate_batch_finished($success, $results);
locale_config_batch_update_components($options, $langcodes, $components, $update_default_config_langcodes);
locale_config_batch_update_default_config_langcodes($context);
locale_config_batch_update_config_translations($names, $langcodes, $context);
locale_config_batch_finished($success, $results);

// After
\Drupal::service(\Drupal\locale\LocaleImportBatch::class)->buildBatch($files, $options);
\Drupal::service(\Drupal\locale\LocaleImportBatch::class)->batchImport($file, $options, $context);
\Drupal::service(\Drupal\locale\LocaleImportBatch::class)->batchSave($context);
\Drupal::service(\Drupal\locale\LocaleImportBatch::class)->batchRefresh($context);
\Drupal::service(\Drupal\locale\LocaleImportBatch::class)->batchFinished($success, $results);
\Drupal::service(\Drupal\locale\LocaleConfigBatch::class)->buildBatch($options, $langcodes, $components, $update_default_config_langcodes);
\Drupal::service(\Drupal\locale\LocaleConfigBatch::class)->batchUpdateDefaultConfigLangcodes($context);
\Drupal::service(\Drupal\locale\LocaleConfigBatch::class)->batchUpdateConfigTranslations($names, $langcodes, $context);
\Drupal::service(\Drupal\locale\LocaleConfigBatch::class)->batchFinished($success, $results);
```

**locale.compare.inc functions:**
```php
// Before
locale_translation_check_projects_batch($projects, $langcodes);
locale_translation_batch_status_build($projects, $langcodes);

// After
\Drupal::service(\Drupal\locale\LocaleProjectChecker::class)->triggerBatch($projects, $langcodes);
// locale_translation_batch_status_build() has no direct equivalent;
// use LocaleProjectChecker::triggerBatch() which calls batch_set() internally.
```

**LocaleFetch method renames** (already deprecated via [#3577671](https://www.drupal.org/node/3577671) but names corrected here):
```php
// Before
\Drupal::service(LocaleFetch::class)->batchUpdateBuild($projects, $langcodes, $options);
\Drupal::service(LocaleFetch::class)->batchFetchBuild($projects, $langcodes, $options);

// After
\Drupal::service(LocaleFetch::class)->buildUpdateBatch($projects, $langcodes, $options);
\Drupal::service(LocaleFetch::class)->buildFetchBatch($projects, $langcodes, $options);
```

**Batch callback strings** used in `BatchBuilder::addOperation()` must also be updated:
```php
// Before (procedural callback strings)
$batch_builder->addOperation('locale_translate_batch_import', [$file, $options]);
$batch_builder->setFinishCallback('locale_translate_batch_finished');

// After (service method strings)
$batch_builder->addOperation(\Drupal\locale\LocaleImportBatch::class . ':batchImport', [$file, $options]);
$batch_builder->setFinishCallback(\Drupal\locale\LocaleImportBatch::class . ':batchFinished');
```

berdir provided the key design direction in comment [#16583848](https://www.drupal.org/node/16583848): rather than splitting callbacks by type (operations, finishers), group them by logical batch, and integrate fetch-related callbacks into existing services while creating a new service for config-related ones. This redirected the initial approach and led to the four-service grouping that shipped. In comment [#16587531](https://www.drupal.org/node/16587531), berdir noted that with this MR all five `.inc` files in locale are now 100% deprecated, with only functions in `locale.module` remaining.
