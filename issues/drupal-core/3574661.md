## Summary

This is a Drupal 12 preparation task that removes deprecated code from the comment module. Several procedural functions, a deprecated class method, a backward-compatibility constructor shim, and a stale library asset alias that had all been scheduled for removal in Drupal 12 are now gone. Site owners and developers who still call any of these removed items will need to update their code before upgrading to Drupal 12.

## Impact

Affects module maintainers.

- **Deprecation:** Removed `comment_uri()`, `comment_preview()`, `template_preprocess_comment()`, `COMMENT_NEW_LIMIT`, `EntityComment::import()`, the `CommentLinkBuilder` constructor backward-compat shim, and the `comment/drupal.comment` library asset alias, all deprecated for drupal:12.0.0.

## Technical details

The diff removes all comment-module deprecations that were scheduled for removal in drupal:12.0.0:

**Procedural functions removed from `comment.module`:**
- `comment_uri(CommentInterface $comment)` (deprecated 11.3.0): callers should use `\Drupal\comment\Entity\Comment::permalink()` directly.
- `comment_preview(CommentInterface $comment, FormStateInterface $form_state)` (deprecated 11.4.0): the logic now lives in `\Drupal\comment\CommentForm::preview()`.
- `template_preprocess_comment(&$variables)` (deprecated 11.3.0): the initial preprocess function is registered directly in `hook_theme()` via `\Drupal\comment\Hook\CommentThemeHooks`.

**Constant removed:** `COMMENT_NEW_LIMIT` is deleted. As andypost noted in the thread, this constant was forgotten when the History module was removed from core; it was never covered by a formal deprecation notice.

**Class method removed:** `EntityComment::import()` in `modules/comment/src/Plugin/migrate/destination/EntityComment.php` (deprecated 11.3.0) is gone. The override temporarily toggled `comment.maintain_entity_statistics` state during stub imports; callers should now use the parent `Entity::import()` directly.

**Constructor shim removed from `CommentLinkBuilder`:** The constructor previously accepted a `ModuleHandlerInterface` as its third argument and an extra `EntityTypeManagerInterface` positional argument for backward compatibility (deprecated 11.3.0). Both the `DeprecatedServicePropertyTrait`, the union type hint, the `#[Autowire]` attribute, and the backward-compat branch are removed. The constructor now accepts only `AccountInterface $current_user`, `CommentManagerInterface $comment_manager`, and `TranslationInterface $string_translation`.

**Library asset alias removed from `comment.libraries.yml`:** The `moved_files` block that redirected `comment/drupal.comment` to the new location (deprecated 11.1.0) is deleted.

**Test class deleted:** `CommentUriDeprecationTest` (a kernel test for the now-removed `comment_uri()` function) is deleted entirely.

**PHPStan baseline cleaned up:** Three suppressed errors that only applied to removed code are deleted from the baseline file:
- `comment_uri()` missing return type
- `$state` might not be defined in `EntityComment.php`
- `CommentUriDeprecationTest::addDefaultCommentField()` missing return type

**Documentation update:** The `uri_callback` example in the entity URI callback docblock is changed from `comment_uri` to `my_entity_uri` to avoid referring to the now-removed function.

`CommentBreadcrumbBuilder` cleanup, initially part of this work, was reverted because that change belongs to [#3459277](https://www.drupal.org/node/3459277) (enforcing the second parameter in `BreadcrumbBuilderInterface::applies()`).

## Upgrade

If your custom or contributed code calls any of the following, update it before upgrading to Drupal 12:

```php
// BEFORE (removed):
comment_uri($comment);

// AFTER:
$comment->permalink();
```

```php
// BEFORE (removed):
comment_preview($comment, $form_state);

// AFTER: No direct replacement. The logic now lives inside
// \Drupal\comment\CommentForm::preview() and is not a public API.
```

```php
// BEFORE (removed):
template_preprocess_comment($variables);

// AFTER: No direct call needed. The function is registered via hook_theme().
```

```php
// BEFORE (removed constructor signature):
new CommentLinkBuilder($current_user, $comment_manager, $module_handler);

// AFTER:
new CommentLinkBuilder($current_user, $comment_manager, $string_translation);
```

```php
// BEFORE (removed constant):
COMMENT_NEW_LIMIT

// AFTER: No replacement exists. Remove any reference to this constant.
```

If you override `EntityComment::import()`, remove the override and rely on the parent `Entity::import()` method.

If your theme registers `comment/drupal.comment` via the old moved-files alias, update the library reference to the current library name directly.
