## Summary

Four procedural functions for rebuilding node access grants have been replaced by a new `NodeAccessRebuild` service in the node module. `node_access_needs_rebuild()` and `node_access_rebuild()` are now deprecated and will be removed in Drupal 13. Any custom module or theme calling these functions should switch to the service. The two internal underscore-prefixed helpers (`_node_access_rebuild_batch_operation` and `_node_access_rebuild_batch_finished`) are removed immediately without deprecation.

## Impact

Affects module maintainers.

- **Deprecation:** `node_access_needs_rebuild()` and `node_access_rebuild()` deprecated in 11.4.0 and removed in 13.0.0; use `NodeAccessRebuild` service methods instead. `_node_access_rebuild_batch_operation()` and `_node_access_rebuild_batch_finished()` removed immediately.
- **New API:** New `Drupal\node\NodeAccessRebuild` service with `needsRebuild()`, `setNeedsRebuild()`, `rebuild()`, `batchOperation()`, and `batchFinished()` methods.

## Technical details

The new `Drupal\node\NodeAccessRebuild` class encapsulates all node access rebuild logic and is registered as an autowired service in `node.services.yml`. It exposes three public methods: `needsRebuild()`, `setNeedsRebuild(bool $rebuild = TRUE)`, and `rebuild(bool $batch_mode = FALSE)`. The state key `'node.node_access_needs_rebuild'` is preserved and exposed as `NodeAccessRebuild::STATE_KEY`. Batch processing logic previously in `_node_access_rebuild_batch_operation()` and `_node_access_rebuild_batch_finished()` is moved into `batchOperation()` and `batchFinished()` instance methods on the service. These are registered with `BatchBuilder` using the `ClassName:methodName` string callable format, which allows the batch system to instantiate the service from the container and use dependency injection rather than static calls. This depends on the `callable` type removal from `BatchBuilder` landed in a companion issue ([#3575096](https://www.drupal.org/node/3575096)).

`RebuildPermissionsForm`, `NodeHelpHooks`, and `NodeModuleHooks` all now receive `NodeAccessRebuild` via constructor injection and call its methods directly instead of the procedural functions. The deprecated `node_access_needs_rebuild()` and `node_access_rebuild()` functions now delegate to the service and emit `E_USER_DEPRECATED` errors; both are deprecated as of drupal:11.4.0 and scheduled for removal in drupal:13.0.0.

## Upgrade

```php
// Before (deprecated in 11.4.0, removed in 13.0.0):
node_access_needs_rebuild();
node_access_needs_rebuild(TRUE);
node_access_needs_rebuild(FALSE);
node_access_rebuild();
node_access_rebuild(TRUE);

// After:
use Drupal\node\NodeAccessRebuild;

$service = \Drupal::service(NodeAccessRebuild::class);

$service->needsRebuild();
$service->setNeedsRebuild(TRUE);
$service->setNeedsRebuild(FALSE);
$service->rebuild();
$service->rebuild(TRUE);
```

In injected classes, add `NodeAccessRebuild` to the constructor:

```php
public function __construct(
  protected readonly NodeAccessRebuild $nodeAccessRebuild,
) {}
```
