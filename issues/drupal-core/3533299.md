## Summary

Four procedural functions for rebuilding node access grants in the node module have been replaced by a new `NodeAccessRebuild` service. `node_access_needs_rebuild()` and `node_access_rebuild()` are now deprecated as of Drupal 11.4.0 and will be removed in 13.0.0; calling them triggers `E_USER_DEPRECATED`. The two internal underscore-prefixed batch helpers (`_node_access_rebuild_batch_operation` and `_node_access_rebuild_batch_finished`) are removed immediately without deprecation because they were never part of the public API. Custom modules or install profiles that call any of these four functions need to update to use the service.

## Impact

Affects module maintainers.

- **Deprecation:** `node_access_needs_rebuild()` and `node_access_rebuild()` deprecated in drupal:11.4.0 and removed in drupal:13.0.0; use `NodeAccessRebuild` service methods instead. `_node_access_rebuild_batch_operation()` and `_node_access_rebuild_batch_finished()` removed immediately without deprecation.
- **New API:** New `Drupal\node\NodeAccessRebuild` service with `needsRebuild()`, `setNeedsRebuild()`, `rebuild()`, `batchOperation()`, and `batchFinished()` methods, and a `STATE_KEY` constant.

## Technical details

The new `Drupal\node\NodeAccessRebuild` class is registered as an autowired service in `node.services.yml` and exposes three public methods: `needsRebuild()`, `setNeedsRebuild(bool $rebuild = TRUE)`, and `rebuild(bool $batch_mode = FALSE)`. The state key `'node.node_access_needs_rebuild'` is preserved and exposed as `NodeAccessRebuild::STATE_KEY`. Batch processing logic previously in the two underscore-prefixed functions is moved into `batchOperation()` and `batchFinished()` instance methods on the service. These are registered with `BatchBuilder` using the `ClassName:methodName` string callable format (e.g. `static::class . ':batchOperation'`), which allows the batch system to resolve and instantiate the service from the container using dependency injection rather than static function calls. This approach depends on the `callable` type removal from `BatchBuilder` that landed in [#3575096](https://www.drupal.org/node/3575096).

`RebuildPermissionsForm`, `NodeHelpHooks`, and `NodeModuleHooks` all now receive `NodeAccessRebuild` via constructor injection and call its methods directly. The deprecated `node_access_needs_rebuild()` and `node_access_rebuild()` functions delegate to the service and emit `E_USER_DEPRECATED`. The `hook_node_grants` API documentation was also updated to reference `\Drupal\node\NodeAccessRebuild::rebuild()` in place of the old procedural function name.

The first merge of this change was immediately reverted (commit `b5d0936a`) after alexpott noticed that some documentation comments still referenced `node_access_rebuild()` by name; the clean final commit (`3a0d690d`) includes those documentation fixes.

## Upgrade

```php
// Before (deprecated in 11.4.0, removed in 13.0.0):
node_access_needs_rebuild();
node_access_needs_rebuild(TRUE);
node_access_needs_rebuild(FALSE);
node_access_rebuild();
node_access_rebuild(TRUE);

// After:
use Drupal\node\NodeAccessRebuild;

$service = \Drupal::service(NodeAccessRebuild::class);

$service->needsRebuild();
$service->setNeedsRebuild(TRUE);
$service->setNeedsRebuild(FALSE);
$service->rebuild();
$service->rebuild(TRUE);
```

In injected classes, add `NodeAccessRebuild` to the constructor:

```php
public function __construct(
  protected readonly NodeAccessRebuild $nodeAccessRebuild,
) {}
```
