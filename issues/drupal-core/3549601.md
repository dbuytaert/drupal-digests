## Summary

When running `core/scripts/run-tests.sh` with both the `--directory` option and a positional `@group` argument, the group filter was previously ignored and all tests in the directory were run. This made it impossible for contrib module maintainers to do the two things at once: limit test discovery to their module's directory (which avoids fatal errors from unrelated third-party modules) and still run only a specific test group. The fix makes the two options work together: `--directory` narrows the file scan and the group argument then filters the discovered tests down to the requested subset. The script now also prints a "Test discovery" header summarizing which directory, module, test suite(s), and group(s) are active for the run.

## Impact

No upgrade or configuration changes required.

## Technical details

The root cause was that in Drupal 11.2+ the `run-tests.sh` script adopted PHPUnit 11's test discovery API (`PhpUnitTestDiscovery`). When `--directory` was in use, `getTestClasses()` called the `getTestListLimitedToDirectory()` code path, but the positional group argument from `Config::getTests()` was never forwarded to the discovery layer; it was only used in the separate "groups-only" code path. The fix adds a `$testGroups` parameter to the public `PhpUnitTestDiscovery::getTestClasses()` and `findAllClassFiles()` methods, and to the private helpers `getTestList()` and `getTestListLimitedToDirectory()`. Both helpers now apply an `array_intersect` check against `$item['groups']` and skip any test class whose groups do not overlap with the requested list. The call sites in `run-tests.sh` (`simpletest_script_get_test_list()` and the `--list-files-json` path) are updated to pass `Config::getTests()` as `$testGroups`. A new guard in `Config::createFromCommandLine()` throws a `RuntimeException` when `--directory` is combined with `--class` or `--file`, since those options are all file-based and mutually exclusive. This fix was built on top of the Symfony Console-based argument parsing introduced in [#3536964](https://www.drupal.org/node/3536964).
