## Summary

This is a public follow-up to security advisory SA-CORE-2025-008. It adds functional test coverage to verify that the `Cache-Control` response header is set correctly when files are served through custom file stream wrapper schemes. The change is test-infrastructure-only and has no user-facing effect beyond the expanded test coverage.

## Impact

No upgrade or configuration changes required.

## Technical details

Drupal's file system supports custom stream wrappers (e.g., `public://`, `private://`) registered as tagged services. SA-CORE-2025-008 fixed a defect where custom stream wrappers could inadvertently serve files with incorrect `Cache-Control` headers. This issue adds the public test suite that was originally developed in the private security issue by larowlan, xjm, and mohit_aghera.

The change introduces two new stream wrapper classes inside the `file_test` module: `DummyPrivateStreamWrapper` (scheme `dummy-private://`) and `DummyPublicStreamWrapper` (scheme `dummy-public://`), both extending `DummyStreamWrapper` and overriding `getExternalUrl()` to point at new test routes. They are registered in `file_test.services.yml` with the `stream_wrapper` service tag. Two matching download routes are added to `file_test.routing.yml`, both handled by `FileDownloadController::download`, at paths `/system/dummy` and `/system/dummy-public` respectively.

The `fileDownload` hook implementation in `FileTestHooks.php` is extended to inspect a `file_test_active_stream_wrapper` key-value flag and return `['Cache-Control' => 'private']` or `['Cache-Control' => 'public']` accordingly, enabling the test to control which header is emitted per scenario.

The new functional test class `DummyStreamWrapperFileTest` in `core/modules/file/tests/src/Functional/` uses a `streamWrapperProvider` data provider to cover four cases: `dummy-public://` (expects `public` at `/system/dummy-public`), `private://` (expects `private` at `/system/files`), `dummy-private://` (expects `private` at `/system/dummy`), and `temporary://` (expects `private` at `/system/temporary`). Each case writes a file, attaches it to an `EntityTest` entity, and asserts the `Cache-Control` response header on the file download URL. The `file_additional_public_schemes` setting is used in test setup to register `dummy-public` as a public scheme.
