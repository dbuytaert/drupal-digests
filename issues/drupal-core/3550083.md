## Summary

Search engines were indexing search result pages with query parameters, creating poor quality indexes and performance problems. Drupal's default robots.txt file only blocked path-based search URLs like `/search/keyword`, but not query-based URLs like `/search?q=keyword`. This caused crawlers to index every possible combination of faceted search results, creating an endless crawl that degraded site performance and SEO quality. The default robots.txt file now blocks both types of search URLs from being crawled. Sites with custom robots.txt files may want to add these rules manually.

## Impact

No upgrade or configuration changes required.

## Technical details

The default `robots.txt` file included a `Disallow: /search/` rule that only matched search URLs with a trailing slash. Core's search module uses paths like `/search/node` and `/search/user`, which are covered by this rule. However, many sites use custom search implementations built with Views, Search API, or faceted search that generate URLs with query parameters like `/search?title=robots`. These were not blocked by the existing rule.

Search engine crawlers were following these unblocked URLs and attempting to index every possible combination of search filters and facets. This created performance problems and poor SEO outcomes. The robots.txt specification supports matching query parameters with the `?` character.

The fix adds two new disallow rules: `Disallow: /search?` and `Disallow: /index.php/search?`. These complement the existing rules and block search URLs with query parameters from being crawled. The change updates both the root `robots.txt` file and the scaffold template in `core/assets/scaffold/files/robots.txt`.

## Upgrade

This change only affects new Drupal installations and sites that use the scaffolded robots.txt file. Existing sites with custom robots.txt files are not automatically updated.

If you maintain a custom robots.txt file, consider adding these rules manually:

```
Disallow: /search?
Disallow: /index.php/search?
```

These rules block search engine crawlers from indexing search result pages with query parameters. This is particularly important for sites using faceted search, Search API views with exposed filters, or any custom search implementation that uses query strings.

See change record [#3567198](https://www.drupal.org/node/3567198) for more details.
