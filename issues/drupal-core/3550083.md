## Summary

Search engines were indexing dynamically generated search result pages on Drupal sites, especially those using query parameters like `/search?q=keyword`. This caused crawlers to index poor quality content and follow infinite combinations of faceted search pages, degrading site performance and SEO. The default robots.txt file already blocked path-based search URLs like `/search/keyword`, but did not block query parameter searches. The updated robots.txt now blocks both patterns, protecting sites from excessive crawler activity on search pages. This only affects new installations and sites that use scaffolded files.

## Impact

No upgrade or configuration changes required.

## Technical details

The default `robots.txt` file included `Disallow: /search/` with a trailing slash, which blocked URLs like `/search/node` but did not block `/search?title=robots`. According to Google's robots.txt specification, the pattern `/search/` matches paths that continue after the trailing slash but does not match when a query string immediately follows.

Sites using Search API, Views-based search, or faceted search typically generate URLs with query parameters rather than path segments. Crawlers would index every possible combination of facet filters, creating performance issues and polluting search engine indexes.

The fix adds two new disallow rules: `Disallow: /search?` and `Disallow: /index.php/search?`. These specifically block URLs where query parameters immediately follow the search path. The original `/search/` rules remain in place to continue blocking path-based search URLs. Both the root `robots.txt` and the scaffold version in `core/assets/scaffold/files/robots.txt` were updated.

Because `robots.txt` is a scaffolded file that sites may customize, existing installations will not automatically receive this update.

## Upgrade

Existing sites should manually update their `robots.txt` file if they want to block crawler access to search pages with query parameters. Add these two lines to the file:

```
Disallow: /search?
Disallow: /index.php/search?
```

Place them near the existing `Disallow: /search/` rules for consistency. If you have customized your `robots.txt` file or use a different search path, adjust the rules accordingly. New installations and sites that regenerate scaffolded files will automatically receive these updated rules.
