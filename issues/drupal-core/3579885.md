## Summary

This is a purely internal test-code change with no effect on production behavior. It updates unit tests in the Render library to be compatible with PHPUnit 12.5's stricter distinction between mock objects and stubs, eliminating deprecation notices that would otherwise fire during the upgrade path to PHPUnit 12.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5 treats mock objects and stubs as fundamentally different: a mock (`createMock()`) must have at least one expectation configured, and `$this->any()` (which expresses no real expectation) is deprecated. An object that only needs to return preset values should be a stub (`createStub()`).

The main structural change is in `RendererTestBase`, the shared base class for the renderer suite. Dependencies that require no call-count verification (`CallableResolver`, `ThemeManagerInterface`, `ElementInfoManagerInterface`, `VariationCacheFactoryInterface`, `CacheContextsManager`) are now created with `createStub()` in `setUp()`. The `expects($this->any())` prefix is removed from all the `.method()->willReturn*` chains on those objects, because stubs do not accept expectations.

To handle tests that do need real expectations on these objects, four new helper methods were added to `RendererTestBase`: `setUpMockThemeManager()`, `setUpMockCallableResolver()`, `setUpMockVariationCacheFactory()`, and `setUpMockCacheContextsManager()`. Each one replaces the stub with a fresh `createMock()` and injects it into the already-constructed `$this->renderer` or `$this->renderCache` via `ReflectionProperty`. Tests in `RendererTest` that configure expectations on `$this->themeManager` now call `$this->setUpMockThemeManager()` before doing so, and `setUpUnusedCache()` and `setUpMemoryCache()` call `setUpMockVariationCacheFactory()` to do the same for the cache factory.

`BubbleableMetadataTest` replaces a `getMockBuilder(Renderer::class)->onlyMethods([])->getMock()` partial mock (used to allow `mergeAttachments()` to execute normally) with a real `new Renderer(...)` instance, wiring its constructor arguments with stubs. This is a cleaner and more forward-compatible pattern.

`ElementInfoManagerTest` removes its shared `setUp()` mock properties entirely; each test now creates its own local stubs and mock, which is more idiomatic and avoids shared state.

Six PHPStan baseline suppressions covering the now-removed `any()` calls in the Render test namespace are deleted from `core/.phpstan-baseline.php`.

The `MetadataBubblingUrlGeneratorTest` changes that were originally in scope were moved to [#3579890](https://www.drupal.org/node/3579890) (Database, Routing), because that test class is a child of a routing test class and the MR there needed it to pass.
