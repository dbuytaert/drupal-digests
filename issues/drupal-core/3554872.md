## Summary

Drupal previously relied on `symfony/phpunit-bridge` to collect and report deprecation notices during test runs. With PHPUnit 11's native deprecation reporting now in place and tested upstream, the Drupal-side test classes written specifically to verify the bridge integration became dead code. This task removes those test classes and their dedicated fixture, reducing maintenance overhead and eliminating a functional test that was also exhibiting random failures.

## Impact

No upgrade or configuration changes required.

## Technical details

Drupal's test framework had a layer built around `symfony/phpunit-bridge` for propagating deprecation notices from the system under test (SUT) back to the test runner. PHPUnit 11 introduced its own native deprecation handling, making the bridge and its associated scaffolding redundant. This issue removed everything that existed solely to test that bridge integration.

**Files deleted:**
- `core/tests/Drupal/FunctionalTests/Core/Test/PhpUnitBridgeTest.php` — functional test for the bridge (`BrowserTestBase`), also noted as flaky in CI.
- `core/tests/Drupal/KernelTests/Core/Test/PhpUnitBridgeTest.php` — kernel test for the bridge (`KernelTestBase`).
- `core/tests/Drupal/Tests/Core/Test/PhpUnitBridgeTest.php` — unit test for the bridge (`UnitTestCase`).
- `core/tests/Drupal/Tests/Core/Test/PhpUnitBridgeIsolatedTest.php` — unit test with process isolation.
- `core/tests/Drupal/Tests/Core/Listeners/DrupalStandardsListenerDeprecationTest.php` — unit test for `DrupalStandardsListener` deprecation handling (needed a deprecated class as a fixture just to verify the listener ignored deprecation errors during coverage analysis).
- `core/tests/Drupal/Tests/ExpectDeprecationTest.php` — test exercising `ExpectDeprecationTrait::expectDeprecation()` from the bridge.
- `core/modules/system/tests/modules/deprecation_test/src/Deprecation/DrupalStandardsListenerDeprecatedClass.php` — the arbitrarily-deprecated fixture class whose sole purpose was serving `DrupalStandardsListenerDeprecationTest`.

**Preserved test, relocated:** During review, mondrake identified that `testErrorOnSiteUnderTest()` from `FunctionalTests\Core\Test\PhpUnitBridgeTest` tests a Drupal-specific mechanism: `TestHttpClientMiddleware` intercepts deprecation notices that originate in the SUT (not in the test process) by reading them out of the HTTP response header and re-firing them as `E_USER_DEPRECATED` so PHPUnit can collect them. That behavior is independent of the symfony bridge and still needs coverage. The test was moved to `BrowserTestBaseTest::testDeprecationTriggeredInSystemUnderTest()` in `core/tests/Drupal/FunctionalTests/BrowserTestBaseTest.php`, with a more detailed docblock explaining the `TestHttpClientMiddleware` and `_drupal_error_handler` chain.

**Ancillary cleanup:**
- `FixtureDeprecatedClass.php` docblock updated to remove stale references to the deleted `DrupalStandardsListenerDeprecationTest` and instead point to its remaining consumer, `ServiceDeprecationTest`.
- A comment in `TestHttpClientMiddleware::__invoke()` that referenced the now-gone `DeprecationBridge\DeprecationHandler::collectActualDeprecation()` was simplified to "collected by PHPUnit".
