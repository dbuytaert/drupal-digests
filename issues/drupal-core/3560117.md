## Summary

This meta issue coordinated the work needed to add the Navigation module to Drupal's Standard profile and recipes. Navigation is the modern replacement for the Toolbar module, providing a new administrative interface. Before Navigation could be enabled by default, critical accessibility issues had to be resolved and the module needed to be marked stable. This change paves the way for eventually deprecating Toolbar and removing legacy JavaScript libraries like Underscore and Backbone from Drupal core.

No upgrade or configuration changes required.

This meta issue tracked the coordination of multiple child issues required to enable the Navigation module in Standard profile and recipes. The main technical work happened in child issue [#3575171](https://www.drupal.org/node/3575171), which added Navigation to Standard profile and recipes, and [#3560118](https://www.drupal.org/node/3560118), which replaced Toolbar with Navigation in the Umami demo profile.

The issue was initially postponed on [#3391723](https://www.drupal.org/node/3391723) until critical accessibility issues were resolved. These included fixing focusable elements in closed sidebars, high contrast mode refinements, and invalid ARIA attributes. Once the accessibility team confirmed all blocking issues were fixed, the work proceeded.

A performance consideration was also addressed in [#3576727](https://www.drupal.org/node/3576727), which integrated Navigation's performance test into `StandardPerformanceTest`. The standalone `\Drupal\Tests\navigation\FunctionalJavascript\PerformanceTest` became redundant after this change.
