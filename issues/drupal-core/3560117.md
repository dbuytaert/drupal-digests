## Summary

This meta issue coordinated adding the Navigation module to Drupal's Standard install profile, standard recipe, and Umami demo profile, replacing the Toolbar module as the default admin navigation experience for new Drupal installations. The change required resolving a set of outstanding accessibility blockers first, which were tracked separately. Once those were fixed, the actual code landed through three child issues: [#3560118](https://www.drupal.org/node/3560118) (Umami), [#3575171](https://www.drupal.org/node/3575171) (Standard profile and recipes), and [#3576727](https://www.drupal.org/node/3576727) (performance tests). Site owners installing Drupal fresh from the Standard profile or standard recipe now get the Navigation module instead of Toolbar.

## Impact

No upgrade or configuration changes required.

## Technical details

The meta issue grouped three "must/should have" child issues under a single tracking node. The real code changes are in the children:

- [#3560118](https://www.drupal.org/node/3560118) replaced Toolbar with Navigation in `demo_umami.info.yml`.
- [#3575171](https://www.drupal.org/node/3575171) added `navigation` to `standard.info.yml` and to `core/recipes/standard/recipe.yml` (including `navigation.block_layout`, `navigation.settings`, and the `system.menu.navigation-user-links` config, plus the `access navigation` permission for the authenticated user role). Toolbar was removed from both.
- [#3576727](https://www.drupal.org/node/3576727) moved the Navigation performance assertions from the now-redundant `NavigationPerformanceTest` into `StandardPerformanceTest`, reflecting that Navigation is now the standard admin bar in a fresh install.

The issue was briefly postponed on accessibility grounds: `longwave` noted (after discussion with `catch`) that proceeding required the critical items in [#3391723](https://www.drupal.org/node/3391723) to be resolved first. Once those blocking accessibility fixes were committed, `catch` unpostponed the work and noted that this meta itself is a blocker for [#3484850](https://www.drupal.org/node/3484850), the task to deprecate the Toolbar module, which in turn blocks removing the Underscore.js and Backbone.js libraries from core.

The current repo confirms `navigation` is listed as a dependency in `core/profiles/standard/standard.info.yml` and `core/recipes/standard/recipe.yml`, and is present in `core/profiles/demo_umami/demo_umami.info.yml`. No `toolbar` dependency remains in those files.
