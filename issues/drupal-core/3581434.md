## Summary

Sites using Layout Builder with content moderation and multiple translations could hit a fatal serialization error when editing node layouts. The error would surface in `PrepareLayout::onPrepareLayout()` when Drupal tried to store the section storage to the layout temporary storage. This bug is fixed in 11.x and 11.3.x. No configuration or code changes are required for site owners.

## Impact

No upgrade or configuration changes required.

## Technical details

The root cause is a broken reference in `ContentEntityBase::__clone()`. When a translation is initialized via `initializeTranslation()`, the `$originalEntity` property is turned into a PHP reference shared between the entity and all its translations, keeping them in sync. This is the same pattern used for several other properties (`enforceIsNew`, `newRevision`, `fields`, etc.). For all those other properties, `__clone()` explicitly breaks the reference by reassigning a local copy. However, `$originalEntity` was omitted from that list (the handling was removed during the original work in [#2839195](https://www.drupal.org/node/2839195) without explanation).

When `ContentEntityStorageBase::createRevision()` clones an entity (as triggered by `ContentModeration\EntityTypeInfo::entityPrepareForm()`), the clone inherits the shared reference. When content moderation then calls `setOriginal()` on the draft, the value propagates via the still-live reference to every other entity that shares it. Worse, because the entity being set as the original is itself one of the objects sharing the reference, `$this->originalEntity` ends up pointing to the entity itself, creating a circular self-reference that breaks PHP serialization.

The fix is a three-line addition to `ContentEntityBase::__clone()` that follows the exact same pattern already in place for the other properties:

```php
$originalEntity = $this->originalEntity;
$this->originalEntity = &$originalEntity;
```

This breaks the shared reference so that post-clone changes to `$originalEntity` on one object do not propagate to others. `ContentEntityCloneTest` received a new dedicated `testEntityOriginal()` method that covers the translation-sharing behavior, the clone-isolation requirement, and the merged-revision-original scenario where `createRevision()` sets an original automatically. The existing skip of the `originalEntity` property in `testEntityPropertiesModifications()` was updated to point to this new test instead.
