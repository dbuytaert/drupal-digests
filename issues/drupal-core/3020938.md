## Summary

A new `view unpublished block content` permission has been added to the `block_content` module. Previously, viewing an unpublished block (for example a Draft block created through Content Moderation) required either `administer block content` or `access block library`, both of which grant much broader access than simply viewing. Editors who had only the `create and edit custom blocks` permission could place unpublished blocks via Layout Builder but could not view them on the actual page. The new permission allows sites to grant scoped view access to unpublished block content without elevating those users to library or admin roles.

## Impact

Affects site owners.

- **Permission change:** New 'view unpublished block content' permission allows viewing draft block content without requiring 'administer block content' or 'access block library'.

## Technical details

The permission is declared in `block_content.permissions.yml`. The access logic in `BlockContentAccessControlHandler::checkAccess()` was updated so that the `view` operation allows access when the entity is published, or when the account has either `access block library` OR the new `view unpublished block content` permission (evaluated with an `OR` combinator via `allowedIfHasPermissions()`). A `addCacheableDependency($entity)` call was also added to the view access result so that the published/unpublished state of the entity is correctly reflected in the cache metadata. Tests were updated to cover the new permission case and to correct an existing test-case label (`view:unpublished:reusable:admin` renamed to `view:unpublished:reusable:library`). Expected unauthorized access messages in REST/JSON:API tests now reflect the two-permission alternative: `'access block library' OR 'view unpublished block content'`.

## Upgrade

No code changes are required. Sites using Content Moderation with block content should review editor roles and consider granting the new `view unpublished block content` permission instead of the broader `access block library` or `administer block content` permissions if the only need is viewing draft blocks.
