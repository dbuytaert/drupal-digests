## Summary

The Telephone module (`telephone`) has been officially deprecated in Drupal core. Sites and contributed modules that currently depend on this module should plan to migrate to the contributed version before it is removed in a future major release. A change record and updated documentation page at https://www.drupal.org/node/3223395#s-telephone describe the recommended migration path.

## Impact

## Technical details

The deprecation was executed following the standard core module removal process approved in [#3266465](https://www.drupal.org/node/3266465). The change adds `lifecycle: deprecated` and `lifecycle_link: https://www.drupal.org/node/3223395#s-telephone` to `telephone.info.yml`. Drupal's extension system reads these keys to emit deprecation notices when the module is installed.

The three test classes in the module (`GenericTest`, `TelephoneFieldTest`, `TelephoneItemTest`) each received the `#[IgnoreDeprecations]` PHPUnit attribute so the module's own test suite does not fail due to its own deprecation notice while it remains in core.

No libraries exclusive to the module required deprecation, and no references to `telephone` outside the module itself were found that needed separate issues.

## Upgrade

Sites that have the Telephone module enabled should install the contributed replacement and migrate their field data before upgrading to the major Drupal version that removes the module. See https://www.drupal.org/node/3223395#s-telephone for instructions.

Contributed modules that declare a dependency on `drupal:telephone` should update their `.info.yml` to depend on the contributed package instead:

```yaml
# Before
dependencies:
  - drupal:telephone

# After (once migrated to the contributed module)
dependencies:
  - telephone:telephone
```
