## Summary

PHPUnit 12.5.0 began emitting notices when a mock object has no call expectations configured, recommending test stubs be used instead. This change updates the `unit-component` test suite in Drupal core to silence those notices by switching to stubs or adding explicit `never()` expectations where appropriate. There is no user-facing or API impact; this is a test hygiene improvement to keep the test suite clean against newer PHPUnit versions.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 issues a notice for any mock object created with `createMock()` or `getMockBuilder()->getMock()` that never has `expects(...)` configured. The correct fix in such cases is to use `createStub()` instead, which creates a test double without expectation tracking. Eight files in `core/tests/Drupal/Tests/Component/` were updated across two strategies:

**Strategy 1: Convert to `createStub()`** — used where the object is only needed for method return values, not for verifying call counts.
- `Datetime/TimeTest.php`: `getMockBuilder('...RequestStack')->getMock()` replaced with `createStub(RequestStack::class)`; `expects($this->any())` calls on `getCurrentRequest()` removed; the property type updated to the intersection type `RequestStack&Stub`.
- `Plugin/DefaultFactoryTest.php`: inline `getMockBuilder(PluginDefinitionInterface::class)->getMock()` replaced with `createStub(PluginDefinitionInterface::class)`.
- `Plugin/Discovery/AttributeBridgeDecoratorTest.php`: `createMock(DiscoveryInterface::class)` with `expects($this->any())` replaced by `createStub()`.
- `Plugin/Discovery/StaticDiscoveryDecoratorTest.php`: two `getMockBuilder(StaticDiscoveryDecorator::class)->disableOriginalConstructor()->onlyMethods([])->getMock()` patterns replaced with direct instantiation `new StaticDiscoveryDecorator($this->createStub(DiscoveryInterface::class))`, which is both cleaner and avoids bypassing the constructor.
- `Plugin/Factory/ReflectionFactoryTest.php`: `getMockBuilder(ReflectionFactory::class)->disableOriginalConstructor()->getMock()` replaced with `createStub(ReflectionFactory::class)`.
- `Component/Utility/ArgumentsResolverTest.php`: four `getMockBuilder('...')->getMock()` calls for `Test1Interface`, `TestClass`, and `Test2Interface` replaced with `createStub()`.

**Strategy 2: Add explicit `never()` expectations** — used where converting to a stub was not straightforward because the mock is also manipulated via reflection or has other expectations in different branches.
- `Plugin/Context/ContextTest.php`: added `expects($this->never())->method('getContextDefinition')` to a branch that exercises `getContextValue()` without going through the definition path.
- `Plugin/Discovery/DiscoveryCachedTraitTest.php`: added `expects($this->never())->method('getDefinitions')` for the branch where definitions are already cached and `getDefinitions()` should not be called.

mondrake noted in a comment that PHPUnit has also signalled plans to deprecate the `any()` matcher entirely (sebastianbergmann/phpunit[#6461](https://www.drupal.org/node/6461)), and that Drupal has approximately 1,700 `createMock()` calls to address across its test suites. This issue covers only the `unit-component` suite.
