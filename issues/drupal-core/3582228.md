## Summary

`HttpKernelUiHelperTrait::drupalGet()`, introduced in [#3390193](https://www.drupal.org/node/3390193), now accepts two optional parameters: `$options` for URL query-string options and `$headers` for custom HTTP request headers. The `$path` argument can now also be a `\Drupal\Core\Url` object instead of a plain string. Kernel tests that previously could not pass query parameters or request headers to `drupalGet()` can now do so the same way functional browser tests do, making it easier to convert functional tests to lighter kernel tests.

## Impact

Affects module maintainers.

- **New API:** New `$options` and `$headers` parameters on `HttpKernelUiHelperTrait::drupalGet()`; `$path` now also accepts `\Drupal\Core\Url`; new `protected buildUrl()` helper method added to the trait.

## Technical details

`HttpKernelUiHelperTrait::drupalGet()` signature changed from `drupalGet($path): string` to `drupalGet($path, array $options = [], array $headers = []): string`. The `$path` parameter now accepts `\Drupal\Core\Url|string`.

A new `protected` helper `buildUrl(Url|string $path, array $options = []): string` was added to the trait. It resolves string paths through `Url::fromUri('base:/' . $path, $options)` and handles the `<front>` special case. When `$path` is a `Url` object and `$options` is non-empty, the object's own options take precedence (merged as `$url_options + $options`). Absolute URLs are explicitly prohibited via an `assert`.

Headers are applied by calling `$session->setRequestHeader()` for each entry before `$session->visit()`. The docblock notes the normalization chain: Mink's `BrowserKitDriver` converts header names to `UPPER_CASE`, while Symfony's `HeaderBag` then converts them to `lower-case`, so `WebAssert::responseHeaderExists()` may not work as expected for request headers set this way.

## Upgrade

If you use `HttpKernelUiHelperTrait::drupalGet()` in a kernel test and need query parameters or custom headers, update the call:

```php
// Before: had to embed query strings in the path string.
$this->drupalGet('/system-test/set-header?name=meaning&value=42');

// After: use the $options array, matching functional test conventions.
$this->drupalGet('/system-test/set-header', [
  'query' => ['name' => 'meaning', 'value' => '42'],
]);

// After: pass a Url object directly.
$url = Url::fromRoute('system_test.set_header', [], [
  'query' => ['name' => 'meaning', 'value' => '42'],
]);
$this->drupalGet($url);

// After: add custom request headers.
$this->drupalGet('/some/path', [], ['Accept-Language' => 'fr']);
```
