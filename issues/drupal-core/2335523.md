## Summary

In Drupal 11.4.0, the layout CSS file `node.module.css` was removed from the `node/drupal.node` asset library, the `node/form` library was deprecated (scheduled for removal in Drupal 12.0.0), and the `node-edit-form.html.twig` template was deleted from both the `demo_umami` and `starterkit_theme` themes. The CSS in that file defined a two-column layout for node edit forms using classes such as `.layout-region-node-main` and `.layout-region-node-secondary`, but investigation confirmed those classes are not present on the node form in a standard Claro or Olivero install. Claro had already been suppressing the file via `libraries-override`. The change removes dead CSS and cleans up the associated library definitions. The `node.module.css` file itself is not yet deleted; that will happen in the follow-up issue [#3573705](https://www.drupal.org/node/3573705).

## Impact

Affects module maintainers and front-end developers.

- **Theme change:** Themes that had `libraries-override` entries suppressing or replacing `css/node.module.css` from `node/drupal.node` or `node/form` can remove those overrides.
- **Deprecation:** `node/form` asset library deprecated in `drupal:11.4.0`, removed in `drupal:12.0.0`. No replacement; callers should attach `node/drupal.node` instead.

## Technical details

The node module's `node.module.css` provided a two-column layout for the node edit form using classes `.layout-region-node-main`, `.layout-region-node-secondary`, and `.layout-region-node-footer`. This file was referenced by two separate libraries: `node/drupal.node` and `node/form`, both defined in `core/modules/node/node.libraries.yml`. Both libraries were attached inside `NodeForm::form()` in `core/modules/node/src/Form/NodeForm.php`: `node/drupal.node` was attached redundantly on the individual "author" and "options" fieldsets, while `node/form` was attached at the form level.

Investigation confirmed that the layout classes are not rendered on the node form with standard profiles in Claro or Olivero. Claro had already removed the CSS from both libraries via `libraries-override` in `claro.info.yml`, and stable9 had been substituting its own copy. The CSS was therefore dead in practice for all core themes.

The commit made the following changes:

- `core/modules/node/node.libraries.yml`: removed the `css/node.module.css` entry under the `drupal.node` library; added a `deprecated` key to the `form` library declaring deprecation in `drupal:11.4.0` and removal in `drupal:12.0.0` with no replacement.
- `core/modules/node/src/Form/NodeForm.php`: removed the redundant per-element `#attached library` calls for `node/drupal.node` from the author and options fieldsets; changed the form-level library attachment from `node/form` to `node/drupal.node`.
- `core/profiles/demo_umami/themes/umami/templates/classy/content-edit/node-edit-form.html.twig`: deleted (the layout classes in the template no longer have backing CSS in core).
- `core/themes/starterkit_theme/templates/content-edit/node-edit-form.html.twig`: deleted for the same reason.
- `core/themes/claro/claro.info.yml` and `core/themes/stable9/stable9.info.yml`: removed the `libraries-override` blocks that had been suppressing or replacing `node.module.css` from both `node/drupal.node` and `node/form`.
- `core/tests/Drupal/KernelTests/Core/Theme/Stable9LibraryOverrideTest.php`: added `node/form` to the list of libraries excluded from the stable9 override check, with a comment pointing to follow-up issue [#3573705](https://www.drupal.org/node/3573705) where the file and library will be fully removed.

Because an empty library definition is not currently valid, `node.module.css` remains on disk and stays referenced by the deprecated `node/form` library. The follow-up issue [#3573705](https://www.drupal.org/node/3573705) will delete the file and the library entry entirely.

## Upgrade

If your module or theme attaches the `node/form` library, switch to `node/drupal.node`:

```yaml
# Before
dependencies:
  - node/form

# After
dependencies:
  - node/drupal.node
```

If your theme's `*.info.yml` has `libraries-override` entries for `node/drupal.node` or `node/form` that suppressed or replaced `css/node.module.css`, remove them:

```yaml
# Remove these blocks from libraries-override:
  node/drupal.node:
    css:
      layout:
        css/node.module.css: false   # or a replacement path

  node/form:
    css:
      layout:
        css/node.module.css: false   # or a replacement path
```
