## Summary

On sites using the Workspaces module, AJAX-enabled form elements that already carried scalar `token` or `persist` query parameters (as can happen when the Media Library module copies request parameters into form elements across AJAX requests) would have those values silently promoted to arrays. The next AJAX request then failed with a Symfony `BadRequestException: Input value contains a non-scalar value`. The fix corrects the merge strategy in the Workspaces form hook so workspace query parameters are always scalar, preventing this crash.

## Impact

No upgrade or configuration changes required.

## Technical details

`FormOperations::setAjaxWorkspace()` in `core/modules/workspaces/src/Hook/FormOperations.php` injects workspace-identifying query parameters (`workspace`, `token`, `persist`) into every AJAX-enabled form element's `#ajax['options']['query']` array. Previously it used `array_merge_recursive()` to combine the workspace-specific `$url_query_options` with any pre-existing query values on the element. `array_merge_recursive()` collapses duplicate scalar keys into arrays rather than overwriting them, so when an element already had its own `token` or `persist` values, the merge produced arrays like `['workspace-token', 'pre-existing-token']` for `token` and `[false, '']` for `persist`. Symfony's `InputBag::get()` then rejects those non-scalar values with a `BadRequestException`.

The fix replaces `array_merge_recursive()` with `NestedArray::mergeDeep()` from `Drupal\Component\Utility\NestedArray`. The argument order was also inverted: `$existing_query` is now the base and `$url_query_options` (the workspace parameters) are applied on top, so workspace values always overwrite pre-existing ones rather than accumulating alongside them. Additionally, the prior guard `!isset($element['#ajax']['options']['query']['workspace'])` was removed; workspace parameters are now unconditionally applied to every `#ajax` element on every form build, which is safe because the new merge direction makes the operation idempotent.

A `collision_test` field was added to `ActiveWorkspaceTestForm` (in the `workspaces_test` module) pre-seeded with a `media_library_opener_id` query parameter. `WorkspaceFormPersistenceTest` asserts that after the merge the pre-existing key survives untouched while `workspace`, `token`, and `persist` are present as scalars with the correct workspace values.
