## Summary

New Drupal sites installed with the Standard profile or the standard recipe now use the Navigation module as the default administrative toolbar, replacing the legacy Toolbar module. The `content_editor` role's toolbar permission is updated to match. This change only affects new installations; existing sites are not automatically migrated.

## Impact

Affects site owners.

- **Permission change:** The `content_editor` role in the Standard profile and standard recipe now grants `access navigation` instead of `access toolbar`.

## Technical details

In `core/profiles/standard/standard.info.yml`, `toolbar` is replaced by `navigation` in the module install list. The corresponding role configuration in `user.role.content_editor.yml` drops the `toolbar` module dependency and substitutes `access navigation` for `access toolbar`.

The standard recipe (`core/recipes/standard/recipe.yml`) receives the same module swap, and additionally declares explicit config imports for the Navigation module: `navigation.block_layout`, `navigation.settings`, `system.menu.content`, and `system.menu.navigation-user-links`. The recipe's `content_editor` role definition is also updated to grant `access navigation` instead of `access toolbar`.

Standard profile performance tests were updated to reflect Navigation's different query and cache behavior: the anonymous front-page scenario drops from 39 to 37 queries and reduces `CacheSetCount` from 45 to 43, while a logged-in node-page scenario adds one query and gains additional cache gets due to Navigation's menu handling. The initial 11.x commit was reverted because `migrate_drupal_ui` tests (present in 11.x but not in main) required updates to expected menu entity counts; a follow-up MR fixed those tests and the change was re-landed.

## Upgrade

Sites that have customized or replicated the `content_editor` role from the Standard profile should update the role's permission and module dependency:

```yaml
# Before
dependencies:
  - toolbar
permissions:
  - 'access toolbar'

# After
dependencies:
  - navigation
permissions:
  - 'access navigation'
```

Existing sites that installed Standard before this change are not affected automatically. To switch an existing site from Toolbar to Navigation, enable the `navigation` module, update any role that holds `access toolbar` to `access navigation` instead, and disable or uninstall `toolbar`.
