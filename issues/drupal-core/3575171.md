## Summary

The Standard installation profile and Standard recipe now install the Navigation module instead of the Toolbar module. This brings the modern navigation interface to new Drupal sites by default. Sites created with the Standard profile will use Navigation as the administrative menu system, providing a more streamlined experience aligned with current Drupal direction. The change affects default permissions for the content editor role, which now grants access to Navigation instead of Toolbar.

## Impact

Affects site owners.

- **Permission change:** `access toolbar` permission replaced with `access navigation` in Standard profile

## Technical details

The Standard profile and Standard recipe configuration was updated to replace Toolbar with Navigation. This includes changing the module dependency in `standard.info.yml` from `toolbar` to `navigation`, and updating the content editor role to grant the `access navigation` permission instead of `access toolbar`.

The Standard recipe explicitly declares Navigation's configuration files including block layout, settings, and the two menus used by Navigation (`system.menu.content` and `system.menu.navigation-user-links`). This ensures the recipe properly imports Navigation's default configuration.

Performance test baselines were updated to reflect Navigation's slightly different caching and query patterns compared to Toolbar. The changes show Navigation results in two fewer database queries for anonymous users and slightly different cache access patterns, with one additional discovery cache get. These minor performance differences are due to Navigation's different architecture and menu handling.

## Upgrade

This change only affects new Drupal installations using the Standard profile or Standard recipe. Existing sites are not impacted.

If you have custom installation profiles or recipes based on Standard, update your module dependencies and permissions:

```yaml
install:
  - navigation  # Replace toolbar

config:
  actions:
    user.role.content_editor:
      ensure:
        permissions:
          - 'access navigation'  # Replace 'access toolbar'
```

For recipes, explicitly import Navigation configuration files as shown in the Standard recipe if needed.
