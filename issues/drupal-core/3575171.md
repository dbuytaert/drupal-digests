## Summary

The Standard installation profile and Standard recipe now install the new Navigation module instead of the legacy Toolbar module. This change makes Navigation the default administrative interface for new Drupal installations, providing a modernized user experience for site administrators and content editors. Sites using the Standard profile will immediately benefit from Navigation's improved interface when installing Drupal.

## Impact

Affects site owners.

- **Permission change:** `access navigation` permission replaces `access toolbar` in the Content editor role

## Technical details

The Standard profile previously installed the `toolbar` module and granted the `access toolbar` permission to the Content editor role. This change replaces both with the `navigation` module and `access navigation` permission.

The Standard recipe now explicitly lists Navigation configuration files including `navigation.block_layout`, `navigation.settings`, `system.menu.content`, and `system.menu.navigation-user-links` to ensure the Navigation interface is properly configured.

Performance test baselines were updated to reflect Navigation's different caching and query behavior. Navigation reduces configuration queries by two and changes cache access patterns, resulting in fewer cache sets but slightly more cache gets. The total query count decreased from 39 to 37 for anonymous users on the front page, and menu link IDs shifted due to Navigation adding its own menu items during installation.

## Upgrade

This change only affects new installations using the Standard profile or Standard recipe. Existing sites are not impacted.

If you maintain a custom installation profile based on Standard, update your profile's `.info.yml` file to install `navigation` instead of `toolbar`, and update role permissions from `access toolbar` to `access navigation` in your role configuration files.
