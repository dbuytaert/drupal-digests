## Summary

Drupal's Standard profile and standard recipe now install the Navigation module instead of Toolbar by default. This change makes the modern Navigation interface the default administrative toolbar for new Drupal sites. The content editor role now uses the `access navigation` permission instead of `access toolbar`. This transition positions Navigation as the standard administrative experience for Drupal, replacing the legacy Toolbar module.

## Impact

Affects site owners.

- **Permission change:** The `access toolbar` permission is replaced by `access navigation` in the Standard profile's content_editor role.

## Technical details

The Standard installation profile (`standard.info.yml`) now installs the `navigation` module instead of `toolbar`. The content_editor role configuration (`user.role.content_editor.yml`) grants `access navigation` permission instead of `access toolbar`, with the module dependency updated accordingly.

The standard recipe (`core/recipes/standard/recipe.yml`) follows suit, declaring `navigation` instead of `toolbar` in its install list. The recipe explicitly imports Navigation's configuration files: `navigation.block_layout`, `navigation.settings`, `system.menu.content`, and `system.menu.navigation-user-links`.

Performance tests were updated to reflect the different behavior of Navigation compared to Toolbar. Query counts and cache access patterns shifted slightly due to Navigation's implementation. Some queries decreased (37 vs 39 in one test scenario) as Navigation optimizes certain operations differently. The diff shows both forward and reverse patches, indicating the change was applied, then temporarily reverted, then reapplied after fixing breaking tests in the 11.x branch.

Migrate Drupal UI tests required updates to entity counts, as Navigation introduces additional menu entities (10 instead of 8 menus for D6 upgrades, 7 instead of 5 for D7). Demo Umami profile tests also needed adjustments to JavaScript byte counts.
