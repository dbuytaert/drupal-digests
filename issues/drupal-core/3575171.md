## Summary

Drupal's Standard installation profile and standard recipe now use the Navigation module instead of the Toolbar module. This modernizes new Drupal sites by providing a better administrative interface experience out of the box. Sites using the Standard profile will automatically get Navigation enabled during fresh installations, though existing sites are not affected by this change. The switch aligns with Drupal's broader goal to make Navigation the default administrative toolbar for the platform.

## Impact

Affects site owners.

- **Permission change:** The `access toolbar` permission is replaced with `access navigation` in the Standard profile's content_editor role.

## Technical details

The Standard profile's module list (`standard.info.yml`) now installs `navigation` instead of `toolbar`. The content_editor role configuration was updated to grant the `access navigation` permission instead of `access toolbar` and to declare a dependency on the `navigation` module rather than `toolbar`.

The standard recipe (`recipe.yml`) received parallel changes: it installs `navigation` instead of `toolbar`, explicitly includes Navigation's configuration files (`navigation.block_layout`, `navigation.settings`, `system.menu.content`, and `system.menu.navigation-user-links`), and updates the content_editor role permissions to use `access navigation`.

Performance tests in `StandardPerformanceTest.php` were updated to reflect Navigation's different caching and query patterns. The changes show Navigation makes slightly different cache lookups than Toolbar, reflected in adjusted expected query counts and cache statistics. The test adjustments ensure the performance baseline accurately tracks the new Navigation-based setup.

## Upgrade

This change only affects fresh installations using the Standard profile or standard recipe. Existing sites continue using Toolbar unless site owners manually switch to Navigation.

If you maintain a custom installation profile or recipe based on Standard, update your module list and role permissions:

```yaml
install:
  - navigation  # instead of toolbar

config:
  actions:
    user.role.content_editor:
      grantPermissions:
        - 'access navigation'  # instead of 'access toolbar'
```

Update any performance or functional tests that assert specific cache or query counts when using the Standard profile, as Navigation has different performance characteristics than Toolbar.
