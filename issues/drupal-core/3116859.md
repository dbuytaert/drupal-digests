## Summary

This meta issue tracked the enforcement of the `Drupal.Arrays.Array.LongLineDeclaration` coding standard across all of Drupal core. The sniff requires that PHP array declarations exceeding a line-length limit be written in multi-line format rather than inline. Previously the rule was excluded from core's PHPCS configuration. After completing a series of sub-issues that reformatted violations throughout the codebase, and after the Coder project raised the line-length limit from 80 to 120 characters, the rule is now fully enabled. This is a pure code-style cleanup with no effect on runtime behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

The `Drupal.Arrays.Array.LongLineDeclaration` sniff (from the Coder project) flags PHP array declarations written inline when the total line length exceeds a configured limit. Prior to this work, `core/phpcs.xml.dist` suppressed the rule with `&lt;exclude name="Drupal.Arrays.Array.LongLineDeclaration"/&gt;`. The remediation strategy, proposed by xjm, was to split fixes into smaller patches organized by directory and test type rather than by module. Sub-issues covered Kernel tests ([#3512960](https://www.drupal.org/node/3512960), [#3513561](https://www.drupal.org/node/3513561)), FunctionalJavascript tests ([#3513119](https://www.drupal.org/node/3513119)), Functional tests ([#3513120](https://www.drupal.org/node/3513120)), Unit tests ([#3513128](https://www.drupal.org/node/3513128)), `core/lib` ([#3508608](https://www.drupal.org/node/3508608)), module tests ([#3508601](https://www.drupal.org/node/3508601)), and several additional directories ([#3549629](https://www.drupal.org/node/3549629), [#3549630](https://www.drupal.org/node/3549630), [#3549652](https://www.drupal.org/node/3549652), [#3549662](https://www.drupal.org/node/3549662), [#3549727](https://www.drupal.org/node/3549727)). A recurring concern during the process was that the default 80-character limit forced reformatting of tightly structured test-data arrays (for example, compact tabular assertions) into deeply nested multi-line form that was harder to read. alexpott documented concrete examples in the thread. jonathan1055 ran counts showing 5,805 violations at limit 80, 2,928 at limit 100, and 1,907 at limit 120. The Coder project resolved this in [#3185082](https://www.drupal.org/node/3185082) by raising the default limit to 120. With all sub-issues merged and the exclusion removed, `core/phpcs.xml.dist` now positively includes `&lt;rule ref="Drupal.Arrays.Array.LongLineDeclaration"/&gt;` (line 34 in the current `main` branch). The final enabling step was tracked in [#3549730](https://www.drupal.org/node/3549730). A small number of exceptional cases throughout core still suppress the rule inline with `// phpcs:ignore Drupal.Arrays.Array.LongLineDeclaration`.
