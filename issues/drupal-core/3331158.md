## Summary

Content editors can now expand the CKEditor 5 text editor to fill the entire browser window, making it easier to work with longer content and see more editing options at once. This restores a feature that was available in CKEditor 4 but missing when Drupal upgraded to CKEditor 5. Site administrators can add the fullscreen button to any text format toolbar through the standard CKEditor configuration interface.

## Impact

No upgrade or configuration changes required.

## Technical details

CKEditor 5 version 45.0.0 introduced a fullscreen editing plugin that was not yet exposed in Drupal core. This change registers the CKEditor 5 fullscreen plugin as an available Drupal plugin by adding the necessary configuration and assets.

The implementation adds a `ckeditor5_fullscreen` plugin definition in `ckeditor5.ckeditor5.yml` that references the `fullscreen.Fullscreen` CKEditor 5 plugin. A new library `core/ckeditor5.fullscreen` in `core.libraries.yml` loads the JavaScript from `assets/vendor/ckeditor5/fullscreen/fullscreen.js`. An admin library `ckeditor5/internal.admin.fullscreen` provides the CSS for the toolbar button icon.

The plugin registers a `fullscreen` toolbar item and sets `elements: false` because it affects the editing interface but does not introduce new HTML elements to the content. The fullscreen button icon is provided as an SVG file at `icons/fullscreen-enter.svg`, styled via `css/fullscreen.admin.css`.
