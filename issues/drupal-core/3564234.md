## Summary

The "Manage display" tab in Field UI now opens a display-builder agnostic overview page instead of dropping administrators directly into the default view mode's field formatter form. The overview lists all view modes for a bundle in two tables: "Enabled view modes" and "Disabled view modes". Each enabled mode has a "Manage" link to its configuration form and a "Disable" link; disabled modes have an "Enable" link. The default view mode is always shown as enabled and cannot be disabled. A "Manage view modes" action link leads to the global view modes administration page. The individual display mode forms are unchanged; only the navigation entry point has moved.

## Impact

Affects module maintainers and front-end developers.

- **Theme change:** New `drupal.field_ui_display_overview` library and `field_ui_display_overview_table.css` stylesheet added to the 'Manage display' overview page.
- **New API:** New `entity.entity_view_display_overview.{entity_type_id}` route and `EntityViewDisplayOverviewController` / `EntityViewDisplayStatusController` for the display mode overview and toggle actions.

## Technical details

Three new routes are registered per entity type in `RouteSubscriber::alterRoutes()`. The overview route `entity.entity_view_display_overview.{entity_type_id}` is placed at `$path/display` and handled by the new `EntityViewDisplayOverviewController::overview()`. The existing default display route `entity.entity_view_display.{entity_type_id}.default` was moved from `$path/display` to `$path/display/default`. A new CSRF-protected toggle route `entity.entity_view_display.{entity_type_id}.toggle_status` at `$path/display/{view_mode_name}/{operation}` is handled by `EntityViewDisplayStatusController::toggleStatus()`, which enables or disables the named view mode and redirects back to the overview. The default view mode is hardcoded as always enabled; attempting to disable it throws an `\InvalidArgumentException`.

The "Custom display settings" collapsible section (the checkboxes for enabling view modes) was removed from `EntityDisplayFormBase::form()` and its corresponding `submitForm()` logic. That same checkbox UI and submit logic was moved into `EntityFormDisplayEditForm`, so form modes still use the checkbox pattern on the default form display. View mode enable/disable is now exclusively managed through the overview page.

Local task derivatives in `FieldUiLocalTask` were updated: `display_overview_{entity_type_id}` now resolves to the new overview route. A new "Overview" secondary tab `field_display_overview_{entity_type_id}` was added, and the "Manage display" operation link emitted by `entityOperation()` in entity listing pages also now targets the overview route. The new `drupal.field_ui_display_overview` library loads `css/field_ui_display_overview_table.css` for the overview layout.

## Upgrade

Any code or test that navigated to `$bundle_path/display` and expected the default view mode form must be updated to `$bundle_path/display/default`.

```php
// Before:
$url = Url::fromRoute("entity.entity_view_display.$entity_type_id.default", $params);

// Still works; no change needed for routes that already use the named route.
// Only bare path strings or links hardcoded to /display are affected.

// Before (hardcoded path):
$this->drupalGet('admin/structure/types/manage/article/display');

// After:
$this->drupalGet('admin/structure/types/manage/article/display/default');
```

Code that previously submitted the `display_modes_custom[{mode}]` checkbox on the default view display form to enable a view mode must switch to either clicking the "Enable" link on the overview page or directly creating/saving an `entity_view_display` config entity with `status: TRUE`:

```php
$storage = \Drupal::entityTypeManager()->getStorage('entity_view_display');
$display = $storage->create([
  'targetEntityType' => 'node',
  'bundle' => 'article',
  'mode' => 'full',
  'status' => TRUE,
]);
$display->save();
```
