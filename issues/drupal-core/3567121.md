## Summary

PHPUnit 12.5 introduced a behavioral distinction between mock objects (which verify that specific methods are called) and stubs (which only provide return values). Drupal core unit tests across 13 modules were still using `createMock()` and `$this->any()` for test doubles that never actually assert call expectations, which now triggers PHPUnit notices and deprecation warnings. This is a test-only internal cleanup with no effect on site behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced a notice when a `MockObject` has no expectations configured, deprecated `$this->any()` (semantically equivalent to "no expectation"), and deprecated `with()` on stubs. Three kinds of changes are applied across 13 test files in this first batch:

1. **`createMock()` → `createStub()`**: Any test double that only needs to return values without asserting call counts is converted. `createStub()` automatically disables the original constructor, so several `getMockBuilder(...)->disableOriginalConstructor()->getMock()` patterns are also collapsed.

2. **`expects($this->any())` removed or replaced**: Where a stub is sufficient, `->expects($this->any())->method(...)` is reduced to `->method(...)`. Where a call must actually be verified, `$this->any()` is replaced with a precise expectation such as `$this->once()` or `$this->atLeastOnce()` while keeping the object as a mock (e.g., `storage->expects($this->once())->method('loadMultiple')` in `ResponsiveImageStyleConfigEntityUnitTest`, and `connection->expects($this->atLeastOnce())->method('getServerVersion')` in the mysql `ConnectionTest`).

3. **`with()` removed from stubs**: PHPUnit 12.5 also deprecated argument constraints on stubs. In `ResponsiveImageStyleConfigEntityUnitTest`, `with('responsive_image_style')`, `with('image_style')`, and `with('test_group')` constraints are removed from stub method configurations.

Additional smaller changes: PHPDoc `@var` type annotations are updated from `MockObject` to `Stub` in several files; `BreakpointTest` replaces `createMock(TranslationInterface::class)` with `$this->getStringTranslationStub()`, the existing `UnitTestCase` helper.

Files changed span these modules: `announcements_feed` (`AnnounceFetcherUnitTest`), `big_pipe` (`FiberPlaceholderTest`), `breakpoint` (`BreakpointTest`), `content_moderation` (`ContentModerationRouteSubscriberTest`), `jsonapi` (`LinkTest`), `media_library` (`MediaLibrarySelectFormTest`), `menu_link_content` (`MenuLinkContentEntityAccessTest`), `mysql` (`ConnectionTest`), `navigation` (`NavigationMenuLinkTreeManipulatorsTest`), `responsive_image` (`ResponsiveImageStyleConfigEntityUnitTest`), `shortcut` (`ShortcutLocalTasksTest`), `sqlite` (`ConnectionTest`), and `workspaces` (`WorkspacePublisherTest`). The commit was targeted only to `main` because it did not cherry-pick cleanly to `11.x`.
