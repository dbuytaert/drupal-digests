## Summary

Drupal core updated its PHPUnit dependency to address a security vulnerability. This is a development-only dependency used for running tests. The vulnerability exists in PHPUnit itself and does not affect production sites. Site owners running custom tests should update their development dependencies to receive the security fix. The update brings PHPUnit to version 11.5.50 on Drupal 11.x, 11.3.x, and 10.6.x branches (which use PHPUnit 9.6.34).

- **Security:** PHPUnit security vulnerability CVE-2026-24765 addressed

## Technical details

This issue updates the `phpunit/phpunit` constraint in `composer.json` from `^11.5.42` to `^11.5.50` on main, 11.x, and 11.3.x branches, and from `^9.6.13` to `^9.6.34` on the 10.6.x branch. These versions contain fixes for CVE-2026-24765, a security vulnerability in PHPUnit.

Running `composer audit` on affected branches would have flagged this vulnerability. The update was performed by running `composer update -W phpunit/phpunit` to allow updating dependencies with the `-W` flag, then manually updating the constraint in both the root `composer.json` and `core/tests/PHPStan/composer.json` files, followed by `composer update --lock` to regenerate lock files.

The update also pulls in updated versions of related packages including `phpunit/php-code-coverage` (11.0.12), `phpunit/php-file-iterator` (5.1.1), `sebastian/comparator` (6.3.3), and `nikic/php-parser` (5.7.0) as required by the dependency tree. These are all development-only dependencies with no impact on production code.
