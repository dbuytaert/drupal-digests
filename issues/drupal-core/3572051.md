## Summary

The hook `hook_icon_pack_alter()` was previously undocumented, making it difficult for developers to discover or use. This hook allows modules to modify icon pack definitions that are part of Drupal's core Icon API. With proper documentation now in place, developers can more easily customize icon packs provided by core or contributed modules. The documentation includes a practical example showing how to change the label of the navigation icon pack.

## Impact

No upgrade or configuration changes required.

## Technical details

The Icon API in Drupal core includes an `IconPackManager` plugin manager that discovers icon pack definitions from YAML files. Like other plugin managers, it supports an alter hook through `alterInfo()` to allow modifications during discovery. However, `hook_icon_pack_alter()` was never documented, making it effectively invisible to developers.

The new documentation was added to `core/lib/Drupal/Core/Render/theme.api.php` alongside other theme and rendering-related hooks. The hook receives an array of icon pack definitions keyed by pack ID and allows modules to modify any aspect of these definitions. The documentation includes a simple example that changes the label of the navigation icon pack.

Discussion during review clarified that `theme.api.php` was the appropriate location rather than `core.api.php`, since it contains other theme-related hooks regardless of whether they can only be invoked by modules or themes. A follow-up issue was opened to explore allowing themes to invoke this alter hook as well.
