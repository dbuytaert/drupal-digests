## Summary

Two locale batch operation callbacks in `locale.bulk.inc` were renamed to better reflect the `LocaleConfigManager` API methods they wrap. `locale_config_batch_set_config_langcodes()` is now `locale_config_batch_update_default_config_langcodes()`, and `locale_config_batch_refresh_name()` is now `locale_config_batch_update_config_translations()`. Both old names are deprecated in Drupal 11.1.0 and will be removed in Drupal 12.0.0. This change only affects code that explicitly references these batch callback functions by name, which is unusual outside of core itself.

## Impact

Affects module maintainers.

- **Deprecation:** `locale_config_batch_set_config_langcodes()` and `locale_config_batch_refresh_name()` deprecated in 11.1.0; replaced by `locale_config_batch_update_default_config_langcodes()` and `locale_config_batch_update_config_translations()`; removed in 12.0.0

## Technical details

The locale module defines batch operation callbacks in `core/modules/locale/locale.bulk.inc` that wrap `LocaleConfigManager` service methods. Before this change the callback names (`locale_config_batch_set_config_langcodes`, `locale_config_batch_refresh_name`) did not correspond to the service methods they called (`updateDefaultConfigLangcodes`, `updateConfigTranslations`), making the code harder to follow.

The initial approach was a straight rename, but catch pointed out (comment #5) that because these are public procedural functions without a leading underscore they require a deprecation period rather than a silent rename. The final change therefore introduces two new canonical functions with `: void` return types: `locale_config_batch_update_default_config_langcodes(&$context)` and `locale_config_batch_update_config_translations(array $names, array $langcodes, &$context)`. The old names are retained as thin wrappers that fire `@trigger_error` with a deprecation notice pointing to change record https://www.drupal.org/node/3475054, then delegate to the new functions. Both are scheduled `deprecated in drupal:11.1.0 and is removed from drupal:12.0.0`.

`locale_config_batch_build()` was updated to register the new callback names via `BatchBuilder::addOperation()`, and the `@see` reference in its docblock was updated accordingly. A follow-up commit removed two now-redundant PHPStan baseline suppressions from `core/.phpstan-baseline.php` for `missingType.return` on the old functions, since the new functions carry explicit `: void` return type hints. A new unit test class `LocaleBulkDeprecationTest` was added under `core/modules/locale/tests/src/Unit/` to assert that calling each old function triggers the expected deprecation message. That test class was removed in Drupal 12.x once the deprecated functions themselves were removed.

## Upgrade

If your custom code or contrib module registers `locale_config_batch_set_config_langcodes` or `locale_config_batch_refresh_name` as batch operation callbacks, update to the new names:

```php
// Before
$batch_builder->addOperation('locale_config_batch_set_config_langcodes');
$batch_builder->addOperation('locale_config_batch_refresh_name', [$names, $langcodes]);

// After
$batch_builder->addOperation('locale_config_batch_update_default_config_langcodes');
$batch_builder->addOperation('locale_config_batch_update_config_translations', [$names, $langcodes]);
```
