## Summary

This is a pure test-code change with no effect on Drupal's runtime behavior. It updates unit tests in the Database and Routing subsystems to be compatible with PHPUnit 12.5.0+, which deprecates `$this->any()` and emits notices when a mock object is created without configuring any expectations. Site owners running Drupal's test suite against a future PHPUnit 12.5 release will no longer see those notices from these test classes.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces two breaking changes for test code: `$this->any()` is deprecated (it was equivalent to having no expectation, which means the object should be a stub), and mock objects created without configuring expectations emit a runtime notice directing authors to use stubs instead. The fix covers 10 test classes across `core/tests/Drupal/Tests/Core/Database/` and `core/tests/Drupal/Tests/Core/Routing/` (plus `MetadataBubblingUrlGeneratorTest` in the Render namespace, which extends `UrlGeneratorTest`).

The mechanical changes are: `createMock()` and `getMockBuilder()->disableOriginalConstructor()->getMock()` calls replaced with `createStub()` wherever no expectations are configured; `->expects($this->any())` chains either dropped entirely or replaced with plain `->` when `willReturn()` is still needed; and string class names replaced with `ClassName::class` constants along with proper `use` imports.

A structural challenge arose in `RouteBuilderTest` and `UrlGeneratorTest`: some properties are set to stubs in `setUp()` but only a subset of test methods need them to carry expectations. Because a mock created in `setUp()` would fail if a test method does not satisfy its expectations, new helper methods were introduced (`setUpMockMatcherDumper()`, `setUpMockCheckProvider()`, and `setUpMockRouteProcessorManager()`) that reinitialize the relevant property as a full mock via `ReflectionProperty::setValue()` immediately before each test that needs it. This approach was chosen over splitting the classes into two, as noted by catch in the commit comment.

The corresponding PHPStan baseline suppressions in `core/.phpstan-baseline.php` for the `any()` calls in `DriverModuleMissingDependenciesTest`, `MetadataBubblingUrlGeneratorTest`, `RedirectDestinationTest`, `RouteBuilderTest`, and `RoutePreloaderTest` were also removed.
