## Summary

This is a purely internal test maintenance change with no effect on site behaviour. Ten unit test classes across the Lock, Logger, Mail, Path, PathPreprocessor, and PreWarm test directories were updated to stay compatible with PHPUnit 12.5.0, which emits notices when mock objects are created without any configured expectations.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced a notice whenever a mock object is created via `createMock()` or `getMockBuilder()->getMock()` but no expectations are ever configured on it: the recommendation is to use a stub (`createStub()`) instead, because stubs carry no expectation-checking overhead and more accurately communicate the test intent. Additionally, `$this->any()` and `with()` on stubs were deprecated as meaningless.

The commit (2f97f97a) rewrites the affected test files in `core/tests/Drupal/Tests/Core/Lock/`, `…/Logger/`, `…/Mail/`, `…/Path/`, `…/PathPreprocessor/`, and `…/PreWarm/`. The primary mechanical change is replacing `createMock()` with `createStub()` for every dependency that is only provided as a collaborator without expectations. Where a mock genuinely has expectations (e.g. `expects($this->once())` calls), it remains a `MockObject`. For `LockBackendAbstract`, which is abstract and therefore cannot be stubbed directly, a concrete `StubLockBackendAbstract` helper class is introduced in the test namespace, allowing `getMockBuilder(StubLockBackendAbstract::class)->onlyMethods([...])->getMock()` to be used in `LockBackendAbstractTest` with only the one method that actually needs an expectation mocked. `PathMatcherTest` and `PathValidatorTest` similarly switch their router, account, and path-processor dependencies to `createStub()`. This issue covers the same fix pattern introduced for the base `UnitTestCase` class in the sibling issue [#3578904](https://www.drupal.org/node/3578904).
