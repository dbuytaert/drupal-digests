## Summary

This is a purely internal test-code cleanup with no effect on production behaviour. It prepares Drupal's unit-test suite for PHPUnit 12.5, which deprecates mock objects that never configure expectations and deprecates the `$this->any()` matcher. Ten test classes across the Session, State, TempStore, Template, Recipe, Transliteration, and TypedData test directories are updated.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduces three deprecations that affect existing Drupal tests: (1) a mock object that never calls `expects()` should be a stub instead, (2) `$this->any()` is deprecated because it is equivalent to configuring no expectation, and (3) `with()` on stubs is deprecated as pointless.

The fix applies a consistent split-setup pattern across the affected files. A shared `setUp()` (or `setUpState()` / `setUpMockWrappedSessionHandler()` equivalents) now creates all dependencies as stubs via `createStub()`. Individual test methods that actually verify call counts or argument matching call a small `setUpMock*()` helper at their start, which reinitializes only the relevant dependency as a full mock via `createMock()`. For `TwigExtensionTest`, where the system-under-test is constructed in `setUp()` and holds its own references, the helpers use `\ReflectionProperty` to swap in the mock after construction.

`SessionConfigurationTest` goes further: the `createSessionConfiguration()` helper was using `getMockBuilder()` to override `drupalValidTestUa()` on the concrete `SessionConfiguration` class. The fix removes this partial mock entirely and instantiates `new SessionConfiguration(...)` directly, since the mocked method was not actually exercised in the surviving test cases.

`StateTest` had an additional complexity because several test methods use `#[Depends]` chains, so the old shared `setUp()` was split into `setUpState()` (stub key-value store, for tests that only read results), `setUpStateWithMockStorage()` (mock store, for tests that assert write calls), and the shared wiring method `doSetUpState()`.

Four `$ignoreErrors[]` suppression entries in `core/.phpstan-baseline.php` that covered the now-fixed `any()` deprecation warnings in `PrivateTempStoreTest`, `TwigExtensionTest`, `PhpTransliterationTest`, and `RecursiveContextualValidatorTest` are removed.
