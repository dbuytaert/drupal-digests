## Summary

This is a purely internal CI pipeline change with no effect on site functionality or site owners. It restructures how Drupal's GitLab CI runs tests, and fixes a variable scoping problem that caused performance test jobs to fail after that restructuring.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue restructured `.gitlab-ci.yml` to run all test jobs in the top-level pipeline instead of delegating to a child pipeline (`.gitlab-ci/pipeline.yml`). Previously, unit tests appeared in the parent pipeline and other tests in a separate child pipeline, which was awkward to navigate in the GitLab UI. The new approach runs everything in the top-level pipeline and uses a recursive call to `.gitlab-ci.yml` for secondary runs (repeat and performance tests) that need different parameters.

The specific diff provided is a follow-up fix committed after the main restructuring (commit ce25fce2). After the restructure, performance test jobs started failing because `MINK_DRIVER_ARGS_WEBDRIVER` (the Chrome WebDriver configuration string for Selenium) had previously been declared only in the `PHPUnit-FunctionalJavascript` job's `variables:` block. Once performance tests were reorganized into the same pipeline structure they also needed that variable, but it was out of scope. The fix moves `MINK_DRIVER_ARGS_WEBDRIVER` from the job-level `variables:` of the FunctionalJavascript job up to the global `variables:` block at the top of `.gitlab-ci.yml`, making it available to all jobs. A debug `echo $MINK_DRIVER_ARGS_WEBDRIVER` line that had been in the FunctionalJavascript `after_script` was also removed as part of the cleanup.
