## Summary

Two procedural functions in `system.module` have been deprecated as part of an ongoing effort to reduce the system module's footprint. `system_sort_themes()` is deprecated with no replacement; its only core caller (`SystemController::themesPage()`) now uses an inlined static closure. `system_check_directory()`, a form `after_build` callback, is also deprecated with no replacement; its sole core usage was already removed in [#3571593](https://www.drupal.org/node/3571593). Both are scheduled for removal in Drupal 13.0.0. Contrib modules using either function have until Drupal 13 to adapt.

## Impact

Affects module maintainers.

- **Deprecation:** `system_sort_themes()` and `system_check_directory()` in `system.module` are deprecated in Drupal 11.4.0 with no replacement and will be removed in Drupal 13.0.0.

## Technical details

`system_sort_themes($a, $b)` sorted the installed-themes list on the themes administration page, placing the default theme first then sorting the rest case-insensitively by name. It was only ever called in one core location, `SystemController::themesPage()`. The fix inlines the identical logic as a `static function` closure passed directly to `uasort()`, then adds the standard `@trigger_error` deprecation notice to the original function body. `system_check_directory($form_element, $form_state)` was a form `after_build` callback that validated a configured directory path and set a form error when the path was invalid. Its last core call (from the translation settings form) was removed in [#3571593](https://www.drupal.org/node/3571593). The function body is preserved verbatim; only the `@trigger_error` call and PHPDoc `@deprecated` tag are added. Both functions reference change record https://www.drupal.org/node/3566774 and are marked deprecated in drupal:11.4.0, removed from drupal:13.0.0 with no replacement. This task is part of the broader system module procedural-function cleanup; [#3015812](https://www.drupal.org/node/3015812) handles the related `system_region_list()` and `system_default_region()` deprecations.

## Upgrade

Contrib modules calling `system_sort_themes()` should inline the sorting logic directly:

```php
// Before
uasort($themes, 'system_sort_themes');

// After
uasort($themes, static function ($a, $b) {
  if ($a->is_default) {
    return -1;
  }
  if ($b->is_default) {
    return 1;
  }
  return strcasecmp($a->info['name'], $b->info['name']);
});
```

Contrib modules using `system_check_directory()` as a form `after_build` callback should remove the callback and reimplement any directory-validation logic they need directly in their form or form submission handler. There is no drop-in replacement.
