## Summary

Two medium-severity security vulnerabilities in the `guzzlehttp/psr7` library (CVE-2026-48998, Host Confusion via Authority Reinterpretation; and CVE-2026-49214, CRLF Injection via URI Host Component) were publicly registered on June 11 2026 against versions below 2.10.2. Drupal core's `drupal/core-recommended` package was pinning `~2.8.0`, which prevented `composer update` from pulling in a safe release and caused `composer audit` to fail on any project using that metapackage. This change raises the minimum to `^2.10.2` (resolved to 2.10.4 in the lock file) across the affected branches, so sites can install a clean dependency tree without workarounds.

## Impact

Affects site owners.

- **Security:** Two medium-severity CVEs in `guzzlehttp/psr7` (<2.10.2): CVE-2026-48998 (Host Confusion via Authority Reinterpretation) and CVE-2026-49214 (CRLF Injection via URI Host Component).
- **Requirements:** Minimum `guzzlehttp/psr7` raised from `~2.8.0` to `~2.10.4` in `drupal/core-recommended`.

## Technical details

The constraint for `guzzlehttp/psr7` was updated in three places within the single commit on `main` (with parallel commits on `11.3.x` and `10.6.x`): `composer.json` at the repo root changed from `^2.8.0` to `^2.10.2`, `core/composer.json` (used by `drupal/core-recommended`) changed its pinned tilde constraint from `~2.8.0` to `~2.10.4`, and `composer.lock` was regenerated to record `guzzlehttp/psr7` at `2.10.4`. The `11.4.x` and `11.x` branches were already on `2.10.4` and needed no changes. The contributor noted that bumping to 2.11.0 introduced deprecations, so 2.10.4 was chosen as the target instead. The two CVEs affect URI parsing: GHSA-hq7v-mx3g-29hw covers CRLF injection via the URI host component, and GHSA-34xg-wgjx-8xph covers host confusion via authority reinterpretation.

## Upgrade

Sites running `drupal/recommended-project` on affected branches (main, 11.3.x, 10.6.x) should update to the patched Drupal core release and run `composer update`. Before the patched release is available, the workaround is:

```bash
composer require "guzzlehttp/psr7":"2.10.4 as 2.8.1"
```
