## Summary

Drupal 11 upgrades its test infrastructure from PHPUnit 9 to PHPUnit 10 and drops the `symfony/phpunit-bridge` dependency entirely. Because Symfony signalled it would not port the bridge to PHPUnit 10, Drupal now ships its own replacements for every feature the bridge provided: deprecation capturing, `@group legacy` suppression, `expectDeprecation()` assertions, and pattern-based ignore files. Sites themselves are not affected, but any contributed or custom module that ships tests will need minor updates to match the PHPUnit 10 API. Tests that run outside Drupal's own test runner (for example, directly from the PHPUnit CLI) need an extra flag for non-unit tests.

## Impact

Affects module maintainers and site owners.

- **Requirements:** PHPUnit upgraded from 9.x to 10.x; `symfony/phpunit-bridge` dependency removed from Drupal 11.
- **API change:** `expectError*()` and `expectWarning*()` methods re-implemented in Drupal's shim (PHPUnit 10 removed them natively); `@group legacy` annotation still works via backward-compat shim but `#[IgnoreDeprecations]` is the new form; process isolation for non-unit tests is now enforced via base class constructors rather than removed `$runTestInSeparateProcess` properties, so running kernel/functional tests directly from the PHPUnit CLI requires `--process-isolation`.
- **Deprecation:** `DrupalComponentTestListenerTrait` removed; replaced by a PHPStan rule (see #3400366).

## Technical details

The diff confirms `phpunit/phpunit` bumped from `^9.6.13` to `^10.5.19` and `symfony/phpunit-bridge` removed from `composer.json`, `composer.lock`, and the vendor `Config` exclusion list. The entire sebastian/* dependency tree was updated to its next major series (e.g. `sebastian/diff` 4→5, `sebastian/global-state` 5→6).

The bridge's deprecation-tracking features are replaced by a custom error handler shim layered on top of PHPUnit's own handler. Each test swaps in the shim at `setUp()` and restores PHPUnit's handler at `tearDown()`. The shim re-implements `expectDeprecation()` for asserting a specific deprecation was triggered, re-implements the removed `expectError*()` and `expectWarning*()` families, and reads `.deprecation-ignore.txt` pattern files for selective suppression. The `@group legacy` docblock annotation is mapped to the same effect as PHPUnit 10's native `#[IgnoreDeprecations]` attribute, preserving backward compatibility; tests can be migrated incrementally.

PHPUnit 10 removed the `$runTestInSeparateProcess` and `$preserveGlobalState` properties that `KernelTestBase` and `BrowserTestBase` previously used to enforce per-test process isolation. Because PHPUnit 10 only reads isolation attributes from the class they are written on (not child classes), the solution is to call `setRunTestInSeparateProcess()` from the constructors of those base classes. Unit tests are excluded for performance. When running non-unit tests directly with `vendor/bin/phpunit` (outside `run-tests.sh`), `--process-isolation` must be passed manually. Follow-up issues [#3445240](https://www.drupal.org/node/3445240) and [#3417650](https://www.drupal.org/node/3417650) track adding `#[RunTestsInSeparateProcesses]` per class and replacing `expectError*()` / `expectWarning*()` calls across the test suite respectively.

`DrupalComponentTestListenerTrait` and the PHPUnit listener infrastructure were removed. A PHPStan rule (landed in [#3400366](https://www.drupal.org/node/3400366)) replaced the trait's responsibility of verifying that component tests do not extend Drupal core test base classes. The `phpunit.xml.dist` configuration file was updated to the PHPUnit 10 schema, test suite definitions were expanded to cover theme and profile test paths, and `--display-deprecations` was added to the runner command so PHPUnit natively surfaces deprecation notices in output.

## Upgrade

Replace `@group legacy` docblock annotations with PHPUnit 10's native attribute on test methods that intentionally exercise deprecated code. The annotation continues to work via Drupal's shim, but migration to the attribute is recommended:

```php
// Before (PHPUnit 9 / symfony/phpunit-bridge)
/**
 * @group legacy
 */
public function testDeprecatedBehaviour(): void { }

// After (PHPUnit 10 native)
#[\PHPUnit\Framework\Attributes\IgnoreDeprecations]
public function testDeprecatedBehaviour(): void { }
```

Replace removed `expectWarning*()` and `expectError*()` calls. PHPUnit 10 no longer converts PHP warnings to exceptions, so code that previously used PHPUnit warnings to interrupt control flow must be restructured:

```php
// Before
$this->expectError();
$this->expectErrorMessage('Some message');
someCodeThatTriggersAnError();

// After: assert the underlying exception directly
$this->expectException(\AssertionError::class);
someCodeThatTriggersAnError();
```

When running Kernel or Functional tests directly from the PHPUnit CLI (outside Drupal's own test runner), pass `--process-isolation` to replicate per-test isolation:

```bash
./vendor/bin/phpunit --process-isolation -c web/core/phpunit.xml.dist \
  web/core/modules/mymodule/tests/src/Kernel/MyKernelTest.php
```

### Collaboration

mondrake opened the issue and built the entire replacement implementation incrementally: prototyping the bridge removal, building the error handler shim, implementing `expectDeprecation()` and `expectError()`, and finally resolving the TestKernel interaction in functional tests that had blocked the error handler from being correctly restored. longwave identified that the removal of `$runTestInSeparateProcess` and `$preserveGlobalState` in PHPUnit 10 was the root cause of widespread kernel test failures, and traced a subtle `SecurityAdvisoryTest` failure to a PHPUnit 9 warning-to-exception side effect that Guzzle middleware had been accidentally relying on. alexpott pointed out that controlling process isolation from the test runner script would break PHStorm users, which prompted longwave to discover that `setRunTestInSeparateProcess()` could be called from the base class constructors instead, producing a cleaner solution than either annotation-per-class or runner-level flags.
