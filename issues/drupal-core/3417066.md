## Summary

Drupal 11 upgrades its test suite from PHPUnit 9 to PHPUnit 10 and drops the `symfony/phpunit-bridge` dependency entirely. Drupal now ships its own replacements for the bridge's deprecation-tracking and error-handling features. Sites running Drupal 11 are not directly affected, but any contributed or custom module tests will need minor updates to match PHPUnit 10 APIs and the new process isolation behavior.

## Impact

Affects module maintainers and site owners.

- **Requirements:** PHPUnit upgraded from 9.x to 10.x; `symfony/phpunit-bridge` dependency removed from Drupal 11.
- **API change:** `expectError*()` and `expectWarning*()` methods removed from test base classes; `@group legacy` annotation replaced by `#[IgnoreDeprecations]` attribute (BC support maintained); process isolation for non-unit tests no longer set by base class properties and requires `--process-isolation` CLI flag or `#[RunTestsInSeparateProcesses]` per-class attribute when running outside Drupal's test runner.
- **Deprecation:** `DrupalComponentTestListenerTrait` removed; replaced by a PHPStan rule.

## Technical details

The `symfony/phpunit-bridge` package provided Drupal's deprecation capture, `@group legacy` suppression, `expectDeprecation()` assertions, and selective ignore via `.deprecation-ignore.txt`. Symfony signalled they would not port the bridge to PHPUnit 10 and would likely drop the deprecation-tracking features even in a PHPUnit 11 port (see the upstream issue linked in the issue body). This made a custom replacement unavoidable.

**Custom error handler shim.** The core of the replacement is an error handler shim layered on top of PHPUnit's own handler. Each Drupal test swaps in the shim at `setUp()` and restores PHPUnit's handler at `tearDown()`. The shim intercepts PHP `E_*` error events that Drupal needs to handle specially and delegates everything else to PHPUnit. This approach re-implements: `expectDeprecation()` (assert a specific deprecation was triggered), `expectError()` and `expectWarning()` (fill the gap from methods removed in PHPUnit 10), and selective deprecation suppression from `.deprecation-ignore.txt` pattern files.

**`@group legacy` and `#[IgnoreDeprecations]`.** PHPUnit 10 ships a native `#[IgnoreDeprecations]` attribute that silences all deprecation notices for a test method. Drupal's shim maps the legacy `@group legacy` docblock annotation to the same behaviour for backward compatibility; tests can be migrated incrementally.

**Process isolation.** PHPUnit 9's `$runTestInSeparateProcess = true` and `$preserveGlobalState = false` properties, which `KernelTestBase` and `BrowserTestBase` set to enforce per-test process isolation, are gone in PHPUnit 10. Annotations and attributes for this only apply to the class they are written on, not to child classes. The solution found during the thread (comment #89/90) is to call `setRunTestInSeparateProcess()` from the constructors of the base test classes. Because Unit tests must not run in isolation for performance, only the non-unit base classes enable this. Follow-up issue [#3445240](https://www.drupal.org/node/3445240) tracked adding the `#[RunTestsInSeparateProcesses]` attribute to all individual kernel and functional test classes via a Rector script.

**`PhpUnitTestRunner` and subprocess spawning.** The internal runner was updated to use Symfony Process instead of `exec()` to spawn PHPUnit subprocesses, adding timeout support (spun out to [#3417805](https://www.drupal.org/node/3417805)). The `--display-deprecations` flag was added to the runner command so PHPUnit natively surfaces deprecation notices in output.

**Listener removal.** The `DrupalComponentTestListenerTrait` and the PHPUnit listener infrastructure were removed. A PHPStan rule replaced the trait's responsibility of checking that component tests do not extend Drupal core test base classes ([#3400366](https://www.drupal.org/node/3400366)).

**`phpunit.xml.dist` updates.** The configuration file was updated for PHPUnit 10 format changes. Test suite definitions were expanded to include paths for themes and profiles in addition to modules. `ignoreSuppressionOfDeprecations="true"` was set on the `<source>` element.

**`TestHttpClientMiddleware` side-effect fix.** In PHPUnit 9, `file_put_contents()` failures triggered warnings that PHPUnit converted to exceptions; those exceptions were inadvertently caught by Guzzle middleware and suppressed HTTP requests. PHPUnit 10 does not convert warnings to exceptions, so requests that were previously never made now succeed. This caused unexpected behaviour in `SecurityAdvisoryTest` and `StandardPerformanceTest`. The fix was addressed within the issue (tracked separately in a related issue) rather than being a change in this issue itself.

The `@group legacy` annotation replacement meta is tracked in [#3417650](https://www.drupal.org/node/3417650). Running PHPUnit against non-unit test files from the CLI requires the `--process-isolation` flag when not using Drupal's own test runner.

## Upgrade

Replace `@group legacy` docblock annotations with PHPUnit 10's native attribute on test methods that intentionally exercise deprecated code:

```php
// Before (PHPUnit 9 / symfony/phpunit-bridge)
/**
 * @group legacy
 */
public function testDeprecatedBehaviour(): void {
    // ...
}

// After (PHPUnit 10 native)
#[\PHPUnit\Framework\Attributes\IgnoreDeprecations]
public function testDeprecatedBehaviour(): void {
    // ...
}
```

Replace removed `expectWarning*()` and `expectError*()` calls. These methods are gone in PHPUnit 10; refactor the runtime code to remove `trigger_error()` calls and assert the actual exception instead:

```php
// Before
$this->expectError();
$this->expectErrorMessage('Some message');
someCodeThatTriggersAnError();

// After: assert the underlying exception directly
$this->expectException(\AssertionError::class);
someCodeThatTriggersAnError();
```

When running Kernel or Functional tests directly from the PHPUnit CLI (outside Drupal's own test runner), pass `--process-isolation` to replicate the per-test isolation that base classes previously enforced via properties:

```bash
./vendor/bin/phpunit --process-isolation -c web/core/phpunit.xml.dist \
  web/core/modules/mymodule/tests/src/Kernel/MyKernelTest.php
```
