## Summary

Unit tests in the Node module have been updated to use PHPUnit test stubs instead of expectation-less mock objects. This is a purely internal test code change with no effect on production behavior. It prepares the Node module's test suite for PHPUnit 12.5+, which emits notices when mock objects are created without configuring expectations, and deprecates `$this->any()` and `with()` on stubs.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced a notice when a mock object created via `createMock()` or `getMockBuilder()->getMock()` has no expectations configured, because such objects are semantically stubs, not mocks. It also deprecated `$this->any()` (which was equivalent to no expectation) and `with()` argument constraints on stubs.

This issue is part of the broader effort tracked in [#3561671](https://www.drupal.org/node/3561671) to clean up all such patterns across Drupal core.

Five test files under `core/modules/node/tests/src/Unit/` were changed:

**`NodeOperationAccessTest.php`**: `CacheContextsManager` was converted from a three-line `getMockBuilder()` chain to `createStub()`. `AccountInterface`, `EntityTypeInterface`, `NodeGrantDatabaseStorageInterface`, `LanguageInterface`, `NodeStorageInterface`, `EntityTypeManagerInterface`, and `ModuleHandlerInterface` were also converted from `createMock()` to `createStub()`, and their `expects($this->any())` calls were dropped. The `NodeInterface` object (`$node`) intentionally remains a mock because it retains a real expectation: `$node->expects($this->atLeastOnce())->method('isDefaultRevision')`. Previously that expectation was wrapped in an `if (isset($isDefaultRevision))` guard; the guard was removed so the expectation always applies. The `->with($nid)` argument constraint on `NodeStorageInterface::load()` and `->with('node')` on `EntityTypeManagerInterface::getStorage()` were removed because `with()` is now deprecated for stubs.

**`NodeBulkFormTest.php`**: `ActionConfigEntityInterface`, `EntityStorageInterface`, `EntityRepositoryInterface`, `LanguageManagerInterface`, `MessengerInterface`, `ResettableStackedRouteMatchInterface`, `ViewExecutable`, and `DisplayPluginBase` were converted to stubs. String-based class name arguments were replaced with `::class` constants, and corresponding `use` statements were added. `ViewsData` and the `ViewEntityInterface` storage object could not be converted to stubs because they retain real `expects($this->once())` calls (tightened from `any()`).

**`RevisionLinkDeleteTest.php`, `RevisionLinkRevertTest.php`, `RevisionLinkTest.php`**: Identical pattern in each: the `TranslationInterface` service registered in the container changed from `createMock()` to `createStub()`, and `AccessManagerInterface`, `EntityTypeManagerInterface`, `EntityRepositoryInterface`, `LanguageManagerInterface`, `ViewExecutable`, and `DisplayPluginBase` arguments passed to the field constructors changed from `createMock()` to `createStub()`.
