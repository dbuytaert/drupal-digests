## Summary

The Umami Food Magazine demo profile is now hidden from Drupal's installer profile selection screen. It remains in core so that the many existing tests that depend on it continue to work, but site builders will no longer encounter it when setting up a new site. This is a preparatory step toward moving Umami to a contributed site template, as Drupal CMS and its site templates have superseded it as the recommended demo experience.

## Impact

No upgrade or configuration changes required.

## Technical details

The entire change is a single line added to `profiles/demo_umami/demo_umami.info.yml`: `hidden: true`. Setting this key in a profile's info file causes Drupal's installer to skip the profile during the profile-selection step, while still allowing the profile to be installed programmatically (as tests do). The issue is explicitly framed as an interim step before the larger work tracked in [#3523324](https://www.drupal.org/node/3523324), which proposes deprecating and removing Umami from core entirely in favour of a contributed site template. As catch noted in the thread, combined with [#3159848](https://www.drupal.org/node/3159848) (merging minimal and standard profiles into a recipe), this change means core can be installed from the UI without presenting any profile selection at all.
