## Summary

`BatchStorageInterface` now declares a `getId(): int` method. Previously the method existed only on the concrete `BatchStorage` class, yet Drupal core called it through the interface type-hint, creating a contract gap. Any module that provides a custom implementation of `BatchStorageInterface` must now add this method.

## Impact

Affects module maintainers.

- **API change:** `getId(): int` added to `BatchStorageInterface`; custom implementations of this interface must now declare this method.

## Technical details

The core batch system calls `\Drupal::service(BatchStorageInterface::class)->getId()` in `form.inc` to assign an ID to a new progressive batch. Because `getId()` was defined on `BatchStorage` but absent from `BatchStorageInterface`, static analysis tools and any custom class implementing the interface would not know the method was required.

The fix adds `getId(): int` to `BatchStorageInterface` with a full docblock including `@throws \Exception` (matching the exception the concrete implementation can throw when a new ID cannot be generated). The corresponding docblock on `BatchStorage::getId()` is replaced with `{@inheritdoc}`.

The `ProxyClass\Batch\BatchStorage` generated proxy already declared `getId(): int`, so no changes to that class were needed.

## Upgrade

If you have a custom class that implements `BatchStorageInterface`, add the `getId()` method:

```php
/**
 * {@inheritdoc}
 */
public function getId(): int {
    // Your implementation here.
}
```
