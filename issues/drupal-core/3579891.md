## Summary

This is an internal test-code cleanup with no effect on production behavior. It prepares the Asset library's unit tests for PHPUnit 12.5.0+, which issues notices for mock objects that configure no call-count expectations and deprecates `$this->any()` and `with()` on stubs.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 distinguishes mocks (objects with call-count expectations) from stubs (objects that only provide canned return values). Using `createMock()` for objects that never call `expects()` triggers a notice, and `$this->any()` is deprecated because it is equivalent to having no expectation at all.

The fix replaces `createMock()` with `createStub()` throughout the Asset library unit tests: primarily `AssetResolverTest.php`, `LibraryDiscoveryParserTest.php`, and `CssCollectionOptimizerLazyUnitTest.php`. Test doubles for `LibraryDiscoveryCollector`, `ModuleHandlerInterface`, `ThemeManagerInterface`, `ActiveTheme`, `LanguageInterface`, `LanguageManagerInterface`, `ThemeHandlerInterface`, `AssetCollectionGrouperInterface`, `AssetOptimizerInterface`, and several others are converted. Where a return value is still required, the tests use `->method(...)->willReturn(...)` or `->method(...)->willReturnCallback(...)` on the stub rather than on a mock with `expects()`.
