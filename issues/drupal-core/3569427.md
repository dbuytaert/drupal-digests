## Summary

Unit tests in the System module have been updated to use PHPUnit stubs instead of mocks wherever no call expectations were configured. This is a pure test maintenance change with no effect on production behaviour. It prepares the test suite for PHPUnit 12.5, which deprecates `$this->any()` and will emit notices for mock objects that configure no expectations.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5 introduces two related changes that affect how test doubles are written. First, a mock object that sets no call expectations (`expects(...)`) now emits a notice recommending a stub instead. Second, `$this->any()` is formally deprecated because "expect any number of calls" is semantically equivalent to having no expectation, which is what a stub is for. Additionally, PHPUnit 12.5.11 added a further deprecation: stubs cannot use `with()` to constrain arguments.

The commit updates five test files under `core/modules/system/tests/src/Unit/`:

- `Batch/BatchControllerTest.php`: `createMock(BatchStorageInterface::class)` converted to `createStub`.
- `Breadcrumbs/PathBasedBreadcrumbBuilderTest.php`: Seven properties (`$titleResolver`, `$accessManager`, `$requestMatcher`, `$context`, `$pathProcessor`, `$pathMatcher`, `$cache_contexts_manager`) plus inline `RouteMatchInterface` doubles all converted from `createMock`/`getMockBuilder` to `createStub`. The `$currentUser` and `$currentPath` properties were dropped as class-level fields and replaced with inline `createStub(...)` calls at the `TestPathBasedBreadcrumbBuilder` constructor site. All `expects($this->any())` and `expects($this->once())` chains on stubs are removed, leaving bare `->method()->willReturn(...)` chains. PHPDoc type annotations on the converted fields are updated from `MockObject` to `Stub`. String-based class names (e.g. `'Drupal\Core\Access\AccessManagerInterface'`) are replaced with `::class` constants, and corresponding `use` statements are added.
- `Menu/MenuLinkTreeTest.php`: Five `createMock`/`getMockBuilder` calls for `MenuLinkTree` constructor arguments converted to `createStub`, plus `CacheContextsManager` stub creation simplified from `getMockBuilder(...)->disableOriginalConstructor()->getMock()` to `createStub`.
- `Menu/SystemLocalTasksTest.php`: `createMock(EntityTypeManagerInterface::class)` with `expects($this->any())` converted to `createStub`, and two `$this->themeHandler->expects($this->any())` calls tightened to `expects($this->once())` (these remain mocks with expectations and are not converted to stubs).
- `Pager/PreprocessPagerTest.php`: Three `getMockBuilder(...)->disableOriginalConstructor()->getMock()` calls for `PagerManager`, `Pager`, and `UrlGenerator`, plus one `createMock(RequestStack::class)` with `expects($this->any())`, all converted to `createStub`.

The PHPStan baseline file `core/.phpstan-baseline.php` is updated to remove the suppression entries for the now-fixed `any()` usages in `PathBasedBreadcrumbBuilderTest`, `SystemLocalTasksTest`, and `PreprocessPagerTest`.
