## Summary

Drupal 11 now requires PHP 8.3.0 as its minimum version. Any site running Drupal 11 must be on PHP 8.3.0 or higher. This was a planned change following the policy decision in [#3330874](https://www.drupal.org/node/3330874) and was blocked until the GitLab CI environment was also standardised on PHP 8.3 ([#3400984](https://www.drupal.org/node/3400984)).

## Impact

Affects site owners.

- **Requirements:** Minimum PHP version raised from 8.1.0 to 8.3.0 for Drupal 11

## Technical details

The change touches every layer of the codebase that declares a PHP version constraint.

`core/lib/Drupal.php`: `Drupal::MINIMUM_PHP` changed from `'8.1.0'` to `'8.3.0'`, and `Drupal::RECOMMENDED_PHP` changed from `'8.2.0'` to `'8.3.0'`. These two constants drive the installer pre-flight check and the runtime status report warning in `system_requirements()` in `core/modules/system/system.install`.

`core/composer.json` and all standalone component `composer.json` files under `core/lib/Drupal/Component/`: the `php` requirement constraint changed from `>=8.1.0` to `>=8.3.0`. The root `composer.json` platform override was likewise updated from `8.1.0` to `8.3.0`, and `composer.lock` regenerated accordingly.

Because PHP 8.2 and 8.3 features are now native, `symfony/polyfill-php82` and `symfony/polyfill-php83` were removed as real dependencies and added instead to the root `composer.json` `replace` block (alongside the already-replaced PHP 8.0 and 8.1 polyfills). They were also removed from `composer/Metapackage/CoreRecommended/composer.json` and `composer/Metapackage/PinnedDevDependencies/composer.json`.

The `sebastian/diff` constraint in `core/composer.json` and `core/lib/Drupal/Component/Diff/composer.json` was widened from `^4` to `^4|^5`, since the Diff component can work with the newer release and no longer needs the polyfill restriction.

`core/lib/Drupal/Core/Utility/PhpRequirements.php`: the PHP 8.1 and 8.2 EOL entries were removed from the `$phpEolDates` array since those versions are now below the minimum.

`core/modules/system/system.install`: the PHP 8.1.0–8.1.5 OPcache bug warning (introduced while 8.1 was the minimum) was removed entirely, along with the corresponding test branch in `PhpRequirementTest`. The warning had been annotated with a `@todo` pointing to this very change.

`core/INSTALL.txt` was updated to reference PHP 8.3.0 in both the prerequisites and requirements sections.

`.gitlab-ci.yml` had a latent typo where the "PHP 8.3 SQLite 3" CI job was pointing `_TARGET_PHP` at `"8.2"` instead of `"8.3"`; this was corrected in a small separate commit (96e597e7) that catch noticed and pushed immediately before the main commit (6469e9a1) landed.

## Upgrade

Sites and distributions that pin a `php` constraint in their own `composer.json` should update it to `>=8.3.0`. No PHP code changes are required; this is a platform requirement bump only.

```json
"require": {
    "php": ">=8.3.0"
}
```

Any code that tested or worked around the PHP 8.1.0–8.1.5 OPcache bug (tracked in `https://github.com/php/php-src/issues/8164`) can be removed, as all supported PHP versions for Drupal 11 are unaffected.
