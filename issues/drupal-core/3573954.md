## Summary

Three symbols used in custom and contrib test classes are deprecated in Drupal 12 and will be removed in Drupal 13: `TestRequirementsTrait`, its `getDrupalRoot()` method, and `BrowserTestBase::setUpAppRoot()`. The underlying functionality (resolving the Drupal root path and registering the debug dump handler) has been consolidated into `DrupalTestCaseTrait`, which all Drupal base test classes already use. Developers whose test code references any of the deprecated symbols will start seeing deprecation notices and must update before upgrading to Drupal 13.

## Impact

Affects module maintainers.

- **Deprecation:** `TestRequirementsTrait`, `TestRequirementsTrait::getDrupalRoot()`, and `BrowserTestBase::setUpAppRoot()` deprecated in drupal:12.0.0 and removed in drupal:13.0.0; use `DrupalTestCaseTrait` and access `$this->root` directly.

## Technical details

The consolidation was motivated by three identical copies of `setDebugDumpHandler()` (a `#[BeforeClass]` hook registering the `VarDumper` CLI handler) and two copies of `getDrupalRoot()` living independently in `KernelTestBase`, `BrowserTestBase`, and `UnitTestCase`. Both methods, plus the `$root` property previously declared redundantly in those classes and in `TestSetupTrait`, are now defined once in `DrupalTestCaseTrait`.

The `$root` property in `DrupalTestCaseTrait` is declared with a PHP 8.4 property hook for lazy initialization: the `get` hook calls `static::getDrupalRoot()` on first access and caches the result, so explicit `setUp()` assignments in subclasses are no longer needed. Because property hooks require PHP 8.4, this change targets the `main` branch (Drupal 12) only. A `phpcs:ignoreFile` directive was added to `DrupalTestCaseTrait` temporarily because PHPCS does not yet parse hooked property syntax.

`TestRequirementsTrait` is deprecated at the file level via a top-of-file `@trigger_error`, its `getDrupalRoot()` method carries an individual deprecation notice, and `BrowserTestBase::setUpAppRoot()` is deprecated with a notice to access `$this->root` directly. The change record is at `https://www.drupal.org/node/3574112`.

## Upgrade

If your test class uses `TestRequirementsTrait`, remove it. All Drupal base test classes (`UnitTestCase`, `KernelTestBase`, `BrowserTestBase`) already pull the same functionality from `DrupalTestCaseTrait`.

```php
// Before
use Drupal\Tests\TestRequirementsTrait;

class MyTest extends UnitTestCase {
  use TestRequirementsTrait;
  ...
}

// After — no use statement needed; functionality comes from UnitTestCase
class MyTest extends UnitTestCase {
  ...
}
```

If your test code calls `static::getDrupalRoot()`, access `$this->root` instead:

```php
// Before
$root = static::getDrupalRoot();

// After
$root = $this->root;
```

If your test class overrides or calls `setUpAppRoot()`, remove the call:

```php
// Before (in setUp())
$this->setUpAppRoot();

// After — remove the call; $this->root is initialized lazily via the property hook
```
