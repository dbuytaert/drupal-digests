## Summary

Two Symfony security vulnerabilities are addressed by bumping `symfony/http-foundation` and `symfony/routing` to version `^8.0.13` across all supported Drupal branches (10.5.x through main). Sites that run `composer audit` before this update will see CVE-2026-48736 and CVE-2026-48784 reported; after updating they will not. Both CVEs were disclosed on 2026-05-26.

## Impact

Affects site owners.

- **Security:** CVE-2026-48736 (SSRF bypass via IPv6 omission in `symfony/http-foundation`) and CVE-2026-48784 (URL path traversal via dot-segment encoding in `symfony/routing`) patched by requiring both packages at `^8.0.13`.

## Technical details

CVE-2026-48736 is in `symfony/http-foundation`: `IpUtils::PRIVATE_SUBNETS` omitted IPv6 transition address forms (6to4, NAT64, Teredo, IPv4-compatible), allowing an SSRF bypass via `NoPrivateNetworkHttpClient`. The fix raises the minimum required version from `^8.0` (previously pinned to `v8.0.8` in the lockfile) to `^8.0.13`.

CVE-2026-48784 is in `symfony/routing`: `UrlGenerator` dot-segment encoding skipped every other chained `../` or `./` sequence, potentially collapsing a generated URL off-route under RFC 3986 normalization. The fix raises the minimum from `^8.0.12` to `^8.0.13`. Note that Drupal ships its own `UrlGenerator` subclass, so fully covering CVE-2026-48784 for Drupal also requires the separate fix tracked in [#3592341](https://www.drupal.org/node/3592341).

The diff touches `core/composer.json`, `composer/Metapackage/CoreRecommended/composer.json`, `composer/Metapackage/PinnedDevDependencies/composer.json`, and `composer.lock`. No PHP source files outside of Composer metadata were changed.
