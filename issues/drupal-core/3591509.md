## Summary

Five `$settings` keys related to Migrate Drupal have been removed from `default.settings.php`. These keys were used exclusively by the Migrate Drupal module, which was removed from core in [#3572280](https://www.drupal.org/node/3572280). If your `settings.php` still defines any of them, they will be silently ignored; you can safely remove them.

## Impact

Affects site owners.

- **Config change:** Five Migrate Drupal `$settings` keys removed from `default.settings.php`; any definitions still present in `settings.php` are now silently ignored.

## Technical details

The commit removes two blocks from both `core/assets/scaffold/files/default.settings.php` and `sites/default/default.settings.php`. The first block was an active assignment, `$settings['migrate_node_migrate_type_classic'] = FALSE`, which forced the classic node migration path in the Migrate Drupal upgrade wizard. The second block was a set of commented-out template entries for `$settings['migrate_source_connection']`, `$settings['migrate_source_version']`, `$settings['migrate_file_public_path']`, and `$settings['migrate_file_private_path']`, which supplied default values to the `/upgrade/credentials` credential form. With `migrate_drupal` removed from core ([#3572280](https://www.drupal.org/node/3572280)), no code in core reads any of these keys, making the settings stubs dead documentation.

## Upgrade

If your `settings.php` still defines any of the following keys from a past D6/D7 migration, they are now unused and can be removed:

```php
$settings['migrate_node_migrate_type_classic'] = FALSE;
$settings['migrate_source_connection'] = '';
$settings['migrate_source_version'] = '';
$settings['migrate_file_public_path'] = '';
$settings['migrate_file_private_path'] = '';
```
