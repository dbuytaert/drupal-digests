## Summary

This META task tracked known accessibility issues affecting the Settings Tray module and its off-canvas dialog, coordinating the work required before Settings Tray could be marked stable. All must-have accessibility blockers were resolved through dedicated child issues: improved tab flow in Settings Tray ([#2784569](https://www.drupal.org/node/2784569)), visible focus styling for off-canvas dialog links ([#2915759](https://www.drupal.org/node/2915759)), constrained contextual tabbing on new page loads in edit mode ([#2919147](https://www.drupal.org/node/2919147)), contrast improvements in the off-canvas dialog ([#2934178](https://www.drupal.org/node/2934178)), and correct focus styling for the off-canvas close button ([#2934499](https://www.drupal.org/node/2934499)). Remaining "should-have" items (returning focus to the originating contextual link, [#2915762](https://www.drupal.org/node/2915762)) and "nice-to-have" items (reduced-motion support for the slide-in animation, [#2940677](https://www.drupal.org/node/2940677)) were tracked but not required for stable. The issue was closed after approximately eight years once all must-haves were confirmed resolved.

## Technical details

This is a coordination META issue with no direct code changes of its own. It served as a living checklist spanning three subsystems: `settings_tray.module`, the off-canvas dialog library (originally added for Settings Tray), and the contextual links module.

Must-have items resolved in child issues:
- **[#2784569](https://www.drupal.org/node/2784569)**: Improved tabbing flow inside the Settings Tray panel.
- **[#2915759](https://www.drupal.org/node/2915759)**: Visible `:focus` styling for links inside the off-canvas dialog.
- **[#2919147](https://www.drupal.org/node/2919147)**: Ensured contextual tabbing was constrained on new page loads while edit mode was active.
- **[#2934178](https://www.drupal.org/node/2934178)**: Contrast ratio fixes in off-canvas dialog CSS.
- **[#2934499](https://www.drupal.org/node/2934499)**: Focus and hover border styling for the off-canvas `[x]` close button, initially postponed on [#2863354](https://www.drupal.org/node/2863354).

Should-have item still open at close:
- **[#2915762](https://www.drupal.org/node/2915762)**: Return keyboard focus to the contextual link tab position when the dialog is dismissed.

Nice-to-have item tracked separately:
- **[#2940677](https://www.drupal.org/node/2940677)** / **[#2928103](https://www.drupal.org/node/2928103)**: `prefers-reduced-motion` support for the off-canvas slide-in animation, noted by andrewmacpherson as aligning with WCAG 2.1 level AAA.

One design decision recorded in the thread: the requirement to display a "You are now in edit mode" announcement ([#2773601](https://www.drupal.org/node/2773601)) was removed from the must-have list after a UX meeting concluded that `Drupal.announce()` (already called on edit-mode entry, matching the contextual module's behavior) provided sufficient programmatic notification alongside the visual cues on the page.
