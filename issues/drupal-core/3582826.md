## Summary

The monolithic `migration/css/base/gin.css` file (5000+ lines of compiled Sass output copied from the Gin contrib theme) has been removed from the `default_admin` theme and replaced with 45 individual PostCSS files. Each file corresponds to one of the original Gin Sass partials. The new files use standard PostCSS features supported by core's build tooling: custom media queries via `postcss-custom-media`, standard CSS nesting, and logical properties instead of `postcss-rtlcss` directives. Both source (`.pcss.css`) and compiled (`.css`) versions exist for each partial. The theme is still in a migration directory and has known pre-existing visual regressions unrelated to this change.

## Technical details

The `default_admin` theme (the new experimental admin theme replacing Claro/Gin) inherited its base styles from a single file, `migration/css/base/gin.css`, which was a direct compiled output of the Gin contrib theme's monolithic `gin.scss`. This file was not compatible with core's PostCSS-based build process and could not be maintained or evolved incrementally.

The refactor followed a structured sequence: Sass media queries were converted to PostCSS custom media (`@custom-media --admin-large` etc., defined in a shared `media-queries.pcss.css`); Sass BEM nesting shortcuts (`&__selector`, `&--modifier`) were expanded to full selectors; Sass icon management was replaced with CSS custom properties holding inline SVG data URIs (in `gin-icons.css`); Sass variables and functions were replaced with CSS custom properties already present in the theme; `postcss-rtlcss` directives were replaced with CSS logical properties (`inset-inline-start`, `padding-block`, etc.); and finally each Sass partial was moved to its own `.pcss.css` source file with a corresponding compiled `.css` output.

The 45 resulting files cover distinct UI areas: `gin-body`, `gin-button`, `gin-dropbutton`, `gin-form`, `gin-field-ui`, `gin-focus`, `gin-icons`, `gin-regions`, `gin-table`, `gin-tabledrag`, and so on. `default_admin.libraries.yml` was updated to list all 45 files in place of the single `gin.css` entry. The compiled `.css` files carry a "DO NOT EDIT" header pointing to change record [#3084859](https://www.drupal.org/node/3084859).

A toggle-button regression introduced during the conversion was identified through visual regression testing (the `ddev-drupal-admin-vrt` DDEV add-on) and fixed before merge. A secondary-button hover regression was caught by a reviewer and fixed by a contributor. Several other pre-existing visual differences from Gin 5.x were noted and deferred to follow-up issues.
