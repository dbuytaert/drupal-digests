## Summary

The `default_admin` admin theme's single `migration/css/base/gin.css` file, a 5000-plus line monolith compiled from the Gin theme's Sass source, has been removed. In its place, the same styles now live in roughly 44 separate PostCSS files, one per UI component (for example `gin-button.css`, `gin-form.css`, `gin-dropbutton.css`, `gin-regions.css`, and so on). The `default_admin.libraries.yml` is updated to reference all of the new files. The visual output is intended to be functionally equivalent; the change is structural preparation for the ongoing admin-theme CSS clean-up tracked in [#3582351](https://www.drupal.org/node/3582351).

## Impact

Affects front-end developers.

## Technical details

The original `gin.css` was a machine-compiled artifact of the Gin contrib theme's Sass build pipeline, not hand-authored PostCSS, so it could not be maintained or patched incrementally within Drupal core. The refactor replayed the Gin Sass build in reverse: Sass media-query mixins were expanded to plain `@media` rules (and then to PostCSS `@import "media-queries.pcss.css"` custom-media aliases), Sass BEM `&__`/`&--` nesting was expanded to valid CSS nesting, `postcss-rtlcss` directionality shims were rewritten using logical CSS properties (`inset-inline-start`, `padding-block`, etc.), and SVG icon management was moved to CSS custom properties (`--admin-icon-*` mask-image tokens) in `gin-icons.css`. Each resulting PostCSS file corresponds to one of the Sass partials from the original Gin source. The compiled output of each new `.pcss.css` source file is the matching `.css` file that ships; both sets of files are in `core/themes/default_admin/migration/css/base/`. The entry in `default_admin.libraries.yml` changes from a single `migration/css/base/gin.css: { minified: false }` line to ~44 individual file references under the `theme` group.

## Upgrade

If you maintain a custom patch or override targeting `core/themes/default_admin/migration/css/base/gin.css`, that file no longer exists. Find the relevant rules in the new per-component files under the same `migration/css/base/` directory. Examples of the new file mapping:

- Button styles → `gin-button.css`
- Form element styles → `gin-form.css`
- Dropbutton styles → `gin-dropbutton.css`
- Layout regions → `gin-regions.css`
- Focus ring styles → `gin-focus.css`
- Icon token definitions → `gin-icons.css`

If you extend the `default_admin` library in a sub-theme or contrib module and referenced `gin.css` by name, update your `libraries.yml` or `hook_library_info_alter()` accordingly.
