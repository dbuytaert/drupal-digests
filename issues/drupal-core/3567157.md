## Summary

Unit tests in the Serialization module have been updated to use PHPUnit stubs instead of expectation-less mocks, and to remove deprecated `$this->any()` expectations. This is part of a broader effort ([#3561671](https://www.drupal.org/node/3561671)) to prepare Drupal core for PHPUnit 12.5+, which emits notices when mock objects are created without configuring any expectations and deprecates the `any()` expectation matcher. The change has no effect on runtime behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced a notice when a mock object (created with `createMock()`) has no expectations configured, because such objects are semantically equivalent to stubs and should be created with `createStub()` instead. Additionally, `expects($this->any())` was deprecated because it is equivalent to having no expectation at all.

The commit touches nine test files under `core/modules/serialization/tests/src/Unit/`:

**Encoder/XmlEncoderTest.php**: The shared `setUp()` method created a `$baseEncoder` mock that was unused in `testSupportsEncoding()` and `testSupportsDecoding()`. The fix splits setup out of `setUp()` into a new `setUpEncoder()` helper called only from the two tests that actually need the mock (`testEncode()` and `testDecode()`). The two format-support tests now instantiate a plain `XmlEncoder` locally.

**EntityResolver/ChainEntityResolverTest.php** and **EntityResolver/UuidResolverTest.php**: `$testNormalizer` and local `$normalizer` variables changed from `createMock(NormalizerInterface::class)` to `createStub(NormalizerInterface::class)`. The `NormalizerInterface` import was added to both files.

**Normalizer/ConfigEntityNormalizerTest.php**: Three service collaborators (`EntityFieldManagerInterface`, `EntityTypeManagerInterface`, `EntityTypeRepositoryInterface`) converted to stubs.

**Normalizer/ContentEntityNormalizerTest.php**: The same three service collaborators converted to stubs in `setUp()`. `ContentEntityInterface`, `ConfigEntityInterface`, and `AccountInterface` mocks for type-checking tests converted to stubs. The helper `createMockForContentEntity()` was renamed to `createStubForContentEntity()`, its visibility changed from `public` to `protected`, its parameter typed as `array`, and its return type changed to `Stub`. Inside the helper, `getMockBuilder()->onlyMethods()->getMock()` was replaced with `createStub()`, and `expects($this->any())->method('getTypedData')` was replaced with a plain `->method('getTypedData')` call. The `Stub` and `ConfigEntityInterface`/`ContentEntityInterface` use statements were added.

**Normalizer/EntityNormalizerTest.php**: `$entityTypeRepository` converted to a stub. `TypedDataInterface` and `FieldItemListInterface` local variables converted to stubs where no expectations were configured. Explicit `expects($this->never())` assertions were added to the still-mocked `$entityFieldManager` and `$entityTypeManager` in `testNormalize()` and `testDenormalizeWithNoEntityType()` to document that those methods must not be called in those test paths.

**Normalizer/ListNormalizerTest.php**: `$typedData` and `$typed_data_manager` converted to stubs. The `Symfony\Component\DependencyInjection\ContainerBuilder` mock (built with `getMockBuilder()->onlyMethods(['get'])->getMock()` plus `expects($this->any())`) was replaced with a real `Drupal\Core\DependencyInjection\ContainerBuilder` instance using `$container->set('typed_data_manager', $typed_data_manager)`.

**Normalizer/NullNormalizerTest.php**: Two local `TypedDataInterface` mocks in `testSupportsNormalization()` and `testNormalize()` converted to stubs.

**Normalizer/TypedDataNormalizerTest.php**: The shared `$typedData` property and its creation in `setUp()` were removed. `testSupportsNormalization()` now creates a local stub inline. `testNormalize()` retains a local mock (with `expects($this->once())`) because it verifies `getValue()` is called exactly once.

The `.phpstan-baseline.php` entries suppressing `any()` deprecation notices for `ContentEntityNormalizerTest` and `ListNormalizerTest` were removed as they are no longer needed.
