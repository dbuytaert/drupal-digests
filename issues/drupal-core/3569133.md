## Summary

The Shortcut module is no longer automatically installed when creating a new Drupal site using the Standard or Demo Umami profiles, or the Standard recipe. This is one step in a broader effort to move the Shortcut module out of Drupal core. Existing sites are unaffected; the module remains available and can still be enabled manually.

## Impact

No upgrade or configuration changes required.

## Technical details

This issue removes `shortcut` module integration from three bundled installation paths as part of the broader project to retire the Shortcut module from core.

**Standard profile** (`core/profiles/standard/`): `shortcut` is removed from the install list in `standard.info.yml`. The entire `standard.install` file is deleted, since its only content was a `standard_install()` hook that created two default `Shortcut` entities ("Add content" at `/node/add` and "All content" at `/admin/content`). The `access shortcuts` permission is dropped from `config/install/user.role.authenticated.yml`.

**Demo Umami profile** (`core/profiles/demo_umami/`): `shortcut` is removed from `demo_umami.info.yml`. The equivalent shortcut-creation block is removed from `demo_umami_install()` in `demo_umami.install`. The Claro admin theme's shortcut integration config file `config/install/claro.settings.yml` (which only contained the `shortcut.module_link: true` third-party setting) is deleted entirely. The `access shortcuts` permission is removed from the authenticated role config and `administer shortcuts` from the editor role config.

**Standard recipe** (`core/recipes/standard/recipe.yml`): Both the `basic_shortcuts` recipe reference and the direct `shortcut` module install entry are removed. andypost noted that these must land in the same commit as the profile changes because `StandardRecipeTest` enforces that profile and recipe installations produce matching configurations.

**Test fixtures**: Two kernel-test recipe fixtures (`config_from_module_and_recipe` and `config_wildcard`) that used `shortcut` as a convenient module with known config are updated to remove that dependency. `ConfigActionValidationTest` drops the `shortcut_set` test case and removes the shortcut module from its setup, and gains a `#[DoesNotPerformAssertions]` attribute on an unrelated test that was left assertion-free.

**Test updates**: Migration upgrade tests (`Upgrade6Test`, `Upgrade7Test`, `MultilingualReviewPageTestBase` across `contact` and `migrate_drupal_ui` modules) now explicitly declare `shortcut` in their `$modules` array, since it is no longer auto-installed via the standard profile, and update expected shortcut entity counts to reflect what the migrated source database contains rather than what the profile previously added. `StandardRecipeInstallTest` and `StandardRecipeTest` remove assertions that exactly two shortcuts were created. OpenTelemetry performance baselines for Umami and Navigation are adjusted downward to reflect the reduced query and cache load.

Further cleanup, including removal of the `basic_shortcuts` recipe itself, is tracked in [#3571079](https://www.drupal.org/node/3571079).
