## Summary

Drupal core now includes a new experimental administrative theme called Admin, based on the contributed Gin theme. This provides a modern editorial experience with improved usability features including dark mode support, customizable accent colors, configurable layout density, and support for sticky action buttons on content forms. Site owners can enable Admin to give content editors a refreshed interface while the theme continues to be refined for eventual stable release.

## Impact

No upgrade or configuration changes required.

## Technical details

The Admin theme is an experimental addition to core derived from the Gin contributed theme. It replaces the current pattern of maintaining separate Claro (core) and Gin (contributed) admin themes, which caused duplication of effort and slower development of administrative UI improvements.

The theme includes substantial styling overrides for core libraries and components, bringing Gin's design patterns into core. It supports both the Navigation module and the legacy Toolbar module during the transition period. The theme provides two feature flags: one to control sticky action buttons on content forms (defaulting to never, with options for content_forms and always) and another to apply the admin theme to the login page (defaulting to true).

The implementation includes hooks for modules to register routes that should use the admin content form layout. Files in the migration directory contain legacy code that will be deduplicated and merged with Claro components in follow-up work. Many CSS custom properties use the gin- prefix as a transitional measure.

As an experimental theme, Admin does not need to meet all core gates immediately and can continue to evolve before reaching stable status.
