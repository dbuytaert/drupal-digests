## Summary

A new experimental admin theme called Admin has been added to Drupal core. This theme evolved from the contributed Gin theme and provides a modern editing experience with improved usability. It includes two feature flags: one to control sticky action buttons on forms and another to optionally use the admin theme for the login page. The theme is experimental in its first release, meaning it will continue evolving before becoming stable. Site owners can enable it to test the new interface but should expect ongoing changes.

## Impact

No upgrade or configuration changes required.

## Technical details

The Admin theme is a fork of the Gin contrib theme being merged into core under a new machine name. The merge includes approximately 10,000 lines of CSS, JavaScript, templates, and PHP across base styles, components, and layout files. The theme overrides and extends many core libraries including dropbuttons, vertical tabs, dialogs, tabledrag, and form elements.

Two settings are available: `$settings['core_admin_theme_use_sticky_action_buttons']` controls where sticky action buttons appear (values: `never`, `content_forms`, `always`), and `$settings['user_login_admin_theme']` determines if the login page uses the admin theme (defaults to `TRUE`).

The theme includes hooks for contrib modules to register routes that should use the content form layout (`hook_admin_content_form_routes()` and its alter hook). Much of the code still references "gin" internally in CSS classes, JavaScript variables, and settings keys. A deduplication effort with Claro components remains in a migration directory and will be completed in follow-up issues. The theme extends rather than replaces Claro initially, maintaining three admin themes during the transition period.
