## Summary

Drupal 11.1 introduces a new experimental admin theme called Admin, evolved from the contributed Gin theme. This provides a modern administrative interface focused on improving the editorial experience, with features like customizable accent colors, dark mode support, and sticky action buttons on content forms. The theme ships with Drupal core as an alpha experimental option, allowing site builders to test and provide feedback while the theme is further refined. Once stable, Admin is intended to become the single default administrative theme in Drupal, consolidating development that has been split between Claro and Gin.

## Impact

No upgrade or configuration changes required.

## Technical details

The Admin theme was merged from Gin (a popular contributed theme) using the machine name `admin`. It ships as alpha experimental, meaning it does not need to meet all core gates initially and can evolve based on community feedback.

The theme includes extensive CSS, JavaScript, templates, and preprocessing logic ported from Gin. It provides two feature flags: `core_admin_theme_use_sticky_action_buttons` (defaults to `never`, can be `content_forms` or `always`) and `user_login_admin_theme` (defaults to `TRUE`). The theme supports both the legacy Toolbar module and the new Navigation module.

A significant amount of code was copied from Claro since Gin originally forked from it. Files in the `migration` directory contain deduplicated components that will be merged back into primary CSS and JavaScript files in follow-up work. The theme uses modern CSS features like color-mix and relative color syntax with fallbacks. The implementation includes hooks like `hook_admin_content_form_routes()` to allow modules to opt their entity forms into Admin's enhanced layout.
