Drupal core now includes a new experimental admin theme called Admin, which merges the popular contributed Gin theme into core. This theme provides a modern administrative interface focused on improving the editorial experience. Site owners can install and test the Admin theme now, though it is marked as experimental while development continues. ## Technical details

The Admin theme is based on Gin 6.x and was merged into core as an experimental theme. The theme lives in `core/themes/admin` with the machine name `admin`. It includes extensive CSS and JavaScript assets, templates, configuration, and hooks to customize the administrative experience.

The merge includes two feature flags in `settings.php`: `core_admin_theme_use_sticky_action_buttons` (defaults to `never`, accepts `content_forms` or `always`) and `user_login_admin_theme` (defaults to `TRUE`). These control sticky action button behavior and whether the login page uses the admin theme.

The theme provides hooks like `hook_admin_content_form_routes()` and `hook_admin_content_form_ignore_form_ids()` to allow modules to register routes that should use the admin theme's content edit form layout. Many CSS and JavaScript files are organized in a `migration` directory, reflecting ongoing work to deduplicate code shared with Claro. The theme overrides numerous core libraries for components like dropbuttons, vertical tabs, autocomplete, and dialog elements to provide consistent styling. While based on Gin, many references to "Gin" remain in internal class names and variables and will be cleaned up in follow-up work.

The path to commit involved significant technical challenges. Multiple contributors helped resolve CSS linting issues, cspell errors, JavaScript formatting, and failing tests.
