## Summary

Drupal core now ships a new experimental administrative theme named Admin (machine name `admin`), derived from the contributed Gin theme. Site owners can enable it to give content editors a modernised back-end interface with dark mode support, configurable accent and focus colours, adjustable layout density, and optional sticky action buttons on content forms. The theme is marked experimental, meaning it does not yet need to meet all core stability gates, and will continue to evolve towards a stable release. Enabling it requires the Navigation module (the legacy Toolbar module is supported during the transition). Two `$settings` keys control behaviour: `core_admin_theme_use_sticky_action_buttons` (default `never`, options `content_forms` and `always`) and `user_login_admin_theme` (default `TRUE`).

## Impact

Affects module maintainers and site owners.

- **Config change:** New `admin.settings` config entity (schema type `theme_settings`) with keys for accent colour, focus colour, dark mode, high contrast mode, layout density, and user-override flags.
- **New API:** Three new hooks in `admin.api.php`: `hook_admin_content_form_routes()`, `hook_admin_content_form_routes_alter()`, and `hook_admin_content_form_ignore_form_ids()` for controlling which routes and forms receive the Admin theme's editorial layout.

## Technical details

The theme lives at `core/themes/admin/` and is declared with `experimental: true` in `admin.info.yml`. Its configuration schema is defined in `config/schema/admin.schema.yml` as `admin.settings`, a `theme_settings` type covering `preset_accent_color`, `preset_focus_color`, `enable_dark_mode`, `high_contrast_mode`, `accent_color`, `focus_color`, `layout_density`, `show_description_toggle`, and `show_user_theme_settings`. Default config ships in `config/install/admin.settings.yml` (blue accent, gin focus colour, dark mode off, default density).

The diff introduces three new hooks in `admin.api.php`: `hook_admin_content_form_routes()` lets modules register route names that should receive the Admin theme's editorial form layout (two-column sidebar with entity meta); `hook_admin_content_form_routes_alter(array &$routes)` allows altering that list; and `hook_admin_content_form_ignore_form_ids()` registers form IDs to exclude from the layout. These are resolved by `Helper::isContentForm()`.

The theme overrides CSS and JS for a large number of core and contrib libraries via `libraries-override` and extends them via `libraries-extend` in `admin.libraries.yml`. A `migration/` subdirectory holds legacy SCSS-derived CSS and JS from Gin that has not yet been deduplicated against Claro; those files are intentionally excluded from the ESLint pass (via `.eslintignore`) until a follow-up consolidation is done. CSS custom properties throughout the theme still carry the `gin-` prefix as a transitional polyfill (e.g. `--gin-color-primary` maps to `--accent-color-500`). Accent colour computation uses `color-mix(in lch, ...)` with a progressive-enhancement `@supports` block that switches to `rgb(from ...)` relative colour syntax in browsers that support it.

The commit also fixes `EntityFilteringThemeTest`, where a term was created with a hardcoded `vid` of `1` dating to 2012. The fix replaces it with `TaxonomyTestTrait::createVocabulary()` and `createTerm()`. A follow-up issue ([#3576431](https://www.drupal.org/node/3576431)) corrected the theme's experimental status display in the UI.

## Upgrade

Modules that provide custom content entity edit forms and want them to receive the Admin theme's two-column editorial layout should implement the new hook:

```php
/**
 * Implements hook_admin_content_form_routes().
 */
function mymodule_admin_content_form_routes(): array {
  return [
    'entity.my_type.edit_form',
    'entity.my_type.add_form',
  ];
}
```

To exclude specific routes or form IDs already registered by another module:

```php
/**
 * Implements hook_admin_content_form_routes_alter().
 */
function mymodule_admin_content_form_routes_alter(array &$routes): void {
  $routes = array_diff($routes, ['entity.my_type.edit_form']);
}
```

No changes are required for sites that do not enable the Admin theme.
