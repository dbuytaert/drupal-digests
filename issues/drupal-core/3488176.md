## Summary

The procedural `system_theme()` hook implementation and its `drupal_common_theme()` helper have been converted to object-oriented code. A new `ThemeHook` class in the system module now implements `hook_theme()` using the `#[Hook]` attribute, and a new `ThemeCommonElements` class holds the shared theme element definitions previously scattered between those two functions. Because the system module is not yet registered when the theme registry is built during site installation, the change also adds special install-time handling in `Registry::build()` so that the installer can still access the required theme elements. This is an internal refactoring with no effect on the rendered output of any Drupal site.

## Impact

Affects module maintainers.

- **API change:** `system_theme()` procedural function removed; replaced by `ThemeHook::theme()` in `Drupal\system\Hook\ThemeHook`. `drupal_common_theme()` removed; replaced by `ThemeCommonElements::commonElements()` in `Drupal\Core\Theme\ThemeCommonElements`.
- **Deprecation:** `drupal_common_theme()` and `system_theme()` are removed without a deprecation period as part of the procedural-to-OOP hook conversion.
- **New API:** New `Drupal\Core\Theme\ThemeCommonElements` class with static `commonElements()` method providing the shared core theme element definitions.

## Technical details

The system module previously declared `system_theme()` in `system.module`, which merged the output of `drupal_common_theme()` (from `core/includes/theme.inc`) with system-specific entries. This was the last procedural `hook_theme()` implementation in core awaiting OOP conversion (following [#3487971](https://www.drupal.org/node/3487971)).

Two obstacles complicated the conversion. First, during site installation (`MAINTENANCE_MODE === 'install'`), the system module is not yet registered, so `Registry::build()` cannot reach the new OOP hook class through the normal `invokeAllWith('theme', ...)` loop. Second, a `RegistryTest` exercises the registry in a partially-bootstrapped state where Drupal is not fully initialised.

The solution introduces two new classes. `Drupal\Core\Theme\ThemeCommonElements` in `core/lib/Drupal/Core/Theme/ThemeCommonElements.php` provides a static `commonElements()` method carrying all of the theme element definitions that were previously in `drupal_common_theme()`, plus `off_canvas_page_wrapper` and `status_report_grouped` which previously lived in the system-specific section. `Drupal\system\Hook\ThemeHook` in `core/modules/system/src/Hook/ThemeHook.php` carries a `#[Hook('theme')]`-attributed `theme()` method that calls `ThemeCommonElements::commonElements()` and merges in the remaining system-specific entries via `array_merge()`.

In `Registry::build()`, when `MAINTENANCE_MODE === 'install'` is detected, the registry bypasses `invokeAllWith` and instead calls `processExtension()` directly for the system module using a new synthetic type string `'install'`. Inside `processExtension()`, this new type causes `ThemeCommonElements::commonElements()` to be called directly for the theme array, and the type is then immediately reset to `'module'` so that preprocess hook discovery proceeds via the normal path. This makes the full set of common theme elements available to the installer without requiring the system module to be registered.

`drupal_common_theme()` is deleted from `core/includes/theme.inc` and its PHPStan baseline entry in `core/.phpstan-baseline.php` is removed. The `system_theme()` function is deleted from `system.module`. A `ContentNegotiationRoutingTest` kernel test gained `system` in its `$modules` list as a side-effect of the change.

## Upgrade

Any code calling `drupal_common_theme()` directly should be updated to call `\Drupal\Core\Theme\ThemeCommonElements::commonElements()` instead:

```php
// Before
$theme = drupal_common_theme();

// After
use Drupal\Core\Theme\ThemeCommonElements;
$theme = ThemeCommonElements::commonElements();
```

The procedural `system_theme()` function has also been removed. It was a hook implementation and should not have been called directly, but if your code did call it, use `\Drupal\system\Hook\ThemeHook::theme()` or invoke it through the hook system.
