## Summary

On multilingual Drupal sites, enabling a module while the interface translations directory (`translations://`) is misconfigured or unwriteable previously caused a fatal PHP error: `TypeError: fgets(): Argument #1 ($stream) must be of type resource, bool given`. This would crash Drush or a deployment pipeline mid-run. The fix makes `PoStreamReader` throw a catchable `RuntimeException` instead, so the locale subsystem can log a clear message and continue rather than halting with an unhandled fatal error.

## Impact

Affects module maintainers.

- **API change:** `PoStreamReader::open()` now throws `\RuntimeException` when `fopen` fails to open the stream, instead of silently setting the internal file descriptor to `false`; callers that invoke `open()` directly should wrap it in a try/catch.

## Technical details

In `PoStreamReader::open()`, the original code called `fopen()` but never checked its return value. When `fopen` fails (for example because `translations://` is not properly configured), it returns `false`. A subsequent call to `readLine()` then passed that `false` value directly to `fgets()`, which in PHP 8 throws a `TypeError`.

The fix restructures `open()`: it calls `@fopen($this->uri, 'rb')` (suppressing the PHP-level warning that would otherwise be emitted) and immediately checks the result. If the handle is falsy, it throws `new \RuntimeException('Cannot open stream for uri ' . $this->uri)`. This gives calling code a catchable, typed exception rather than an unhandled fatal. A new `PoStreamReaderTest` unit test was added in `core/tests/Drupal/Tests/Component/Gettext/PoStreamReaderTest.php` to verify this path.

The related issue [#3546047](https://www.drupal.org/node/3546047) was opened separately to improve the upstream logging in `locale_translation_download_source()` when the download itself fails, covering the complementary failure mode.

## Upgrade

If you call `PoStreamReader::open()` directly, wrap it to handle the new failure mode:

```php
// Before: open() would silently succeed even if fopen() failed.
$reader->setURI($uri);
$reader->open();

// After: open() throws \RuntimeException when the stream cannot be opened.
$reader->setURI($uri);
try {
  $reader->open();
}
catch (\RuntimeException $e) {
  // Handle misconfigured translations directory or unreadable file.
  \Drupal::logger('mymodule')->error($e->getMessage());
}
```
