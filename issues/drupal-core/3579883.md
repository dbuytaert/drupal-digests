## Summary

This is a test-only change with no effect on production code or site behavior. It updates unit tests in the `EventSubscriber`, `StackMiddleware`, and `Update` test suites to be compatible with PHPUnit 12.5.0, which emits notices when mock objects are created without any configured expectations.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced a notice for mock objects created via `createMock()` that never set any expectations: "No expectations were configured for the mock object … you should use a test stub instead." It also deprecated `$this->any()` (which was equivalent to having no expectation) and deprecated `with()` on stubs. The fix converts those usage patterns across roughly 10 test classes in three directories: `core/tests/Drupal/Tests/Core/EventSubscriber/`, `core/tests/Drupal/Tests/Core/StackMiddleware/`, and `core/tests/Drupal/Tests/Core/Update/`. Concretely, `$this->createMock(Foo::class)` calls that carried no `expects()` assertion were changed to `$this->createStub(Foo::class)`, and any remaining `->expects($this->any())` calls were dropped. When a real call-count expectation existed (`once()`, etc.) the mock was left as a mock. The `Update` suite uses Prophecy, so the relevant changes there address related but analogous patterns. This issue is part of a broader PHPUnit 12 compatibility effort; a companion issue ([#3578904](https://www.drupal.org/node/3578904)) applied the same pattern to `UnitTestCase` base-class tests and was merged first, resolving five additional notices that appeared when running the affected tests in isolation.
