## Summary

Olivero's `.theme` file and three test theme `.theme` files have been removed. The hooks from these files have been converted to hook classes using PHP attributes. Olivero's `_olivero_hex_to_hsl()` helper function has been moved to a trait. This change has no effect on site functionality or external code.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue converts procedural hooks and helper functions in `.theme` files to modern hook classes and traits.

For Olivero, the `_olivero_hex_to_hsl()` function is moved to `HexToHslTrait` as a protected `convertHexToHsl()` method. The function converts hex color codes to HSL values for CSS custom properties. `OliveroPagePreprocessHooks` now uses the trait to call the method. The unit test is updated to use the trait directly instead of requiring the `.theme` file. The function had no contrib usage according to a tresbien.tech search.

For test themes, `test_installer_theme_form_install_select_language_form_alter()` moves to `TestInstallerThemeHooks::formInstallSelectLanguageFormAlter()`, and `views_test_checkboxes_theme_form_views_exposed_form_alter()` moves to `ViewsTestCheckboxesThemeHooks::formViewsExposedFormAlter()`. The empty `test_missing_base_theme.theme` file is deleted. The issue leaves `stable9.theme` and `access_test.theme` unchanged for backwards compatibility and testing.
