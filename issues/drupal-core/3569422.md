## Summary

This is a purely internal change to Drupal's migrate module test code. It has no effect on site behavior, configuration, or the public API. The change prepares the test suite for full compatibility with PHPUnit 12.5 by replacing expectation-less mock objects with proper stubs.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 12.5.0 introduced two breaking changes relevant to this work: mock objects that configure no call expectations now emit a deprecation notice telling you to use a stub instead, and `$this->any()` is deprecated because it is equivalent to having no expectation. A stub (created with `createStub()`) is the correct PHPUnit concept for a test double that simply returns configured values without verifying call counts.

The migrate module's unit test suite made heavy use of `createMock()` or `getMockBuilder()->getMock()` even for objects where no `expects(...)` call followed, and used `->expects($this->any())` as a no-op preamble before `->method()->willReturn()` chains. This issue replaces all such patterns with `createStub()` and removes the `any()` calls, across approximately 17 test files under `core/modules/migrate/tests/src/Unit/`.

The central base classes were updated first. `MigrateTestCase::getMigration()` now calls `createStub()` on the migration class instead of `getMockBuilder()->disableOriginalConstructor()->getMock()`, and the default `$idMap` created there switches from `createMock(MigrateIdMapInterface::class)` to `createStub()`. `MigrateProcessTestCase::setUp()` converts its shared `$row` and `$migrateExecutable` properties to stubs via `createStub(Row::class)` and `createStub(MigrateExecutableInterface::class)`.

Tests that do need real call-count expectations on the ID map (such as `testImportWithValidRowWithoutDestinationId` and `testImportWithValidRowWithProcessMigrateException` in `MigrateExecutableTest`) explicitly opt back in to a mock by calling a new helper method `setUpMigrationWithMockIdMap()`, which re-creates the migration and executable with a `createMock(MigrateIdMapInterface::class)` ID map. Similarly, `GetTest`, `LogTest`, and `MachineNameTest` create local mocks inline for the specific test methods that assert exact call counts, rather than relying on the shared stub from the base class.

A small correctness fix was also included: `StubSourcePlugin::initializeIterator()` previously returned a bare `[]` array where the return type declares `\Iterator`; it now correctly returns `new \ArrayIterator([])`.

The PHPStan baseline file `core/.phpstan-baseline.php` had nine suppression entries for the `any()` deprecation across migrate test files; all nine are removed by this commit.

The `any()` usage in `MigrateSourceTest::testCount()` and `testCountCacheKey()` was explicitly excluded from this issue because those tests are structurally invalid; they were addressed separately in [#3572915](https://www.drupal.org/node/3572915). Residual notices caused by `UnitTestCase::getStringTranslationStub()` (which internally returns a mock despite its name) are tracked in [#3578904](https://www.drupal.org/node/3578904). Changes from [#3569415](https://www.drupal.org/node/3569415) (`MigrateProcessTestCase` stub conversion) were folded into this issue when that issue was closed as a duplicate.

The 11.x backport (commit `0b318c43`) was reverted (commit `e0001c7f`) the same day after a similar backport caused CI failures when running against PHPUnit 10; the changes therefore land only on `main`.
