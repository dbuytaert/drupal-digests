## Summary

Single Directory Components (SDC) now support font preloading via the `libraryOverrides` key in a component's definition. Before this fix, declaring a `fonts` entry in a component's `libraryOverrides` would not work correctly because the path translation step ignored font entries. Component authors can now add a `fonts` block with relative paths and `preload: true`, and Drupal will resolve those paths and emit the appropriate preload headers just as it does for fonts declared in regular `.libraries.yml` files.

## Impact

Affects module maintainers.

- **New API:** SDC `libraryOverrides` now accepts a `fonts` key with relative paths and `preload: true`, enabling font preloading from Single Directory Components.

## Technical details

`ComponentPluginManager::translateLibraryPaths()` was responsible for rewriting component-relative file paths to paths that the core library system can resolve. It handled `css` and `js` entries, but the `fonts` key introduced by [#3366561](https://www.drupal.org/node/3366561) was never included, so font paths stayed component-relative and never reached the preload pipeline.

The fix adds `fonts` to the same path-translation loop. The `fonts` key is unset from the passthrough overrides array, each font filename is checked for `UrlHelper::isExternal()`, relative filenames are expanded to an absolute path and then converted to a library-root-relative path via `makePathRelativeToLibraryRoot()`, and the translated entry is written back into `$altered_overrides['fonts']`. External font URLs are passed through unchanged. A test component fixture (`lib-overrides`) was extended with a `test-component-font.woff2` entry (`preload: true`) and the functional test `testLibraryOverrides()` now asserts the font filename appears in the rendered page output.

## Upgrade

To preload a font from an SDC component, add a `fonts` block to the component's `libraryOverrides` in its component definition file:

```yaml
libraryOverrides:
  fonts:
    my-font.woff2:
      preload: true
```

The path is relative to the component directory. External URLs are also supported and passed through unchanged.
