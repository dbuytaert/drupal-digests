## Summary

This is an internal code-quality change with no user-facing effect. It resolves PHPStan level-6 errors in the link module's test suite by adding missing return types, narrowing generic `array` type hints to precise generic shapes, and adding typed parameter annotations. The companion fix to `PathAliasTestTrait` eliminated dozens of PHPStan suppression entries across many unrelated modules in the shared baseline file.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPStan was run at level 6 against `core/modules/link/tests` using the command `vendor/bin/phpstan analyze -c ./core/phpstan.neon.dist -l 6 -- core/modules/link/tests`. The errors fell into three categories, each addressed differently.

**Missing return types on data providers and helpers.** `LinkFieldUITest::providerTestFieldUI()` returned `\Generator` but had no declared return type; it was annotated accordingly. `LinkFieldFormStatesTest::linkFieldFormStatesData()`, `LinkExternalProtocolsConstraintValidatorTest::providerValidate()`, and the return-type annotation on `LinkAccessConstraintValidatorTest::providerValidate()` (which yields, so `\Generator`) were all corrected. `assertPathAliasExists()` and `assertPathAliasNotExists()` in the shared trait `core/tests/Drupal/Tests/Traits/Core/PathAliasTestTrait.php` had no return type; adding `: void` to both methods cascaded across the entire codebase, removing over two dozen `missingType.return` suppression entries from `core/.phpstan-baseline.php` for modules including filter, locale, menu_link_content, mysql, path, path_alias, shortcut, views, and core functional and kernel tests.

**Imprecise generic array types in PHPDoc.** Several `@param array` and `@return array` annotations were narrowed to typed generics: `@param array<string, string>` for `assertValidEntries()` and `assertInvalidEntries()` in `LinkFieldTest.php`; `@return array<int, array<string, string>>` in `LinkInputValuesTraits::getLinkInputValues()`; `@return list<list{string, list<string>}>` in `LinkItemUrlValidationTest`; `@var array<string>` for `$testViews` in `LinkViewsTokensTest`; and `@param array<string, mixed>` pairs in `AttributeXssTest::testSanitizeAttributes()`.

**PHPStan blind spots on untyped parameters and entity variables.** `LinkExternalProtocolsConstraintValidatorTest::testValidate()` had untyped parameters `$url` and `$valid`; they were declared `string` and `bool` respectively. In `LinkFormatterTest::testLinkFormatter()`, the result of `EntityStorageInterface::create()` was annotated with an inline `/** @var \Drupal\Core\Entity\ContentEntityInterface $entity */` comment so PHPStan could resolve calls on the returned object. A string-to-int coercion (`$id = $match[1]`) was also corrected to `$id = (int) $match[1]` in `LinkFieldTest.php`. The redundant custom docblock on `LinkFormatterTest::$modules` was replaced with `{@inheritdoc}`.
