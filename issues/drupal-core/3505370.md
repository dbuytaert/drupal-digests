## Summary

Drupal now deprecates the filter tips page that shows examples of HTML formatting. This page appeared at `/filter/tips` and linked from every text field by default. With modern rich text editors like CKEditor 5, these technical HTML examples are no longer relevant or helpful. Site administrators can add formatting guidance directly in field descriptions, which provides more flexibility and can include links to external resources. The filter tips page looked dated and had no way to hide it through the user interface.

## Impact

Affects module maintainers.

- **API change:** The `$long` parameter removed from `FilterInterface::tips()` and `FilterBase::tips()` method signatures
- **Deprecation:** The `/filter/tips` page, `filter.tips` and `filter.tips_all` routes, `FilterController::filterTips()`, the `$long` parameter to `FilterInterface::tips()` and `_filter_tips()`, and the 'About text formats' link on text fields are deprecated and will be removed in Drupal 12

## Technical details

The change deprecates the entire "long format" filter tips system while keeping the short format tips that appear inline on forms when multiple text formats are available. The `filter.tips` and `filter.tips_all` routes are deprecated in `filter.routing.yml` and will be removed in Drupal 12. The `FilterController::filterTips()` method is also deprecated.

The `$long` parameter has been removed from the `FilterInterface::tips()` and `FilterBase::tips()` method signatures. The `_filter_tips()` function no longer accepts a `$long` parameter. When legacy code calls these methods with the `$long` parameter, Drupal uses reflection to detect the parameter count and triggers deprecation errors while maintaining backward compatibility.

The "About text formats" help link has been removed from `TextFormat::processFormat()` in the `filter` element, so it no longer appears below text fields. The menu link definition for `filter.tips_all` has been removed from `filter.links.menu.yml`.

Tests that access the `/filter/tips` routes now use the `#[IgnoreDeprecations]` attribute. The `UserBlocksTest` was updated to use node paths instead of the deprecated `/filter/tips` path for testing login behavior.

## Upgrade

Custom filter plugins that implement `FilterInterface::tips()` should remove the `$long` parameter from their method signature:

```php
// Before:
public function tips($long = FALSE) {
  if ($long) {
    return $this->t('Detailed explanation...');
  }
  return $this->t('Short tip');
}

// After:
public function tips() {
  return $this->t('Short tip');
}
```

If you have custom code that calls `_filter_tips()` or `FilterInterface::tips()` with the `$long` parameter, remove that parameter from your calls. Only short tips are now used.

See the change record at [#3567879](https://www.drupal.org/node/3567879) for complete details.
