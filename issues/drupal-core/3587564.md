## Summary

A new `search_node` core submodule now owns all of the Search module integration that was previously bundled inside the Node module. Sites that use the built-in content search (the `node_search` search page) will need this new module enabled. An update hook handles the transition automatically on existing sites. This work is part of the broader effort to deprecate the Search module by extracting its integrations into dedicated submodules.

## Impact

Affects module maintainers.

- **Deprecation:** `\Drupal\node\Plugin\Search\NodeSearch` deprecated in drupal:11.4.0, removed in drupal:12.0.0; use `\Drupal\search_node\Plugin\Search\SearchNode` instead.
- **New API:** New `search_node` core submodule introduced, owning the `node_search` plugin, its hooks, config, and Views data integration.

## Technical details

The canonical `node_search` plugin class has moved from `core/modules/node/src/Plugin/Search/NodeSearch.php` (`\Drupal\node\Plugin\Search\NodeSearch`) to `core/modules/search/modules/search_node/src/Plugin/Search/SearchNode.php` (`\Drupal\search_node\Plugin\Search\SearchNode`). The old class is retained as a deprecated shell that simply extends the new one and fires `E_USER_DEPRECATED` in its constructor; the `#[Search(id: 'node_search')]` attribute now lives on `SearchNode`.

Several hooks previously implemented inside the Node module have been relocated. `NodeSearchHooks` (cron rankings aggregation) was re-namespaced into `Drupal\search_node\Hook`. A new `SearchNodeHooks` class implements `hook_entity_predelete` (clearing the search index when a node is deleted), `hook_node_update` (clearing removed translations from the index), `hook_entity_view_display_alter` (hiding field labels in the `search_index` view mode), and a `hook_form_node_preview_form_select_alter` to unset the `search_index` view mode option. The Views data for `node_search_index`, `node_search_total`, and `node_search_dataset` was extracted from `NodeViewsData::getViewsData()` into a new `SearchNodeViewsHooks::viewsDataAlter()` hook.

On the configuration side, the `search.page.node_search.yml` optional config file now declares `search_node` (not `node`) as its required module; the schema entry for `search.plugin.node_search` moved from `node.schema.yml` to a new `search_node.schema.yml`; and the `core.entity_view_mode.node.search_index` and `core.entity_view_mode.node.search_result` install configs moved from the Node module to `search_node`. The `Node` entity class itself was cleaned up: the `preDelete()` static method and the translation-removal call in `postSave()` were deleted, since those responsibilities now live in `SearchNodeHooks`.

## Upgrade

Code that subclasses or type-hints `\Drupal\node\Plugin\Search\NodeSearch` must be updated:

```php
// Before (deprecated in drupal:11.4.0, removed in drupal:12.0.0):
use Drupal\node\Plugin\Search\NodeSearch;

// After:
use Drupal\search_node\Plugin\Search\SearchNode;
```

The `search_node` module must be enabled wherever the `node_search` search page is in use. On existing sites an update hook handles this automatically, but custom install profiles or test environments that enable modules explicitly should add `search_node` to their module list.
