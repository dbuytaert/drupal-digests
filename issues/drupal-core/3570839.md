## Summary

Four underscore-prefixed functions in the media_library module have been moved into class-based methods and marked internal. In Drupal 11.4, the original functions are deprecated for removal in Drupal 12. In Drupal 12, they have been removed entirely. These functions were always intended as internal helpers for form callbacks and display configuration.

## Impact

Affects module maintainers.

- **Deprecation:** Four underscore functions in media_library deprecated in 11.4.0 and removed in 12.0.0 with no public replacement: `_media_library_views_form_media_library_after_build()`, `_media_library_media_type_form_submit()`, `_media_library_configure_form_display()`, and `_media_library_configure_view_display()`

## Technical details

The media_library module contained four underscore-prefixed functions that served as form callbacks and display configuration helpers. The refactoring moves these into two locations based on their purpose.

The two form callbacks (`_media_library_views_form_media_library_after_build` and `_media_library_media_type_form_submit`) moved to the existing `MediaLibraryHooks` class as public instance methods `viewsFormAfterBuild()` and `mediaTypeFormSubmit()`. They must be public because the form API requires callable callbacks, but they are marked `@internal` to signal they are not part of the public API. Call sites in `formAlter()` now reference them as `static::class . ':methodName'`.

The two display configuration functions (`_media_library_configure_form_display` and `_media_library_configure_view_display`) moved to a new `MediaLibraryDisplayManager` class as static methods `configureFormDisplay()` and `configureViewDisplay()`. The class is marked `@internal`. These methods are static so they can be called from both the hook class and the install hook without requiring dependency injection. The install hook now calls `MediaLibraryDisplayManager::configureFormDisplay($type)` instead of the underscore function.

In the 11.x branch, the original functions remain in `media_library.module` with `@deprecated` tags and `@trigger_error()` calls. The deprecation messages state there is no replacement, signaling these were always internal. In the main branch (12.x), the functions are removed entirely and `media_library.module` contains only the remaining public procedural functions.

## Upgrade

No action required. The deprecated functions were internal helpers and should not have been called by external code.
