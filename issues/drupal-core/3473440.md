## Summary

This meta-issue coordinated the removal of all Twig 3.x deprecation warnings that Drupal core was triggering. The warnings were introduced in twig/twig versions 3.9 through 3.12 and were temporarily suppressed in `core/.deprecation-ignore.txt`. Left unaddressed, they were producing over 190,000 false-positive deprecation errors across contrib modules in Upgrade Status tooling, which also blocked the project update bot from sending patches to those projects. Three child issues delivered all the actual fixes, and all twig/twig 3.11 and 3.12 suppression entries have been removed from `core/.deprecation-ignore.txt`. The changes are internal and have no functional impact on site behaviour beyond minor HTML whitespace differences in a few templates.

## Impact

Affects module maintainers and front-end developers.

- **Theme change:** Removal of `{% apply spaceless %}` from core templates changes HTML whitespace in some rendered output; custom templates overriding those files should verify rendering.
- **Deprecation:** Three classes of twig/twig 3.x deprecated API usage removed from Drupal core: `getNode('filter')` node accessor, standalone `twig_escape_filter()` function, and the `{% apply spaceless %}` filter.

## Technical details

The work was split across three child issues, each targeting a distinct class of deprecation.

**[#3474692](https://www.drupal.org/node/3474692) — `FilterExpression` node accessor (twig 3.11/3.12).** In twig/twig 3.11+ the `getNode('filter')` accessor on `FilterExpression` nodes was deprecated in favour of a `twig_callable` attribute that carries a proper `TwigFilter` object. Two files in `core/lib/Drupal/Core/Template/` were updated:
- `TwigNodeVisitor::leaveNode()`: the check `$node->getNode('filter')->getAttribute('value')` became `$node->getAttribute('twig_callable')->getName()`, and the escape-filter swap from `$node->getNode('filter')->setAttribute('value', 'drupal_escape')` became `$node->setAttribute('twig_callable', $env->getFilter('drupal_escape'))`, passing an actual `TwigFilter` object.
- `TwigNodeTrans::compileString()`: the same `getNode('filter')->getAttribute('value')` pattern was replaced with `getAttribute('twig_callable')->getName()` to inspect filter names while traversing the trans-string AST.

**[#3457168](https://www.drupal.org/node/3457168) — Removed `twig_escape_filter()` call (twig 3.9).** The standalone `twig_escape_filter()` function became internal to Twig and was removed from the public API. `TwigExtension::escapeFilter()` in `core/lib/Drupal/Core/Template/TwigExtension.php` was updated to replace `twig_escape_filter($env, $return, $strategy, $charset, $autoescape)` with `$env->getRuntime(EscaperRuntime::class)->escape($arg, $strategy, $charset, $autoescape)`, and `use Twig\Runtime\EscaperRuntime` was added. A `testRenderStrategies()` test was added to `TwigExtensionTest` covering both the `html` and `js` escape strategies.

**[#3486170](https://www.drupal.org/node/3486170) — Deprecated `spaceless` filter (twig 3.12).** The `{% apply spaceless %}` tag was deprecated in twig/twig 3.12 because whitespace collapsing belongs to the template author, not a filter. All uses in core templates were removed across modules (`block_content`, `ckeditor5`, `link`, `system`, `toolbar`) and themes (Olivero, Claro, Stable9, Starterkit, Umami). Where whitespace trimming was still desired, Twig whitespace-control markers (`{{- -}}`, `{%- -%}`) were used instead. Two files in the `default_admin` theme were not addressed and still carry `{% apply spaceless %}`.

After all three fixes, the `# Twig 3.` section of `core/.deprecation-ignore.txt` was fully vacated.

## Upgrade

Contrib modules or custom code that creates `Twig\Node\Expression\FilterExpression` nodes and reads the filter name via `getNode('filter')->getAttribute('value')` must switch to the `twig_callable` attribute:

```php
// Before (deprecated since twig/twig 3.12):
$name = $node->getNode('filter')->getAttribute('value');
$node->getNode('filter')->setAttribute('value', 'my_filter');

// After:
$name = $node->getAttribute('twig_callable')->getName();
$node->setAttribute('twig_callable', $env->getFilter('my_filter'));
```

Code that calls the removed `twig_escape_filter()` function should use `EscaperRuntime` instead:

```php
// Before (removed in twig/twig 3.9+):
use Twig\Environment;
return twig_escape_filter($env, $value, $strategy, $charset, $autoescape);

// After:
use Twig\Runtime\EscaperRuntime;
return $env->getRuntime(EscaperRuntime::class)->escape($value, $strategy, $charset, $autoescape);
```

Custom or contrib templates using `{% apply spaceless %}` should remove the tag and apply Twig whitespace-control markers (`{{- -}}`, `{%- -%}`) where trimming is needed, as the filter is deprecated in twig/twig 3.12.
