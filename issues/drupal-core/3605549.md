## Summary

Install profiles can now list recipes in their `.info.yml` file under a `recipes` key, and Drupal will apply those recipes automatically as part of the installation batch, immediately after modules are installed. Previously, a profile that wanted to apply a recipe had to use workarounds like Drupal CMS does. This change eliminates that need and lets profiles combine traditional module installation with recipe-based configuration in a single, declarative step.

## Impact

Affects module maintainers and site owners.

- **Config change:** Profile `.info.yml` files gain an optional `recipes` list key recognized by `install_profile_modules()`.
- **New API:** New `recipes` key supported in install profile `.info.yml` files; lists recipe directory paths to apply during installation.

## Technical details

The change is a small addition to `install_profile_modules()` in the installer. After the module-install batch is assembled, the function reads `$install_state['profile_info']['recipes']` and maps each path to a `Recipe` object using `Recipe::createFromDirectory(...)`. It then calls `RecipeRunner::toBatchOperations()` on each recipe and appends the resulting operations to the same `BatchBuilder`, so recipes run in the order listed, back-to-back with module installs.

Path resolution follows the existing convention: core recipe paths (e.g. `core/recipes/audio_media_type`) work as-is because `RecipeConfigurator` already has special handling for the `core/` prefix. Paths to contributed recipes require the caller to provide a fully-resolvable directory path; a dedicated follow-up ([#3612240](https://www.drupal.org/node/3612240)) will address resolving contributed recipe paths via Composer configuration. Recipe translation during install (downloading `.po` files before applying) is also deferred to a separate issue.

## Upgrade

To use this feature in a custom install profile, add a `recipes` key to the profile's `.info.yml`:

```yaml
name: My Profile
type: profile
install:
  - some_module
recipes:
  - core/recipes/audio_media_type
```

Recipes are applied in the listed order after module installation completes. Core recipe paths (prefixed with `core/`) resolve out of the box. Contributed recipe paths must currently be absolute or relative to the web root; a path-resolution follow-up is tracked in [#3612240](https://www.drupal.org/node/3612240).
