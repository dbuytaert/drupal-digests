## Summary

The Settings Tray module is being removed from Drupal core and moved to a contributed project. Settings Tray was added to core alongside Quick Edit to allow block configuration in an off-canvas sidebar without leaving the page, but it was never meaningfully expanded and Quick Edit itself was removed from core more than three years ago. The module depends on the Toolbar module, which is itself approved for removal, and it does not function correctly when the Navigation module is installed. With little usage on live sites and no active plans to extend its functionality, the community decided removal from core is the right path. The module will be maintained in contrib.

## Impact

No upgrade or configuration changes required.

## Technical details

Settings Tray (`settings_tray.module`) requires the Toolbar module as a dependency. Because Toolbar is approved for removal from core, Settings Tray either had to be updated to work with the Navigation module or removed. Investigation showed the module was broken even with both Toolbar and Navigation installed, opening the tray form as a full page instead of the off-canvas drawer. The module's functionality is narrow: for most blocks it only allows changing the title; for Views blocks it also exposes the items-per-page setting. The off-canvas rendering infrastructure was decoupled from Settings Tray long ago (see the `drupal.dialog.ajax` library), so Layout Builder's use of the off-canvas tray is unaffected. This issue is the policy decision; the concrete deprecation and removal steps are tracked in the meta issue [#3576667](https://www.drupal.org/node/3576667). A search of contributed modules showed very limited dependency on Settings Tray's API (some styling support in Gin, menu_block, and menu_block_override).
