## Summary

Drupal is cleaning up legacy code to prepare for removing `.module` files. Five internal helper functions in the Editor module that tracked file usage in text fields have been moved into a protected class method. These functions were never intended for external use and were marked with underscore prefixes to signal they were internal. The functions are deprecated in Drupal 11.4 and removed entirely in Drupal 12.0. Site editors and content creators will see no changes. The Editor module continues to track files embedded in formatted text exactly as before.

## Impact

Affects module maintainers.

- **Deprecation:** Five underscore-prefixed functions in `editor.module` are deprecated in Drupal 11.4 and removed in Drupal 12.0, with no replacement provided.

## Technical details

The Editor module provided five underscore-prefixed functions in `editor.module` to track file usage when images and other files are embedded in formatted text fields. According to Drupal's backward compatibility policy, underscore-prefixed functions are considered internal and not public API.

These functions have been moved to protected methods in the `EditorHooks` class: `_editor_record_file_usage()` becomes `EditorHooks::recordFileUsage()`, `_editor_delete_file_usage()` becomes `EditorHooks::deleteFileUsage()`, `_editor_get_file_uuids_by_field()` becomes `EditorHooks::getFileUuidsByField()`, `_editor_get_formatted_text_fields()` becomes `EditorHooks::getFormattedTextFields()`, and `_editor_parse_file_uuids()` becomes `EditorHooks::parseFileUuids()`.

In Drupal 11.4, the original functions remain but trigger deprecation notices referencing change record [#3568136](https://www.drupal.org/node/3568136). In Drupal 12.0, the functions are completely removed from `editor.module`. The protected methods in `EditorHooks` now handle all file usage tracking internally when entities are inserted, updated, or deleted.

The hook implementation methods (`entityInsert()`, `entityUpdate()`, `entityDelete()`, and `entityRevisionDelete()`) now call the new protected methods directly. Documentation examples in `entity.api.php` that used these functions have been replaced with generic examples.

## Upgrade

Contributed modules calling any of these functions should stop using them immediately. These were never public API and should not have been used outside the Editor module.

If your code calls `_editor_record_file_usage()`, `_editor_delete_file_usage()`, `_editor_get_file_uuids_by_field()`, `_editor_get_formatted_text_fields()`, or `_editor_parse_file_uuids()`, remove those calls. There is no direct replacement because these methods are now protected. If you need similar functionality, implement your own file tracking logic or file an issue to request a public API.

See [#3568136](https://www.drupal.org/node/3568136) for the change record.
