## Summary

Drupal 11.x now ships jQuery 4.0.0-beta instead of jQuery 3.7.1. The upgrade is internal and most sites will notice no difference, but contributed and custom code that calls jQuery APIs removed in version 4 — such as `$.isArray()`, `$.isWindow()`, and the `push`, `sort`, and `splice` properties on `$.fn` — will need to be updated. The migrate plugin was intentionally excluded because Drupal core was already on jQuery 3.7.x, which the jQuery team considers a safe upgrade baseline.

## Impact

Affects module maintainers and site owners.

- **Requirements:** Applies to 11.x only; the jQuery 3.7.x line remains in 10.x.
- **API change:** `$.isArray()`, `$.isWindow()`, and `$.fn.push`/`$.fn.sort`/`$.fn.splice` prototype properties were removed in jQuery 4; contrib and custom JavaScript using these APIs must be updated.

## Technical details

jQuery is the JavaScript DOM and Ajax utility library vendored in `core/assets/vendor/jquery/`. This issue upgraded the uncompressed file, its minified version, and its source map from 3.7.1 to 4.0.0-beta, and updated the version string in `core.libraries.yml`. The update was performed using the core vendor tooling (`yarn add -D jquery@4.0.0-beta && yarn vendor-update`) rather than manual file replacement.

**Selector engine.** jQuery 4 dissolves the former Sizzle bundled selector engine back into the main library. The internal `matchExpr` variable is split into a new `filterMatchExpr` (for ID, CLASS, TAG, ATTR, PSEUDO, CHILD token types) and a separate `matchExpr` that extends it with `needsContext`. The large `setDocument()` function no longer returns the document object and no longer runs dynamic feature tests; browser capability flags such as `support.getById`, `support.disconnectedMatch`, `support.scope`, `support.sortDetached`, and `support.sortStable` are gone entirely. IE/Edge detection is now done at parse time via `document.documentMode` into a top-level `isIE` constant. The `:has()` and IE `:enabled`/`:disabled` quirk detection that was previously inside `setDocument` is now evaluated once at module load. The `find.contains`, `find.attr`, and `find.error` sub-functions are removed; `jQuery.contains` is now a plain `jQuery.extend` property and `find.matchesSelector` is simplified to drop the IE 9 disconnected-node workaround.

**Removed APIs.** `$.isArray()` was removed upstream; Drupal's `states.js` had already been fixed in [#3419730](https://www.drupal.org/node/3419730). `$.isWindow()` had been used in the vendored `jquery-ui/ui/position.js` and was fixed in [#3420499](https://www.drupal.org/node/3420499). The internal `isFunction` helper variable was removed in favour of inline `typeof target !== "function"` checks. The `push`, `sort`, and `splice` properties on `$.fn.prototype` are no longer present.

**Module export.** The CommonJS factory pattern that conditionally exported a factory function for environments lacking a `window.document` (e.g., bare Node.js) was replaced with `module.exports = factory(global, true)`. The guard for a missing document is now an explicit `throw` near the top of the module body.

**Prerequisite and follow-on issues.** The jquery-form plugin was unmaintained and not jQuery 4-compatible; it was forked into core in [#3419734](https://www.drupal.org/node/3419734), resolving most functional JavaScript test failures. The `:tabbable` jQuery-UI pseudo-selector used in `dialog.js` was removed in [#3296098](https://www.drupal.org/node/3296098), which unblocked the Nightwatch test suite. A Nightwatch test that had verified the tabbable-shim integration through the dialog code path was removed because that code path no longer executes with jQuery 4. The `jquery.tabbable.shim` library itself was separately deprecated and queued for removal in [#3278625](https://www.drupal.org/node/3278625). Asset size budget numbers in the test suite were adjusted after [#3441844](https://www.drupal.org/node/3441844) landed.

## Upgrade

If your custom or contributed JavaScript calls any of the following removed jQuery 4 APIs, update as shown:

```js
// $.isArray() removed — use native Array.isArray()
- if ( $.isArray( myVar ) ) { ... }
+ if ( Array.isArray( myVar ) ) { ... }

// $.isWindow() removed — check directly
- if ( $.isWindow( obj ) ) { ... }
+ if ( obj != null && obj === obj.window ) { ... }

// $.fn.push / $.fn.sort / $.fn.splice removed from the jQuery prototype
// These were internal array methods; do not call them on jQuery objects.
```
