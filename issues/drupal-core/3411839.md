## Summary

Drupal 11.x now ships jQuery 4.0.0-beta in place of jQuery 3.7.1. The upgrade is limited to 11.x because the jQuery team considers a jQuery 3.5+ baseline safe to upgrade without the jquery-migrate compatibility plugin, and core was on 3.7.1. Most sites will notice no change, but contributed and custom JavaScript that calls APIs removed in jQuery 4, such as `$.isArray()`, `$.isWindow()`, or the `push`, `sort`, and `splice` properties on `$.fn`, will need to be updated. The jquery-migrate plugin was intentionally excluded.

## Impact

Affects module maintainers and site owners.

- **Requirements:** Applies to 11.x only; the jQuery 3.7.x line remains in 10.x.
- **API change:** `$.isArray()`, `$.isWindow()`, and `$.fn.push`/`$.fn.sort`/`$.fn.splice` prototype properties were removed in jQuery 4; contrib and custom JavaScript using these APIs must be updated.

## Technical details

The vendored files in `core/assets/vendor/jquery/` (uncompressed, minified, and source map) were replaced using core's vendor tooling (`yarn add -D jquery@4.0.0-beta && yarn vendor-update`), which also updated the version string in `core.libraries.yml`.

The most significant internal change is that jQuery 4 dissolves the former Sizzle bundled selector engine back into the main library. `setDocument()` no longer returns the document and no longer runs dynamic browser-capability tests at call time; the flags `support.getById`, `support.disconnectedMatch`, `support.scope`, `support.sortDetached`, and `support.sortStable` are gone entirely. IE detection moves to a top-level `isIE` constant (`document.documentMode`), and `:has()` quirk detection and `rbuggyQSA` construction now run once at module load. The former `matchExpr` object is split into `filterMatchExpr` (covering ID, CLASS, TAG, ATTR, PSEUDO, CHILD token types) and a `matchExpr` that extends it with `needsContext`. The sub-functions `find.contains`, `find.attr`, and `find.error` are removed; `jQuery.contains` becomes a plain `jQuery.extend` property. The internal `isFunction` helper is removed in favour of inline `typeof target !== "function"` checks.

The CommonJS module export is simplified: the conditional factory pattern that returned a factory function for Node.js environments without a `window.document` is replaced with `module.exports = factory(global, true)`, and an explicit `throw` near the top of the module body guards against a missing document.

The `push`, `sort`, and `splice` properties are removed from `jQuery.fn.prototype`. Prerequisite issues that had to land first include the fix for `$.isArray` in `states.js` ([#3419730](https://www.drupal.org/node/3419730)), the fork of the unmaintained jquery-form plugin into core ([#3419734](https://www.drupal.org/node/3419734)), the removal of the `:tabbable` jQuery-UI pseudo-selector from `dialog.js` ([#3296098](https://www.drupal.org/node/3296098)), and the asset-size budget adjustments ([#3441844](https://www.drupal.org/node/3441844)). A Nightwatch test for the tabbable-shim dialog integration was removed because that code path is no longer reached with jQuery 4.

## Upgrade

If your custom or contributed JavaScript calls any of the following removed jQuery 4 APIs, update as shown:

```js
// $.isArray() removed — use native Array.isArray()
- if ( $.isArray( myVar ) ) { ... }
+ if ( Array.isArray( myVar ) ) { ... }

// $.isWindow() removed — check directly
- if ( $.isWindow( obj ) ) { ... }
+ if ( obj != null && obj === obj.window ) { ... }

// $.fn.push / $.fn.sort / $.fn.splice removed from the jQuery prototype.
// These were internal array methods; do not call them on jQuery objects.
```
