## Summary

Sites using Symfony Mailer now get stricter validation on the `system.mail` mailer_dsn configuration. The `scheme`, `host`, and `port` fields are validated against RFC 3986 and valid port ranges, while the `options` mapping is validated per transport type (smtp, smtps, sendmail, null, native). The `mailer_dsn` schema type is now a reusable core data type, so contrib and custom modules that define their own transport config entities can reference it directly instead of duplicating the schema.

## Impact

Affects module maintainers and site owners.

- **Config change:** `system.mail` config now uses the new `mailer_dsn` schema type and has the `FullyValidatable` constraint applied, so stricter validation runs on save.
- **New API:** New reusable `mailer_dsn` config schema type in `core.data_types.schema.yml` and new `UriHost` validation constraint plugin, both available to contrib and custom modules.

## Technical details

This issue builds on [#3399645](https://www.drupal.org/node/3399645), which introduced a structured `mailer_dsn` mapping in `system.schema.yml`. The mapping was inline and had only minimal constraints, so this issue tightens it and extracts it as a reusable core data type.

**Reusable `mailer_dsn` core data type (`core/config/schema/core.data_types.schema.yml`)**

The inline mapping from `system.schema.yml` was moved to `core.data_types.schema.yml` as `mailer_dsn` with the following field constraints:

- `scheme`: `NotBlank` plus a `Regex` constraint enforcing the RFC 3986 ABNF (`/^[a-z][a-z0-9+\-\.]*$/i`), so schemes like `smtp`, `sendmail+smtp`, and `drupal.test-capture` are valid but `_underscore` is not.
- `host`: `NotBlank` plus a new `UriHost` constraint (described below) that accepts hostnames, IPv4 addresses, and bracket-enclosed IPv6 literals such as `[::1]`.
- `user` and `password`: nullable strings with no constraints. Per discussion, some mailer APIs use basic auth in unusual ways (e.g., the entire API key in the user field with a blank password), so `NotBlank` was deliberately not applied.
- `port`: nullable integer restricted to 0-65535 via `Range`.
- `options`: typed dynamically by scheme via the `mailer_dsn.options.[%parent.scheme]` interpolation key.

**Scheme-specific option sub-types** are also defined in `core.data_types.schema.yml`:

- `mailer_dsn.options.*`: fallback sequence accepting any non-null string values for unknown schemes.
- `mailer_dsn.options.null` and `mailer_dsn.options.native`: empty mappings (no options allowed).
- `mailer_dsn.options.sendmail` (also aliased as `mailer_dsn.options.sendmail+smtp`): a `command` field validated with a Unicode control-character regex (`/([^\PC])/u` with `match: false`) drawn from the same pattern used by the `label` data type (see [#3379102](https://www.drupal.org/node/3379102)).
- `mailer_dsn.options.smtp` and `mailer_dsn.options.smtps`: a structured mapping with `verify_peer` (boolean), `peer_fingerprint` (hex-only regex to enforce colon-stripped fingerprints), `local_domain` (control-character regex), and `Range`-constrained integers for `restart_threshold`, `restart_threshold_sleep`, `ping_threshold`, and `max_per_second`.

**New `UriHost` constraint (`core/lib/Drupal/Core/Validation/Plugin/Validation/Constraint/`)**

`UriHostConstraint` and `UriHostConstraintValidator` are added as a new Drupal validation plugin with id `UriHost`. The validator uses `filter_var($value, FILTER_VALIDATE_DOMAIN, FILTER_FLAG_HOSTNAME)` to accept valid hostnames and IPv4 addresses (PHP's domain filter also accepts valid IPv4), and separately handles IPv6 literals by stripping the enclosing brackets and calling `filter_var($address, FILTER_VALIDATE_IP, FILTER_FLAG_IPV6)`. Symfony's built-in `Hostname` constraint was considered but rejected because it does not handle bracket-enclosed IPv6 literals as required by RFC 3986 section 3.2.2.

**`system.mail` schema (`core/modules/system/config/schema/system.schema.yml`)**

The inline `mailer_dsn` mapping was replaced with `type: mailer_dsn` to use the new core data type. The `FullyValidatable` constraint was also added to the `system.mail` config object so that the full validation graph is exercised during config validation.

**Tests**

A new kernel test `MailerDsnConfigValidationTest` covers each field and each scheme-specific option type. `ConstraintsTest` gained a `testUriHost` method covering hostnames, IPv4, and IPv6 literals. `TypedConfigTest` was updated to include the new `host` field used in the test config fixture.
