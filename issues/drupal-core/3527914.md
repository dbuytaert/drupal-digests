# #3527914: Use UMD installation method for CKEditor5

## Summary

CKEditor 5's DLL (Dynamic Link Library) build method is being discontinued: CKEditor 5 v48 already removed DLL support from its published npm packages, making an upgrade impossible without switching build methods. Drupal 12 now switches to the UMD (Universal Module Definition) build method, which ships all CKEditor 5 functionality as a single aggregated file loaded into a `CKEDITOR` global, along with consolidated per-language translation files. This change lands in Drupal 12 only; Drupal 11 retains the DLL build because CKSource confirmed continued LTS support for the version bundled with Drupal 11 through its end-of-life. Contrib modules that ship CKEditor 5 plugins will need a new major version to support Drupal 12, but a backward-compatibility shim eases the migration.

## Impact

Affects module maintainers.

- **API change:** Contrib CKEditor 5 plugins must replace webpack `DllReferencePlugin` with `externals` mapping `ckeditor5`/`@ckeditor/*` imports to the `CKEDITOR` UMD global; per-plugin library declarations should depend on `core/ckeditor5` instead of the deprecated stub libraries.
- **Deprecation:** Per-plugin stub libraries (`core/ckeditor5.autoformat`, `core/ckeditor5.basic`, and ~23 others) deprecated in drupal:12.0.0, removed in drupal:13.0.0; `ckeditor5-dll-shim.js` (and `CKEditor5.dll()`) deprecated in drupal:12.0.0, removed in drupal:13.0.0.

## Technical details

The DLL build exposed each plugin as a separate JavaScript file that delegated to `CKEditor5.dll` via webpack's `DllReferencePlugin`. The UMD build replaces all those individual files with a single `core/assets/vendor/ckeditor5/ckeditor5.umd.js` that exports everything through the `CKEDITOR` global. Translation files similarly consolidated from one file per plugin per language to one file per language under `core/assets/vendor/ckeditor5/translations/*.umd.js`.

On the Drupal side, `core.libraries.yml` gains the top-level `ckeditor5` library entry pointing at `ckeditor5.umd.js`. The dozens of old per-plugin stub libraries (`core/ckeditor5.autoformat`, `core/ckeditor5.basic`, `core/ckeditor5.essentials`, etc.) are deprecated in drupal:12.0.0 and removed in drupal:13.0.0; they now simply redirect to `core/ckeditor5`. `Ckeditor5Hooks.php` and `LanguageMapper.php` already looked for `.umd.js` files when scanning for translation assets, wiring the new consolidated translation files into the library system.

Drupal's own internal plugins under `js/ckeditor5_plugins/` are still built separately with webpack, but `webpack.config.js` now uses an `externals` callback that maps all `ckeditor5` and `@ckeditor/*` imports to the `CKEDITOR` global instead of the old `DllReferencePlugin` approach.

For backward compatibility, a new `js/ckeditor5-dll-shim.js` is included in the `internal.drupal.ckeditor5` library. It sets `window.CKEditor5.dll` to a function that returns the `CKEDITOR` global (since the UMD build is flat), but fires a `Drupal.deprecationError()` on every invocation. This lets many existing DLL-built contrib plugins continue to load in Drupal 12 while triggering console deprecation notices. The shim is deprecated in drupal:12.0.0 and removed in drupal:13.0.0.

The decision to use UMD rather than ESM with import maps (which had been the originally planned path, blocked on [#3398525](https://www.drupal.org/node/3398525)) was made because UMD requires no additional platform API work and the DLL EOL deadline had become urgent after CKEditor 5 v48 removed DLL support entirely.

## Upgrade

Contrib modules that ship CKEditor 5 plugins built with webpack's `DllReferencePlugin` need to switch to webpack `externals` for Drupal 12 compatibility. Replace the `DllReferencePlugin` configuration with an `externals` entry that maps `ckeditor5` and `@ckeditor/*` imports to the `CKEDITOR` global:

```js
// webpack.config.js (Drupal 12 / UMD approach)
externals: [
  function ({ request }, callback) {
    if (request === 'ckeditor5' || request.startsWith('ckeditor5/')) {
      return callback(null, 'CKEDITOR');
    }
    if (request.startsWith('@ckeditor/')) {
      return callback(null, 'CKEDITOR');
    }
    callback();
  },
],
```

Plugin source files can keep their existing `import` statements from `ckeditor5` or `@ckeditor/*` packages; only the webpack build configuration changes. Libraries that previously declared a dependency on a now-deprecated stub library (e.g. `core/ckeditor5.basic`) should be updated to depend on `core/ckeditor5` directly. See the change record at https://www.drupal.org/node/3581531 for full details.
