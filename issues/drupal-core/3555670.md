## Summary

The procedural functions `user_load_by_mail()` and `user_load_by_name()` are deprecated as part of efforts to clean up `user.module`. Both functions were thin wrappers around entity storage methods and can be replaced directly with `loadByProperties()`.

## Impact

Affects module maintainers.

- **Deprecation:** `user_load_by_mail()` and `user_load_by_name()` deprecated in 11.4.0, removed in 13.0.0

## Technical details

Both `user_load_by_mail()` and `user_load_by_name()` were simple wrappers that called `\Drupal::entityTypeManager()->getStorage('user')->loadByProperties()` with either `['name' => $name]` or `['mail' => $mail]` and returned `reset($users)` or FALSE. The deprecation updates all core usages to call the entity storage directly.

In `update.fetch.inc`, the change also optimizes `_update_cron_notify()` by loading all users at once with `loadByProperties(['mail' => $notify_list])` and indexing them by email, rather than loading each user individually in a loop with `user_load_by_mail()`.

## Upgrade

```php
// Before
$user = user_load_by_name($name);

// After  
$users = \Drupal::entityTypeManager()
  ->getStorage('user')
  ->loadByProperties(['name' => $name]);
$user = reset($users);

// For user_load_by_mail():
$users = \Drupal::entityTypeManager()
  ->getStorage('user')
  ->loadByProperties(['mail' => $mail]);
$user = reset($users);
```
