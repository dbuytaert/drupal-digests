Drupal core cleaned up deprecated code related to file handling that was scheduled for removal. This maintenance work removes old code paths and helper functions that modules and sites should have already migrated away from in earlier versions. The change reduces technical debt and simplifies the file handling codebase without affecting external code that has already been updated to use current APIs.

## Technical details

This issue removed deprecated code in the File component and module that was marked for removal in Drupal 12. The cleanup includes:

Removed deprecated integer constants for file exists behavior (`FileSystemInterface::EXISTS_RENAME`, `EXISTS_REPLACE`, `EXISTS_ERROR`) and the backward compatibility method `FileExists::fromLegacyInt()`. All file operations now require the `FileExists` enum.

Removed the `LegacyMimeTypeMapLoadedSubscriber` event subscriber and the deprecated `hook_file_mimetype_mapping_alter()`. MIME type customization now requires implementing an event listener for `MimeTypeMapLoadedEvent`.

Removed deprecated methods from `ExtensionMimeTypeGuesser` (`setMapping()`, constructor accepting `ModuleHandlerInterface`) and `MimeTypeMap` (`getMapping()`, `setMapping()`). Removed legacy mapping arrays from `ExtensionMimeTypeGuesser`.

Removed deprecated functions from `file.module`: `file_get_content_headers()`, `file_managed_file_submit()`, `file_system_settings_submit()`, and template preprocess functions now registered directly in hook_theme.

Cleaned up deprecated service properties using `DeprecatedServicePropertyTrait` from `FileSystem` and `ExtensionMimeTypeGuesser`. Removed the `Drupal\file\Upload\FileUploadHandler` service alias.

Removed all type hints that allowed legacy integer values for `FileExists` parameters across `FileSystemInterface`, `FileSystem`, `FileRepository`, `FileUploadHandler`, and migrate's `FileCopy` process plugin.
