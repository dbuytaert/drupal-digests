## Summary

Drupal 12 removed all deprecated code in the File component and module that had been marked for removal since Drupal 10.3 and 11.2/11.3. Custom modules that still use the old integer constants for file-exists behavior, the deprecated MIME-type hook, or the procedural file functions must migrate before upgrading. The codebase is now smaller and strictly typed in these areas.

## Impact

Affects module maintainers.

- **API change:** Removed FileSystemInterface::EXISTS_RENAME/EXISTS_REPLACE/EXISTS_ERROR constants, FileExists::fromLegacyInt(), hook_file_mimetype_mapping_alter(), ExtensionMimeTypeGuesser::setMapping(), MimeTypeMap::getMapping()/setMapping(), Drupal\file\Upload\FileUploadHandler service alias, and several procedural file.module functions; all FileExists parameters are now strictly typed as the FileExists enum.

## Technical details

Three clusters of deprecated code were removed.

**File-exists integer API.** `FileSystemInterface::EXISTS_RENAME`, `EXISTS_REPLACE`, and `EXISTS_ERROR` constants are gone. `FileExists::fromLegacyInt()` is gone. Every method that previously accepted `FileExists|int` now declares a strict `FileExists` enum type: `FileSystemInterface::copy()`, `move()`, `saveData()`, `getDestinationFilename()`, `FileRepository::writeData()`, `copy()`, `move()`, `FileRepositoryInterface` counterparts, and the internal `file_save_upload()` and `_file_save_upload_from_form()` functions in `file.module`. The `FileSystem` class also lost its `DeprecatedServicePropertyTrait` usage and the associated `$deprecatedProperties` array.

**MIME type legacy layer.** `LegacyMimeTypeMapLoadedSubscriber` (the event subscriber that bridged `hook_file_mimetype_mapping_alter()` to `MimeTypeMapLoadedEvent`) is deleted, as is `hook_file_mimetype_mapping_alter()` itself from `hooks.file.inc`. `ExtensionMimeTypeGuesser` lost its entire legacy body: the `$defaultMapping` array (~900 lines), the deprecated `$mapping` property, `setMapping()`, the old constructor accepting `ModuleHandlerInterface`, and the `DeprecatedServicePropertyTrait` plumbing. `MimeTypeMap::getMapping()` and `MimeTypeMap::setMapping()` are also gone. The `Drupal\file\Upload\FileUploadHandler` deprecated service alias is removed; callers must use `FileUploadHandlerInterface`.

**Procedural file.module functions.** `file_get_content_headers()`, `file_managed_file_submit()`, `file_system_settings_submit()`, `template_preprocess_file_managed_file()`, `template_preprocess_file_link()`, `template_preprocess_file_widget_multiple()`, and `template_preprocess_file_upload_help()` are all deleted. The `drupal.file` library entry for the old `file/file.js` moved-files record is also removed. `HtaccessWriter::__construct()` now requires a non-nullable `Settings` argument instead of an optional one with a service-locator fallback.

## Upgrade

Replace removed integer constants with the `FileExists` enum:

```php
// Before (removed)
use Drupal\Core\File\FileSystemInterface;
$fileSystem->copy($source, $dest, FileSystemInterface::EXISTS_RENAME);
$fileSystem->copy($source, $dest, FileSystemInterface::EXISTS_REPLACE);
$fileSystem->copy($source, $dest, FileSystemInterface::EXISTS_ERROR);

// After
use Drupal\Core\File\FileExists;
$fileSystem->copy($source, $dest, FileExists::Rename);
$fileSystem->copy($source, $dest, FileExists::Replace);
$fileSystem->copy($source, $dest, FileExists::Error);
```

Replace `hook_file_mimetype_mapping_alter()` with a `MimeTypeMapLoadedEvent` subscriber:

```php
// Before (removed hook)
function mymodule_file_mimetype_mapping_alter(&$mapping) {
  $mapping['mimetypes']['my_type'] = 'application/x-myformat';
  $mapping['extensions']['myext'] = 'my_type';
}

// After: tagged service implementing EventSubscriberInterface
use Drupal\Core\File\Event\MimeTypeMapLoadedEvent;
public function onMimeTypeMapLoaded(MimeTypeMapLoadedEvent $event): void {
  $event->map->addMapping('application/x-myformat', 'myext');
}
public static function getSubscribedEvents(): array {
  return [MimeTypeMapLoadedEvent::class => 'onMimeTypeMapLoaded'];
}
```

Replace removed procedural functions:

```php
// file_get_content_headers($file)  →  $file->getDownloadHeaders()
// file_managed_file_submit()  →  ManagedFile::submit()
// file_system_settings_submit()  →  FileHooks::settingsSubmit()
// template_preprocess_file_managed_file/link/widget_multiple/upload_help
//   → registered directly in hook_theme(); remove standalone calls
```

Replace the deprecated service alias:

```php
// Before (removed alias)
\Drupal::service('Drupal\file\Upload\FileUploadHandler');

// After
\Drupal::service(\Drupal\file\Upload\FileUploadHandlerInterface::class);
```
