## Summary

The Navigation module's mobile sidebar (below 1024px) had cramped spacing around the logo, close button, and back button in the toolbar header area. Hover and focus ring styles on those buttons were visually clipped because there was insufficient clearance. The fix adjusts the header height and padding in the Navigation module's CSS so the top area matches desktop proportions on mobile viewports, giving focus styles room to display properly.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue was entirely within `core/modules/navigation/css/components/admin-toolbar.css` (and its PostCSS source `admin-toolbar.pcss.css`). Three CSS changes were made together:

1. **Header height variable increased.** The `--admin-toolbar-sidebar-header` custom property value was raised from `calc(3.5 * var(--admin-toolbar-rem))` to `calc(3.75 * var(--admin-toolbar-rem))`. This property drives the negative `margin-block-start` used to pull the header content up, so increasing it slightly expands the available height for the header row at the mobile breakpoint.

2. **Bottom padding added to the header.** `padding-block-end: var(--admin-toolbar-space-4)` was added to the `.admin-toolbar__header` block. This provides the clearance below the logo/close/back-button row that was previously missing, preventing focus and hover ring styles from being clipped by the element below.

3. **Incorrect negative margin hack removed.** A `& + .toolbar-block` rule that pulled the adjacent block up by `calc(var(--admin-toolbar-space-10) * -1)` in mobile (with a `margin-block-start: 0` reset at the desktop breakpoint) was deleted. As finnsky noted in the review, the 10px value was already wrong (should have been 12px), and after the two fixes above it became unnecessary. The corresponding `padding-block-end: 0` override in the desktop media query was also removed.

The net result is that spacing between the logo and the navigation buttons is now consistently 16px at both breakpoints, matching the desktop layout.
