## Summary

Drupal is eliminating procedural code from module files. This change moves the last procedural functions from the CKEditor 5 module into object-oriented methods on a hooks service class. The functions remain callable for now but will trigger deprecation warnings. This change only affects contrib modules that directly call these functions, which is rare. Most sites will see no impact.

## Impact

Affects module maintainers.

- **Deprecation:** Three `ckeditor5.module` procedural functions deprecated: `ckeditor5_filter_format_edit_form_submit()`, `_update_ckeditor5_html_filter()`, and `_ckeditor5_theme_css()` moved to `Ckeditor5Hooks` service.

## Technical details

The change moves three procedural functions from `ckeditor5.module` to methods on the `Ckeditor5Hooks` service class:

`ckeditor5_filter_format_edit_form_submit()` becomes `Ckeditor5Hooks::filterFormatEditFormSubmit()`. This form submission handler updates the allowed HTML tags when manually editable tags or styles are configured.

`_update_ckeditor5_html_filter()` becomes `Ckeditor5Hooks::updateCkeditor5HtmlFilter()`. This AJAX callback handler updates multiple areas of the text format form when switching editors or changing CKEditor 5 settings.

`_ckeditor5_theme_css()` becomes `Ckeditor5Hooks::themeCss()` and is marked protected with no replacement. This internal function retrieves theme stylesheets for use within CKEditor 5.

The deprecated procedural functions now call the new methods via dependency injection. All internal references (form callbacks, AJAX callbacks, and hook implementations) updated to use the class methods directly. The `Ckeditor5Hooks` constructor gains four new dependencies: `RendererInterface`, `ConfigFactoryInterface`, `ThemeExtensionList`, and `MessengerInterface` to support the moved logic.

This change is part of the broader initiative to eliminate `.module` files from Drupal core.

## Upgrade

Only contrib modules that directly call the deprecated functions need to update:

If you call `ckeditor5_filter_format_edit_form_submit()`, inject the `Ckeditor5Hooks` service and call `filterFormatEditFormSubmit()` instead.

If you call `_update_ckeditor5_html_filter()` as a form callback, change the callback reference to `Ckeditor5Hooks::updateCkeditor5HtmlFilter`.

If you call `_ckeditor5_theme_css()`, this was an internal function with no direct replacement. The CKEditor Premium module is known to use it and will need an alternative approach.

See the change record at [#3570104](https://www.drupal.org/node/3570104) for details.
