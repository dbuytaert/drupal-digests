## Summary

Drupal 12 removes a set of theme system features that were deprecated across Drupal 11.1 through 11.3. Procedural `template_preprocess_*()` functions are gone, the theme registry no longer auto-discovers them, and the `file` and `includes` keys in `hook_theme()` no longer work. Theme engine preprocess callbacks are also removed. Custom modules or themes that relied on any of these patterns must migrate to the `initial preprocess` callback key in `hook_theme()` before deploying Drupal 12. Contributed themes including Bootstrap are known to be affected.

## Impact

Affects module maintainers.

- **API change:** `Registry::__construct()` signature changed: `$root`, `$theme_name`, and `$theme_initialization` parameters removed; remaining parameters use constructor property promotion. The `theme.registry` service definition is updated accordingly.
- **Deprecation:** Procedural `template_preprocess_*()` functions, `file`/`includes` keys in `hook_theme()`, theme engine preprocess callbacks, and `drupal_common_theme()` are all removed after being deprecated in Drupal 11.1 through 11.3.

## Technical details

The `Registry` class constructor is simplified and modernized using PHP constructor property promotion. The `$root` (app root) argument, the optional `$theme_name` argument, and the `$theme_initialization` argument are all removed from `Registry::__construct()`, and the corresponding `$initialized` flag (unused for over a decade) is deleted. The `theme.registry` service definition is updated to match: `%app.root%` and `@theme.initialization` are dropped, `~` (the theme name placeholder) is removed, and `@keyvalue` becomes required. The `init()` method is simplified to a single line calling `$this->themeManager->getActiveTheme()`.

`Registry::processExtension()` no longer auto-discovers `template_preprocess_{hook}()` functions. The code path that checked `function_exists('template_preprocess_' . $hook)` and pushed the function name into `preprocess functions` is removed entirely. Along with it, the handling for `file` and `includes` in theme hook definitions is gone from both `processExtension()` and `ThemeManager::render()`. Theme engine preprocess callbacks following the `{engine_name}_engine_preprocess_{hook}()` naming pattern are also no longer registered.

The procedural wrappers deleted from `core/includes/theme.inc` include `template_preprocess()`, `template_preprocess_html()`, `template_preprocess_page()`, `template_preprocess_links()`, `template_preprocess_image()`, `template_preprocess_table()`, `template_preprocess_item_list()`, `template_preprocess_container()`, `template_preprocess_breadcrumb()`, `template_preprocess_pager()`, `template_preprocess_region()`, `template_preprocess_maintenance_page()`, `template_preprocess_install_page()`, `template_preprocess_maintenance_task_list()`, `template_preprocess_tablesort_indicator()`, `template_preprocess_menu_local_task()`, `template_preprocess_menu_local_action()`, `template_preprocess_time()`, `template_preprocess_datetime_form()`, `template_preprocess_datetime_wrapper()`, `template_preprocess_field()`, `template_preprocess_field_multiple_value_form()`, `_field_multiple_value_form_sort_helper()`, and `drupal_common_theme()`. Tests that previously instantiated separate `Registry` objects per theme now switch the active theme via `ThemeManager::setActiveTheme()` on the shared service instead.

## Upgrade

**If you extended `Registry`**, update the constructor. The old signature began with `($root, $cache, $lock, $module_handler, $theme_handler, $theme_initialization, $runtime_cache, $module_list, $kernel, $theme_name = NULL, $key_value_factory = NULL)`. The new signature is:

```php
public function __construct(
  protected CacheBackendInterface $cache,
  protected LockBackendInterface $lock,
  protected ModuleHandlerInterface $moduleHandler,
  protected ThemeHandlerInterface $themeHandler,
  protected CacheBackendInterface $runtimeCache,
  protected ModuleExtensionList $moduleList,
  protected HttpKernelInterface $kernel,
  protected KeyValueFactoryInterface $keyValueFactory,
)
```

**If you defined procedural `template_preprocess_*()` functions**, register them as `initial preprocess` callbacks in `hook_theme()` instead:

```php
function mymodule_theme(): array {
  return [
    'my_template' => [
      'variables' => ['content' => NULL],
      'initial preprocess' => [MyClass::class, 'preprocessMyTemplate'],
    ],
  ];
}
```

**If you used `file` or `includes` in `hook_theme()` definitions**, remove those keys and move preprocess logic into autoloaded classes.

**If you maintained a theme engine** with `{engine_name}_engine_preprocess_{hook}()` callbacks, those functions will no longer be called. There is no replacement; migrate this logic into theme or module preprocess hooks.
