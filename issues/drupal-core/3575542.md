## Summary

Drupal 12 removes several theme system features that were deprecated in earlier versions. The theme registry no longer discovers `template_preprocess_*()` functions automatically, and the `file` and `includes` properties in `hook_theme()` definitions are removed. Theme engines can no longer register preprocess functions. Many deprecated preprocess functions from `core/includes/theme.inc` are deleted entirely.

## Impact

No upgrade or configuration changes required.

## Technical details

This removal targets multiple deprecation notices introduced between Drupal 11.1 and 11.3. The theme registry (`Registry` class) no longer auto-discovers procedural `template_preprocess_*()` functions. Modules must now register preprocess callbacks explicitly using the `initial preprocess` key in `hook_theme()` definitions.

The `file` and `includes` properties in `hook_theme()` are removed entirely. These allowed modules to specify include files containing preprocess functions. All preprocess logic must now be in autoloaded classes.

Theme engines previously could register preprocess callbacks using naming patterns like `{engine_name}_engine_preprocess_{hook}()`. This capability is removed with no replacement.

The `Registry` and `ThemeManager` constructors are simplified. The `$root` parameter is removed from `Registry`, and the optional `$theme_name` parameter is eliminated. Tests that needed to work with multiple themes now switch the active theme via the theme manager instead of instantiating separate registry instances.

Many procedural preprocess functions from `theme.inc` are removed: `template_preprocess()`, `template_preprocess_html()`, `template_preprocess_page()`, `template_preprocess_links()`, `template_preprocess_image()`, `template_preprocess_table()`, and numerous others. These functions now exist only as service methods registered directly in `hook_theme()` definitions.

## Upgrade

If you have custom modules or themes extending `Registry` or `ThemeManager`, update constructor signatures. The `$root` parameter is removed from `Registry` as the first argument. The optional `$theme_name` parameter is also removed from `Registry`.

If you defined `template_preprocess_*()` functions, register them explicitly in your `hook_theme()` implementation using the `initial preprocess` key:

```php
function mymodule_theme(): array {
  return [
    'my_template' => [
      'variables' => ['content' => NULL],
      'initial preprocess' => [MyClass::class, 'preprocessMyTemplate'],
    ],
  ];
}
```

Remove any `file` or `includes` keys from your `hook_theme()` definitions. Move preprocess functions into autoloaded classes.

If you maintained a theme engine with preprocess functions, those functions will no longer be invoked. Migrate this logic into theme preprocess hooks or contributed module code.
