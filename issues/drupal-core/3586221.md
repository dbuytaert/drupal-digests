## Summary

The Stable 9 theme has been removed from Drupal core as of the `main` branch. If your site runs a custom or contributed theme that declares `base theme: stable9`, that theme will break after upgrading unless you take action. The theme continues to exist as a contributed project on Drupal.org, so the quickest path is to install that contrib package and leave your theme's `.info.yml` unchanged. Alternatively, you can switch to a different base theme such as Stable or Stark, or remove the base theme declaration entirely if your theme provides all its own markup.

## Impact

Affects front-end developers.

- **Theme change:** The `stable9` base theme is no longer shipped with core; themes declaring `base theme: stable9` must install the contrib Stable 9 package or switch base themes.

## Technical details

The entire `core/themes/stable9/` directory was deleted, including all Twig templates, CSS files, JavaScript, and test files. The machine name `stable9` was added to the `DRUPAL_CORE_REMOVED_THEME_LIST` constant in `core/modules/system/src/Install/Requirements/SystemRequirements.php`. That constant drives the site requirements check that surfaces a warning when a formerly-core theme is still referenced in a site's configuration, which gives site owners a clear signal at update time.

Tooling references were also cleaned up: a PHPStan baseline entry for `Stable9LibraryOverrideTest.php` was removed from `.phpstan-baseline.php`, a Prettier ignore rule and a Stylelint ignore rule for the theme's vendored CSS paths were removed from `.prettierignore` and `.stylelintrc.json`, the Stable9 maintainer entry was removed from `MAINTAINERS.txt`, and the word `necolas` (the author credit in stable9's vendored copy of `normalize.css`) was removed from `core/misc/cspell/dictionary.txt`.

Review by `dcam` flagged that the constant had moved from `system.install` to `SystemRequirements.php` since similar removals, and separately identified that `necolas` needed to be removed from the CSpell dictionary. Both were corrected before the final commit.

## Upgrade

If your custom theme's `.info.yml` currently reads:

```yaml
base theme: stable9
```

you have two options:

**Option A — Install the contrib Stable 9 theme (no `.info.yml` change needed):**
```
composer require drupal/stable9
drush en stable9
```

**Option B — Switch to a different base theme:**
```yaml
# Switch to core Stable (earlier compatibility layer):
base theme: stable

# Or remove the base theme declaration entirely if your theme is self-contained:
# (remove the base theme line)
```

Run `drush updatedb` after upgrading core so that the requirements check in `SystemRequirements` can surface any remaining references to the removed theme.
