## Summary

All references to the JSON:API specification website in Drupal core were updated to use the secure HTTPS protocol instead of HTTP. This change affects documentation links, code comments, error messages, and constant values throughout the JSON:API module. The update ensures that all links to external specification documentation use modern web security standards.

## Impact

No upgrade or configuration changes required.

## Technical details

This issue updated 38 instances of `http://jsonapi.org` URLs to use `https://jsonapi.org` across the JSON:API module. The changes were made systematically using a sed command to replace all occurrences: `git grep -l http://jsonapi.org | xargs sed -i 's#http://jsonapi.org#https://jsonapi.org#g'`.

Most changes appear in code comments and PHPDoc `@see` tags that reference the JSON:API specification. Three non-comment changes occur in production code: the `SUPPORTED_SPECIFICATION_PERMALINK` constant in `JsonApiSpec.php`, a Link header set in `JsonApiRequestValidator.php` for spec-violation error responses, and an expected error response URL in a functional test.

All URLs were verified to work correctly with the HTTPS protocol. This is part of a broader initiative to replace HTTP URLs with HTTPS URLs throughout Drupal core where the target site supports secure connections.
