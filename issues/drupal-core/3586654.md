## Summary

Sites that check for translation updates via the locale module will see a dramatic performance improvement. On a site with 200 modules and 38 languages, `locale_translation_check_projects_local()` dropped from around 5 minutes to roughly 7 seconds. The change removes an unnecessary file-system directory scan that was triggered for every project and language combination during a local `.po` file check.

## Impact

No upgrade or configuration changes required.

## Technical details

`LocaleSource::sourceCheckFile` previously located local `.po` files by calling `FileSystemInterface::scanDirectory()` with a regex-escaped filename pattern, even though the file's URI was already stored on `$source_file->uri`. This scan ran for every project-language pair, causing severe I/O and serialization overhead at scale.

The fix replaces the entire directory scan with a direct `file_exists($source_file->uri)` check. When the URI is present and the file exists, `filemtime()` and `hash_file()` are called on the known path. The `$source_file->uri` assignment that the old scan performed is now skipped because the URI is already populated before `sourceCheckFile` is called.

The scan was a historical artefact. As traced by comment archaeology against [#1804688](https://www.drupal.org/node/1804688) (where it was introduced) and [#1998056](https://www.drupal.org/node/1998056) (where cron-based auto-updates arrived), the original code tracked multiple versions of `.po` files, which made a scan meaningful. When [#1998056](https://www.drupal.org/node/1998056) shifted tracking to only the most recent download, the multi-version lookup rationale disappeared but the expensive scan was never removed. Because this functionality had effectively been absent since Drupal 8.8.0 with no reported breakage, removing the scan carries no compatibility risk. Broader optimisations to `locale_translation_get_status` and the KV store reads are tracked separately in [#3037156](https://www.drupal.org/node/3037156) and [#3590050](https://www.drupal.org/node/3590050).
