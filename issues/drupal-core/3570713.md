## Summary

Drupal core now requires PHPStan 2.1.38 or later for static analysis during development. This update fixes compatibility issues with newer PHP versions where PHPStan's older type stubs incorrectly flagged duplicate array keys in MySQL PDO connection options. The upgrade only affects developers running static analysis on Drupal core itself and does not change runtime behavior or affect production sites.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPStan version 2.1.38 includes updated type stubs from JetBrains/phpstorm-stubs that correctly handle PHP 8.4's PDO MySQL constants. Earlier versions incorrectly reported duplicate array keys for `\Pdo\Mysql::ATTR_USE_BUFFERED_QUERY`, `\Pdo\Mysql::ATTR_FOUND_ROWS`, and `\Pdo\Mysql::ATTR_MULTI_STATEMENTS` in `core/modules/mysql/src/Driver/Database/mysql/Connection.php` at line 125.

The `composer.json` and `composer.lock` files were updated to require `phpstan/phpstan` version `^2.1.38` (previously `^1.12.27 || ^2.1.34`) and `phpstan/phpstan-phpunit` version `^2.0.11` (previously `^1.4.2 || ^2.0.11`). This removes support for PHPStan 1.x entirely from the main branch.

The PHPStan baseline file `core/.phpstan-baseline.php` was updated with additional baseline entries for newly detected missing return type declarations and generic type specifications. The change only affects the main development branch (future Drupal 12), not the 11.x or 10.6.x branches.

This issue was triggered when a verification bot running on a contributor's local machine with PHP 8.4.6 detected the array key error, causing it to incorrectly mark RTBC issues as "Needs work" even though GitLab CI pipelines passed. The root cause was that PHPStan's bundled stubs were out of date.

### Collaboration

The issue revealed tension between local verification tooling and GitLab CI as sources of truth. After the bot incorrectly flagged approximately half of all RTBC issues overnight, nod_ stopped the bot and proposed moving verification checks into CI rather than running them locally. This prevented further disruption while the PHPStan version was corrected. The discussion also surfaced concerns from cmlara about contrib module compatibility, since many modules support Drupal 10.6 through 12 in a single release but PHPStan 1.x and 2.x have incompatible configuration files. The decision to commit only to the main branch preserves backward compatibility for contrib while allowing core development to move forward.
