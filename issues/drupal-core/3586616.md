## Summary

The `search` module dependency has been removed from `olivero_test`, the internal test helper module for the Olivero theme. Search-related Nightwatch integration tests for Olivero's search form blocks have been moved into the `search` module's own test directory. This change was backported to 11.x specifically to unblock deprecation of the `search` module there.

## Impact

No upgrade or configuration changes required.

## Technical details

The `olivero_test` module listed `drupal:search` as a dependency in its `.info.yml`. Because `olivero_test` is a core module, this coupling prevented `search` from being deprecated. Removing the dependency required relocating the Olivero-search Nightwatch tests that relied on it.

A new PHP setup class `TestSiteOliveroInstallTestScript` is placed at `core/modules/search/tests/src/Nightwatch/Olivero/` (implementing `TestSetupInterface`). It installs both `olivero_test` and `search`, then sets Olivero as the default theme. A new JS test file at `core/modules/search/tests/src/Nightwatch/Tests/Olivero/oliveroSearchFormTest.js` handles disabling the narrow and wide Olivero search form blocks and verifying no console errors result. The `oliveroRemoveBlockConsoleTest.js` test in core is updated to point to the new setup script path and no longer disables the search form blocks, since `search` is no longer installed as part of that test's setup.

During review, dcam observed that the initial placement of the JS files was not under a `Tests` subdirectory, which caused them to be silently skipped by the Nightwatch runner. This matches the convention followed by every other core module with Nightwatch tests (ckeditor, navigation, toolbar). After correcting the path, the `oliveroRemoveBlockConsoleTest.js` ran twice as expected, once per setup file.
