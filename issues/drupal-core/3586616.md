## Summary

The `olivero_test` testing module no longer declares a hard dependency on the Search module. Search-related Nightwatch test setup and tests that were previously tied to `olivero_test` have been moved into the Search module's own test directory. This is an internal test infrastructure change with no effect on production sites, but it unblocks a future deprecation of the core Search module.

## Impact

No upgrade or configuration changes required.

## Technical details

The `olivero_test` module listed `drupal:search` as a dependency in `olivero_test.info.yml` solely to support Nightwatch browser tests that verified Olivero's search form blocks. Removing that dependency required extracting the search-specific test concerns into the Search module itself.

The fix adds a new `TestSiteOliveroInstallTestScript` class under `core/modules/search/tests/src/Nightwatch/Olivero/` (implementing `TestSetupInterface`) that installs both `olivero_test` and `search` and sets Olivero as the default theme. A new Nightwatch test file placed under `core/modules/search/tests/src/Nightwatch/Tests/Olivero/` handles disabling the narrow and wide Olivero search form blocks, which was the search-specific setup previously embedded in `oliveroRemoveBlockConsoleTest.js`. That existing test (in `core/tests/Drupal/Nightwatch/`) was updated to reference the new setup script path and had its search block interaction steps removed.

During review, dcam observed that the new Nightwatch JS files were not actually being picked up by the test runner because they were not inside a `Tests/` subdirectory, matching the convention used by ckeditor, navigation, and toolbar modules. The files were corrected to follow that convention. alexpott backported the change to the 11.x branch explicitly to allow the Search module to be deprecated there.
