## Summary

Drupal's test infrastructure now requires PHPUnit 12, simultaneously dropping support for PHPUnit 11. This change lands in Drupal 12 (`main` branch) only. It is a developer-only change: no user-facing site behavior changed. Anyone who writes or runs automated tests for Drupal core or contrib modules must update their test code to be compatible with PHPUnit 12.

## Impact

Affects site owners.

- **Requirements:** PHPUnit minimum version raised from ^11.5.50 to ^12.5.14; phpspec/prophecy-phpunit raised from ^2 to ^2.4

## Technical details

The version bump itself is straightforward: `composer.json` moves `phpunit/phpunit` from `^11.5.50` to `^12.5.14` and `phpspec/prophecy-phpunit` from `^2` to `^2.4`. All downstream `sebastian/*` sub-packages (comparator, diff, exporter, global-state, environment, etc.) are advanced to their PHPUnit 12-compatible major versions in `composer.lock`. The packages `sebastian/code-unit` and `sebastian/code-unit-reverse-lookup` are removed entirely, as PHPUnit 12 dropped them; the allowlist in `core/scripts/dev/commit-code-check.sh`'s `Config` class is updated accordingly.

Two prerequisite streams had to land first. Annotation-to-attribute conversion (tracked in [#3445240](https://www.drupal.org/node/3445240)) was completed before this issue opened: PHPUnit 12 silently ignores docblock annotations (`@group`, `@dataProvider`, etc.), so all test metadata in core had to move to PHP 8 native attributes (`#[Group(...)]`, `#[DataProvider(...)]`, etc.). The mock-to-stub refactor (tracked in [#3561671](https://www.drupal.org/node/3561671)) was the final blocker: PHPUnit 12.5 deprecated `createMock()` calls that configure no expectations. Anywhere `createMock()` was used purely to get a type-compatible double without `->expects(...)`, it was replaced with `createStub()`.

The commit also cleans up PHPStan suppressions that were only needed to silence PHPUnit 11 compatibility warnings. The `phpstan-baseline.php` entries suppressing `KernelTestBase::__construct()` and `BrowserTestBase::__construct()` override errors are removed, as is the matching pattern in `phpstan.neon.dist`. The custom PHPStan rules (`TestClassClassMetadata` and `TestClassMethodMetadata`) that flag obsolete PHPUnit docblock annotations are deliberately retained for the Drupal 12 lifecycle to catch stale annotation usage in both in-flight core MRs and contrib modules that adopt the rules in their own PHPStan configuration.

## Upgrade

Contrib module developers need to make up to three changes for PHPUnit 12 compatibility.

**1. Replace PHPUnit docblock annotations with PHP attributes**

```php
// Before (PHPUnit 11 and earlier)
/**
 * @group mymodule
 * @dataProvider myDataProvider
 */
public function testSomething(): void {}

// After (PHPUnit 12)
use PHPUnit\Framework\Attributes\Group;
use PHPUnit\Framework\Attributes\DataProvider;

#[Group('mymodule')]
#[DataProvider('myDataProvider')]
public function testSomething(): void {}
```

**2. Replace expectation-less mocks with stubs**

```php
// Before: mock with no expectations configured
$service = $this->createMock(MyService::class);
$service->method('getFoo')->willReturn('bar');

// After: use a stub when no ->expects() call is made
$service = $this->createStub(MyService::class);
$service->method('getFoo')->willReturn('bar');
```

**3. Add `#[RunTestsInSeparateProcesses]` to kernel and browser test classes**

```php
use PHPUnit\Framework\Attributes\RunTestsInSeparateProcesses;

#[RunTestsInSeparateProcesses]
class MyKernelTest extends KernelTestBase { ... }

#[RunTestsInSeparateProcesses]
class MyFunctionalTest extends BrowserTestBase { ... }
```

Unit tests and build tests do not require `#[RunTestsInSeparateProcesses]`.
