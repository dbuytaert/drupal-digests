## Summary

Drupal's test infrastructure now requires PHPUnit 12, dropping support for PHPUnit 11. This is a developer-only change: it affects anyone who runs Drupal's automated tests or writes automated tests for contrib modules. No user-facing behavior changed.

## Impact

Affects site owners.

- **Requirements:** PHPUnit minimum version raised from 11 to 12 (specifically ^12.5.14); phpspec/prophecy-phpunit raised from ^2 to ^2.4

## Technical details

PHPUnit 12 was released in February 2025. Several preparatory streams had to be completed before the version bump was safe to land.

**Annotation-to-attribute conversion** (prerequisite, tracked in [#3445240](https://www.drupal.org/node/3445240)): PHPUnit 12 ignores doc-comment annotations entirely. All test metadata in Drupal core had to be expressed as PHP 8 attributes (`#[Group(...)]`, `#[DataProvider(...)]`, etc.) rather than `@group`, `@dataProvider`, and similar docblock tags.

**Stub-vs-mock cleanup** (the main blocker, tracked in [#3561671](https://www.drupal.org/node/3561671)): PHPUnit 12.5 deprecated mocks that configure no expectations. Anywhere `$this->createMock()` was called purely to get a type-compatible double without asserting any calls, it had to be replaced with `$this->createStub()`. This affected more than 1,500 test sites across core. The sub-issue [#3581058](https://www.drupal.org/node/3581058) closed the final batch, and dcam drove a large portion of this work.

**Additional cleanup**: `Assert::isType()` removal ([#3530453](https://www.drupal.org/node/3530453)) and a second round of PHPUnit 12 test fixes ([#3550292](https://www.drupal.org/node/3550292)) were merged into this MR.

**The version bump itself** (`composer.json`, `composer.lock`): `phpunit/phpunit` moved from `^11.5.50` to `^12.5.14`. `phpspec/prophecy-phpunit` moved from `^2` to `^2.4`. All downstream `sebastian/*` sub-packages (comparator, diff, exporter, global-state, etc.) were updated accordingly in `composer.lock`. PHPUnit 11 support was dropped simultaneously; it was considered undesirable to carry dual-version compatibility into a new Drupal major.

**`#[RunTestsInSeparateProcesses]` enforcement**: Classes extending `KernelTestBase` or `BrowserTestBase` must carry the `#[RunTestsInSeparateProcesses]` attribute. This is enforced at runtime in `KernelTestBase::setUp()` and `BrowserTestBase::setUp()`, and statically by the custom `TestClassClassMetadata` PHPStan rule in `core/tests/PHPStan/Rules/TestClassClassMetadata.php`.

**PHPStan annotation rules retained**: The custom PHPStan rules (`TestClassClassMetadata` and `TestClassMethodMetadata`) that flag obsolete PHPUnit doc-comment annotations were deliberately kept for the Drupal 12 lifecycle. In PHPUnit 12 these annotations are silent no-ops, so leaving the rules in place catches them early in CI for both core MRs still in the pipeline and contrib modules that adopt the rules in their own PHPStan configuration.

The `RunnerVersion` helper class at `core/tests/Drupal/TestTools/PhpUnitCompatibility/RunnerVersion.php` and related compatibility infrastructure were discussed as candidates for a follow-up removal, but were not removed in this commit.

## Upgrade

Contrib module developers need to make three changes for PHPUnit 12 compatibility:

**1. Replace PHPUnit doc-comment annotations with PHP attributes**

```php
// Before (PHPUnit 11 and earlier)
/**
 * @group mymodule
 * @dataProvider myDataProvider
 */
public function testSomething(): void {}

// After (PHPUnit 12)
use PHPUnit\Framework\Attributes\Group;
use PHPUnit\Framework\Attributes\DataProvider;

#[Group('mymodule')]
#[DataProvider('myDataProvider')]
public function testSomething(): void {}
```

**2. Replace expectation-less mocks with stubs**

```php
// Before: mock with no expectations configured
$service = $this->createMock(MyService::class);
$service->method('getFoo')->willReturn('bar');

// After: use a stub when no ->expects() call is made
$service = $this->createStub(MyService::class);
$service->method('getFoo')->willReturn('bar');
```

**3. Add `#[RunTestsInSeparateProcesses]` to kernel and browser test classes**

```php
use PHPUnit\Framework\Attributes\RunTestsInSeparateProcesses;

#[RunTestsInSeparateProcesses]
class MyKernelTest extends KernelTestBase { ... }

#[RunTestsInSeparateProcesses]
class MyFunctionalTest extends BrowserTestBase { ... }
```

Unit tests and build tests do not require `#[RunTestsInSeparateProcesses]`.
