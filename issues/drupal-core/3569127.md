## Summary

Drupal's automated update path tests now start from new 11.3.0 database dump fixtures instead of the older 10.3.0 dumps. The new fixtures ship without the modules that are deprecated for removal in Drupal 12 (such as History, Ban, and Contact), eliminating the need for separate per-module uninstall scripts as intermediate steps. A follow-up fix ensured the experimental `mysqli` database driver module is properly activated when tests run against these new fixtures. This is a testing infrastructure change with no impact on production sites.

## Impact

No upgrade or configuration changes required.

## Technical details

The Drupal update path test framework (`UpdatePathTestBase`) relies on pre-built gzipped SQL dumps as starting points for functional upgrade tests. Before this change the two canonical base fixtures were `drupal-10.3.0.bare.standard.php.gz` and `drupal-10.3.0.filled.standard.php.gz`, and tests that needed a post-deprecation state had to chain multiple `uninstall-*.php` PHP scripts (for the history, ban, and contact modules) on top of those dumps inside each test's `setDatabaseDumpFiles()` method.

This issue added two new fixtures, `drupal-11.3.0.bare.standard.php.gz` and `drupal-11.3.0.filled.standard.php.gz` (under `core/modules/system/tests/fixtures/update/`), generated by starting from a 10.3 standard installation, uninstalling the modules listed as deprecated for 12.x, and running updates through to 11.3. The new dumps therefore arrive in the correct state without any supplemental uninstall scripts.

`UpdatePathTestBaseTest::setDatabaseDumpFiles()` and `UpdatePathTestBaseFilledTest::setDatabaseDumpFiles()` were updated to load the single 11.3 fixture in place of the 10.3 dump plus the chained `uninstall-history.php`, `uninstall-ban.php`, and `uninstall-contact.php` scripts. The `databaseDumpFiles` count assertions in `UpdatePathTestBaseTest::testSetup()` were adjusted accordingly (from 5 to 3 in the first commit, then from 3 to 4 after the mysqli fix). The system module schema version verified in `UpdatePathTestBase::testDatabaseLoaded()` was updated from `10201` to `11300`.

After the initial commit landed, `mondrake` observed that `mysqli` driver tests were failing because the experimental `mysqli` module was not present in the new fixtures. The fix added a new fixture script, `core/modules/system/tests/fixtures/update/install-mysqli.php`, which directly manipulates the `core.extension` config row in the database to register the `mysqli` module. Both `UpdatePathTestBaseTest` and `UpdatePathTestBaseFilledTest` append this script to their dump file lists. The full fixture change targets `main` (the 12.x development branch); a backport to 11.x was committed and then reverted, leaving 11.x on the 10.3 dumps. Only the `install-mysqli.php` follow-up was cherry-picked to 11.x.

Once the old 10.3 upgrade paths and their associated test coverage are removed (the next planned step), the 10.3 dump files themselves can also be dropped. The longer-term direction of moving away from shipped dump files entirely is tracked in [#3403649](https://www.drupal.org/node/3403649).
