## Summary

Attempting to add or edit a view mode for an entity type that has no bundles (for example, the Node entity type after all content types have been deleted) threw a fatal `TypeError` and crashed the form. This is especially likely to surface now that Drupal core no longer ships with default content types. The bug is now fixed and the form renders correctly with an empty bundle selection list.

## Impact

No upgrade or configuration changes required.

## Technical details

In `EntityDisplayModeFormBase::form()`, the code called `array_keys($bundles[$definition->id()])` directly. When no bundles exist for the entity type, `$bundles[$definition->id()]` returns `null`, and `array_keys()` does not accept `null`, causing a `TypeError` at line 119 of `EntityDisplayModeFormBase.php`.

The fix adds a null-coalescing fallback: `$entity_bundles = $bundles[$definition->id()] ?? [];`, then iterates over `$entity_bundles`. The `bundles_by_entity` form element now simply renders with an empty `#options` array instead of crashing.

A regression test was added to `FieldUiFormTest` as a kernel test (`testViewModeAddFormWithoutBundles`), which builds the `EntityDisplayModeAddForm` directly for a node entity type that has no bundles and asserts the form builds successfully. The bug was introduced in [#3359240](https://www.drupal.org/node/3359240), which added bundle selection to the view mode creation form.
