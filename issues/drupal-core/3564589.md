## Summary

When JavaScript errors occur during functional JavaScript tests, Drupal now includes the URL path where the error happened in the test failure message. This makes it much easier to debug tests that visit multiple pages, since developers can immediately see which page caused the error instead of having to guess or add extra debugging code. The error messages now show the path followed by a pipe character and then the error details.

## Impact

No upgrade or configuration changes required.

## Technical details

The `js_testing_log.js` file now prepends `window.location.pathname` to both JavaScript errors and warnings before storing them in session storage. The format is `path|message`, for example `/js_errors_test|Error: A manually thrown error`.

The `DrupalSelenium2Driver` class needed updates to parse deprecation warnings correctly. Since the path prefix comes before the `[Deprecation]` marker, the code now uses `str_contains()` instead of `str_starts_with()` to detect deprecation messages, and uses a regex to extract the deprecation text after the marker.

Test expectations in `JavascriptErrorsTest` were updated to expect the path prefix in error messages. The Nightwatch assertion for deprecation errors also received a similar regex update to strip the path prefix before matching messages.

This change only affects the test framework's internal logging format. No production code or public APIs are impacted.
