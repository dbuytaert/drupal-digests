## Summary

As groundwork for moving the Shortcut module to contributed modules, the Shortcut-specific CSS and preprocessing logic that was previously bundled with the Claro, Olivero, and default_admin themes has been relocated into the Shortcut module itself. Sites using Shortcut with the Claro or Olivero theme will see no visible change, but the CSS is now delivered by the module rather than by the theme. The default_admin theme has had all Shortcut styling removed outright, since Shortcut display there is handled by the Navigation module. A post-update function cleans up a stale `third_party_settings.shortcut` config entry from `olivero.settings` on existing sites.

## Impact

Affects front-end developers and site owners.

- **Database update:** `olivero_post_update_remove_shortcut_settings_if_not_installed` clears `third_party_settings.shortcut` from `olivero.settings` on upgrade.
- **Config change:** `olivero.schema.yml` no longer declares a schema for `third_party_settings.shortcut.module_link`; the key is absent from `olivero.settings` when Shortcut is not installed.
- **Theme change:** Shortcut CSS for Claro and Olivero is now owned and delivered by the Shortcut module via `claro.shortcut.backport` and `olivero.shortcut.backport` libraries; `claro/drupal.shortcut` no longer exists. The default_admin theme has had all Shortcut-specific CSS, icons, and template classes removed.

## Technical details

Claro's `drupal.shortcut` library (defined in `claro.libraries.yml` with `css/components/shortcut.css`) and its `shortcut/drupal.shortcut` entry in `libraries-extend` are removed. The CSS moves to `css/backport-themes/claro/shortcut.css` inside the Shortcut module, served via the new `claro.shortcut.backport` library. Similarly, Olivero's `css/components/page-title.css` (which contained shortcut-wrapper positioning styles) is removed from Olivero's `global-styling` library; it moves to `css/backport-themes/olivero/page-title.css` in the Shortcut module, served via `olivero.shortcut.backport`. Both libraries are attached dynamically in `ShortcutThemeHooks::preprocessPageTitle()` when the active theme is `claro` or `olivero`, replacing the old `libraries-extend` mechanism.

Olivero's `OliveroHooks::preprocessPageTitle()` hook, which wrapped the page title and shortcut link in a positioning `div`, is deleted. The equivalent logic moves into the new `ShortcutThemeHooks::preprocessPageTitleOlivero()` method, called conditionally from `ShortcutThemeHooks::preprocessPageTitle()`. `ShortcutThemeHooks` gains a `ThemeManagerInterface` constructor injection to support this. Shortcut's `shortcut_install()` and `shortcut_uninstall()` functions now manage `third_party_settings.shortcut.module_link` on Olivero in addition to Claro. Olivero previously hardcoded `module_link: true` in its `config/install/olivero.settings.yml` and declared the key in its schema; both are removed. `olivero_post_update_remove_shortcut_settings_if_not_installed()` clears the stale `third_party_settings.shortcut` key from `olivero.settings` on upgrade.

The default_admin theme loses its `drupal.shortcut` library definition, its `shortcut/drupal.shortcut` library extension, shortcut icon SVG files, `--admin-icon-shortcut` and `--admin-icon-shortcut-filled` CSS custom properties, all `.shortcut-action` CSS rules, `.toolbar-icon-shortcut` / `.toolbar-button--icon--shortcuts` icon selectors, `#toolbar-item-shortcuts` visibility rules, and the `menu--shortcuts` class from `menu-region--top.html.twig`. Stable9 and Claro are intentionally left in place pending their respective deprecations.

## Upgrade

If you maintain a **sub-theme of Claro** that previously overrode `claro/drupal.shortcut` via `libraries-extend`, that library no longer exists. Override `shortcut/claro.shortcut.backport` instead, or provide your own shortcut CSS.

If you maintain a **sub-theme of Olivero** that overrode the page-title shortcut wrapper styles, those styles are no longer in Olivero's `global-styling` library. They are now attached by the Shortcut module via `shortcut/olivero.shortcut.backport`.

If any code reads `olivero.settings:third_party_settings.shortcut.module_link` directly, the key is removed from the schema and will be cleared by the post-update function `olivero_post_update_remove_shortcut_settings_if_not_installed`. The setting is now managed entirely by the Shortcut module's install/uninstall hooks.

If you use the **default_admin** theme with Shortcut enabled, the theme-level shortcut CSS (icon, action link styling) is gone. The Navigation module continues to handle shortcut display; theme-level star-icon support is not present in default_admin.
