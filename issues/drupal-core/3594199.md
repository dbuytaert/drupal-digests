## Summary

The Telephone module has been removed from Drupal core. It provided a `telephone` field type, a `telephone_link` formatter, and a `telephone_default` widget for storing and displaying phone numbers. Sites that have telephone fields in use must uninstall the core module and switch to the community-maintained contrib version before or after upgrading. A change record with migration guidance is available at https://www.drupal.org/node/3606383.

## Impact

Affects module maintainers.

- **API change:** `TelephoneItem`, `TelephoneLinkFormatter`, and `TelephoneDefaultWidget` are removed from core; the `telephone` field type and its plugins no longer exist in `Drupal\telephone\*`.

## Technical details

The entire `core/modules/telephone/` directory was deleted, including `TelephoneItem` (field type), `TelephoneLinkFormatter` (formatter), `TelephoneDefaultWidget` (widget), `TelephoneHooks` (hook implementations), CSS icon assets, config schema (`telephone.schema.yml`), and the `drupal.telephone-icon` library. All associated tests were also removed.

`telephone` was added to the `DRUPAL_CORE_REMOVED_MODULE_LIST` constant in `core/modules/system/src/Install/Requirements/SystemRequirements.php`. This constant is checked during requirements verification, so any site with the telephone module still enabled will receive a blocking error when trying to run updates. The `telephone` namespace was also removed from the PHPStan suppression regex in `core/phpstan-baseline.neon`, and the `telephone/drupal.telephone-icon` library reference was removed from `Stable9LibraryOverrideTest`.

A follow-up issue ([#3607231](https://www.drupal.org/node/3607231)) was opened to regenerate the core database fixtures so that update path tests no longer need to carry a separate fixture to uninstall telephone before running.

## Upgrade

Sites using the telephone field type must uninstall the core module before upgrading:

1. Install the contrib replacement (https://www.drupal.org/node/3223395#s-telephone).
2. Uninstall the core `telephone` module via Drush or the Extend UI.
3. Update Drupal core.

If the module is still enabled at update time, `SystemRequirements` will block the update with an error because `telephone` is now listed in `DRUPAL_CORE_REMOVED_MODULE_LIST`.
