## Summary

Controllers and plugins using `AutowiredInstanceTrait` can now autowire container parameters directly in their constructors, not just services. This brings the trait's autowiring capabilities in line with Symfony's container behavior, reducing the need for custom `create()` factory methods when a plugin or controller needs to inject both services and configuration parameters.

## Impact

Affects module maintainers.

- **New API:** The `#[Autowire]` attribute in `AutowiredInstanceTrait` now supports container parameters using `#[Autowire('%parameter_name%')]` or `#[Autowire(param: 'parameter_name')]` syntax

## Technical details

The `AutowiredInstanceTrait::getAutowireArguments()` method now detects when an `#[Autowire]` attribute value is wrapped in percent signs (e.g. `%parameter_name%`) using a regex pattern. When detected, it extracts the parameter name and calls `$container->getParameter()` instead of `$container->get()`. This follows the same convention Symfony uses to distinguish parameters from services.

The implementation supports both the verbose `#[Autowire(param: 'name')]` syntax and the shorthand `#[Autowire('%name%')]` syntax. Nullable parameters work the same way as nullable services: if the parameter doesn't exist and the constructor argument has a default value, that default is used instead of throwing an exception.

Error messages now distinguish between missing services and missing parameters to make debugging easier. The change also works with setter injection via the `#[Required]` attribute, which was added in [#3558306](https://www.drupal.org/node/3558306).

## Upgrade

When using `AutowiredInstanceTrait` (typically in plugins or controllers), you can now autowire container parameters directly in constructors:

```php
use Symfony\Component\DependencyInjection\Attribute\Autowire;

class MyPlugin extends PluginBase {
  use AutowiredInstanceTrait;

  public function __construct(
    // Autowire a parameter using shorthand syntax
    #[Autowire('%serializer.format_providers%')]
    public readonly array $formatProviders,
    // Or using explicit param syntax
    #[Autowire(param: 'authentication_providers')]
    public readonly array $authProviders,
    // Services still work as before
    public readonly MyService $service,
  ) {}
}
```

This eliminates the need for custom `create()` methods that previously had to inject parameters manually. See the change record at https://www.drupal.org/node/3575335 for more examples.
