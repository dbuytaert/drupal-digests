## Summary

The `default_admin` theme shipped two CSS libraries (`settings_tray` and `settings_tray_edit`) intended to style the Settings Tray edit mode, but those styles never had any effect: `default_admin` strips contextual links from all non-layout-builder blocks, which is the only mechanism that opens the Settings Tray. The dead CSS files and library definitions have now been removed. The corresponding integration in Stable 9 is intentionally left alone, since Stable 9 is itself being removed from Drupal core in Drupal 12.

## Impact

Affects front-end developers.

- **Theme change:** Libraries `default_admin/settings_tray` and `default_admin/settings_tray_edit` removed from the `default_admin` theme; any external code depending on them must remove those dependencies.

## Technical details

The `default_admin` theme declared two library entries (`settings_tray` and `settings_tray_edit`) in `default_admin.libraries.yml`, each wrapping a pair of CSS files under `migration/css/components/`. These libraries were attached to the `navigation` and `toolbar` library groups. The theme's `preprocessBlock` hook (which removes contextual links from all blocks except layout builder blocks, inherited originally from the Seven theme via [#2487025](https://www.drupal.org/node/2487025)) makes it impossible to launch the Settings Tray on any admin-theme block, so the styling was unreachable.

The commit removes the four CSS files (`settings_tray.css`, `settings_tray_edit.css`, and their minified counterparts), drops the two library definitions, and removes the `default_admin/settings_tray` and `default_admin/settings_tray_edit` entries from the `navigation` and `toolbar` dependency lists. The Stable 9 `settings_tray/drupal.settings_tray` overrides in `stable9.info.yml` are untouched because Stable 9 is scheduled for removal from core itself.

## Upgrade

Any custom module or theme that declares a dependency on the removed libraries must be updated:

```yaml
# Before (now broken)
dependencies:
  - default_admin/settings_tray
  - default_admin/settings_tray_edit
```

These libraries no longer exist and should be removed from your dependency declarations. Because Settings Tray does not function in admin themes (contextual links are suppressed there), no replacement dependency is available within `default_admin`.

### Collaboration

During review, dcam discovered that the original procedural hook implementation for loading the Settings Tray libraries was silently not firing. Investigating why led to the broader finding, confirmed by longwave, that Settings Tray is entirely non-functional in admin themes because `default_admin` and Claro both implement `hook_preprocess_block` to remove contextual links from all blocks. Longwave traced this back to [#2487025](https://www.drupal.org/node/2487025), which removed contextual links from the Seven theme, with Claro and Default Admin later inheriting that behavior. This discovery shifted the resolution from moving the CSS into the `settings_tray` module to simply removing it entirely from `default_admin`.
