## Summary

When running Drupal's PHPUnit test suite, contrib and custom extensions that override a core (or other extension) method without adding a native return type will now trigger a deprecation notice, provided the parent method documents its intended return type via a `@return` docblock annotation. This early warning gives extension authors time to add native return types before Drupal adds them to interfaces and abstract classes in the next major release, which would otherwise cause fatal errors.

## Impact

No upgrade or configuration changes required.

## Technical details

Symfony's `DebugClassLoader` already warns when a child class overrides a method that carries a `@return` annotation but the child does not add a native return type. However, it uses the first namespace segment as the vendor boundary, so all code under `\Drupal\` is treated as a single vendor and no cross-extension warnings are ever emitted.

The fix introduces `DrupalDebugClassLoader` in `core/tests/Drupal/TestTools/ErrorHandler/DrupalDebugClassLoader.php`, marked `@internal`. It extends `DebugClassLoader` and overrides `checkAnnotations()`. After calling `parent::checkAnnotations()`, it reads the static `$returnTypes` property from `DebugClassLoader` via `ReflectionProperty` and iterates over the collected return-type data for the class being loaded. For each method that carries a `@return`-derived type, the override extracts the Drupal extension name (the second namespace segment, or the third when `Tests` is the second) for both the declaring class and the implementing class via the private `getExtensionName()` helper. If these extension names differ, the implementing method is actually overriding something in a different extension. The override then checks whether the method is truly overridden in the current class, whether it already has a native return type, and whether its docblock contains `@deprecated` or `@return` (either of which suppresses the message). If none of those conditions are met, a deprecation in the standard Symfony format is appended.

The test bootstrap (`core/tests/bootstrap.php`) is updated to call `DrupalDebugClassLoader::enable()` instead of `DebugClassLoader::enable()`.

To prevent the new loader from immediately breaking CI on known existing violations, two regex patterns are added to `core/.deprecation-ignore.txt`: one suppressing the new message for all current core modules, and one for test modules and fixture extensions. These allow incremental follow-up cleanup (see [#3579258](https://www.drupal.org/node/3579258)) rather than requiring every existing violation to be fixed in this commit.

As part of fixing violations that would not be suppressible in tests, a handful of return types were added to test helpers: `TestableSqlContentEntityStorage::getTableMapping()` gains `: TableMappingInterface`; several methods in `core/tests/Drupal/KernelTests/RouteProvider.php` gain native return types; and `ClaroPreRender::trustedCallbacks()` gains `: array`.

A unit test `DrupalDebugClassLoaderTest` in `core/tests/Drupal/Tests/TestTools/ErrorHandler/` covers the positive case (cross-extension child without a return type triggers a deprecation) and five negative cases (same-extension child, child with native return type, child with `@return` annotation, child with `@deprecated`, child that does not override the method).

## Upgrade

Extension authors who override methods from a different Drupal extension and see a new deprecation notice in their test runs have two options:

**Option 1: Add a native return type (preferred)**
```php
// Before
public function myMethod() {
  return $this->doSomething();
}

// After
public function myMethod(): string {
  return $this->doSomething();
}
```

**Option 2: Add an explicit `@return` annotation to suppress the notice**
```php
/**
 * {@inheritdoc}
 *
 * @return string
 */
public function myMethod() {
  return $this->doSomething();
}
```

The deprecation notice only appears during test runs (the loader is active only in the PHPUnit bootstrap). No runtime behavior changes for production sites.
