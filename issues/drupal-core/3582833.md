## Summary

The `default_admin` admin theme shipped to Drupal core as Gin with Claro's CSS copied wholesale, leaving two parallel CSS trees: `css/` (original Claro styles) and `migration/css/` (Gin overrides). This issue consolidates all 40+ files from `migration/css/base/` into the canonical `css/` directory, merging each file into its corresponding component file or creating a new file where no Claro equivalent existed. The end result is a single authoritative CSS tree per component, eliminating the override layer. Along the way, several intentional visual corrections landed: focus rings are now a visible solid `0.25rem` ring using `--gin-color-focus` (previously 1px dotted transparent), heading font-weight dropped from bold to 400 (tracking Gin's variable-font design), and blockquotes changed from a quotation-mark decoration to a colored left border. Styles that belonged to contrib modules (Diff, Upgrade Status, Insert) were removed from core and separate issues filed in those queues.

## Impact

Affects front-end developers.

- **Theme change:** Focus rings changed from invisible dotted to solid 0.25rem; heading font-weight changed from bold to 400; blockquote styling and action-link icon technique changed in default_admin theme.

## Technical details

The two CSS trees existed because `default_admin` was built by flattening Gin (which used Claro as a base theme) into core. The `migration/css/` directory held all the Gin override and additive files named with a `gin-*` prefix; the canonical `css/` directory held the original Claro structure. Each `gin-*.pcss.css` file was either merged into an existing file in `css/` or promoted to a new standalone file there.

Key structural changes visible in the diff: `css/base/variables.pcss.css` now imports `media-queries.pcss.css` at the top and contains the full `--gin-*` custom-property block (colors, typography, spacing, layout, dark-mode overrides, and three density variants keyed on `data-gin-layout-density`). `css/base/icons.pcss.css` is a new file that centralizes all icon references as `--admin-icon-*` custom properties pointing to SVG files. `css/components/action-link.pcss.css` dropped per-state inline SVG `background-image` declarations for each icon variant in favor of a unified `mask-image` approach using those icon custom properties, dramatically reducing duplication.

Focus styles changed globally: `--focus-outline` went from `1px dotted transparent` to `0.25rem solid var(--gin-color-focus)` and the `box-shadow` focus variant was removed from the `elements` file. The `--gin-color-focus` and `--gin-color-focus-border` custom properties are now declared in the consolidated variables block. The `css/base/font.pcss.css` path correction (`../../../font/` to `../../font/`) fixes a broken `@font-face` reference that would have prevented the Inter/Ginter font from loading.

The `migration/css/` directory is not yet fully removed; a follow-up ([#3590364](https://www.drupal.org/node/3590364)) handles the remaining files (components, layout, theme layers not covered here).

## Upgrade

No PHP or API changes. If a contrib theme or module overrides CSS files from `default_admin`'s `migration/css/base/` directory, those files have been merged into `css/` and the `migration/` originals may be removed in a follow-up. Contrib modules that had styles bundled in the Gin migration layer (Diff, Upgrade Status, Insert) should expect those styles to be absent from core; see the issues opened in those queues for how to relocate them.

The focus-ring change is the most noticeable visual shift:
```css
/* Before */
--focus-outline: 1px dotted transparent;

/* After */
--focus-outline: 0.25rem solid var(--gin-color-focus);
```
