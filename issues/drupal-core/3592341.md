## Summary

Drupal's routing system maintains its own copy of Symfony's URL generation logic in `UrlGenerator::generateFromRoute()`. That copy contained the same vulnerability described in Symfony CVE-2026-48784: when a route parameter contains chained dot-segments such as `../../..` or `././.`, the URL generator failed to percent-encode all of them, leaving raw `../` or `./` sequences in the generated URL. A browser or HTTP client following such a URL would resolve the dot-segments as path traversal. This fix ensures every `.` and `..` segment is encoded to `%2E` and `%2E%2E` respectively, regardless of how many are chained. The companion issue [#3592421](https://www.drupal.org/node/3592421) updated the upstream Symfony packages; this issue fixes Drupal's own forked copy.

## Impact

Affects site owners.

- **Security:** Incomplete percent-encoding of chained `../` and `./` dot-segments in `UrlGenerator::generateFromRoute()` could allow path traversal via crafted route parameters (Symfony CVE-2026-48784).

## Technical details

The old code in `UrlGenerator::generateFromRoute()` used `strtr($path, ['/../' => '/%2E%2E/', '/./' => '/%2E/'])` plus two `str_ends_with` checks for trailing `/.` and `/..`. Because `strtr()` performs a single non-overlapping scan, a sequence like `/../../..` would only encode some of the segments: the replacement of the first `/../` consumed the surrounding slashes, leaving the adjacent `../` without leading or trailing slashes to match on the next pass. The test run by `smustgrave` confirmed the symptom: expected `/test-path/%2E%2E/%2E%2E/%2E%2E/bar` but received `/test-path/%2E%2E/../%2E%2E/bar`.

The fix, cherry-picked from Symfony commit `4b63c3a3`, replaces the `strtr` approach with an explicit segment-by-segment pass: `explode('/', $path)`, encode any segment that is exactly `.` or `..`, then `implode('/', $segments)`. This handles any number of consecutive dot-segments and also covers segments at the tail of the path without special-casing. Tests covering all three chained patterns (`../../..`, `././.`, `../././..`) and tail routes were ported directly from the Symfony commit.
