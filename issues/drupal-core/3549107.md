## Summary

Content fields on Drupal sites can contain control characters or invalid UTF-8 byte sequences (for example, a backspace byte 0x08 copied from a word processor). Before this fix, requesting such content through JSON:API caused a `SyntaxError: Bad control character in string literal in JSON` parse error in clients, because Drupal was emitting the raw bytes unescaped. The fix makes all JSON responses from Drupal, including JSON:API, REST, and AJAX, automatically substitute the Unicode replacement character (U+FFFD) for any invalid UTF-8 sequences, producing valid JSON output in all cases.

## Impact

No upgrade or configuration changes required.

## Technical details

The serialization module's `JsonEncoder` (`core/modules/serialization/src/Encoder/JsonEncoder.php`) constructs a Symfony `JsonEncode` instance with a set of PHP JSON encoding flags. Before this fix, only the HTML-safety flags were present: `JSON_HEX_TAG | JSON_HEX_APOS | JSON_HEX_AMP | JSON_HEX_QUOT`. These flags escape HTML-special characters but do nothing for invalid UTF-8 byte sequences. PHP's `json_encode()` silently returns `false` when it encounters invalid UTF-8 without the `JSON_INVALID_UTF8_SUBSTITUTE` flag, which causes the serializer to emit a broken or empty response body that clients then reject as malformed JSON.

The fix adds `JSON_INVALID_UTF8_SUBSTITUTE` to the bitmask in `JsonEncoder::__construct()`. With this flag, PHP replaces any invalid UTF-8 sequence with the Unicode replacement character `\ufffd` instead of aborting encoding. The fix was applied at the `serialization` module level rather than in the JSON:API-specific `JsonEncoder` (`core/modules/jsonapi/src/Encoder/JsonEncoder.php`), which merely extends the parent. This means the protection is inherited by REST JSON and AJAX responses as well.

The contributor scottatdrake initially proposed patching only the JSON:API encoder, then in a follow-up comment recognized that fixing the parent `serialization` encoder would cover REST and AJAX as well. That is the approach that was merged. New unit tests were added to both `core/modules/serialization/tests/src/Unit/Encoder/JsonEncoderTest.php` and `core/modules/jsonapi/tests/src/Unit/Encoder/JsonEncoderTest.php` covering: invalid UTF-8 substitution, preservation of existing HTML-unsafe character escaping (`\u003C`, `\u003E`, etc.), and structured nested-array data. The word `ufffd` was also added to the cspell dictionary at `core/misc/cspell/dictionary.txt` to prevent spell-check failures in CI. The fix was committed to 11.x and 11.3.x simultaneously, then backported to 10.6.x via a separate MR by scottatdrake.
