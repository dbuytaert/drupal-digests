## Summary

The Migrate Drupal module has been removed from Drupal core's main branch (targeting Drupal 12). It was deprecated in Drupal 11.4.0 following Drupal 7's end of life, and no contributed replacement exists. Rather than deleting the module directory entirely, Drupal retains a minimal stub with its info file marked `lifecycle: obsolete` and an uninstall hook, so sites that still have the module enabled can cleanly uninstall it before upgrading. The companion Migrate Drupal UI module was removed in a separate issue ([#3574714](https://www.drupal.org/node/3574714)). This is a purely internal cleanup with no behavioral change for sites that have already uninstalled both modules.

## Impact

Affects module maintainers.

- **Deprecation:** `DrupalSqlBase`, `I18nQueryTrait`, `MigrateFieldPluginManager`, all Migrate Drupal source/process/destination plugins, and the `plugin.manager.migrate.field` service are fully removed from core; there is no replacement.

## Technical details

The removal was approved under the policy established in [#3371229](https://www.drupal.org/node/3371229) following Drupal 7's end of life. The `migrate_drupal` module had been deprecated in drupal:11.4.0 targeting removal in drupal:12.0.0.

The retained stub consists only of the module's `.info.yml` (marked `lifecycle: obsolete`), an uninstall hook, and a `hook_update_N()` that triggers that uninstall. Both `migrate_drupal` and `migrate_drupal_ui` were added to `DRUPAL_CORE_REMOVED_MODULE_LIST` in `system.install`.

The diff shown is a portion of the ~853 file change (234 insertions, 192,542 deletions). It covers cleanup in PHPStan tooling files:

- `cspell.json`: removed the ignore-list entries for the Drupal 6 and Drupal 7 fixture dump files (`modules/migrate_drupal/tests/fixtures/drupal6.php` and `drupal7.php`).
- `phpstan-baseline.neon`: removed the blanket deprecation-warning suppression patterns for `DrupalSqlBase`, `I18nQueryTrait`, `MigrateFieldPluginManager`, the `plugin.manager.migrate.field` service, and dozens of specific process plugins (`BlockPluginId`, `BlockRegion`, `FieldType`, `ProcessField`, `FilterID`, `LanguageNegotiation`, etc.) and destination plugins (`EntityBlock`, `EntityContentComplete`, `EntityNodeType`, `EntityShortcutSet`, etc.).
- A separate PHPStan ignore-errors baseline file: removed `ignoreErrors` entries that suppressed `class.extendsDeprecatedClass` and `method.deprecatedClass` violations across source plugins in `block`, `block_content`, `comment`, `content_translation`, `field`, and other core modules that had extended `DrupalSqlBase`, as well as `missingType.return` entries for numerous migrate-related test classes.

Issue [#3573895](https://www.drupal.org/node/3573895) resolved remaining test integrations in the core migrate module that had depended on `migrate_drupal` directly, including `MigrateEntityContentBaseTest` (which used `StubTestTrait`) and the `migrate_source_annotation_bc_test` test module (which extended `EmptySource` from `migrate_drupal`). `MigrationPluginConfigurationTest` was updated to avoid the legacy `Variable` source plugin.

## Upgrade

Sites that have `migrate_drupal` or `migrate_drupal_ui` enabled must uninstall both modules before upgrading to Drupal 12. The retained stub module and its `hook_update_N()` handle the uninstall automatically during a database update run on Drupal 11 before the upgrade. Custom code that extends `DrupalSqlBase`, uses `MigrateFieldPluginManager`, or references any of the removed process or destination plugins has no supported replacement in core and must be removed or replaced with custom implementations.
