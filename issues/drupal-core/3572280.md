## Summary

The `migrate_drupal` module has been removed from Drupal core's `main` branch (targeting Drupal 12). It was deprecated in Drupal 11.4.0 following Drupal 7's end of life, and no contributed replacement will be provided. Rather than deleting the module directory entirely, core retains a minimal stub consisting only of the `.info.yml` file (marked `lifecycle: obsolete`) and an uninstall hook, so sites that still have the module enabled can cleanly uninstall it before upgrading. Both `migrate_drupal` and `migrate_drupal_ui` were added to `DRUPAL_CORE_REMOVED_MODULE_LIST` in `system.install` (the latter was missed in the companion issue [#3574714](https://www.drupal.org/node/3574714)). This is the largest negative diffstat in core history: 853 files changed, 234 insertions, 192,542 deletions.

## Impact

Affects module maintainers.

- **Deprecation:** `DrupalSqlBase`, `I18nQueryTrait`, `MigrateFieldPluginManager`, the `plugin.manager.migrate.field` service, and all Migrate Drupal source, process, and destination plugins are fully removed from core in Drupal 12 with no replacement.

## Technical details

Removal was authorized by the policy established in [#3371229](https://www.drupal.org/node/3371229) after Drupal 7 reached end of life. The retained stub contains only `migrate_drupal.info.yml` (marked `lifecycle: obsolete`), a `hook_uninstall()`, and a `hook_update_N()` that triggers uninstallation during a Drupal 11 database update run ahead of the major-version upgrade.

The commit diff visible in `phpstan-baseline.neon` and `core.phpstan-baseline.php` reflects cleanup of hundreds of suppressed deprecation warnings that were added in Drupal 11.3.0 when these symbols were marked deprecated. Removed suppressions cover `class.extendsDeprecatedClass` violations for every core module that extended `DrupalSqlBase` (block, block_content, comment, content_translation, field, file, filter, image, language, menu_link_content, and others), `method.deprecatedClass` violations for calls to `variableGet()`, `moduleExists()`, and `checkRequirements()` on `DrupalSqlBase`, and `missingType.return` violations for dozens of migrate-related test classes. A new `class.notFound` suppression for `EntityContentComplete` in `IdAuditor.php` was added to acknowledge the now-missing class. The fixture dump filenames `drupal6.php` and `drupal7.php` were also removed from the `cspell.json` ignore list.

Dependent test integrations were resolved before this landed: `MigrateEntityContentBaseTest` had used `StubTestTrait` from `migrate_drupal`, which was copied into the `migrate` module; the `migrate_source_annotation_bc_test` test module extended `EmptySource` from `migrate_drupal` and was removed along with `MigrateSourceDiscoveryTest`; and `MigrationPluginConfigurationTest` was updated to avoid the legacy `Variable` source plugin. See [#3573895](https://www.drupal.org/node/3573895) for those fixes.

## Upgrade

Sites must uninstall `migrate_drupal` (and `migrate_drupal_ui`) before upgrading to Drupal 12. Running database updates on Drupal 11 first will trigger the included uninstall hook automatically.

Custom code that depends on any of the following has no supported core replacement and must be removed or rewritten:

- `DrupalSqlBase` (source plugin base class)
- `I18nQueryTrait`
- `MigrateFieldPluginManager` / `plugin.manager.migrate.field` service
- Any process plugin under `Drupal\{block,field,file,filter,image,language,link,node,path,responsive_image,search,system,taxonomy,user}\Plugin\migrate\process\`
- Any destination plugin such as `EntityBlock`, `EntityContentComplete`, `EntityNodeType`, `EntityShortcutSet`, `EntitySearchPage`, `ThemeSettings`, `EntityUserRole`
- Any source plugin extending `DrupalSqlBase` in contributed or custom modules
