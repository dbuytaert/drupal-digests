## Summary

A recent change to Drupal core updated the method signature for `HtmlRenderer::buildPageTopAndBottom()`, adding two optional parameters. This broke the Canvas module because its subclass override did not match the new signature, causing fatal errors during PHPStan analysis and test runs. The fix updates Canvas to include the optional parameters in its method declaration, restoring compatibility with core while preserving its existing behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

The Canvas module provides `CanvasPreviewRenderer`, which extends `HtmlRenderer` from core. It overrides the `buildPageTopAndBottom()` method with an empty implementation to prevent the toolbar and other page elements from appearing in preview mode.

Core issue [#3339905](https://www.drupal.org/project/drupal/issues/3339905) changed the signature of `HtmlRenderer::buildPageTopAndBottom()` to accept two additional optional parameters: `array $page_top = []` and `array $page_bottom = []`. This change was committed on 2026-02-03.

Canvas's override did not include these new parameters, creating a method signature mismatch that violated PHP's inheritance rules. This triggered fatal errors during autoloading when PHPStan tried to analyze code that included Canvas, and also broke test execution.

The fix updates `CanvasPreviewRenderer::buildPageTopAndBottom()` to match the parent signature by adding the optional `$page_top` and `$page_bottom` parameters. Since both parameters have default values, this maintains backward compatibility and conforms to the Liskov Substitution Principle. The method body remains empty as intended.
