## Summary

In the Canvas module's "Manage Libraries" tab, clicking "Add Folder" now opens a proper modal `Dialog` component instead of an inline popover. The new dialog includes client-side validation with inline error messages, loading state on the submit button, and graceful in-dialog display of API errors such as duplicate folder names. Separately, the "Add new code component" dialog had its confirm button text changed from "Add" to "Create" for consistency.

## Impact

Affects module maintainers.

- **New API:** New `validateFolderNameClientSide` function added to `@/features/validation/validation`

## Technical details

The `AddFolderButton` component in `ManageLibrary` previously used Radix UI's `Popover.Root`/`Popover.Trigger`/`Popover.Content` to host the folder creation form. The rewrite replaces this with the shared `Dialog` component from `@/components/Dialog`, which provides a modal overlay, a cancel/confirm footer, and a structured error slot. State management was also updated: instead of handling success synchronously in the submit handler, the component now uses `useEffect` hooks to react to `isSuccess` and `isError` from `useCreateFolderMutation`, keeping the dialog open on error and resetting on success.

A new `validateFolderNameClientSide` function was added to `@/features/validation/validation.ts` to enforce that folder names do not start or end with a hyphen, underscore, or whitespace, and contain no special characters. The existing `validateMachineNameClientSide` function was renamed to `validateCodeMachineNameClientSide` to distinguish it from the new folder validator; all call sites and tests were updated accordingly.

The `Dialog` component itself received a minor fix: the close `IconButton` now calls `handleOpenChange(false)` via an explicit `onClick` handler, ensuring the `onOpenChange` callback fires correctly when the header close button is clicked. Playwright tests were updated to use role-based selectors (`getByRole('button', { name: 'Add' })`) instead of the previous `data-testid` selectors that were tied to the old popover markup.

## Upgrade

The function `validateMachineNameClientSide` in `@/features/validation/validation` was renamed:

```ts
// Before
import { validateMachineNameClientSide } from '@/features/validation/validation';
validateMachineNameClientSide(name);

// After
import { validateCodeMachineNameClientSide } from '@/features/validation/validation';
validateCodeMachineNameClientSide(name);
```
