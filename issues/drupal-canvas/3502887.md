## Summary

The Canvas editor URL structure has changed. The editor path moved from `/canvas/{entity_type}/{entity_id}` to `/canvas/editor/{entity_type}/{entity_id}`, and the preview path moved from `/canvas/{entity_type}/{entity_id}/preview/{width}` to `/canvas/preview/{entity_type}/{entity_id}/{width}`. Navigating to `/canvas` now shows a welcome screen rather than immediately opening an editor. These changes are groundwork for seamless SPA navigation between different pages and content types without full page reloads, unblocking issue [#3538104](https://www.drupal.org/node/3538104).

## Impact

Affects module maintainers.

- **API change:** `drupalSettings.canvas.entityType` and `drupalSettings.canvas.entity` removed; `drupalSettings.canvas.entityTypeKeys` changed from a flat object to a map keyed by entity type ID; `useGetLayoutByIdQuery` no longer accepts an `entityId` argument

## Technical details

The `canvas.boot.entity` route path changed from `/canvas/{entity_type}/{entity}` to `/canvas/editor/{entity_type}/{entity}`. `CanvasPathProcessor::processInbound()` was broadened: it now rewrites all `/canvas/` paths (except `/canvas/api`) to `/canvas`, rather than only matching the entity-type/entity pattern, so the React SPA handles its own sub-routing entirely.

On the PHP side, `CanvasController` no longer emits `drupalSettings.canvas.entityType` or `drupalSettings.canvas.entity`. The `drupalSettings.canvas.entityTypeKeys` value changed shape from a flat map of the currently-edited entity's field keys to an object keyed by entity type ID, populated for `node` and every entity type implementing `ComponentTreeEntityInterface`. A `CacheableAccessDeniedHttpException` is now thrown (previously `LogicException`) when no compatible entity type is accessible.

In the React SPA all routes that previously used `/editor`, `/editor/region/:regionId`, `/preview/:width`, and `/code-editor/...` as flat paths now embed `/:entityType/:entityId` into the path: e.g. `/editor/:entityType/:entityId`, `/preview/:entityType/:entityId/:width`, `/editor/:entityType/:entityId/code-editor/component/:codeComponentId`. Components throughout the app (`PageInfo`, `PageStatus`, `UnpublishedChanges`, `PreviewControls`, `SideMenu`, `ComponentContextMenu`, etc.) switched from reading `entityId`/`entityType` out of Redux selectors (`selectEntityId`, `selectEntityType` from `configurationSlice`) to reading them from `useParams()`. A new `useEntityTitle` hook replaces direct use of `canvasSettings.entityTypeKeys.label` to look up the title field key. `useGetLayoutByIdQuery` no longer accepts an `entityId` argument and reads the ID from URL params internally. A new `Welcome` component is rendered at the base `/canvas` path and at `/editor/` when no entity is selected. `PreviewControls` was extracted into its own component.

## Upgrade

Any code or bookmarks pointing to `/canvas/{entity_type}/{entity_id}` must be updated to `/canvas/editor/{entity_type}/{entity_id}`.

Preview URLs change from:
```
/canvas/{entity_type}/{entity_id}/preview/{width}
```
to:
```
/canvas/preview/{entity_type}/{entity_id}/{width}
```

`drupalSettings.canvas` no longer contains `entityType` or `entity` keys. Code reading those must switch to reading `entityType`/`entityId` from the URL or from the new `entityTypeKeys` map:

```js
// Before
const labelKey = drupalSettings.canvas.entityTypeKeys.label;

// After
const entityType = /* read from URL params */;
const labelKey = drupalSettings.canvas.entityTypeKeys[entityType]?.label;
```

API responses for content entity links (`LINK_REL_EDIT`, `LINK_REL_SET_AS_HOMEPAGE`) now return `/canvas/editor/{entity_type}/{entity_id}` instead of `/canvas/{entity_type}/{entity_id}`.
