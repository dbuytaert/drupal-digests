## Summary

Canvas now includes utilities for converting between its internal component tree format and json-render specifications. This enables rendering Canvas pages in React applications using the json-render library. The conversion functions handle single and multi-root component trees, nested structures, and named slots. These utilities are part of the Canvas module's effort to support external rendering environments like Canvas Workbench.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas stores component pages as flat arrays where each component links to its parent via `parent_uuid`. The json-render library expects a nested spec with a single root and a flat map of elements linked by `children` and `slots` arrays.

Three new functions in the `drupal-canvas` package handle conversion. `canvasTreeToSpec()` walks the flat Canvas tree and builds a json-render spec, wrapping multi-root trees in a synthetic `canvas:component-tree` element. `specToCanvasTree()` reverses the process, generating UUIDs for each node and unwrapping the synthetic wrapper. `renderCanvasTree()` combines these with `@json-render/react` to render Canvas trees directly in React applications.

The implementation depends on a forked version of json-render that adds named slot support (PR #105 in the upstream repository). The Canvas tree format uses `parent_uuid` and `slot` to link components; json-render uses element keys in `children` and `slots` arrays. The converter validates parent references and throws errors for malformed trees. Component inputs are mapped to json-render props, while `component_version` and `label` fields have no json-render equivalent and are set to null during conversion.
