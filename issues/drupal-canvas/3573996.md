## Summary

Canvas pages can now be rendered and edited in local development environments using json-render, a library that renders React components from JSON specifications. This enables developers to work on Canvas pages outside of Drupal, using familiar tools like IDEs and version control. The new utilities convert between Canvas's flat component tree format (where components are linked by parent IDs) and json-render's nested specification format, making it possible to prototype Canvas workbenches and editing experiences in standalone React applications.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas stores component trees as flat arrays where each node references its parent by UUID. The json-render library expects a nested specification with a single root element and slots. Three new utilities bridge these formats.

`canvasTreeToSpec()` walks the flat Canvas array and reconstructs parent-child relationships into json-render's nested structure. When multiple root components exist, it wraps them in a synthetic `canvas:component-tree` element since json-render requires a single root. Named slots map directly; the default `children` slot maps to json-render's special `children` array.

`specToCanvasTree()` reverses this conversion. It traverses the json-render spec depth-first, generating Canvas nodes with parent UUIDs and slot names. It preserves existing UUIDs when round-tripping and strips the synthetic wrapper element to restore multi-root trees.

`renderCanvasTree()` combines conversion with rendering using `@json-render/react`. A fallback component handles the synthetic wrapper transparently and logs warnings for unknown component types.

These utilities depend on a forked version of json-render that supports named slots (proposed in upstream PR #105), installed via npm aliases until the feature merges upstream. The implementation includes comprehensive round-trip tests validating structure preservation, slot ordering, and edge cases like orphaned components.
