## Summary

A new "Disable max-width" option is proposed for the PayPal Checkout payment gateway. When enabled, it removes the max-width constraint that PayPal normally applies to Smart payment buttons, giving merchants fuller control over how the buttons are sized in their storefront layout.

## Impact

Affects site owners.

- **Config change:** New `disableMaxWidth` boolean key added to the `commerce_paypal_checkout` gateway style configuration; defaults to `FALSE` (no change to existing behavior).

## Technical details

The change touches three layers. In `commerce_payment.commerce_payment_gateway.plugin.paypal_checkout` config schema, a new `disableMaxWidth` boolean key is added under the `style.buttons_customizable` mapping. In the `Checkout` plugin class, `'disableMaxWidth' => FALSE` is added to the default configuration array so existing gateways keep the current capped behavior. A `#type => 'checkbox'` element labelled "Disable max-width" is inserted into the `$form['style']` group, bound to that default value, so administrators can toggle it through the standard gateway edit form.
