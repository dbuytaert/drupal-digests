## Summary

The Canvas module's content-template field-linking UI now surfaces "List (integer)" and "List (float)" fields as candidates when connecting content fields to component props. Previously, all three `list_*` field types were excluded from shape-matching because a predefined value set is unlikely to exactly match a component's enumerated inputs. This change recognises that mapping a predefined integer or float field to a generic `type: integer` or `type: number` prop is perfectly safe and useful, so those two field types are now eligible to be linked. "List (string)" remains blocked for now (tracked in [#3548749](https://www.drupal.org/node/3548749)).

## Impact

Affects module maintainers.

- **New API:** New `exceptions` key in `JsonSchemaFieldInstanceMatcher::IGNORE_FIELD_TYPES` entries allows per-schema opt-outs from the ignore list.

## Technical details

`JsonSchemaFieldInstanceMatcher::IGNORE_FIELD_TYPES` was restructured from a flat `array<string, class-string>` to `array<string, array{class: class-string, exceptions: array<array>}>`. Each entry now carries an `exceptions` list of JSON Schema fragments; when the target prop schema exactly matches one of those fragments the field type is no longer skipped during matching.

`list_integer` gains two exceptions (`['type' => 'integer']` and `['type' => 'number']`), and `list_float` gains one (`['type' => 'number']`). The loop inside the field-instance filtering path (around line 342) was updated to `continue` (skip the ignore rule) when `in_array($schema, $allowed_schemas, TRUE)` is true, so only a prop with additional constraints such as `minimum` or `multipleOf` would still be skipped. `list_string`, `map`, and `password` carry empty `exceptions` arrays and behave exactly as before.

The `FieldInstanceSupportTest::COMPLETION` constant rose from ~88.46% to ~89.29% and `COMPLETION_PROPS` from ~89.74% to ~90.24%. Tests add a `one_from_an_integer_list` field storage and confirm it appears in the matched instances for `type: integer` and `type: number` prop shapes, including via the `unix_to_date` adapter.
