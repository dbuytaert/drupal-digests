## Summary

Canvas's `semi_coupled` theme engine became incompatible with Drupal 11.3.x after core issue [#1685492](https://www.drupal.org/node/1685492) converted theme engines from hook-based `.engine` files into tagged services. Any Canvas page on an 11.3.x site threw a 500 error with the message `Service "semi_coupled" not found: the container inside "Drupal\Core\Theme\ThemeManager" is a smaller service locator that only knows about the "twig" service`. This issue proposed converting the engine to comply with the new service-based API. It was ultimately closed as fixed when [#3555062](https://www.drupal.org/node/3555062) removed the semi-coupled theme engine from Canvas entirely, making Canvas compatible with both 11.2.x and 11.3.x without needing the engine approach at all.

## Impact

No upgrade or configuration changes required.

## Technical details

The root cause is that Drupal 11.3.x (core [#1685492](https://www.drupal.org/node/1685492)) requires theme engines to be registered as tagged services implementing `ThemeEngineInterface` rather than as standalone `.engine` files providing procedural hooks (`hook_extension()`, `hook_theme()`, `hook_render_template()`). The old `semi_coupled.engine` file and its accompanying `semi_coupled.info.yml` (of type `theme_engine`) were incompatible with this model.

MR !296 proposes the intermediate fix: move all engine logic from procedural functions into a `SemiCoupledEngine` class implementing `ThemeEngineInterface`, and register it in `canvas.services.yml` with `tags: [{name: theme_engine, engine_name: semi_coupled}]`. The methods `theme()` and `renderTemplate()` map directly to the old `semi_coupled_theme()` and `semi_coupled_render_template()` hooks; the helper functions `_semi_coupled_collect_props()`, `_semi_coupled_build_html_tag()`, `_semi_coupled_get_variable()`, `_semi_coupled_snake_case()`, and `_semi_coupled_kebab_case()` become private methods on the class.

This MR was not merged. The actual fix arrived via [#3555062](https://www.drupal.org/node/3555062), which removed the `semi_coupled` theme engine from Canvas entirely and improved React embedding without relying on a custom engine, resolving the 11.3.x incompatibility in a way that also kept 11.2.x working.
