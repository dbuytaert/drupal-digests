## Summary

The Canvas module's library panels received a significant UI overhaul. A live search box (debounced at 300 ms) was added to both the Library and Manage Library panels so site builders can filter components, patterns, and code components by name. The "Add new" (code component) button was removed from the Library panel and consolidated into a "New" dropdown in the Manage Library panel. The "Add new folder" button is now gated behind the `canvas_dev_mode` feature flag so it does not appear in production. Library items no longer insert on click; instead, each item exposes a contextual menu (right-click or the "Open contextual menu" button) with an "Insert" option alongside other actions such as Edit, Rename, and Delete. A spurious "0" entry that appeared under the Code tab when there were no folders but internal code components existed is also fixed.

## Impact

No upgrade or configuration changes required.

## Technical details

The core of the refactor is a new generic `LibraryItemList` component that replaces duplicated folder/item rendering logic in `ComponentList` and `PatternList`. It accepts typed items, folder data, a `searchTerm`, and a `renderItem` callback, and internally handles alphabetical sorting, folder grouping via the existing `folderfyComponents`/`sortFolderList` helpers, Radix UI `Skeleton` loading states, and a `Callout` empty state (including a "no results for search term" variant).

`ExposedJsComponent` was renamed `CodeComponentItem` and extended to handle both exposed (`JSComponent`) and non-exposed (`CodeComponentSerialized`) code components through a unified `CodeComponentItemProps` interface with an `exposed` flag. A new `ComponentItem` component was extracted from the inline `SidebarNode` usage in `ListItem`. `PatternNode` was renamed `PatternItem`. All three now accept `insertMenuItem` and `menuTitleItems` render props so that `ListItem` can inject a titled "Insert" menu entry into each item's dropdown and context menu without the item components needing to know about layout dispatch logic.

Click-to-insert was removed from `ListItem`'s drag wrapper; inserting a component now requires using the contextual menu's "Insert" option (`handleInsertClick`). The `onMouseEnter` preview handler is now always attached regardless of draggable state. In `SideMenu`, navigating to the code editor now also dispatches `setManageLibraryTab('code')` so the Manage Library panel automatically opens on the Code tab. A new `openManageLibraryPanel()` helper was added to the `CanvasEditor` Playwright page-object class, and several tests were updated to use `addComponent({ name })` through the contextual menu rather than direct locator clicks.
