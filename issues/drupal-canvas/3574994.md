## Summary

The right sidebar in Drupal Canvas can now be resized by dragging a handle. Users can adjust the sidebar width to fit their workflow and screen size. The chosen width is remembered across sessions. This improves the editing experience when working with component properties that need more space or when maximizing the preview area.

## Impact

No upgrade or configuration changes required.

## Technical details

The implementation replaces the `react-resizable-panels` library with `allotment` for both the editor right sidebar and the code editor layout. The new library provides better control over pane visibility and sizing constraints.

In the main editor, the `Editor` component now wraps the editor frame and `ContextualPanel` in an `Allotment` component. The sidebar pane has constraints: `SIDEBAR_MIN_PX` (320px) minimum, `SIDEBAR_MAX_PX` (640px) maximum, and a default of 24% width. The pane is hidden when no component is selected in the template editor or when in interactive mode, using the `visible` prop rather than CSS display none.

The code editor similarly uses nested `Allotment` components for horizontal and vertical splits. Both layouts persist their sizes to localStorage using new `layoutStorage.ts` helpers (`getLayoutItem` and `setLayoutItem`). These helpers catch storage errors gracefully so the UI falls back to defaults if localStorage is unavailable.

The `ContextualPanel` CSS changed from a fixed width to flex-based sizing to work within the resizable pane. The layout token `--sidebar-right-width` was removed since width is now dynamic. Playwright tests were updated to wait for the contextual panel to mount after component selection, as the Allotment pane visibility transitions can delay rendering.
