## Summary

Canvas Workbench is a new local developer tool prototype for previewing Drupal Canvas Code Components and pages without a running Drupal site. It automatically discovers components and pages from your project's source code, displays them in a browser interface, and supports hot module reload for live updates as you edit. This tool aims to provide a Storybook-like development experience tailored specifically for Canvas components.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas Workbench introduces a Vite-powered development server that scans a project directory for Canvas component metadata files (`component.yml` and `*.component.yml`) and page specifications (`pages/*.json`). The `@drupal-canvas/discovery` package handles filesystem scanning with `.gitignore` support, resolves JavaScript and CSS entry files by extension precedence, and returns component records with stable IDs and file paths.

The `@drupal-canvas/vite-compat` package provides shared Vite configuration for host project imports. It resolves `@/` alias imports from host components, enables SVGR for SVG components, and handles side-effect CSS imports including Tailwind entrypoints.

The `drupal-canvas` package gains json-render utilities: `canvasTreeToSpec` and `specToCanvasTree` convert between Canvas component trees and json-render specs, `renderSpec` and `renderCanvasTree` render components using a registry, `defineComponentRegistry` dynamically imports components, and `defineComponentCatalog` generates a json-render catalog from metadata for AI prompting and validation.

The Workbench runtime uses one Vite process for both the UI shell and preview iframe. Discovery data is exposed via `/__canvas/discovery` and `/__canvas/preview-manifest` endpoints. A custom Vite plugin watches host files and sends targeted HMR updates. Components must export a default render function to be previewable. Currently requires a hardcoded Tailwind entrypoint at `src/components/global.css`.
