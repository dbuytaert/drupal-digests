## Summary

Canvas Workbench is a new local developer preview tool for Drupal Canvas Code Components and Canvas Pages. It automatically discovers components and pages from your source code, displays them in a browsable UI, and provides live previews with hot module reload. Component inputs defined in `component.yml` files can be previewed with example values. Pages represented as json-render specs can also be previewed locally. This early prototype explores whether a Storybook-like experience can work directly with Canvas component metadata and eliminate manual story authoring.

## Impact

No upgrade or configuration changes required.

## Technical details

This prototype introduces several new packages. The `@drupal-canvas/discovery` package scans filesystems for `component.yml` and `*.component.yml` files, respects `.gitignore` rules, resolves JavaScript entries by extension precedence (`.ts`, `.tsx`, `.js`, `.jsx`), attaches optional CSS entries, and discovers top-level page specs from `pages/*.json`. The `@drupal-canvas/workbench` package runs a Vite dev server that serves a React-based UI shell and an iframe preview runtime from a single process. A custom Vite plugin exposes discovery and preview manifest data as JSON endpoints. The `@drupal-canvas/vite-compat` package provides shared host import compatibility, including host alias resolution (`@/...`), third-party imports, SVGR for SVG components, and Tailwind CSS entrypoint support. Component previews load the default export from each discovered JavaScript entry. Pages use json-render specs rendered via the `drupal-canvas` package. New utilities in `drupal-canvas/json-render-utils` include `defineComponentRegistry` for dynamically importing components, `defineComponentCatalog` for converting Canvas metadata to json-render catalogs, and `renderSpec` for rendering json-render specs. The `renderCanvasTree` function now normalizes Canvas prop payloads such as rich text wrappers and image metadata before rendering.
