## Summary

Canvas Workbench is an early prototype for a local developer tool that lets developers preview Canvas Code Components and Canvas Pages directly from source code, similar to Storybook. It automatically discovers components by scanning component.yml files, displays them in a preview UI, and supports hot module reload for live updates as you code. The tool runs as a bundled Vite server and reads example values from component metadata to generate interactive previews without requiring a Drupal backend.

## Impact

No upgrade or configuration changes required.

## Technical details

The prototype introduces three new packages. `@drupal-canvas/discovery` scans the filesystem for `component.yml` and `*.component.yml` files, resolves JavaScript entries by extension priority (.ts, .tsx, .js, .jsx), attaches optional CSS entries, and discovers top-level page specs from `pages/*.json`. It respects `.gitignore` rules and excludes common build directories. `@drupal-canvas/vite-compat` provides shared Vite compatibility helpers, including host import aliasing (`@/`), SVGR for SVG components, and support for side-effect CSS imports. `@drupal-canvas/workbench` runs a Vite dev server with custom middleware that exposes discovery data and preview manifests as JSON endpoints, uses routes for component and page previews, and watches the host project for changes to trigger targeted HMR updates.

The `drupal-canvas` package gains new json-render utilities: `defineComponentRegistry` dynamically imports discovered components, `defineComponentCatalog` converts component metadata to json-render catalog format with Zod schemas, and `renderSpec` renders json-render specs with prop resolution and state binding. The preview iframe imports component modules through Vite `@fs` URLs and normalizes Canvas prop payloads (like rich text `{value, format}` wrappers and image metadata) into component-friendly values.

Workbench currently runs one Vite process for both UI shell and preview iframe to simplify early iteration, with explicit guardrails to separate workbench (`@wb/*`) and host (`@/`) aliases. It temporarily hardcodes a Tailwind entrypoint at `src/components/global.css` loaded through a virtual module.
