## Summary

Canvas Workbench is a new local developer tool that lets you preview and work with Canvas Code Components and pages directly from your source code. Similar to Storybook, it automatically discovers components from your codebase, reads their configuration from component.yml files, and shows you live previews with example values. It includes hot module reload support and uses json-render to preview entire Canvas pages locally without needing a Drupal backend running.

## Impact

No upgrade or configuration changes required.

## Technical details

This prototype introduces three new packages. The `@drupal-canvas/discovery` package scans directories for `component.yml` and `*.component.yml` files, respects `.gitignore` rules, and discovers both components (with JavaScript and optional CSS entries) and pages (from `pages/*.json` files). It resolves JavaScript entries by extension precedence (.ts, .tsx, .js, .jsx) and generates stable component IDs using SHA1 hashes.

The `@drupal-canvas/vite-compat` package provides shared Vite compatibility helpers for host project imports. It resolves `@/` alias imports from host components, supports asset imports (images, SVGs, CSS), enables SVGR for SVG components, and handles extensionless directory imports with automatic index file resolution.

The `@drupal-canvas/workbench` package provides the Workbench UI itself. It runs a Vite dev server that serves both the component browser UI and preview iframes. Discovery middleware exposes component and page data at `/__canvas/discovery` and preview manifests at `/__canvas/preview-manifest`. The preview system imports component modules through Vite `@fs` URLs and requires a default export from each component module. A hardcoded Tailwind entrypoint at `src/components/global.css` is loaded through a virtual CSS module.

The `drupal-canvas` package gains new json-render utilities. `renderSpec()` renders json-render specs using a component registry, resolving `$state` expressions and conditional logic. `defineComponentRegistry()` builds registries by dynamically importing component JavaScript entries. `defineComponentCatalog()` converts component metadata to json-render catalogs with Zod schemas for validation and AI prompt generation. Canvas-specific prop normalization unwraps rich text `{value, format}` objects and image metadata wrappers.
