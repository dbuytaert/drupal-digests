## Summary

Canvas now supports dragging and dropping folders to reorder them within the library panel. Previously, folders could only be reordered using contextual menu options to move them up or down. This feature makes organizing content libraries faster and more intuitive, especially when managing multiple folders. The drag operation is restricted to the sidebar area and includes visual feedback with drop indicator lines to show where the folder will be placed.

## Impact

No upgrade or configuration changes required.

## Technical details

The folder reordering implementation extends the existing drag-and-drop system to handle folder items in addition to components. The `SidebarFolder` component now uses the `useDraggable` hook from `@dnd-kit/core`, passing folder metadata including `type: 'folder'`, `folderId`, and `weight` in the drag data. The component is only draggable when `devMode` is enabled and the user has the `folders` permission.

The `useDropOnFolderHandler` hook detects when a folder is being dropped on another folder by checking if `activeData.type === 'folder'`. It prevents self-drops, then retrieves all folders of the same type (components, patterns, etc.) and sorts them by weight. The dragged folder is removed from its current position and inserted at the target position, then all affected folders have their weights updated sequentially to reflect the new order.

Visual feedback includes CSS classes `isDragging`, `dropIndicatorAbove`, and `dropIndicatorBelow` that show where the folder will be placed based on the relative weights of the dragged and target folders. A custom modifier `restrictFolderToSidebar` constrains folder dragging to the sidebar area by checking if the dragged element exceeds the sidebar's right boundary. When at the boundary, a `folderAtBoundary` class changes the cursor to indicate the operation is not allowed.

The `DragEventsHandler` component filters drag events to prevent folders from being dropped into layout regions. During `handleDragOver`, if a folder is being dragged over a non-folder destination, the target slot is unset to prevent visual feedback. The `DragOverlay` is hidden when dragging folders to avoid confusion.

Playwright tests verify the drag-and-drop behavior by creating two test folders, calculating their bounding boxes, and performing a manual pointer-based drag that moves past the `@dnd-kit` 3px activation distance before completing the drop. The test confirms the folder order changes as expected.
