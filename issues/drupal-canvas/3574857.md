## Summary

The Canvas editor now supports the `entity_reference_autocomplete` field widget when used as a component prop. Previously, selecting an entity via the autocomplete field would not produce a usable value in the Canvas prop system. With this change, the widget is registered as supported and the selected autocomplete value is correctly transformed into a numeric entity ID before being stored.

## Impact

Affects module maintainers.

- **New API:** New `entityAutocompleteTargetId` client-side transform registered in Canvas; available for use in custom Canvas field widget integrations.

## Technical details

On the PHP side, `ReduxIntegratedFieldWidgetsHooks::fieldWidgetInfoAlter()` gains an entry for `entity_reference_autocomplete` with two transforms: `mainProperty` (pointing to `target_id`) and the new `entityAutocompleteTargetId`. The `FieldWidgetSupportTest` completion metric advances from 15/28 to 16/28.

On the JavaScript side, a new `entityAutocompleteTargetId` transformer is added in `ui/src/utils/transforms.ts`. It extracts the numeric ID from Drupal's autocomplete display format (`"Label (id)"`) using `ENTITY_AUTOCOMPLETE_MATCH = /.+\s\(([^)]+)\)\s*$/`, which mirrors `\Drupal\Core\Entity\Element\EntityAutocomplete::extractEntityIdFromAutocompleteInput`. For comma-separated tags-mode input (multiple references in a single field), a quote-aware `splitAutocompleteValues` function mirrors `Drupal.autocomplete.splitValues` so labels containing commas (e.g. `"Hello, world" (3)`) are handled correctly. The transform is idempotent: if the value is already a bare ID (because the transform ran on a prior commit cycle), it passes through unchanged rather than being dropped. Invalid entries that do not match the autocomplete pattern also pass through as bare IDs so that a single bad entry does not discard the entire selection; server-side validation surfaces the actual error.

`shouldSkipPropValidation` in `componentFieldValidation` is extended to skip client-side JSON schema validation for props whose schema type is `object` or `array` of objects, because their source shape differs from the resolved value and cannot be validated client-side. A new `canvas.transform.entityAutocompleteTargetId` Drupal library is declared in `canvas.libraries.yml`. This work follows the same pattern used for `daterange_default` in [#3522130](https://www.drupal.org/node/3522130).
