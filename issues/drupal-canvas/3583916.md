## Summary

The `@drupal-canvas/create` CLI now includes an `agents` subcommand that lets developers set up AI agent compatibility symlinks in an existing project, not just during initial scaffolding. Previously, if agent setup was skipped at scaffold time or a developer wanted to add support for additional agents later, there was no way to do it without rescaffolding. Running `npx @drupal-canvas/create@latest agents` (optionally with a comma-separated agent list or `none`) fills that gap.

## Impact

Affects module maintainers.

- **New API:** New `agents` subcommand in `@drupal-canvas/create`: `npx @drupal-canvas/create@latest agents [agents]`

## Technical details

The core problem was that `setupAgentSkills` and the interactive agent-selection prompt were only reachable through the main `create` flow. The fix introduces a standalone `agentsCommand` function in a new `src/agents.ts` file, registered on the `commander` program instance in `index.ts` via `agentsCommand(program)`. It accepts an optional `[agents]` argument (comma-separated list or `"none"`), resolves the project directory from `process.cwd()`, and delegates to the existing `setupAgentSkills` and `parseAgentSelection` helpers. To support context-appropriate error messages when called from the new command, `parseAgentSelection` gains an optional `label` parameter (defaulting to `'--agents'`) that replaces the previously hard-coded string in all three error paths. Two smaller UX improvements landed alongside: `setupAgentSkills` now logs an informational message when no `.agents/skills` directory is found (previously it silently returned), and `defaultPromptForAgents` emits an empty `p.log.message('')` to add visual spacing before the multiselect prompt.
