## Summary

When an SDC component defined more than one image prop (for example `image_desktop`, `image_tablet`, and `image_mobile`), editing any one of those images in the Canvas UI caused all the others to be lost on save. Only the most recently uploaded image was retained. This is now fixed: all image prop values are preserved correctly when any single prop is updated.

## Impact

No upgrade or configuration changes required.

## Technical details

The Canvas editor sends a `patchComponent` request to the backend whenever a prop changes via AJAX (such as an image upload). The request payload was built by merging `selectedModel.resolved` with the freshly changed `values`. The problem was that any other prop values still held in the React form state were not run through the `getPropsValues` transform before being included in the patch, so they were either dropped or sent in a form the backend did not accept.

The fix, in `ComponentInstanceFormRenderer`, reads the full current form state via `selectFormValues(state, FORM_TYPES.COMPONENT_INSTANCE_FORM)` before building the patch. It then calls `getPropsValues(formState, { ...inputAndUiData, model: { [selectedComponentId]: selectedModel } }, currentData.transforms)` to produce a properly transformed snapshot of all existing prop values. The resolved payload is now assembled as `{ ...selectedModel.resolved, ...transformedFormState, ...values }`, ensuring existing props are normalized and the newly changed value still wins on conflict.
