## Summary

The Canvas CLI `push` command now correctly handles build failures and looks for compiled assets in the right directory. Previously, the command would throw errors when components failed to build, preventing developers from seeing which components had issues. It also looked for global CSS and JavaScript files in the wrong location. The command now displays a helpful table showing which components succeeded or failed, and correctly finds built assets in the output directory.

## Impact

No upgrade or configuration changes required.

## Technical details

This fix addresses three bugs in the Canvas CLI `push` command workflow. First, `buildAndPushComponents()` now returns build results instead of throwing exceptions when builds fail, allowing the caller to display a detailed error table before exiting. Second, `uploadGlobalAssetLibrary()` now looks for compiled `index.css` and `index.js` files in `config.outputDir` instead of `componentDir/dist`, fixing a path resolution issue. Third, the import analyzer now skips `@/components/*` imports when collecting dependencies, since Canvas provides these via an importmap and they should not be bundled into individual components. Tests were updated to verify that build failures return results rather than throwing errors.
