## Summary

Canvas now supports editing multiple images and videos in a single field using drag-and-drop reordering and inline controls. Previously, multi-value media fields lacked a user-friendly interface for managing multiple items. Now, media items display as thumbnails with drag handles for reordering and remove buttons for deletion, matching the workflow already available for other multi-value field types like text and numbers.

## Impact

No upgrade or configuration changes required.

## Technical details

This change extends the Redux-integrated form UI to support multi-value media fields by wrapping the media library widget in a sortable list container. The implementation uses `@dnd-kit` to enable drag-and-drop reordering of media items. Three new React components handle the behavior: `DrupalMediaLibraryFieldset` manages AJAX replacements of the media widget while preserving React component tree position, `DrupalMediaListContainer` coordinates sorting and removal operations, and `DrupalMediaLibraryItem` renders individual items with drag handles.

The sorting implementation tracks media items by auto-generated IDs and updates both the Redux store and DOM weight fields when items are reordered. For the component editing form, the order is determined by the sequence of `target_id` values in the model, which are patched after drag operations. For entity forms, the order is managed via weight fields.

Special handling ensures the remove button works correctly when only one item remains. Because removing the final item deletes the input element from the DOM before its change event can propagate to Redux, the remove handler directly patches the component model after waiting for any in-flight AJAX requests to complete.

The UI disables drag handles when only one item is present and hides the "Add media" button when the configured cardinality limit is reached. Drupal's weight toggle UI is removed in favor of drag-to-reorder. Two test fields (`field_cvt_unlimited_media` and `field_cvt_limited_media`) were added to the test article bundle to support functional testing of both unlimited and limited cardinality scenarios.
