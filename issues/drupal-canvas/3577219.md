## Summary

The Canvas component editor now supports multi-value Image and Video prop types. Editors can view media items as inline thumbnails, reorder them using a six-dot drag handle, remove individual items with an inline button, and add further items via an "Add media" button. The "Add media" button is hidden once the field's configured cardinality limit is reached, and drag handles are visually disabled when only one item is present. This brings media props in line with multi-value text and number props already supported in the editor.

## Impact

No upgrade or configuration changes required.

## Technical details

Three new React components handle the behavior and are registered in `twigToJSXComponentMap`. `DrupalMediaLibraryFieldset` wraps the entire media fieldset and intercepts `Drupal.AjaxCommands.prototype.insert`: when Drupal's AJAX would normally do a `replaceWith` on a `[data-canvas-media-library-fieldset]` element, the override fires a `canvas:updateMediaWidget` custom event carrying the new markup instead. The component listens for that event, parses the HTML via `DOMParser`, calls `Drupal.Hyperscriptify`, and stores the result in React state. This keeps the component's position in the React tree stable, which is required for sorting to work correctly when the list contains a mix of pre-existing and newly added items.

`DrupalMediaListContainer` coordinates sorting and removal using `@dnd-kit/core` and `@dnd-kit/sortable`. After a drag-end event, the component reorders its internal state and calls `updateComponentPropOrder`. In the component editing form (`component_instance_form`), it reads the current sequence of `[name*="target_id"]` inputs from the DOM and dispatches `patchComponent` with `buildReorderPatch` to update both `source[propName].value` and `resolved[propName]`. In entity forms, it instead updates the `[data-canvas-media-weight]` DOM attributes directly. When only one item remains, a `mousedown` listener on the remove button triggers `buildRemovePatch` directly; the patch dispatch is wrapped in a `setInterval` that polls `isAjaxing()` and waits for all in-flight AJAX to finish before calling `patchComponent`. This addresses a race condition where removing the last item deletes the tracking input from the DOM before its change event can reach Redux. `DrupalMediaLibraryItem` renders individual items using `useSortable` and conditionally renders the drag handle based on the `data-is-multiple` attribute set by PHP.

In `ReduxIntegratedFieldWidgetsHooks`, the widget alter now iterates all selection delta keys rather than only index 0, adding `data-track-hidden-value` to both `target_id` and `weight` hidden inputs, a `data-canvas-media-weight` marker to weight fields, and `data-is-multiple` on each selection wrapper. The `canvas_stark` theme overrides `media_library/widget` to disable core's `js/media_library.widget.js`, and `canvas_stark_preprocess_fieldset__media_library_widget` calls `unset($variables['prefix']['weight_toggle'])` to remove Drupal's weight-toggle UI in favour of drag-to-reorder. Twig templates `fieldset--media-library-widget.html.twig` and `media-library-selection.html.twig` now wrap their contents in the new custom elements; a new `media-library-item.html.twig` renders each item via `<drupal-canvas-media-library-item>`.

The earlier `VALUE_THAT_MEANS_REMOVE` sentinel constant and `flaggedForRemoval` utility were deleted from `function-utils` as the new approach handles removal through direct model patching, making the sentinel unnecessary. `InputBehaviorsComponentPropsForm` was simplified accordingly. Two test fields, `field_cvt_unlimited_media` (cardinality -1) and `field_cvt_limited_media` (cardinality 2), were added to the `canvas_test_article_fields` module and pre-seeded with image media entities to support functional Cypress tests.
