## Summary

When an SDC or code component under active development has its props changed (added, renamed, removed), Canvas now handles the mismatch between stored component instances and the updated component gracefully. Previously, editing a page that used an SDC whose props had changed would crash with a PHP `TypeError` or `OutOfRangeException`. Now the component instance editing form renders from the version's stored metadata regardless of what the live component looks like. Additionally, when Drupal's verbose error logging is enabled, broken components (for example, an SDC whose directory was deleted) now display the actual exception detail in both the Canvas editor and on the live page, making it much easier to diagnose what went wrong during development.

## Impact

Affects site owners.

- **Database update:** Two post-update hooks: `canvas_post_update_0001_track_props_have_required_flag_in_components` backfills `required` flag in all `Component` config entity versions; `canvas_post_update_0007_respect_prop_ordering` restores SDC-defined prop order in active versions.
- **Config change:** New `required` boolean key added to each `prop_field_definitions` entry in `canvas.generated_field_explicit_input_ux`; `prop_field_definitions` sequence `orderby` changed from `key` (alphabetical) to `~` (insertion order).

## Technical details

The root cause of the editing crash was that `GeneratedFieldExplicitInputUxComponentSourceBase::buildConfigurationForm()` iterated over the live SDC schema (`getComponentInputsForMetadata()`) rather than the `prop_field_definitions` stored in the `Component` config entity version. The fix inverts this: the form now iterates over the stored `prop_field_definitions`, cross-referencing live metadata only for the prop label. Because storable props are a subset of all props, a pre-computed list of storable prop names is built from live metadata and used to skip non-storable entries in the stored list.

To support the form's `#required` flag for props that may no longer exist in the live component, a new `required` boolean key was added to each entry in `prop_field_definitions` inside the `canvas.generated_field_explicit_input_ux` config schema. `GeneratedFieldExplicitInputUxComponentSourceBase` now writes this flag when building the stored prop definitions, and reads it directly in `buildConfigurationForm()`. Two post-update hooks handle existing data: `canvas_post_update_0001_track_props_have_required_flag_in_components` (via `CanvasConfigUpdater::updatePropFieldDefinitionsWithRequiredFlag()`) backfills the `required` key across all versions of every `Component` config entity, creating a new active version when the active version's hash must change; `canvas_post_update_0007_respect_prop_ordering` (via `CanvasConfigUpdater::updatePropOrder()`) reorders `prop_field_definitions` entries to match the component author's declared prop order. The latter was necessary because the schema previously used `orderby: key` (alphabetical), which silently discarded the SDC-defined order; it is now `orderby: ~`.

`RenderSafeComponentContainer::handleComponentException()` was updated to read `system.logging`'s `error_level` setting: when verbose, it renders the exception class and message (sanitized via `Xss::filterAdmin()`) in both the Canvas editor preview and the live-page view. `Component::normalizeForClientSide()` was updated to wrap the `renderComponent()` call inside a `RenderSafeComponentContainer` render element, ensuring that exceptions thrown during that call are caught and rendered rather than propagating. `BlockComponent::renderComponent()` was changed to throw an `OutOfBoundsException` immediately when the block plugin is `Broken`, replacing a `#pre_render` callback that achieved the same effect indirectly. An `@internal` utility class `ComponentMetadataHelper::getNonAttributeComponentProperties()` was extracted to centralize the logic for filtering `Attribute`-typed props.

## Upgrade

Run database updates after upgrading Canvas:

```
drush updb
```

Two post-update hooks run automatically:
- `canvas_post_update_0001_track_props_have_required_flag_in_components`: backfills the `required` boolean into every prop entry in every version of every `Component` config entity.
- `canvas_post_update_0007_respect_prop_ordering`: reorders `prop_field_definitions` entries to match the prop order declared in each SDC or code component's schema.

No other manual steps are required.
