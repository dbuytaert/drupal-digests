## Summary

The `canvas pull` command now detects when local component files exist that have been deleted remotely and offers to remove them. Previously, pulling updates would only add new or updated components but leave behind local files for components that no longer exist on the server. This helps keep local development environments in sync with the remote state by cleaning up stale components.

## Impact

No upgrade or configuration changes required.

## Technical details

The pull command now compares remote component machine names against locally discovered components. Components that exist locally but not remotely are identified as "local-only" and counted in the pull summary. Before executing the pull, users are prompted to confirm deletion of these local-only components (defaulting to "No" for safety). If confirmed or running with `--yes`, the component directories are removed recursively. The `prepare()` method of pull tasks now returns both summary lines and a local-only count, and the `execute()` method accepts a `deleteLocalOnly` option to control deletion behavior. Results for deleted components are reported with "Deleted" status in the final summary.
