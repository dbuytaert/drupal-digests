## Technical details

Canvas Workbench is a local developer tool similar to Storybook but focused on Canvas Code Components and Canvas pages. After the initial version in [#3573867](https://www.drupal.org/node/3573867) and component mocks in [#3579801](https://www.drupal.org/node/3579801), this issue cleans up the UI.

The discovery system now tracks `projectRelativeDirectory` for each component and stores `projectRoot` in the discovery result. This allows the Workbench header to display the full project-relative path to the selected component or page in monospace font.

The main Workbench UI (`App.tsx`) replaces component variant buttons with a tabbed interface using the new `Tabs` component. The sidebar reorders to show pages before components and removes the warnings section. The reload frame button is removed from the header.

Warning display moves from a sidebar section to toast notifications using the `sonner` library. Warnings appear as toasts with infinite duration and are dismissed when they no longer apply. The `useEffect` hook tracks active toast IDs and dismisses stale ones when the preview manifest updates.

Preview errors now render inline using the new `Alert` component. The `PreviewFrameApp.tsx` error boundary stores the error message in state and displays it in a `PreviewErrorAlert` component instead of just posting a message to the parent frame. The preview area removes status text and debug sections.

Theme support uses a provider pattern in `theme-provider.tsx` that tracks system, light, and dark modes. The theme persists to localStorage and responds to system preference changes. A settings menu in the header lets users switch themes. The `Toaster` component from `sonner` integrates with the theme system.

The sidebar width reduces from 16rem to 12rem. The Geist Mono variable font is added for monospace text. Several shadcn/ui components are added: `Alert`, `Tabs`, and `Toaster`.
