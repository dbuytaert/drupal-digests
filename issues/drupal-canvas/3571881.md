## Summary

A bug in Canvas 1.1.0 caused fatal errors when site administrators tried to manage view modes that were not controlled by Canvas templates, such as Teaser or Search Index displays. The crash occurred when Layout Builder was installed, preventing administrators from configuring these view modes. This issue affected sites running Drupal CMS with Canvas 1.1.0 and made it impossible to manage certain content displays. The fix ensures that all view mode management pages work correctly, regardless of whether they use Canvas templates.

## Impact

No upgrade or configuration changes required.

## Technical details

The `ViewModeDisplayController` was attempting to build the entity view display form manually by retrieving a form object from the entity type manager and calling `getForm()`. When Layout Builder was installed, it altered the route to use `LayoutBuilderEntityViewDisplayForm`, which expects a specific entity to be passed. For view modes without a Canvas template, the controller's manual form building didn't properly set up the entity, causing `getLayoutBuilderUrl()` to be called on null.

The fix replaces the manual form building logic with a delegation to `HtmlEntityFormController`. Instead of using `EntityTypeManagerInterface` and `FormBuilderInterface` to manually construct the form, the controller now injects `HtmlEntityFormController` and calls its `getContentResult()` method. This ensures the standard Drupal form handling is used for non-Canvas view modes, which properly handles Layout Builder's route alterations.

The test coverage adds `testLayoutBuilderCompatibility()`, which installs Layout Builder, enables it on a teaser view mode, and verifies that visiting the display management page returns a 200 status code without errors. This prevents regression of the issue.
