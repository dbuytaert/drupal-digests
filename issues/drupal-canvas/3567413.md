## Summary

When you open a page, template, or region for editing in Canvas, component instances that reference an outdated version of their component are now automatically upgraded to the active (latest) version, provided the change is safe. Safe upgrades include adding optional props, adding slots, making props optional, changing default values, and expanding allowed media types on image props. Unsafe changes (such as removing props or adding new required props) still block auto-upgrade, leaving the instance on its old version. For site owners, the most visible effect is that new image media types automatically appear in existing image-prop component instances the next time those entities are edited, with no manual intervention required.

## Impact

Affects module maintainers.

- **API change:** `ComponentInterface::getSettings()` and `ComponentInterface::getSlotDefinitions()` now each accept an optional `?string $version = NULL` parameter; any implementing class must add it.
- **New API:** New `ComponentSource` attribute `updater` parameter and `ComponentInstanceUpdaterInterface` extension point for component source plugins to provide automatic instance upgrading logic.

## Technical details

The feature introduces a new `updater` parameter on the `ComponentSource` attribute, which accepts an FQCN implementing the new `ComponentInstanceUpdaterInterface` (marked `@internal`). That interface exposes three methods: `isUpdateNeeded(ComponentTreeItem)`, `canUpdate(ComponentTreeItem)`, and `update(ComponentTreeItem)`, returning the new `ComponentInstanceUpdateAttemptResult` enum (`NotNeeded`, `NotAllowed`, `Latest`). The concrete implementation `GeneratedFieldExplicitInputUxComponentInstanceUpdater` is wired to both the SDC and JsComponent sources; Block and Fallback sources pass `updater: FALSE`.

`ComponentSourceManager::updateComponentInstances(ComponentTreeItemList)` is the new orchestration method. It iterates the tree, resolves the updater class via the `classResolver`, and calls `isUpdateNeeded` + `canUpdate` before invoking `update`, which sets `component_version` on the item in place. This runs in `ApiLayoutController::buildRegion()` before building the client-side representation, and in `ApiConfigControllers::normalize()` before serving Pattern previews, ensuring both the editor form and the preview always reflect the active version.

`GeneratedFieldExplicitInputUxComponentInstanceUpdater::canUpdate` compares the currently loaded version's `prop_field_definitions` and `slot_definitions` (retrieved via the updated `Component::getSettings(?string $version)` and `Component::getSlotDefinitions(?string $version)`, which now accept an optional version argument) against the active version. Compatibility is checked using the new `StorablePropShape::fieldDataFitsIn(StorablePropShape)` method, which compares field type, cardinality, storage settings, and instance settings. Entity-reference fields receive special treatment: the active version's allowed target bundles must be a superset of the current version's bundles (covering the new-image-media-type use case). Changing the field widget or default values is explicitly allowed. The `StaticPropSource::hasSameShapeAs()` method was refactored to reuse `fieldDataFitsIn` rather than duplicating the comparison logic.

`VersionedConfigEntityBase` gains the protected helper `assertVersionExists(string $version)`, extracted from `loadVersion` to allow `getSettings` and `getSlotDefinitions` to call it without loading the version. `JavaScriptComponent::setProps()` was also fixed to prune removed props from the `required` list. A known limitation (adding a new required prop with a default value is blocked, not auto-upgraded) is tracked in [#3568602](https://www.drupal.org/node/3568602).

## Upgrade

If you maintain a custom component source plugin using the `ComponentSource` attribute, add the `updater` parameter explicitly:

```php
// No updater for this source type:
#[ComponentSource(
  id: 'my_source',
  // ... other params ...
  updater: FALSE,
)]

// Or provide a ComponentInstanceUpdaterInterface implementation:
#[ComponentSource(
  id: 'my_source',
  // ... other params ...
  updater: MyComponentInstanceUpdater::class,
)]
```

If you implement `ComponentInterface`, add the optional `?string $version = NULL` parameter to `getSettings()` and `getSlotDefinitions()`:

```php
public function getSettings(?string $version = NULL): array { ... }
public function getSlotDefinitions(?string $version = NULL): array { ... }
```
