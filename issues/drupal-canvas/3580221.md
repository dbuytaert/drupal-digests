## Summary

The Canvas CLI can now push and pull Brand Kit fonts, allowing font configuration to be version-controlled and deployed like other Canvas assets. When a `canvas.brand-kit.json` file is present in the project root, `canvas push` resolves each font family (via `unifont` for provider-based fonts, or a local file path), uploads the font files to the Drupal site, and syncs the full font list to the global Brand Kit. `canvas pull` fetches fonts from the global Brand Kit, downloads the files into a local `fonts/` directory, and writes local `src` entries back to `canvas.brand-kit.json`. Font sync is entirely opt-in: the file must be present for any font operations to run. The `administer brand kit` permission is no longer hidden from the Drupal permissions page when the `canvas_dev_mode` module is not installed.

## Impact

Affects module maintainers and site owners.

- **Config change:** New optional `canvas.brand-kit.json` file in the project root for font configuration; default CLI OAuth scope now includes `canvas:brand_kit`
- **Permission change:** `administer brand kit` permission is no longer hidden from the Drupal permissions page when `canvas_dev_mode` is not installed
- **New API:** New `createFontsPullTask` export in `pull.ts` and `pushFonts` / `buildFontPushPlannedResults` exports in `font-push.ts`; new TypeScript interfaces `FontsConfig`, `FontFamilyEntry`, `FontDefaults`, `FontAxisDefaults`, `BrandKitConfigFile` exported from `config.ts`

## Technical details

The global Brand Kit entity stores font configuration site-wide. Previously it was editor-only; the CLI had no way to version or deploy fonts. This change adds font sync to both push and pull as an optional step gated on the presence of `canvas.brand-kit.json` in the project root.

**Config (`packages/cli/src/config.ts`):** The `Config` interface gains a `fonts?: FontsConfig` field. At startup, `loadFontsFromBrandKitFile()` reads and parses `canvas.brand-kit.json` (if present) and stores the `fonts` block. New TypeScript interfaces describe the full font config shape: `FontsConfig`, `FontFamilyEntry` (a union of `LocalFontFamilyEntry` with a `src` path and `ProviderFontFamilyEntry` with an optional `provider`), `FontDefaults`, `FontProviderOptions`, and `FontAxisDefaults`. The `BRAND_KIT_CONFIG_FILENAME` and `BRAND_KIT_GLOBAL_ID` constants are exported. The default OAuth scope gains `canvas:brand_kit` (was `canvas:js_component canvas:asset_library`; now `canvas:js_component canvas:asset_library canvas:brand_kit`).

**Font push (`packages/cli/src/lib/fonts/font-push.ts`):** `pushFonts()` validates the config first (via `font-validate.ts`), then fetches the current Brand Kit to diff against remote. For provider-based families it calls `createFontResolver()` (a `unifont` wrapper in `font-resolver.ts`) and then `downloadResolvedFaces()` to fetch font files to temp paths. For local `src` entries it reads the file from disk. Variable font axes are extracted from the font binary using `opentype.js` (decompressing woff2 with `woff2-encoder` first) via `font-metadata.ts`. Axis tags get human-readable display names (e.g. `wght` â†’ `"Weight"`) for the Brand Kit UI. `axisDefaults` overrides are clamped to each axis min/max. Only variants not already present on the remote Brand Kit are uploaded; push replaces the remote font set in full so fonts not in config are deleted. All outcomes (create/update/delete/unchanged) are reported. An empty `families` array clears all fonts on the Brand Kit.

**Font pull (`packages/cli/src/lib/fonts/font-pull.ts`):** `pullFonts()` fetches the Brand Kit via `api.getBrandKit()`, skips variants already represented in `canvas.brand-kit.json` (matched by a normalized `family\0weight\0style` key), and for each new variant downloads the file via `api.downloadFile()` into a `fonts/` directory with slugified filenames. Variable font weight ranges are reconstructed from stored axes (`wght` min/max). `updateBrandKitConfig()` merges new `LocalFontFamilyEntry` objects into the existing file, preserving other top-level keys. `createFontsPullTask()` in `pull.ts` wraps this as a `PullTask` that runs alongside components and global CSS.

**OAuth (`CanvasOauthAuthenticationProvider`):** `BrandKit::ENTITY_TYPE_ID` is added to the list of protected config entity types so the existing config entity REST endpoints (`canvas.api.config.get`, `canvas.api.config.patch`, etc.) accept OAuth tokens for Brand Kit operations.

**`canvas.module`:** `canvas_form_alter()` is removed. This hook was hiding the `administer brand kit` permission entry from the Drupal permissions UI when `canvas_dev_mode` was not installed. Its removal means the permission is always visible.

New npm dependencies added to the CLI: `unifont ^0.7.4`, `opentype.js ^1.3.4`, `woff2-encoder ^2.0.0`.

## Upgrade

If you have `CANVAS_SCOPE` set explicitly in `.env` or your environment, add `canvas:brand_kit` to enable font push/pull:

```
CANVAS_SCOPE="canvas:js_component canvas:asset_library canvas:brand_kit"
```

If `CANVAS_SCOPE` is unset, the new default already includes `canvas:brand_kit`.

To opt in to font sync, create `canvas.brand-kit.json` in your project root:

```json
{
  "fonts": {
    "families": [
      {
        "name": "Inter",
        "provider": "google",
        "weights": ["400", "700"],
        "styles": ["normal", "italic"]
      },
      {
        "name": "My Font",
        "src": "fonts/MyFont-Regular.woff2",
        "weights": ["400"],
        "styles": ["normal"]
      }
    ]
  }
}
```

No changes are required if you do not create `canvas.brand-kit.json`; the push and pull commands continue to work as before.
