## Summary

Drupal Canvas now supports Tailwind CSS plugins in global CSS files. This makes it possible to use popular Tailwind extensions like typography styling, form controls, animations, and accessibility tools when building design systems. The initial release bundles the official typography plugin, which provides prose styles for content-rich elements. Teams can now create more sophisticated and production-ready user interfaces without hitting the previous limitations of Canvas's build process.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas now bundles `tailwindcss-in-browser` version 0.6.0, which adds support for loading Tailwind CSS plugins through JavaScript's native `import()` function. The update enables using the `@plugin` directive in global CSS with full HTTPS URLs or bare specifiers that resolve through an import map.

A new `GlobalImports` service centralizes the import map configuration that was previously embedded in `JsComponent::renderComponent()`. This service provides the list of available imports and cache-busting query strings. The import map is now attached at the controller level in `CanvasController`, making it available to all Canvas UI routes.

The `@tailwindcss/typography` plugin is now bundled in `packages/astro-hydration` and added to the global import map, allowing developers to use `@plugin "@tailwindcss/typography";` in their global CSS. The plugin is also exported through a new `tailwindcss-typography.ts` module and imported in the `Stub.jsx` component for browser availability.

Plugins loaded via full URLs (like `@plugin "https://esm.sh/@tailwindcss/typography@0.5.19";`) work in the browser but not in local Node.js environments using Canvas CLI, due to differences in how `import()` handles remote URLs. For compatibility between browser and local development, plugins should use bare specifiers with corresponding import map entries.
