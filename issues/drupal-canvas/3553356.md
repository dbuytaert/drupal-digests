## Summary

Canvas now supports Tailwind CSS `@plugin` directives in Global CSS. The `@tailwindcss/typography` plugin is bundled and immediately usable via `@plugin "@tailwindcss/typography";`. Any other plugin can be loaded by full URL (e.g. `@plugin "https://esm.sh/tailwindcss-animate";`), though URL-based plugins only work in the browser and will not compile under Canvas CLI local builds. Plugins can also be made available by bare specifier if their module is added to the Canvas import map.

## Impact

Affects module maintainers.

- **New API:** Tailwind CSS `@plugin` directives are now supported in Canvas Global CSS; `@tailwindcss/typography` is bundled and available by bare specifier out of the box.

## Technical details

The underlying enabler is a bump of `tailwindcss-in-browser` from 0.5.0 to 0.6.0, which adds Tailwind plugin loading via the native JavaScript `import()` function. This allows `@plugin` directives to reference either full HTTPS URLs or bare specifiers that appear in the page's import map.

To wire in `@tailwindcss/typography`, the commit introduces an `@internal` `GlobalImports` service (`src/GlobalImports.php`) that owns the complete import map previously inlined in `JsComponent::renderComponent()`. The map now includes an `@tailwindcss/typography` entry pointing to a bundled `tailwindcss-typography.js` file. Cache-busting is handled by `GlobalImports::getQueryString()`, which uses the module version string or falls back to Drupal's `AssetQueryStringInterface` when the version is `0.0.0` (development installs). The same `getQueryString()` is reused for `modulepreload` link hints in `JsComponent`.

The import map population was moved from `JsComponent` into `CanvasController`, because `CanvasController` returns an `HtmlResponse` directly, bypassing the `page_attachments` and `page_attachments_alter` hooks that normally process attachments. Placing `GlobalImports` here ensures plugins are available across the full Canvas UI, not just inside individual component renders. Contrib modules that need to inject additional import map entries must decorate `html_response.attachments_processor` with a `decoration_priority` lower than -100 to run before `ImportMapResponseAttachmentsProcessor`, as documented by wotnak in comment #9 of this issue.
