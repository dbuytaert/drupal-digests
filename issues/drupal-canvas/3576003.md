## Summary

The Canvas editor now supports multi-value integer and number fields with a modern popover-based interface. Users can edit numeric values by clicking on them, add new values with a clear button, reorder items with drag-and-drop handles, and remove items when appropriate. Required fields with only one value prevent deletion to ensure data integrity. The interface provides immediate visual feedback and validates numeric constraints like minimum, maximum, and step values before accepting changes.

## Impact

No upgrade or configuration changes required.

## Technical details

This feature extends the multi-value form UI pattern introduced for text fields ([#3573096](https://www.drupal.org/node/3573096)) to integer and number (float) field types. The `DrupalInputMultivalueForm` component renders a collapsed list view where each item shows a numeric preview. Clicking an item opens a popover dialog with a number input field.

The implementation adds HTML5 validation support for numeric constraints. When users commit a value by pressing Enter, the component checks the input element's validity using `checkValidity()` and displays native browser validation messages via `reportValidity()` if constraints are violated. This ensures min, max, and step attributes are enforced before updating the form state.

Remove button behavior adapts to field configuration. For required fields, the component queries the DOM to count rows using `data-canvas-row-count` and checks for the `.form-required` class. When only one item remains in a required field, the remove button is disabled. The component also passes numeric input attributes (min, max, step) from the Drupal render array through to the popover input.

CSS updates ensure proper styling for number and integer field types within the table-based multi-value layout. The implementation reuses existing drag-and-drop infrastructure and AJAX behaviors from Drupal's core tabledrag system.
