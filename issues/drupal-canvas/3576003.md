## Summary

The Canvas editor now supports multi-value integer and number (float) fields with the same popover-based interface previously available only for text fields ([#3573096](https://www.drupal.org/node/3573096)). Users can click a list item to open a popover, edit the numeric value, commit with Enter, add items with an "+ Add new" button, reorder with drag-and-drop handles, and remove items with a trash icon button. For required fields, the remove button is automatically disabled when only one value remains, preventing the field from being emptied. HTML5 numeric constraints (min, max, step) are enforced before a value is committed.

## Impact

No upgrade or configuration changes required.

## Technical details

The PHP hook `ReduxIntegratedFieldWidgetsHooks` previously checked only for `textfield` when marking elements with `#is_multivalue_form`. The `'number'` type (used for both integer and float widgets) was added to that check so the multi-value React component is activated for numeric fields too.

The `DrupalInputMultivalueForm` React component received several changes. A new `NumericInputAttributes` TypeScript interface was added to `DrupalAttribute.ts` with optional `min`, `max`, and `step` properties. The popover `TextField` now passes `type`, `min`, `max`, and `step` through from the Drupal render array attributes, enabling native number input behavior. On commit (Enter key or button click), the component calls `checkValidity()` on the input element and invokes `reportValidity()` to surface browser-native constraint errors rather than accepting an out-of-range or non-step-aligned value.

Remove-button enabling logic was extracted into `isRemoveButtonEnabled()`. It queries the DOM for the presence of a Drupal `remove_button` submit input in the table row, then reads the `data-canvas-row-count` attribute (now written by `canvas_stark_preprocess_field_multiple_value_form`) and checks for a `.form-required` element in the enclosing field wrapper. When the field is required and `rowCount === 1`, the popover remove button is rendered as disabled. The hidden input wrapper was converted from an inline `left: -9999px` style to a `visuallyHiddenInput` CSS module class. The `setTimeout`-based focus workaround was replaced by a `useEffect` that queries the actual `<input>` element inside `popoverTextFieldWrapperRef` once the popover opens.

On the template side, the "Add new" button was moved inside the `.multivalue-container` wrapper in the shared `field-multiple-value-form.html.twig`, and a redundant integer/number-specific template was deleted. Three new test fields (`field_cvt_unlimited_number`, `field_cvt_unlimited_int`, `field_cvt_required_number`) were added to the test article type, and a comprehensive parameterized Cypress suite covers render, edit, add, drag-and-drop, remove, and required-field behaviors across all three configurations.
