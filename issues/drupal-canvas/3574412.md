## Summary

Canvas now allows you to create content templates for any view mode, not just Full content. This means you can design custom layouts for teasers, cards, and other view modes used in lists, views, and previews. Previously, only the Full content view mode could be customized with Canvas templates, which limited how content appeared in different contexts across your site. This expands Canvas editing capabilities to match how Drupal content is actually displayed throughout a site.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas content templates were originally restricted to the Full view mode as a deliberate scope decision for the 1.0 release. This limitation was removed because alternate view modes like Teaser and Card can already be used in Views and other contexts, making template support for them immediately useful.

The restriction existed primarily in the UI layer. The TypeScript code filtered available view modes to only `full`, even though the server-side API already supported other view modes. Exposed slots (a feature for injecting field content into template regions) remain Full view mode only, but the ADR already documented that alternate view modes would not support exposed slots in 1.0.

The change removes the `mode === 'full'` filter from the template creation dialog and updates validation tests to prove that view modes like `teaser` work correctly. The `ContentTemplateAwareViewBuilder` already handled alternate view modes properly. New kernel tests prove templates render correctly for non-full view modes, and a Playwright test demonstrates the complete workflow of creating a Teaser template and verifying it renders on a Views-generated page.
