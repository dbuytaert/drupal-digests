## Summary

The Drupal Canvas CLI validate command now uses updated ESLint rules that check for currently unsupported import patterns in components. The new rules detect and flag imports of fonts (which should come from the brand kit), asset files (images, video, SVG), CSS side-effect imports, and helpers from within component directories. Components can still import from third-party packages, other components, and shared utility locations outside component directories. The validate command runs with the new rules by default, with a `--deprecated` flag available to use the previous rule set.

## Impact

No upgrade or configuration changes required.

## Technical details

The ESLint configuration in `@drupal-canvas/eslint-config` adds a new `component-imports` rule that validates import statements against Drupal Canvas's supported patterns. The rule now explicitly checks for and rejects font package imports (`@fontsource`), asset file imports (using extensions from the `@drupal-canvas/discovery` package), CSS side-effect imports, and imports from nested paths within component directories. It continues to support imports from third-party packages, other components via `@/components/`, and shared utilities from non-component directories.

The previous `component-imports` rule is renamed to `component-imports-deprecated` and moved to the deprecated rule set. The CLI validate command integrates with `discoverCanvasProject` from the `@drupal-canvas/discovery` package to automatically find components instead of requiring manual selection. The `--all` and `--components` options are now deprecated unless used with the new `--deprecated` flag.

Shared utilities like `resolveCanvasConfig` and asset extension constants are consolidated into `@drupal-canvas/discovery` to avoid duplication across packages.
