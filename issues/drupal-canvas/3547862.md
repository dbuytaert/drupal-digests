## Summary

The admin listing at `/admin/content/pages` in the Canvas module now shows a proper Operations column with Edit and Delete links for each Canvas page. Previously the listing had a custom workaround field that only linked to the Canvas editor, with no way to delete a page from the UI. The page title in the listing now also links to the live (canonical) page rather than showing as plain text.

## Impact

No upgrade or configuration changes required.

## Technical details

The root cause was that the `Page` entity type had no `list_builder` handler registered. The Views `entity_operations` field plugin requires a list builder to be present on an entity type in order to resolve and render entity operations (edit, delete, etc.). Without it, the operations field was simply unavailable in Views.

The fix registers `EntityListBuilder::class` as the `list_builder` handler directly on the `Page` entity's attribute definition in `Page.php`. On the view side, the old workaround field `edit_canvas_page` (using the `entity_link_edit` plugin, excluded from display, output as text) is removed and replaced with the standard `operations` field using the `entity_operations` plugin. At the same time, the `title` field's `link_to_entity` setting is flipped from `false` to `true` so that clicking the page title navigates to the canonical live page. A new functional test `CanvasPageListTest` asserts that a user with `edit canvas_page` and `delete canvas_page` permissions sees canonical, edit-form, and delete-form links on the listing page.
