## Summary

The Drupal Experience Builder module has been completely renamed to Drupal Canvas. The machine name changed from `experience_builder` to `canvas`, and the legacy `xb` shorthand prefix has been replaced throughout with `canvas`. This is a sweeping change that touches every layer of the codebase: module files, PHP namespaces, hook names, route names, permissions, entity type IDs, submodule names, library identifiers, JavaScript settings keys, API endpoint paths, and HTML output. Any contrib module, custom module, or theme that references Experience Builder or XB identifiers must be updated to use the new Canvas names.

## Impact

Affects module maintainers and site owners.

- **Permission change:** Permissions `create xb_page`, `edit xb_page`, and `delete xb_page` renamed to `create canvas_page`, `edit canvas_page`, and `delete canvas_page`.
- **API change:** PHP namespace `Drupal\experience_builder` renamed to `Drupal\canvas`; hooks renamed from `experience_builder_*` to `canvas_*`; library IDs, JS settings keys (`xb`/`xbData`/`xbExtension` → `canvas`/`canvasData`/`canvasExtension`), API paths (`/xb/api/v0/` → `/canvas/api/v0/`), and plugin namespace `Plugin\ExperienceBuilder` → `Plugin\Canvas` all changed.

## Technical details

The rename was executed via a `composer rename` script that performed search-and-replace across file contents, filenames, and directory names. Key identifier mappings include: PHP namespace `Drupal\experience_builder` → `Drupal\canvas`; hook prefix `experience_builder_` → `canvas_`; route prefix `experience_builder.` → `canvas.`; library key `xb-ui` → `canvas-ui`; `drupalSettings.xb` → `drupalSettings.canvas`; `drupalSettings.xbData` → `drupalSettings.canvasData`; `drupalSettings.xbExtension` → `drupalSettings.canvasExtension`; internal API base path `/xb/api/v0/` → `/canvas/api/v0/`; CSS class `xb-page` → `canvas-page`; HTML comment markers `xb-region-content-start-*` → `canvas-region-content-start-*`; and the plugin namespace `Plugin\ExperienceBuilder` → `Plugin\Canvas`.

Submodules were also renamed: `xb_vite` → `canvas_vite`, `xb_stark` → `canvas_stark`, `xb_personalization` → `canvas_personalization`, and test modules such as `xb_test_extension` → `canvas_test_extension`. The `xb_page` entity type became `canvas_page`. The field widget plugin definition key used in `hook_field_widget_info_alter()` changed from `'xb'` to `'canvas'`. The `drupalSettings.canvasExtension` key replaced `drupalSettings.xbExtension` for identifying third-party Canvas UI extensions. The `canvas_requirements()` hook checks for the `canvas_stark` theme and surfaces an admin warning if it is missing. An update hook `experience_builder_update_112000` that was present in the old `.install` file was removed in this commit (it had already run on existing sites).

## Upgrade

Any code integrating with Experience Builder must be updated. Common replacements:

**PHP namespace:**
```php
// Before
use Drupal\experience_builder\SomeClass;
// After
use Drupal\canvas\SomeClass;
```

**Hook implementations:**
```php
// Before
function mymodule_experience_builder_some_hook() {}
// After — hooks themselves were renamed; update per the new hook names in canvas.api.php
```

**`hook_field_widget_info_alter()` plugin definition key:**
```php
// Before
$info['my_widget']['xb'] = ['transforms' => [...]];
// After
$info['my_widget']['canvas'] = ['transforms' => [...]];
```

**Library dependency in `*.libraries.yml`:**
```yaml
# Before
dependencies:
  - experience_builder/xb-ui
# After
dependencies:
  - canvas/canvas-ui
```
