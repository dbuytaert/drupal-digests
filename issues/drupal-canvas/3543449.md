## Summary

If you have Drupal Canvas and Webform both installed, submitting a webform and being redirected to the rendered submission would crash with a fatal error. Canvas was replacing the view builder for every fieldable entity type, including `webform_submission`, whose native view builder implements a Webform-specific interface. Canvas's replacement view builder does not implement that interface, so any code calling Webform-specific methods on it would fail. The fix restricts Canvas's view builder override to `node` entities only, leaving every other entity type's view builder untouched. Support for additional entity types will be added later under [#3498525](https://www.drupal.org/node/3498525).

## Impact

No upgrade or configuration changes required.

## Technical details

The `ContentTemplateHooks::entityTypeAlter()` hook was iterating over all entity types and, for any that implemented `FieldableEntityInterface`, swapping in `ContentTemplateAwareViewBuilder` as the view builder. This is a greedy override: it fires at hook invocation time regardless of whether a content template exists for that entity type, and it discards the original view builder class (saving it under `ContentTemplateAwareViewBuilder::DECORATED_HANDLER_KEY`).

The problem is that some modules register view builders that implement module-specific interfaces beyond `EntityViewBuilderInterface`. Webform's `webform_submission` entity type uses a view builder implementing `WebformSubmissionViewBuilderInterface`, which exposes methods like `buildElements()`. Once Canvas replaced that class, any caller expecting those extra methods would get a fatal "undefined method" error, even with no content templates configured at all.

The one-line fix adds `&& $entity_type->id() === 'node'` to the condition in `ContentTemplateHooks::entityTypeAlter()`, so only `node` entities receive the `ContentTemplateAwareViewBuilder` override. A `@todo` comment references [#3498525](https://www.drupal.org/node/3498525), the meta issue that will eventually generalize this to other entity types in a safe, opt-in way.
