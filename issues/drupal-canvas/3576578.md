## Summary

Canvas now provides clearer error messages when component developers misconfigure array props. Previously, when a component schema defined an array with `maxItems` set to less than 2, developers received a vague "Nonsensical array size limit specified" error. The new error message explains the specific problem, shows the actual value, and suggests using a non-array type for single-value props. This helps developers understand why arrays must allow at least two items to be useful in Canvas.

## Impact

No upgrade or configuration changes required.

## Technical details

When Canvas added support for `type: array` props in a previous issue, it included a generic exception for the edge case where `maxItems < 2`. This made little sense for Canvas because a single-item array is just a pointless wrapper for a component prop, and zero or negative values make even less sense.

This issue improved developer experience by replacing the vague error message with a helpful one. The new message appears in two places: `ComponentMetadataRequirementsChecker` (which validates component metadata) and `JsonSchemaType::computeStorablePropShape()` (which interprets JSON schema for storage). 

The enhanced message now shows the actual `maxItems` value received and explicitly suggests using a non-array type for single-value props. A test component called `image-gallery-nonsensical` was added with `maxItems: 1` and a photo of a judging dog to verify the validation works correctly. This pays off technical debt from the initial array support implementation that intentionally left this validation incomplete.
