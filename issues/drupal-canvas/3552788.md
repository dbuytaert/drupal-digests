## Summary

Creating a new release of the Canvas module previously required manual steps: building the UI assets locally, committing them, creating a tag, and pushing to drupal.org. This process is now fully automated through GitLab CI. When maintainers create a version tag (like `v1.0.3`), the CI pipeline automatically builds the UI, commits the built assets to a detached commit, and creates the properly formatted release tag that drupal.org expects. This reduces the risk of human error and makes the release process faster and more consistent.

## Impact

No upgrade or configuration changes required.

## Technical details

The Canvas module stores UI assets built from source but doesn't commit the built files to the main branch. Previously, a shell script (`scripts/tag-release.sh`) semi-automated the release process by cloning the repository to a temporary directory, building assets, committing them, tagging, then reverting the build artifacts. This approach required local execution and manual verification.

The new `.gitlab-ci.yml` release job eliminates manual work. When a tag matching `/^v\d+\.\d+\.\d+$/` (like `v1.0.3`) is pushed, the job builds UI assets using `npm ci` and `npm run build` in the `ui/` directory. It temporarily modifies `.gitignore` to un-ignore `ui/dist/` and `packages/astro-hydration/dist/`, stages these directories, and creates a detached commit containing the built assets.

The job then creates a second tag without the `v` prefix (transforming `v1.0.3` to `1.0.3` using `cut -c2-`). This unprefixed tag points to the commit with built assets and is what drupal.org recognizes as the official release. Because GitLab CI checks out tags in detached HEAD state, this release commit never pollutes the `1.x` branch. The process uses a `GITLAB_CI_RELEASE_TOKEN` to authenticate git operations and creates a release archive as a CI artifact.

The `.gitignore` file was consolidated to manage exclusions at the repository root rather than in subdirectories. The old `scripts/tag-release.sh` and duplicate `.gitignore` files in `ui/` and `packages/astro-hydration/` were removed. New documentation in `docs/release-process.md` explains the automated flow.
