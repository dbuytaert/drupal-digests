## Summary

The Canvas code component editor now provides a graphical interface for marking a prop as multi-valued. Component builders see an "Allow multiple values" checkbox for supported prop types (text, integer, number, link, image, video, date, and list variants). Once enabled, a dropdown lets them choose between unlimited mode (dynamic add/remove buttons per value) and limited mode (a fixed count with increment/decrement controls). Example values in either mode can be reordered by drag-and-drop. Boolean and formatted text props do not show this option. The feature is visible only when Canvas dev mode is active. Persistence of this configuration is handled separately in [#3572553](https://www.drupal.org/node/3572553), and required-field validation for multi-value props is deferred to [#3576124](https://www.drupal.org/node/3576124).

## Technical details

The `Props.tsx` component now renders an "Allow multiple values" checkbox below each prop's type-specific form, gated on `getCanvasSettings()?.devMode` and a list of supported `derivedType` values. Checking the box dispatches `updateProp` with `allowMultiple: true`, sets `items` to the appropriate JSON Schema item descriptor (e.g. `{ type: 'string' }` for text, `{ type: 'string', format: prop.format }` for date/link), initialises `example` to `[]`, and sets `valueMode` to `'unlimited'` and `limitedCount` to `1`. Unchecking clears all of those fields and resets `example` to an empty string. The `VALUE_MODE_LIMITED` and `VALUE_MODE_UNLIMITED` string constants from `@/types/CodeComponent` drive the dropdown and conditional rendering.

A new `FormPropTypeArray` component handles the example-value UI for simple types (string, integer, number) when `allowMultiple` is true. For link, date, and enum types the existing `FormPropTypeDate`, `FormPropTypeLink`, and `FormPropTypeEnum` components receive the new `allowMultiple`, `valueMode`, and `limitedCount` props and branch internally. All multi-value UI shares a new `PropValuesSortableList` component that wraps `dnd-kit` for drag-and-drop row reordering. Utility helpers `createArrayDragEndHandler`, `handleArrayAdd`, `handleArrayRemove`, and `handleArrayValueChange` in `arrayPropUtils` keep array mutations out of the render path. A `createArrayWithCount` helper in `Props.tsx` manages resizing the example array when the limited count changes, preserving existing values and padding with empty slots. For list prop types the limited count is capped at the number of defined non-empty enum options.
