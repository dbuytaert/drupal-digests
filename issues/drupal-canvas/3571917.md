## Summary

The Canvas code component editor now provides a graphical interface for configuring props that accept multiple values. Component builders can choose between unlimited values (with add/remove buttons) or a fixed limit, and can reorder example values via drag-and-drop. This UI supports text, link, image, video, integer, number, date, and list prop types. Boolean and formatted text props remain single-value only. The feature is front-end only and requires dev mode to be visible.

## Impact

No upgrade or configuration changes required.

## Technical details

Previously, component builders had no UI to mark a prop as multi-valued when creating code components in Canvas. This issue adds checkbox controls to enable multi-value mode for compatible prop types. Once enabled, a dropdown lets builders choose between limited (fixed count with numeric input and increment/decrement buttons) or unlimited (dynamic add/remove buttons) modes. Example values render as draggable rows powered by dnd-kit, preserving order during reordering. Type changes reset multi-value state to prevent schema mismatches.

The implementation introduces `FormPropTypeArray` for simple types (string, integer, number), extends `FormPropTypeLink`, `FormPropTypeDate`, and `FormPropTypeEnum` to handle both single and array modes, and adds `PropValuesSortableList` for reusable drag-and-drop rows. The UI syncs with Redux state via the `allowMultiple`, `valueMode`, `limitedCount`, `items`, and `example` fields on `CodeComponentProp`. List types automatically cap the limited count at the number of defined enum options. Validation states track per-input validity for links and dates.

The change is gated behind Canvas dev mode and omits backend persistence, which is handled in [#3572553](https://www.drupal.org/project/canvas/issues/3572553). Required field validation for multi-value props is deferred to [#3576124](https://www.drupal.org/project/canvas/issues/3576124). Over 1000 lines of vitest coverage ensure drag-and-drop, mode switching, count adjustment, and value persistence work correctly.
