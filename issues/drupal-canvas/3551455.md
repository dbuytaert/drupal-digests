## Summary

`HostEntityUrlPropSource` in the Canvas module previously only generated absolute canonical URLs for the host entity. It now supports both absolute and relative URLs, controlled by an `absolute` flag. The prop source suggester has been updated to automatically propose an absolute URL when the target prop is typed `format: uri` or `format: iri`, and a relative URL when the prop is typed `format: uri-reference` or `format: iri-reference`. This matters for cacheability: absolute URLs always vary by the `url.site` cache context, so using relative URLs where the schema allows them avoids that overhead.

## Impact

Affects module maintainers.

- **New API:** New `$absolute` constructor parameter on `HostEntityUrlPropSource`; stored JSON gains an `absolute` boolean key

## Technical details

`HostEntityUrlPropSource` gains a constructor that accepts `public readonly bool $absolute`. The `toArray()` method now serializes the `absolute` flag into the stored JSON (e.g. `{"sourceType": "host-entity-url", "absolute": true}`). The `parse()` method defaults `$absolute` to `TRUE` when the key is absent, so existing stored data without the flag continues to behave as before. The `evaluate()` method passes `$this->absolute` to `Url::setAbsolute()` instead of always calling `setAbsolute()` with no argument.

`PropSourceSuggester::matchHostEntityUrlPropSources()` previously returned a single hardcoded `new HostEntityUrlPropSource()`. It now computes `$supports_only_absolute_urls` from the target prop's string format: `JsonSchemaStringFormat::Uri` and `JsonSchemaStringFormat::Iri` require absolute URLs, while `uri-reference` and `iri-reference` do not. The suggester then constructs `new HostEntityUrlPropSource(absolute: $supports_only_absolute_urls)` and returns that single context-appropriate suggestion.

The `label()` and `asChoice()` methods update accordingly: labels are now `'Absolute URL'` or `'Relative URL'` (replacing the single fixed label `'Canonical absolute URL'`), and `asChoice()` returns `host-entity-url:absolute:canonical` or `host-entity-url:relative:canonical`. The PHPStan type alias `HostEntityUrlPropSourceArray` is updated to `array{sourceType: string, absolute?: bool}`. Generalised URL option passthrough (query strings, fragments, etc.) was deliberately excluded to avoid exposing `\Drupal\Core\Url`'s full API as a leaky abstraction.

## Upgrade

Existing stored prop sources with no `absolute` key continue to work unchanged; `parse()` defaults to `TRUE`. When Canvas re-saves a component instance the serialized form will gain the explicit `absolute` key:

```json
// Before (still valid, parsed as absolute: true)
{"sourceType": "host-entity-url"}

// After re-save with absolute URL
{"sourceType": "host-entity-url", "absolute": true}

// After re-save with relative URL
{"sourceType": "host-entity-url", "absolute": false}
```

No manual migration is needed. Code that constructs `HostEntityUrlPropSource` directly must now pass the `$absolute` argument:

```php
// Before
new HostEntityUrlPropSource()

// After
new HostEntityUrlPropSource(absolute: true)
// or
new HostEntityUrlPropSource(absolute: false)
```
