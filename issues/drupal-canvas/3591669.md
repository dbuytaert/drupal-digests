# #3591669: Add heuristics for exposing fields/props to the content-entity-reference selection UI

## Summary

In the Canvas module's `content-entity-reference` "Typed Data browser" UI (used by Code Component Developers to wire entity data to component props), the field and property picker previously offered items that were either useless or outright dangerous: internal metadata fields, duplicate image URL properties, translation and revision bookkeeping fields, unprocessed raw text, non-navigable raw URIs (such as `entity:node/1` or `public://image.jpg`), and datetime properties that carried `DrupalDateTime` objects instead of strings (causing a `preg_match()` TypeError crash at render time). This change adds a set of heuristics that trim the picker down to only what a Code Component Developer can meaningfully consume, and adds matching server-side validation so expressions targeting a raw URI cannot be saved by any route.

## Impact

No upgrade or configuration changes required.

## Technical details

The core problem was that `DataDefinitionInterface::isInternal()` cannot distinguish an explicit `->setInternal(TRUE)` from the value it silently defaults to for all computed properties when the flag is unset: both return `TRUE`. The original guard `isInternal() && !isComputed()` therefore missed explicitly-internal computed properties such as the `date` property on datetime fields (marked internal by `DateTimeItemOverride`). Two new static helpers in `TypedDataHelper` fix this: `isExplicitlyInternal()` reads the raw definition array to detect an explicit mark, and `isEffectivelyInternal()` combines that with the non-computed case to give a single correct answer. All call sites in `ApiUiContentEntityReferenceControllers`, `EntityFieldExpressionMustNotTargetInternalPropertyConstraintValidator`, and `EntityFieldPropSourceMatcher` are updated to use these helpers. A new PHPStan rule `RequireTypedDataHelperForInternalCheckRule` enforces that no future code calls `->isInternal()` or reads `$definition['internal']` directly on a `DataDefinitionInterface`.

Beyond the internal-property fix, a new `isConsideredIrrelevant()` method on the controller hides translation and revision bookkeeping base fields that are not marked internal in core (`default_langcode`, `revision_translation_affected`, `revision_log_message`, `content_translation_source`, `content_translation_outdated`). For the image field, `src_with_alternate_widths` and `srcset_candidate_uri_template` are excluded by name from the picker, and `src` is relabelled "Image URL". For formatted text fields (`TextItemBase` subtypes), `value` and `summary` are excluded, keeping only `processed`, `summary_processed`, and `format`. For URI-typed properties, a new `TypedDataHelper::isRestrictedToHttpSchemes()` helper checks whether a `UriSchemeConstraint` limited to `http`/`https` is present; properties lacking it (link's raw `uri`, a file's stream-wrapper `value`) are excluded from the picker and rejected by the new `EntityFieldExpressionMayOnlyTargetResolvableUrisConstraint`, which is wired into the `canvas.js_component.*` config schema alongside the existing `EntityFieldExpressionMustNotTargetInternalProperty` constraint.
