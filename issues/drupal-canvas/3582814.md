## Summary

The `@drupal-canvas/create` CLI (version 1.3.0) gains a new `--agents` flag that lets developers control AI agent compatibility symlink creation without interactive prompts. Previously, the agent setup step always showed an interactive prompt when a `.agents/skills` directory was present, blocking fully unattended use in CI pipelines or scripts. Now callers can pass a comma-separated agent list (e.g. `--agents claude-code,roo`) to set up symlinks without prompting, pass `--agents none` to skip the step explicitly, or omit the flag to retain the existing interactive prompt on TTY runs. On non-TTY runs with no flag provided, the step is silently skipped.

## Technical details

The change adds a new `--agents <agents>` option to the `commander` program in `src/cli.js`. Parsing is delegated to a new `parseAgentSelection()` function in `src/lib/agent-selection.js`, which splits on commas, deduplicates with `Set`, rejects unknown agent names (via a new `isAgentType()` guard in `agents.js`), rejects universal agents that already use `.agents/skills`, rejects mixing `none` with other names, and returns `undefined` when the flag is omitted (preserving the "undecided" state). The resolved `AgentType[] | undefined` value is threaded through the `Context` type and passed to `createApp()`, which forwards it to `setupAgentSkills()` alongside a new `interactive` boolean derived from `process.stdin.isTTY && process.stdout.isTTY`. Inside `setupAgentSkills()` the logic is: if `selectedAgents` is a defined array, use it directly and skip the prompt; if it is `undefined` and `interactive` is true, call `promptForAgents` as before; if it is `undefined` and `interactive` is false, return immediately without output. An empty explicit selection (`[]`) prints "No additional agents selected. Skipping compatibility setup." and returns early. The README is also updated to document the new flag and to reflect the currently available predefined templates (`acquia-nebula`, `balintbrews-canvas-starter`).
