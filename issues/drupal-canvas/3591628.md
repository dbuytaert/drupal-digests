# #3591628: On Drupal 11.3, Components are not regenerated when configuration changes (e.g. creating a media type)

- **Project:** Drupal Canvas
- **Type:** Critical bug report
- **Version:** 1.x-dev
- **Status:** Fixed
- **Discussion:** [#3591628](https://git.drupalcode.org/project/canvas/-/work_items/3591628) · 2 contributors · 5 comments

## Summary

On Drupal 11.3+, Canvas Components were not regenerated when site configuration changed (for example, adding a new media type) unless a full cache rebuild was performed first. This meant that widgets expected to reflect the new configuration (such as the media library selector on an image prop) would remain stale until `drush cr` was run. The fix resolves the issue for fresh installs and any install sequence where `canvas_stark` is installed before Canvas has been used for the first time.

## Impact

No upgrade or configuration changes required.

## Technical details

Since Drupal 11.3, `ThemeInstaller` calls `DrupalKernel::updateThemes()` when a theme is installed or uninstalled, which rebuilds the entire service container. Canvas's module install hook installs `canvas_stark` partway through, triggering that container rebuild. After that point, the `ComponentSourceHooks` service object — and all services it had injected, including `ComponentSourceManager` and `PersistentPropShapeRepository` — are orphaned instances pointing at the pre-rebuild container. Any prop shapes written to the orphaned `PersistentPropShapeRepository` are lost because its `::destruct()` is never called; the live container's repository starts cold, so Components are never persistently generated.

The fix injects `DrupalKernelInterface` into `ComponentSourceHooks` using `#[Autowire(service: 'kernel')]`. In the module install hook path (`hook_install`), instead of calling `$this->rebuild()` (which uses the stale injected `$this->componentSourceManager`), the code now calls `$this->kernel->getContainer()->get(ComponentSourceManager::class)` to obtain the live post-rebuild instance, then calls `generateComponents()` on that. This ensures prop shapes land in the cache that belongs to the live container and will actually outlive the request.
