# #3586959: Validate image prop example URLs in Code Component metadata files

- **Project:** Drupal Canvas
- **Type:** Normal feature request
- **Version:** 1.x-dev
- **Status:** Fixed

## Summary

Canvas Code Component authors now get an ESLint error if their `component.yml` example values for `canvas.module/image` props use relative or root-relative `src` URLs instead of fully-qualified ones. The error message recommends using a placeholder such as `https://placehold.co/600x400`. This gives faster feedback during local development rather than waiting for backend schema validation to catch the problem. The fix shipped in `@drupal-canvas/eslint-config@0.3.0` and `@drupal-canvas/cli@0.14.0`.

## Impact

Affects module maintainers.

- **New API:** New ESLint rule `component-image-example-urls` added at severity `error` to the `required` config in `@drupal-canvas/eslint-config`; enforces fully-qualified URLs for `canvas.module/image` prop example `src` values in `component.yml` files.

## Technical details

The issue adds a new ESLint rule `component-image-example-urls` to the `required` config in `@drupal-canvas/eslint-config`. The rule lives in `rules/component-image-example-urls.js` and operates on the `yaml-eslint-parser` AST. It only activates for files matched by `isComponentYmlFile` (i.e. `component.yml`). For each entry in `props.properties`, `hasImageRef` checks whether the prop's mapping contains a `$ref` equal to `json-schema-definitions://canvas.module/image`, or whether its `items` mapping does (covering array-of-image props). When an image prop is found, `getInvalidImageExampleNodes` walks the `examples` sequence and inspects each image object's `src` scalar. `isFullyQualifiedUrl` uses the standard `URL` constructor and requires both a non-empty `protocol` and `hostname`; anything else (relative paths, root-relative paths) produces an ESLint error with a message pointing to `https://placehold.co/600x400`. The rule is registered at severity `'error'` alongside existing rules such as `component-dir-name` and `component-prop-names`. The underlying backend schema gap that permits the same invalid values through the JSON schema validator is tracked separately in [#3586958](https://www.drupal.org/node/3586958).

## Upgrade

Update the ESLint config package directly:

```
npm install @drupal-canvas/eslint-config@0.3.0
```

Or, if using the Canvas CLI, upgrade to the new release which pulls in the updated config automatically:

```
npm install @drupal-canvas/cli@0.14.0
```

After upgrading, any `component.yml` file that has an image prop example with a relative or root-relative `src` (e.g. `./hero.jpg` or `/sites/default/files/hero.jpg`) will produce an ESLint error. Replace those values with fully-qualified URLs such as `https://placehold.co/600x400`.
