## Summary

Canvas component instances can be configured to populate their props from entity fields using `DynamicPropSource` (for example, linking a Hero component's heading to a node's title field, or following a reference to the author's name). Before this fix, no access checks were performed when resolving those linked values, meaning a component could silently expose field or entity data to users who lack view access to that data. The fix adds access checking to the evaluation pipeline: if the current user cannot view the entity or field being resolved, the prop falls back to `NULL`, the component shows its fallback "failed to render" output, and a warning is logged. Component instance edit forms are unaffected and continue to load normally.

## Impact

Affects site owners.

- **Security:** Information disclosure: `DynamicPropSource` previously resolved entity and field values with no access checks, exposing restricted data; evaluation now enforces `view` access and falls back to NULL on denial.

## Technical details

The evaluation of `DynamicPropSource` expressions is handled by `Evaluator` in `Drupal\canvas\PropExpressions\StructuredData`. A new `Evaluator::validateAccess()` static method now runs before any field or entity value is resolved in the `DynamicPropSource` code path. It calls `access('view', NULL, TRUE)` on both the entity and the specific `FieldItemListInterface` being accessed. If access is not allowed, it throws a `CacheableAccessDeniedHttpException` whose cacheability metadata includes both the access result and the entity or field, ensuring that downstream render caches can vary or invalidate correctly.

The caller-side handling lives in `GeneratedFieldExplicitInputUxComponentSourceBase::getExplicitInput()`. Previously that method used `array_map` to evaluate all prop sources in one pass. It now iterates explicitly and wraps each `PropSource::parse()->evaluate()` call in a try/catch for `AccessDeniedHttpException`. On a catch, it logs a structured warning via the canvas logger channel (recording the prop name, component UUID, raw input, and exception message) and stores `NULL` as the resolved value for that prop. The unchanged `source` key in the returned array still carries the original configured expression, so the component instance form can render it without re-evaluating the inaccessible value.

To support the new logger dependency, both `GeneratedFieldExplicitInputUxComponentSourceBase` and its concrete subclass `SingleDirectoryComponent` were updated to accept and inject `logger.channel.canvas` via their constructors and `create()` factories. Many kernel test classes received `UserCreationTrait` and explicit `setUpCurrentUser()` calls, because the evaluator now requires a current user context with appropriate permissions for any test that exercises `DynamicPropSource` resolution.
