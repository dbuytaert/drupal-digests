## Summary

Site owners using the Translation Management Tool (TMGMT) to translate content entities (nodes, Canvas pages, etc.) that contain `component_tree` fields can now see and edit each component instance's translatable props as individual strings in the TMGMT job item review form. Previously, without a field processor, TMGMT treated the entire `component_tree` field as an opaque blob, making content translation impractical. The change also ensures the TMGMT review form correctly loads CKEditor 5 for rich-text component props.

## Impact

Affects module maintainers.

- **New API:** New `ComponentTreeFieldProcessor` TMGMT field processor and `ComponentInputsTranslatablesExtractor` shared service added to the Canvas module.

## Technical details

The new `ComponentTreeFieldProcessor` class in `src/Tmgmt/ComponentTreeFieldProcessor.php` extends `DefaultFieldProcessor` from `tmgmt_content`. Its `extractTranslatableData()` method converts each `ComponentTreeItem`'s `inputs` JSON property into a `ComponentInputsMapping` typed config object and delegates to `ComponentInputsTranslatablesExtractor` to discover translatable props via config schema metadata. `setTranslations()` writes accepted translations back into the correct nested positions in the raw `inputs` array using a recursive walker (`writeNestedTranslationsToInputs()`), necessary because component source props can have translatable values at arbitrary nesting depth with no field item to delegate to. The processor is registered via `TmgmtHooks::fieldInfoAlter()` by setting `tmgmt_field_processor` on the `component_tree` field info entry when `tmgmt_content` is installed.

The extraction logic previously embedded in `ComponentInputsConfigProcessor` was extracted into a new `ComponentInputsTranslatablesExtractor` service so it can be shared between both the config-entity processor (`ComponentInputsConfigProcessor`) and the new content-entity processor. `ComponentInputsConfigProcessor` is now a thin wrapper around this service. As part of this refactor, `TmgmtHooks` takes over the `hook_config_schema_info_alter` registration of `ComponentInputsConfigProcessor` that previously lived in `canvas_dev_translation`'s `ConfigTranslationSupportHooks`.

`tmgmt_job_item_edit_form` is added to the list of form IDs that use static prop sources in `ReduxIntegratedFieldWidgetsHooks`, ensuring Redux-integrated field widgets initialise correctly inside the TMGMT review form. A filter in `ComponentInputsTranslatablesExtractor` works around a TMGMT upstream bug ([#3601246](https://www.drupal.org/node/3601246)) where a `translatable type: ignore` element with an array `#text` value would crash later TMGMT processing.
