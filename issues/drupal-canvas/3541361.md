## Summary

In the Canvas ContentTemplate UI, optional image and video fields were not appearing as linkable data sources for SDC component props with a `type: object` shape (such as an image or video prop). Additionally, when a matching optional field was selected and the field happened to be empty, Drupal threw a `Twig\Error\RuntimeError` with SDC validation errors about NULL values. Both problems are now fixed: optional image/video fields show up as candidates, and selecting an empty optional field no longer causes a render crash — Canvas simply omits the prop when the field is empty.

## Impact

No upgrade or configuration changes required.

## Technical details

Two separate bugs were fixed across two commits.

**Commit 1 — Shape matching (`JsonSchemaFieldInstanceMatcher`):** Three related problems in `JsonSchemaFieldInstanceMatcher` prevented optional `type: object` prop matches from being found.

First, sub-props of an optional parent object were being evaluated as required even when the parent was optional. The fix changes the `$sub_required` argument passed to `::matchEntityPropsForScalar()` from the bare sub-prop flag to `$is_required_in_json_schema && $sub_required`, correctly propagating optional status.

Second, the code that iterates over `target_bundles` to find bundle-specific fields was gated on `count($target_bundles) > 1`, so single-bundle entity reference fields (the common case for image/video media fields) were never walked. The fix changes the condition to `count($target_bundles) > 0`.

Third, `::getFieldDefinitions()` only called `entityFieldManager->getFieldDefinitions()` when exactly one bundle was specified. For bundleless entity types like `User` (which has no `bundle` key), the code fell back to `getBaseFieldDefinitions()`, which excludes bundle fields like `user_picture`. The fix detects bundleless entity types via `EntityTypeManagerInterface::getDefinition()->hasKey('bundle')` and treats the entity type ID itself as the bundle in that case. This required adding `EntityTypeManagerInterface` as a new constructor dependency to `JsonSchemaFieldInstanceMatcher`, covered by a `canvas_post_update_0006_add_service_argument` update hook.

A fourth fix works around a long-standing Drupal core bug: `EntityReferenceItem::propertyDefinitions()` never constrains its `entity` property to a target bundle (see core issue [#3531905](https://www.drupal.org/node/3531905) and [#2169813](https://www.drupal.org/node/2169813)). `::getConstrainedTargetDefinition()` now calls the new `BetterEntityDataDefinition::createFromBuggyInCoreEntityDataDefinition()` factory method to convert the core `EntityDataDefinition` into a `BetterEntityDataDefinition` with correct bundle constraints applied. `BetterEntityDataDefinition::setBundles()` also gained a `$this->propertyDefinitions = NULL` line to invalidate the cached property definitions when bundles are updated.

**Commit 2 — Component source hydration (`GeneratedFieldExplicitInputUxComponentSourceBase`):** Once shape matching correctly surfaced optional fields, selecting an empty optional field caused `hydrateComponent()` to pass `{src: NULL, alt: NULL, width: NULL, height: NULL}` to SDC, which triggered a `ComponentValidator` error. The fix adds a post-resolution pass in `::hydrateComponent()`: for any optional prop whose resolved value is `NULL`, the prop is dropped from the hydration array entirely; for optional `type: object`-shaped props (identified via `FieldTypeObjectPropsExpression`), if all key-value pairs are NULL (checked via `empty(array_filter($resolved_value))`), the entire prop is likewise omitted. This approach sidesteps the active core issue [#3531905](https://www.drupal.org/node/3531905) by ensuring Canvas never passes `NULL` for typed optional SDC props. New test coverage is provided in `GeneratedFieldExplicitInputUxComponentSourceBaseTest`, extended by both `SingleDirectoryComponentTest` and `JsComponentTest`.
