# #3591638: Add a computed `src` field property to the image field type

## Summary

Canvas's image field type now exposes a `src` field property that code-component developers can use to bind an image URL to a component prop. Previously the only browser-ready URL was named `src_with_alternate_widths`, which leaked an internal implementation detail into the data-selection UI. The new `src` is a clean alias: it resolves to the same URL (including the `?alternateWidths` query string needed for Canvas's responsive-image handling) but is presented with a name that is meaningful to component authors. Automatic shape matching and content-template suggestions are unaffected and continue to resolve `src_with_alternate_widths`.

## Impact

Affects module maintainers.

- **New API:** Image field type in Canvas now exposes a computed `src` property (alias of `src_with_alternate_widths`) for direct use in code-component data selection.

## Technical details

The change is in `ImageItemOverride::propertyDefinitions()`. After the existing `src_with_alternate_widths` property is constructed (a computed `DataDefinition` backed by `ComputedUrlWithQueryString`), a new `src` property is added by cloning it and calling `setSetting('is source for', 'src_with_alternate_widths')` on the clone. The `is source for` setting is the same mechanism already used in `TextWithSummaryItemOverride` to exclude raw `value`/`format` properties in favour of `processed`. In `EntityFieldPropSourceMatcher`, any property carrying `is source for` is skipped during shape matching iteration (`continue`), which keeps `src` out of automatic prop matching while still making it available for explicit selection. Both `src` and `src_with_alternate_widths` produce identical dependency graphs, as confirmed by the test case at line 1011 of `PropExpressionTest`. The `@todo` on this property points to [#3464003](https://www.drupal.org/node/3464003), which tracks replacing computed field properties with adapters; that change will require an update path for any component trees that consume `src`.
