## Summary

The Canvas module has adopted `shipmonk/dead-code-detector` 1.1.1 as a PHPStan extension to automatically flag unused methods, properties, and classes during CI. Because Drupal's dynamic dispatch patterns produce many false positives, eleven custom usage providers were written to teach the detector about hooks, routing callbacks, DIC service calls, schema validation callbacks, and test framework conventions. Once false positives were eliminated, the analysis confirmed and removed approximately 200 lines of genuinely dead code: a whole exception class, several public methods, unused constructor parameters, and unreferenced test properties.

## Impact

Affects module maintainers.

- **API change:** Several public symbols removed as confirmed dead code: `InvalidRequestBodyValue` class, `JsonSchemaType::isIterable()` and `::isTraversable()`, `withDelta()` on three prop expression classes, and three methods on `CanvasResourceLinkCollection` (which is `@internal`).

## Technical details

The dead-code detector is wired into `phpstan.neon` by tagging eleven new provider classes with `shipmonk.deadCode.memberUsageProvider`. All providers live in `phpstan_rules/Canvas/PHPStan/UsageProvider/` and extend `ReflectionBasedMemberUsageProvider`, returning `VirtualUsageData` for symbols Drupal invokes through paths static analysis cannot follow.

The providers cover: `DrupalHookUsageProvider` marks methods carrying `#[Hook]` and their DIC-instantiated constructors; `DrupalRoutingUsageProvider` parses all canvas `*.routing.yml` files at analysis time to identify `_controller` and `_title_callback` classes, and marks `access()` on classes under `Drupal\canvas\Access\`; `DrupalServiceCallsUsageProvider` parses `*.services.yml` `calls:` directives; `ConfigSchemaValidationUsageProvider` parses `*.schema.yml` `callback:` entries; `DrupalTestUsageProvider` handles LSB reads of test properties (`$modules`, `$defaultTheme`, `$configSchemaCheckerExclusions`, etc.), abstract-base property writes, and polymorphic test method overrides; `CanvasApiContractUsageProvider` suppresses false positives on all canvas interface and abstract-class methods and their implementations; `CanvasTraitUsageProvider` works around a PHP 8.0+ change where `ReflectionMethod::getDeclaringClass()` returns the using class instead of the trait, breaking ShipMonk's candidate matching; `DrupalDynamicPropertyUsageProvider` covers `EntityBase` and `Constraint` subclass properties written via `$this->$key = $value` dynamic assignment; `DrupalConfigTranslationUsageProvider`, `DrupalPolymorphicDispatchUsageProvider`, and `ContribPluginsUsageProvider` cover further dynamic dispatch patterns. Two submodules (`canvas_ai`, `canvas_personalization`) are excluded via `ignoreErrors` entries pending their own cleanup.

The confirmed dead code deletion removed: the `InvalidRequestBodyValue` exception class; `JsonSchemaType::isIterable()` and `::isTraversable()`; `withDelta()` on `FieldObjectPropsExpression`, `FieldPropExpression`, and `ReferenceFieldPropExpression`; `CanvasResourceLinkCollection::hasLinkWithKey()`, `filter()`, and `merge()` (the collection class itself is `@internal`); `CanvasPropVisitor::extractAttributeName()`; `ShapeMatchingHooks::getMediaSourceFieldName()`; `ConfigEntityVersionDeriver::$basePluginId`; `ComponentPluginManager::$isRecursing`; injected dependencies removed from `ValidComponentTreeItemConstraintValidator` (`TypedDataManagerInterface`) and `CanvasTemplateRenderer` (`ElementInfoManagerInterface`, `RequestStack`, `LanguageManagerInterface`); and unused test class properties and imports. Symbols that must be kept but cannot yet be reached by analysis are suppressed inline with `@phpstan-ignore shipmonk.deadMethod` or `@phpstan-ignore shipmonk.deadConstant`.

## Upgrade

If you reference any of the following removed symbols in custom code, remove the references:

```php
// Deleted class — remove all usages:
\Drupal\canvas\InvalidRequestBodyValue

// Deleted enum methods:
JsonSchemaType::isIterable()
JsonSchemaType::isTraversable()

// Deleted methods on prop expression classes:
FieldObjectPropsExpression::withDelta(int $delta)
FieldPropExpression::withDelta(int $delta)
ReferenceFieldPropExpression::withDelta(int $delta)

// Deleted methods on CanvasResourceLinkCollection (@internal, but deletions are real):
CanvasResourceLinkCollection::hasLinkWithKey(string $key)
CanvasResourceLinkCollection::filter(callable $f)
CanvasResourceLinkCollection::merge($a, $b)
```

`ValidComponentTreeItemConstraintValidator` and `CanvasTemplateRenderer` no longer accept their previously injected services in their constructors; if you subclass or manually instantiate these, update the constructor calls accordingly.
