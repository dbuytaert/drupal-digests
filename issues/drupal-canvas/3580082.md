The interactive setup tool for Drupal Canvas now supports additional coding agents (Cortex Code, Deep Agents, and Warp) and reflects the current standard that most agents share a common `.agents/skills` directory. The agent selection prompt now clarifies that universal agents are always included and presents additional agents with updated terminology.

## Technical details

The `@drupal-canvas/create` package maintains metadata about which coding agents (like Cursor, Claude Code, Cline) work with Drupal Canvas Code Components and where each agent expects to find its skills directory. Agents that follow the newer `.agents/skills` convention are "universal" and require no compatibility symlinks. Agents with legacy directories like `.claude/skills` or `.cursor/skills` need symlinks created during project setup.

This update synchronizes the agent list with the upstream `vercel-labs/skills` repository. Three new agents were added: `cortex`, `deepagents`, and `warp`. Several existing agents (`antigravity`, `cline`, `cursor`, `deepagents`, `warp`) were reclassified to use `.agents/skills` instead of agent-specific directories, reflecting their adoption of the universal standard.

The interactive prompt was updated to make the distinction clearer: universal agents are now labeled "always included" and the selectable section was renamed from "Other agents" to "Additional agents." Status messages throughout the setup flow were shortened to refer to "compatibility setup" rather than "agent compatibility setup."

Tests were updated so compatibility symlinks are only validated for truly non-universal agents (like `.roo/skills`), while universal agents like Cursor no longer trigger symlink creation.
