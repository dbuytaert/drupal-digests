## Summary

The static method `BlockComponent::componentIdFromBlockPluginId()` has been removed from the Canvas module. This method was a temporary alias that simply forwarded its argument to `BlockComponentDiscovery::getComponentConfigEntityId()` and was always marked for removal. Any code that called it must now call `BlockComponentDiscovery::getComponentConfigEntityId()` directly. There is no change in behavior.

## Impact

Affects module maintainers.

- **API change:** `BlockComponent::componentIdFromBlockPluginId()` removed; use `BlockComponentDiscovery::getComponentConfigEntityId()` instead.

## Technical details

The removed method in `BlockComponent` carried a `@todo Remove in #3561267` marker, making this a pre-announced cleanup. The only internal call site was in `PageRegion`, which resolved component IDs for blocks by calling `BlockComponent::componentIdFromBlockPluginId($block->getPluginId())`; that call is now replaced with `BlockComponentDiscovery::getComponentConfigEntityId($block->getPluginId())`. The kernel test `BlockComponentTest::testComponentIdFromBlockPluginId()` had its `@covers` annotation and assertion updated to target `BlockComponentDiscovery::getComponentConfigEntityId` directly, confirming the two methods were functionally identical.
