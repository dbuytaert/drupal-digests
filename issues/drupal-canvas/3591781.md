# #3591781: Release content entity reference props for code components: drop the `canvas_dev_er` feature flag module

- **Project:** Drupal Canvas
- **Type:** Normal feature request
- **Version:** 1.x-dev
- **Status:** Fixed
- **Discussion:** [#3591781](https://git.drupalcode.org/project/canvas/-/work_items/3591781) · 1 contributor · 4 comments

## Summary

The "Content entity reference" prop type in the Canvas code editor is now generally available. Previously, developers had to enable a hidden `canvas_dev_er` module to expose this option when creating or editing code component props. That gate has been lifted: the option now appears unconditionally for all users of the code editor. The `canvas_dev_er` module itself is deprecated in place (marked `lifecycle: deprecated`) and flagged for removal before Canvas 2.0.

## Impact

Affects module maintainers.

- **Deprecation:** The `canvas_dev_er` submodule is deprecated and flagged for removal before Canvas 2.0
- **New API:** The `contentEntityReference` prop type in the Canvas code editor is now unconditionally available; the `drupalSettings.canvas.devEntityReferenceMode` flag is removed.

## Technical details

The feature flag was implemented as a hidden submodule (`canvas_dev_er`) whose presence caused the PHP attachment layer (`CanvasSettingsAttachment` or equivalent) to emit `drupalSettings.canvas.devEntityReferenceMode = true`. The frontend `Props` component read that setting via `getCanvasSettings()` and used it to filter `derivedPropTypes`, hiding the `contentEntityReference` option unless the flag was set or the existing prop already used that type.

The fix removes `devEntityReferenceMode` from `DrupalSettings` and from the PHP `drupalSettings` attachment. On the React side, the `isContentEntityReferenceEnabled` variable and the `propTypeOptions` filter in `Props.tsx` are deleted; the `Select` now iterates directly over `derivedPropTypes`, which always includes the `contentEntityReference` entry. Tests that toggled `setCanvasDevEntityReference` and asserted conditional visibility are replaced with a single test asserting the option is always present. A test-module's dependency on `canvas_dev_er` is also dropped. The `canvas_dev_er` `.info.yml` receives `lifecycle: deprecated` and a link to change record [#3609370](https://www.drupal.org/node/3609370).

## Upgrade

Sites that explicitly enabled `canvas_dev_er` to access entity reference props in the code editor no longer need it. The module still exists but is deprecated; uninstall it and stop enabling it. No code changes are needed: the prop type is now always available.

If custom code checks `drupalSettings.canvas.devEntityReferenceMode`, remove those checks:

```js
// Before (remove this guard):
if (drupalSettings.canvas.devEntityReferenceMode) {
  // use content entity reference
}

// After (no guard needed):
// content entity reference is always available
```
