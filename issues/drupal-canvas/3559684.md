## Summary

When uploading a large number of components (e.g. ~90 at once) via the Canvas CLI tool, the process would time out because uploads ran one by one and the OAuth access token could expire mid-run. This fix parallelises both the existence checks and the actual uploads, and adds automatic token refresh so long-running bulk operations no longer fail partway through.

## Impact

No upgrade or configuration changes required.

## Technical details

Three interrelated changes landed in the CLI tool's TypeScript source:

**Lazy OAuth with automatic refresh.** `ApiService.create()` no longer eagerly fetches an access token on construction. Instead, a new private `refreshAccessToken()` method is called on-demand. An Axios request interceptor calls it before any request when no token is present. An Axios response interceptor handles 401 responses by transparently refreshing the token and retrying the original request once (guarded by an `_retry` flag to prevent loops). A `refreshPromise: Promise<string> | null` field deduplicates concurrent refresh attempts: if multiple requests trigger a refresh simultaneously, they all await the same promise rather than each firing a separate token request.

**Bulk existence check.** `checkComponentsExist()` replaces the old per-component `apiService.getComponent(machineName)` calls with a single `apiService.listComponents()` call whose results are loaded into a `Set`. All N machine names are then checked locally in O(1) each, reducing N round trips to one.

**Concurrent uploads via `processInPool`.** A new `request-pool.ts` utility exports `processInPool<T, R>(items, processor, concurrency = 10)`. It slices the input array into batches of at most `concurrency` items, awaits each batch with `Promise.allSettled`, and collects results in original index order. Individual upload failures are recorded without aborting the rest of the batch. A companion `createProgressCallback` helper updates the CLI spinner with a `(n/total - pct%)` message as each item completes. `getBuildAndUploadResults()` in `upload.ts` now calls these utilities for both the existence check phase and the upload phase.
