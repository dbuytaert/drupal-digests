## Summary

The `@drupal-canvas/create` tool now helps set up coding agent compatibility when creating a new project. If a template includes AI agent skills in the `.agents/skills` directory, the setup wizard asks which coding agents you plan to use and automatically creates the necessary compatibility links. This ensures skills work correctly across different AI coding assistants like Claude Code, Cursor, and others, without manual configuration.

## Technical details

The `@drupal-canvas/create` package adds agent skills compatibility detection and setup during project initialization. When a cloned template repository contains an `.agents/skills` directory, the new `setupAgentSkills()` function discovers skill directories and prompts the user to select coding agents via `searchMultiselect()`.

The implementation borrows from the Vercel Skills project. It maintains a registry of 40+ coding agents in `agents.ts`, categorizing them as "universal" (those that already use `.agents/skills`) or requiring compatibility symlinks. The `agents.ts` module exports `AgentType`, `AgentConfig`, and utility functions like `getUniversalAgents()` and `isUniversalAgent()`.

For non-universal agents, `createAgentSkillSymlinks()` creates relative symlinks from agent-specific directories (like `.claude/skills`, `.cursor/skills`) to the canonical `.agents/skills` directory. The function handles existing paths gracefully, logs warnings for skipped or failed symlinks, and uses junction links on Windows for compatibility.

The setup includes a custom `searchMultiselect()` prompt component that supports keyboard navigation, real-time filtering, a locked section for universal agents, and visual feedback. Tests in `agent-skills-setup.test.ts` verify symlink creation, conflict handling, and mixed agent selection scenarios. The package adds `vitest`, `picocolors`, and `string-width` dependencies for testing and terminal output.
