## Summary

Canvas AssetLibrary entities can now track custom font CSS files. This allows Canvas to load font files by adding their CSS to the asset library system. Previously, there was no structured way to attach font files through the AssetLibrary API. This feature supports programmatic font management for Canvas components and may be exposed in the editor interface in the future.

## Impact

No upgrade or configuration changes required.

## Technical details

The `AssetLibrary` entity gained a `fonts` property that stores an array of font entries. Each entry contains a `name` (e.g. '@fontsource/inter') and a `uri` pointing to a managed file containing the font CSS.

The `getFonts()` and `setFonts()` methods provide typed access to this array. The property is nullable and defaults to an empty array when read. Font entries are included in the entity's client-side normalization for editor integration.

In `LibraryHooks::libraryInfoBuild()`, font CSS URIs are transformed into absolute URLs via `FileUrlGeneratorInterface` and added to the library definition's `css.theme` array. This makes the fonts available when the asset library is attached to a page.

Config schema defines fonts as a nullable sequence of mappings with name and uri fields. The feature adds no UI; fonts are managed programmatically through the AssetLibrary API. Saving the entity invalidates library info cache tags to ensure font changes are picked up immediately.
