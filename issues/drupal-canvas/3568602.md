## Summary

Canvas now gracefully handles components that gain new required props or have optional props made required, without breaking existing published pages. Before this change, such component schema evolution was blocked from automatic upgrade, and any not-yet-edited page using the old component version would fail to render correctly. After this change, pages render using the first example value defined for the newly required prop, and when an editor opens the page, the stored component instance is upgraded automatically, preserving any value the editor had previously set.

## Impact

Affects module maintainers.

- **API change:** `ComponentSourceInterface::getDefaultExplicitInput()` gains `bool $only_required = FALSE`; `ComponentSourceInterface::hydrateComponent()` gains a required third parameter `array $active_required_explicit_inputs`.

## Technical details

This extends the automatic component instance upgrading system from [#3567413](https://www.drupal.org/node/3567413) to cover two previously unsafe cases: a new required prop added to a component, and an existing optional prop promoted to required.

The implementation operates at two distinct points. At edit time, `GeneratedFieldExplicitInputUxComponentInstanceUpdater::update()` now iterates over the active version's required props and populates missing inputs with their default values (the first example value stored in `prop_field_definitions`). For brand-new required props, the default is always written. For optional-to-required promotions, the existing user-supplied value is preserved if present; only if absent is the default written. The prior `canUpdate()` guard that returned `FALSE` for these two cases has been removed entirely. At render time, `ComponentTreeItemList::getHydratedTree()` calls `getDefaultExplicitInput(only_required: TRUE)` on the active version's component source and passes the resulting map as a new `$active_required_explicit_inputs` argument to `hydrateComponent()`. Inside `GeneratedFieldExplicitInputUxComponentSourceBase::hydrateComponent()`, those default values are wrapped as `EvaluationResult` objects and merged into the hydrated props via array union, so they act as a fallback for any required prop missing from the stored component instance data.

The design relies on a baseline assumption already enforced by `ComponentMetadataRequirementsChecker`: every required prop in a component must have at least one example value, or the `Component` config entity becomes disabled. This guarantee makes the first example value a safe stand-in during both render-time hydration and edit-time upgrade.

Two method signatures on `ComponentSourceInterface` changed. `getDefaultExplicitInput()` gains `bool $only_required = FALSE` (backward-compatible default). `hydrateComponent()` gains a required third parameter `array $active_required_explicit_inputs`. All built-in implementations (`JsComponent`, `SingleDirectoryComponent` via `GeneratedFieldExplicitInputUxComponentSourceBase`, `BlockComponent`, `Fallback`, `InvalidSlots`, `Personalization`) have been updated accordingly. `BlockComponent` includes a `@todo` noting that `$only_required` handling is deferred pending [#3521221](https://www.drupal.org/node/3521221).

## Upgrade

Any custom class implementing `ComponentSourceInterface` must update two method signatures:

```php
// Before:
public function getDefaultExplicitInput(): array { ... }
public function hydrateComponent(array $explicit_input, array $slot_definitions): array { ... }

// After:
public function getDefaultExplicitInput(bool $only_required = FALSE): array { ... }
public function hydrateComponent(array $explicit_input, array $slot_definitions, array $active_required_explicit_inputs): array { ... }
```

If your implementation needs to filter to required-only inputs, skip props where `$def['required'] === FALSE` when `$only_required` is `TRUE`. The `$active_required_explicit_inputs` array contains prop sources keyed by prop name; use it as a fallback (array union) inside `hydrateComponent()` for required props that may be absent from stored instance data.
