## Summary

Contributors to the Drupal Canvas (formerly Experience Builder) module can now run PHP static analysis tools locally using the same commands that CI uses. New `composer run` scripts (`phpcs`, `phpcbf`, `phpstan`, `install-dev-deps`) make it straightforward to validate code without guessing at the correct invocation. Documentation was added in `docs/testing/` covering global setup, PHP static testing, and CI replication via `gitlab-ci-local`.

## Impact

No upgrade or configuration changes required.

## Technical details

Four `composer run` scripts were added to `composer.json`: `install-dev-deps` (installs dev dependencies declared in the module's own `composer.json` into the parent Drupal project), `phpcs` (runs PHP_CodeSniffer with JUnit output to `test-results/phpcs.xml`), `phpcbf` (auto-fixes style violations), and `phpstan` (runs static analysis with JUnit output). A helper `composer-root` script walks up the directory tree to locate the Drupal project root so the tools work regardless of where the module is installed.

The GitLab CI configuration was substantially reorganised. The monolithic `.drupal` setup anchor in `.gitlab-ci.yml` was extracted into a new `.gitlab/ci/drupal.yml` file split into three composable anchors: `.drupal-files` (creates the Drupal project and links the module), `.drupal-webserver` (configures Apache and filesystem permissions), and `.drupal-site` (writes `settings.testing.php` and builds the frontend). A new `.gitlab/ci/test.yml` provides `.test_runner` and `.test_summary` shell helpers that collect per-command pass/fail state and emit a summary table rather than failing immediately on the first error. The upstream template's separate `phpcs`, `phpstan`, `cspell`, and `composer-lint` jobs are now disabled (`when: never`) and replaced by a single consolidated `lint (php)` job that runs `composer validate`, `phpcs`, and `phpstan` in sequence using these helpers. The old `lint (frontend)` job was renamed `lint (node)` and updated to use the same `justafish/drupalci-environments-playwright` image as the PHP job. The `cspell` check moved from a standalone job into the Node lint path; `cspell` was added to `package.json` as a dev dependency and `.cspell.json` was expanded to include Drupal core's remote dictionaries and a much wider set of language dictionaries. Two return type annotations were added to `xb_vite_install()` and `xb_vite_library_info_alter()` as a minor cleanup included in the same commit.

## Upgrade

Developers contributing to the module can now install dev dependencies from within the module directory:

```shell
# From within modules/custom/experience_builder (or equivalent path)
composer run install-dev-deps
```

Then run static checks locally:

```shell
composer run phpcs     # Detect coding standard violations
composer run phpcbf    # Auto-fix coding standard violations
composer run phpstan   # Static analysis
```

To replicate a CI job exactly using [gitlab-ci-local](https://github.com/firecow/gitlab-ci-local):

```shell
gitlab-ci-local \
    --remote-variables git@git.drupal.org:project/gitlab_templates=includes/include.drupalci.variables.yml=main \
    --variable="_GITLAB_TEMPLATES_REPO=project/gitlab_templates" "lint (php)"
```
