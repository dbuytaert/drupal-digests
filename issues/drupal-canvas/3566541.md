## Summary

The Code Component Editor now includes a "Date and time" prop type in the Type drop-down menu. Content editors can define component props that accept date or datetime values, which can be populated either statically using a calendar widget or dynamically by linking to a date field on a node. This enables components to display and work with date information.

## Impact

No upgrade or configuration changes required.

## Technical details

The Code Component Editor's prop type system already supported `type: string, format: date` and `type: string, format: date-time` at the config schema level through `canvas.json_schema.prop.string`. This change adds UI support to make these formats accessible.

A new `FormPropTypeDate` component provides the interface for date and datetime props. Users select between "Date only" or "Date and time" via a `Date type` dropdown, which sets the `format` field to either `date` or `date-time`. The form includes an `Example value` field that renders as an HTML5 date or datetime-local input.

The implementation includes timezone conversion utilities in `date-utils.ts`. The `localTimeToUtcConversion` function converts datetime-local input values to UTC ISO strings for storage, while `utcToLocalTimeConversion` handles the reverse for display. This ensures datetime values use UTC internally while presenting local times in the UI.

The `derivedPropTypes` configuration adds a new `date` type that matches props where `type === 'string'` and `format` is either `date` or `date-time`. The initialization sets `type: 'string'` and `format: 'date'` as defaults.

Comprehensive test coverage includes both unit tests in `Props.test.tsx` and fixtures in `code-component-props.json`. Tests verify switching between date types, setting example values, and proper serialization/deserialization.
