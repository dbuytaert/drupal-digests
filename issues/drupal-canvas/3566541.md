## Summary

Code components can now receive date and datetime values through props. Content editors can use calendar widgets to select dates statically or link to date fields on nodes. This enables code components to display formatted dates and times pulled from Drupal's content system.

## Impact

No upgrade or configuration changes required.

## Technical details

The Code Component Editor now includes "Date and time" as a prop type in the Type dropdown. This surfaces the existing support in the JSON schema for `type: string, format: date` and `type: string, format: date-time` props, which was already present in the `canvas.json_schema.prop.string` config schema but lacked a user interface.

The new `FormPropTypeDate` component provides a date type selector (date-only or datetime) and appropriate HTML5 input elements. For datetime values, the component handles conversion between local datetime-local format (required by the browser input) and UTC ISO strings (required by the server) using new utility functions `localTimeToUtcConversion` and `utcToLocalTimeConversion` in `date-utils.ts`.

The `derivedPropTypes` array gained a new entry that identifies props with `type: string` and `format: date` or `format: date-time`. The `FormPropTypeLink` component was updated to accept and preserve the `format` prop, ensuring link props maintain their URI format metadata. Comprehensive vitest unit tests cover date-only and datetime scenarios, including timezone conversion and validation.
