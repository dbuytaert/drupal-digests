## Summary

The Code Component Editor now includes a "Date and time" option in the Type dropdown when defining props. Content editors can choose between "Date only" or "Date and time" formats, providing calendar-based input for static values and the ability to link to date fields on content. This completes support for the JSON Schema date and date-time formats that Single Directory Components already allowed in their schema definitions.

## Impact

No upgrade or configuration changes required.

## Technical details

The Code Component Editor already supported `type: string, format: date` and `type: string, format: date-time` at the config schema level through `canvas.json_schema.prop.string` since the initial `ComponentSource` plugin implementation. This change adds UI support to make these formats accessible when creating code components.

The `derivedPropTypes` array now includes a `date` type that derives when `prop.type === 'string'` and `prop.format` is either `date` or `date-time`. The new `FormPropTypeDate` component renders a "Date type" select that toggles between the two formats, and an appropriate input field (`type="date"` or `type="datetime-local"`).

For date-time values, the component converts between the UTC ISO string format required by the server and the local datetime format required by the HTML5 `datetime-local` input element using new utility functions `localTimeToUtcConversion()` and `utcToLocalTimeConversion()` in `date-utils.ts`. Date-only values pass through unchanged.

The changes include comprehensive vitest unit tests in `Props.test.tsx` that verify both date and datetime prop creation, format switching, and example value handling. The existing Cypress unit tests were not extended, with a note directing new tests to vitest instead.
