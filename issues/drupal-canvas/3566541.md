## Summary

The Code Component Editor now lets site builders choose Date and Date and Time as prop types when defining component properties. This makes it easier to work with dates in custom components. Builders can set example dates using a calendar picker, and can link these props to date fields on content to display dynamic date information.

## Impact

No upgrade or configuration changes required.

## Technical details

Single-file components (SDC) have always supported `type: string, format: date` and `type: string, format: date-time` in their schema. However, the Code Component Editor UI did not expose these types in the Type dropdown when defining props. This feature adds a new "Date and time" option that allows selecting between "Date only" (format: `date`) and "Date and time" (format: `date-time`).

The implementation adds `FormPropTypeDate.tsx` to render date/datetime input fields with a type selector. For date-only props, the component uses a standard HTML5 date input. For datetime props, it uses a datetime-local input and converts between the local datetime format the browser expects and the UTC ISO string format the server requires via new utility functions `localTimeToUtcConversion()` and `utcToLocalTimeConversion()` in `date-utils.ts`.

The `derivedPropTypes.ts` configuration registers the new "Date and time" type, mapping it to props with `type: 'string'` and `format` of either `date` or `date-time`. The serialization and deserialization logic in `code-editor-utils.test.jsx` was extended to handle these formats. Comprehensive test coverage was added in `Props.test.tsx` using vitest, verifying date type selection, format switching, example value handling, and UTC/local time conversion.
