## Summary

Optional image and video properties in components can now be properly cleared. Previously, when users removed a default image from an optional property, it would reappear after page refresh or publish. The system now correctly distinguishes between "no value set yet" (use default) and "user intentionally removed the value" (respect empty state). This fix ensures that removing a default image from an optional property persists across page loads and publishes, preventing the default from unexpectedly returning.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue stemmed from the backend incorrectly falling back to `DefaultRelativeUrlPropSource` whenever an optional image/video prop had an empty value, even when the user explicitly removed it. The fix introduces explicit tracking of user deletion intent by preserving NULL values in prop sources.

Backend changes in `GeneratedFieldExplicitInputUxComponentSourceBase` now check whether the client sent an explicit `value` key that differs from the default. When a user removes an optional prop, the system stores it as `StaticPropSource` with `value: null` instead of falling back to the default. The validation step was updated to skip `ComponentValidator` checks for optional props whose value evaluated to NULL, preventing false errors. The model serialization preserves NULL values explicitly to maintain the distinction between "unset" and "deleted".

Frontend changes added a new `DefaultImagePreview` React component that displays a thumbnail of the default image with a "Remove default" button (only for optional props). Clicking the button sends a PATCH request with `value: null` to persist the deletion. The component is integrated via a custom element in the media library fieldset template and only renders when no user-selected media exists and the current value matches the default.

The `fieldWidgetCompleteFormAlter` hook now passes `#component_prop_name` through the render array to identify which prop each widget manages, enabling correct scoping when multiple image props exist in the same component form.

## Lessons learned

- When implementing optional form fields with defaults, distinguish between 'no value set yet' (use default) and 'user intentionally cleared the value' (respect empty state). Store explicit deletion as a NULL value in the data layer rather than falling back to defaults, so the user's intent to leave the field empty is preserved across saves. *(via bnjmnm, narendrar, wim leers, tedbow)*
- When async operations complete after assertions run, tests may pass while the system enters an incorrect state. Add delays or wait conditions that ensure the full state transition completes before asserting the final outcome. *(via bnjmnm)*
- When multiple instances of the same widget exist in a form, ensure state changes target only the intended instance. Pass unique identifiers through render arrays to prevent cross-contamination when JavaScript manipulates widget state. *(via bnjmnm)*
