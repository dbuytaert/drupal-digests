## Summary

This change proves and documents that `hook_storage_prop_shape_alter()` is a fully capable extension point for contrib modules: it can override Canvas's built-in prop shape decisions and add support for completely new prop shapes and field types. A previous assumption in the issue queue that the hook could only alter existing matches was incorrect. Alongside the test coverage, a bug in the daterange field mapping was corrected: `value` (start date) and `end_value` (end date) were mapped to the wrong `from`/`to` keys, and this is now fixed.

## Impact

No upgrade or configuration changes required.

## Technical details

The test module `canvas_test_storage_prop_shape_alter` was extended to demonstrate three scenarios via `CanvasTestStoragePropShapeAlterHooks::storagePropShapeAlter()`:

1. **Changing** Canvas's default decision: replacing the `link` field type/widget with `uri` for `type=string&format=uri` props.
2. **Adding** support for an arbitrary prop shape Canvas does not natively handle: a schema of `type=integer&multipleOf=12` is matched to an imaginary `multiple_of` field type provided by the test module's `MultipleOfItem` class. To enable this, `JsonSchemaType` was updated so that `array_key_exists('multipleOf', $schema)` now returns `NULL` explicitly rather than falling through to the generic unrestricted-integer branch, leaving such schemas for contrib to handle via the hook.
3. **Adding** support for a well-known `$ref`-based prop shape not covered by core: the `json-schema-definitions://canvas.module/date-range` schema is matched to `daterange` via `ShapeMatchingHooks::datetimeRangeStoragePropShapeAlter()`, which already existed but is now verified by the test.

The daterange bugfix corrects `ShapeMatchingHooks`: `from` is now mapped to `FieldTypePropExpression('daterange', 'value')` (the start value) and `to` to `end_value`, reversing the prior incorrect assignment. Corresponding assertions in `PropShapeToFieldInstanceTest` were updated to match.
