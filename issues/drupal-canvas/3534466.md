## Summary

Canvas pages can now be indexed by Search API, making their component content available in site search results. Previously, the text entered into Canvas component instances (headings, body text, labels, and so on) was invisible to Search API. After this change, site owners can add Canvas pages as a datasource to a Search API index, add the "Canvas component tree inputs" field as a Fulltext type, reindex, and search will find content that lives inside Canvas components. The `drupal/search_api: ^1` module is now a required dependency of Canvas.

## Impact

Affects module maintainers and site owners.

- **Requirements:** `drupal/search_api: ^1` added as a required Composer dependency of Canvas.
- **New API:** New `ComponentTreeInputExtractor` service and `canvas_component_tree_inputs` Search API processor plugin for indexing Canvas page component inputs.

## Technical details

A new Search API processor plugin, `ComponentTreeInputs` (plugin id `canvas_component_tree_inputs`), is registered under `Drupal\canvas\Plugin\search_api\processor\ComponentTreeInputs`. It implements `add_properties` stage and exposes a single processor property, `canvas_component_tree_inputs`, typed as `string`. The plugin is available only on indexes whose datasource includes the `canvas_page` entity type (`supportsIndex` checks for that).

The actual extraction is handled by a new service, `ComponentTreeInputExtractor`, which walks every `ComponentTreeItem` in a page's component tree via `ComponentTreeLoader`. For each component instance it retrieves explicit prop inputs and filters them: only props whose resolved JSON schema `type` is `string` are considered, enum-typed string props (pick lists) are excluded, and a configurable list of prop names is ignored. The default ignored list is `['id', 'class', 'cssClasses', 'extraClasses']`, configurable per-index through a textarea in the processor's configuration form.

Extraction only works for components backed by `GeneratedFieldExplicitInputUxComponentSourceBase` (i.e. SDC and JS/code components). Block components are intentionally skipped because Search API's built-in "Rendered item" processor handles those. A new config schema key `plugin.plugin_configuration.search_api_processor.canvas_component_tree_inputs` covers the `ignored_prop_names` sequence. Kernel tests cover the extractor directly (`ComponentTreeInputExtractorTest`) and the full Search API processor integration including end-to-end query execution (`ComponentTreeInputsTest`).

## Upgrade

To enable Canvas content in search, add the following to a Search API index configuration after updating Canvas:

1. Add `entity:canvas_page` as a datasource on the index.
2. In the index fields, add the "Canvas component tree inputs" processor property and set its type to **Fulltext**.
3. Optionally configure the processor's ignored prop names list (defaults to `id`, `class`, `cssClasses`, `extraClasses`).
4. Reindex and clear caches.

No code changes are required for sites that do not use Search API.
