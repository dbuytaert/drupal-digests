## Summary

When editing a content template for a view mode other than Full (such as Teaser or Card), global regions like header and footer are now hidden from both the editor and preview. These regions do not render when the view mode is displayed on the actual site, so hiding them makes it clearer what the template will actually look like. The editor now shows only the content region for non-full view modes.

## Impact

No upgrade or configuration changes required.

## Technical details

After [#3574412](https://www.drupal.org/node/3574412) enabled creating content templates for view modes other than Full, the global regions (header, footer, etc.) were still appearing in the editor and preview even though they are not part of the display for those view modes.

The solution adds a `shouldIncludeGlobalRegions()` method in `ApiLayoutController` that returns false for `ContentTemplate` entities whose view mode is not "full". This method is called throughout the controller to conditionally load global regions from `PageRegion::loadForActiveTheme()`, include editable regions in auto-save validation, and set a `#canvas_hide_global_regions` flag on the preview build.

In `CanvasPreviewRenderer::prepare()`, when this flag is set, all page regions except `CanvasPageVariant::MAIN_CONTENT_REGION` are emptied before rendering.

On the frontend, the component context menu checks whether global regions exist before showing the "Move to region" option. The viewport sets minimum height to 0 for non-full view modes so the preview size matches the actual content rather than requiring a minimum page height.

After Wim raised concerns about preview accuracy being broken, they clarified via direct message that the implementation does preserve page-level markup and CSS.
