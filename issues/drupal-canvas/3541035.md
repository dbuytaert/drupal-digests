## Summary

In the Drupal Canvas module (formerly Experience Builder), the "Review changes" publish dialog previously displayed content templates with a generic file icon and the raw machine-readable label `content-templates`. After this change, content templates are shown with a dedicated template icon (a document with layout zones) and the human-readable group heading "Content templates". Page entries across the UI also received a visual update, switching from a plain file icon to a file-with-text icon to better distinguish them from templates. Several alignment and spacing issues in the publish dialog rows were also corrected.

## Impact

No upgrade or configuration changes required.

## Technical details

The core functional fix is in `getGroupLabel()` (`src/utils/review.ts`), which now returns `'Content templates'` for the `content_template` entity type instead of falling through to the `kebabCase` default that produced `content-templates`.

A new custom SVG asset, `src/assets/icons/template.svg`, was added to represent content templates with a distinct icon. `ChangeIcon` (previously an inline component in `ChangeRow.tsx`) was extracted into its own file (`src/components/review/changes/ChangeIcon.tsx`) and gained an explicit `case 'content_template'` branch returning the new `TemplateIcon`. The same `TemplateIcon` is now used in the side-menu button for the templates section, in `SideBarNode` VARIANTS, and in the page-type `iconMap` in the sidebar navigation.

Across several components (`Navigation.tsx`, `PageList.tsx`, `SideMenu.tsx`, `SideBarNode.tsx`, `NameTag.tsx`, `EmptyRegionDropZone.tsx`), `FileIcon` was replaced with `FileTextIcon` for page-type entries. `ErrorGroup.tsx` was also updated to use the shared `ChangeIcon` component instead of its own hardcoded `FileIcon`, meaning error groups now receive the same per-entity-type icon treatment as change rows.

Layout tweaks in `ChangeRow.tsx` switched several flex containers from `align="center"` to `align="start"` and added `pt="1"` wrappers to keep the icon, timestamp, and action button vertically aligned with multi-line label text. The `.avatar` border was moved from a CSS class to an inline style, `.labelIcon` CSS was removed (replaced by the shared `.changeIcon` class), and `.changeIcon` color was changed from `--purple-9` to `--gray-10` with no explicit size constraint.
