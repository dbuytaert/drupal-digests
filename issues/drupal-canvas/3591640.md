# #3591640: Selects based on enums defined in schema.json not working because id not a supported key in schema or registering duplicated JSON Schema in Ajv

- **Project:** Drupal Canvas
- **Type:** Major bug report
- **Version:** 1.x-dev
- **Status:** Fixed
- **Discussion:** [#3591640](https://git.drupalcode.org/project/canvas/-/work_items/3591640) · 1 contributor · 5 comments

## Summary

Selecting an option from a Canvas component's enum/select field (for example a heading level or column width) threw a JavaScript error and the change never applied, leaving the preview frozen. This happened because the PHP `justinrainbow/json-schema` library injects `id` or `$id` keys when it resolves `$ref` references in a component's `schema.json`, and when those keys reached the front-end Ajv validator they caused either a "NOT SUPPORTED: keyword id, use $id" error or a "schema already exists" duplicate-registration error. The fix strips `id` and `$id` from the resolved schema before it is sent to the browser, so Ajv receives a clean schema and select interactions work correctly again.

## Impact

No upgrade or configuration changes required.

## Technical details

The method `stripNonStandardJsonSchemaKeys` in `JsonSchemaPropsComponentSourceBase` recursively removed Canvas-specific extension keys (`meta:enum`, `x-translation-context`) from prop schemas before they were serialised into the `field_data` payload sent to the React UI. The PHP library `justinrainbow/json-schema` populates `id` (Draft 4) and `$id` (Draft 7) on every object it produces when resolving `$ref`s against `schema.json` files; these keys were not being stripped. Ajv on the client side rejected `id` entirely in strict mode, and when multiple props referenced the same shared definition (e.g. `json-schema-definitions://canvas.module/column-width`), the same `$id` value was registered with Ajv more than once, producing a duplicate-schema error.

The fix renames the method to `simplifySchemaForAjvClient` and adds `'id'` and `'$id'` to the `$keys_to_remove` list. The stripping is recursive, so nested object and array sub-schemas are also cleaned. Test fixture expected-data arrays throughout `JsComponentTest` and `SingleDirectoryComponentTest` are updated to omit the `$id` values that were previously present in the assertions. A new Playwright end-to-end test case exercises `editComponentProp` with a `select` input type to confirm that enum props update the preview correctly after the fix.
