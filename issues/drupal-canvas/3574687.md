## Summary

Canvas' test suite converted all PHPUnit annotations to PHP attributes. PHPUnit 11 stopped recognizing comment-based annotations like `@group` and `@dataProvider`, so tests could not be grouped or filtered. After conversion to attributes like `#[Group('…')]` and `#[DataProvider('…')]`, the test suite can once again run targeted subsets. A new PHPCS sniff enforces attributes going forward.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 11 dropped support for docblock annotations, requiring a wholesale migration to PHP 8 attributes. Canvas used PHPUnit's `--group` flag to run test subsets in CI, such as JavaScript component tests or slow tests. Without attributes, `--group JavaScriptComponents` found no tests.

The conversion touched every annotation PHPUnit reads: `@group`, `@covers`, `@coversClass`, `@dataProvider`, and `@testWith`. PHPUnit does not allow mixing annotations and attributes in a single file, so partial migration was impossible. The Rector rule from #3446380 automated the bulk conversion.

One test class required refactoring: `PropSourceTest` declared `@covers` on individual test methods, which PHPUnit 11 forbids. That class split into an abstract base and one concrete subclass per prop source, each covering only its own class.

A new `PhpunitAnnotationsSniff` detects forbidden annotations and points developers to the correct attribute syntax. This sniff can be deleted once Canvas requires Drupal 12, which will enforce attributes core-wide.

## Lessons learned

- When upgrading to PHPUnit 11+, test annotations like `@group`, `@dataProvider`, and `@covers` must all be converted to PHP attributes in the same file. PHPUnit no longer allows mixing annotations and attributes within a single test class—convert all annotations at once to restore functionality like group-based test filtering. *(via wim leers, longwave)*
- When a framework upgrade breaks existing workflows, enforce the new patterns with automated tooling. After manually converting code (e.g., using Rector to migrate annotations to attributes), add linting rules to prevent regressions and guide future contributors toward the correct approach. *(via wim leers)*
