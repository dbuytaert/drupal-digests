## Summary

Running specific test groups in PHPUnit stopped working after upgrading to PHPUnit 11 because annotations like `@group canvas` are no longer recognized. All test annotations must now use PHP attributes like `#[Group('canvas')]`. This restores the ability to run focused test subsets during development, and a new PHPCS rule prevents future use of deprecated annotations.

## Impact

No upgrade or configuration changes required.

## Technical details

PHPUnit 11 requires PHP attributes instead of docblock annotations for metadata like groups, data providers, and coverage declarations. The change converted every `@group`, `@dataProvider`, `@covers`, and `@coversClass` annotation to their attribute equivalents: `#[Group(…)]`, `#[DataProvider(…)]`, `#[CoversClass(…)]`. PHPUnit does not allow mixing annotations and attributes in the same file, so all annotations had to be converted atomically.

A new PHPCS sniff (`PhpunitAnnotationsSniff`) now detects and forbids these four annotation types, enforcing attributes going forward. The sniff will be removed once Canvas requires Drupal 12, which enforces this policy in core.

One test class (`PropSourceTest`) required refactoring because PHPUnit 11 no longer allows per-method `@covers` declarations. It was split into an abstract base class with concrete subclasses, one per formerly-tested method, each with a single `test()` method and class-level coverage attributes.

## Lessons learned

- When a testing framework deprecates annotations in favor of attributes, you cannot mix both syntaxes in a single test file. Convert all annotations to attributes at once to avoid tests being silently ignored and not executed. *(via longwave, wim leers)*
