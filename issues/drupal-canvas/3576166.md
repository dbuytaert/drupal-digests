## Summary

Canvas component editor now supports multi-value Date and DateTime fields with a popover-based UI. When editing a component, users can add, edit, remove, and reorder multiple date or datetime values. Each value appears as a formatted list item; clicking it opens a popover with separate date and time inputs. This is UI-only and does not change how field data is stored.

## Impact

No upgrade or configuration changes required.

## Technical details

The Canvas component editor previously supported multi-value text and numeric fields but not date or datetime fields. This adds that support using the same popover pattern introduced in [#3573096](https://www.drupal.org/node/3573096).

A new React component `DrupalDatetimeMultivalueForm` wraps the Drupal datetime widget when it appears in a multivalue field table. The component reads initial values from the hidden native date and time inputs, displays them as a single formatted list item using `Intl.DateTimeFormat`, and presents separate date and time inputs in a Radix popover when clicked. The popover commits changes only when the user presses Enter, triggering validation via `checkValidity()` before updating the hidden inputs. Closing the popover without Enter discards temporary changes.

The component handles both datetime fields (date + time inputs in popover) and date-only fields (date input only). A new Twig template `datetime-wrapper--multivalue-form.html.twig` wraps datetime widgets with a custom element that passes the field label to the React component. The theme suggestion is added in `canvas_stark_theme_suggestions_datetime_wrapper_alter()` when `#multivalue_field_label` is present. That property is set in `ReduxIntegratedFieldWidgetsHooks::markMultivalueFormElements()`, which now skips recursing into datetime elements since they are handled as a unit.

Four utility functions were extracted to `multivalueFormUtils.ts` and shared with `DrupalInputMultivalueForm`: `isRemoveButtonEnabled()` checks required field constraints and Drupal button state, `triggerDrupalRemoveButton()` clicks the hidden AJAX remove button, `copyInputAttributes()` transfers validation attributes from hidden inputs to visible ones, and `updateInputValue()` uses the native value setter to trigger both React and Drupal event handlers.

CSS positions custom calendar and clock icons to the left of date and time inputs in the popover. The preprocessing in `canvas_stark_preprocess_field_multiple_value_form()` now adds a `canvas-multivalue-field-content` class to the field content cell and a `data-canvas-remove-button` attribute to the remove action cell, making them reliably identifiable without relying on element order.

Test fields `field_cvt_unlimited_datetime`, `field_cvt_unlimited_date`, and `field_cvt_required_datetime` were added to `canvas_test_article_fields`. The required datetime field includes a default value so existing test nodes pass validation when published by unrelated tests.
