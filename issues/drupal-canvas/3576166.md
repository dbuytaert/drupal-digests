## Summary

The Canvas component editor now supports multi-value date and datetime fields with the same popover-based list interface introduced for text fields in [#3573096](https://www.drupal.org/node/3573096). Each value appears as a collapsed list item showing the locale-formatted date and, for datetime fields, the time. Clicking a list item opens a popover with separate date and time inputs; pressing Enter commits the change while closing the popover without Enter discards it. The "+ Add new" button, drag-to-reorder handles, and per-item remove control all behave consistently with other multi-value field types. This UI is currently gated on the `canvas_dev_mode` module pending a fix in a follow-up issue.

## Impact

Affects module maintainers.

- **New API:** New `DrupalDatetimeMultivalueForm` React component and `multivalueFormUtils.ts` utility module for multi-value datetime field support in the Canvas editor

## Technical details

A new `DrupalDatetimeMultivalueForm` React component (in `DrupalDatetimeMultivalueForm.tsx`) renders Drupal's native date and time inputs off-screen and drives a Radix UI popover interface. Whether the field is date-only or datetime is detected at runtime by checking for a `time` input in the hidden DOM. A corresponding Twig template `datetime-wrapper--multivalue-form.html.twig` outputs a `<drupal-canvas-datetime-wrapper--multivalue-form>` custom element that the React component hydrates, and it is registered in the `twigToJSXComponentMap` under that tag name.

On the PHP side, `canvas_stark_theme_suggestions_datetime_wrapper_alter()` adds the `datetime_wrapper__multivalue_form` theme suggestion when `#multivalue_field_label` is set on the element. `canvas_stark_preprocess_datetime_wrapper()` was extended to pass `fieldLabel` to the template. The `markMultivalueFormElements()` method in `ReduxIntegratedFieldWidgetsHooks` now stops recursing into `datetime` elements and instead stamps `#multivalue_field_label` on them directly, so the entire date+time widget is treated as one unit rather than having its child inputs processed independently. The `hook_preprocess_field_multiple_value_form` implementation was also moved from `ReduxIntegratedFieldWidgetsHooks` into `canvas_stark.theme`, and extended to add a `canvas-multivalue-field-content` class on the table's content column and a `data-canvas-remove-button` data attribute on the action column.

Shared utilities previously duplicated in `DrupalInputMultivalueForm` were extracted to a new `multivalueFormUtils.ts` module: `isRemoveButtonEnabled`, `triggerDrupalRemoveButton`, `copyInputAttributes`, and `updateInputValue`. These are now consumed by both `DrupalInputMultivalueForm` and `DrupalDatetimeMultivalueForm`. Three test fields were added to the `canvas_test_article_fields` test module (`field_cvt_unlimited_datetime`, `field_cvt_unlimited_date`, `field_cvt_required_datetime`) along with comprehensive Cypress tests covering popover editing, date-only vs. datetime detection, "+ Add new", drag-to-reorder, and remove interactions.
