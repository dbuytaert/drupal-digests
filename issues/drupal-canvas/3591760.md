# #3591760: Canvas should mark multi-line string props on SDCs and code components as translatable

- **Project:** Drupal Canvas
- **Type:** Critical task
- **Version:** 1.x-dev
- **Status:** Fixed
- **Discussion:** [#3591760](https://git.drupalcode.org/project/canvas/-/work_items/3591760) · 2 contributors · 5 comments

## Summary

In the Canvas module, multi-line string props (`type: string` with `pattern: (.|\r?\n)*`) on SDCs and code components were not being offered for translation in either the config translation UI or TMGMT. Additionally, the config schema generator and TMGMT extractor were using separate, independently maintained rules to decide which prop shapes are translatable, meaning they could silently diverge and produce mismatches where TMGMT would offer an input that the config schema would then reject on save (or the reverse). Both issues are now resolved.

## Impact

No upgrade or configuration changes required.

## Technical details

The fix extracts the inline translatability check from `JsonSchemaPropsComponentInstanceInputsConfigSchemaGenerator` into a new `@internal public static isTranslatableShape(array $prop_shape): bool` method on the same class. `ComponentInputsTranslatablesExtractor` (the TMGMT side) now calls `isTranslatableShape()` instead of carrying its own copy of the logic, making the two subsystems share a single source of truth.

The extracted method also fixes two correctness gaps in the original inline check. First, SDC appends `object` to every declared `type` value, producing an array like `['string', 'object']` instead of the scalar `'string'` the old code expected; `isTranslatableShape()` normalizes this with `((array) $prop_shape['type'])[0]` before comparing. Second, the old array-peeking code asserted the presence of an `items` key only after checking `type === 'array'`, which could miss raw SDC metadata that has `items` but a compound type; the new code simply checks `array_key_exists('items', $prop_shape)`.

`ComponentInputsTranslatablesExtractor` was also refactored to count non-computed stored properties on the field item via `FieldItemDataDefinitionInterface::getPropertyDefinitions()` filtered by `!isComputed()`, replacing a looser approach that compared against `FieldStorageDefinitionInterface::CARDINALITY_UNLIMITED`. This gives an accurate stored-property count to determine whether a single-property field item collapses to a scalar (and the correct config schema type to emit) regardless of cardinality.

A new `canvas_test_translation` test module adds an `untranslatable-prop-shapes` SDC component with `date`, `email`, `integer` and `boolean` props to lock in that those shapes are never offered for translation, and a new unit test class `JsonSchemaPropsComponentInstanceInputsConfigSchemaGeneratorTest` covers the full range of translatable and non-translatable shapes.
