## Summary

When building or editing a content template in Canvas, site builders can now select which content entity to use for previewing that template directly from the top navigation bar. A new dropdown lists all available entities of the matching bundle; choosing one navigates the canvas to that entity. When no content exists yet for the template's bundle, a dedicated full-screen notice explains the situation and provides an "Add new {bundle}" button that opens Drupal's content creation UI in a new tab.

## Impact

Affects module maintainers.

- **New API:** New `useGetPreviewContentEntitiesQuery` hook and `PreviewContentEntitiesResponse` type for listing preview-eligible entities by entity type and bundle.

## Technical details

The `TopBar` component now conditionally renders a new `ContentPreviewSelector` dropdown on any route starting with `/template/`. It feeds the dropdown by calling `useGetPreviewContentEntitiesQuery`, a new RTK Query endpoint that hits `canvas/api/v0/ui/content_template/suggestions/preview/{entityTypeId}/{bundle}`. Selecting an item navigates to `/template/:entityType/:bundle/:viewMode/:previewEntityId`.

A new route `/template/:entityType/:bundle/:viewMode` (without `previewEntityId`) is registered alongside the existing `:previewEntityId` route. It renders the new `TemplateRoot` component, which on mount queries `useGetContentTemplatesQuery` and immediately redirects (with `replace`) to the suggested entity if one is available. When none is found, it renders the no-content notice; the "Add new" button constructs a URL as `{baseUrl}{entityType}/add/{bundle}` using `getBaseUrl()`.

`PageInfo` gains template-route awareness: it shows a `FileIcon` and a caption of the form `{bundleLabel} - {viewModeLabel} template`, resolved by looking up bundle and view mode labels from the existing `useGetContentTemplatesQuery` response.

New types `PreviewContentEntity` and `PreviewContentEntitiesResponse` are added in the services file, and `PreviewContentEntities` is registered as a new RTK cache tag. The `TemplateListItem` link fallback is also corrected from `0` to an empty string for cases where `suggestedPreviewEntityId` is absent.
