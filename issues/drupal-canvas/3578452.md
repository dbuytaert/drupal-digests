## Summary

The Canvas CLI can now handle pages that contain image props during both push and pull. On pull, the resolved image data (src, alt, width, height) is written to the local JSON spec alongside a new `_provenance` field that preserves the Drupal media entity reference. A new `npx canvas reconcile-media` command downloads external images referenced in local specs, uploads them to Drupal, and records the resulting media entity IDs in `_provenance`. The push command now guards against unreconciled external image URLs and directs users to run `reconcile-media` first. A new `canvas:media:view` OAuth scope is also installed automatically on existing sites via a database update.

## Impact

Affects module maintainers and site owners.

- **Database update:** Post-update hook `canvas_oauth_post_update_0004_media_view_scope` creates the `canvas:media:view` OAuth scope on existing sites.
- **Config change:** New `canvas_media_view` OAuth scope config entity installed by `canvas_oauth_post_update_0004_media_view_scope`.
- **New API:** New `reconcile-media` CLI command and `ApiService.uploadMedia()` method for uploading external images to Drupal media entities.

## Technical details

The core challenge: json-render page specs store resolved media props (src/alt/width/height) for local rendering, but Drupal's API expects entity references (`target_id`/`target_uuid`). The solution is a new `_provenance` field on `AuthoredSpecElement` that holds the Drupal-native form of media inputs alongside the human-readable `props`.

**Pull direction** (`pageToAuthoredSpec`): A new `extractMediaProvenance` helper compares each component node's `inputs` and `inputs_resolved`. When a prop in `inputs_resolved` looks like resolved media (has a `src` string) and the corresponding entry in `inputs` is an entity reference (`target_id` or `target_uuid`), that `inputs` value is written as `_provenance[propName]` in the local spec file.

**Push direction** (`serializeElementMapForServer`): A new `mediaTransformer` is first in the transformer chain in `prop-transforms.ts`. When `_provenance` carries a `target_id` or `target_uuid` for a media prop, only that key is forwarded to Drupal (the CLI-only `source_url` is stripped). `serializePropsForServer` receives the element's `_provenance` map as a third argument so each prop transformer can access it via the context object.

**`reconcile-media` command** (`commands/reconcile-media.ts`):
1. Scans all local page specs with `collectUnreconciledMediaProps`, which calls `getUnreconciledMedia` per prop. A prop is unreconciled when its `src` is an absolute external URL or a `data:` URL.
2. Deduplicates by URL so each image is downloaded and uploaded only once.
3. Downloads via `downloadExternalMedia` (axios with `request-filtering-agent`) or `downloadDataUrlMedia` for inline data URLs.
4. Validates the MIME type against `SUPPORTED_IMAGE_TYPES` (jpeg, png, gif, webp, avif); SVG is explicitly rejected.
5. Uploads via the new `ApiService.uploadMedia()` method, which POSTs multipart/form-data to `/canvas/api/v0/media/:mediaType/upload` (the endpoint added in [#3578449](https://www.drupal.org/node/3578449)).
6. Applies results: updates `props[propName]` to `inputs_resolved` from Drupal and writes `_provenance[propName] = { target_id, source_url }`.
7. Writes updated specs back to disk.

**Push guard**: `preparePages` now calls `collectUnreconciledMediaProps` per page and aggregates the results as `pendingMediaReconciliations`. If any are found, `pushCommand` throws before contacting Drupal, instructing the user to run `npx canvas reconcile-media`.

**OAuth scope**: A new `canvas_media_view` config entity and `canvas_oauth_post_update_0004_media_view_scope` post-update hook ensure the `canvas:media:view` scope exists on upgraded sites. The CLI's default scope string (`DEFAULT_SCOPE`) now includes `canvas:media:image:create` and `canvas:media:view`.

**`handleApiError` fix**: The method's return type was corrected from `void` to `never`, which allowed removal of all unreachable `throw` statements that followed calls to it throughout `ApiService`.

This work builds on the json-render conversion utilities from [#3573996](https://www.drupal.org/node/3573996) and the `inputs_resolved` computed property from [#3577654](https://www.drupal.org/node/3577654).

## Upgrade

Run database updates to install the new `canvas:media:view` OAuth scope on existing sites:

```bash
drush updb
```

For CLI users: if your local page specs contain image props whose `src` is an external URL, run the new reconcile step before pushing:

```bash
npx canvas reconcile-media
npx canvas push
```
