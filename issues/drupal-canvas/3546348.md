## Summary

The Canvas module's OpenAPI spec (`openapi.yml`) is now validated automatically on every merge request by a new `lint (OpenAPI)` CI job. Alongside adding that gate, the spec itself was corrected throughout: field types were updated to OpenAPI 3.1 syntax, operation metadata was reorganized, and several schema definitions were made more precise. The `devizzent/cebe-php-openapi` dev dependency was also bumped from `^1.0.3` to `^1.1.4`. These changes improve the reliability of the spec as a source of truth for anyone building clients or reading documentation, without altering any actual HTTP API behavior.

## Impact

No upgrade or configuration changes required.

## Technical details

The existing PHP-based `OpenApiSpecValidationTest` was not catching the structural errors that appeared in external tools like Swagger Editor. The new `lint (OpenAPI)` job in `.gitlab-ci.yml` runs `npx @redocly/cli@latest lint openapi.yml` on a `node:24-alpine` image with no stage dependencies (`needs: []`). A `.redocly.lint-ignore.yaml` file records 87 currently-known violations across three Redocly rules (`struct`, `no-required-schema-properties-undefined`, `path-parameters-defined`) that are deferred to follow-up [#3546404](https://www.drupal.org/node/3546404).

The concurrent `openapi.yml` corrections are extensive. The most systematic change replaces the OpenAPI 2.0-style `nullable: true` pattern with the OAS 3.1-correct `type: [X, "null"]` array form across all nullable fields (`string`, `integer`, `object`). Operation-level `description` fields are moved to `summary` fields as the spec requires. Inline object shapes that were repeated across multiple response bodies are extracted into new reusable component schemas: `ClientSideTransforms` (component prop transforms keyed by prop name) and `AutoSaveConcurrencyControl` (auto-save entries with hash values for concurrency control, used in layout patch/post responses). Several `items` sub-schemas that incorrectly used a nested `schema:` key (OAS 2.0 style) are corrected to direct `$ref` references. The `Component` schema gains `additionalProperties: false` and corrected required fields (`source`, `css`, `js_header`, `js_footer`, `version`). The `StagedConfigUpdate` request body fixes an incorrect `schema: $ref:` nesting to a direct `$ref:`. A top-level `security: []` declaration is added to `openapi.yml` to document that routes rely on Drupal session auth or OAuth2 via the `canvas_oauth` submodule.
