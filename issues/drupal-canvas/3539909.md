# #3539909: Access bypass - ApiLayoutController::componentInstanceExistInComponentData doesn't correctly recurse the tree

## Summary

An access bypass existed in the Canvas page builder's layout API. When a PATCH request was made to update a component instance, the controller was supposed to check whether the user had edit access to the page region containing that component. Due to a flawed recursive tree search, the component could fail to be located, causing the access check to be silently skipped. This allowed an authenticated user to patch components in restricted page regions without the required permission.

## Impact

Affects site owners.

- **Security:** Access bypass in `ApiLayoutController::patch()` allowed updating components in restricted page regions without edit permission

## Technical details

The bug lived in `ApiLayoutController::componentInstanceExistInComponentData()`. Its recursion was broken: it checked `!empty($slotData)` before examining the slot's child components, meaning slots without deeply nested children were never searched. The outer `getRegionForComponentInstance()` caller returned `NULL` when no region was found, and the access check in `patch()` was written as `if ($regionForComponentId !== MAIN_CONTENT_REGION && NULL !== $regionForComponentId)`, so a `NULL` result bypassed the `PageRegion::access('edit')` guard entirely.

The fix landed as part of [#3541057](https://www.drupal.org/node/3541057). The entire `componentInstanceExistInComponentData` / `getRegionForComponentInstance` pair was deleted and replaced with `getEntityWithComponentInstance()`, which delegates tree traversal to `ComponentTreeItemList::getComponentTreeItemByUuid()`. That method correctly walks the full component tree. The refactored `patch()` method calls `getEntityWithComponentInstance()` to identify which entity (or `PageRegion`) owns the target component, then performs a direct `$entity_to_patch->access('edit')` check on that entity before proceeding. A `NotFoundHttpException` is thrown if the UUID is absent from the tree, eliminating the silent-bypass path.
