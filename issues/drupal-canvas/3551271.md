## Summary

The Canvas module's pages overview at `/admin/content/pages` now has an "Add page" action button, matching the standard Drupal pattern for content overview pages. Clicking it immediately creates a new unpublished Canvas page titled "Untitled page" and redirects the user to its edit form. The button is only shown to users who have the `create canvas_page` entity access.

## Impact

Affects module maintainers.

- **New API:** New route `canvas.canvas_page.add` at `/admin/content/pages/add` and named overview route `canvas.canvas_pages_overview` at `/admin/content/pages`.

## Technical details

The change introduces a new `PageController` class in `Drupal\canvas\Controller` with a single `addPage()` method. The method calls `Page::create()` with `title = 'Untitled page'` and `status = 0` (unpublished), saves the entity, then issues a `RedirectResponse` to `entity.canvas_page.edit_form`.

Two new routes are registered in `canvas.routing.yml`. The `canvas.canvas_pages_overview` route (path `/admin/content/pages`) formalises the existing overview as a named admin route protected by the `edit canvas_page` permission. The `canvas.canvas_page.add` route (path `/admin/content/pages/add`) points to `PageController::addPage`, enforces `_entity_create_access: 'canvas_page:canvas_page'`, and requires a CSRF token. A local action link (`canvas.add_canvas_page`) is wired to appear on the overview route via `canvas.links.action.yml`, and a menu link entry for the overview is added to `canvas.links.menu.yml` under `system.admin_content`.

A minor typo was also corrected in `canvas.permissions.yml`: a stray backtick was appended to the "Delete Canvas Page" description.
