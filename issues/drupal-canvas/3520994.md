## Summary

The code editor in Drupal Canvas (formerly Experience Builder) has replaced its panel layout library. The previous implementation used `react-mosaic-component`, which brought in a large drag-and-drop dependency chain (`react-dnd`, `react-dnd-multi-backend`, `rdndmb-html5-to-touch`) for a feature the editor never used (window drag-and-drop). That cluster also caused a crash when users navigated quickly between routes, requiring a workaround involving `DndProvider` context placement. The panel layout is now built on `react-resizable-panels`, which is lighter and purpose-built for resizable split panes. The editor, preview, and component-data panels remain in the same three-pane arrangement, but the toggle icon and visual styling of resize handles have been updated.

## Impact

No upgrade or configuration changes required.

## Technical details

The old `MosaicContainer` component used `MosaicWithoutDragDropContext` and `MosaicWindow` from `react-mosaic-component`. Because `react-mosaic` bundles `react-dnd`, a `DndProvider` wrapper was required in `App.tsx` to prevent a "Cannot have two MultiBackends at the same time" crash (see [#3510436](https://www.drupal.org/node/3510436)). That wrapper, along with the `HTML5toTouch` multi-backend configuration, has been entirely removed.

The replacement is `CodeEditorContainer`, which uses `PanelGroup`, `Panel`, and `PanelResizeHandle` from `react-resizable-panels`. Layout state that was previously a `MosaicNode<string>` tree (`defaultLayout` / `fullEditorLayout`) is now a single boolean `maximizedEditorLayout`. When `maximizedEditorLayout` is false, the right-side `Panel` and `PanelResizeHandle` are rendered; when true, they are hidden, giving a full-width editor. The iframe overlay hack in `Preview` that suppressed mouse-event interception during Mosaic resize events has been removed, as `react-resizable-panels` handles this natively.

The CSS approach also changed: the 174-line `canvas-react-mosaic-component.css` global override file and the `canvas-code-mirror.css` global file were removed. Styles are now in CSS Modules (`CodeEditorContainer.module.css` and `CanvasCodeMirror.module.css`). Playwright test selectors were updated from Mosaic-specific CSS classes (`.canvas-mosaic-window-editor`, `.canvas-mosaic-window-preview`, `.canvas-mosaic-window-component-data`) to `data-testid` attributes (`canvas-code-editor-main-panel`, `canvas-code-editor-preview-panel`, `canvas-code-editor-component-data-panel`, `canvas-code-editor-container`). A Vitest optimizer workaround (`include: ['react-mosaic-component'], exclude: ['react-dnd']`) that was needed to prevent a `require()`-of-ESM error in tests was also removed.
