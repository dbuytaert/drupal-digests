# #3555487: default-relative-url may crash with JSON storage

## Summary

On MySQL 8.0, which stores JSON natively rather than as a string, the Canvas module can crash with a 500 error when rendering pages that use default prop values for image or video components. This caused the Byte demo home page of Drupal CMS 2.0 to fail on MySQL 8.0. The fix landed in the related issue [#3550165](https://www.drupal.org/node/3550165).

## Impact

No upgrade or configuration changes required.

## Technical details

The crash originates in `Drupal\canvas\PropSource\DefaultRelativeUrlPropSource`, which compares a stored JSON schema against a minimal expected shape using `ksort()` before an equality check. That one-level sort does not recurse into nested `properties` keys, so when MySQL 8.0's native JSON type returns property keys in a different order than MariaDB's string-encoded JSON, the comparison fails and an exception is thrown. Versions of MariaDB that DDEV defaults to store JSON as strings, preserving insertion order and masking the bug; MySQL 8.0 does not, exposing it. The fix, applied in [#3550165](https://www.drupal.org/node/3550165), adds a second pass of `ksort()` over each entry inside `properties` for both the stored schema and the minimal schema before the equality check.
