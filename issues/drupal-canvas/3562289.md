## Summary

Code Components can now use TypeScript syntax in addition to JavaScript. This enables developers to write components with type annotations for better code quality and IDE support. The TypeScript code is parsed and transpiled to JavaScript, but type checking itself is not performed. Developers using the CLI tool can add their own type-checking workflow. This enhancement makes the Code Components feature more aligned with modern JavaScript development practices.

## Impact

No upgrade or configuration changes required.

## Technical details

The JavaScript compiler powered by SWC was updated to parse TypeScript syntax by changing the parser configuration from `syntax: 'ecmascript'` with `jsx: true` to `syntax: 'typescript'` with `tsx: true`. This allows the compiler to handle TypeScript syntax including type annotations, interfaces, and type parameters.

The CodeMirror editor instance was updated to use the TypeScript language extension, enabling proper syntax highlighting for TypeScript code. The Babel parser used in the preview functionality also received the `typescript` plugin to correctly parse TypeScript when analyzing imports and exports.

The implementation does not perform type checking. SWC strips type annotations during transpilation, converting TypeScript to JavaScript. Developers using the CLI tool can add separate type checking using standard TypeScript tooling if desired.

A test fixture component was converted from `Menu.jsx` to `Menu.tsx` with TypeScript interfaces and type annotations to ensure Playwright test coverage for TypeScript components.
