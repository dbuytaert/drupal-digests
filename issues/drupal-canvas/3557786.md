## Summary

When a Canvas component instance had one or more props linked to a `DynamicPropSource`, opening the Media Library widget for an image prop would crash with a `LogicException`. This is now fixed. Sites using Canvas components with dynamic prop sources and image fields can open the Media Library normally again.

## Impact

No upgrade or configuration changes required.

## Technical details

The crash originated from a `LogicException` introduced in [#3554207](https://www.drupal.org/node/3554207), which deliberately prevented `Component` config entities from being serialized into form state. That earlier fix removed `#component` from the top-level `ComponentInstanceForm`, but missed a second location: `GeneratedFieldExplicitInputUxComponentSourceBase` was still setting `'#component' => $component` on each individual prop's form element array. When the Media Library widget triggered form-state serialization, it encountered the `Component` entity on those child elements and hit the guard exception.

The fix is a single-line removal of `'#component' => $component` from the per-prop form element in `GeneratedFieldExplicitInputUxComponentSourceBase`. The `$component` reference was already unnecessary in that context, mirroring the same cleanup done for the parent form in [#3554207](https://www.drupal.org/node/3554207).
