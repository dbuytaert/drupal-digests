## Summary

Site editors and content authors using the Canvas editor on multilingual Drupal sites will now see a checkmark icon next to each language in the Canvas language switcher dropdown that has a dedicated translation for the current entity. Languages without a translation show no indicator, making it immediately clear that selecting them will display a default-language fallback rather than a dedicated translation. The goal is not to hide untranslated languages (since other referenced entities may still render differently), but to communicate translation state clearly.

## Impact

Affects module maintainers.

- **New API:** New `translations.available` array of language codes added to the `ApiLayoutController::get()` JSON response.

## Technical details

The `ApiLayoutController::get()` method is extended to append a `translations` key to its JSON response. It calls `getTranslationLanguages()` on the entity (guarded by `method_exists` to safely handle non-translatable entity types), extracts language codes with `array_keys()`, and returns them as `translations.available` (an array of language code strings). The OpenAPI spec in `openapi.yaml` is updated to document the new `translations` object, which is added to both the path-level response schema and the reusable component schema.

On the frontend, `LayoutModelSliceState` in `layoutModelSlice.ts` gains a `translations` field typed as `Record<string, any>`, with a new `selectTranslations` selector. The `setInitialLayoutModel` reducer stores the translations payload from the API response. The `LanguageSelect` component reads this via `useAppSelector(selectTranslations)` and, for each language option, renders a `CheckIcon` (from `@radix-ui/react-icons`) in a fixed-width container when the language code appears in `translations.available`. A `data-canvas-has-translation="true"` attribute is also set on the language name text element for Playwright test targeting; the new end-to-end test assertions verify that translated languages (English, French) carry this attribute while untranslated ones (Spanish) do not. The `filterLayoutForPost` helper in the PHP test base is updated to strip `translations` from POST payloads, and kernel tests assert that the `translations.available` array is present in GET responses.
