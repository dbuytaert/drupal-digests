## Summary

The Canvas module now has a formal, YAML-based plugin system for building extensions. Developers can ship a `[module].canvas_extension.yml` file alongside their Drupal module to register one or more extensions, and use the new `@drupal-canvas/extensions` npm package to read live Canvas editor data (current preview HTML and the selected component UUID) from within their extension's iframe. A new REST endpoint at `GET /canvas/api/v0/extensions` lists all discovered extensions and their metadata. The Canvas editor UI now checks whether any extensions are registered and adjusts its interface accordingly.

## Impact

Affects module maintainers.

- **New API:** New `CanvasExtensionPluginManager` plugin type discovered from `[module].canvas_extension.yml` files, plus the `@drupal-canvas/extensions` npm package with `getPreviewHtml()`, `subscribeToPreviewHtml()`, `getSelectedComponentUuid()`, and `subscribeToSelectedComponentUuid()` for iframe-based extensions.

## Technical details

The PHP side is built on `CanvasExtensionPluginManager`, a `DefaultPluginManager` subclass that uses `YamlDiscovery` to locate `[module].canvas_extension.yml` files across all enabled modules. `processDefinition()` enforces two validation rules: `api_version` is required (throws `ExtensionValidationException` if missing), and `url`/`icon` values must not begin with `/` (relative paths are resolved to absolute server paths under the module root; full HTTP URLs are passed through unchanged). Definitions are returned as `CanvasExtension` instances (implementing `CanvasExtensionInterface`) backed by a `CanvasExtensionTypeEnum` PHP 8.1 backed enum with `canvas` and `code-editor` variants. Multiple extensions can be defined in a single YAML file.

`ApiExtensionControllers::list()` serves `GET /canvas/api/v0/extensions` as a `CacheableJsonResponse`. Extension `permissions` are intentionally omitted from the API response as a security measure; the normalized payload includes `id`, `name`, `description`, `icon`, `url`, `type`, and `api_version`. `CanvasController` now injects `CanvasExtensionPluginManager` and sets `drupalSettings.canvas.extensionsAvailable` based on whether any extensions are discovered.

The `@drupal-canvas/extensions` npm package (at `packages/extensions/`, published as `@drupal-canvas/extensions`) exposes four TypeScript functions: `getPreviewHtml()`, `subscribeToPreviewHtml(callback)`, `getSelectedComponentUuid()`, and `subscribeToSelectedComponentUuid(callback)`. The underlying protocol uses typed `window.postMessage()` messages following the pattern `canvas:data:{get|subscribe|unsubscribe}:{dataType}` between the extension iframe and the Canvas parent app. Subscriptions return an unsubscribe function that sends a matching `unsubscribe` message and removes the event listener.

As a housekeeping change in this same commit, the Twig helper classes `CanvasTwigExtension`, `CanvasWrapperNode`, and `CanvasPropVisitor` were moved from the `Drupal\canvas\Extension\` namespace to `Drupal\canvas\Twig\`. The existing legacy test extension was renamed to `canvas_test_extension_legacy` to make way for a new `canvas_test_extension` that demonstrates the new plugin and JS API.

## Upgrade

**Define a Canvas extension in PHP (YAML plugin):**

```yaml
# [your_module].canvas_extension.yml
my_canvas_extension:
  name: My Extension
  description: What this extension does.
  url: dist/index.html        # relative to your module root, or a full https:// URL
  icon: assets/icon.svg       # relative or full URL; must NOT start with "/"
  api_version: "1.0"
  type: canvas                # or: code-editor
  permissions:
    - access content
```

**Read Canvas editor data from inside your extension iframe (JavaScript):**

```bash
npm install @drupal-canvas/extensions
```

```typescript
import {
  getPreviewHtml,
  subscribeToPreviewHtml,
  getSelectedComponentUuid,
  subscribeToSelectedComponentUuid,
} from '@drupal-canvas/extensions';

// One-time fetch:
const html = await getPreviewHtml();
const uuid = await getSelectedComponentUuid();

// Subscribe to changes:
const unsubscribe = subscribeToSelectedComponentUuid((uuid) => {
  console.log('Selected:', uuid);
});
// Call unsubscribe() when done.
```

**Namespace change for Twig helper classes** (if your code referenced them directly):

```php
// Before:
use Drupal\canvas\Extension\CanvasTwigExtension;
use Drupal\canvas\Extension\CanvasWrapperNode;
use Drupal\canvas\Extension\CanvasPropVisitor;

// After:
use Drupal\canvas\Twig\CanvasTwigExtension;
use Drupal\canvas\Twig\CanvasWrapperNode;
use Drupal\canvas\Twig\CanvasPropVisitor;
```
