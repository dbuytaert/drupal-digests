## Summary

The `@drupal-canvas/eslint-config` package now properly parses TypeScript syntax when linting Canvas component code. Previously, the package could identify TypeScript files for linting but couldn't understand TypeScript-specific syntax like interfaces and type annotations. This fix ensures TypeScript code in Canvas components is correctly validated, preventing false errors and enabling proper code quality checks for TypeScript developers building Canvas components.

## Impact

No upgrade or configuration changes required.

## Technical details

The issue arose from [#3577508](https://www.drupal.org/project/canvas/issues/3577508) which added support for linting `.ts` and `.tsx` files but didn't include the parser needed for TypeScript syntax. Without `typescript-eslint`, ESLint could target these files but would fail on TypeScript-specific constructs like interfaces, type annotations, and generics.

The fix integrates the `typescript-eslint` package across all config variants. The `required` config now uses `tseslint.parser` as the language parser. The `recommended` config includes `tseslint.configs.recommended` rules, and the `strict` config adds `tseslint.configs.strict`. The `typescript-eslint` dependency was moved from `devDependencies` to regular `dependencies` by removing the `"dev": true` flags from multiple packages in `package-lock.json`.

Tests were updated to use the TypeScript parser and include a test case with TypeScript syntax (interface and type annotations) to verify proper parsing.
