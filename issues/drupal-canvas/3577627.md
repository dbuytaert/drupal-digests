## Summary

The `@drupal-canvas/eslint-config` package can now properly parse and lint TypeScript syntax in Canvas components. Previously, the package could lint TypeScript files but lacked the parser needed to understand TypeScript-specific syntax like interfaces and type annotations. This fix ensures TypeScript components are validated correctly during development, catching errors that would otherwise slip through.

## Impact

No upgrade or configuration changes required.

## Technical details

The `@drupal-canvas/eslint-config` package added `typescript-eslint` as a dependency and integrated it into all configuration variants. The `required` config now uses `tseslint.parser` as its language parser, enabling ESLint to understand TypeScript syntax like interfaces, type annotations, and generics.

The `recommended` config includes `tseslint.configs.recommended` rules, while the `strict` config adds `tseslint.configs.strict` for more rigorous TypeScript validation. These changes build on top of the existing React and YAML linting rules.

The fix addresses an issue introduced in #3577508, where TypeScript file extensions were added to the linting file patterns but the parser was not configured to handle TypeScript syntax. The package-lock.json changes remove the `"dev": true` flag from `typescript-eslint` and its dependencies, making them production dependencies rather than dev-only packages. Test coverage was expanded to include a TypeScript component example with interface declarations.
