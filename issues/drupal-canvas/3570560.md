## Summary

Canvas now requires the `in_array()` function to be written as `\in_array()` throughout the codebase. This is a code style change that makes PHP slightly faster by avoiding namespace lookups for this commonly used function. The change applies only to Canvas's internal code and does not affect how site builders or module developers write their own code.

## Impact

No upgrade or configuration changes required.

## Technical details

This change adds `in_array` to the list of global functions that must be fully qualified with a leading backslash in Canvas's `phpcs.xml` configuration. The `SlevomatCodingStandard.Namespaces.FullyQualifiedGlobalFunctions` rule now enforces `\in_array()` instead of `in_array()`.

The leading backslash tells PHP to skip namespace resolution and call the global function directly. This provides a small performance benefit in namespaced code by eliminating unnecessary symbol table lookups. The change affects 180+ call sites across the codebase.

The diff shows the mechanical replacement of `in_array(` with `\in_array(` and `!in_array(` with `!\in_array(`. The updated phpcs.xml now includes a reference to PHP's internal compile-time optimization list, documenting why certain functions receive this treatment.

This is part of a gradual rollout to eventually apply this standard to all global function calls in Canvas, building on earlier work with `assert()`, `sprintf()`, `array_map()`, and type-checking functions like `is_array()`.
