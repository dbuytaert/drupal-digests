## Summary

Drupal Canvas now organizes all its JavaScript packages in a unified workspace structure with consistent linting and TypeScript configuration. The `cli` package moved from the root directory into `packages/cli`, and the `astro-hydration` library moved from inside the UI directory to `packages/astro-hydration`. A single ESLint configuration now covers the entire repository, replacing multiple separate configs. All packages share a common TypeScript base configuration, making it easier to maintain code quality standards across the project. This cleanup prepares Canvas to adopt Turborepo for better dependency management and faster builds in future work.

## Impact

No upgrade or configuration changes required.

## Technical details

The Canvas project had grown organically with JavaScript packages scattered across different directories and inconsistent tooling. The `cli` package lived at the repository root, `astro-hydration` nested inside `ui/lib/`, and various packages had their own ESLint and TypeScript configurations that didn't align.

This change consolidates the structure. The `cli` package moved to `packages/cli` alongside other published packages like `@drupal-canvas/create` and `@drupal-canvas/eslint-config`. The `astro-hydration` library moved from `ui/lib/astro-hydration` to `packages/astro-hydration`, requiring updates to library paths in `canvas.libraries.yml` and the `JsComponent` plugin class.

All packages now use npm workspaces properly, with a root `package.json` listing `packages/*`, `ui`, and `docs/user` as workspaces. A shared `eslint.config.mjs` at the repository root replaces separate configs, incorporating rules for React, TypeScript, Vitest, Cypress, and Mocha tests. The configuration uses flat config format with plugins for `eslint-plugin-react`, `eslint-plugin-react-hooks`, `eslint-plugin-vitest`, and `eslint-plugin-cypress`.

A base `tsconfig.json` in the repository root provides common TypeScript compiler options that all packages extend. Individual packages have minimal `tsconfig.json` files with just `"extends": "../../tsconfig.json"`. A new `@drupal-canvas/types` package centralizes shared TypeScript types like `DrupalSettings`, `FormatType`, and `PropsValues`.

GitLab CI pipelines were refactored to work with the new structure. The `UI` job now runs `npm ci` at the root and builds with `npm run build --workspace=@drupal-canvas/ui`. The `eslint` job consolidates what were previously separate `eslint` and `UI eslint` jobs into a single validation step. Cypress tests configure `CYPRESS_CACHE_FOLDER` as a variable and use `npx cypress` instead of direct node module paths. New `prettier` and `type-check` jobs validate code formatting and TypeScript compilation across all workspaces.

The `CLI Test` and `CLI Publish` jobs updated their workspace references from `cli` to `@drupal-canvas/cli`. Import map entries in `JsComponent.php` changed paths from `ui/lib/astro-hydration/dist` to `packages/astro-hydration/dist`.

## Lessons learned

- When packages in a monorepo are added incrementally without proper tooling, dependencies between packages become difficult to manage and publishing becomes error-prone. Consolidate all packages into a unified workspace with proper monorepo tooling (like Turborepo) early to enable automatic publishing, caching, and parallelization. *(via wotnak)*
- npm workspaces don't handle nested packages well when one package directory contains another. Flatten the structure by moving nested packages to sibling directories and use build scripts to copy artifacts to expected locations if needed. *(via wotnak)*
