## Summary

AI agents in the Canvas editor were broken: tools such as `ai_agent:get_entity_information` could not determine which entity they were operating on, causing every agent request to fail. The root cause was a prior change that removed `entityType` and `entity` from the `drupalSettings.canvas` object, leaving the AI Wizard component with no way to supply those values to the server or to the post-operation redirect. After this fix the AI chat interface works correctly and, once all operations complete, the editor redirects to the correct entity-scoped URL instead of the generic `/editor` path.

## Impact

No upgrade or configuration changes required.

## Technical details

The `AiWizard` component was reading `drupalSettings.canvas.entityType` and `drupalSettings.canvas.entity` when building request bodies for AI agent calls. Both keys were removed in a prior refactor (work item 3502887), so those reads produced `undefined`, breaking every agent that depended on entity context.

The fix threads the React Router `params` object (which carries `entityType` and `entityId` from the URL) into the component's `currentValuesRef` so it stays current inside async closures. `operationsHandler` also receives `params` as an argument. Request body construction now reads `currentValuesRef.current.params.entityType` and `currentValuesRef.current.params.entityId` instead of the removed `drupalSettings.canvas` keys, covering both the JSON and `FormData` request paths.

A secondary bug was also fixed: after placing components the handler was calling `navigate('/editor')`, a generic path that navigated away from the current entity. It now calls `navigate(\`/editor/${entityType}/${entityId}\`)` using the same `params` values, returning the user to the correct editor page.

During review, narendrar noted that simply removing the `navigate` call was not enough: the earlier addition of `navigate('/editor')` had been deliberate (to reset component selection context after page creation, per [#3537739](https://www.drupal.org/node/3537739)).
