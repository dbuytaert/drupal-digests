# #3571116: PageRegion's cache tags not bubbling, resulting in stale results being served by (Dynamic) Page Cache

## Summary

When you published changes to a Canvas page region, those changes would not appear on the site until you cleared all caches. The page cache and dynamic page cache continued serving stale versions of the page because cache tags from page regions were not properly bubbling up to the final rendered page. This meant updates to page regions were invisible to visitors until a full cache rebuild, which is impractical for live sites and would require invalidating an entire CDN.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas component trees can be stored in either content entities (like `Page`) or config entities (like `PageRegion`, `ContentTemplate`, and `Pattern`). When rendering a component tree, `ComponentTreeItemList::toRenderArray()` was creating cacheability metadata from the `RenderableComponentTree` but not from the host entity itself. This meant that for `PageRegion` config entities rendered by `CanvasPageVariant`, the `config:canvas.page_region.*` cache tags were never added to the final page render array.

The fix ensures `ComponentTreeItemList::toRenderArray()` calls `addCacheableDependency()` with the host entity after getting cacheability from the renderable tree. This bubbles the host entity's cache tags (like `config:canvas.page_region.olivero.header`) to the render array, ensuring pages are invalidated when page regions change.

The fix also required plumbing the host entity through more of the component tree infrastructure. `ComponentTreeItemListInstantiatorTrait::createDanglingComponentTreeItemList()` now accepts `ComponentTreeEntityInterface` in addition to `FieldableEntityInterface`, and `ComponentTreeItem::calculateFieldItemValueDependencies()` was updated to handle both types. Two workarounds were added to `ComponentTreeConfigEntityBase` for Typed Data static caching bugs in core (tracked in [#3571532](https://www.drupal.org/node/3571532)).

## Collaboration

This critical bug was discovered by penyaskito during Canvas page region testing and diagnosed by Wim Leers, who traced the missing cache tags through multiple layers of the rendering system. The fix evolved through four iterations on the merge request, starting with a minimal fix and expanding into a systemic improvement that makes Canvas more robust. The process uncovered technical debt in how host entities were passed through the component tree infrastructure, which was addressed to prevent similar issues in the future. Test coverage was expanded to verify cache tags bubble correctly for all entity types that can contain component trees.

## Lessons learned

- When rendering nested components that bubble cache metadata, setting cacheable metadata on the wrong render array level prevents it from reaching the outer layers. Cache tags and contexts must be bubbled to the top-level render array that will be cached, not to intermediate child elements. *(via wim leers)*
