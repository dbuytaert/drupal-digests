## Summary

The Canvas module's `ComponentSourceManager::generateComponents()` method can now regenerate `Component` config entities for a single component source plugin and/or a specific subset of components within that source, instead of always regenerating every component across all sources. The immediate effect is a performance improvement when saving a JavaScript component entity: Canvas now only regenerates the one affected `Component` config entity rather than the entire set.

## Impact

Affects module maintainers.

- **New API:** Two optional parameters added to `ComponentSourceManager::generateComponents()`: `?string $source_id` and `?array $source_specific_ids`, enabling targeted Component config entity regeneration.

## Technical details

`ComponentSourceManager::generateComponents()` previously took no arguments and iterated over all registered `ComponentSource` plugin definitions. It now accepts two optional parameters: `?string $source_id` to restrict processing to one plugin and `?array $source_specific_ids` (a list of source-specific component IDs) to restrict processing to particular components within that plugin. Internally, `generateComponentsForSource()` received the same `$source_specific_ids` parameter and uses `array_intersect()` to filter the discovered component IDs before eligibility checking and entity creation or update.

The concrete call site that motivated this change is `JavascriptComponentStorage::doPostSave()`, which previously called `generateComponents()` with no arguments, triggering a full regeneration. It now calls `generateComponents(JsComponent::SOURCE_PLUGIN_ID, [$entity->id()])`, scoping the work to the single JavaScript component that was just saved.

A `@phpstan-import-type ComponentSourceSpecificId` annotation was added to `ComponentSourceManager` from `ComponentCandidatesDiscoveryInterface`, and a `\assert()` guard enforces that the `$source_specific_ids` array is a list (not an associative array). A `@phpstan-ignore-next-line` pragma was added to an existing `assert` inside the update loop where PHPStan had narrowed the type too aggressively. Tests were updated to use the targeted API, including a call to `generateComponents('sdc', ['canvas_test_sdc:image'])` in `DefaultContentExportImportTest` to mirror what would happen in a real request.

## Upgrade

If you call `generateComponents()` without arguments (the old form), it still works unchanged. To regenerate only one source or one component, use the new optional parameters:

```php
// Regenerate all components for one source plugin:
$componentSourceManager->generateComponents('my_source_plugin_id');

// Regenerate specific components within one source plugin:
$componentSourceManager->generateComponents('my_source_plugin_id', ['component_id_a', 'component_id_b']);
```
