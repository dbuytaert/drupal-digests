## Summary

When building Canvas content templates, the suggestion list that appears while linking a component prop to an entity field now omits fields that are considered unlikely to be useful for display. For Node entities, "Promoted to front page" (`promote`) and "Sticky at top of lists" (`sticky`) are no longer offered. For referenced File entities, the "User ID" (`uid`) owner field is hidden. For all content entity types, "Default translation" (`default_langcode`), "Default revision" (`revision_default`), and "Revision log message" (`revision_log_message`) are removed from suggestions. The result is a cleaner, less noisy suggestion UI when setting up content templates.

## Impact

No upgrade or configuration changes required.

## Technical details

The filtering lives in `PropSourceSuggester` (`src/ShapeMatcher/PropSourceSuggester.php`). A new private method `isConsideredIrrelevant()` accepts a `FieldPropExpression`, `ReferenceFieldPropExpression`, or `FieldObjectPropsExpression` and returns `true` when the expression targets a field deemed unfit for display.

Entity-type-specific heuristics check whether the host entity type is `node` (excluding `promote` and `sticky`) or whether a referenced entity type is `file` (excluding `uid` on that reference chain). Generic heuristics avoid hardcoding field names: they call `ContentEntityTypeInterface::getKey('default_langcode')` and `ContentEntityTypeInterface::getRevisionMetadataKey()` to resolve the actual field names for `default_langcode`, `revision_default`, and `revision_log_message` per entity type. `EntityTypeManagerInterface` is injected into the service to support these lookups. The method recurses into `ReferenceFieldPropExpression::$referenced` and each sub-expression in `FieldObjectPropsExpression::$objectPropsToFieldProps`, so an irrelevant field nested anywhere in a reference chain is also excluded.

A `@todo` in the code marks the hardcoded approach for future replacement once Drupal core issue [#3557353](https://www.drupal.org/node/3557353) provides a way to annotate field definitions with irrelevance metadata directly on Typed Data definitions.
