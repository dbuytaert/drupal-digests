## Technical details

This issue added Playwright end-to-end tests for the Content Templates feature, focusing on field linking. The test suite verifies linking component props to entity fields across multiple data types (boolean, string, email, URI, integer, HTML, image objects) and field widget configurations. 

A test recipe provides field setup on the Page content type along with sample content. The main test file systematically verifies each prop type in the `all-props` component can be linked to appropriate fields, checking both the UI state and rendered preview output. The tests validate hierarchical field paths (e.g., "Image Media > Media > Image > Height") and ensure suggestion menus work correctly.

Performance improvements were made to `PropSourceSuggester` to avoid repeatedly fetching suggestions in loops. The `LinkedFieldBox` component now displays hierarchical paths with arrows (→) in tooltips and only the final segment as the visible title. The `PropLinker` component was enhanced with test-friendly data attributes to enable automated testing of nested suggestion menus. The Playwright configuration increased the global timeout to one hour to accommodate the comprehensive test suite.

## Lessons learned

- When end-to-end tests call an API repeatedly in a loop, fetch shared data once before the loop rather than on each iteration. Tests that stress performance edge cases (like components with many props) will expose inefficiencies that would otherwise remain hidden in typical use cases. *(via bnjmnm)*
- When writing tests for systems with extensible options (like plugin systems or configurable widgets), design tests to fail when new options are added but not covered by tests. This prevents silent test decay as the system evolves. *(via bnjmnm)*
- When planning major refactors to UI code, ensure end-to-end test coverage is in place first. Without automated tests for areas not covered by other test layers, changes that break large parts of the UI may go undetected. *(via jessebaker)*
