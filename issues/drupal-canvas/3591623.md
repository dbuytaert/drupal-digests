## Summary

The `entity_blueprint` module's workspace-enforcement settings have been completely removed. The `require_workspace` option (which defaulted to `true`, blocking entity saves outside an active workspace) and its admin settings page at `/admin/config/content/entity-blueprint` no longer exist. Sites that had explicitly set this option to `false` to permit saves in the live environment will lose that configuration on the next config import. The change was made in response to the Canvas autosave issue where enforcing workspace context caused repeated false 409 conflict errors on non-English monolingual sites.

## Impact

Affects site owners.

- **Config change:** `entity_blueprint.settings` config object removed; the `require_workspace` key no longer exists and any stored value will be orphaned.

## Technical details

The commit deletes `config/install/entity_blueprint.settings.yml` (holding the `require_workspace: true` default), `config/schema/entity_blueprint.schema.yml` (the schema for that config object), the route definition (`entity_blueprint.settings`), the menu link, `src/Form/SettingsForm.php`, `src/Form/SettingsFormTrait.php`, and `src/Hook/SettingsFormAlter.php`. The `SettingsFormAlter` hook was wiring entity_blueprint settings into a `navigation_plus` settings form via `hook_form_navigation_plus_settings_alter`; that integration is also gone. After this change there is no `entity_blueprint.settings` config object and no means to toggle workspace enforcement through entity_blueprint's own UI or through navigation_plus.

## Upgrade

If your site had `entity_blueprint.settings` in configuration management (e.g. `config/sync/entity_blueprint.settings.yml`), remove that file before running `drush config:import`, otherwise the import will error on an unknown config object. No code substitution is needed; the workspace enforcement feature is simply removed.
