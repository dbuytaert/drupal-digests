## Summary

Canvas code components rely on JavaScript import maps to resolve module specifiers. Previously, the import map logic was duplicated between the backend and the code editor preview UI, creating maintenance burden and inconsistency risk. This change unifies import map generation on the backend, passing it to the UI via drupalSettings. A new hook allows modules and themes to add or replace import map entries, enabling custom JavaScript libraries and Tailwind CSS plugins to be made available to code components.

## Impact

Affects module maintainers.

- **New API:** New `hook_canvas_importmap_alter()` allows modules and themes to add or modify JavaScript import map entries for Canvas code components

## Technical details

The `GlobalImports` service now builds the complete import map structure including both global imports and scoped imports, returning an array with `imports` and `scopes` keys matching the import map specification. The service invokes `hook_canvas_importmap_alter()` to allow modules and themes to modify the map before cache-busting query strings are appended to global imports.

The code editor preview previously duplicated the import map in the TypeScript `Preview` component. It now reads existing import map tags from the main document, merges in preview-specific additions (the auto-save endpoint for component dependencies), and injects them into the iframe. This eliminates the duplication and ensures the preview uses the same import map as the backend.

The `JsComponent` plugin and `CanvasController` now call `getImportMap()` instead of the former `getGlobalImports()` method. Scoped imports (component-specific dependencies) are merged with the base map structure.

A test module demonstrates the hook by adding a new package, overriding an existing one, and adding scoped imports. The Astro bundler configuration was updated to handle Preact subpath imports correctly, ensuring bare specifiers are preserved in the output so they resolve through the import map with proper cache-busting.

## Upgrade

Implement `hook_canvas_importmap_alter()` to add custom JavaScript libraries or Tailwind CSS plugins:

```php
function my_module_canvas_importmap_alter(array &$import_maps): void {
  $module_path = \Drupal::service('extension.path.resolver')
    ->getPath('module', 'my_module');

  // Add a new package.
  $import_maps['imports']['my-library'] = \base_path() . $module_path . '/js/my-library.js';

  // Replace an existing package.
  $import_maps['imports']['clsx'] = \base_path() . $module_path . '/js/custom-clsx.js';

  // Add scoped imports.
  $import_maps['scopes']['/my-scope/'] = [
    'preact' => \base_path() . $module_path . '/js/custom-preact.js',
  ];
}
```

The hook runs for both modules and themes. Cache-busting query strings are appended automatically after the hook fires. See the new documentation at `/apis/import-maps` for Tailwind CSS plugin examples.
