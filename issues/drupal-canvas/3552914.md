## Summary

Canvas code components rely on a JavaScript import map to resolve module specifiers such as `preact`, `clsx`, and `swr`. Previously the import map was defined independently in two places: the PHP backend and the TypeScript code editor preview component, creating a maintenance burden and a risk of drift. This change moves the authoritative definition entirely to the backend and introduces `hook_canvas_importmap_alter()` so modules and themes can add, override, or remove entries. The code editor preview now reads `<script type="importmap">` tags already present in the main document instead of maintaining its own copy.

## Impact

Affects module maintainers.

- **New API:** New `hook_canvas_importmap_alter()` lets modules and themes add, replace, or remove JavaScript import map entries used by Canvas code components, including Tailwind CSS plugins

## Technical details

The `GlobalImports` service gained a new `getImportMap()` method (replacing `getGlobalImports()`) that returns a full import-map-spec array with `imports` and `scopes` keys. After building the base map it invokes `hook_canvas_importmap_alter()` via both `ModuleHandlerInterface::alter()` and `ThemeManagerInterface::alter()`, so both modules and themes can participate. Cache-busting query strings are appended to global import entries only after the hook fires, so implementors do not need to add them manually.

`JsComponent` and `CanvasController` were updated to call `getImportMap()`. The `SCOPED_IMPORTS` key is now always present in the returned structure (empty array by default), and per-component scoped dependencies are merged into it with `array_merge` rather than conditionally assigned.

The TypeScript `Preview` component dropped its hardcoded import map object. It now queries all `script[type="importmap"]` elements from the main document, injects the preview-specific `@/components/` entry pointing at the auto-save endpoint, and writes the merged maps into the iframe's `srcDoc`. This makes the preview automatically inherit any entries contributed by `hook_canvas_importmap_alter()`.

The Astro bundler configuration was updated so that Preact subpath imports (`preact`, `preact/hooks`, `preact/compat`) emitted by cross-subpath references within the `preact` or `@astrojs/preact` packages use bare specifiers in the output bundle. Previously Rollup emitted relative paths for these, which bypassed the import map and its cache-busting query strings, causing the browser to load two separate Preact instances.

A test module (`canvas_test_importmap_alter`) and a `GlobalImportsTest` kernel test class cover adding a new global entry, overriding an existing one, and adding scoped imports.

`GlobalImports` carries `@internal`, so the method rename is not a public API change. The hook itself is the public surface. A documentation page was added at `/apis/import-maps` with usage examples including Tailwind CSS plugin registration. The hook notes an intention to deprecate it in favour of Drupal core's `hook_importmap_alter()` when [#3398525](https://www.drupal.org/node/3398525) lands. The related issue [#3553356](https://www.drupal.org/node/3553356) introduced `@plugin` support in Global CSS and motivated the need for extensible import maps.

## Upgrade

Implement `hook_canvas_importmap_alter()` to register custom JavaScript packages or Tailwind CSS plugins for use in Canvas code components:

```php
function my_module_canvas_importmap_alter(array &$import_maps): void {
  $module_path = \Drupal::service('extension.path.resolver')
    ->getPath('module', 'my_module');

  // Add a new globally available package.
  $import_maps['imports']['my-library'] = \base_path() . $module_path . '/js/my-library.js';

  // Replace a built-in package with a custom build.
  $import_maps['imports']['clsx'] = \base_path() . $module_path . '/js/custom-clsx.js';

  // Add scoped imports (cache-busting is NOT applied to scopes).
  $import_maps['scopes']['/my-scope/'] = [
    'preact' => \base_path() . $module_path . '/js/custom-preact.js',
  ];
}
```

To register a Tailwind CSS plugin for use with `@plugin` in Global CSS:

```php
function my_theme_canvas_importmap_alter(array &$import_maps): void {
  $theme_path = \Drupal::service('extension.path.resolver')
    ->getPath('theme', 'my_theme');
  $import_maps['imports']['@tailwindcss/forms'] = \base_path() . $theme_path . '/js/tailwindcss-forms.js';
}
```

Cache-busting query strings are appended automatically to all `imports` entries after the hook fires; do not append them manually. The hook fires for both modules and themes. See `/apis/import-maps` for full documentation.
