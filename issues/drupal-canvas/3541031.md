## Summary

The Canvas editor can now open and edit content templates, not just individual pages. When a user clicks a template view-mode entry in the Templates sidebar menu, the editor navigates to a new route (`/canvas/template/:entityType/:bundle/:viewMode/:previewEntityId`) and renders the template layout in the same editor frame used for page editing. Component operations (add, reorder, delete) work the same way they do for pages. The active sidebar panel is now persisted to `localStorage` as a temporary workaround for the full-page reload that occurs when switching between templates.

## Impact

Affects module maintainers.

- **New API:** Three new RTK Query hooks exported from `componentAndLayoutApi`: `useGetTemplateLayoutQuery`, `usePostTemplateLayoutMutation`, `useUpdateComponentInTemplateMutation`; `useGetLayoutByIdQuery` renamed to `useGetPageLayoutQuery`.

## Technical details

A new React route pattern `/template/:entityType/:bundle/:viewMode/:previewEntityId` is registered in `AppRoutes.tsx`, reusing the existing `SideMenu` and `Editor` shell. Inside `Editor`, the current `pathname` determines whether to mount `Layout` (for pages) or the new `TemplateLayout` component (for templates). `TemplateLayout` mirrors `Layout` but calls `useGetTemplateLayoutQuery`, which targets the endpoint `canvas/api/v0/layout-content-template/{entity_type}.{template_bundle}.{template_view_mode}/{entity_id}`.

Three new RTK Query endpoints are added to `componentAndLayoutApi`: `getTemplateLayout` (GET, loads initial layout and dispatches `setInitialPageData` and `setHtml`), `postTemplateLayout` (POST mutation, re-renders the preview after in-place edits), and `updateComponentInTemplate` (PATCH mutation, performs individual component updates and refreshes the layout model). The existing `getLayoutById` query is renamed `getPageLayout` throughout the codebase. The `baseQuery` layer is extended to parse `templateBundle` and `templateViewMode` from the current URL and substitute `{template_bundle}` and `{template_view_mode}` placeholders in API URLs.

The `Preview` component detects whether the path contains `/template/` and routes preview POST requests to `usePostTemplateLayoutMutation` instead of the page preview mutation. `useComponentSelection` and `useEditorNavigation` are refactored to compute navigation paths from `location.pathname` rather than from hardcoded `/editor/:entityType/:entityId` base URLs, using a new shared `route-utils.ts` module (`removeComponentFromPathname`, `removeRegionFromPathname`, `setComponentInPathname`, `setRegionInPathname`). `TemplateListItem` gains an `href` prop and a `selected` state driven by the new `useGetEditedTemplateId` hook, which derives `entityType.bundle.viewMode` from route params. `primaryPanelSlice` persists `activePanel` to `localStorage` under key `canvas_panel_state` so the Templates menu stays open across the full-page navigations that currently occur when switching templates.
