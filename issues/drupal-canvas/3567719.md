## Summary

The Canvas module's `src/PropExpressions/StructuredData` subsystem gained a formal interface hierarchy that replaces the flat, organically-grown class structure in place since before the module's initial 1.0.0 release. Previously the six prop expression classes had no common interfaces to express what kind of starting point or evaluation result they represented, making it difficult to write generic code against them. The refactor introduces clear abstractions so that callers can reason about expression behavior without depending on specific concrete classes. Some public methods on those concrete classes were renamed or removed in the process.

## Impact

Affects module maintainers.

- **API change:** Public methods `getLeaf()` and `isMultiBundle()` removed from `ReferenceFieldPropExpression`; `isMultiBundle()` removed from `FieldPropExpression` and `FieldObjectPropsExpression`; `StorablePropShape::getFieldType()` removed. Replacements: `ReferencePropExpressionInterface::getFinalTargetExpression()`, `ReferencePropExpressionInterface::targetsMultipleBundles()`, `$storable_prop_shape->fieldTypeProp->getFieldType()`.

## Technical details

Five `@internal` interfaces and one `@internal` trait were added to `src/PropExpressions/StructuredData/`:

- `EntityFieldBasedPropExpressionInterface` (used by `DynamicPropSource`s): exposes `getHostEntityDataDefinition()`, `getFieldName()`, `getDelta()`, and `hasSameStartingPointAs()`. Implemented by `FieldPropExpression`, `FieldObjectPropsExpression`, and `ReferenceFieldPropExpression`.
- `FieldTypeBasedPropExpressionInterface` (used by `StaticPropSource`s): exposes `getFieldType()`. Implemented by `FieldTypePropExpression`, `FieldTypeObjectPropsExpression`, and `ReferenceFieldTypePropExpression`.
- `ScalarPropExpressionInterface`: carries the PHPStan type aliases for scalar evaluation results (previously declared inline in `EvaluationResult`); exposes `getFieldPropertyName()`. Implemented by `FieldPropExpression` and `FieldTypePropExpression`.
- `ObjectPropExpressionInterface`: carries PHPStan type aliases for object evaluation results; exposes `getObjectExpressions()`, which replaces direct reads of the `objectPropsToFieldProps` / `objectPropsToFieldTypeProps` public properties. Implemented by `FieldObjectPropsExpression` and `FieldTypeObjectPropsExpression`.
- `ReferencePropExpressionInterface`: carries PHPStan type aliases for entity-reference results; exposes `getTargetExpression()`, `getFinalTargetExpression()` (replacing `getLeaf()`), `targetsMultipleBundles()` (replacing `isMultiBundle()` on reference expressions), and `getFieldPropertyName()`. Implemented by `ReferenceFieldPropExpression` and `ReferenceFieldTypePropExpression`.

`EntityFieldBasedExpressionTrait` implements `hasSameStartingPointAs()` using the new interface accessors instead of direct property comparison, which was error-prone for `ReferenceFieldPropExpression` where the starting entity data lives on the inner `$referencer` object.

`Labeler::label()` now accepts `EntityFieldBasedPropExpressionInterface` instead of the explicit three-class union, and internally dispatches on `ReferencePropExpressionInterface` rather than `instanceof ReferenceFieldPropExpression`. `Evaluator` similarly switches from `match ($expr::class)` with direct property access to interface method calls. `ShapeMatchingHooks::getFieldTypeProps()` (protected) now throws `\InvalidArgumentException` for unsupported media source classes instead of silently returning an empty array. This work was described as a prerequisite for [#3550750](https://www.drupal.org/node/3550750), which relies on `getFieldName()` and `targetsMultipleBundles()` to clean up multi-bundle reference handling.

## Upgrade

Canvas extension code that calls methods on the concrete prop expression classes needs to update:

```php
// getLeaf() renamed to getFinalTargetExpression() on reference expressions
// Before:
$leaf = $referenceExpr->getLeaf();
// After:
$leaf = $referenceExpr->getFinalTargetExpression();

// isMultiBundle() removed; use targetsMultipleBundles() on reference expressions only
// Before:
$multi = $referenceExpr->isMultiBundle();
// After:
$multi = $referenceExpr->targetsMultipleBundles();

// Direct ->referenced property access replaced by a method
// Before:
$target = $referenceExpr->referenced;
// After:
$target = $referenceExpr->getTargetExpression();

// StorablePropShape::getFieldType() removed
// Before:
$fieldType = $storable_prop_shape->getFieldType();
// After:
$fieldType = $storable_prop_shape->fieldTypeProp->getFieldType();
```
