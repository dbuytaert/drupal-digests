## Summary

In Drupal Canvas (dev mode), creating a new folder in the code component library no longer opens a separate popup dialog. Clicking "Add folder" from the New dropdown now renders an inline editable text field directly in the library list, pre-filled with "New folder" and auto-focused. The folder is saved by pressing Enter or clicking away, cancelled with Escape, and newly created folders appear at the top of the list rather than alphabetically at the bottom.

## Impact

No upgrade or configuration changes required.

## Technical details

The `LibraryToolbar` component was refactored to remove the `Dialog` component and Radix UI `Form.Root` in favor of an inline block rendered below the toolbar row when `isCreatingFolder` state is `true`. A new `onFolderCreating` prop callback notifies the parent when the inline creation state changes. Auto-focus and text selection on mount are handled via a `useEffect` + `setTimeout` + `textFieldRef` ref, because `autoFocus` alone fires before the dropdown close animation releases focus. An `isSubmittingRef` guard prevents double-submission. `handleCreateFolder` cancels silently if the name is untouched (still "New folder") or empty, submits on Enter or blur, and `cancelFolderCreation` restores state on Escape. API errors are now displayed inline beneath the input rather than in a dialog error state. The `onCloseAutoFocus` handler on `DropdownMenu.Content` is suppressed when a folder is being created so the inline input can receive focus instead of the trigger button. In `componentAndLayoutApi`, an `onQueryStarted` hook added to `createFolder` reads the cached folder list from Redux state (`getFolders(undefined)` cache key) and sets `body.weight` to `minWeight - 1`, placing new folders at the top. `validateFolderNameClientSide` was updated to trim the input before checking for leading/trailing hyphens or underscores, relaxing the whitespace constraint during typing, and its error message now says "hyphen or underscore" rather than "hyphen, underscore, or whitespace". The Playwright folder-management tests were updated throughout to follow the new inline flow and the revised expected folder sort order.

### Collaboration

During manual testing, tedbow raised two UX concerns: that focus was not automatically placed on the inline input after clicking "Add folder", and that the folder was not pre-created with a default name (as macOS Finder does). utkarsh_33 addressed the auto-focus concern in a follow-up commit and argued that pre-creating a folder with a default name would cause name-collision errors on repeated use. jessebaker agreed the current approach was the right tradeoff, and also fixed edge cases where API error messages stuck around after the user changed the name or cancelled folder creation.
