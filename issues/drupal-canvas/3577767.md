## Summary

The linker feature was not working for date-only fields in component templates. When site builders tried to connect a date field to dynamic data, the linker button would not appear. This prevented date fields from being linked to content properties like "Authored on" or "Changed" dates. The fix ensures that date fields show the linker button just like other field types, allowing site builders to make date fields dynamic in their component templates.

## Impact

No upgrade or configuration changes required.

## Technical details

The linker appears for most field widgets automatically, but some widgets use custom wrapper templates instead of the standard form element label template. Previously, only fieldsets received the prop link data needed to render the linker. Date-only fields use `datetime_wrapper` as their theme wrapper, which was not included in this logic.

The fix adds `datetime_wrapper` to the list of wrappers that receive prop link data in `GeneratedFieldExplicitInputUxComponentSourceBase`. The corresponding theme preprocess function and Twig template were updated to render the linker using the same `canvas-drupal-label` component already used by fieldsets.

This mirrors the approach from [#3550848](https://www.drupal.org/node/3550848) which fixed linker rendering for date-time fields. The test suite was expanded to verify that date-only fields can be linked to various date properties and that the linked values display correctly in component previews.
