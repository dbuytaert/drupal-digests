## Summary

Canvas editors can now link data sources to date-only fields in components. Previously, the linking interface (linker) failed to appear for fields defined as `type: string, format: date` in component schemas. This prevented editors from connecting dynamic data to date fields, forcing them to use static values. The fix ensures that date fields work the same way as other linkable fields, restoring the ability to pull dates from content like publication times or custom date fields.

## Impact

No upgrade or configuration changes required.

## Technical details

The linker attaches to form elements through `#label_attributes['prop_link_data']` or `#prop_link_data` depending on how the widget renders its label. Fieldsets already had support for `#prop_link_data` because they render custom label markup instead of using the standard form element label template.

Date-only fields use `DateTimeDefaultWidget` with a `datetime_wrapper` theme wrapper that also renders custom label markup. The fix adds `datetime_wrapper` to the list of wrappers that receive `#prop_link_data`, matching the existing fieldset logic.

The `canvas_stark` theme preprocessor for `datetime_wrapper` now exposes this data to the template, and the template was updated to render the `canvas-drupal-label` web component with `direct_linker_data` attribute, matching the fieldset implementation.

Test coverage was expanded to verify that date format suggestions appear in the linker and successfully link to date fields. The test only links one suggestion per prop (via `suggestionsToLinkByProp`) to reduce test runtime while ensuring the mechanism works. Prop-specific value overrides (using `::` syntax in `entityValuesNode4`) handle cases where the same suggestion renders differently across different date formats.
