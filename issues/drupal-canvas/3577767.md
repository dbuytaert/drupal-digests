## Summary

Canvas provides a linker feature that allows site builders to connect component properties to dynamic data sources from Drupal entities. This feature was not working for date-only fields (as opposed to date-time fields). The linker button would not appear when configuring these fields, preventing builders from connecting date properties to entity data. This fix restores the linker functionality for date-only fields by updating how the Canvas theme handles datetime wrapper elements.

## Impact

No upgrade or configuration changes required.

## Technical details

The linker relies on prop link data being passed through form element render arrays to special label components in the Canvas theme. For most field types, this data is attached to the `#label_attributes` property. However, some field widgets use custom theme wrappers that render their own label UI instead of using the standard form element label template.

The datetime widget for date-only fields uses a `datetime_wrapper` theme wrapper that renders its own title element. Previously, prop link data was only passed through to `fieldset` wrappers. The fix adds `datetime_wrapper` to the list of wrappers that receive the `#prop_link_data` element property.

The Canvas Stark theme's `datetime-wrapper.html.twig` template was updated to render the `canvas-drupal-label` component (which includes the linker button) instead of a plain `<h4>` tag, matching the pattern already used for fieldsets.

The test suite was expanded to verify linker functionality for date-only fields and added prop-specific value overrides to handle cases where the same entity field renders differently depending on the target prop's format requirements.
