## Summary

The `webdev` module's `.info.yml` no longer declares any module `dependencies:`. Previously it listed all 12 bundled contributed modules (ctools, token, diff, pathauto, metatag, field_group, smart_trim, entityqueue, inline_entity_form, better_exposed_filters, link_attributes, token_filter) as hard dependencies, which Drupal enforced at install time. Those constraints are now gone, so enabling webdev alone does not guarantee those modules are present. Sites relying on webdev to pull in or enforce those contrib modules must now manage them independently. The same commit also adds a complete GitLab CI pipeline with BDD functional acceptance tests (webship-js 2.x + Playwright Chromium) covering login, module presence, admin pages, content authoring, access control, and frontend meta tag checks.

## Impact

Affects module maintainers.

- **API change:** All 12 module `dependencies:` removed from `webdev.info.yml`; Drupal no longer enforces that pathauto, metatag, diff, entityqueue, and 8 other contrib modules are installed when webdev is enabled.

## Technical details

The `dependencies:` block was removed in full from `webdev.info.yml`. In Drupal 11, declaring a module in `dependencies:` causes the module system to refuse to enable the declaring module unless all listed modules are already present. Stripping that block decouples the webdev module from any enforcement of its companion contrib modules.

The CI pipeline is defined in `.gitlab-ci.yml` and extends the drupalci shared templates (`include.drupalci.main.yml`, `.variables.yml`, `.workflows.yml`). A custom `.drupal-node-base` job template chains the drupalci `composer` artifact with a `drush site:install` + `drush en webdev`, then installs Node 20, Yarn 4 (corepack), and the `webship-js ~2.0` package along with Playwright Chromium. The test job `webship-js-test` runs `cucumber-js --config cucumber.js --tags "not @wip"` with JUnit output and on-failure screenshots/videos as artifacts.

Feature coverage in `tests/features/` spans role provisioning (`01-01-01-users-login.feature`), module presence on `/admin/modules`, admin pages for pathauto, metatag, diff, entityqueue, token/token_filter, field_group, and better_exposed_filters, content authoring with Pathauto alias generation, anonymous access-denial checks, and front-end canonical/generator meta tag presence. Custom step definitions in `tests/step-definitions/webdev.steps.js` add steps for named-user login (keyed off `cucumber.js` `worldParameters.users`), bulk test-user provisioning via `drush`, field/button/meta/link-rel assertions, and HTTP status checks. Selector maps for Claro (`cms-drupal-core-claro.json`) and Gin (`cms-drupal-cms-gin.json`) are registered in `cucumber.js`.

## Upgrade

If you use the `webdev` module and relied on Drupal's module dependency enforcement to ensure companion modules are installed, you must now manage those modules explicitly. Add them directly to your `composer.json` and enable them in your deployment process or recipe:

  "drupal/ctools": "^4",
  "drupal/token": "^1",
  "drupal/diff": "^1",
