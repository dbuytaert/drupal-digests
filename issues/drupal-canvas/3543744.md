## Summary

The Canvas AI submodule now provides a dedicated `AiResponseValidator` service for validating AI-generated component structures before they are placed on the canvas. Previously this validation logic lived directly inside the `SetAIGeneratedComponentStructure` tool plugin, making it unavailable to other tools. The extraction means any AI tool that receives component structures from an AI model can reuse the same validation without reimplementing it.

## Impact

Affects module maintainers.

- **New API:** New `canvas_ai.response_validator` service (`AiResponseValidator`) exposes `validateComponentStructure()` for reuse across Canvas AI tool plugins.

## Technical details

The three private methods on `SetAIGeneratedComponentStructure` (`validateComponentStructure`, `convertToComponentTreeData`, `buildPathTranslationMap`) and the two traits it used (`ComponentTreeItemListInstantiatorTrait`, `ConstraintPropertyPathTranslatorTrait`) were moved into a new `Drupal\canvas_ai\AiResponseValidator` class. The class is registered as the `canvas_ai.response_validator` service, injected with `BasicRecursiveValidatorFactory` and the `uuid` service. The plugin itself, which is `final`, now receives `AiResponseValidator` via its `create()` factory instead of the two lower-level services it previously held.

The public entry point is `AiResponseValidator::validateComponentStructure(array $componentGroups): void`. It converts the YAML-derived component group array into the internal `ComponentTreeItem` flat list format, validates it via `ComponentTreeItemListInstantiatorTrait::createDanglingComponentTreeItemList()`, and throws a `ConstraintViolationException` with human-readable paths if violations are found. The path translation logic maps flat numeric indices back to the original YAML paths (e.g. `components.0.[my_component].props`) so error messages are meaningful to callers.

## Upgrade

Developers writing custom Canvas AI tool plugins who previously duplicated validation logic can now inject and call the shared service:

```php
// In your plugin's create() factory:
$instance->responseValidator = $container->get('canvas_ai.response_validator');

// In your execute() method:
$this->responseValidator->validateComponentStructure($componentGroups);
```
