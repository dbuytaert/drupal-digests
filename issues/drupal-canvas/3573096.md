## Summary

Canvas now provides a redesigned user interface for editing multi-value text fields. Instead of showing all input boxes at once, the new design displays a compact list where each value appears as a collapsed item. Clicking an item opens a popover dialog for editing. Users can reorder items with drag handles, remove items with trash icons, and add new items with a "+ Add new" button. The interface respects field cardinality limits and required field constraints. This new UI is currently behind a feature flag requiring the `canvas_dev_mode` module to be enabled.

## Impact

No upgrade or configuration changes required.

## Technical details

This feature introduces a React-based multivalue form widget that replaces the default Drupal field widget display when `canvas_dev_mode` is enabled. The implementation consists of several layers:

The backend preprocessing hooks (`canvasStarkPreprocessFieldMultipleValueForm` and `fieldWidgetCompleteFormAlter`) detect multivalue fields and inject custom CSS classes and attributes. A custom Twig template (`field-multiple-value-form--string-textfield.html.twig`) wraps the field table in a `.multivalue-container` div and customizes the "+ Add new" button label.

The frontend `DrupalInputMultivalueForm` component renders each text field as a collapsed list item that triggers a Radix UI Popover when clicked. The popover contains the actual input field, a close button, and a remove button. Changes are committed when the user presses Enter. The component maintains a hidden copy of the actual Drupal form input for form submission and AJAX compatibility.

A custom `tabledrag.theme.js` overrides Drupal's drag handle rendering to use an SVG icon. CSS hides the original Drupal remove buttons and the "You have unsaved changes" message. The remove button in the popover triggers clicks on the hidden Drupal buttons to maintain AJAX functionality. Cypress tests verify the complete editing workflow including popover interactions, drag-and-drop reordering, and item removal.
