## Summary

Multi-value text fields in Canvas now display with a more compact, modern interface when the development module is enabled. Instead of showing all input boxes at once, the field shows a list of existing values. Clicking any value opens a popover where you can edit the text, and you can reorder items by dragging them. An "Add new" button appears at the bottom when you can add more values. This makes forms with many values easier to scan and work with, especially on smaller screens or when managing long lists of items.

## Impact

No upgrade or configuration changes required.

## Technical details

This feature adds a popover-based editing interface for multi-value text fields, available behind the `canvas_dev_mode` flag. The new UI replaces the traditional stacked textfield approach with a collapsed list view where each value appears as a clickable row with a drag handle.

The implementation uses a new `DrupalInputMultivalueForm` React component that renders list items in a compact format. Clicking a row opens a Radix UI popover containing the actual input field. The component maintains a hidden real input for form submission while presenting a visual duplicate in the popover for editing. Changes commit when the user presses Enter, and closing the popover without Enter reverts any uncommitted edits.

The backend marks multi-value form elements via `hook_field_widget_complete_form_alter()` and routes them to a custom Twig template. A new `input--multivalue-form.html.twig` template suggestion connects these elements to the React component. CSS overrides hide default Drupal tabledrag weight controls while preserving AJAX functionality. The remove button in the popover triggers the hidden Drupal remove button to ensure proper AJAX updates.

The feature includes custom drag handle SVG icons via `Drupal.theme.tableDragHandle()` override and comprehensive Cypress tests covering editing, adding, removing, and reordering items. The design respects cardinality limits and shows/hides the Add button accordingly.
