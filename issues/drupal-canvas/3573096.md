## Summary

Multi-value text fields in the Canvas editor now display as a more compact and manageable list interface when the development mode module is enabled. Each field value appears as a single row with a drag handle for reordering. Clicking a row opens a popover dialog where you can edit the text or remove the item. An "Add new" button appears below the list when the field can accept more values. This new interface makes it easier to work with fields that allow multiple entries, such as tags or keywords, by reducing visual clutter and providing clear controls for common actions.

## Impact

No upgrade or configuration changes required.

## Technical details

This feature introduces a new React component `DrupalInputMultivalueForm` that renders when `canvas_dev_mode` is enabled and a text field is configured for multiple values. The component displays a collapsed list view with custom drag handles (styled using a Radix UI icon) and opens a Radix UI `Popover` for editing individual values. The popover includes the field label, an input field, and a remove button.

The implementation uses a "hidden real input" pattern: the actual Drupal form input remains in the DOM but is positioned off-screen for accessibility and form functionality. The visible UI is a controlled React component that programmatically updates the hidden input using native value setters and dispatches both `input` and `change` events to ensure React and Drupal AJAX handlers are notified.

Changes to `ReduxIntegratedFieldWidgetsHooks.php` add preprocessing logic to mark multivalue form elements with `#is_multivalue_form` and attach the new `multivalue-form` library. A new Twig template suggestion `input--multivalue-form.html.twig` routes these inputs to the React component. The `tabledrag.theme.js` file overrides Drupal's default drag handle theme to use custom SVG icons matching the design system.

CSS in `multivalue-form.css` hides default Drupal elements like weight columns and "unsaved changes" warnings, applies borders only to the table body, and styles the "Add new" button. The implementation preserves all existing Drupal AJAX behaviors by triggering the hidden remove buttons rather than reimplementing removal logic.
