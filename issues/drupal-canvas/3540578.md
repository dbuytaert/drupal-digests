## Summary

Drupal Canvas (formerly Experience Builder) now lets site builders organize components into folders using drag and drop in the Library panel. While the Library panel is open, the editor canvas dims and blurs to signal that drag operations target folders rather than the page layout. The work from this MR formed the foundation for the larger refactor in [#3561018](https://www.drupal.org/node/3561018), where the feature ultimately landed.

## Impact

Affects module maintainers.

- **New API:** New `useUpdateFolderMutation` hook exported from `componentAndLayoutApi` for PATCH operations on `canvas/api/v0/config/folder/{id}`

## Technical details

A new `FolderDragContext` component wraps `LibraryItemList` in a `DndContext` from `@dnd-kit/core`. Its `dragEndHandler` checks whether the drop target carries `destination: 'folders'` in its data, then issues sequential `updateFolder` PATCH mutations: first removing the component from its prior folder (if any) by filtering it out of `priorFolder.items`, then appending it to the target folder's items array.

`SidebarFolder` gains a required `id` prop and registers itself as a droppable via `useDroppable` when `activePanel === 'manageLibrary'`. A new `.isOver` CSS rule applies a blue outline and background during hover. `ListItem` previously blocked all drag operations in the `manageLibrary` panel; that guard is removed, and the drag data's `origin` field is set to the active panel name rather than the hardcoded string `'library'`.

The new `updateFolder` RTK Query mutation sends `PATCH canvas/api/v0/config/folder/{id}` and invalidates both `Folders` and `Layout` cache tags. It is excluded from `withAutoSavesInjection` alongside `createFolder` and `createContentTemplate`. `EditorFrame` and `ContextualPanel` each accept a new `inactive` boolean prop; when `activePanel === 'manageLibrary'`, an absolutely-positioned overlay with `backdrop-filter: blur(10px)` and 30% black background covers the canvas. The sidebar panel labels are also renamed: the add-components panel changes from "Library" to "Add" and the folder-management panel from "Manage library" to "Library".
