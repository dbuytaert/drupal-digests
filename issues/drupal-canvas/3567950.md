## Summary

Canvas now requires all calls to global PHP functions like `assert()` and `sprintf()` to use a leading backslash (for example, `\assert()` and `\sprintf()`). This coding standard improves performance because modern PHP can apply compile-time optimizations only when global functions are fully qualified. The change applies to two commonly used functions initially, with follow-up issues created to expand the standard to all global functions. Drupal core is considering the same approach.

## Impact

No upgrade or configuration changes required.

## Technical details

This change enforces the Slevomat Coding Standard rule `FullyQualifiedGlobalFunctions` through PHPCS, starting with `\assert()` and `\sprintf()`. The coding standard rule is configured in `phpcs.xml` with an `include` property listing the two function names. All existing uses of `assert()` and `sprintf()` throughout the codebase were updated to their fully-qualified equivalents using simple find-and-replace operations.

The motivation comes from PHP's ability to optimize calls to global functions at compile-time when they are fully qualified with a leading backslash. Without the backslash, PHP must check the current namespace for a function definition before falling back to the global namespace, which adds runtime overhead. This optimization is only possible when the function call explicitly uses the global namespace via the `\` prefix.

The incremental approach starts with just two functions to avoid disrupting all open merge requests. Follow-up issues target additional high-usage functions like `\in_array()`, `\array_key_exists()`, `\array_map()`, `\array_keys()`, and type-checking functions. Eventually all global function calls will be covered. The pattern is inspired by Drupal core issue [#3346987](https://www.drupal.org/project/drupal/issues/3346987) and work in the phpstan-drupal project.

## Collaboration

The issue was opened by wim leers after noticing inconsistent application of the pattern across Canvas contributions. One contributor (larowlan) had been manually applying fully-qualified function calls in merge requests, but without automated enforcement the pattern was applied inconsistently and created review noise. The team adopted an incremental rollout strategy to minimize disruption: enforce the rule for just two functions initially, then create novice-friendly follow-up issues to expand coverage gradually. This approach balances code consistency with practical merge request management.
