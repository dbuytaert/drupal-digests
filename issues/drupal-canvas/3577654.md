## Summary

Canvas added a new computed property `inputs_resolved` to component instances. This property resolves stored inputs (prop sources, media entity IDs, block settings) into their final output values (image URLs, text strings, booleans). This makes resolved data available to API consumers like JSON:API and the Canvas Page API, enabling use cases like Canvas Workbench displaying component images without needing to resolve media entities client-side.

## Impact

Affects module maintainers.

- **New API:** `inputs_resolved` computed property on `ComponentTreeItem` exposes resolved component input values (e.g. image URLs instead of media entity IDs) to normalizers like JSON:API and REST. Also adds `ComponentSourceInterface::getResolvedExplicitInput()` method.

## Technical details

`ComponentTreeItem` gains an `inputs_resolved` computed property via the new `ResolvedComponentInputs` TypedData class. The property is marked non-internal so it's exposed to normalizers. When accessed, it calls the component source's new `getResolvedExplicitInput()` method, which delegates to `hydrateComponent()` to get resolved values. For component sources using prop sources (SDC, JS components), the implementation unwraps `EvaluationResult` objects to extract plain PHP values and collects cacheability metadata. Block components and other sources that return flat values have that data passed through directly. The property returns NULL when the component cannot be loaded or when a host entity is required but unavailable (e.g. content templates with entity field prop sources accessed via JSON:API). The computed value is invalidated in `ComponentTreeItem::onChange()` whenever `inputs`, `component_id`, or `component_version` changes. The implementation uses `ComputedDataTypeWithCacheabilityTrait` to cache the computed value and associated cacheability metadata, with an added `invalidateComputedValue()` method for explicit cache invalidation.
