## Summary

The CLI tool's `pull` and `push` commands can now transfer Canvas Pages in addition to components and global CSS. Pages are opt-in via the `--include-pages` flag or `CANVAS_INCLUDE_PAGES` environment variable. Pages are stored locally as JSON files containing a component tree. The CLI converts between the local format and Canvas's API format, including resolving component versions that may have changed after pushing components.

## Impact

Affects module maintainers and site owners.

- **Config change:** New `CANVAS_INCLUDE_PAGES` environment variable (accepts true/false, 1/0, yes/no)
- **Permission change:** New OAuth scopes `canvas:page:create`, `canvas:page:read`, and `canvas:page:edit` required when pages are enabled
- **New API:** New `--include-pages` option for `pull` and `push` commands to include Canvas pages

## Technical details

Canvas pages require `component_version` for each component in their tree, but the json-render library doesn't track versions and versions can change after pushing components. The CLI now fetches active component versions from `/canvas/api/v0/config/component` after pushing components and before pushing pages. This required adding the `Component` entity type (in addition to `JavaScriptComponent`) to the OAuth-protected config entities in `CanvasOauthAuthenticationProvider`.

Pages are stored locally in an "authored spec" format with a `slots` object for better editing ergonomics, rather than the API's flat array with `parent_uuid` and `slot` properties. The `pageToAuthoredSpec()` and `authoredSpecToComponentTree()` functions handle conversion between formats.

When pulling pages, the CLI matches local files by UUID and updates existing files. When pushing, pages without a UUID or with a UUID not found remotely are created as new pages. The server-assigned UUID is written back to the local file after creation. Pages containing non-JS components (like SDC components) are skipped during pull with a validation error.

The feature defaults to disabled to avoid requiring additional OAuth scopes for users who don't need page management.

## Upgrade

To enable page push/pull, configure your OAuth consumer to include the page scopes and use the `--include-pages` flag:

```bash
npx canvas pull --include-pages
npx canvas push --include-pages
```

Or set the environment variable:

```bash
CANVAS_INCLUDE_PAGES=true npx canvas pull
```

Update your OAuth consumer scopes in the Drupal admin UI to include `canvas:page:create`, `canvas:page:read`, and `canvas:page:edit`.
