## Summary

The Publishing UX interface now provides smoother navigation when working with changes and errors. When publishing fails due to validation errors, clicking the error link navigates to the problem component in the same tab without reloading the page. When discarding changes, the interface updates immediately without a full page refresh, keeping you in context while the system reverts to the published state. These improvements reduce interruptions and make the publishing workflow feel more responsive and modern.

## Impact

No upgrade or configuration changes required.

## Technical details

This issue implements two single-page application (SPA) navigation improvements in the Publishing UX canvas editor. First, error links in the publish review modal now use React Router `Link` components instead of opening new tabs. The error path construction was simplified to relative URLs since navigation stays within the SPA. The `target="_blank"` attribute was removed so clicking navigates in-place.

Second, discarding pending changes no longer triggers `window.location.reload()`. Instead, the discard mutation explicitly invalidates relevant Redux Toolkit Query cache tags for layout, content, and pending changes. For the current page, it clears form field values and resets the layout initialization state so the canvas and sidebar reflect refetched published data. For code components, it invalidates the component cache and sets a `forceRefresh` flag that bypasses the normal skip logic in `useGetCodeEditorData`, forcing the editor to refetch canonical data and re-initialize.

The changes preserve staged changes during error navigation and maintain proper state synchronization after discards by selectively clearing cached form values for both component instance and entity forms. End-to-end tests verify SPA navigation behavior and confirm the URL path remains unchanged after discards.
