## Summary

The Canvas module's library sidebar now supports renaming folders. Editors can trigger rename mode either by selecting "Rename" from a folder's three-dot contextual menu or by double-clicking the folder name. While in rename mode the folder name becomes an inline text input, pre-filled with the current name. Pressing Enter commits the change, pressing Escape cancels it, and clicking outside (blur) also cancels without saving. Duplicate names and empty names are rejected with an inline error message. The folder's open or closed state is preserved through the rename operation.

## Impact

Affects module maintainers.

- **New API:** New `nameSlot`, `errorSlot`, and `onNameDoubleClick` props on the `SidebarFolder` component

## Technical details

All rename logic lives in the `FolderList` component (`src/.../FolderList`). It uses `useUpdateFolderMutation` from `@/services/componentAndLayout` to PATCH the folder. State is managed with `useState` for `isRenaming`, `folderName`, and `validationError`, plus a `isSubmittingRef` ref to guard against double-submissions during the async mutation. A `useEffect` wired to `folder.name` keeps the local name in sync after a successful server round-trip. Client-side validation runs on every keystroke via `validateFolderNameClientSide` from `@/features/validation/validation`; API-level errors are parsed by `extractErrorMessageFromApiResponse` and shown in the same error slot.

The `SidebarFolder` component received three new optional props: `nameSlot` (a `ReactNode` that replaces the plain `Text` when set), `errorSlot` (rendered below the accordion row), and `onNameDoubleClick` (wired to the name area's `onDoubleClick`). When `nameSlot` is present the container alignment switches from `center` to `start` and overflow from `hidden` to `visible` to accommodate the `TextField.Root` input without clipping it. The three-dot menu button is hidden while rename mode is active (the parent passes `menuItems={undefined}` when `isRenaming` is true).

A CSS fix in the same commit corrects the `contextualAccordionVariant` style: the dots-button hover and selected-hover background variables were previously cleared to `none`, making the button invisible on hover. They now use `var(--blue-3)` and `var(--blue-5)` respectively, with the dot icon coloring to `var(--blue-9)` on hover.
