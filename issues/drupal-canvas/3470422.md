## Summary

When an SDC, block plugin, or JS code component disappears between requests (for example, after switching git branches during development, or after a Drush config delete), Canvas previously threw an unrecoverable fatal error. Pages would white-screen, the component list API would return HTTP 500, and the component instance form would crash. After this fix, Canvas degrades gracefully: the page renderer shows the existing "something went wrong" fallback, the `/canvas/api/v0/config/component` API marks the affected component with `broken: true`, and the component instance form displays the previously stored inputs with a message prompting the developer to fix or copy the component.

## Impact

Affects module maintainers.

- **API change:** `ComponentSourceInterface::renderComponent()` and `hydrateComponent()` each gained a new required `array $slot_definitions` parameter; new required `isBroken(): bool` method added to the interface.
- **New API:** `ComponentInterface::getSlotDefinitions()` added to read stored slot definitions from `fallback_metadata` without contacting the live component source.

## Technical details

A new `isBroken(): bool` method was added to `ComponentSourceInterface`. Each built-in source implements it differently: `SingleDirectoryComponent` catches `ComponentNotFoundException` from `getMetadata()`; `BlockComponent` checks whether `getBlockPlugin()` returns an instance of Drupal core's `Broken` block plugin; `JsComponent` checks whether the backing `JavaScriptComponent` config entity exists via storage. The `Fallback`, `Personalization`, and `InvalidSlots` plugins always return `false`.

The signatures of `renderComponent()` and `hydrateComponent()` were updated to accept an explicit `array $slot_definitions` parameter. Previously, implementations read slot definitions directly from the source (which would crash if the source was broken). Now callers pass the slot definitions stored in the `Component` config entity through a new `Component::getSlotDefinitions()` helper, which reads from `fallback_metadata` without touching the potentially-unavailable source. `ComponentTreeItemList` was updated to load slots this way throughout hydration and rendering.

`Component::normalizeForClientSide()` now gates on `isBroken()`: when broken, it catches any rendering exception from `renderComponent()`, falls back to `RenderSafeComponentContainer::handleComponentException()`, and returns `broken: true` in the API response. `ComponentInstanceForm` checks `isBroken()` and, when true, instantiates a `Fallback` source plugin with the message "Component is missing. Fix the component or copy values to a new component." and passes it the best available stored inputs. `ClientServerConversionTrait` (used when saving component trees) was similarly hardened to catch `ComponentNotFoundException` and fall back to `Fallback`.

`JsComponent::getJavaScriptComponent()` was hardened to throw `ComponentNotFoundException` instead of an assertion error when the backing config entity is absent. `BlockManager` was changed from `final` to non-final to allow test-subclassing. Test infrastructure (`BrokenPluginManagerTrait`, `BrokenBlockManager`, `BrokenComponentManager`) and a new `ComponentSourceTestBase::testIsBroken()` abstract test method now enforce that every `ComponentSource` plugin is covered for this scenario. A follow-up issue ([#3532514](https://www.drupal.org/node/3532514)) handles the related case where a component's props change while in development.

## Upgrade

Any custom implementation of `ComponentSourceInterface` must add the new `isBroken()` method and update `renderComponent()` and `hydrateComponent()` to accept the new `$slot_definitions` parameter:

```php
// Before:
public function renderComponent(array $inputs, string $componentUuid, bool $isPreview): array { … }
public function hydrateComponent(array $explicit_input): array { … }

// After:
public function isBroken(): bool {
  // Return TRUE if the underlying component source (block plugin, SDC, etc.)
  // is currently unavailable.
  return FALSE;
}

public function renderComponent(array $inputs, array $slot_definitions, string $componentUuid, bool $isPreview): array { … }
public function hydrateComponent(array $explicit_input, array $slot_definitions): array { … }
```

Callers that previously passed slot definitions from `$source->getSlotDefinitions()` should now pass them from `$component->getSlotDefinitions()` so the stored metadata is used when the live source is unavailable.
