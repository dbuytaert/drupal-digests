## Summary

When a Canvas JavaScript component placed on an entity page declares a `content-entity-reference` prop, site builders previously had only two ways to populate it: pick a specific entity statically, or link to an entity-reference field on the host entity. This change adds a third option: "use the rendered page's own entity as the prop value." The stored source configuration is simply `{"sourceType": "host-entity"}`. When no host entity is available (for example in a pattern preview), the prop is silently skipped rather than causing an error. The change builds directly on the `content-entity-reference` prop shape added in [#3586613](https://www.drupal.org/node/3586613).

## Impact

Affects module maintainers.

- **New API:** New `PropSource::HostEntity = 'host-entity'` enum case in the public `PropSource` enum; `PropSourceSuggester::suggest()` return type gains a `host-entity` bucket alongside `entity-field` and `host-entity-url`.

## Technical details

The new `HostEntityPropSource` class (marked `@internal`) is wired into the `PropSource` enum as `PropSource::HostEntity = 'host-entity'`, extending the existing `parse()` dispatch table. Its `evaluate()` method returns `new EvaluationResult($host_entity, CacheableMetadata::createFromObject($host_entity))` when a host entity is present, and throws `MissingHostEntityException` when `$host_entity` is `NULL`. The existing try/catch in `GeneratedFieldExplicitInputUxComponentSourceBase::getExplicitInput()` already handles this exception, which is what silently skips the prop in host-less contexts.

A new `HostEntityPropSourceMatcher` (`@internal`) performs a two-step gate: the prop's `x-allowed-entity-type-id` must equal the host entity type ID (strict equality), and if `x-allowed-bundle` is present it must equal the host bundle. It returns at most one `HostEntityPropSource`, injected into `PropSourceSuggester` as a new `host-entity` bucket alongside the existing `entity-field` and `host-entity-url` buckets.

The commit also introduces `LinkedPropSourceInterface` (`@internal`), which unifies the three "linked" prop source types (`EntityFieldPropSource`, `HostEntityUrlPropSource`, and the new `HostEntityPropSource`) under a common contract. As part of this, the `label()` method on those two existing `@internal` classes changed signature from `label(): TranslatableMarkup|string` (no parameter) to `label(EntityDataDefinitionInterface $host_entity_data_definition): TranslatableMarkup`. Callers in `LinkedPropSource` (the render element) and `PropSourceSuggester` were updated accordingly. The `HostEntityPropSource::label()` produces context-aware strings such as "This Article content item" (bundled types) or "This user" (bundle-less types).
