## Summary

Canvas adopted PHPStan architecture testing to automatically enforce code organization rules that were previously only caught through manual code review. The system now prevents lower-level components like prop expressions from depending on higher-level Canvas code, ensuring architectural layers remain clean. It also requires every test to declare a PHPUnit group attribute, making it easier to run specific subsets of the test suite. This automated enforcement helps the codebase remain maintainable as it evolves and reduces the chance of architectural violations slipping through during deadline pressure.

## Impact

No upgrade or configuration changes required.

## Technical details

Canvas integrated phpat/phpat, a PHPStan-based architecture testing tool. Two architecture tests were added in `phpstan_rules/Canvas/PHPStan/Architecture/`.

The `PropExpressions` test enforces that the entire `Drupal\canvas\PropExpressions` namespace remains standalone, only depending on itself, specific Drupal core namespaces, and standard PHP classes. This is critical because prop expressions are foundational infrastructure that may eventually move to Drupal core. The test also enforces that all concrete prop expression implementations are `final`.

The `Tests` test requires all test classes extending `TestCase` to declare at least one `#[Group]` attribute. This enables running targeted test subsets.

During implementation, the architecture test discovered three violations: incorrect `FieldConfigInterface` import, a method that needed relocation from `JsonSchemaFieldInstanceMatcher` to the new `FieldItemAnalyzer` helper, and `ContentAwareDependentInterface` living in the wrong namespace. These were fixed to make prop expressions truly standalone.

A new `FieldItemAnalyzer` class was extracted to house field item analysis logic previously scattered in `JsonSchemaFieldInstanceMatcher`. Several test classes missing group attributes were updated.
