## Summary

The Drupal Canvas module's administration navigation now includes a dedicated "Pages" link that goes directly to the Canvas pages listing at `/admin/content/pages`. At the same time, the existing "Content" link in the navigation menu is renamed to "CMS" and its icon is changed to a database icon, to distinguish structured content from Canvas pages. Before this change, there was no discoverable path to the Canvas pages view from the admin navigation.

## Impact

Affects module maintainers.

- **New API:** New `entity.canvas_page.collection` route at `/admin/content/pages` and `navigation.pages` menu link for the Canvas pages listing.

## Technical details

Two changes land together. First, the `canvas_page` entity type annotation gains a `collection` link template pointing to `/admin/content/pages` and a `collection_permission` set to `self::EDIT_PERMISSION`. A new navigation menu link definition (`navigation.pages`) is declared in `canvas.links.navigation.yml`, routing to `entity.canvas_page.collection` with `weight: -5` (placing it before the CMS link) and a `file` icon from the `navigation` pack.

Second, `ModuleHooks` gains a `hook_menu_links_discovered_alter` implementation (ordered after the `navigation` module using `#[Hook(..., order: new OrderAfter(['navigation']))]`). When the `navigation.content` link is present it overwrites the title to `'CMS'` and changes its icon to `database`. `ModuleHooks` is updated to inject `TranslationInterface` and use `StringTranslationTrait` so the title can be translated. A new `NavigationIntegrationTest` kernel test verifies both link definitions and the rename logic.
