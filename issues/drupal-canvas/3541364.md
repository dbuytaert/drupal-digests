## Summary

Experience Builder now allows users to delete empty component folders from the component library. When a folder contains components, the delete option appears disabled with a helpful tooltip explaining that folders with components cannot be deleted. This prevents accidental data loss while giving users control over organizing their component library by removing folders they no longer need.

## Impact

No upgrade or configuration changes required.

## Technical details

The `FolderList` component now integrates the `useDeleteFolderMutation` hook from the `componentAndLayout` service. When a user clicks "Delete folder" from a folder's context menu, the component checks whether the folder contains any components using `hasItems`. If the folder is empty, it calls the `deleteFolder` mutation which sends a DELETE request to `canvas/api/v0/config/folder/{id}`. The mutation invalidates both `Folders` and `Components` cache tags to refresh the UI.

The delete menu item is conditionally disabled based on whether the folder has items. When disabled, a native HTML `title` attribute provides a tooltip explaining why deletion is blocked. The menu only appears when `devMode` is enabled in canvas settings, matching the existing behavior for the rename option.

A new Playwright test verifies both successful deletion of empty folders and proper disabling of the delete option for folders containing components. The test creates a temporary folder, confirms deletion, then checks that a folder with components shows a disabled delete menu item. Error handling follows the same pattern as folder renaming, displaying API errors via `extractErrorMessageFromApiResponse`.

Eight contributors collaborated over six months to deliver this feature. The acceptance criteria evolved through discussion, with the team initially considering disabling components when folders are deleted, then pivoting to a simpler approach of only allowing deletion of empty folders. The UX discussion around tooltip implementation showed thoughtful consideration of native browser behavior versus custom components, with the team choosing to accept the native tooltip delay to avoid additional CSS complexity for a single use case.

## Lessons learned

- When deleting a container entity that references other entities, prevent deletion if the container is not empty to avoid orphaning data. Provide user feedback explaining why the operation is blocked. *(via vipin.mittal18)*
