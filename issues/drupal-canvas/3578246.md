## Summary

The Canvas CLI now includes a `pull` command that downloads all code components and global CSS from the Canvas UI back to the local filesystem. This complements the existing `push` command, enabling a two-way sync workflow. The command automatically places new components in the configured directory and updates existing components in-place while preserving custom files. Use `--skip-overwrite` to pull only new items, and `--yes` for non-interactive operation in CI/CD pipelines.

## Impact

No upgrade or configuration changes required.

## Technical details

The `canvas pull` command uses the existing `listComponents()` and `getGlobalAssetLibrary()` API endpoints to fetch all components and global CSS. It leverages the discovery system to identify which components already exist locally, then either updates them in-place or creates new directories for new components.

The implementation uses a task-based architecture where `createComponentsPullTask()` and `createAssetsPullTask()` handle components and CSS respectively. Each task has a `prepare()` phase that fetches remote data and discovers local state, then an `execute()` phase that writes files.

When updating existing components, the pull command preserves any additional files in the component directory (like helper modules or extra assets) and only overwrites the three core files: `component.yml`, `index.jsx`, and `index.css`. The `--skip-overwrite` flag allows pulling only net-new components without touching existing ones, useful for avoiding conflicts.

The global CSS path is now configurable via `globalCssPath` in `canvas.config.json`. This config resolution logic was centralized in the `@drupal-canvas/vite-compat` package's new `resolveCanvasConfig()` function, replacing the previous `loadCanvasConfig()` implementation and eliminating hardcoded paths.
