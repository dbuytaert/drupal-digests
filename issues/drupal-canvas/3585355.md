# #3585355: `content-entity-reference` prop selection UI: a "Typed Data browser" to let a Code Component Developer pick which data to consume

## Summary

Canvas code component developers can now configure a `content-entity-reference` prop by choosing an entity type and bundle through a "Content Relationship" modal dialog, then selecting individual entity fields via a collapsible checkbox tree. Reference fields expand lazily to show fields from the related entity type. Selections persist to `entityFieldExpressions` on the prop and are auto-saved through the existing auto-save path. Previously there was no editor UI for this prop type.

## Impact

Affects module maintainers.

- **New API:** New `contentEntityReferenceApi` RTK Query service (`useListEntityTypesQuery`, `useListFieldsQuery`) and `ContentRelationshipDialog` component exported from the Canvas UI package for use by custom integrations.

## Technical details

The implementation centers on three new areas. First, `contentEntityReferenceApi` in `ui/src/services/contentEntityReferenceApi.ts` is an RTK Query service registered in `store.ts` with two endpoints: `listEntityTypes` (fetches `ContentEntityReferenceEntityTypes` from `/canvas/api/v0/ui/content-entity-reference`) and `listFields` (fetches `ContentEntityReferenceField[]` from an href supplied by the entity type or bundle definition via the `typed-data-browser` link relation). The recursive `FieldsTree` / `FieldRow` components in `ContentRelationshipDialog` call `listFields` lazily as the user expands reference nodes, so the full field graph is never fetched upfront.

Second, `ContentRelationshipDialog` in `ui/src/features/code-editor/dialogs/ContentRelationshipDialog.tsx` is a two-step modal: step one lets the developer pick entity type and bundle; step two shows the collapsible field tree with checkboxes. The dialog holds draft state locally and only dispatches on Save, keeping cancel side-effect-free. The `useSelectedRelationshipFields` hook tracks checked expressions as a `Map` keyed by expression string, supporting indeterminate state when a field has multiple properties of which only some are selected.

Third, `FormPropTypeContentEntityReference` in `ui/src/features/code-editor/component-data/forms/FormPropTypeContentEntityReference.tsx` wires the dialog into the props panel. It calls `setEntityFieldExpressions` (a new reducer case in `codeEditorSlice`) on save, which writes to `prop.entityFieldExpressions: string[]`. The existing `useAutoSave` hook then fires the existing PATCH mutation. On load, `useResolvedExpressionFields` in `ui/src/features/code-editor/utils/contentEntityReferenceLabelResolver.ts` walks the `listFields` API tree recursively (up to depth 8) to translate raw expression strings back into human-readable labels so the field list in the panel renders correctly after a page reload.
